## 2. 불일치 / codex 신규 후보 → 조사 큐 (게이트 라우팅)

### Q-A ★[돈크리티컬·진성 확정후보] 표지 용지비 누락 (codex NEW-A·Top3 #1)
- **codex 주장**: 분담안에 표지 **용지비** 귀속처가 없다(저청구). 권위 seq63=6비목(…+용지비+…)·이전사이트 표지 용지 슬롯 보유. 071 표지×2라 영향 더 큼.
- **Claude 설계 현황**: D6 단일귀속표 = 부모공식(표지인쇄+표지코팅+제본) + 내지구성원(내지인쇄+**내지**용지). line90 "용지비=내지용지"로 해석. → 표지 용지비 항목 부재.
- **★reconcile 1차 교차확증(라이브 소스)**: 072 선례(`03_design/hardcover-book-design.md` L203·L208)는 표지(073)가 **PRF_DGP_A로 인쇄비+전용지 용지비+코팅비**를 자기 구성원공식으로 가져 6비목 정확 분담. 068~071 설계는 표지를 "가격0"으로 두면서 표지용지를 부모공식에도 안 넣음 → **072 선례와 불일치 = 진성 결함 후보 강함**.
- **판정(가설)**: 표지 용지비 누락 → 표지 종이값만큼 저청구(특히 071 ×2). 설계의 "표지 가격0" 전제와 6비목 권위가 충돌.
- **라우팅**: NO-GO 후보 → set-designer 보정. 선택지 (a) 표지 구성원에 PRF_DGP_INNER/DGP_A형 공식 부여(인쇄0이면 용지만)해 072처럼 표지용지를 구성원에 / (b) 부모공식에 표지용지 comp 4번째 추가. 어느 쪽이든 표지용지 1비목 단일귀속처 명시 필수. 게이트 G-068/071 골든에 표지용지 비목 검출 케이스 추가.

### Q-B [조사신호 P-1·합의] 070 PUR A5 내지 단면 vs 이전사이트 양면 (codex 4a·Top3 #2)
- Claude(P-1 중)·codex(조사필요) **합의**: 권위 booklet-l1 A5=단면이면 설계가 맞음·이전사이트 양면만은 강한 조사신호. 돈영향 ~2배(내지인쇄 단/양면).
- **라우팅**: 게이트 G-070 골든에서 A5 면 매칭(권위 단면 vs 라이브 양면) 양쪽 계산·권위 booklet-l1 PUR 행 재확인(행 어긋남 가능성). 자동교정 금지·권위 절대.

### Q-C [조사신호 P-2·부분합의] 071 표지×2 (codex 4b)
- Claude(P-2 중·"자동 ×2 보류"·게이트 골든 표지1 vs ×2 양쪽)·codex(GO조건부·"오히려 용지비 ×2 저청구 위험") — **방향 다름**: Claude는 과대청구 경계(×2 보류), codex는 ×2는 정당하되 표지용지까지 ×2 안 하면 저청구 경계.
- **reconcile**: 둘 다 게이트 골든에서 표지비(인쇄+코팅+용지) ×1 vs ×2 양쪽 계산해 권위 가격표(seq72)와 대조로 확정하면 양 신호 동시 해소. "표지없음" 선택 시 부모 표지비 전체 0 처리(codex 추가 지적)도 게이트 케이스 추가.
- **라우팅**: 게이트 G-071. Q-A(표지용지) 보정과 함께 처리.

### Q-D ★[codex 신규·1차 반증] 셋트행 min/max/incr 컬럼 의미 오용 (codex NEW-C·Top3 #3)
- **codex 주장(가설)**: t_prd_product_sets min/max/incr가 페이지축이 아니라 구성수량 제약이면 페이지범위 주입은 모델 오용.
- **★reconcile 1차 교차반증(라이브 소스)**: `models.py:469-471` db_comment="구성최소/최대개수·개수증가단위"(컬럼 자체는 구성수량축 맞음). 그러나 `price_views.py:1674`가 셋트 구성원의 `min_cnt/max_cnt/cnt_incr`를 그대로 내지 멤버의 **페이지 입력 가변범위 UI**로 노출하고 `dflt_qty=sub_prd_qty`로 씀. 실 페이지파생은 `pages`(별도 호출인자)→`derive_inner_sheets`(L1889). → 라이브가 이 컬럼을 내지 페이지 UI 범위로 재활용하는 게 **072 선례·라이브 정합**이며 모델 오용 아님.
- **판정**: codex NEW-C = **false-positive(반증)**. 단 codex가 "pages는 셋트행이 아니라 호출인자"를 정확히 짚은 부분은 설계 D7과 일치(보강 가치).
- **라우팅**: 게이트 통과 후보(이월). 추가 조사 불요.

### Q-E [codex 신규·조사] 부모 표지인쇄 S1 단일 한계 (codex NEW-B)
- **codex 주장**: 부모공식 표지인쇄가 COMP_PRINT_DIGITAL_S1(단면)만이면 표지 양면(068 A4·071) 단가 못 담음.
- **reconcile**: 설계 §3.2가 "표지인쇄=양면 시 S2 별행"·CFM-S2-REVIVE(S2 부활)를 이미 BLOCKED로 둠 → 부분 인지. 단 부모공식 comp 배선에 S2 분기를 명시 안 함(use_dims 도수 자연택일 가정). 게이트에서 표지 양면 골든으로 S2 매칭 검출.
- **라우팅**: CFM-S2-REVIVE에 표지측 S2도 포함 명시(현재 내지 위주 기술). 게이트 G-068(A4 표지양면)·G-071 골든.

### Q-F [codex 신규·조사] 옵션 비목성 (codex NEW-D)
- **codex 주장**: 071 투명커버·링컬러·069/070 개별포장이 무료인지 비용 비목인지 미확정. 투명커버는 물성상 비용 가능성→누락 위험.
- **Claude 설계 현황**: 071 투명커버=CONFIRM-071-TRANSPARENT(SEMI_ROLE.05·본 스코프 미포함·후속). 링컬러/개별포장=옵션 처리(가격 미명시).
- **reconcile**: codex가 옳게 지적 — 투명커버가 별도 비용이면 미포함 = 저청구. 단 사용자 directive로 투명커버는 본 스코프 밖(별 옵션축).
- **라우팅**: 후속 CONFIRM(투명커버 가격 비목성 권위 재확인). 본 스코프 GO에는 영향 없으나 "071 완전 견적"에는 잔여. 게이트 비스코프 표기.
