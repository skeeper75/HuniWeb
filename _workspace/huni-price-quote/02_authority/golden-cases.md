# Golden Cases — 시뮬레이터 검증용 대표 케이스 + 기대 final_price

> **Phase 1 — hpq-authority-curator** · 2026-06-18 · `huni-price-quote`
> 각 케이스 = 대표 선택값 + 수량 → **기대 final_price**(엑셀 명시값으로 유도). Phase 3 `hpq-quote-gate-validator`가 라이브 `evaluate_price` 재계산값과 대조하는 수치 기준.
> **유도 추적성:** 각 단가에 셀 좌표를 박았다. 합산형은 원자별 소계를 모두 표기(검증자가 어디서 틀렸는지 역추적 가능).
> **추정 0:** 산정식이 권위에서 닫히지 않는 케이스(예 아크릴 개당/총액 엔진계약)는 **🟡 가설** 표기 + `authority-gaps.md` 컨펌ID 링크.

## 케이스 1 — 엽서 (합산형, 200매)

### 선택값
- 상품: 프리미엄엽서 (마스터 `디지털인쇄`:행3, ID 14529)
- 사이즈: **100x150 mm** → 판걸이수 **8** [`판걸이수`:F4]
- 인쇄: 칼라(CMYK) **단면**
- 종이: 백색모조지 220g → 국4절 단가 **70.64** [`출력소재(IMPORT)`:I6]
- 코팅: 없음
- 후가공: 없음
- 수량: **200매**

### 유도 (verbatim 단가 × 권위 산정식)

```
출력매수(판수) = ceil(200 / 8) = 25                              [공식집:행5]
인쇄비 단가(칼라단면, 출력매수25행) = 800                         [디지털인쇄비:D16]  (수량행 25.0)
인쇄비 = 800 × 25 = 20,000                                       [공식집:행6]
용지비 = 70.64 × (25 + 5) = 70.64 × 30 = 2,119.2                 [공식집:행9]
코팅비 = 0
후가공비 = 0
----------------------------------------------------------------
기대 final_price = 20,000 + 2,119.2 + 0 + 0 = 22,119.2
```

> **🟡 반올림 컨펌(Q-ROUND):** 용지비 2,119.2의 원/소수 반올림 규칙이 권위에 미명시. final_price = **22,119.2** (반올림 전 verbatim). 라이브 반올림 정책 확인 필요 → `authority-gaps.md` Q-ROUND. 검증자는 인쇄비 20,000(정수·확정)을 1차 앵커로 사용.

### 확정 앵커(반올림 무관)
- **인쇄비 = 20,000** (800 [`디지털인쇄비`:D16] × 출력매수25). 이 값이 틀리면 판수 계산/단가행 룩업 결함.

## 케이스 1b — 엽서 (코팅 포함, 검증 보강)

### 선택값
- 케이스1 동일 + **무광 단면 코팅** 추가

### 유도
```
코팅비 단가(무광단면, 출력매수25행) = 800                        [코팅:B8]  (수량행 20.0 구간; 25 → 20행 룩업, 다음 행 30도 800)
코팅비 = 800 × 25 = 20,000                                       [공식집:행8]
기대 final_price = 20,000(인쇄) + 2,119.2(용지) + 20,000(코팅) = 42,119.2
```

> **🟡 코팅 수량행 룩업(Q-COAT-TIER):** `코팅` 시트 행은 1/2/5/10/20/30… 25는 명시행 없음 → 구간 룩업(20행=800 또는 30행=800, 둘 다 800이라 본 케이스는 무영향). 검증자는 코팅비 소계 20,000을 앵커로.

## 케이스 2 — 실사 (면적매트릭스, 60×90cm 1개)

### 선택값
- 상품: 아트프린트포스터(인화지) (`포스터/사인`:행1, 코팅포함가)
- 가로(width) = **600mm** (=60cm) → 열헤더 600 [`포스터/사인`:C2]
- 세로(height) = **900mm** (=90cm) → off-grid (행헤더 800/1000 사이) → **ceiling 1000mm 행** [`포스터/사인`:행5]
- 수량: 1개

### 유도
```
세로 900 → 임계 {600,800,1000,…} 중 ≤ 만족 최소 = 1000 행      [공식집:행101 / off-grid ceiling]
가로 600 → 열 600
매트릭스[세로1000행][가로600열] = 20,000                         [포스터/사인:C5]
기대 final_price = 20,000
```

> **off-grid 입증 케이스:** 세로 900은 매트릭스에 없음 → 다음 큰 구간 1000 ceiling. 검증자가 엔진 ceiling 동작을 확인하는 핵심 케이스.

### 케이스 2b — 실사 grid-exact (off-grid 무관 앵커)
- 아트프린트 가로1000×세로1000 = **20,000** [`포스터/사인`:E5] (헤더 정확 일치).
- 아트프린트 가로1200×세로1800 = **43,200** [`포스터/사인`:F9] (큰 면적 검증).
- 접착투명 가로1000×세로1000 = **55,000** [`포스터/사인`:E73] (소재축 분기 검증).

## 케이스 3 — 아크릴 (면적매트릭스+두께, 30×30 3T 1개)

### 선택값
- 상품: 투명아크릴3T (`아크릴`:행1)
- 가로 = **30mm** → 열헤더 30 [`아크릴`:C2]
- 세로 = **30mm** → 행헤더 30 [`아크릴`:행4]
- 수량: **1개** → 수량구간 1~49 → 할인율 **0.0** [`아크릴`:B51]
- 후가공: 없음

### 유도
```
인쇄가공비 = 매트릭스[세로30행][가로30열] = 3,100                [아크릴:C4]
수량할인(1개) = 1~49 = 0.0 → (1 - 0.0) = 1.0                    [아크릴:B51]
후가공 = 0
기대 final_price = 3,100 × 1 × (1-0.0) = 3,100
```

### 케이스 3b — 아크릴 1.5T 두께축
- 투명1.5T 30×30 = **2,480** [`아크릴`:C24] (=3,100×0.8 두께축 직교 검증).

### 케이스 3c — 아크릴 수량할인 (100개)
```
인쇄가공비(개당) = 3,100  [아크릴:C4]
수량 100 → 구간 100~299 → 할인율 0.2  [아크릴:B53]
기대 = 3,100 × 100 × (1-0.2) = 248,000
```
> **🟡 개당/총액 엔진계약(Q-ACR-ENGINE):** [공식집:행107] "인쇄가공비 × 수량별구간할인". 인쇄가공비가 **개당×수량 후 할인**(=248,000)인지, 라이브 prc_typ .02가 총액형인지 미확정. **가설 = 개당×수량×(1−할인율)** → 248,000. 컨펌 → `authority-gaps.md` Q-ACR-ENGINE. 검증자는 단일수량(케이스3 = 3,100)을 1차 앵커로.

### 케이스 3d — 아크릴 후가공 가산 (30×30 3T + 은색고리, 1개)
```
인쇄가공비 = 3,100  [아크릴:C4]
할인(1개) = 0.0
후가공(은색고리) = 1,100  [아크릴:C63]  (제작수량당 1개)
기대 final_price = 3,100 × 1 + 1,100 × 1 = 4,200
```
