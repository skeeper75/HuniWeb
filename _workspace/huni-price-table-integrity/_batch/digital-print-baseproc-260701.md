# 디지털인쇄 base 공정(PROC_000004) 미바인딩 → 인쇄비 영구 미청구 진단·교정 명세

## 결함 (가격구성요소 중심 · 돈 크리티컬 · 저청구)

가격구성요소 **`COMP_PRINT_DIGITAL_S1/S2`(디지털인쇄비)** 의 전 단가행은 `proc_cd=PROC_000004`(디지털인쇄)로 키잉돼 있다. 엔진 `_row_matches`는 단가행의 채워진 `proc_cd`를 손님 선택과 정확매칭하므로, **손님이 PROC_000004를 선택해야** 인쇄비가 합산된다. 그런데 16개 디지털인쇄 상품의 `t_prd_product_processes`에 **PROC_000004가 미바인딩** → 위젯 공정 드롭다운(=`TPrdProductProcesses`)에 안 뜸 → 손님이 선택할 수단 없음 → **디지털인쇄비가 영영 0원**(인쇄=핵심 비목이라 저청구 규모 큼).

## 근거 사슬 (5중)

1. **공식 설계**: 영향 상품의 공식(PRF_DGP_A/B/C/D/E)이 `COMP_PRINT_DIGITAL_S1`을 배선 — 인쇄비는 가격모델의 핵심 비목.
2. **기준점 016 프리미엄엽서**: 유일하게 `PROC_000004`를 `mand_proc_yn='Y', disp_seq=-1`로 보유 → 인쇄비 정상 청구(검증된 작동 상품).
3. **채점 하네스 자인**: `score_batch.py:291` 주석 *"접지카드 등=인쇄공정이 proc 옵션에 안 보임(후가공만) → base_print_proc 주입(거짓 0 방지)"* — 골든 채점은 인쇄 proc를 **코드로 주입**해 통과시켰으나, 실제 위젯은 product_processes를 읽으므로 재현 불가.
4. **위젯 자동적용**: `price_simulator.html:325` `PROCSEL=PROC_OPTS.filter(o=>o.mand)` — 필수공정은 기본 선택·제거불가. 즉 PROC_000004를 mand로 추가하면 인쇄비 자동 청구(016과 동일 동작).
5. **엔진 실증(시뮬레이터, qty 800)**: PROC_000004 주입 전/후 인쇄비 차이 — 027: +71,200 · 028: +51,000 · 047/026/284: +62,000 · 엽서류: +27,000~33,500.
