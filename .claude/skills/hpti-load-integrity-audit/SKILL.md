# 가격테이블 적재 무결성 진단 방법론

## 왜 이 진단이 선행돼야 하는가
가격공식·구성요소(차원 포함)를 아무리 잘 설계해도, **권위 엑셀의 가격 데이터·차원이 라이브 DB에 이 빠짐 없이 적재되지 않았으면** 정확한 값이 안 나온다. 예: 면적 격자에 70×30 셀이 비어 있으면 그 사이즈는 견적 0/최소가; 등록사이즈(siz_cd) 차원이 라이브 구성요소에 없으면 손님이 사이즈를 못 고르고 엔진이 최소가만 계산. 따라서 **설계 이전에 "권위→라이브 적재가 완전·정확한가"를 시트별로 확신**해야 한다. 이 진단의 GO가 §18 설계·§13/15 검증의 신뢰 기반이다.

## 원칙 [HARD]
- **권위 = 두 엑셀(절대 진실). 라이브 = 감사 대상.** 불일치 시 라이브가 틀린 것(역방향 금지).
- **차원 단위·셀 단위 정밀.** "상품이 존재한다"가 아니라 "이 가격격자의 이 셀/이 차원이 적재됐나"를 본다(§21 종단 정합과 다른 결).
- **라이브 읽기전용 SELECT만**(`.env.local RAILWAY_DB_*`). 실 적재/교정은 인간 승인 후 dbmap 위임.
- **기존 추출 재사용**(06_extract·24_master-extract). 같은 엑셀 재파싱 금지.
- **생성≠검증**(inspector/codex 생성 → gate 독립 재실측). 날조 0·근거(시트:셀·테이블:행·SQL).

## 1. 차원 분류 (가격축 식별 — 적재방식이 갈린다)
각 시트 가격테이블이 무엇으로 가격을 매기는지 먼저 확정한다. 적재 방식이 차원 유형에 종속:

| 가격축 | 라이브 표현 | 적재 방식 | 이 빠짐 위험 |
|---|---|---|---|
| 등록사이즈 | `use_dims=[siz_cd]` + 사이즈별 단가행 | 정확매칭 | 사이즈별 행 누락 |
| 면적(자유치수) | `use_dims=[siz_width,siz_height]` + 격자행 | 구간 ceiling 올림 | **격자 빈 셀(sparse)**·max 초과 |
| 자재 | `use_dims=[mat_cd]` + 자재별 행 | 정확매칭 | 자재 collapse(대표 1개만)·mat 코드 불일치 |
| 도수/인쇄옵션 | `print_opt_cd`/`clr_cd` | 정확매칭 | 면 미분리·이중합산 |
| 수량구간 | `min_qty` 티어 | 이상 하한 | 구간 누락 |
| 공정 | `proc_cd`+`dim_vals` | 정확매칭 | 상세파라미터 누락 |
★등록사이즈 상품을 면적 구성요소에 적재하면 siz_cd 미환원→최소가(코드버그). 차원 유형↔구성요소 일치 필수.

## 2. 정답 격자 추출 (extractor)
시트별로 권위 엑셀에서 **차원열 + 전 데이터셀**을 정규 격자 CSV로 펼친다. 면적이면 전 (가로,세로) 조합, 자재축이면 전 자재, 수량이면 전 구간. "*가격표참고"는 어느 표인지 해소. 차원 의미·기대 적재방식을 `-dims.md`에 명시.

## 3. 적재 대조 (inspector) — 3종 결함
정답 격자 각 셀/차원을 라이브와 대조:
1. **미적재(이 빠진 셀)**: 정답 셀이 라이브 단가행/차원행에 없음. 면적이면 어느 (w,h) 셀이 비었는지, 자재면 어느 mat 행이 없는지 특정. → 돈영향=견적0/최소가/특정자재 견적불가.
2. **차원 누락**: 권위 가격축이 라이브 `use_dims`·`t_prd_product_*` 차원에 없음. → 손님 선택 불가·엔진 미환원.
3. **정합 불일치**: 적재값≠권위·mat_cd가 등록자재와 코드 불일치·표시명↔실제치수 불일치(오적재).
각 결함: {유형·차원조합·정답값·라이브값·돈영향(저/과/견적불가)·재현SQL·라우팅}.

**false-positive 가드:** 의미축 정당 차이(원형/사각 동일가·작업/재단 구분·상품전용 사이즈)는 결함 아님.

## 4. codex 교차 (codex-verifier)
`hqv-codex-cross-verify` 재사용. 독립 2nd opinion으로 놓친 gap + 과적발 적발. codex 주장=가설(확증 전 사실 아님)·미가용 시 Claude 단독 명시.

## 5. 게이트 (integrity-gate) — I1~I7
I1 격자 완전성·I2 미적재 실재·I3 차원누락 실재·I4 불일치 실재(의미축 가드)·I5 돈영향 정확(evaluate_price 재계산)·I6 codex 수렴·I7 생성검증 독립성. 단일 FAIL=NO-GO. GO 시트=설계 신뢰 기반. 결함=교정 명세(대상 t_*·정답값·dbmap 트랙: dbm-price-import-prep 그릇·dbm-correctness-audit 교정·dbm-load-execution 적재)→인간 승인.
