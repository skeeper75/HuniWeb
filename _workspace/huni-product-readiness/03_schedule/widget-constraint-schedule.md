## 0. 핵심 게이트 [HARD]: 위젯은 L3(계산 가능) 토대 위에서만

위젯은 손님 선택 → **가격이 계산되는** 토대 위에서만 의미가 있다(D5 PASS=evaluate_price PRICE≠0 AND 골든 일치).
- **위젯 착수 가능 = L3 + L4 = 83상품** (이 문서의 wave 대상)
- **위젯 선행 불가 = L3 미만 = 200상품** → `blocked-before-widget.md`로 분리(가격계산부터 채워야 함)
- **돈크리티컬 분리**: PRICED-0 6건·silent 합산·×qty 과대청구 의심은 **위젯 wave에 절대 포함 금지**(가격이 틀린 채 위젯 얹으면 손님에게 틀린 값 노출). 전부 L2 이하로 떨어져 자동 차단됨.

### 위젯 착수 가능 83상품의 클래스 분포
| 위젯클래스 | 상품수 | 상품군 | 대표 |
|-----------|--------|--------|------|
| W-AREA 면적입력 | 38 | 실사 26 · 아크릴 12 | 미니보드스탠딩·아크릴마그넷 |
| W-CASCADE 옵션캐스케이드 | 27 | 디지털인쇄 19 · 문구 8 | 스탠다드쿠폰·만년다이어리 |

## 1. 위젯 wave (등급×클래스) — 의존 순서

순서 원칙 = **난이도 낮은 단순 모델 → 복합 모델**, 동형 높은 클래스는 **대표 1개 완주 → 나머지 자동 전파**.

### W0 — 위젯 코어·BFF 배선 (전 wave 선행·상품무관) · 규모 M
- **범위**: `evaluate_price`를 읽기전용 BFF로 API화(§28 `POST /price/evaluate`) + `selections` 데이터 계약 + Shadow DOM 위젯 shell. §6 huni-widget이 이미 4차원×4모델 동등성 GO([[huni-widget-equivalence-gate-go]]) → **재구현 아님, 라이브 배선만**.
- **선행 결정**: 가격 권위=서버(위젯은 표시만, 위젯 단 재계산 금지). PRICE=0=결함 신호([[huni-widget-red-price-never-zero]]).
- **이후 모든 wave가 이 위에 얹힘.**

### W1 — W-FIX 스티커 (L3·15) · 규모 M
- **모델**: siz_cd별 고정단가 → 사이즈 드롭다운→고정가(최저 난도).
- **대표**: 낱장 자유형 스티커(PRD_000055) 완주 → 반칼/투명/홀로/팬시/정사각/직사각 등 14 동형 전파.
- **선행**: W0 + siz_cd 차원 적재(이미 PASS). **제약 C1 병행**(반칼+코팅 동시불가 등·현재 D7 WARN·OC 50~60).

### W2 — W-CASCADE 디지털인쇄 (L3·19) · 규모 L
- **모델**: 원자합산(sizes→dosu→paper→weight→process) 단계 의존 드롭다운 캐스케이드.
- **대표**: 스탠다드 쿠폰/상품권(PRD_000041, OC=100) → 쿠폰/전단/리플렛/접지카드 등 전파.
- **선행**: W0 + 캐스케이드 차원 충전 확인. §28 B1~B7 옵션위젯 축과 1:1.

### W3 — W-AREA + W-ADDON 아크릴 (L3/L4·12) · 규모 M
- **모델**: 본체=면적격자(siz_cd×면적) **+** addon=고리/볼체인 등 → **2클래스 합산**.
- **대표**: 아크릴마그넷(PRD_000147·L4·100%) → 키링/스탠드/코롯토 전파.
- **선행 [HARD]**: addon은 반드시 **product_addons/templates 모델**로. opt_cd 가산형이면 라이브 미작동([[addon-optcd-model-broken-live]]) → 위젯 클래스 W-ADDON 강제.

### W4 — W-AREA 실사 (L3·26) · 규모 L
- **모델**: 가로×세로 면적 입력(off-grid ceiling) → 포스터사인 매트릭스 환원([[dbmap-silsa-price-via-poster-sign]]).
- **대표**: 미니보드스탠딩(PRD_000144·96.7%) → 포스터/현수막/배너/패브릭/폼보드 25 전파(동형 매우 높음).
- **선행**: W0 + 면적 매트릭스 적재(이미 PASS). **OC 50~67 다수 → 제약 C2(면적 min/max) 필수 병행.**

### W5 — W-CASCADE 문구 (L4·8) · 규모 M
- **모델**: 다이어리/노트 캐스케이드. 일부 page/구성원 성격.
- **대표**: 만년다이어리(소프트커버)(PRD_000172·L4·100%).
- **선행**: W0 + DESIGNED_NOT_LOADED 일부 단가행 적재 확인.

### W6 — W-SET 책자 (L3·3) · 규모 M
- **모델**: t_prd_product_sets 구성원 합산 + 셋트공식 + page-step(2P 증분).
- **대표**: 떡메모지(PRD_000097·완제품가) → 메모지류.
- **선행**: W0 + 셋트 구성원 적재 + **제약 C3(page min/max/incr)**. ★본격 책자 셋트(중철/무선/PUR/하드커버 068~082)는 현재 L2+/L2(공식 미바인딩·골든 미달)=**위젯 선행 불가**(blocked 문서 참조).

## 2. 제약조건 트랙 (C) — 위젯과 병행하는 별도 줄

D9 제약(JSONLogic·동시불가·min/max)은 위젯 UX와 함께 가지만 **데이터 정리가 선행**. webadmin devDep에 `json-logic-js` 이미 존재(재사용). 착수 가능 83상품 중 D9=WARN 46·N/A 37 → 대부분 **제약 데이터 미완**.

| 트랙 | 범위 | 종속 wave | 규모 | 선행 |
|------|------|-----------|------|------|
| **C1 동시불가** | 스티커/명함류 반칼+코팅 등 양립 불가 조합 | W1·W2 | M | D7 option_groups 적재 선행 |
| **C2 면적 min/max** | 아크릴/실사/현수막 최소·최대 면적 가드 | W3·W4 | M | D9 면적축 + 면적 입력 UI |
| **C3 셋트 min/max/incr** | 책자/포토북 page 2P증분·구성원 수량 한계 | W6 | M | §23 셋트 구성과 연계 |

- **선행 데이터 정리**: 제약은 권위 엑셀의 옵션성 속성(사이즈→추가상품/박 min·max)에서 도출 → 라이브 t_prd_product_constraints에 적재(미적재가 D9 WARN의 원인).
- **옵션 적재(D7)가 제약(D9)의 선결**: 손님 선택 엔티티(option_groups/items)가 있어야 그 위에 제약을 건다. 현재 D7 FAIL 13·WARN 52 → C트랙은 사실상 "옵션 적재 + 제약 적재" 2단.

## 3. 전체 의존 그래프 (요약)

```
W0(코어·BFF) ──┬─ W1 스티커 ──── C1 동시불가
               ├─ W2 디지털 ──── C1
               ├─ W3 아크릴(+addon)
               ├─ W4 실사 ─────── C2 min/max
               ├─ W5 문구
               └─ W6 책자셋트 ─── C3 page min/max/incr
```

- W1~W6은 W0 후 **병렬 가능**(상품군 독립). 난이도/전파효율상 권장 착수 순서 = W1→W2→W3→W4→W5→W6.
- C1~C3은 각 wave에 **종속 병행**(위젯 UI 없이 제약만 적재해도 무의미·반대로 제약 없이 위젯 내면 손님이 불가 조합 주문).

## 4. 한계
- wave 대상 83상품 전부 D5 PASS이나 **D7/D9는 미완 다수** → 위젯이 "보이긴 해도 주문 완결 불가"인 구간이 존재(C트랙 없이는 L4 미도달).
- 위젯 클래스는 **현재 지배 모델** 기준. 미바인딩(TBD) 상품은 클래스 미정 → blocked 문서.
- 실제 구현 난이도/규모(S/M/L)는 §6 huni-widget 구현팀의 컴포넌트 재사용도에 따라 변동 가능(여기선 가격모델 복잡도 기준 상대 견적).
