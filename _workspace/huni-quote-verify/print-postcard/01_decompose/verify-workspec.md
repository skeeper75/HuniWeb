## 축1 — SOT 일치 (상품마스터 행 ↔ 가격표 구역)

검사: 상품마스터 디지털인쇄 시트 엽서 행이 허용하는 차원 ↔ 가격표 시트 구역이 정합하는가.

| # | 상품마스터 행(권위) | 가격표 구역(권위) | 검사 |
|---|------|------|------|
| A1-1 | 디지털인쇄 엽서 사이즈 7종 (73x98~A5) | 판걸이수 시트 엽서 행(F2~F8) | 사이즈↔판걸이수 1:1 매핑·누락 0 |
| A1-2 | 종이(필수) 열 종이목록 | 출력소재(IMPORT) 디지털인쇄용지 단가 | 종이별 국4절 단가 존재 (220g=70.64 ✅) |
| A1-3 | 인쇄(옵션) = 도수×면 | 디지털인쇄비 2D 매트릭스(흑백/칼라/별색 × 단면/양면) | **★도수축 누락 여부** (D-COLOR) |
| A1-4 | 코팅·후가공 옵션 | 코팅·인쇄후가공 시트 | 코팅 무광/유광·단/양면 매트릭스 정합 |
| A1-5 | 계산공식집초안 r3 엽서 group | PRF_DGP_A note(엽서·상품권·슬로건) | 상품 group ↔ 공식 scope 일치 ✅ |

> SOT 1 권위 = 상품마스터 11시트 = 상품군 차원 경계. 엽서 시트에 없는 차원은 엽서 차원이 아니다.

## 축2 — 공식↔구성요소 매핑 정합 (배선 목록 + 차원경계 + 유효성)

검사: PRF_DGP_A에 배선된 18 comp가 권위 가격축대로·엽서 차원경계 내에서 배선됐는가.

### A2-1 배선 정합 (formula_components 18행)
| comp_cd | 권위 산식 항목 | 정당? | 검사 포인트 |
|---------|------|:--:|------|
| COMP_PRINT_DIGITAL_S1/S2 | (2) 인쇄비 | ✅ | S1=plt_siz(국4절)·S2=siz(작업) 판형축 분리. **양키 이중 위험(V-5/D-5)** |
| COMP_PRINT_SPOT_WHITE_S1 | (3) 별색인쇄비 | ✅(엽서 정당) | 별색=국4절 디지털 독립 comp |
| COMP_COAT_MATTE | (4) 코팅비 | ✅ | 92행 |
| **COMP_COAT_GLOSSY** | (4) 코팅비 | ⚠️ | **단가행 0 = 고아(V-4)** |
| COMP_PAPER | (5) 용지비 | ✅ | siz·mat_cd·56행 |
| COMP_PP_CREASE_1/2/3L·PERF·VARTEXT·VARIMG·CORNER | (6) 후가공비 | ✅(엽서 정당) | **줄수/개수 이중매칭 위험**(1L+2L 동시매칭=D-1/2) |

> ★엽서는 오시·가변·별색 comp의 **정당 귀속 상품군** (이 comp들이 엽서 후가공이 맞음). 위반(D-1/2/3)은 동일 comp가 포스터 28공식에 복사된 것 → 엽서 검증 시 이 배선은 정당으로 본다. (인용: `huni-price-engine-diag/04_binding_validity/comp-product-validity-matrix.md`)

### A2-2 시트 차원경계 위반 후보 (U-7 binding-validity)
- 엽서 자기 후가공(오시/미싱/귀돌이/가변/별색) = 엽서 시트에 있음 → 위반 아님.
- **검사**: PRF_DGP_A에 엽서 시트 밖 차원의 comp가 silent 합산되는가 (addtn_yn=Y 무차별 합산). 라이브 재현쿼리로 비선택 comp의 단가행 매칭 여부 확인.

### A2-3 불필요분 (판별차원 없음·고아·중복)
- COMP_COAT_GLOSSY = 배선됐으나 단가행 0 → 0원 침묵 (제거 후보).
- 줄수/개수 dim_vals 이중매칭(CREASE_1L 줄수2행 + CREASE_2L 동시) → 라이브 dim_vals 분포 재실측.

## 축3 — 차원 매칭 (use_dims ↔ 가격테이블 차원 ↔ component_prices 충전 차원)

검사: 각 comp의 use_dims 선언 ↔ 단가행 충전 차원 ↔ 권위 가격축 3원 정합.

### ★ A3-1 (1순위·돈크리티컬) — 디지털인쇄비 도수축 드롭 (D-COLOR)
| 3원 | 내용 |
|------|------|
| 권위 가격축(디지털인쇄비) | **도수(흑백1도/칼라CMYK/별색4종) × 면(단/양)** 2D |
| use_dims(라이브 S1) | proc·plt_siz·**print_opt(면)**·min_qty — **도수 차원 없음** |
| 단가행 충전(라이브 S1@SIZ_000499) | min_qty=1단면=3000·15=450·25=350·200=130 = **전부 흑백(1도) 열** |
| **불일치** | 칼라(D: 4000/900/800/310)·별색 열 **미적재**, 도수 선택 무반영 |

**재현 SQL (검증가 실행):**
```sql
-- 라이브 칼라/흑백 구분 행 존재 여부 (없으면 도수축 드롭 확정)
SELECT print_opt_cd, min_qty, count(*) n, string_agg(DISTINCT unit_price::text,',') prices
FROM t_prc_component_prices
WHERE comp_cd='COMP_PRINT_DIGITAL_S1' AND plt_siz_cd='SIZ_000499' AND min_qty IN (1,15,25)
GROUP BY print_opt_cd, min_qty ORDER BY 1,2;
-- 기대(결함): min_qty당 print_opt별 1행씩만 = 도수 미구분
-- 권위 정답: 도수(흑백/칼라/별색)별 별도 행/차원 있어야 함
```
**판정**: 라이브 evaluate_price에 "칼라 단면" selection 넣고 200매 → 8,750(흑백) 산출하면 D-COLOR 확정. (골든 PC-2/PC-4 쌍 대조)

### A3-2 사이즈 차원 정합 (siz_cd 이산 vs plt_siz vs 작업사이즈)
- 엽서 사이즈 = **이산 siz_cd 7종** (siz_width/height 구간축 아님 — 면적매트릭스 상품 아님).
- 검사: S1 plt_siz_cd(국4절 SIZ_000499) ↔ S2 siz_cd(작업사이즈) 혼동 없는가. 같은 의미 사이즈 중복 등록 없는가.

### A3-3 용지비 차원 정합 (✅ 기 확인)
- use_dims=[siz·mat_cd]. 백색모조지220g 국4절=70.64 라이브=권위 일치 ✅.
- 검사: siz_cd가 작업사이즈인가 국4절 판형인가 (용지비 산식 = 크기별 기준단가 — 국4절 단가 사용).

### A3-4 코팅 차원 정합
- use_dims=[siz·coat_side_cnt·min_qty]. 무광 92행 ✅ / 유광 0행 ❌.

## 검증가 실행 체크리스트

1. **라이브 evaluate_price 재계산** (PC-2 칼라·PC-4 흑백·PC-3 코팅) → 골든 대조 (P2: 인쇄비 20,000 / 8,750 앵커).
2. **A3-1 재현 SQL** 실행 → 도수축 드롭 확정/반증 (D-COLOR).
3. **A2-2 silent 합산** 재현 → 비선택 comp 매칭 여부.
4. **V-4 유광코팅 0행**·**D-BIND 투명엽서 바인딩 0** 재실측 (생성자 주장 비신뢰·직접 psql).
5. **시뮬레이터/가격뷰어** 표시값 ↔ 재계산값 ↔ 골든 3원 대조.

> 라이브 읽기전용 SELECT만·DB 미적재. 권위 엑셀 절대(v03/STALE 금지). dodge-hunt: 생성자(본 spec) 주장도 라이브 직접 재실측으로 검증.
