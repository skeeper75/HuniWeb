## ① 요약

1차 추출은 속성별 단일 컬럼만 평면화(사이즈=E만)하여 같은 행의 작업사이즈 컬럼을 버렸고, 실사 포맥스보드 A1 false-MISSING을 낳았다. **정확한 신호는 블리드(H)가 아니라 작업사이즈 컬럼 공백이며, 그 컬럼 위치는 시트마다 다르다**(실사 I, 디지털 H, 아크릴 J, 캘린더/굿즈 G). G의 처방은:

- **L1(추출)은 무해석·무손실**: 시트별 전 컬럼을 행 단위로 그대로 뜬다. 어느 컬럼이 중요한지 판단(적재게이트·정합)은 L2 영역.
- **그룹헤더 composite 귀속맵 13시트 전수**: row1 가로병합 그룹명을 하위 컬럼에 forward-fill 귀속해 `그룹명_하위명` composite 컬럼명을 부여(내지/표지 2축 구분 등).
- **실무진 메타 전수 보존**: 스레드댓글(24건 실텍스트 추출 성공)·배경색(footnote 범례로 의미코드 도출)·COMMENT 컬럼·footnote 안내문·★제약텍스트를 행/셀 귀속으로 보존. L2가 해석.
- **완전성 검증 게이트**: 컬럼 커버리지·non-empty 보존율 100%·행카운트·round-trip diff=0 + **메타 보존 검증**(댓글 수·배경색 셀 수 원본 일치). 미통과 시 L2 진입 차단.

핵심 확정 사실(원본 실측):
- 그룹헤더 가로병합 **62개**(13시트 중 11 상품시트에 분포; MAP·계산공식집초안은 그룹헤더 없음).
- 스레드댓글 **24건** 실텍스트 추출 성공(파일 5개: 실사 9·아크릴 8·굿즈파우치 5·계산공식집초안 1·MAP 1). 작성자 전부 `HP`(google-sheets).
- 배경색 의미: **노랑(FFFFFF00)=신규(MES미등록)**, **그레이배경(FFD9D9D9)=품절/준비중**(단 문구·굿즈파우치 footnote에만 명시 — 타 시트 FFD9D9D9는 "확인 필요"), **초록(FFB6D7A8)=footnote 범례셀 마커**(시트당 1개). footnote 범례 근거로 **확정**.

## ② L1 ↔ L2 경계

| 구분 | L1 추출 (faithful extract) | L2 정규화/정합 (interpret) |
|------|---------------------------|---------------------------|
| 목적 | 원본을 **충실·무해석**으로 보존 | 9속성 매핑·DB 정합·적재게이트 판정 |
| 산출 | 시트별 tidy 1행1레코드(전 컬럼 + composite + 셀메타) | MATCH/MISSING/EXTRA/MISMATCH 분류 |
| 성공기준 | **누락 0**(완전성 게이트 통과) | 의미 정확성 |
| 허용 | 머지 펼침·그룹헤더 composite 합성·키 ffill(화이트리스트, 기록 동반)·셀메타 귀속 | 컬럼 선택·unpivot·코드값 매핑·적재게이트 해석 |
| 금지 | 컬럼 drop·의미가정 박기·빈값 삭제·파이프 평면화·속성별 단일컬럼 추출 | L1 raw 변경 |

**[HARD] L1이 하지 않는 것**:
- "작업사이즈 공백 → MISSING" 판정 **금지**. L1은 작업사이즈 컬럼(I/H/J 등)을 행에 그대로 보존만 한다. 공백/0/값 판정은 L2.
- 속성별 단일 컬럼 평면화·파이프(`|`) join **금지**(false MISSING 직접 원인).
- 별색 5컬럼(R~V)·내지/표지 2축을 단일 컬럼으로 합치기 **금지**.

**경계 판정 규칙**: "이 정보가 다운스트림에 필요한가?"라는 **판단이 들어가는 순간이 L2**. L1은 그 판단 없이 전부 가져온다. "작업사이즈 컬럼 공백을 적재게이트 신호로 본다"는 해석은 L2 영역(G는 "I/J 등 행내 전 컬럼을 함께 떠서 L2가 판단 가능하게"까지만 규정).

## ③ 공통 추출 모델

### 3-1. 관찰단위 (grain) — 시트별 상이, L1은 행 보존이 우선
- **관찰단위 = 상품 × 행**. 1행 = 1레코드, 전 컬럼 보존. grain 자체(상품? 옵션? 사이즈?)는 시트마다 다르나, L1은 **원본 1행 = 추출 1레코드**를 고수하면 grain 해석과 무관하게 무손실이 보장된다.
- **상품 앵커**(상품 귀속용 메타): A형 ragged(D 비어있지않음=새상품), B형 병합(상품악세사리 D병합범위=블록경계), C형 단일템플릿(포토북). 앵커는 ffill 화이트리스트로 처리(3-3).

### 3-2. 빈 셀 보존 (공백 ≠ 없음, 유효0 구분)
- 빈 셀은 명시적 빈값(`""`)으로 보존. drop 금지.
- **유효 0 구분**: 실사 H(블리드)=`0`은 "0mm 블리드"라는 **유효 수치**, 공백은 미정의. `0`을 결측 치환 **금지**. L1은 원본 셀이 `0`이면 `0`, 공백이면 `""`로 그대로 보존(타입 강제 금지 — 숫자/문자 원형 유지).

### 3-3. 세로 리스트 ffill (화이트리스트 한정)
- ragged 세로블록의 상품 귀속 키만 forward-fill. **ffill 대상은 명시 화이트리스트로 한정**(전 컬럼 무차별 ffill 금지 — 진짜 빈값을 덮어 거짓 데이터 생성).
- **ffill 화이트리스트(기본 제안, 사용자 확정 대상 → ⑧)**: A=구분(대분류), B=ID, C=MES ITEM_CD, D=상품명. (E 사이즈 이하 옵션 컬럼은 ragged 그대로 — ffill 금지.)
- ffill 적용 사실을 메타 플래그(`ffilled=true`)로 레코드에 기록(round-trip 정확성).

### 3-4. 머지 펼침
- 머지 범위는 좌상단에만 값 → openpyxl `ws.merged_cells.ranges`로 감지해 범위 전체에 좌상단값 전파(unmerge-and-fill). 펼침 사실을 메타로 기록.
- **B형(상품악세사리) D 세로병합**은 병합범위 = 상품 블록경계 권위. 병합 하위행 D=None을 continuation으로 정확히 처리(병합/비병합 혼재 의식).

### 내지/표지 2축 composite 핵심 (책자·포토북·문구)
2축 시트는 작업/재단사이즈가 **두 그룹에 각각** 존재 → composite로 반드시 구분:
- **책자**: `내지파일사양_작업사이즈`(H) vs `표지파일사양_작업사이즈`(T). 표지에는 **재단사이즈 없음**(R=MES,S=블리드,T=작업,U=약어,V=출력파일,W=폴더 — 실측 확인). 페이지룰(O~Q `내지페이지`)은 내지 전용.
- **포토북**: `내지파일사양_작업사이즈`(H)·`내지파일사양_재단사이즈`(I) vs `표지사양_작업사이즈`(U)·`표지사양_재단사이즈`(V).
- **문구**: `내지파일사양_작업사이즈`(H) vs `표지파일사양_작업사이즈`(U). 표지에 재단사이즈 없음.

## ⑤ 실무진 작성 메타 추출 + 의미/맥락 기준

1차가 버린 비정형 실무 맥락. 정규화 매핑(L2)의 결정적 맥락(품절/제약/별도설정 무시 등)이므로 **버리지 말고 행/셀 귀속으로 보존**한다.

## ⑥ 완전성 검증 기준 (E 기반, 누락 0 기계보증)

시트별 자동 산출 → PASS/FAIL. 하나라도 FAIL이면 **L2 진입 차단**.

| 게이트 | 기준 | 비고 |
|--------|------|------|
| `row_count_match` | 원본 시트 데이터행 수 == 추출 레코드 수 | 머지펼침·ffill로 행 증감 없음 |
| `column_count_match` | 원본 시트 컬럼 수(maxcol) == 추출 필드 수(composite 포함) | **1차 false-MISSING 직접 방어선** |
| `nonempty_cell_preservation` | 원본 non-empty 셀 총수 == 추출 non-empty 값 총수 | **100% 필수**(머지펼침분 보정) |
| `value_checksum_match` | 원본 셀값 집합 해시 == 추출 값 집합 해시 (또는 1% 샘플 diff=0) | 타입강제·trim·반올림 변형 탐지 |
| `roundtrip_diff` | 추출본→원본 재구성 diff == 0 | **핵심시트(실사·아크릴·책자·문구·굿즈·디지털)** 우선 |
| **`comment_count_match`** | 원본 스레드댓글 수(24) == 추출 댓글메타 수 | **메타 보존 검증** |
| **`fill_cell_match`** | 원본 의미배경색 셀 수(노랑·그레이) == 추출 배경색메타 셀 수 | **메타 보존 검증** |
| **`footnote_preserved`** | 시트별 footnote 범례셀(초록) 전문 보존 | 시트메타 |

**기준 카운트(원본 실측, 검증 anchor)**: 스레드댓글 24건 / 그룹헤더 가로병합 62개 / footnote 범례셀 8개(상품시트 중 캘린더·디캘 제외 — 확인 필요) / 노랑·그레이 의미배경 셀은 추출 시 전수 카운트 산출.

### 7-1. 데이터 레코드 (시트별 raw 보존 테이블, 1행1레코드)

{
  "sheet": "실사",                         # 시트명
  "row_seq": 71,                            # 원본 행번호 (1-based)
  "prd_nm": "포맥스보드",                   # 상품명(D, ffill 적용)
  "_anchor_ffilled": true,                  # 상품키 ffill 적용 여부 메타
  "cols": {                                 # 전 컬럼 보존 (composite 컬럼명)
    "구분": "...",
    "ID": "...",
    "MES ITEM_CD": "",
    "상품명": "포맥스보드",
    "사이즈(필수)": "A1 (594 x 841 mm)",
    "비규격(최소/최대)_가로": "",
    "비규격(최소/최대)_세로": "",
    "파일사양_블리드": "3",                  # 유효 0/3 그대로, 공백은 ""
    "파일사양_작업사이즈": "594 x 841",      # ← 1차가 버린 컬럼, 행 보존
    "파일사양_재단사이즈": "...",
    "파일사양_출력용지규격": "...",
    "...": "..."                            # 시트 전 컬럼
  },
  "cell_meta": {                            # 셀단위 메타 (해당 셀에만)
    "사이즈(필수)": {"fill_rgb": "FFFFFF00", "fill_meaning": "신규", "font_rgb": null, "comment": null, "has_constraint_star": false}

## ⑧ 사용자 검토 포인트 (확정 필요)

사람이 확정해야 L1 추출 코드 구현으로 진행할 결정:

1. **시트별 grain(관찰단위)**: L1은 "원본 1행=1레코드"로 무손실 보장하되, 상품 귀속(prd_nm ffill)을 어느 키로 할지. A형 ragged(D 비어있지않음=새상품) / B형 병합(악세사리 D병합) / C형 단일템플릿(포토북) 처리 확정.
2. **ffill 화이트리스트**: 기본 제안 = A(구분)·B(ID)·C(MES)·D(상품명)만. E 사이즈 이하 옵션 컬럼은 ffill 금지 확정(무차별 ffill 시 거짓데이터). 추가/제외 컬럼 있는지.
3. **유효 0 컬럼**: 블리드(실사 H=0 유효 0mm)처럼 `0`을 결측 치환 금지할 컬럼 목록. 다른 수치 0 컬럼(가격 0? 수량 0?) 유효성 확정.
4. **배경색→의미 코드맵 확정**: 노랑=신규·(문구·굿즈)그레이=품절은 footnote 근거 확정. **미확정 색**(실사 파랑 FF92CDDC/FF93CDDD·회색 FFB7B7B7/FFCCCCCC, 아크릴 FF92D050/FFE5B8B7, 타 시트 FFD9D9D9)의 의미를 실무진에게 확인. font color(그레이=비노출·오렌지=무시) 별도 축 추출 범위 확정.
5. **코멘트 귀속 규칙**: 굿즈 E2/O2처럼 **row2(헤더) 위치 댓글 = 컬럼레벨 주석** vs 데이터행 댓글 = 셀레벨 귀속. `레이어창`류 단순 편집기 안내(20건)를 보존만 할지, L2에서 무시할지.
6. **round-trip 핵심시트 범위**: 실사·아크릴·책자·문구·굿즈·디지털 6시트 우선 제안 — 전 13시트 전수로 확대할지.
7. **(확인 필요) footnote 범례셀**: 캘린더·디자인캘린더 시트에 동일 footnote 범례 존재 여부 미확인(상품시트 8개에서만 확정). 두 시트 footnote 유무 확인.

| ⑦ | **하이퍼링크** | `cell.hyperlink.target` | 책자 1건(실사·캘린더 0) | 셀메타 `hyperlink` |
| ⑧ | 병합 | `ws.merged_cells.ranges` | 가로병합 그룹헤더 62개 | composite 컬럼명(④절) |

**[HARD] 행/열 숨김 = 비활성/품절/참고 신호 — 제외·삭제 금지, hidden 플래그 필수 보존**. L1은 숨김행도 레코드로, 숨김열도 컬럼으로 그대로 뜨고 `_row_hidden`/`col_hidden` 플래그만 표기한다. "숨김=비활성→미적재" 판정은 **L2 영역**(②경계 원칙 동일). 예: 포맥스보드 A1(실사 r73)은 `_row_hidden=true` + 작업사이즈공백이 **둘 다** 레코드에 보존되어 L2가 다중 신호로 판정.

**[HARD] 수식 셀 파생 플래그**: 수식 셀은 `data_only=True` 계산값과 `data_only=False` 원본 수식을 **둘 다** 보존한다(`is_formula=true`+`formula`). 수식값은 다른 셀 의존 파생값임을 L2가 알 수 있게 한다(실사 S=VAT포함가는 R가의 ×1.1 파생).
