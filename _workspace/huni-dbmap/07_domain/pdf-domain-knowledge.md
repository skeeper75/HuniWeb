# PDF 도메인 지식 수확 — 인쇄방식·공정·용어의 후니 공식 정의

작성: 2026-06-05 · 전략 H Phase 0. 설명 한국어, 식별자/컬럼/코드/SQL 영어.

> **인과 체인(사용자 명시)**: 인쇄 도메인 지식 → 엑셀 해석 → DB 엔티티(인쇄자동견적 위젯 구축). 본 문서는 그 *첫 칸*을 후니 공식 PDF에서 수확한다.
>
> **HARD 원칙**: PDF 권위 문서에 적힌 것만 도메인 지식으로 단정. PDF에 없는 것은 "PDF 미수록(실무 컨펌/리서치 대상)"으로 표기하고 가설로 분리. DB 추정 금지(라이브는 다른 에이전트가 확인 중).
>
> **확정도 범례**: ✅PDF근거(권위 직접 인용) · 🟡추정/교차(PDF 부분+DB/표준 보강) · 🔴PDF미수록(가설, 실무 컨펌 대상).

## 1. 인쇄방식(인쇄타입) Taxonomy — 후니 공식

### 1-1. 자체제작 vs 외주제작 (주문프로세스 p3 §03 "인쇄타입 및 제작옵션") ✅

후니는 인쇄타입을 **자체제작**과 **외주제작** 2계열로 명시 구분한다.

**자체제작 인쇄출력**(p3 상단 박스, 코드 001~006):
| 인쇄출력 | 코드 | 접수파일 포맷(p8 §03-1) | 담당 팀(p1 공정현황·p10) |
|---------|:---:|----------------------|---------------------|
| 디지털출력(디지털인쇄) | 001 | PDF | 디지털인쇄팀 |
| 실사출력 | 002 | JPG / (PDF+커팅AI) | 특수인쇄팀(실사출력) |
| 특수출력 | 002 | — | — |
| 라텍스(화이트)출력 = 화이트인쇄 | 005 | PDF / (PDF+커팅AI) | 쿠마샵(외주) |
| 특수출력+미싱 | 005 | — | — |
| UV평판출력(UV인쇄) | 005 | PDF / (PDF+레이저커팅AI) | 쿠마샵(외주) |
| 전사인쇄 | 005 | JPG | 쿠마샵(외주) |
| 패브릭출력(패브릭인쇄) | 005 | JPG | 쿠마샵(외주)·봉제가공파트 |
| 도장 | 006 | AI | — |

**자체제작 커팅/후가공**(p3, 코드 002~004):
- 반칼/완칼커팅(003), 시트커팅(002), 레이저커팅(004) → 커팅파트
- 라미네이팅코팅·오시/접지·기계재단·수동도무송·타공·귀돌이·떡제본·링제본·PUR제본·싸바리·중철제본·조립/후가공 — 후가공 항목(p3 하위 박스)
- 콜코팅·(손)재단·조립/가공·미싱가공·도장제작 — 추가 후가공

**외주제작**(p3 우측, 코드 006 "외주 및 파일가공필요한"):
- 책자제작, 현수막, 합판인쇄, 고주파, 폰케이스, 동판제작

**외주공정**(p3): 동판제작·박작업. **선택적 외주공정**: 톰슨/도무송·중철제본.

> ✅ **핵심 권위**: 인쇄타입이 상품의 최상위 분류축. 파일 포맷(PDF/JPG/AI)이 인쇄타입을 *구분 가능*(p8 "파일포맷으로 구분가능"). → DB note "Decision 14 v3 — 1상품=1인쇄방식"과 **독립 2중 도달**.

### 1-2. DB ref-processes 교차 (PROC_000001 인쇄 자식 5종) 🟡

DB `PROC_000001 인쇄`의 자식 = `옵셋(003)·디지털(004,"토너 후니 현재 운영")·실크(005)·실사(006)·UV(002, 변형enum)`. PDF 자체제작 인쇄출력 리스트와 대응:
- 디지털출력 ↔ PROC_000004 디지털 ✅
- 실사출력 ↔ PROC_000006 실사 ✅
- UV평판출력 ↔ PROC_000002 UV ✅
- 화이트/패브릭/전사 — PDF상 쿠마샵 외주, DB PROC_000001 자식엔 별도 없음 🔴(외주방식이 인쇄방식 taxonomy에 어떻게 들어가는지 PDF 미명시)
- 옵셋(PROC_000003)·실크(PROC_000005) — **PDF 인쇄타입 리스트에 명시 없음** 🔴(후니 현재 디지털 운영 중심, 옵셋/실크는 DB에만 존재 → 미운영 가설)

## 2. 17 Case 공정 플로우 — 후니 공식 생산 순서 (공정관리 PDF p1) ✅

공정관리 PDF p1 "02. 공정플로우". 각 Case = **상품군별 공정 시퀀스**. 화살표=공정 순서, `출고`/`입고` 말풍선=팀 간 이동, 점선 박스=조건부(생략 가능) 공정.

| Case | 상품군 | 공정 시퀀스 (PDF 명시) | 비고 |
|:----:|-------|----------------------|------|
| 1 | 낱장인쇄 | 디지털출력 → (코팅) → 재단 → 1차포장 | 코팅 점선=조건부 |
| 2 | 스티커 | 디지털출력 → (코팅) → 반칼커팅/완칼커팅 → 재단 → 1차포장 | 반칼·완칼 분기 |
| 3 | 인쇄후가공 | 디지털출력 → (코팅) → 재단 → 오시/접지/타공/귀돌이/수동도무송 등 → 1차포장 | 후가공 다양 |
| 4 | 책자(링제본) | 디지털출력 → (코팅) → 재단 → 링제본(입고) → 1차포장 | |
| 5 | 책자 | 디지털출력 → (코팅) → 재단 → 중철제본/PUR제본/떡제본(입고) → 재단 → 1차포장 | 제본 택일 3종 |
| 5b | 커버/클립보드(레더+싸바리) | 디지털출력+패브릭출력 → 싸바리(입고) → 조립/후가공 → 1차포장 | 출력 2종 합류 |
| 6 | 실사/사인 | 실사출력 → (쿨코팅) → (조립/후가공) → 1차포장+박스포장 | 박스포장 동반 |
| 7 | 투명실사 | 라텍스(화이트)출력 → (조립/후가공) → 1차포장+박스포장 | |
| 8 | 커팅 | 레이저커팅/시트커팅 → 1차포장+박스포장 | 커팅 택일 |
| 9 | 봉제미싱 | 패브릭출력 → 봉제미싱(입고) → 1차포장 | |
| 10 | UV단품 | UV평판출력 → 1차포장 | |
| 11 | 투명클립보드 | UV평판출력 → 레이저커팅 → 조립/후가공 → 1차포장 | |
| 12 | 아크릴가공 | UV평판출력 → 레이저커팅 → 아크릴가공(입고) → 1차포장 | |
| 13 | 전사 | 전사인쇄 → 1차포장 | |
| 14 | 도장 | 도장제작 → 1차포장 | |
| 15 | 외주공정 | 동판제작 → 박작업 → 생산본부(입고) | 외주 후 본부 합류 |
| 16 | 재고상품 | 상품액세서리 → 1차포장 | 무인쇄(재고) |

> ✅ **공정 순서 권위**: 출력 → (코팅) → 재단/커팅 → 후가공/제본 → 포장. **코팅은 항상 출력 후·재단 전**(점선=조건부). **제본·아크릴가공·봉제·박작업·링제본은 `입고` 동반 = 별도 팀/외주 후 합류**. (DB L2 공정 레시피 트리 토대.)

> 🟡 공정관리 PDF는 "시행초안"(draft) — 운영 변경 가능. 주문프로세스 PDF(2025-10)가 더 최신일 수 있으나 두 문서 간 큰 충돌 없음(공정명 일치).

### 2-1. 공정팀 조직 (공정관리 p2 "공정별 현황표") ✅

14개 공정팀/파트 — 각 공정의 담당 부서이자 생산 단위:

## 3. 후가공 공정 용어 — 후니 공식 정의 (PDF + DB note 교차)

> PDF는 공정 *플로우/명칭*은 주나 *정의(물리적 의미)*는 대부분 미수록. 정의는 DB note(후니가 직접 기입)에서 권위 확보 + 표준 보조.

|------|-------------------|---------|------------------|:------:|
| **오시(Crease)** | 누름선 — 접지 전 압선(종이 갈라짐 방지). 줄수 0~3 | 공정관리 p1 Case3 "오시" | `PROC_000029` note "Crease 누름선" {줄수 max3} | ✅ |
| **미싱(Perforation)** | 점선 절취선 — 손으로 뜯음(티켓/쿠폰). 줄수 0~3 | p1 Case9 봉제미싱·p3 미싱가공 | `PROC_000030` note "Perforation 점선 절취" {줄수 max3} | ✅ |
| **귀돌이(Corner Rounding)** | 모서리 둥근 재단. 직각(default)/둥근(R라운딩) | p1 Case3 "귀돌이"·p3 귀돌이 | `PROC_000026` note "Corner Rounding" + 자식 직각027/둥근028 | ✅ |

## 4. 사이즈/판형 의미축 — PDF 단서 (결함1·3 핵심)

> 사용자 결함1("작업사이즈=재단+블리드 모름")·결함3("전지≠작업사이즈")의 PDF 권위 단서.

### 4-1. 파일명 Rename 규약 = 사이즈 의미 단서 (주문프로세스 p6 §06) ✅

후니 파일명 규약: `품목_출력사이즈_양단면_소재명_거래처명_고객명_파일고유번호_수량`. 예) `1002_접지카드_92x57_D_랑240_BIZ_박현정_30146712_00100ea.pdf`

- **"출력사이즈"** = 파일명 2번째 토큰(`92x57`=가로x세로). p6 박스 명시: `[가로]x[세로]`. → **출력사이즈 = 작업물 실제 출력 치수** ✅
- **"양단면"** = `D(양면)`/`S(단면)` 토큰 ✅
- **"소재명"** = `랑240`(랑데뷰 240g)·`몽240`·`린넨` 등(디지털용지/소재 약어) ✅
- 사이즈 토큰은 **출력 단위 1개**(가로x세로)만 — PDF 파일명엔 전지/재단/블리드 3축 구분 *없음* 🔴(파일명은 출력사이즈만, 작업/재단/블리드 분해는 PDF 미수록 → 가격표 판걸이수 시트가 권위, B-rules R-SIZE-2)

### 4-2. 조판(임포지션) 맥락 = 전지 단서 (주문프로세스 p5 §05 Pain point "조판") ✅

- **아크릴상품**: "아크릴원판 396×480에 5도/9도 구분하여 배치하고 돔보넣음" → **원판(press sheet)에 여러 도수 배치 = 조판/임포지션** ✅
- "★판걸이, 건수, 수량에 따른 조판"(다도안 상품) → **판걸이=원판당 앉히는 작업물 개수** ✅
- **돔보(Tombo/가늠표)** = 조판 시 재단 기준 마크. 출력파일(PDF)과 칼선파일(AI)을 분리 분배 ✅

> ✅ **전지/판형 의미축 PDF 확증**: "아크릴원판 396×480"은 **전지(원판)**, 파일명의 "출력사이즈 92x57"은 **작업사이즈(작업물)**. 조판=작업물을 원판에 판걸이 개수만큼 앉히는 행위. → 결함3("전지≠작업사이즈") **PDF 근거로 확정**. 단, 전지 규격 마스터(국전/46전)는 PDF 미수록 🔴(표준 보조).

### 4-3. 도수(별색) 단서 (p5 "5도/9도")
- "아크릴원판…5도/9도 구분" + 파일명 메모 "4도/5도가 있어서 파일명에 구분필요?"(p6 스티커) → **도수 = CMYK(4도) + 별색 추가(5도~)**. 별색이 도수를 늘림 ✅(결함2 별색=공정과 정합: 별색은 도수칸이 아닌 공정).

## 5. 주문 프로세스 — 공정 택일 구조 맥락 (BLOCK-3 단서)

> 사용자 결함4("공정택일그룹 N/A로 닫음")의 PDF 단서 = "여러 공정 중 택일하는 UI 맥락".

### 5-1. 발주 업무 프로세스 (주문프로세스 p9 §09) ✅
- **접수파일 등록 → (server)썸네일 생성 → 의뢰서 출력 + 주문상태변경 → 제작중**
- **case1 원본파일 그대로 접수**: 업체 완성파일(품목관리 "디자인파일여부 Y/N"으로 자동처리) / Edicus Prepress 랜더링
- **case2 원본 다운→가공→접수파일 등록**: 고객파일 다운→변환(후니프린팅 파일업로드 주문) / 1파일→2파일 분리(ex.자유형스티커) / AI파일이면 썸네일용 JPG 함께 업로드(ex.아크릴스티커,박)
- **수동조판**: 여러 주문 모아 수동조판(ex.틴거울, 아크릴상품, 자석북마크) → 수동조판 파일 별도전달

### 5-2. 인쇄타입별 공정 택일/조합 (주문프로세스 p8 §03-1) ✅
**"인쇄타입별 접수파일 속성"** = 인쇄타입마다 공정 조합이 다름:
- 디지털인쇄: PDF 단독 / PDF+커팅(반칼/완칼)AI / PDF+박(외주)AI → **출력 후 커팅·박 택일 조합**
- 실사출력: JPG 단독 / PDF+커팅AI / 시트커팅PDF / 레이저커팅AI → **커팅 방식 택일**
- 화이트인쇄: PDF 단독 / PDF+커팅AI
- UV평판출력: PDF 단독 / PDF+레이저커팅AI

> ✅ **공정택일 구조 = "인쇄타입(base) + 후가공 공정 택일/조합"**. 디지털인쇄는 커팅(반칼∨완칼)·박을 택일/추가. p1 Case5(제본 중철∨PUR∨떡 택일)·Case8(레이저∨시트커팅 택일)도 동일 구조. → 공정택일그룹 = **인쇄타입별로 표출되는 후가공 옵션 중 택일/다중 선택**. **N/A 아님** — PDF가 택일 구조 존재를 명시. 단, DB excl-group 13행과의 정확한 매핑은 미수록 🟡(L2 재구성 대상).

### 5-3. 파일 다운로드 권한 = 공정팀별 분배 (p8 우측) ✅

## 6. 주문 상태머신 — 후니 공식 (주문프로세스 p1 §01 + p2 §02 + p10 §10)

> order-flow(상태머신)는 lead가 종합할 수 있게 PDF 권위로 수확. 본 작업은 도메인 지식 중심이나, 위젯/DB 엔티티가 의존하므로 상태축을 수확.

### 6-1. 주문 상태값 시퀀스 (p1·p10 §10 server 자동화) ✅
```
미입금 → 결제완료 → 주문완료 → 접수완료 → 제작대기 → 제작중 → 출고준비중 → 출고완료
                                                                    ↘ 주문취소
```
| 상태 | 전이 트리거 (PDF 명시) | 처리주체 |
|------|----------------------|---------|
| 미입금 → 결제완료 | 카드/포인트/에스크로/가상계좌/신용결제(이니시스 승인) | 시스템 자동 |
| → 주문완료 | 결제 완료 | server 자동 |
| → 접수완료 | 접수파일 업로드 → 인쇄공정 파일로 자동 썸네일 생성 | CS(접수팀) 수동 + server |
| → 제작대기 | 의뢰서 출력하면 빨간점 표시 | 발주 |
| → 제작중 | (제작번호별)바코드 입력 → 제작중 상태변경 | 생산 |
| → 출고준비중 | 모든상품 포장완료이면 제작완료 → 송장부여 | 출고 |
| → 출고완료 | (주문번호별)바코드 입력. 택배출고/퀵/직접방문 | 출고 |
| → 주문취소 | 고객직접취소 / 가상계좌 미입금 자동취소(1일전 알림) | 시스템·관리자 |

> 🟡 **"제작완료"** 상태: p1엔 "제작완료"(포장완료로 상태변경, if 모든상품 포장완료이면 제작완료), p10 server엔 "출고준비중". 명칭 차이 — 문서간 불일치(p1 구버전 "제작완료" vs p10 "출고준비중"). 실무 컨펌 대상.

### 6-2. 파일 단위 상태 (p1 파일번호) ✅
- **case1 배송상품(재고)**: 상품준비중 → 포장완료
- **case2 파일업로드상품**: 확인전 → 파일확인 / 재업로드요청 → 재업로드완료 → 다운로드완료 → 포장완료

### 6-3. 자동 알림 규칙 (p2 §02) ✅

## 7. Pain Point — 도메인 설계 동인 (주문프로세스 p5 §05) ✅

리뉴얼/위젯 설계 동인(시스템이 풀어야 할 도메인 문제):
