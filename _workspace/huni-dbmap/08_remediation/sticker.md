# sticker(스티커) 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 **누락·오매칭·모호**가 생겼다.
> 본 문서는 그 결함을 *올바로 처리(수정)*하기 위해, **라이브 Railway DB 직접 read-only SELECT를 권위**로
> sticker 16상품을 종단 규명한다. 도메인 지식(07_domain KB)은 결함을 옳게 분류하는 렌즈다.
> 정제된 DB가 인쇄 자동견적 위젯의 기반이 된다.
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본·v2 ref-CSV는 stale 가능).
> 추정·임의판단 0 — 라이브·엑셀 L1·도메인 KB·PDF 실재 근거만. 미지/추정은 가설+컨펌 플래그.
> INSERT/UPDATE/DDL 0. DB 미적재 유지(본 문서는 처리 *방향* 설계). `db railway`·비표준포트·JOIN KEY=`prd_nm`.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. **라이브 SELECT 수행 12회**(전부 read-only).
> **템플릿** `08_remediation/digital-print.md` 종단 흐름·원칙·산출형식 준수.

## ⚠️ 머리말 — 본 검증이 v2/예단 대비 뒤집은 가장 큰 사실 (권위 반전)

**digital-print 파일럿은 R-PROC-2(줄수/개수 공정) 누락이 결함의 핵심이었으나, sticker는 정반대다.**

라이브 직접 확인 결과:

1. **sticker의 커팅 공정(반칼/완칼/스티커완칼)은 *이미* 16상품 중 14상품에 올바로 적재돼 있다.**
   원래 우려(스티커 핵심=커팅인데 누락?)와 달리, **라이브에서 커팅은 도메인 의미대로 분기 적재됨**:
   - 반칼(자유형) 상품(052·053·054·064) → `PROC_000054 반칼`(Kiss Cut, 모양+조각수)
   - 완칼(자유형) 상품(055·056·057) → `PROC_000053 완칼`(Die Cut, 모양)
   - 반칼 도형/사이즈 enum 상품(058~063·066) → `PROC_000055 스티커완칼`(조각수)
   → **digital-print식 "공정 행 자체 누락"은 sticker엔 거의 없다.** 커팅 매핑은 건전.

2. **`가변(텍스트)`·`가변(이미지)` 컬럼은 sheet 전체 154행 전부 공백 = 신호 0** (라이브·L1 동시 확인).
   이는 **digital-print 템플릿 잔재 컬럼**으로, VDP 공정(PROC_000031/032)은 sticker에 **존재하지 않는 게 정상**.
   → digital-print의 R-PROC-2 식 거짓 MISSING을 **non-empty 게이트로 차단**(기대행 미생성). VDP MISSING = **0건(false 차단)**.
   마찬가지로 `코팅`·`추가상품` 컬럼도 전 상품 공백 → 코팅 공정·addon **0행이 정상**.

3. **진짜 잔존 결함은 다른 축이다 — "사이즈/커팅 형상 enum 축의 평면화"와 "화이트별색 공정 누락".**
   sticker 결함의 뿌리는 줄수/개수가 아니라, **상품별 도형·치수 커팅 enum(원형 25~90mm 등)이 DB 사이즈 축에서 드롭**된 것과,
   **투명/홀로그램 상품의 화이트인쇄(별색=공정)가 미적재**된 것이다(§4 G-SK-1·G-SK-2).

**교훈(HARD 재확인):** 시트마다 결함 축이 다르다. digital-print의 R-PROC-2 렌즈를 sticker에 무비판 적용하면
(a) 이미 건전한 커팅을 오보, (b) 빈 가변 컬럼에 거짓 기대행 생성, (c) 진짜 결함(형상 enum 평면화·화이트별색)을 놓침 — 셋 다 위험.
본 문서의 모든 수치는 **라이브 12 SELECT로 재산출**.

**핵심 도메인 캐스케이드(sticker 고유):**
1. **커팅 = 스티커의 핵심 공정**이며 **세 갈래로 의미 분기**한다 — 반칼(Kiss Cut, 종이만)·완칼(Die Cut, 종이+후지)·스티커완칼(완칼+조각수).
   상품명/`커팅` 값이 이 분기를 결정한다(07_domain `prcs_dtl_opt`: 053 완칼=`모양`, 054 반칼=`모양`+`조각수`, 055 스티커완칼=`조각수`).
2. **`커팅` 컬럼 안의 도형·치수 enum**(원형 60mm(6ea), 30x30mm(20ea), 하트55mm 등)은 단순 텍스트가 아니라
   **그 상품에서 가능한 커팅 형상/판당 EA를 정의하는 별도 축**이다. 도형 enum 상품(058~062·066)에서 이 축이 옵션 캐스케이드의 본체다.
3. **빈 `가변` 두 컬럼은 digital-print 템플릿이 남긴 잔재** — sticker엔 VDP가 없다. 신호0이므로 기대행 생성 금지(R-PROC-2 거짓 적용 방지).

## ④ 결함 표 — 누락(MISSING)·오매칭(MISMATCH)·모호(CONFIRM) (도메인 렌즈, 라이브 재확인)

> 분류 기준: **MISSING** = 정규화 규칙대로 엑셀을 변환하면 적재돼야 하는데 라이브에 없는 행.
> **MISMATCH** = 잘못 연결/오매칭. **CONFIRM** = 도메인 의미 미확정(실무 컨펌 대상).
> 모든 판정은 라이브 SELECT로 재확인.

### G-SK-1 [MISSING · 핵심] 화이트별색 공정 미적재 (투명/홀로그램 상품)

- **도메인 근거:** 엑셀 `별색인쇄(옵션)_화이트 = 화이트인쇄(단면)`은 빈값이 아니라
  **별색(화이트) 공정이 이 상품에 존재한다는 신호**다. 도메인 KB §별색: 별색 = CMYK 외 별도 잉크 후공정,
  DB 정규화상 **인쇄 옵션이 아니라 공정**(PROC_000007 별색인쇄 family, 화이트 = 자식 PROC_000008).
  투명/홀로그램 원단은 바탕이 비쳐 **화이트 베이스 인쇄가 필수** — 도메인상 당연한 공정.
- **엑셀 신호 보유 상품(L1 권위):** 반칼 자유형 투명스티커(053)·반칼 자유형 홀로그램스티커(054)·
  낱장 자유형 투명스티커(056)·반칼팬시투명스티커(063, use_yn=N). **4상품 화이트인쇄(단면) 보유.**
- **라이브 재확인(SELECT #6,7):** 053·054·056·063 processes = 커팅 공정 1행뿐(반칼/완칼/스티커완칼).
  **PROC_000007/008(화이트별색) 0행 — 전 상품 부재.** print_option도 단면 1행뿐(별색 흔적 없음).
- **MISSING:** **활성 3상품(053·054·056) + 미출시 1상품(063)** = 별색 공정 행 누락. 원적재가 별색을
  "인쇄 옵션 위치의 부속 텍스트"로 오해해 공정으로 정규화하지 않은 결함(digital-print G-DP-7과 동형).
- **처리방향(§5 R1):** 053·054·056(·063)에 PROC_000007(또는 자식 008 화이트) `t_prd_product_processes` 적재.
  단 별색 부모/자식 적재 규칙(부모 007 vs 자식 008만)은 digital-print R-PROC-4 패턴 확인 후 통일 — **컨펌**.

### G-SK-2 [MISSING · 의미축 평면화] 커팅 도형·치수 enum 축 드롭

- **도메인 근거:** 도형 enum 상품(058 원형·059 정사각·060 직사각·061 띠지·062 팬시·066 합판도무송)의
  엑셀 `커팅` 컬럼은 **그 상품에서 선택 가능한 도형/치수/판당 EA**를 정의하는 옵션 축이다
  (예: 058 = `원형 25mm(24ea)`~`원형 90mm(6ea)` 10종 + ★사이즈선택 분기).
- **라이브 재확인(SELECT #11,12):**
  - **058 반칼원형:** DB size = **A4·A5 2행뿐**(인쇄 판형). 엑셀 원형 도형 enum **10종이 어디에도 없음**.
  - **059/060/061도 동형:** size = A4·A5 2행, 형상 enum(정사각 8종/직사각 25종/띠지 17종) 미적재.
  - **066 합판도무송:** DB size = **26행**(정사각 12 + 직사각 14). 엑셀 37종 중 **원형 11종(10~60mm) 전부 드롭**.
    라이브 `siz_nm like '원형%'` → **0건 확인**.
- **MISSING(라이브 확정):** 합판도무송 **원형 11종**(정확 행수 라이브 직접 카운트). 058~062 형상 enum은
  **DB의 어느 축에도 표현 안 됨**(size=판형만, excl_group=0, process=조각수만) → 형상 enum 축 자체 부재.
  이는 digital-print의 plate_size 의미축 평면화와 동형 — **속성별 단일 컬럼 평면화로 형상 축 drop**(L1/L2 교훈 재현).
- **처리방향(§5 R2):** 형상·치수 커팅 enum을 어느 축에 적재할지 **설계 결정 필요**(size 보조축 vs 신규 옵션 테이블 vs
  process param 확장). 우선 합판도무송 원형 11종은 **size 누락이 명확** → `t_prd_product_sizes` 보강. **설계 컨펌.**

### G-SK-3 [MISMATCH] 소량자유형(064) 사이즈에 배경지·헤더택 치수 오공유

- **도메인 근거:** 064 소량자유형스티커 사이즈는 50×70~94×94 소형 스티커 7종이어야 한다.
- **라이브 재확인(SELECT #10):** 064 size 7행 중 **SIZ_000043(80x80) note=`적용=인쇄해더택`**,
  **SIZ_000036(94x94) note=`적용=인쇄배경지`** — 즉 **다른 상품군(헤더택/배경지)의 size_cd를 공유**해
  스티커에 매단 상태. 나머지 5행(SIZ_000061~065)은 note=`적용=소형반칼스티커`로 정상.
- **판정:** **MISMATCH(오매칭)** — 치수값(80×80·94×94)은 우연히 맞으나 **size 마스터 엔트리의 적용 상품이 헤더택/배경지**.
  master size 재사용으로 의미 오염(브리들). 가격·전지 메타(`전지=316x467 / 적용=인쇄배경지`)가 스티커와 무관.
- **처리방향(§5 R3):** 064 전용 80×80·94×94 size_cd 신규 생성 후 재연결, 또는 공유가 의도면 note 정정. **컨펌**(064는 use_yn=N).

### G-SK-4 [CONFIRM→false MISSING 차단] 가변(텍스트/이미지)·코팅·추가상품 = 신호0

- **digital-print 잔재 컬럼:** 엑셀 `가변(텍스트)`·`가변(이미지)`·`코팅`·`추가상품` = **sheet 전체 154행 전부 공백**(L1·라이브 동시 확인).
- **판정:** 이 컬럼들에 R-PROC-2식 기대행을 생성하면 **거짓 MISSING**. sticker엔 VDP·코팅·addon 공정이 **존재하지 않는 게 정상**.
  → **non-empty 게이트로 차단. MISSING 0건(false 철회).** (digital-print 파일럿이 명시 경고한 함정 회피.)

### G-SK-5 [CONFIRM→MATCH] 자재 — 스티커 원단 직접명 정합

- **라이브 재확인(SELECT #5):** 16상품 material = 유포/무광코팅/유광코팅/미색/비코팅스티커·투명·홀로그램·
  유포지·투명전용지·데드롱·타투전용지 등 — 엑셀 `종이` 직접명과 **mat_nm 1:1 매치**(usage_cd=USAGE.07 일관).
- **판정:** **MATCH.** sticker는 `*별도설정`(IMPORT 포인터)가 아니라 **자재 직접명** → digital-print의 G-DP-3(IMPORT 미적재) 결함은 sticker엔 **없음**.
  자재 축은 건전. (단 065 스티커팩 2종·057 대형 1종이 엑셀 종이 종수와 일치하는지는 §6 컨펌.)

### G-SK-6 [CONFIRM] 반칼 도형상품을 PROC_000055(스티커완칼)로 매핑한 의미 정합

- **도메인 모호:** 058~062는 상품명이 "**반칼**원형/정사각/...스티커"인데 DB는 `PROC_000055 스티커완칼`(완칼+조각수)로 적재.
  반칼=Kiss Cut(종이만)인데 "완칼" 계열 공정 코드를 부여한 것이 **명칭상 불일치**로 보일 수 있다.
- **라이브 근거:** PROC_000055 note=`Die Cut + 조각수`. 도형 enum(원형 25mm 등 판당 다EA)은 합판/조각 단위 커팅이라
  스티커완칼이 의미상 맞을 가능성(판 단위 다조각 도무송형). 단 상품명 "반칼"과의 정합은 **도메인 컨펌 필요**.
- **판정:** **CONFIRM** — 058~062의 반칼 vs 스티커완칼 코드 선택이 의도된 것인지(판당 다조각=스티커완칼 규칙) 실무 확인.

### G-SK-7 [글로벌 갭, sticker 한정 아님] qty_unit_typ_cd 전 NULL

- **라이브:** `t_prd_products.qty_unit_typ_cd` = **272 전상품 NULL**(sticker 16 포함).
- **판정:** sticker 고유 결함 아님 → 글로벌 적재 갭으로 별도 처리(QTY_UNIT 코드 부여 정책 컨펌). digital-print G-DP-9와 동일.

## ⑦ 큰 그림 적용 (도메인 → 엑셀 → DB결함 → 처리 → 위젯)

- **도메인 KB(07_domain):** 반칼(Kiss Cut)/완칼(Die Cut)/스티커완칼 분기·별색=공정·조각수 param 규칙이
  "엑셀 커팅값/화이트인쇄/조각수"를 옳게 해석하는 렌즈가 됨. 커팅 매핑이 이미 건전함을 확인하고(과잉 보고 방지),
  진짜 결함(별색 공정·형상 enum 축)만 규명.
- **엑셀 L1(무손실):** 전 컬럼 보존 덕에 커팅 형상 enum·화이트인쇄 신호·빈 가변 컬럼을 상품별로 정확 추출(추정 0).
  특히 **non-empty 게이트로 digital-print 잔재 컬럼의 거짓 MISSING 차단**(파일럿 교훈 적용).
- **DB 라이브 권위:** 14상품 커팅 정상 적재·자재 직접명 정합을 라이브로 확인해 false 결함을 걷어내고,
  진짜 잔존 결함(화이트별색 3~4상품·합판 원형 11종 size·058~062 형상 enum 축·064 size 오공유)만 확정.
- **처리방향:** 마스터 무변경 + 상품 연결 테이블만 적재(FK 안전). 화이트별색은 digital-print 별색 패턴 재사용.
  형상 enum 축은 적재 축 설계 결정 선행(R2).
- **위젯 기반:** 정제된 9속성이 자동견적 위젯 옵션 캐스케이드(커팅→반칼/완칼, 화이트별색→투명상품 옵션,
  형상 enum→도형 선택 UI, 조각수→수량)의 데이터 소스. 형상 enum 축 누락 시 위젯에서 도형 선택 불가 → R2가 위젯 선결 조건.
