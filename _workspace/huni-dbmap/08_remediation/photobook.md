# photobook 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 **누락·오매칭·모호**가 생겼다.
> 본 문서는 그 결함을 *올바로 처리(수정)*하기 위해, **라이브 Railway DB 직접 read-only SELECT를 권위**로
> photobook(포토북) 상품군을 종단 규명한다. 도메인 지식(07_domain KB)은 결함을 옳게 분류하는 렌즈다.
> photobook 핵심 = **단일 상품 1종 + 표지타입(하드/레더하드/소프트)·사이즈 variant** + **내지/표지/면지 셋트 구조** +
> **제본**(도메인상 레이플랫 PROC_000025 "포토북 전용" — 그러나 라이브·엑셀은 PUR, §4 G-PB-1 충돌).
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본·v2 stale 가능). 추정·임의판단 0 — 라이브·엑셀 L1·도메인 KB 근거만.
> 미지/추정은 가설+컨펌 플래그. INSERT/UPDATE/DDL 0. DB 미적재 유지. `db railway`·비표준포트·JOIN KEY=`prd_nm`.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. 라이브 SELECT는 booklet과 공유(공동 15회) + photobook 전용 4회.

## ⚠️ 머리말 — 본 검증이 과업 가정 대비 뒤집은 가장 큰 사실 (권위 반전)

1. **과업 브리프 "photobook=레이플랫제본(PROC_000025) 단일템플릿"** → **라이브 반전(충돌)**.
   라이브 PRD_000100 제본공정 = **PROC_000020 PUR제본**이고, 엑셀 photobook 전 행 `제본사양_제본=PUR`이다.
   **레이플랫제본 PROC_000025("포토북 전용")는 어떤 상품에도 적재 0건**(SELECT 확인). → photobook은 도메인 KB의
   "레이플랫=포토북 전용" 정의와 달리 **실제로는 PUR로 적재·표기**됨. 단정 금지 — **도메인 충돌로 컨펌**(§4 G-PB-1).
2. **R-MAT-4 "parent 자재 비어있음이 정상"** → **라이브 반전**(booklet과 동일). photobook도 자재·사이즈를
   **parent(PRD_000100)에 usage_cd로 적재**하고, 표지타입 5종·내지·면지 sub_prd(PRD_TYPE.02 101~107)는 **전부 빈 껍데기**. sets만 연결.
3. **B-rules 예측 "page_rule MISSING"** → **라이브 반전(false)**. photobook page_rule 적재됨(24/150/2).

## ① photobook 상품군 — 도메인 이해 (라이브 권위)

photobook 시트 = **단일 상품 `포토북 [디자인명]`**(엑셀 `구분=포토북`, 가격포함 시트). 라이브 PRD_000100,
`prd_typ_cd=PRD_TYPE.04 디자인상품`, use_yn=Y, editor_yn=**Y**(편집기 전용·file_upload_yn=N).

엑셀 17행은 별개 상품이 아니라 **하나의 상품의 variant 행**이다:
- **표지타입(필수)** = 하드커버 / 레더하드커버 / 소프트커버 — variant 축(표지 sub_prd로 적재).
- **사이즈** = 8x8(200x200)·10x10(250x250)·A5·A4 + 표지타입×사이즈 조합별 작업사이즈/가격 행.
- 마지막 행(row 17) = **메타 코멘트**(제목=필수/빨강, 그레이=비노출, 핑크=편집기, "Figma 참고"). 상품 데이터 아님.

| 라이브 prd_cd | prd_nm | type | 역할 |
|:-------------:|--------|:----:|------|
| PRD_000100 | 포토북 [디자인명] | .04 | parent(자재·사이즈·공정 적재 권위) |
| PRD_000101 | -내지(몽블랑130) | .02 | 내지 sub_prd (빈 껍데기) |
| PRD_000102 | -표지(하드커버) | .02 | 표지 variant |
| PRD_000103 | -표지(아트250+무광코팅) | .02 | 표지 variant(=하드커버 표지지) |
| PRD_000104 | -면지(그레이) | .02 | 면지 sub_prd |
| PRD_000105 | -표지(레더하드커버) | .02 | 표지 variant |
| PRD_000106 | -표지(레더) | .02 | 표지 variant(=레더하드 표지지) |
| PRD_000107 | -표지(소프트커버) | .02 | 표지 variant |

> **표지타입 vs 표지종이 2축:** 엑셀은 `표지타입(필수)`(하드/레더하드/소프트)과 `표지종이사양`(아트250+무광코팅/레더)을
> 분리한다. 라이브는 이를 표지 sub_prd 5종으로 평면화 적재(타입+종이 혼재). USAGE.02 자재로 parent에도 5행. → §4 G-PB-3 모호.
> **가격 정보(가격_기본 24P·추가 2P당)는 round-2 가격엔진 영역** — 본 문서 9속성 매핑 범위 밖(이연).

## ④ 결함 표 — 누락·오매칭·모호 (도메인 렌즈, 라이브 재확인)

### G-PB-1 [CONFIRM · 도메인 충돌] 제본 = PUR vs 레이플랫제본(025) 충돌

- **도메인 KB:** PROC_000025 레이플랫제본 note="포토북 전용"(decision-trail·db-domain-structure). 과업 브리프도 "photobook=레이플랫 단일템플릿".
- **라이브:** PRD_000100 process = **PROC_000020 PUR제본** 단독. PROC_000025 레이플랫은 **전 상품 적재 0건**.
- **엑셀:** photobook 전 행 `제본사양_제본=PUR`.
- **판정:** 엑셀·DB는 일치(PUR)하나 도메인 KB의 "레이플랫=포토북 전용"과 충돌. **단정 금지 — CONFIRM.**
  (가설①: 후니 포토북은 실제 PUR 제본이고 레이플랫은 미운영 마스터. 가설②: 레이플랫이어야 하는데 PUR로 오적재.)
- **처리방향:** 실무 컨펌으로 PUR/레이플랫 확정 후, 레이플랫이면 PROC_000025로 교체 + 레이플랫 미운영 마스터 정리.

### G-PB-2 [MISMATCH] 표지 sub_prd·parent 자재의 타입↔종이 평면 혼재

- **엑셀:** `표지타입`(하드커버/레더하드커버/소프트커버 = 3 타입)과 `표지종이사양`(아트250+무광코팅/레더 = 종이)은 **별개 2축**.
- **라이브:** 표지 sub_prd 5종(102 하드커버·103 아트250+무광코팅·105 레더하드커버·106 레더·107 소프트커버) =
  **타입(하드/레더하드/소프트)과 종이(아트250/레더)가 한 평면에 혼재**. 하드커버(타입)와 아트250+무광코팅(그 표지지)이 별 sub_prd로 분리됨.
- **판정:** 표지타입 variant 축과 표지종이 자재 축이 평면화돼 **의미 중복/모호**(MISMATCH). 위젯에서 표지타입 선택 시 종이가 종속돼야 하나 평면 5종은 그 위계를 잃음.
- **처리방향(컨펌):** 표지타입(3) → 종이(타입별 1) 2계층으로 재구조화할지, 현 평면 유지할지.

### G-PB-3 [MISMATCH] 복합표기 `아트250+무광코팅` 미분해 (R-MAT-3)

- **도메인 근거:** `아트250+무광코팅` = 자재(아트250)+공정(무광코팅 PROC_000015). R-MAT-3은 2축 분해 요구.
- **라이브:** `아트250+무광코팅`이 **단일 mat_nm으로 통째 적재**(USAGE.02). 무광코팅이 process(PROC_000015)로 분리 안 됨.
- **판정:** 자재축에 공정이 섞임 = MISMATCH. 위젯 코팅옵션·가격(코팅비 PRC_COMPONENT_TYPE.02) 산정 시 누락.
- **처리방향:** `아트250`=자재 + `무광코팅`=PROC_000015 process 분해 적재(컨펌).

### G-PB-4 [CONFIRM] 책등(10/12/14/16) param 미적재 여부

- **엑셀:** `제본사양_책등` = 하드/레더=`10/12/14/16`, 소프트=`4/6/8/10/12/14` 선택지.
- **라이브:** process PROC_000020(PUR) 1행 — 책등 param(`prcs_dtl_opt.책등`) 적재 여부 라이브 JSON 미확인(본 검증 범위). 제본 PROC_000017 스키마 `책등`(mm) 입력 존재.
- **판정:** CONFIRM — 책등 선택지가 process param 또는 사이즈 variant로 적재됐는지 확인 필요.

### G-PB-5 [정상] sub_prd 빈 껍데기·page_rule·sizes

- 내지(몽블랑130 USAGE.01)·면지(그레이 USAGE.03)·page_rule(24/150/2)·sizes(4)는 parent 적재 정상. sub_prd 빈 껍데기는 라이브 적재 모델(parent-carries-all)상 정상.

### G-PB-6 [글로벌 갭] qty_unit_typ_cd NULL

- PRD_000100 qty_unit NULL(글로벌 272 NULL). photobook 고유 결함 아님 → 글로벌 정책 컨펌.

## ⑦ 큰 그림 적용 (도메인 → 엑셀 → DB결함 → 처리 → 위젯)

- **도메인 KB:** 레이플랫제본(포토북 전용)·USAGE 슬롯·복합표기 분해 규칙이 photobook 엑셀을 옳게 해석하는 렌즈가 됐고,
  **레이플랫↔PUR 충돌**을 결함이 아닌 *도메인 미확정*으로 정확 분류(단정 금지).
- **엑셀 L1(무손실):** 표지타입·표지종이·책등·제본을 분리 추출 → 평면화·복합표기 미분해 결함을 정밀 포착.
