# codex 레시피 생성 프롬프트 (감사 추적)

> 호출: `codex exec -s read-only` (gpt-5.5, AVAILABLE 2026-06-18). 입력=`_authority-context.md` 전문 + 아래 지시.
> codex 산출=가설(권위/라이브 검증 전 사실 아님). 본문은 codex가 생성, Claude는 수집·구조점검만.

## 시스템 지시 (codex에 전달한 프롬프트 본문)

너는 후니프린팅 인쇄 가격엔진 도메인 전문가다. 아래 [권위 컨텍스트]만을 사실 근거로, **디지털인쇄 시트 전 상품의 작업지시서/생산지시서형 레시피**를 한국어로 작성하라.

규칙[HARD]:
1. 컨텍스트에 없는 사실 날조 금지. 없으면 "GAP: ..." 로 명시.
2. 3배지 강제: [권위-확정](상품마스터/가격표 명시), [라이브](t_* 실측), [codex-가설](도메인 추론·검증 필요).
3. 상품별 7항목: ① 정체(무엇·생산형태·고객카테고리) ② 자재 BOM(본체·부속) ③ 공정(인쇄·후가공 순서) ④ 옵션(자재/공정 BUNDLE·사이즈·도수·인쇄옵션·수량 — 각 t_* 차원 연결) ⑤ 가격공식 바인딩(PRF_*) ⑥ 가격구성요소(comp+use_dims+단가행 키) ⑦ 견적 입력→가격 흐름(한 줄 사슬).
4. 가격공식 패밀리(PRF_DGP_A~F·FIXED)별로 공통 사슬을 한 번 정의하고, 상품별로는 차이만 기술(중복 최소화).
5. 미바인딩 상품(투명엽서·지그재그엽서·펄명함·모양명함 등)은 "라이브 미바인딩 → 견적 불가" GAP으로 정직 기록.
6. premium-postcard 검증의 F-1/F-2(print_opt 도수/면 충돌·S1+S2 이중배선)·출력매수 환산을 패밀리 레시피의 "견적 주의" 로 반영.

산출 = 마크다운 한 문서. 패밀리 사슬 정의 → 30 상품 레시피 → 견적 흐름 요약.
