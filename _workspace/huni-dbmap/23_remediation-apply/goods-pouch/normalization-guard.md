# 굿즈파우치 — 정규화 가드 (normalization-guard)

> **작성** 2026-06-14 · round-13. 사용자 기준 4 = "정규화·오염 방지". apply.sql이 라이브에 쓰는 모든 값이 **3중 화이트리스트**(① table-spec 도메인 ② webadmin FK ③ 260610 L1) 안에 있고, 밖의 값을 **새로 만들지 않음(0)** 을 실증.
> **핵심 원칙:** 굿즈 오염의 진원은 v03 마이그레이션(`자재구분`="파우치" 무차별 채움)이다. 본 정정은 **새 오염을 추가하지 않고**, 즉시적용분은 **이미 실재하는 정상 코드값만 재참조**(search-before-mint)한다.

---

## 1. 즉시적용분(W-G01)이 쓰는 값 — 전수 화이트리스트 검증

apply.sql 즉시적용 블록은 `t_prd_product_categories`에만 쓴다. 쓰는 컬럼별 값 출처:

| 컬럼 | 쓰는 값 | 화이트리스트 출처 | 밖의 값 |
|------|---------|-------------------|:--:|
| `prd_cd` | 33개 굿즈 상품코드 | ③ 라이브 실재(PRD_000183~280) — psql `count=33/33` | **0** |
| `cat_cd` | 33개 정상 카테고리코드 | ② FK 대상 `t_cat_categories` 실재 + 전부 upr NOT NULL(정상) — psql `total=33·normal=33` | **0** |
| `main_cat_yn` | 'Y'(INSERT)·'N'(강등) | ① table-spec YN 도메인('Y'/'N') | **0** |
| `disp_seq` | 1 | ① integer | **0** |
| `note` | "정정 2026-06-14: …" 한글 | (자유 텍스트·변경이력) | N/A |
| `reg_dt`/`upd_dt` | now() | ① timestamp | **0** |

> **결론:** 즉시적용분이 만드는 값은 **신규 코드값 0**(새 cat_cd·새 prd_cd·새 enum 0). 전부 라이브 실재 코드 재참조 = search-before-mint 완전 준수. 화이트리스트 밖 값 **0건**.

**재현 SELECT:**
```sql
-- 정상 cat_cd 33개 전부 실재+정상:
SELECT count(*) total, count(*) FILTER (WHERE upr_cat_cd IS NOT NULL) normal
FROM t_cat_categories WHERE cat_cd IN ('CAT_000165',...,'CAT_000272');  -- → 33 | 33
-- prd_cd 33개 전부 실재:
SELECT count(*) FROM t_prd_products WHERE prd_cd IN ('PRD_000183',...,'PRD_000280');  -- → 33
```

## 2. 자동매칭의 정규화 규칙 (오매핑 방지)

고아→정상 노드 매칭은 **이름 동일성**으로 도출했다. 정규화 규칙:

| 매칭 단계 | 규칙 | 결과 |
|----------|------|------|
| 1차 | `prd_nm = cat_nm` 정확 일치 + 정상 노드(upr NOT NULL) | 30건 |
| 2차 | 공백 무시 `replace(nm,' ','')` 일치 | +3건 (초슬림마우스패드·말랑네임택·레더아이패드/노트북파우치) |
| 매칭 실패 | 동명 정상 노드 부재 | 2건 → **W-G02 컨펌**(자동 적용 제외) |

> **오매핑 가드:** "비슷한 이름"(부분일치·LIKE)으로 추론 매칭하지 **않음**. 정확/공백무시 일치만 자동 적용. 애매한 2건(타이벡북커버→타이벡 노드 다수·이미지피켓→이미지피켓(우치와))은 **컨펌으로 라우팅**(추측 0).

## 3. 컨펌대기분이 건드리지 않는 화이트리스트 (보존 원칙)

| 대상 | 보존 이유 |
|------|----------|
| `t_prd_product_sizes` size행(클립보드 SIZ_000413/415·레더라벨 SIZ_000492/494/496 등) | [HARD] size→option 재분류(W-G04)는 **기계삭제 금지** — 가격사슬 파손. 본 트랙은 size행 0건 변경 |
| `t_mat_materials.mat_typ_cd` (.09 파우치 오염) | 소재 판정 컨펌(Q-GP-6) 전 변경 금지 — 잘못 정정하면 새 오염 |
| `t_mat_materials` 비소재 자재행(형상·잉크색·용량) | 정확축(size/도수) 신규 적재 검증 전 논리삭제 금지 |
| `t_cat_categories` 고아 노드 자체(301~306) | hard-delete 금지 — main_cat_yn 강등만(논리정리). 노드 완전 정리는 컨펌(Q-GP-4) |

> 본 트랙이 **값을 새로 만들거나 바꾸는 것은 즉시적용분(카테고리 재연결)뿐**이며, 그것마저 신규 코드값 0. 나머지는 전부 컨펌 후 별 트랙.

## 4. 가드 결론

- **신규 코드값 mint: 0** (cat_cd·prd_cd·enum·siz_cd·mat_cd 전부 재참조).
- **화이트리스트 밖 값: 0** (3중 화이트리스트 전수 통과).
- **오매핑 가드: 적용** (정확/공백무시 일치만 자동, 애매분 컨펌 라우팅).
- **보존 원칙: 준수** (size행·자재행·고아노드 hard-delete 0).
