# 전체 카탈로그 적재 가능성 매트릭스 — 275 상품

> 라이브 DB(= huni-admin 상품뷰어가 렌더하는 현재 상태)를 read-only SELECT 로 스냅샷하고,
> GO 판정된 매핑 산출물(round-5 `_exec` 델타 · 가격 `_exec_price` · 정정 묶음수)을 결합해
> 상품별 적재상태를 판정했다. **DB 쓰기 0** — 본 문서·생성기 모두 read-only 평가.
> 생성기: `gen_catalog_loadability.py`(재현 가능). 스냅샷: `.live-catalog-snapshot.tsv`.
> 식별자/SQL 영어, 설명 한국어. 권위: 라이브 DB > 스키마 시트 > 추출본.

## 0. 한 줄 결론

전체 **275상품** 중 **완비 179** · **적재대기 91** · **차단·GAP 0** · **검토 5**.

> **현재 적재 가능(검증 GO) = round-5 47상품 델타 + 정정 묶음수 18행(9상품) + 가격 2,320행. 나머지 상품은 라이브 완비(적재 불요) 또는 미매핑(라운드 범위 밖).** 가격은 라이브 전건 0 (공식바인딩 N) — 가격 적재본 45상품만이 유일한 가격 공급원.

## 1. 적재상태 verdict 정의

| verdict | 의미 |
|---------|------|
| `완비` | 라이브가 이 상품에 필요한 속성을 보유 — 대기 델타 없음(적재 불요) |
| `적재대기` | round-5/정정/가격 델타가 존재(괄호에 종류 표기: R5-mat·R5-proc·R5-bdl·정정-bdl·가격) |
| `차단·GAP` | round-5 차단/GAP 가 이 상품을 건드림 |
| `검토` | 코어 속성이 비정상 공란(인간 확인 권장) — 카테고리 norm 적용해 과다 플래깅 회피 |

> 정상 공란은 검토로 올리지 않음: 엽서·완성굿즈는 묶음/페이지룰 부재가 정상, 상품권·포토카드 등은 자재-less 가 정상, 책자류는 사이즈/판형이 페이지룰로 대체될 수 있음.

## 부록. 스냅샷 재생성 SQL (read-only)

```sql
-- db=railway. 9속성 건수 + main 카테고리 + 가격 공식바인딩. SELECT 전용.
SELECT p.prd_cd, p.prd_nm, COALESCE(mc.cat_nm,'(미분류)') AS main_cat,
  (SELECT count(*) FROM t_prd_product_categories x WHERE x.prd_cd=p.prd_cd) AS n_cat,
  (SELECT count(*) FROM t_prd_product_sizes x WHERE x.prd_cd=p.prd_cd) AS n_siz,
  (SELECT count(*) FROM t_prd_product_print_options x WHERE x.prd_cd=p.prd_cd) AS n_print,
  (SELECT count(*) FROM t_prd_product_plate_sizes x WHERE x.prd_cd=p.prd_cd) AS n_plate,
  (SELECT count(*) FROM t_prd_product_materials x WHERE x.prd_cd=p.prd_cd) AS n_mat,
  (SELECT count(*) FROM t_prd_product_processes x WHERE x.prd_cd=p.prd_cd) AS n_proc,
  (SELECT count(*) FROM t_prd_product_bundle_qtys x WHERE x.prd_cd=p.prd_cd) AS n_bdl,
  (SELECT count(*) FROM t_prd_product_page_rules x WHERE x.prd_cd=p.prd_cd) AS n_page,
  (SELECT count(*) FROM t_prd_product_addons x WHERE x.prd_cd=p.prd_cd) AS n_addon,
  (SELECT count(*) FROM t_prd_product_price_formulas x WHERE x.prd_cd=p.prd_cd) AS n_priceformula
FROM t_prd_products p
LEFT JOIN t_prd_product_categories pc ON pc.prd_cd=p.prd_cd AND pc.main_cat_yn='Y'
LEFT JOIN t_cat_categories mc ON mc.cat_cd=pc.cat_cd
ORDER BY main_cat, p.prd_cd;
```
