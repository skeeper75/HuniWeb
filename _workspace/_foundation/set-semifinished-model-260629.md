# 반제품/셋트 레이어 분석 정본 (§23 입력·A codify 선행)

> 2026-06-29. 사용자 우려("반제품 미분석 채 접근하면 재작업")에 따라 셋트/반제품 레이어를
> 공식집·라이브로 정밀 분석. 권위=상품마스터 공식집. 라이브 읽기전용 실측.
> ★사용자 도메인 확정: **면지=제본비 포함(가격0)이되 생산에 어느 면지인지 "연결"은 필요(택1).**

## 1. 셋트 2레이어 가격 모델 (확정)

셋트 가격 = **부모 셋트공식(권당) + Σ 반제품 구성원(각자 qty)** = `evaluate_set_price`.

| 레이어 | 무엇 | 가격 모델 | 생산연결 |
|---|---|---|---|
| **부모 셋트공식** | 표지+제본(+면지) 합산·권당 | 공식(prc_typ·표지타입 opt_cd 차원) × 부수 | — |
| **내지 구성원** | 속지 | **공식O · page파생**(부수×⌈page/판걸이수⌉) | 종이 선택 |
| **표지 구성원** | 겉표지 | **공식X · 가격0**(비용=부모공식·표지타입이 부모 차원) | **택1 선택**(전용지/레더/소프트…) |
| **면지 구성원** | 속지 사이 색지 | **공식X · 가격0**(비용=제본비 포함) | **택1 선택**(화이트/블랙/그레이/인쇄) |

핵심: **표지·면지 구성원은 "가격"이 아니라 "선택지(생산연결)"** 다. 내지만 별도 가격(page파생). 부모공식이 표지인쇄+표지코팅+제본+면지비를 권당으로 합산.

내지 page공식 실측:
- 하드커버 내지(284) `PRF_DGP_INNER` = 용지비[plt_siz_cd,mat_cd] + 디지털인쇄비[proc_cd,plt_siz_cd,print_opt_cd] (장당·page→장수 환산).
- 포토북 내지(101) `PRF_PHOTOBOOK_INNER` = 포토북 추가2P당(사이즈별)[siz_cd,min_qty].

## 3. 구조 결함 종합 (재작업 방지 위해 선해소)

1. **면지/표지 택1 미모델** — 3색 면지가 "항상 포함 3구성원"(min/max 비어있음). 가격0이라 금액무해하나 **생산연결(어느 면지)이 택1로 안 잡힘**. → 셋트 구성원에 택1 의미(또는 옵션그룹) 부여 필요.
2. **부모공식 미보유 3**(077·082·088) — 표지+제본 합산 부모공식 신설(072 패턴·면지=제본포함).
3. **내지 구성원 누락**(082·077) — 내지 반제품 승격·page공식 바인딩.
4. **셋트 미구성 4**(068~071) — 셋트 신규 구성(가장 큰 저청구).
5. **엽서북/떡메 내지 공식X**(095·098) — 엽서북 page(20/30장) 가격 차이 반영 점검.

## 4. A codify(공식집-정합 검증)에 주는 함의 — 3-tier 필수

A 검증기는 반드시 **3계층 분기**로 짜야 재작업이 없다:

| 계층 | 검증 기준 |
|---|---|
| **단순 완제품(.01 non-set)** | 공식형=구성요소 합산 정합 / 가격포함=직접단가 적재 / 기성=템플릿 |
| **셋트 완제품(.01 set)** | 부모 셋트공식 존재 + 구성원 완전성(내지·표지) + 면지/표지 택1 연결 + 2레이어 합산 무중복 |
| **반제품(.02)** | 역할별: 내지=page공식 정합 / 표지·면지=가격0·생산연결(부모 spec로 대조 금지=이번 false-positive 원인) |

→ 이 모델로 A를 codify하면 책자 결함(068~071·077·082·088)과 면지 택1까지 한 틀에서 검증·교정 가능. **§23(반제품/셋트 설계·적재) → A codify(3-tier 검증)** 순서가 재작업 없는 길.
