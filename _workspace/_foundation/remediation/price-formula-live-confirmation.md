# 계산공식 라이브 역확인 — off-grid 수량·사이즈 probe (2026-06-28)

> 운영 중인 구 사이트(huniprinting.com)에 **가격표에 없는 수량(95·97)·임의 사이즈**를 넣어
> 실제 계산공식을 역확인. 목적=단가/합가 공식을 정답으로 **확정**(특히 합가 오접근 방지).
> 위상=라이브=운영 정답 공식 오라클. 방법=goods.asp 폼에 off-grid 값 입력→공급가 관찰.

## 확정된 4대 공식 규칙 (라이브 실증)

### 규칙 1 — 자유수량 단가형 = 공급가 = 단가 × 수량 (완전 선형)
가격표에 없는 수량 95·97을 넣어도 **정확한 배수** = 구간·할인 0의 순수 선형.
| 상품 | 단가 | qty=1 | qty=95 | qty=97 | qty=100 |
|---|---:|---:|---:|---:|---:|
| 아크릴 키링(50x50 은색) | 6,000 | 6,000 | **570,000** | **582,000** | 600,000 |
| 현수막(규격 90x500) | 29,000 | 29,000 | **2,755,000** | **2,813,000** | 2,900,000 |
→ 95×단가·97×단가 정확 일치. **자유수량 상품(아크릴 굿즈·실사 규격)은 ×수량 선형**(밴드/할인 없음).

### 규칙 2 — per-unit 합가 = base + 옵션 가산 (단순 덧셈·이중합산 없음)
아크릴 키링 50x50 qty1 옵션 분해:
| 구성 | 공급가 | 가산 |
|---|---:|---|
| 면적 base(고리없음·볼체인없음) | 5,000 | base |
| + 고리 은색 | 6,000 | +1,000 |
| + 고리 금색 | 6,100 | +1,100 |
| + 고리은색 + 볼체인 | 6,900 | 볼체인 +900 |
→ per-unit = **면적 base + 고리 + 볼체인**(additive). 합가=단순 덧셈, **이중합산 흔적 0**. 그 뒤 ×수량.

### 규칙 3 — 면적 사이즈 = 격자 조회(preset cell), off-grid = 견적문의 (연속 보간 아님)
- 즉시가는 **preset 사이즈만**: 아크릴=10mm 격자 select(20~100)·실사=규격 드롭다운(90x500·A1/A2)·아크릴판넬=A1/A2.
- **임의 사이즈(비규격)는 즉시가 미노출 → 견적문의**(현수막 비규격=고객 문의·게시판 "비규격현수막문의" 실재).
→ ★면적 공식은 **격자 lookup**이지 연속 area×rate 보간이 아니다. **off-grid는 ceiling 또는 견적**(임의 단가 생성 금지). 우리 DB 면적격자 + off-grid ceiling 모델이 정합([[dbmap-price-formula-types-authority]]).

### 규칙 4 — 밴드수량 상품 = 밴드 lookup(×qty 아님)
디지털인쇄·스티커·명함·봉투·엽서는 **수량이 SELECT 밴드**(명함 100/200/.../1000·엽서 15/30/50·스티커 4/8/12장·봉투 1000/2000). off-grid 수량(97)은 **손님이 입력 불가**(밴드가 양자화). 가격=밴드별 L2 선조립표 조회.
→ ★밴드수량 상품을 ×qty 선형으로 모델링하면 **오류**. 반대로 자유수량 상품을 밴드로 보면 오류.

## ★단가 vs 합가 — 오접근 방지 정답 (사용자 관심)
| 유형 | 수량 모델 | 가격 공식 | 오접근 위험 |
|---|---|---|---|
| 자유수량 단가형(아크릴 굿즈·실사 규격) | 자유입력(text) | **단가 × 수량**(선형) | 밴드로 오모델→오류 |
| 자유수량 합가형(아크릴 addon) | 자유입력 | (면적 base + 옵션 가산) × 수량 | 옵션 가산 누락=저청구·이중=과대 |
| 밴드수량 L2 선조립(디지털·명함·봉투·스티커·엽서) | SELECT 밴드 | 밴드별 완성가 lookup | ×qty 선형 오모델·L1 단가 또 더함=이중합산 |
| 면적 격자 | preset 사이즈 | grid cell lookup(off-grid=ceiling/견적) | 연속 보간 오모델·임의단가 |

→ **합가 오류 핵심 = ① 밴드수량을 ×qty로 본다 ② L2 선조립표인데 L1 단가를 또 더한다(이중합산) ③ 면적을 연속 보간한다.** 셋 다 라이브가 정답을 보여줌(밴드=select·L2=권위셀 일치·면적=격자/견적).

## off-grid probe 절차 (헬퍼 추가)
```
# 자유수량 단가 검증: qty=text 상품에 95·97 입력 → 단가×수량 일치 확인
$B fill 'input[name=qty]' "97"; $B js "setActionCheck()"; <공급가 읽기>   # = 단가×97 이어야
# 합가 분해: 옵션 select를 하나씩 토글하며 per-unit 가산액 측정
# 면적 off-grid: 자유 size 상품(실사 비규격)은 견적문의 확인(즉시가 0=정상·임의단가 금지)
```
※ live JS는 select 'change' 레이스 주의 — `$B select`/`$B fill`(네이티브 이벤트) 사용·setActionCheck 후 sleep.

## 결론
- **단가형은 예상대로 단순 ×수량**(라이브 95/97 정확 배수로 확증).
- **합가형 정답 확정**: per-unit additive(이중합산 0)·밴드수량은 lookup(×qty 아님)·면적은 격자(보간 아님·off-grid=견적).
- 우리 DB가 **자유수량/밴드수량 구분 + L2 선조립 verbatim + 면적격자(off-grid ceiling/견적)**를 지키면 오차 0. 위반 3종이 합가 오류의 근원.
