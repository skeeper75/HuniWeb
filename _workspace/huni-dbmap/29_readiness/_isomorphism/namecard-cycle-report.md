### 1-1. 1차 합산형 vs 2차 고정가형 — 자율 검증 견고성

| 항목 | 1차 엽서(합산형) | 2차 명함(고정가형) |
|------|------|------|
| 가격 자율 산출 1차 결과 | V3 REFUTED(순환 골든·출력매수 변환 누락) → 보정 폐루프로 정정 | **결함 없이 NO-GO 정직 판정**(NAMECARD-WIRE) |
| 함정 원인 | 출력매수 변환(`ceil(주문수량/판걸이수)`) 누락 → 인쇄비 3배 과대 | (구조적으로 없음) 고정가형은 ×수량 안 함·단가행=세트당 총액 |
| 골든 | 최종가 골든 부재(component 단가표만) | **단가행=최종가**(가격표 명함포토카드 셋당 총액이 곧 골든·STD 3,500 직접 대조) |
| 검증 견고성 | 보정 필요(생성≠검증 폐루프 작동) | 1패스 견고(고정가형 단순성) |

→ **고정가형은 골든이 가격표에 직접 존재**(명함포토카드 셋당 총액)해 합산형의 "최종가 골든 부재" 문제가 없고, 출력매수 변환 함정도 없어 **자율 가격 검증이 합산형보다 신뢰도 높다**. 파이프라인이 두 공식 유형에서 모두 작동하되, 고정가형에서 더 견고.

### 3-2. Sp 가능 판정 = **선결 후 가능 (NAMECARD-WIRE 1건이 핵심 선결)**

- **고정가형 Sp의 특징(합산형과 다름):** 합산형 엽서는 동형 나머지(019~022)가 **CPQ option_items 미적재**(0행)라 적재가 선결이었다. 명함은 **3상품 다 CPQ 완비**(items 30/8/11)이고 **가격 배선만 선결**(NAMECARD-WIRE). 즉 명함 Sp는 "CPQ 적재"가 아니라 "**명함종 comp 배선**"이 병목.
- **즉시 전파 가능:** 033(STD)은 단순 부분집합·STD 배선 완비 → 종단 거의 완료(MATGROUP 단가행 복제 1건만).
- **선결 후 전파:** 031(PREMIUM)·032(COAT)는 명함종 comp 배선(NAMECARD-WIRE)이 선결 → 배선 COMMIT 후 즉시 종단.

**batch 단위 = 명함 고정가형 클래스(3상품)**, 전파 방식:

| batch 그룹 | 상품 | 전파 방식 |
|-----------|------|----------|
| B-STD | 033 | STD 배선 완비 + MATGROUP 단가행 복제(dbm-batch-load gen_batch_upsert·멱등 NOT EXISTS) |
| B-WIRE | 031·032 | ⓑ 공식분리(PREMIUM/COAT) + 바인딩 UPDATE + 명함종 comp 배선. 데이터 실재·배선만 |

**선결 조건(Sp 진입 전):**
1. **NAMECARD-WIRE 배선**(round-2 공식분리 결정 + round-16 배선) — 031/032 가격 정확화. **제안본+게이트 GO 존재**(실 COMMIT만 인간 승인).
2. MATGROUP 단가행 복제 — 033 5소재 완결.

> **결론: Sp는 명함 클래스 3상품 모두 CPQ·차원환원 완비라 "전파 인프라"는 입증 완료. 가격 배선(NAMECARD-WIRE) 1건이 핵심 선결이며, 이미 제안본+독립 게이트 GO 상태 → 실 COMMIT 1건이면 B-WIRE(031/032) 즉시 전파 가능.** 합산형 엽서(CPQ 미적재 선결)보다 Sp 준비도가 높다(명함은 CPQ 다 됨·가격 배선만).

### 3-3. Sp 확장 후보 (현 클래스 외)

미출시 명함종(펄 034·투명 039·화이트 040)은 가격 comp(PEARL/WHITE/CLEAR) 단가행 이미 실재 → 출시(CPQ 적재)+NAMECARD-WIRE 배선 패턴 재사용으로 빠른 클래스 확대. 포토카드(PRF_PHOTOCARD_FIXED)는 별 클래스지만 고정가형 종단기법(명함 사이클) 그대로 적용 가능.

## 4. ★ 고정가형 일반성 입증 (round-21 2차 사이클 핵심)

### 4-1. Q-게이트가 고정가형에서 올바로 작동한 증거

| 검사 | 합산형 가정(틀린 적용) | 명함 Q-게이트 실제(올바름) | 입증 |
|------|------|------|------|
| 수량 처리 | ×수량(합산형 인쇄비) | **×수량 안 함**(R42 세트당 고정총액·STD 100매=3,500 그대로) | C1 보정0 GO |
| 가격사슬 검사 | "comp 다 있나"(합산형 단가행 존재) | "**내 명함종 comp가 배선됐나**"(고정가형 택1 룩업) | NAMECARD-WIRE 포착 |
| 결함 유형 구분 | prc_typ 메타 오적재(D-1b·합산형) | **순수 배선 누락**(.03 부재·메타 정합·§2-5) | 메타 vs 배선 구분 |
| 거짓 GO 함정 | comp 있으면 합산해 "가격 나옴" | **032 STD로 떨어진 3,800을 오산으로 차단** | C3 거짓 GO 차단 |

### 4-2. 한 줄 입증

**round-21 2차 사이클은 동일 Q-게이트가 공식 유형(1차 합산형 PRF_DGP_A → 2차 고정가형 PRF_NAMECARD_FIXED)을 넘어 일반화됨을 입증했다 — 합산형 가정(×수량·comp 합산·prc_typ 메타 오적재)을 적용하지 않고 라이브 공식 정의를 무가정 실측해 "고정가형은 명함종 comp 택1 룩업·×수량 안 함·단가행=세트당 최종가"임을 확인하고, Q4에서 배선 누락(NAMECARD-WIRE)을 합산형의 단가행 누락·메타 오적재(D-1b)와 정확히 구분해 잡아냈으며, Q5에서 032가 STD로 떨어진 3,800을 "가격 나옴"으로 오판하지 않고 오산(과소)으로 정직 차단(거짓 GO 방지)했다 — 게다가 고정가형은 단가행이 곧 골든이라 1차 합산형의 출력매수 변환 함정·최종가 골든 부재가 구조적으로 없어 자율 가격 검증이 1차보다 견고했다.**
