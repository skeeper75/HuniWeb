## 질문별 codex 판정

### Q1. 설계 건전성·엔진 정합 — **결함** (확신도 High)
- 권위 구조 자체는 정합: 대형=동판비 B01 면적매트릭스 + 일반/특수 B03/B05 등급×수량표, 소형=B01 5,000 + B03/B05.
- 박가공비를 proc_cd 단가행으로 일반/특수 중 한쪽만 매칭시키는 설계는 엔진이 먹는다(`pricing.py:41 NON_QTY_DIMS`·`:94`).
- ★**치명 결함(codex 신규 발굴):** 동판비 use_dims가 대형 `[siz_width,siz_height]`·소형 `[]`라 **proc_cd 게이트가 없다** → 박을 안 골라도 본체 공식에 들어간 순간 항상 매칭. `proc_grp:PROC_000033`은 코드상 매칭 조건이 아니며, proc_cd 없으면 `_evaluate_formula`의 공정별 평가 대상도 아님(`pricing.py:675`). 동판비에도 proc_cd 게이트 필요.

### Q2. 돈크리티컬 — **부분 정합·setup 게이트 결함** (확신도 High)
- 동판비 `.03` ×qty 폭발 없음(`pricing.py:201`=금액 그대로).
- 박가공비 `.02 + min_qty` ×qty 폭발 없음(`pricing.py:205`=`unit_price / tier_min_qty * qty`·수량=구간하한이면 표 금액 그대로).
- addon 템플릿 부결 타당(`pricing.py:441`=`unit_price * qty` 단일 경로·75,000→1,000매면 75,000,000 폭발).
- 단, `.03` setup이 선택 게이트 없이 본체 공식에 직접 들어가면 ×qty 폭발은 없으나 **박 미선택 주문에 5,000 또는 면적별 동판비가 붙는 상시 과금** 발생.

### Q3. 면적→등급 환산 — **현재 미지원 확정**(조사필요 아님) (확신도 High)
- siz_width/siz_height ceiling tier는 지원(`pricing.py:165`).
- grade 차원·면적→A~E 변환은 NON_QTY_DIMS/TIER_DIMS에 **없음**(`pricing.py:41`).
- dim_vals는 임의 키 정확매칭만(`pricing.py:103`) — 앱이 `grade=C` selection을 주입하면 등급별 단가행 매칭은 가능하나, 엔진이 90×90→C를 계산하지는 않음.
- 설계가 "앱 계산 / C트랙" 분리한 것은 타당. 단 가변 면적 상품은 트랙 완료 전 적재하면 견적 누락/0원 위험.

### Q5. 놓친 결함·false-positive
- **놓친 결함 1(신규):** 동판비 setup에 proc_cd 게이트 없음 → 박 미선택 상시 과금.
- **놓친 결함 2:** 문서 내부 stale 표기 — C-3/C-5 표는 박가공비 `PRICE_TYPE.01`이라 쓰나 바로 아래 가드는 `.02`로 정정. 구현자가 표만 보면 ×qty 폭발.
- **놓친 결함 3:** `proc_grp:PROC_000033`은 코드상 매칭 차원이 아님 — 실제 차단력은 proc_cd·dim_vals에만 있음.
- **false-positive 없음 / G6 처리 타당:** 라이브 `COMP_NAMECARD_FOIL_S1_STD` 1000구간=63,000·setup 별도 5,000·권위 소형 일반 E 1000 M18=64,000 → **1,000원 단일 셀 갭**으로 보는 것이 맞음.
