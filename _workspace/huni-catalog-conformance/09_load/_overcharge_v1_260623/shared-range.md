## C-1 — PRINT_OPT 코드 실재 (★명세 가정 정정)

명세는 `PRINT_OPT.01/02`(t_cod_base_codes)로 가정했으나 **라이브 실측 결과 다름**:

- `t_prc_component_prices.print_opt_cd` FK → **`t_prt_print_options`** (`fk_compprice_prtopt`), NOT `t_cod_base_codes`.
- 실재 코드 (use_yn=Y·del_yn=N, mint 불요):
  - **`POPT_000001` = 단면** (print_side=단면) → S1 단가행에 충전
  - **`POPT_000002` = 양면** (print_side=양면) → S2 단가행에 충전
- 라이브 이미 사용 중: POPT_000001=689행·POPT_000002=742행 (검증된 코드).

→ **C-1 GO**: 코드 실재·판별 명확(단면/양면). 거버넌스 §12 코드등록 **불요**. 명세의 `PRINT_OPT.01/02`는 placeholder였음 → 실 코드 `POPT_000001/002` 사용.

## 공유범위 — 명함 `COMP_NAMECARD_STD_S1/S2`

| 항목 | 실측 |
|------|------|
| 배선 공식 | `PRF_NAMECARD_FIXED` **단 1개** (다른 공식 0) |
| 공식 바인딩 상품 | **PRD_000031(프리미엄명함)·PRD_000032(코팅명함)·PRD_000033(스탠다드명함)** — 3개 |
| 공식 comp 전체 | S1·S2 **2개뿐** (판별 페어 그 자체) |
| dim_vals | 없음 (`<null>`) — 매칭 복잡도 0 |

**판정: 안전(GO).** 공유 3상품 **전부 명함**(동일 상품군) — 단면/양면 택일은 세 상품 모두에 의미축 정합. 단가행에 print_opt_cd 충전은 세 상품에 **동일하게 올바른** 교정. ★작업지시는 031/032만 언급했으나 **033도 같은 comp를 공유**하므로 한 번 교정에 033도 동시 해소(정합·누락 아님).

→ **무관 상품 공유 0.** COMMIT 안전.

## 공유범위 — 094 `COMP_PCB_S1_20P/S2_20P`

| 항목 | 실측 |
|------|------|
| 배선 공식 | `PRF_PCB_FIXED` **단 1개** (다른 공식 0) |
| 공식 바인딩 상품 | **PRD_000094(엽서북)** — 1개 (전용) |
| 공식 comp 전체 | S1_20P·S2_20P **2개뿐** |
| dim_vals | 없음 (`<null>`) |
| 30P variant | `COMP_PCB_S1_30P/S2_30P` 단가행 실재하나 **어느 공식에도 미배선**(도달불가) → 본 교정 범위 밖(라우팅만·별건) |

**판정: 안전(GO).** 094 **전용** comp — 무관 상품/타 책자 공유 0. use_dims 변경이 094만 영향.

→ **무관 상품 공유 0.** COMMIT 안전.
