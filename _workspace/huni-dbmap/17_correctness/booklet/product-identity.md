## 2. 핵심 정체 발견

### F-ID-1 [정체 정합 — 책자 family는 전부 일반 인쇄물 책자] ✅ (의심 반증)
- 책자 10 완제품은 전부 **CAT_000006 책자 트리** 또는 **CAT_000104 하드커버 / CAT_000026 엽서북 / CAT_000124 노트** 하위 범주. digital-print(인쇄배경지=포장재 012 오분류)·goods-pouch 같은 **정체 오분류는 책자에 없다.** 책자는 전형적 일반 인쇄물이라 사이트 재확인 불요(product-master 책자 트리 + 라이브 카테고리 트리 정합으로 충분).
- **단 카테고리 연결 깊이는 불일치(F-ID-5)** — 정체(범주)는 책자가 맞으나, 068~071은 **잎노드(전용 lvl2 노드)가 아닌 lvl1 책자에 직결**돼 있다(F-GATE-BK-2 보정). 정체 오분류는 아니지만 연결 위계 결함.
- **단 생산방식 3구조(A 통합·B 셋트·떡제본)는 정체의 핵심** — 표지/면지/링/D링이 자재 usage 슬롯·반제품 sets로 정확히 표현돼야 한다(C2 추출규칙 전제). 이 구조 자체는 라이브에 잘 적재됨(sets 21행·usage 슬롯).

### F-ID-5 [카테고리 연결 위계 결함 — 전용 잎노드 고아] 🟡 (F-GATE-BK-2 신규)
- **무엇:** 책자 완제품이 **잎노드(전용 카테고리)가 아닌 상위 중간노드에 직결**돼, 그에 대응하는 전용 노드가 상품 0(고아)이 된다. digital-print F-GATE-1(배경지가 정상 잎노드 273/274 대신 잉여 고아 296에 연결)과 **동형 결함**.
- **라이브 실측(읽기전용):**
  - `SELECT pc.prd_cd, cc.cat_cd, cc.cat_nm, cc.cat_lvl FROM t_prd_product_categories pc JOIN t_cat_categories cc ON pc.cat_cd=cc.cat_cd WHERE pc.prd_cd IN ('PRD_000068','PRD_000069','PRD_000070','PRD_000071')` → 전부 **`CAT_000006 책자 | lvl1`** 직결.
  - `SELECT cc.cat_cd, cc.cat_nm, count(pc.prd_cd) FROM t_cat_categories cc LEFT JOIN t_prd_product_categories pc ON cc.cat_cd=pc.cat_cd WHERE cc.cat_cd IN ('CAT_000100','CAT_000101','CAT_000102','CAT_000103','CAT_000105','CAT_000106','CAT_000107') GROUP BY 1,2` → **CAT_000100 중철책자·101 무선책자·102 PUR책자·103 트윈링책자·106 레더하드커버책자·107 하드커버링책자 = 전부 상품 0(고아)**, **CAT_000105 하드커버책자 = 22 연결**.
- **함의:** 전용 잎노드 6개(100~103·106·107)가 v03 정규화로 생성됐으나 상품이 안 걸렸다. 068~071은 lvl1 책자에, 077·082는 lvl3 하드커버책자(CAT_000105)에 잘못 묶임. 정상 잎노드가 이미 실재(search-before-mint = 신설 0) → 교정은 상품을 전용 잎노드로 재연결 + 빈 고아 노드 논리정리. → **BK-CAT [correction-manifest]**.

### F-ID-2 [레더 링바인더(PRD_000088) — 제본·공정 전무] 🟡
- **무엇:** 088은 L1 C31(제본) 빈값 = 바인더링(D링)이 제본 결합을 대신한다(round-11 BK-2, round-12 C4 확정). 라이브 088 공정 적재 **0행** — 제본뿐 아니라 **수축포장(PROC_000076)도 없다.**
- **비교:** 다른 책자 완제품 8/10이 수축포장 보유(068 중철·097 떡메모지는 L1 anchor 개별포장 빈값이라 미적재 — 정당). 088은 L1 R55 개별포장 빈값 + 제본 빈값 → v03이 088 공정 행을 **하나도 생성 안 함.**
- **정체 함의:** 088은 "제본 없는 바인더"라는 정체가 맞으나, 그렇다고 공정 0행이 옳은지는 별개(수축포장·재단 등 후공정 존재 가능). product-bom §8이 "공정 = 표지(레더 특수인쇄)→면지 부착→D링 바인더 결합" 🟡로 남긴 영역. **컨펌 Q-BK-D**(088 후공정 적재 여부).

### F-ID-3 [떡메모지(PRD_000097) — 카테고리 2중 연결] 🟡
- **무엇:** 097이 `CAT_000129 떡메모지`(lvl3) + `CAT_000124 노트`(lvl2 중간노드) **둘 다** 직접 연결. 다른 책자는 lvl 단일 노드 1개씩.
- **함의:** lvl2 중간노드(노트)에 상품을 직접 거는 것은 잉여(자식 lvl3 노드 떡메모지가 정답 잎). main_cat_yn 확인 필요(둘 중 어느 게 주카테고리인지). EXTRA 후보(논리정리).

### F-ID-4 [잉여 고아 카테고리 노드 — CAT_000297 레드프린팅 책자 가이드] 🟡
- **무엇:** `CAT_000297 레드프린팅 책자 가이드`(upr_cat_cd=NULL·lvl3 고아)가 라이브에 실재. digital-print의 `CAT_000296 배경지`·`CAT_000295 상품권` 고아 노드와 **동형 결함**(L1 R85 URL 가이드행이 카테고리로 잘못 파생). 책자 상품은 이 노드에 연결돼 있지 **않음**(상품 오연결 아님) — 단순 잉여 고아 노드.

## 3. 정체 → 추출규칙 전제 요약

- **A 통합(중철/무선/PUR/트윈링)** = 내지(.01)+표지(.02) parent 자재 + 표지 공정(코팅/박/형압/투명커버). sets=0(반제품 없음). page_rule 활성.
- **B 셋트(하드커버/레더/하드링/레더바인더)** = 내지(.01) parent + 표지(.02) + 면지(.03) parent 자재 **+ sub_prd + sets 병행**(자재 권위=parent usage_cd, sub_prd는 빈 껍데기여야 정상). 링/D링=USAGE.07.
- **떡제본(엽서북/떡메모지)** = 엽서북=내지+표지 page 활성, 떡메모지=내지만·묶음수(권)·page 무의미.
- **레더 링바인더(088)** = 제본 family 미연결(D링이 결합), 후공정 적재 여부 컨펌.
- 책자는 정체 오분류 0(digital-print/goods-pouch와 다름). 결함은 **속성축 적재**(자재 오염·sub_prd 오적재·카테고리 잉여)에 집중.
