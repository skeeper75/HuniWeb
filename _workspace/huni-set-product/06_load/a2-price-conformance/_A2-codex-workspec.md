# A2 돈크리티컬 HIGH — codex 독립 재확인 work-spec

당신(codex)은 후니프린팅 가격 적재정합 검증의 **독립 2차 검증자**다. 아래 4건은 Claude 검증가가 라이브 DB 실측으로 적발한 "돈크리티컬 결함" 주장이다. **검증가 결론을 사실로 받아들이지 말고**, 권위 추출본 CSV(읽기전용)와 아래 박제된 라이브 raw 데이터를 독립 대조해, 각 결함이 **CONFIRMED(진짜 결함) / REFUTED(결함 아님·검증가 오판) / UNCERTAIN(추가증거 필요)** 중 무엇인지 판정하라. 가설을 사실로 단정 말고 근거를 들어라.

권위[HARD] = 인쇄상품 가격표 260527 + 상품마스터 260610. 단가값은 권위 verbatim이어야 한다.

## 결함 1 — [D-C1-02] 디지털 S1 흑백/칼라 축 오적재 (검증가 주장: 색상엽서 저청구)

**라이브 raw (Claude 실측, min_qty=1):**
- `COMP_PRINT_DIGITAL_S1`: 단면(POPT_000001)=3000·3500 / 양면(POPT_000002)=4000·4500
- `COMP_PRINT_DIGITAL_S2`: 단면(POPT_000001)=4000·5000 / 양면(POPT_000002)=6000·7000

**검증가 주장:** 색상엽서(PRF_DGP_A 10상품)는 흑백/칼라 선택축이 없는데, S1 단면에 흑백단가(3000)가, S1 양면에 칼라단면단가(4000)가 들어가 있다. 색상엽서는 단면=칼라단면(4000)·양면=칼라양면(6000)이 정답인데, 단면 주문 시 흑백3000으로 매칭돼 매당 −1,000(단면)/−2,000(양면) 저청구.
**판정 과제:** 권위 디지털인쇄비 시트에서 흑백단면/칼라단면/칼라양면 단가가 각각 얼마인지 확인하고, 라이브 S1 단가 배치(단면=3000,3500·양면=4000,4500)가 "흑백/칼라" 축인지 "단면/양면" 축인지 판정. 색상엽서에 흑백 단가가 적용되는 게 맞는가?

## 결함 2 — [D-C1-01] 디지털 S2(칼라양면 6000) 배선 0

**라이브 raw:** `t_prc_formula_components`에서 PRF_DGP_A~F 6공식 전부 `COMP_PRINT_DIGITAL_S1`만 배선. `COMP_PRINT_DIGITAL_S2`는 어느 공식에도 배선 0행(단가행 212개는 존재).
**검증가 주장:** S2에 진짜 칼라양면(6000)이 있는데 배선이 없어 양면 주문 시 도달 불가.
**판정 과제:** 양면 주문 시 칼라양면 단가가 S1으로 표현 가능한가, 아니면 S2 배선이 반드시 필요한가? (결함1과 연계)

## 결함 3 — [C4-D01] PRF_BIND_SUM 제본 다종-1배선

**라이브 raw:**
- `PRF_BIND_SUM` 배선 comp = `COMP_BIND_JUNGCHEOL`(제본비 중철) **1개뿐**.
- `PRF_BIND_SUM` 바인딩 상품 = 무선책자(PRD_069)·중철책자(068)·트윈링책자(071)·PUR책자(070) **4종**.
- 라이브에 제본 comp 다 존재: `COMP_BIND_MUSEON`(무선)·`COMP_BIND_PUR`·`COMP_BIND_TWINRING`·`COMP_BIND_JUNGCHEOL`(중철).
**검증가 주장:** 4책자가 공유공식 PRF_BIND_SUM을 쓰는데 중철 제본단가만 배선 → 무선/PUR/트윈링은 제본비를 중철값으로 오청구하거나 0으로 누락(PUR ~5배 과소).
**판정 과제:** booklet-l1.csv에서 4책자 제본단가가 실제로 서로 다른가? 다르다면 단일 comp 배선이 오청구인가? 권위상 정답 배선 구조는?

## 결함 4 — [C3-D01] FOMEXBOARD 본체 단가 오적재 (과대청구)

**라이브 raw:** `COMP_POSTER_FOMEXBOARD_WHITE3MM`: A3(SIZ_000315)=**8500**, A2(SIZ_000317)=**13000**.
**검증가 주장:** 권위 포맥스보드 A3=8300·A2=11500인데 라이브가 +200/+1500 과대청구.
**판정 과제:** 권위 추출본에서 포맥스보드(화이트3mm) A3/A2 단가를 찾아 라이브 8500/13000과 대조. 검증가의 권위값(8300/11500)이 맞는가?

## 출력 형식

```
[결함N] 판정: CONFIRMED|REFUTED|UNCERTAIN
권위값(파일:행/셀): ...
라이브값: ...
근거: ...(왜 그렇게 판정했는가·검증가가 놓치거나 과장한 점 포함)
```
마지막에 4건 종합 한 줄(몇 건 CONFIRMED).
