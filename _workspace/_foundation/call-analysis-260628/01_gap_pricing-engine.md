## D2. 필수 공정 자동 가격 반영

### 코드 근거
- **플래그 실재**: `models.py:451` `mand_proc_yn = CharField(max_length=1, db_comment='필수공정여부')` (테이블 `t_prd_product_processes`). → SOT "필수공정여부 플래그 이미 구현됨"과 정합 **확인**.
- **엔진(pricing.py) 자동가산 로직**: **미발견**. `evaluate_price`/`_evaluate_formula`는 `proc_sels`(호출자가 넘긴 공정 목록)만 평가(`pricing.py:694-705`). `mand_proc_yn`를 읽어 자동 포함하는 코드가 pricing.py에 0건.

- **POST 핸들러**: `price_views.py:1769-1781` — `proc_sels`는 전적으로 클라이언트 POST body `p.get("procs")`에서 옴. **서버가 필수공정을 강제 주입하지 않음**.
- **자동선택의 실제 위치 = 프론트 JS**: `price_simulator.html:325` `PROCSEL=(PROC_OPTS||[]).filter(o=>o.mand).map(o=>({proc_cd:o.v}))` — 필수공정을 초기 선택·계산에 포함. `:381-383`은 필수공정을 "필수" 배지+제거불가로 잠금. 셋트도 동일(`:626`).

→ **검사/인코딩 위치**: §18 설계(`hpe-engine-design`)에서 "엔진이 상품의 `mand_proc_yn=Y` 공정을 호출자 미제공 시 자동 주입"을 엔진 계약으로 승격(또는 §7 적재에서 모든 호출 경로가 필수공정을 채우도록 강제). 패키지 내재공정 제외는 §18 설계에서 comp 모델로 정의.
