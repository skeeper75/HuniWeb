### 1.1 권위 재확인 (verbatim)

### 1.2 엔진 합가형 의미 (pricing.py verbatim)
- `component_subtotal`(:185): `.02 → unit_price ÷ tier_min_qty × qty`. `tier_min_qty`=매칭행 min_qty(:174). base≤0 → `ValueError`(:188).
- 수량→구간(:42·:163): min_qty='이상' 하한, **주문 이하 최대 임계**. 예 주문2 → min_qty {1,3} 중 ≤2 최대 = 1.
- 한 comp는 자기 행을 min_qty 티어로 단일 매칭. 여러 comp는 `_evaluate_formula`가 합산(:401).

### 1.3 사용자 확정 구조 → 엔진 표현 (3안 비교)
사용자 directive: **1~2장 = 2,000원(기본) · 3장부터 = 3장마다 4,000원**.

| 안 | 표현 | 1~2장 | 3장 | 6장 | 적합 |

| **A: 단일 comp .02 2행** | COMP_STK_TATTOO(.02): min_qty=1·up=2000 / min_qty=3·up=4000 | 주문2→tier1→2000÷1×2=**4000** ❌ | tier3→4000÷3×3=4000 ✅ | 4000÷3×6=8000 ✅ | ❌ 1~2장 틀림(2000÷1×2=4000) |
| **B: 단일 comp .02·base 매칭폭** | min_qty=1·up=2000(1~2 의도) | 동일 ❌ ×수량 발현 | | | ❌ 동일 결함 |

### 1.4 권고 — 안 C (qty_incr=3 활용) + base 잔여 컨펌
- **핵심: 상품 min_qty=3·qty_incr=3이라 위젯 정상 주문수량은 3,6,9…뿐**. 따라서 **3장부터 4000 = COMP_STK_TATTOO(.02·min_qty=3·4000) 그대로 정확**(이미 COMMIT됨·1.5 확인). 정상 경로에서 base 2000은 발현 안 됨.
- **base 2000(1~2장)의 위치 = 별도 경로 확정 필요**: ① 1~2장 = "샘플/소량 별 SKU"인가 ② 위젯이 min_qty=1 허용하고 sub-3는 flat 2000인가 ③ 단순 최소주문 안내가(사문)인가. 엔진 .02는 flat-총액(수량무관 2000)을 직접 표현 불가(항상 ×qty). → **flat 2000은 min_qty 구간 .02로 표현 불가** → 만약 1~2장 주문 허용·flat 2000이면 **별 처리 필요**(최소청구액 floor·앱 레벨·또는 별 base comp .02 min_qty=1 up=2000이되 위젯이 qty=1만 보내는 경우 2000 정확·qty=2면 4000).
- **정직 분류:** 사용자가 "1~2장=2000"을 명시했으나 **위젯이 1~2장 주문을 받는지**(상품 min_qty=3과 모순) 가 미해소. **min_qty=3 정상 경로는 GO**(base 무관). 1~2장 경로는 컨펌 Q-TAT-1.

### 1.5 라이브 현황 (이전 COMMIT 실측)

→ **3장당 4000은 이미 COMMIT·정확.** 본 지시의 추가분 = base 2000 처리만.
