## 축1 — SOT 일치: **깨짐 (불일치)**

- 권위 가격표 디지털인쇄비 축 = 인쇄방식 × 면(단/양) × **도수(흑백1도/칼라CMYK/별색)** × 출력매수 수량행.
- 라이브 인쇄비 구성요소 use_dims에 **도수/색 차원 없음.** 단가행도 흑백 열 값만 적재.
- codex가 명세 표를 토대로 권위 칼라 열을 **명시적으로 추정·재구성**:
  - 권위 흑백: `1→3000, 15→450, 20→400, 25→350, 200→130`
  - 권위 칼라: `1→4000, 15→900, 20→850, 25→800, 200→310`
  - 라이브 S1: 흑백 열과 일치, 칼라 열 없음
- **단서 조건 (codex 명시)**: "상품마스터가 실제로 '엽서는 흑백만 허용'이라면 이야기가 달라질 수 있다. 단 골든 케이스가 칼라단면을 요구하므로 현재 정보로는 SOT 불일치로 보는 게 맞다."

## 축2 — 공식↔구성요소 매핑: **부분 정합·계산 안전성 깨짐**

큰 틀(인쇄비/코팅비/용지비/후가공비/별색인쇄비)은 권위 산식 항목을 나눠 가짐 → 구조 유사. 단, 합산형(전 comp addtn='Y'·차감/택일/우선순위 장치 없음)이라:
1. **S1/S2 이중합산 위험** — S1=국4절 판형 / S2=작업사이즈. 호출측이 두 키 동시 제공+둘 다 매칭 시 인쇄비 2회 합산. 명세만으론 호출이 둘 중 하나만 주는지 모름 → "확정 결함까지는 아니나 방어장치 없음."
2. **유광코팅비 0행** — 유광 선택지 있고 권위 유광 매트릭스 존재인데 단가행 0 → 유광 선택 시 코팅비 미계산/매칭 실패. **고아/미적재 결함.**
3. **후가공 동시매칭 위험** — 오시1L/2L/3L·가변텍스트/이미지 1/2/3EA 별도 comp+합산형이면 배타 매칭 보장 필요. 줄수/개수 차원 분리 여부 불명 → 동시매칭 시 이중합산.

## 축3 — 차원 매칭: **가장 크게 깨짐 (1순위)**

필요 3원 정합 = 권위(인쇄방식×도수×면×수량) ↔ use_dims(동일) ↔ 단가행(도수/면/수량 조합별). 라이브는:
```
use_dims: 인쇄방식 × 판형/작업사이즈 × 면 × 수량
단가행: 흑백 열 값만 존재
누락: 도수/색
```
→ 칼라단면 vs 흑백단면을 가격적으로 구분할 축 없음. 칼라 선택도 흑백 단가행으로 떨어질 가능성 큼.

### 골든 산식 독립 재검산 (codex 직접 검증 — 일치)

- **PC-4 흑백단면**: 출력매수 25 → 흑백 25행 350 → 인쇄비 350×25=**8,750**. ✅
- **돈크리티컬 수치화 (codex)**: 칼라 20,000 vs 흑백 8,750 = **차액 11,250.** 라이브가 칼라에도 350 적용 시 칼라 200매 인쇄비만 **11,250원 과소산출.**
