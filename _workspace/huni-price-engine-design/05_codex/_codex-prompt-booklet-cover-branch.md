# 독립 설계 검토 요청 — 책자(booklet) 셋트상품 "표지 펼침/개별 분기" 가격계산 설계

당신은 인쇄 자동견적 가격엔진의 **독립 검토자**입니다. 아래는 후니프린팅(한국 인쇄소)의 책자류 셋트상품
가격공식 설계입니다. 이 설계가 "상품을 실제로 가격계산 가능하게 하는가"를 **당신 스스로 독립 판정**하세요.
다른 누구의 판정도 주어지지 않았습니다. 근거와 함께 당신만의 결론을 내리고, 모르면 모른다고 하세요.

★ 가장 중요한 것: **돈 크리티컬 결함** (저청구 undercharge / 과청구 overcharge / silent 합산 /
이중계상 double-count / 가격엔진 계약 위반). 가격이 0원이 나오거나 절반만 청구되거나 두 배 청구되는
경로를 집요하게 찾으세요. "그럴듯하지만 틀린" 설계를 잡는 것이 당신의 임무입니다.

## A. 가격엔진 계약 (evaluate_price / evaluate_set_price — 라이브 코드 pricing.py)

- `evaluate_price(prd_cd, selections, qty)`: 한 상품의 가격공식(frm_cd)에 묶인 가격구성요소(component)들을
  각 component의 단가(component_prices)로 룩업해 **합산**한다.
- 단가형(`.01`, prc_typ): `subtotal = unit_price × qty` (각 component에 같은 단일 qty를 곱함).
- 차원 매칭: 각 component는 `use_dims`(예: `[proc_cd, plt_siz_cd, print_opt_cd, min_qty]`)로 단가행을 찾는다.
  use_dims에 있는 축은 **선택값이 주입돼야 단가행 1개로 좁혀짐**. 주입 안 되면 여러 행이 와일드 매칭될 수 있음.
- `min_qty`(수량 TIER): '이상' 하한. 주문수량 이하 최대 임계 행 선택 (qty=50 → min_qty=50 행). **단가가
  수량 TIER에 따라 달라짐**(볼륨할인: 수량↑ → 부당단가↓).
- 종이류 인쇄 component(use_dims에 `plt_siz_cd` 있음): 출력매수를 `plate_qty = ceil(qty / pansu)`로 환산
  (한 판에 여러 장 앉히는 임포지션). 즉 **÷pansu(나눗셈) 메커니즘만** 존재.
- `evaluate_set_price(prd_cd, selections, copies)`: 셋트 완제품 = Σ(구성원 evaluate_price) + 부모공식
  evaluate_price + 할인. 구성원(sub_prd) 중 가격공식이 없으면(0행) 그 구성원의 가격 기여 = 0.
- 셋트 호출자가 구성원의 유효수량(member qty)을 직접 산출해 주입할 수 있다(호출자 산출 유효수량).
- `grep cover_mult|cover_sheets`로 라이브 코드 검색 시 **0건**(이 변수는 코드에 없음).

## C. 설계자의 핵심 설계 결정 (이걸 당신이 독립 재검토)

### C1. 표지 "펼침/개별" 분기 = cover_mult ∈ {1, 2}
- 책등(책 등짝)이 있는 제본(중철·무선·PUR·하드커버무선)은 앞표지+책등+뒤표지를 **한 펼침면**으로 출력 →
  표지 출력매수 배수 **cover_mult = 1**.
- 책등이 없는 제본(트윈링·하드커버링)은 앞표지/뒤표지가 **물리적으로 2장** → **cover_mult = 2**.
- cover_mult는 제본방식(proc_cd)에서 자동 파생(손님이 직접 선택 안 함). 링(021/024)이면 2, 아니면 1.
- 설계자 주장: 표지인쇄/코팅/용지 **단가는 출력 1매 기준**이라 ×2 곱해도 "이중계상 아님".

### C2. 분기 구현 = "출력매수 앱계산" (설계자 채택안)
- evaluate_set_price 런타임에서 `cover_sheets = copies × cover_mult` 계산.
- 부모공식의 표지인쇄(S1)·표지코팅(COAT)·표지용지(COMP_PAPER) component는 **1매 기준 단가 그대로 두고**,
  곱해지는 출력매수(qty 인자)만 cover_mult 반영.
- 설계자는 **표지 3개 component를 부모공식에 직접 배선**하고, cover_sheets를 그 component들의 qty로 쓴다.
- 설계자가 부결한 대안: ②sub_prd_qty=2(표지 구성원 가격공식이 0이라 가격에 안 닿음) / ③같은 comp 2회 배선(combo_key 충돌).

### C3. 두 개의 수량축
- cover_sheets(표지 출력매수) = copies × cover_mult.
- inner_sheets(내지 출력매수) = copies × ceil(page / pansu).
- copies(부수) = 제본비 min_qty TIER 조회·전 항 곱.

### C4. 표지 가격모델 2종
- **분해형(068~071)**: 표지인쇄(S1)+표지코팅(COAT)+표지용지(PAPER) 3개 component 별 배선. cover_mult를 cover_sheets에 적용.
- **통합형(072·077·082)**: COVERBIND(표지+제본 권당 통합단가). 072 COVERBIND는 "펼침 1매 전제"로 봄.
  082는 링(책등없음=2장)인데 COVERBIND를 072와 공유하면 ~50% 저청구라 보고, 별 COVERBIND_TWINRING 또는
  분해형 필요하다고 봄(미결 — 권위 표지단가가 1면기준인지 2면포함인지 확인 필요).

### C5. 077/082 내지 구성원 누락
- 077/082는 page_rule(내지 페이지 입력받음)이 있는데 셋트 구성원에 **내지(sub_prd)가 0행** →
  내지비가 합산 안 됨. 설계자는 072의 내지284(PRF_DGP_INNER) 패턴을 077/082에 복제(mint)하자고 제안.

### C6. proc_cd 고정 주입
- 071이 COMP_BIND_TWINRING(4 proc_cd 공유)을 쓰는데 proc_cd 미주입 시 4행 다중매칭 위험 →
  상품→proc_cd=PROC_000021(트윈링) 고정 주입. 082는 PROC_000024.

## D. 골든 케이스 (설계 공식으로 재계산해 검증)

- **G-068A** 중철·표지단면 백모120 무광·내지28p양면·100부 (cover_mult=1, cover_sheets=100):
  표지인쇄 350×100=35,000 + 표지코팅 500×100=50,000 + 표지용지 36.88×100=3,688 + 제본 700×100=70,000
  = **부모소계 158,688**. (+ 내지 별도)
- **G-068B** 068A에서 표지만 양면: 인쇄 700×100=70,000 + 코팅 1000×100=100,000 → 부모소계 243,688 (068A 대비 **+85,000**).
- **G-071** 트윈링·표지단면 아트150 무광·내지100p단면·50부 (cover_mult=**2**, cover_sheets=50×2=**100**):
  표지인쇄 350×100=35,000 + 코팅 500×100=50,000 + 용지 46.65×100=4,665 + 제본 1500×50=75,000
  = **부모소계 164,665**. (cover_mult 누락=×1이면 표지 3비목 절반 저청구)
- **G-077/082**: 설계자는 COVERBIND ×100 = 796,900을 부모로 가정했으나 라이브 077/082 공식은 0행(미바인딩).

## ★당신의 독립 판정 요청 (근거와 함께)

1. **이 설계로 068·069·070·071·072·077·082·094·100이 실제 가격계산이 되는가?** 견적 0원/누락이 나는 상품은?
2. **cover_mult ×2 모델이 위 가격엔진 계약(A)과 정합하는가?** evaluate_price는 한 component에 단일 qty를
   곱하고, 종이류는 plate_qty=ceil(qty/pansu) **나눗셈만** 한다. 표지를 부모공식에 직접 배선하면서
   cover_sheets=copies×cover_mult를 그 component의 qty로 주입하는 게 **현행 엔진에서 실제 가능한가?**
   부모공식 안의 다른 component(제본비=copies)와 표지(=cover_sheets) 둘이 **서로 다른 qty**를 요구하는데
   evaluate_price는 단일 qty만 받는다 — 이 충돌을 설계가 어떻게 푸는가? 안 풀리면 무슨 일이 생기나?
3. **cover_sheets가 단가 TIER(min_qty) 조회에도 영향을 주는가?** G-071에서 표지인쇄 단가를 100매(=50×2)
   기준으로 골랐다 — copies=50인데 표지 TIER를 cover_sheets=100으로 조회하는 게 맞나? 이게 "매수만 곱"인가
   "TIER 자체가 바뀌는 것"인가? 골든값에 영향이 있나?
4. **077/082가 미바인딩(공식 0행)이면 표지/제본/내지 중 무엇이 누락되나?** 설계자는 "내지 누락"을 강조하는데,
   당신이 보기에 더 선행하는/더 심각한 누락은? G-077 골든 796,900은 재현 가능한가?
5. **이중계상/silent 합산 위험**: ① 표지 ×2가 정말 이중계상이 아닌가? ② COMP_PAPER가 표지용지·내지용지
   양쪽에 쓰이는데 충돌은? ③ proc_cd 미주입 시 TWINRING 4행 다중매칭이 silent 합산을 일으키나?
6. **068~071 부모공식이 "제본비 1개 component만" 가졌다** — 표지/내지/용지가 전혀 없다. 이 상태로 견적하면?
7. 설계가 자기검증으로 놓친 **돈 크리티컬 경로**가 더 있나? (있으면 구체적으로)

각 항목에 PASS/FAIL/UNSURE + 근거. 특히 돈크리티컬은 "어느 경로에서 얼마나 저/과청구되는지" 명시.
