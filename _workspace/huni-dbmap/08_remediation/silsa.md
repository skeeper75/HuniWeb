# silsa(실사) 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 실사(대형) 상품군에
> **누락·오매칭·모호**가 생겼다. 본 문서는 그 결함을 *올바로 처리(수정)* 하기 위해,
> **라이브 Railway DB 직접 read-only SELECT를 권위**로 silsa 29상품을 종단 규명한다.
> digital-print 파일럿(`08_remediation/digital-print.md`)의 종단 흐름·원칙·산출형식을 템플릿으로 삼았다.
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본 `ref-*.csv` stale 가능).
> 추정·임의판단 0 — 라이브·엑셀 L1·도메인 KB·PDF 실재 근거만. 미지/추정은 가설+컨펌 플래그.
> INSERT/UPDATE/DDL 0. DB 미적재 유지(처리 *방향* 설계). `db railway`·비표준포트·JOIN KEY=`prd_nm`.
> EXTRA 삭제단정 금지. 숨김/미출시(`use_yn=N`)·행숨김=비활성 분류.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. **라이브 SELECT 수행 약 13회**(전부 read-only).

## ⚠️ 머리말 — 본 검증이 v2/직관 대비 뒤집은 가장 큰 사실 (권위 반전)

**실사 시트는 acrylic/digital-print 대비 적재 상태가 가장 양호하다 — 자재(MAT_TYPE.08)·실사전용 공정(족자/봉제/타공/부착)이 실제로 적재돼 있다. 결함은 "전무"가 아니라 "특정 공정·자재의 부분 누락 + 액자/보드류 자재 0행"에 집중된다.**

라이브 직접 확인 결과:
- **실사전용 공정이 도메인대로 적재됨(권위 반전 — 누락 아님):** 족자포스터=족자제작(PROC_000082), 일반현수막=타공+봉제+부착(079/080/081),
  배너류=타공, 패브릭류=봉제. KB §2-1 D-24/b.9 실사군 공정이 **실제 연결돼 있다.** digital-print류의 "공정 전무" 결함과 대비.
- **자재도 MAT_TYPE.08 실사소재로 정확 적재:** 인화지/매트지/PET/PVC/투명PVC/그래픽천/린넨/캔버스/레더/타이벡/메쉬/현수막천 — L1 소재명과 **1:1 정합**(MISMATCH 거의 없음).
- **그러나 자재 0행 5상품:** **폼보드·포맥스보드·프레임리스우드액자·레더아트액자·미니보드스탠딩**. 이들은 L1 `소재(필수)`가 공백(보드/액자 = 소재가 보드 자체)이라 자재 미적재 → **R-MAT 판정 필요**(보드를 자재로 볼지).
- **화이트별색·코팅 부분 누락:** 접착투명포스터 L1 `화이트별색=단면`인데 PROC_000008(화이트별색) 미적재(공정 0행).

**교훈(HARD 재확인):** 실사는 "도메인 이해가 반영된 적재"의 모범 사례에 가깝다(실사전용 공정·소재 정합).
따라서 실사 결함 처리는 **전면 재적재가 아니라 부분 보정**(자재 0행 5상품·화이트별색·추가옵션 거치대/족자고리)에 국한된다. 라이브로 봐야 "이미 잘 된 부분"과 "잔존 누락"이 갈린다.

silsa 시트 = **실사(대형 잉크젯) 출력물** — 포스터/패브릭/액자/족자/배너/현수막/시트커팅/스티커.

**핵심 도메인 캐스케이드(실사 특화):**
1. **실사 전용 공정 정합** — `오버로크/말아박기/봉미싱` = 봉제(PROC_000080 `유형` enum), `N구타공` = 타공(079 `구수`), `원형/사각족자` = 족자제작(082 `모양`), `양면테입/큐방/끈/각목` = 부착(081) 또는 추가. **이 매핑이 실제 DB에 반영돼 있음.**
2. **보드/액자 = 소재 공백** — 폼보드/포맥스/프레임리스/레더아트액자/미니보드는 L1 소재칸 공백(보드 자체가 소재). DB 자재 0행 = "공백→미적재" 결과. **보드를 MAT_TYPE.08 자재로 등록할지 판정 대상.**
3. **포맥스 A1 작업사이즈 공백** — L1이 보존한 false-MISSING 토대(report §④). A1행 숨김+회색 = 미생산 후보(L2 판정).
4. **추가옵션의 거치대/우드봉은 addon, 가공의 봉제/타공은 process** — 같은 "부가물"이 축이 갈림(거치대=완제 부속 addon, 봉제=가공 공정).

## ④ 결함 표 — 누락(MISSING)·오매칭(MISMATCH)·모호(CONFIRM) (도메인 렌즈, 라이브 재확인)

> 분류 기준: **MISSING** = 정규화 규칙(B)대로 엑셀 변환 시 적재돼야 하는데 라이브에 없는 행.
> **MISMATCH** = 잘못된 축/코드. **CONFIRM** = 도메인 의미 미확정(실무 컨펌).

### G-SL-1 [MISSING·MAJOR] 보드/액자 5상품 자재 0행

- **도메인 근거:** 폼보드·포맥스보드·프레임리스우드액자·레더아트액자·미니보드스탠딩 = 보드/액자 자체가 자재.
  L1 가공칸에 `블랙보드/화이트보드`·`화이트포맥스(3mm/5mm)` 등 보드 종류 명시(폼보드·포맥스). 자재 부재가 아니라 **소재칸 공백으로 미적재**.
- **라이브 재확인(SELECT #11):** 5상품 `t_prd_product_materials` **0행** 확정.
- **판정:** **MISSING**(보드/액자를 MAT_TYPE.08 또는 신규 자재로 등록 대상). 폼보드 블랙/화이트, 포맥스 3mm/5mm는 자재 variant.
- **처리방향(§5 R1):** 보드/포맥스/우드액자를 자재로 등록·연결. (가공칸의 `화이트포맥스(3mm)`가 자재 신호 — R-MAT-3 소재형 분해).
- **가설 플래그:** 액자(프레임리스/레더아트액자)는 "출력물+프레임 결합"이라 자재가 프레임인지 출력소재인지 컨펌.

### G-SL-2 [MISSING] 화이트별색 공정 누락 (접착투명포스터)

- **도메인 근거:** L1 접착투명포스터 `화이트별색(옵션)=단면`. R-PROC-3: 별색 = 공정(PROC_000007 family, 화이트=PROC_000008). 투명 소재에 화이트 베이스 인쇄.
- **라이브 재확인(SELECT #11):** 접착투명포스터(122) PROC_000008 **0행**(process 전무).
- **판정:** **MISSING 1**(화이트별색 공정). 투명/접착투명 계열 위젯에서 "화이트 인쇄" 옵션 누락.
- **처리방향(§5 R2):** PROC_000008(화이트별색) 적재 + `단면` 면 정보.

### G-SL-3 [MISSING] 코팅 공정 부분 누락

- **도메인 근거:** L1 코팅(옵션) 보유 상품 = 아트프린트·방수·접착방수·폼보드·포맥스·프레임리스·PET배너·미니배너(`무광/유광코팅`).
- **라이브 재확인(SELECT #6):** 아트프린트(118)·방수(120)·접착방수(121)·폼보드(129)·포맥스(130)·프레임리스(131)·PET배너(136)·미니배너(145) = 코팅공정(014/015) **적재됨(정합).**
  → **대부분 MATCH.** 단 아트페이퍼(119) 등 코팅칸이 `없음`인 상품은 비적재(정상).
- **판정:** 코팅은 대체로 **MATCH**(부분 누락 미발견). v2가 코팅 누락으로 봤다면 false → **라이브 정합 확인.**

### G-SL-4 [MISSING/CONFIRM] 추가옵션(거치대·족자고리·끈) addon 누락

- **도메인 근거:** L1 추가(옵션) = PET/메쉬배너 `실내/실외 배너거치대`, 족자포스터 `천정형고리`, 현수막 `끈/각목/큐방`, 행잉 `우드행거/우드봉`.
- **라이브 재확인(SELECT #10):** addon 적재 = 캔버스행잉(133)→우드행거(PRD_000014)·린넨우드봉족자(134)→우드봉(PRD_000013) **2건뿐.**
  배너 거치대·족자 천정형고리·현수막 끈/각목/큐방 = **addon 미적재.**
- **판정:** **MISSING**(거치대/고리/끈 addon 다수). 단 거치대·끈이 addon인지 가공(부착공정)인지 축 분기 → CONFIRM. 현수막 `끈/각목/큐방`은 부착 성격, `거치대`는 완제 addon 성격.
- **처리방향(§5 R3):** 거치대/우드봉/고리 = addon, 끈/타공/봉제 = process로 축 분리 후 적재.

### G-SL-5 [CONFIRM] 가공칸 봉제/타공/족자 = 공정 (적재 정합 확인)

- **도메인 근거:** L1 가공 `오버로크/말아박기/봉미싱` = 봉제(080), `N구타공` = 타공(079), `원형/사각족자` = 족자제작(082), `열재단/양면테입` = 부착/완칼.
- **라이브 재확인(SELECT #6):** 린넨패브릭(124)·캔버스패브릭(125)·캔버스행잉(133)·린넨우드봉(134) = 봉제(080) 적재. 배너/현수막 = 타공(079). 족자포스터 = 족자제작(082). 일반현수막 = 봉제+타공+부착. **= 도메인 정합(MATCH).**
- **판정:** **MATCH(권위 반전 — 누락 아님).** 실사 가공 공정은 도메인대로 적재돼 있다.
- **잔존:** 봉제 `유형`(오버로크/말아박기/봉미싱)·타공 `구수`(4/6/8) 등 **param 적재 여부**는 별도 확인 대상(공정행은 있으나 세부옵션 미확인 — 컨펌).

### G-SL-6 [MISSING] nonspec(사용자입력) 범위 전무

- **도메인 근거:** L1 포스터/패브릭류 다수 `사용자입력`·현수막 `5000x900`+사용자입력. R-SIZE-5: nonspec 대형 치수 범위.

## ⑦ 큰 그림 적용 (도메인 → 엑셀 → DB결함 → 처리 → 위젯)

- **도메인 KB(07_domain):** 봉제 `유형`·타공 `구수`·족자제작(b.9)·D-24 실사 후가공군·MAT_TYPE.08 실사소재·화이트별색(008)이
  "엑셀 오버로크/N구타공/원형족자/소재칸"을 옳게 해석하는 렌즈 → **실사는 이 렌즈대로 잘 적재된 모범 사례임을 확인**하고, 잔존 누락(보드자재·화이트별색·거치대)만 결함으로 분리.
- **엑셀 L1(무손실):** 전 컬럼 보존으로 보드 종류(블랙/화이트포맥스)·화이트별색·거치대 종류·포맥스 A1 작업사이즈 공백·★숨김을 정확 추출(추정 0).
- **DB 라이브 권위:** 라이브 13 SELECT로 "이미 정합한 부분(족자/봉제/타공/소재)"과 "잔존 누락(보드자재 5·화이트별색1·addon·nonspec)"을 갈라냄. **실사는 권위 반전이 false-누락 철회 방향**(코팅·실사공정 정합 확인)으로 작용.
- **처리방향:** 마스터 무변경 + 부분 보정(전면 재적재 불요). 보드/액자 자재가 최우선.
- **위젯 기반:** 정제된 9속성이 실사 위젯 옵션 캐스케이드(소재→자재, 코팅/봉제/타공/족자→공정, 거치대→addon, 대형치수→nonspec)
  데이터 소스. 실사는 토대가 양호해 **보드자재·nonspec 범위만 보정하면 위젯 구동 가능**.
