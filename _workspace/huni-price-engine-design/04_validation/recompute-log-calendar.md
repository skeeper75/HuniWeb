## 1. pricing.py 동치 재구현 (동치 입증 선행)

`component_subtotal`(pricing.py:177-188) 동치:
```python
def component_subtotal(prc_typ, unit_price, tier_min_qty, qty):
    up, q = Decimal(str(unit_price)), Decimal(qty)
    if prc_typ == "PRICE_TYPE.01":      # 단가형: 장당가 × 수량 (:180)
        return up * q
    elif prc_typ == "PRICE_TYPE.02":    # 합가형: 총액 ÷ min_qty × 수량 (:181)
        if not tier_min_qty: raise ValueError   # :186-188
        return (up / Decimal(tier_min_qty)) * q
```
tier 선택 = min_qty '이상' 하한, 주문수량 이하 최대 임계(pricing.py:42·:146 `_tier_order_val`).

## 5. ★inline 합산 골든 BLOCKED 독립 역산 (designer 판정 재검)

**designer = BLOCKED(정직). 검증 = BLOCKED 정당(추측 회피=옳음·designer 산식 오류 아님).**

역산 방법: inline에서 제본비(라이브 q1 verbatim) 차감 → 인쇄+용지 잔여가 `출력판수 × (인쇄단가+용지단가)` 정수해를 주는가(qty=1).

```
탁상220 : 10400-5000=5400 ÷ (4000양면+112.58) = 1.313  ❌ non-integer
미니    :  6500-4500=2000 ÷ 4112.58            = 0.486  ❌
엽서    :  4000-   0=4000 ÷ (3000단면+112.58)   = 1.285  ❌
벽걸이  :  9900-5000=4900 ÷ 3112.58            = 1.574  ❌
와이드  : 24000-5000=19000 (3절·종이단가 미상)         —
```

**전 케이스 정수 출력판수 해 없음** → inline 가격 = 에디터형 1부 정찰가 스냅샷(formula-map §6 정합)이지 단가행 합산 결과 아님. inline을 product_prices INSERT 시 ① 단가행 산식과 충돌(권위 모호) ② PRODUCT_PRICE 선점으로 FORMULA 우회(G-CAL-2 발동). **BLOCKED 정직 표기·추측 단가 0 = 정당.** Q-CAL-GOLDEN 인간 컨펌 라우팅 옳음.
