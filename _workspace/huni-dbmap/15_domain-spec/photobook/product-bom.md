# 포토북 — 상품 BOM 명세 (round-11 확대 #3)

> **작성** 2026-06-10 · round-11. 포토북은 **1 상품명(`포토북[디자인명]`)** 이나 **size 4종 × 표지타입 3종 variant**로 펼쳐진다. 각 variant의 **구성요소(자재 BOM) + 공정 + 제본 + 생산방식**을 도출. 설명 한국어, 식별자 영어.
>
> **권위:** ① 후니 PDF ② 07_domain KB(entity-semantic-model §2 variant·§4 생산방식·§5 제본·decision-trail D-29 면지) ③ **경쟁사 SKU 구분(Shutterfly/Mixbook/Blurb, 보조)** ④ 엑셀 실측. 추정 0.
> **생산구조(07_domain §4):** 포토북=**B 셋트**(하드/레더 표지=반제품 sub_prd USAGE.02·면지 USAGE.03) + 소프트커버는 종이표지 PUR(A 통합 근접). **자재 권위는 항상 parent + usage_cd**(B 셋트라도 sub_prd 9속성 0행=정상).
> **확정도:** ✅ DB/PDF · 🟡 부분/표준 · 🔴 컨펌.

## 0. variant 구조 — 1상품 × size 4 × 표지타입 3 (경쟁사 SKU 정합)

## 1. 하드커버 포토북 (표지타입=하드커버) — B 셋트

- **제본:** PUR제본. **표지=반제품**(하드보드+아트250 싸바리). **책등(10/12/14/16mm)** = 페이지수 따라.
- **자재(BOM):** 내지 **몽블랑130**(usage.01) + 표지 **아트250**(usage.02, 반제품) + **무광코팅**(표지 공정) + **면지 그레이**(usage.03 D-29). ✅
- **인쇄:** 내지 양면 / 표지 단면. 내지폴더=디지털인쇄·표지폴더=디지털인쇄. ✅
- **page_rule:** 24~150·증가2. ✅
- **가격:** size별 base(8x8=15000·10x10=22000·A5=12000·A4=16000) + 추가(2P)당(500·1000·300·600).
- **공정:** `내지출력 → 표지출력/무광코팅 → 면지 부착 → PUR제본(책등) → 재단 → 포장`. ✅
- **셋트 주의:** 표지 sub_prd 빈 껍데기(9속성 0행 정상), 자재 권위=parent+usage_cd. ✅
- **확정도:** ✅

## 2. 레더 하드커버 포토북 (표지타입=레더하드커버) — B 셋트

- **제본:** PUR제본. 표지=**레더** 반제품(특수출력 별 라인).
- **자재:** 내지 몽블랑130(usage.01) + 표지 **레더**(usage.02, MAT_TYPE.06 가죽?) + 면지 그레이(.03). ✅
- **인쇄방식 분리:** 내지폴더=디지털인쇄 / **표지폴더=특수출력**(레더는 별 라인). ✅
- **page_rule:** 24~150·증가2. ✅
- **가격:** size별 base(8x8=23000·10x10=32000·A5=19000·A4=26000) — **하드커버 대비 +8000~10000**(레더 프리미엄, 경쟁사 leather cover upgrade 정합).
- **공정:** `내지출력(디지털) → 레더표지(특수출력) → 면지 부착 → PUR제본 → 재단 → 포장`. ✅
- **확정도:** ✅ (레더 자재유형 🟡 — CONFIRM-PB-3)

## 3. 소프트커버 포토북 (표지타입=소프트커버) — A 통합 근접

- **제본:** PUR제본. 표지=**아트250+무광코팅 종이**(반제품 아님, 종이표지 PUR).
- **자재:** 내지 몽블랑130(usage.01) + 표지 아트250+무광코팅(usage.02 종이) + 면지 그레이(.03). ✅
- **인쇄:** 내지 양면 / 표지 단면. 표지폴더=디지털인쇄. ✅
- **page_rule:** **4~14·증가2**(하드보다 페이지 범위 다름 — 소프트커버 경량). 🟡
- **표지작업사이즈:** 420x213(하드 460x235보다 작음 — 책등 얇음).
- **가격:** size별 base(8x8=12000·A5=10000·A4=13000) — **가장 저렴**(경쟁사 softcover most affordable 정합). 10x10 소프트=비활성(회색).
- **공정:** `내지출력 → 표지출력/무광코팅 → PUR제본 → 재단 → 포장`(면지 유무 컨펌). ✅
- **생산구조 주의:** 소프트커버는 표지=종이라 **A 통합 근접**(반제품 아닐 수 있음). 하드/레더만 확실한 B 셋트. **CONFIRM-PB-1**(표지타입 sub_prd 분해 범위). 🟡
- **확정도:** 🟡 (생산구조·page 범위 컨펌)

> **제본=PUR 단일**(책자는 GRP-BOOK-제본 택일그룹·7종, 포토북은 PUR만 → **excl_group 불요**, mand_proc_yn=Y 단일). 레이플랫(PROC_000025) 미운영(07_domain §144, **C-10 컨펌 1순위**).

**page_rule:** 하드/레더하드=24~150·증가2. 소프트=4~14·증가2(컨펌). 페이지=spread 단위(증가2 ↔ 가격 추가2P당).

**가격(가격포함 시트):** base(24P 기준) + per-page(2P당 증분). size×표지타입 매트릭스. **round-2 가격엔진(고정가 base + 증분 component)** — `dbm-price-formula` 트랙. 경쟁사 base+per-page 동형.

**미저장(앱 런타임):** 책등 두께→PUR 책등(C30) = 페이지수 따라 계산 가능(메모리 `compute-in-app`).

**생산방식 함의:** 하드/레더하드=B 셋트(표지 sub_prd 빈 껍데기 + parent usage_cd 자재 권위). 소프트커버=A 통합 근접(종이표지). 적재 감사에서 sub_prd 9속성 0행=정상, parent 내지종이 0행이 결함.
