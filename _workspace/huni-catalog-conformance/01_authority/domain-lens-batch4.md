## 0. 굿즈파우치 = "완제품 굿즈/파우치/에코백" — 인쇄가 아니라 완제 제작상품

굿즈파우치 시트는 11시트 중 **유일하게 "인쇄물"이 아니다.** 거울·코스터·머그컵·보틀·쿠션·티셔츠·파우치·
에코백·필통·키링·폰케이스 — 전부 **완제 굿즈**다. 도메인 정체:

- **상품 = 완제 소품/잡화 + (그 위에 인쇄/승화/UV/고주파한) 디자인**. 출력소재(전지)에 잉크를 얹는
  디지털인쇄·책자와 본질이 다르다. 굿즈는 **이미 완성된 물건(블랭크)에 고객 이미지를 입히는** 모델.
- 따라서 가격은 **출력 면적·도수·판수로 분해되지 않고**, **완제품 개당 고정단가**로 결정된다
  (RedPrinting `tmpl_price`·WowPress 완제 블랭크와 동형). [[dbmap-print-domain-recipe-philosophy]]·§18 §7.

## 1. 12축의 굿즈파우치 도메인 의미 (인쇄물 시트와 다른 점을 명시)

| 축 | 굿즈파우치에서의 의미 | 인쇄물 시트와의 차이 |
|----|---------------------|---------------------|
| **사이즈코드** | variant 사이즈등급(S/M/L·75x130mm 등)·기종(아이폰15) | 치수 자체가 아니라 **비치수 등급/기종**(siz_nm 형상 융합). 102 중 65 상품만 옵션 보유(단일 사이즈 33상품은 needed=N) |
| **도수** | 본체 인쇄 단/양면(벨벳쿠션 단면/양면·이미지피켓 양면) | 도수=색수 매트릭스가 아니라 **인쇄면 variant**. 완제 BOM의 일부로 항상 needed=Y(편집기/업로드 경로 존재) |
| **인쇄옵션** | 고주파(말랑PVC)·승화·UV — **공정에 가까운 가공** | 별색/코팅/커팅이 아니라 굿즈 표면처리. 가공(옵션)_가공 컬럼 보유 8상품만 needed=Y |
| **판형** | **해당 없음(N)** — 굿즈는 전지 출력 안 함 | 출력용지규격(I열)=hidden·완제 블랭크라 전지 판형 없음. ★라이브 85 plate_sizes=**EXTRA 후보**(round-22 잔재 의심·인스펙터 검토) |
| **자재** | 본체 원단/소재 = **상품 정체**(레더·캔버스·타이벡·메쉬·PVC·코르크·우드·규조토) | 인쇄용지가 아니라 **완제 본체 소재**. 항상 needed=Y(BOM 핵심)·단 가격은 inline 고정가에 baked-in(자재 단가 합산 아님·§18 §5) |
| **공정** | 고주파융착·재봉(에코백)·박·고리부착 | 인쇄 후가공이 아니라 **굿즈 제작 공정**. 8상품 needed=Y |
| **묶음수** | N구/팩(키캡 2~4구·자석북마크 2개1팩·CD앨범 세트) | 페이지 묶음이 아니라 **개수 단위**(GAP-COUNT). 옵션 보유 65상품 needed=Y |
| **페이지룰** | **해당 없음(N)** — 굿즈는 내지페이지 없음 | 책자 전용 축. 굿즈파우치 전건 needed=N |
| **옵션그룹** | variant 택1(사이즈등급/용량/면/기종)·가공 택1 | 옵션 또는 가공 보유 67상품 needed=Y·라이브 0행 적재(전건 MISSING) |
| **제약규칙** | 엑셀 별표(★) = 폰케이스 신규기종 마커일 뿐·진짜 제약 0 | needed 전건 N(별표는 "준비 필요" 노트). 라이브 EXTRA 제약은 인스펙터 |
| **추가상품** | 잉크 5cc·볼체인·아크릴스탠드·맥세이프 | 5상품만 추가상품(옵션) 보유 needed=Y |
| **추가상품 템플릿** | 추가상품 SKU 묶음 | 추가상품 보유 5상품과 동치 needed=Y |

## 2. ★가격계산방식 판별 (면적매트릭스 vs 규격티어 vs 고정가) — 고정가 100%

★[HARD] 굿즈파우치는 **면적매트릭스(가로×세로) 0건·규격티어 0건·세트조합 0건 = 전부 고정가형**이다.
calc-formula-draft row 122~123 절대 권위: `[고정가형: 굿즈/파우치] = 고정가 + 수량별구간할인 타입 + 추가상품`.

| 계산방식 | 굿즈파우치 적용 | 도메인 근거 |
|---------|---------------|------------|
| **면적매트릭스(siz_w/h)** | **없음** | 완제 블랭크는 면적으로 안 갈림(실사·아크릴과 결정적 차이) |
| **규격티어(연속 사이즈)** | **없음** | 사이즈는 이산 variant(S/M/L)·연속 입력 아님 |
| **GP-1 단일고정가(55상품)** | 완제 1개당 단가 1건(차원 없음) | 카드거울 2,500 단 1가·거울/코스터/파우치/에코백 본체 |
| **GP-2 변형고정가(31상품)** | variant별(S/M/L·용량·면·기종) 각 고정가 | 사각손거울 S5000/M5500/L6000·머그 11온스/대용량·벨벳 단면/양면 |
| **수량구간할인 타입(횡단)** | 4종 택1(GOODSA/B·FABRIC·SQUISHY) | base에 정률% 곱·문구 단일 DSC와 결정적 차이 |

★ **돈크리티컬 도메인 함정 2종(§18 G-GP-3·G-GP-5 계승):**
1. **GP-2 평탄화 금지** — S/M/L variant를 단일 고정가 1개로 평탄 적재하면 M 주문에 S가격 오청구.
   variant축(opt_cd/siz_cd)을 판별차원으로 충전 필수(평탄화=과대/과소청구).
2. **GP-2 PRODUCT_PRICE 선점 가드** — GP-2 상품에 product_prices 1행이라도 있으면 FORMULA 경로가
   통째로 우회되어 variant 단가가 영영 안 먹힘(가장 silent한 실패).

## 3. 완제 vs 반제(세트) — 굿즈파우치는 세트조합 레이어 불요

- 완제품(GP-1·GP-2): 거울·코스터·머그·보틀·쿠션·파우치·에코백·필통·말랑류 전부 = 각 1 고정가(또는
  variant별 고정가). 면적/원자합산/세트조합 분해 없음.
- **세트(반제 조합) 상품 부재** — 디지털 엽서북식 내지+표지 별 SKU 합산·책자 부품 합산이 굿즈엔 없다.
- 단 파우치 본체+조립(지퍼/끈) BUNDLE은 자재+공정(생산 BOM)일 수 있으나 **가격은 inline 고정가에
  baked-in**(BUNDLE은 MES·dbmap 위임·[[dbmap-acrylic-price-chain-link]]). 가격 합산 아님.
