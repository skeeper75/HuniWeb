# 가격구성요소 그룹핑 모델 정립 — round-23 (사용자 의도 재정렬)

> **작성** 2026-06-17 · round-23. **사용자 의도: 가격구성요소 그룹핑 관리.** 같은 차원을 쓰는 동종 요소(제본={무선/중철/PUR/트윈링}, 별색={화이트/핑크/금/은/클리어}, 오시/미싱/귀돌이/가변)를 1 구성요소 그룹으로 묶고 세부 종류는 차원값(키)으로 두어 `(구성요소, 종류)` 키로 단가 조회. 목적 = 동종 요소 난립 방지·그룹 관리.
>
> **★이전 분석 정정:** 우리(arbiter 포함)가 별색 424행을 "오염 삭제", 제본을 "통합 비대상", 귀돌이를 "통합 보류"로 본 것은 **사용자 그룹핑 의도와 어긋난 오판**. 본 문서가 라이브 근거로 재판정·철회한다.
>
> **라이브 실측:** Railway read-only SELECT, 2026-06-17. 비밀값 비노출·DB 쓰기 0. 생성자≠검증자(GO는 dbm-validator).

## 0. 핵심 결정 (5)

1. **그룹핑 표현 모델 = ① 단일 comp + 종류축 차원 키.** 종류가 **공정성(제본·별색·오시·미싱·귀돌이)이면 `proc_cd`(t_proc_processes 부모-자식 그룹)**, **순수 파라미터(줄수·개수)면 `dim_vals` jsonb**. 색은 별색의 경우 이미 proc_cd(PROC_000008~012)가 색축. `(comp_cd, 종류키)`로 단가 조회. **comp_grp 신설 불요**(proc_cd 트리가 이미 그룹 역할).
2. **별색 재판정 = 이전 "424 오염" 판정 전면 철회.** WHITE_S1 530행은 **5색(proc_cd PROC_000008~012)×2면(POPT)×53구간의 정상 통합 comp**. 확장키(proc_cd+print_opt_cd) 기준 **중복 0**(실측). 내가 자연키에서 proc_cd를 누락해 "424 중복"을 오산했음. **삭제 금지·보존.** WHITE_S1이 곧 사용자 의도(색=조회 키)의 정답 형태.
3. **제본 재판정 = 이전 "통합 비대상" 철회.** 제본종류는 **proc_cd(PROC_000017 "제본" 그룹의 자식: 중철/무선/PUR/트윈링…)가 종류 키**. 라이브에 이미 통합 선례 존재(`COMP_BIND_TWINRING`이 4종 proc 흡수 32행·`CAL_WALL`이 4종). 상호배타(UI 택1)는 가격구성요소 통합과 별개 — 통합 가능·권고.
4. **귀돌이·미싱 재판정:** 귀돌이=PROC_000026 그룹(둥근/직각=proc_cd 자식) → proc_cd 종류축 통합 가능(이전 "보류" 완화). 미싱=오시 동형(proc_cd + dim_vals.줄수). 단 prc_typ 불일치(1L=.02) 정정 동반.
5. **일반 원칙(타 시트 확장):** 종류축 결정 트리 = **공정 종류 → proc_cd(t_proc 그룹 트리) / 색·재질 → clr_cd·mat_cd / 순수 수량파라미터(줄·개·면) → dim_vals**. 같은 use_dims·같은 prc_typ·동종이면 1 comp로 그룹핑.

## 1. 그룹핑 표현 모델 정립

### 1.1 후니 t_prc_* 구조 사실 (라이브 실측)

- `t_prc_price_components` 컬럼: comp_cd·comp_nm·comp_typ_cd·prc_typ_cd·**use_dims(jsonb)**·note·use_yn. **comp_grp 컬럼 없음**(PC_COLS 실측).
- `t_prc_component_prices` 종류축 후보 컬럼: `proc_cd`·`clr_cd`·`mat_cd`·`dim_vals(jsonb)`·`opt_cd`·`print_opt_cd`.
- 즉 "구성요소 그룹"을 표현하는 전용 컬럼은 없지만, **종류축 = 단가행의 proc_cd/clr_cd/mat_cd/dim_vals**가 이미 `(comp, 종류)` 키 조회를 가능케 함. `use_dims`가 "이 comp는 어느 차원으로 단가를 가르는가"를 선언.

### 1.2 3개 후보 모델 비교

| 모델 | 표현 | 후니 정합 | 키 조회 | 정규화 | DDL |
|------|------|:--:|:--:|:--:|:--:|
| **① 단일 comp + 종류축 차원** | 1 comp, 종류=proc_cd/clr_cd/dim_vals | ✅ **라이브 선례 실재**(WHITE_S1 5색·BIND_TWINRING 4종·CREASE_1L 줄수) | ✅ `(comp_cd, 종류키)` | ✅ 무손실 | **불요** |
| ② comp_grp(상위)+comp(종류) | grp 테이블/컬럼 신설 | ❌ comp_grp 컬럼 부재 | △ 2단 조인 | △ | **필요(신규)** |
| ③ 종류 전용 코드축만 | clr_cd 등 | △ clr_cd 5종(도수)뿐·별색 자리 없음(메모리 G-1) | △ | △ | 부분 |

### 1.3 결정 — ① 단일 comp + 종류축 차원 (권장)

**근거(라이브 선례):**
- **별색:** WHITE_S1 1 comp가 proc_cd(PROC_000008~012=화이트/클리어/핑크/금/은)로 5색 흡수 → `(COMP_PRINT_SPOT_WHITE_S1, proc_cd=PROC_000011)` = `별색-금색` 조회. **이미 사용자 의도 구현.**
- **제본:** COMP_BIND_TWINRING 1 comp가 proc_cd(중철/무선/PUR/트윈링) 4종 흡수 → `(comp, proc_cd=PROC_000019)` = `제본-무선`.
- **오시:** CREASE_1L 1 comp가 dim_vals.줄수 1/2/3 흡수 → `(comp, dim_vals={"줄수":2})` = `오시-2줄`.

→ **종류축 컬럼 선택 규칙:** 종류가 **공정 카탈로그(t_proc_processes)에 부모-자식으로 등재된 것**(제본종류·별색색·오시·미싱·귀돌이)이면 **proc_cd**. 종류가 **순수 수량 파라미터(줄수·개수·면수)**면 **dim_vals**. 색이 별색처럼 공정으로 등재됐으면 proc_cd(별색 색은 PROC). comp_grp 신설은 proc_cd 트리가 이미 그룹 역할을 하므로 **불요**(과적 DDL 회피·search-before-mint).

> **WHITE 통합 comp의 명명 정정:** 5색 흡수 comp가 `COMP_PRINT_SPOT_WHITE_S1`로 명명된 것은 부적절(WHITE는 5색 중 1색일 뿐). 정본 통합 comp는 색 무관 명명 권고: `COMP_PRINT_SPOT_S1`(단면 별색)·`COMP_PRINT_SPOT_S2`(양면). 색은 proc_cd 차원. 이는 명명 정비이지 행 삭제 아님(Q-G1).

## 3. 제본 통합 정립 (이전 "통합 비대상" 철회)

### 3.1 실측 — 제본종류 = proc_cd 그룹 트리

- PROC_000017 "제본"(부모) → 자식: 중철18·무선19·PUR20·트윈링21·떡22·하드커버무선23·하드커버트윈링24·레이플랫25·싸바리98·캘린더99~102.
- 라이브 통합 선례: `COMP_BIND_TWINRING`이 proc 18/19/20/21 **4종 흡수**(32행)·`COMP_BIND_CAL_WALL` 4종(24행)·`COMP_BIND_SSABARI` 3종(18행). → **종류축 통합 패턴이 제본에도 이미 실재.**
- 개별 comp 잔존: JUNGCHEOL(18만)·MUSEON(19만)·PUR(20만)·HC_MUSEON(23만)·HC_TWINRING(24만)·CAL_DESK130/220/MINI(101/100/102 각). → **혼재(통합 미완)이지 통합 불가 아님.**

### 3.2 결론 — **철회·통합 권고**

- ❌ **철회:** "제본 11 comp 공정종류 본질 차이 → 통합 비대상"(silsa-quote-design §2.4).
- ✅ **정정:** 제본종류는 proc_cd로 키가 잡히는 **동일 차원(제본 공정·.01 단가형·use_dims proc_cd+min_qty)** → 사용자 의도대로 **그룹 통합 가능**.
- **통합 모델:** 정본 `COMP_BIND`(또는 책자/하드커버/캘린더 3 그룹 분할) + proc_cd 종류축. 잔존 개별 comp는 정본으로 흡수·use_yn=N. **상호배타(UI 택1)는 옵션 레이어(L2) 표현이지 가격구성요소 통합과 무관** — 사용자 지적이 정확. 통합 후 `(COMP_BIND, proc_cd=무선)` 조회.
- **단 책자 vs 하드커버 vs 캘린더는 use_dims/공정대분류가 달라**(하드커버=양장 추가공정·캘린더=별 proc 99~102) 3 그룹 분할이 정규화상 타당할 수 있음 → Q-G2.

### 3.3 귀돌이·미싱 보강

- **귀돌이:** PROC_000026 그룹(둥근=028·직각=027 추정) → proc_cd 종류축 통합 가능. 이전 "보류"는 proc_cd 혼재(ROUND=027+028) 때문 → **proc_cd 정합 후 통합**(보류 완화). 형상=공정등재이므로 proc_cd 축(dim_vals 아님).
- **미싱:** 오시 동형(proc_cd + dim_vals.줄수). prc_typ 1L=.02→.01 정정 동반(silsa-quote §2.2 유지).

## 4. 일반 그룹핑 원칙 (타 시트 확장 — 결정 트리)

> 사용자: "이게 정리돼야 다른 시트 통합 가능." 재사용 결정 트리.

```
가격구성요소 후보 N개가 동종인가?
│
├─ [Q1] 같은 prc_typ_cd(.01단가/.02합가) + 같은 use_dims(차원집합) + 같은 comp_typ_cd 인가?
│       NO → 그룹핑 금지(이질). 별 comp 유지.
│       YES ↓
│
├─ [Q2] N개의 "차이"가 무엇인가?
│       ├─ 공정 종류(제본방식·별색색·오시/미싱/귀돌이) → t_proc_processes에 부모-자식 등재?
│       │     YES → ★종류축 = proc_cd. 정본 1 comp + proc_cd 종류키. (제본·별색·오시·미싱·귀돌이)
│       │     NO(미등재) → 공정 카탈로그 등재 선행 후 proc_cd, 또는 Q3.
│       ├─ 순수 수량 파라미터(줄수·개수·면수) → ★종류축 = dim_vals jsonb. (오시 줄수·가변 개수)
│       ├─ 색(도수 아닌 별색) → 별색은 proc_cd(공정등재). 일반 색=clr_cd(단 5종 도수뿐, 별색 자리 없음)
│       └─ 재질/소재 → mat_cd
│
├─ [Q3] 종류별 단가가 실제로 다른가?
│       YES → 종류축 차원행으로 단가 분기(정상). 1 comp에 종류×구간 전건.
│       NO(동일가) → 종류축 불요·단일행(예: 시트커팅 화이트/블랙 동일가).
│
└─ [Q4] 통합 시 의미손실?
        정본에 전 종류 단가행 존재 → 잔존 형제 comp use_yn=N + 배선 1개 축소(재적재 0).
        정본에 일부만 → 부족분 dim_vals/proc_cd 부여해 이설(값동일·신규0).
```

**핵심 원칙:**
1. **그룹핑 = comp 1개 + 종류축 차원**(comp_grp 테이블 신설 불요·proc_cd 트리가 그룹).
2. **종류축 우선순위:** 공정등재 → proc_cd / 수량파라미터 → dim_vals / 별색색 → proc_cd / 도수 → clr_cd / 재질 → mat_cd.
3. **상호배타(UI 택1)는 그룹핑 판정과 무관**(옵션 레이어 L2 문제).
4. **통합 = use_yn=N + 배선 축소**(행 삭제 아님·단가행 보존). 단 정본 명명은 종류중립.
5. **확장키로 중복 검사**(proc_cd·print_opt_cd 포함) — 자연키만으로 "중복" 오판 금지(별색 교훈).

### 4.1 타 시트 적용 예시

| 시트 | 그룹 후보 | 종류축 | 통합 키 |
|------|----------|--------|---------|
| 코팅 | 무광/유광/벨벳 코팅 | proc_cd(코팅종류) + coat_side_cnt | `(COMP_COAT, proc_cd, 면수)` |

| 봉투 | 규격×색 | siz_cd + mat_cd | `(COMP_ENV, siz, mat)` |
