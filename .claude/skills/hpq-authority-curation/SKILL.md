# hpq-authority-curation — 권위 가격 골든값 추출 방법론

## 목적

라이브가 옳은지 판정하려면 **정답이 먼저 고정**돼야 한다. 권위 엑셀에서 상품군별 정답 가격 구성과
골든 단가값을 추출해 검사·검증의 기준점을 만든다.

## 절차

### 1. 가격축 도출
각 상품군에서 "가격을 바꾸는 차원"을 권위 엑셀 기준으로 확정한다(사이즈·자재·도수·인쇄옵션·공정·
옵션·수량 중 어느 것). 이것이 라이브 use_dims·단가행 차원의 **정답 기준**이 된다.

### 2. 가격산정 방식 4분류 태깅
dbmap 정립 분류로 상품군을 태깅(검증 케이스 설계 입력):
- **MATRIX**(매트릭스) · **합산형**(원자 합산) · **면적매트릭스**(가로×세로 구간·[[dbmap-area-matrix-wh-dimension]]) ·
  **2단룩업** · **고정가**(수량×옵션).

### 3. 골든 단가 추출 (verbatim)
- 단가를 **변형·재계산·정규화하지 않는다**. 셀 좌표(시트·행·열)를 출처로 박는다.
- 면적매트릭스는 매트릭스 그대로, 고정가는 고정가로, 수량구간은 구간경계+구간단가 그대로.
- 합가형/단가형을 권위 기준으로 구분(엔진 환산 검증의 입력).

### 4. 검증 케이스 설계
시뮬레이터 검증용 대표 케이스: {상품·선택값(차원별)·수량·grade} → **기대 final_price**(엑셀에서 유도).
- 케이스는 4분류를 대표하도록(MATRIX·합산형·면적·고정가 각 1+).
- 기대값 유도 과정을 함께 적어(검증자가 역추적 가능) — 골든은 항상 `값 + 셀출처 + 유도`.

### 5. 갭 분리
권위 엑셀에 없거나 모호하면 골든값으로 박지 않고 `authority-gaps.md`에 `가설 + 출처후보 + 컨펌ID`로.

## 권위 규칙 [HARD]
- 상품마스터(260610)+인쇄상품 가격표(260527) = 절대 권위. 라이브·rpmeta·경쟁사 = 갭헌팅(덮어쓰기 금지).
- v03 마이그레이션 산출·STALE 등급 원천 = 정답 참조 금지(round-14 freshness).
- 추정 0. 골든값은 엑셀 명시값 verbatim + 셀 출처 필수.
