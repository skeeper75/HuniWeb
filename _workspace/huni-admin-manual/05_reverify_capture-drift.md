## 핵심 결론 (3줄)

1. **로그인 OK** — redirect 흐름·275 상품·사이드바 전부 베이스라인과 동일.
2. **재점검 14개 화면 중 DRIFT 0건** — 모든 화면이 베이스라인 캡션과 일치(MATCH). PNG 덮어쓰기 0건.
3. **옵션 레이어(silsa PRD_000138) = 이미 라이브 CPQ 데이터로 렌더 중** — **그러나 이것은 베이스라인 캡처 시점(06:38)에 이미 반영된 상태**였다. round-9 CPQ 적재(2026-06-09 silsa 43행 COMMIT)가 베이스라인 캡처(2026-06-10 06:38)보다 **먼저** 일어났으므로, 옵션 L1/L2 베이스라인 PNG는 이미 적재된 상태를 찍은 것. 즉 "빈 화면 → 적재됨" 드리프트는 **이미 베이스라인에 내재**되어 있어 신규 드리프트 아님.

## 옵션 레이어 라이브 렌더 — 상세 확인 (최우선 의심 항목)

**질문:** round-9 CPQ 적재로 silsa(PRD_000138) 옵션 화면이 빈 화면 → 적재된 행으로 바뀌었는가?

**답:** 라이브는 **CPQ 데이터로 정상 렌더 중**이다. 단 이 상태는 **베이스라인 캡처 시점에 이미 동일하게 반영**되어 있었다.

- silsa 상세: `옵션그룹 (2)` — 가공·추가 두 그룹.
- 가공 그룹(OPT_000003): 6옵션(열재단·타공4/6/8·양면테입·봉미싱), 각 옵션이 **자재+공정 BUNDLE** 구조로 라이브 렌더 — 양면테입=「양면테입(자재)/공통」+「부착(공정)」, 봉미싱=「봉제사(자재)/공통」+「봉제(공정)」.
- 추가 그룹: 없음·큐방(4개)·끈(4개)·각목(세로)+끈(4개)·각목(가로)+끈(4개) — round-9 **각목 방향(세로/가로) 재모델**이 라이브에 반영됨.

**타임라인 정합:** round-9 silsa 43행 COMMIT = 2026-06-09. 베이스라인 캡처 = 2026-06-10 06:38. ⇒ 캡처가 적재 **이후**라 베이스라인 L1/L2 PNG는 이미 적재본을 찍음. 픽셀 대조 결과 라이브 = 베이스라인(뷰포트 폭 차이 외 동일). **신규 드리프트 0.**

**대조 검증:** 엽서(PRD_000016) 옵션그룹 = 0행(빈 화면) 그대로 — round-9 엽서 적재본이 DRY-RUN GO이나 미COMMIT임과 정합. 옵션 레이어가 "전 상품 일괄 적재"가 아니라 **silsa만 선별 COMMIT**된 round-9 상태를 라이브가 정확히 반영.

## QA·리더 통지 사항

- 옵션 레이어가 라이브에 **CPQ 데이터로 정상 렌더됨**을 시각 재확인. 매뉴얼 옵션 챕터의 권위(silsa 예시)가 라이브와 일치 — QA 게이트의 "캡처 인덱스=시각 권위" 가정이 이번 라이브 재대조로 **검증됨**(가정이 옳았음).
- 신규 시각 드리프트 0 → 매뉴얼 재집필·재캡처 트리거 불요. 캡션 정밀화 2건(자재유형 코드값·옵션 추가 그룹 예시)은 Low 우선, 작가 재량.
