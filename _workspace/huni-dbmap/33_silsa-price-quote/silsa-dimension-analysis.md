## 0. 핵심 발견 (한 눈)

1. **포스터사인 = 가격표 최대 시트** — **본가 테이블 26개** + **추가옵션(add-on) 테이블 8종**. 본가 26 중 면적매트릭스 13(`가로/세로` 헤더 13개 실측) + 수량/사이즈 밴드·이산표 13.
2. **사이즈 차원은 라이브에서 `siz_cd`(FK→t_siz_sizes) 1개로만 받는다.** `t_prc_component_prices`에 가로/세로 수치 컬럼 **없음**. 가로×세로 1조합 = 1 siz_cd(좌표 규격코드).
3. **기존 모든 면적매트릭스 선례(실사 포스터·현수막·아크릴)가 전건 `siz_cd` 방식**으로 적재됨(라이브 실측: plt_siz_cd·dim_vals 사이즈 용도 사용 0건). → A안이 곧 라이브 선례.
4. **t_siz_sizes는 `work_width/work_height/cut_width/cut_height` 수치 컬럼 보유** — siz_cd 한 행이 가로×세로를 정규 표현. 좌표형 siz **383종 이미 등록**(`^\d+x\d+$` 명명·use_yn=Y 517종 중). 면적매트릭스 사이즈축의 정규 그릇이 이미 존재.
5. **가격사슬 부분단절 라이브 재확인** — 28상품 전건 `PRF_POSTER_FIXED` 단일 바인딩이나 공식 배선은 `COMP_POSTER_ARTPRINT_PHOTO` **1 comp만**(`formula_components` 1행). 인화지 외 27상품 엔진 조회불가(돈-크리티컬 결함, Phase B 보강 대상).

> **잠정 권고(이번엔 근거 정리, 결정은 Phase B):** **A안(siz_cd 기초코드)**. 라이브 그릇·선례·정규화가 전부 A안을 가리킴. B안(옵션/템플릿/제약)은 가격엔진 매칭 규칙(component_prices가 siz_cd로만 사이즈 매칭)과 충돌 → 면적단가를 옵션화하면 가격사슬을 벗어남. 단 **siz 채번 규모(좌표 미실재 분)**는 별도 컨펌 필요.

## 2. 항목 2 — 테이블별 차원 분해 + 견적 필요 구성요소

### 2.1 면적매트릭스 13블록 (본가)

```
셀 (가로 g, 세로 s) = 단가 p
  → component_prices: comp_cd=COMP_POSTER_<MAT>, siz_cd=<(g,s) 좌표 규격코드>,
     unit_price=p, (clr/mat/proc/coat/opt/bdl/min_qty = NULL)
```
- **차원 = `siz_cd` 1개**(가로×세로 = 1 규격). 수량축 없음(라이브 `use_dims=["siz_cd"]` 실측). 도수·소재 무관(소재는 comp_cd로 분기).
- **off-grid** = 격자에 없는 가로×세로 → 한 단계 큰 크기 ceiling(엔진/위젯 런타임·DB 미저장). 메모리 [[dbmap-compute-in-app-db-stores-lookup]] 정합.
- **견적 성립 필요 구성요소:** ① 상품(prd_cd) → ② 소재별 공식(현재 단일 `PRF_POSTER_FIXED` — §4 보강 필요) → ③ 소재 comp(`COMP_POSTER_<MAT>`) → ④ 좌표 siz_cd(가로×세로). 도수/코팅은 매트릭스 통가(별도 차원 불요).

### 2.2 사이즈/수량 밴드 9블록

```
(siz_cd, min_qty) = 구간 개당단가
  → component_prices: comp_cd=COMP_POSTER_<X>, siz_cd=<규격>, min_qty=<구간하한>, unit_price=p
```
- **차원 = `siz_cd` + `min_qty`**(라이브 `use_dims=["siz_cd","min_qty"]` 실측). 수량 많을수록 개당 싸지는 **단가형**(.01)·구간총액 아님.
- **견적 필요:** 상품 → 공식 → comp → siz_cd + 주문수량(→ min_qty 매칭 구간 단가 × 수량). 밴드류 siz는 표준규격(A시리즈·600x1800)이라 siz 실재율 높음(BLOCKED 낮음).

### 2.3 이산 사이즈/색·소재 4블록

- **차원 = siz_cd + 색/소재 분기**. 색은 가격차 있으면 별 comp_cd(폼보드 화이트/검정), 가격 같으면 단일 comp(시트커팅 무광 화이트/블랙=동일가). 미러/홀로그램은 가격 상이 → 별 comp.
- **견적 필요:** 상품 → 공식 → 색/소재별 comp → siz_cd(표준규격, round-2 §6.5 전건 라이브 실존 확증).

### 2.4 추가옵션(add-on) 8테이블

- **옵션축 = 가공(타공/봉제/재단)·부속(거치대/우드행거/우드봉/끈/큐방/천정고리)**. 본가 면적/사이즈 차원과 직교.
- 라이브 실측: `COMP_POSTEROPT_*` **별 component**로 모델(opt_cd 차원 아님). use_dims = `[]`(고정 add) 또는 `["siz_cd"]`(거치대=사이즈 종속).
- **견적 필요:** 본가 comp(본체가) + 선택 옵션 comp(add-on) **합산**(formula_components addtn_yn=Y). 합산 경로 vs CPQ option add_price는 컨펌(round-16 Q-PS-2).

## 3. 항목 3 — 매트릭스 사이즈 처리방식 A안 vs B안 (핵심 판정 근거)

### 3.1 라이브 가격엔진 매칭 규칙 실측 (결정적 근거)

| 실측 항목 | 결과 | 출처 |
|-----------|------|------|
| component_prices 사이즈 차원 컬럼 | **`siz_cd` varchar(50) FK → t_siz_sizes**. 가로/세로 수치 컬럼 **없음** | information_schema (CP_COLS) |
| 가로×세로 직접 수치 입력 가능? | **불가** — 사이즈는 siz_cd로만 매칭. (엔진: comp_cd+siz_cd+… 조건으로 unit_price 1건 조회) | price-engine-ddl §4·§5 |
| 면적매트릭스 선례 적재방식 | 실사 포스터·현수막·아크릴 comp **전건 siz_cd**. plt_siz_cd 사이즈 용도 0건, dim_vals 사이즈 용도 0건 | POSTER_CP 실측 |
| t_siz_sizes 좌표 표현력 | `work_width/height·cut_width/height` 수치 보유 → siz_cd 1행 = 가로×세로 정규 표현 | SIZ_COLS 실측 |
| 좌표형 siz 기등록 규모 | `^\d+x\d+$` 명명 **383종**(use_yn=Y 517 중) | COORD_SIZ 실측 |
| `dim_vals jsonb` 신규 컬럼 정체 | "개수" 등 비-사이즈 축(280행). 사이즈 좌표용 아님 | DIMVALS_SAMPLE 실측 |
| `plt_siz_cd` 신규 컬럼 정체 | 별색인쇄(SPOT) 판형 단가용. 면적매트릭스 사이즈축 아님 | PLTSIZ_SAMPLE 실측 |

### 3.2 A안 vs B안 트레이드오프 표

| 축 | **A안 — 기초코드 t_siz_sizes 등록** | **B안 — 옵션/템플릿/제약 처리** |
|----|-------------------------------------|----------------------------------|
| **무엇이 필요** | 좌표 가로×세로마다 siz_cd 1행(work/cut 수치) | option_groups/options/option_items에 가로·세로 선택지 + constraints + 가격을 옵션가로 |
| **어느 t_*** | `t_siz_sizes`(부모) → `t_prc_component_prices.siz_cd`(자식 FK) | `t_prd_product_option_*` + `t_prd_product_constraints` + (가격은 여전히 component_prices siz_cd 또는 별 경로) |
| **가격엔진 정합** | ✅ **완전 정합** — 엔진이 siz_cd로 사이즈 매칭(설계 그대로) | ❌ **충돌** — 면적단가는 component_prices(siz_cd)에 있어야 엔진이 조회. 옵션은 가격축이 아님(option_items에 add_price 컬럼 부재 — 메모리 [[dbmap-acrylic-price-chain-link]]) |
| **라이브 선례** | ✅ 실사·현수막·아크릴 전건 A안 | ❌ 면적매트릭스 B안 선례 0건 |
| **기존 행 영향** | siz 부모행 추가(자식 가격행 FK 연결). 기존 가격행 무손상 | 가격사슬을 옵션으로 우회 → 기존 component_prices 면적행과 이중모델 위험 |
| **정규화** | ✅ 가로×세로 = 규격 = siz의 본질(work/cut 수치 보유). 1정규형 | ❌ 옵션 선택지에 좌표를 넣으면 가격이 옵션값에 종속(반정규화) |
| **메모리 정합** | ✅ [[dbmap-l2-requires-l1-price-table]](사이즈=이산 면적매트릭스·가격표 권위) / [[dbmap-compute-in-app-db-stores-lookup]](DB=룩업) | ❌ L2(옵션)는 L1(차원/가격) 위에 얹는 층 — 사이즈를 L2로 내리면 경계 위반 |
| **실무진 관리용이성** | △ 좌표 siz가 많으면 siz 마스터 비대(BLOCKED 채번 부담) | △ 옵션 폼은 직관적이나 가격 연결이 끊김(돈 결함 위험) |
| **off-grid ceiling** | ✅ 격자 siz 룩업 + 런타임 ceiling(메모리 정합) | △ 옵션은 이산 선택지라 off-grid 개념 부재 |

### 3.3 잠정 권고 (결정은 Phase B)

**A안(기초코드 t_siz_sizes 등록) 권고.** 근거:
1. **가격엔진 매칭 규칙이 사이즈를 siz_cd로만 받음** — B안은 가격사슬을 벗어나 면적단가를 엔진이 조회할 경로가 없어진다(돈-크리티컬).
2. **라이브 선례 전건 A안**(실사·현수막·아크릴) — 기계적 신규 모델 금지(메모리 [[dbmap-schema-design-intent-first]]).
3. **t_siz_sizes가 좌표(work/cut 수치)의 정규 그릇**이며 좌표형 siz 383종 이미 등록 — 그릇이 이미 존재(search-before-mint).
4. **사이즈는 L1 차원**(가격표 권위)이지 L2 옵션이 아님(메모리 [[dbmap-l2-requires-l1-price-table]]).

**단, A안의 미해소 부담(Phase B 컨펌 대상):**
- **좌표 siz 채번 규모** — round-16은 687 면적조합 중 667(97%) 미실재로 봤으나, round-2 정정은 search-before-mint로 POSTER 106 NONE + ACRYL 105 NONE = **211 신규 등록**으로 산정(EXACT 68·REVERSED 29 재사용). **두 산정치 불일치** → Phase B에서 좌표 siz 실재율 재실측 필요(라이브 383 좌표 siz 대조).
- **영구 격자 전건 등록 vs 부분 등록** — round-2 "좌표 siz 등록 요청서" 범위 컨펌(Q-PS-3).
- **REVERSED 29종**(역방향만 존재) 재사용 여부 — 직접입력형이라 면적동일 시 재사용 정당 가능 vs 의미축(가로/세로) 보존(Q-PS-3 연계).

## 4. 부수 발견 — 가격사슬 부분단절 (돈-크리티컬·Phase B 보강)

라이브 실측 재확인:
- **28상품(PRD_000118~145) 전건 `PRF_POSTER_FIXED` 단일 바인딩** (BIND 실측).
- **`PRF_POSTER_FIXED` 배선 = 1 comp**(`formula_components` 1행, `COMP_POSTER_ARTPRINT_PHOTO`만) (WIRE 실측).
- → **인화지(PRD_000118) 외 27상품 엔진 조회불가**. 자기 소재 단가행이 적재돼 있어도 공식이 그 comp를 안 봄.
- **추가 결함:** round-2 정정은 고정가형 15·면적 13 분기를 권고했으나 라이브는 28상품 전건 동일 단일 공식 — 고정가/면적 혼재 미반영.

**해소 방향**(메모리 [[dbmap-price-chain-dwire-per-product-formula]]): 소재별/유형별 공식 `PRF_POSTER_<MAT>` 분리 + 상품→자기 공식 바인딩 교체. **컨펌 Q-PS-1**(단일공식+조건분기 vs 소재별 공식 — webadmin Phase11 엔진 설계 확인). 이는 사이즈 처리(항목 3)와 독립한 별개 트랙이나, 견적 성립의 전제이므로 Phase B에서 동반 정립 필요.
