## 1. B1 — 커팅(완칼) 가공비 (국4절) [A1:B38]

```
A1='커팅(완칼) 가공비 (국4절)'   C1='인쇄배경지'(라벨노트)
A2='수량(국4절)'  B2='단면'        C2='인쇄비+소재+커팅'(라벨노트)
A3=1   B3=2000
A4=2   B4=4000
...
A38=1000  B38=2000000
A39='1장당 2000원씩 올립니다.'(증분룰)
```

- **축**: `수량(국4절)` = 출력판형(국4절) 단위 수량. `단면` = 인쇄면(단면 인쇄).
- **데이터셀 36개**: 수량 1~1000(18개 이산 구간 아님 — 1~10 낱개 + 15~1000 점프).
- **가격 거동(실측)**: `가격 = 수량 × 2000`이 36행 전부 성립(검산 True). 즉 **장당 2,000원 × 국4절 수량 = 단가형**.
- **국4절 = 출력판형 = siz_cd** (메모리 [[dbmap-output-plate-mapping.md]] 권위). 라이브 = **SIZ_000499 "316x467"**(=국4절 판형). 단가행은 siz_cd=SIZ_000499로 고정.
- **단면(B2)**: 단면만 존재(양면 컬럼 없음). 단면 인쇄 = coat_side_cnt 차원 아님(완칼은 인쇄면 무관 가공) → 차원 비사용. "단면" 헤더는 가격표 라벨일 뿐 가격 분기 아님.
- **증분룰 A39**: "1장당 2000원씩" = B1 가격공식의 자연어 설명. 이산 구간을 넘어선 수량은 `수량×2000`. 별도 명시 노트로 보존(이산구간 외 룰).

### B1 라이브 정합 ✅
`COMP_CUT_FULL_DIECUT` 36행, siz_cd=SIZ_000499, min_qty=1~1000, unit_price=수량×2000 — **시트와 byte 일치**.
use_dims=`["siz_cd","min_qty"]`. PRF_DGP_B(모양엽서·라벨택)·PRF_DGP_F(썬캡)에 disp_seq=4로 배선.

## 2. B2 — 타공(단가) [A42:C66] — 전부 0원 placeholder

```
A42='타공 (단가)'  D42='헤더택 / 벽걸이캘린더'  (F42='타공 (합가)' → B3)
A43='옵션/ 제작수량'  B43='1구(6mm)'  C43='2구(6mm)'
A44=1     B44=0   (C44~C66 전부 빈칸)
A45=100   B45=0
...
A66=9000  B66=0
```

- **축**: 옵션(1구/2구·6mm) × 제작수량. 제작수량 23구간(1~9000).
- **데이터(실측)**: B(1구) 23행 **전부 0.00**. C(2구) **전부 빈칸**(데이터 없음).
- **해석**: 이 "단가" 블록은 **미사용 placeholder**. 0원은 "타공 단가 차원은 비워두고 합가(B3)로 가격을 매긴다"는 의미. D42='헤더택/벽걸이캘린더' = 이 단가표가 의도된 상품 라벨(미사용).
- **라이브 정합**: `COMP_CUT_PERF_1H6` 23행 **전부 0.00** — 시트와 일치. use_dims=`["min_qty"]`. PRF_DGP_C/D/E(헤더택·소량전단지·접지)에 배선됨(0원 기여 = 실질 무가격).

### B2 처리 판정
- **0원 단가 블록은 그릇에 보존하되 "미사용/0원" 플래그**. 침묵 삭제 금지 — 라이브에 실재(23행·배선됨)하므로 round-trip 보존 필수.
- 실 타공 가격은 B3(합가)가 담당. B2는 "단가형 경로가 비활성"임을 나타내는 의미 있는 데이터.

## 3. B3 — 타공(합가) [F42:H54] — 진짜 타공 가격

```
F42='타공 (합가)'
F43='옵션/ 제작수량'  G43='1구(6mm)'  H43='2구(6mm)'
F44=1     G44=2000    H44=4000
F45=100   G45=2000    H45=4000
F46=300   G46=4000    H46=8000
F47=500   G47=6000    H47=12000
F48=1000  G48=11000   H48=22000
F49=2000  G49=21000   H49=42000
F50=3000  G50=31000   H50=62000
F51=4000  G51=41000   H51=82000
F52=5000  G52=51000   H52=102000
F53='1구는 100장당 1000원씩 올립니다.'(증분룰)
F54='2구는 100장당 2000원씩 올립니다.'(증분룰)
```

- **축**: 옵션(1구/2구·6mm) × 제작수량(9구간 1~5000).
- **헤더가 직접 "합가"라고 명시** — 이 블록은 가격표 저자가 의도적으로 "합가"로 분류. 셀값 = **수량구간 총액**(100구간=2,000원 전체, 장당 아님).
- **2구 = 1구 × 2** (9행 전부 성립).
- **증분룰 F53/F54**: 1구 100장당 +1000, 2구 100장당 +2000 = 이산구간 외 외삽 룰. 별도 명시 보존.

### B3 라이브 정합 — 🔴 min_qty 구간 불일치 발견
- 라이브 `COMP_CUT_FULL_PERF_1H6`(1구) / `_2H6`(2구) 각 9행 존재. unit_price **시퀀스는 동일**(2000/2000/4000/6000/11000/21000/31000/41000/51000).
- **그러나 min_qty 경계가 다름**:
  - 시트(260527): 1 / 100 / **300 / 500 / 1000 / 2000 / 3000 / 4000 / 5000**
  - 라이브: 1 / 100 / **200 / 300 / 400 / 500 / 600 / 700 / 800**
- → 라이브는 **다른(이전) 버전의 구간 경계**로 적재됨. 가격값은 같으나 수량 매칭 지점이 다름.
- **판정**: 260527 시트가 권위 → 라이브 min_qty는 **stale/divergent**. import 그릇은 **시트 구간(300/500/1000/2000…)**으로 작성하고, 라이브 교정 필요 플래그.
- comp_nm "커팅 합가(완제품가)"인데 prc_typ_cd=**.01 단가형**으로 등록됨 → 헤더 "합가"와 충돌(§decomposition P4).
