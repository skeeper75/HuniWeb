## 0. 한 줄 결론

**Claude(라이브 실측)와 codex(계약 추론)가 합의한 NO-GO는 옳다.** 결함의 진원은 설계 골든값이 아니라
**라이브 prc_typ 오적재(전 고정가형이 단가형인데 단가는 묶음/구간 총액)** + **인쇄면 판별차원 부재(silent 이중합산)**.
교정 정답은 **prc_typ 단가형(.01) → 합가형(.02) 전환**이며(엔진이 `÷min_qty`로 장당가 환산), **명함은 추가로 min_qty=NULL 가드** 필요.
단 ① 명함 주문 UX(고정세트 vs 누적) ② 박 동판비 정액 의도 ③ 인쇄면 통합 방식은 **사용자/도메인 컨펌**이 선행해야 안전.

이번 심의에서 ★새로 확정한 사실(라이브 실측):
- **엔진 계약 확정**: `pricing.py:component_subtotal` — 단가형=`unit×qty`(unit=장당가 가정), **합가형=`(unit÷min_qty)×qty`(unit=구간총액 가정)·min_qty≤0/NULL이면 ValueError**.
- **합가형 선례 실재**: 라이브 PRICE_TYPE.02 comp 4건 중 `COMP_STK_PACK`(스티커 54장1세트·min_qty=54)·`COMP_STK_TATTOO`(3장세트·min_qty=3) = **명함/포토카드와 동형인 "N장 1세트 총액"을 이미 합가형으로 교정한 선례**(round-23 스티커). → 교정안 A는 후니 자체 선례를 따르는 것이지 신규 패턴이 아니다.
- **고정가형 두 하위패턴 분리**(트레이드오프 핵심): ⓐ단일구간(명함 8종·포토카드 SET, min_qty=100/1) ⓑ다구간 하향(PCB 39구간·BULK 50구간·FOIL 9구간, min_qty=주문수량 브래킷·unit=해당 브래킷 묶음총액). **두 패턴 모두 합가형 ÷min_qty가 정답**이지만 주문 UX 함의가 다름.

## 2. R-6/CV-1/CV-2 — 교정안 A vs B 트레이드오프

### 엔진 계약 (라이브 pricing.py:177-192 실측 — 트레이드오프의 자[尺])

```
단가형(.01): subtotal = unit_price × qty                    (unit = 장당가로 해석)
합가형(.02): subtotal = (unit_price ÷ tier_min_qty) × qty   (unit = 구간총액으로 해석)
             ★ tier_min_qty ≤ 0 또는 NULL → ValueError("수량구간 없어 장당가 환산 불가")
```

현 상태: 단가가 **구간총액**인데 prc_typ가 **단가형** → 엔진이 구간총액을 장당가로 오해 → ×qty 폭발.

### 교정안 A — 합가형(.02) 전환 (÷min_qty = 장당가×qty)

prc_typ만 .01→.02로 바꾼다(단가행 값 불변·verbatim 보존). 엔진이 `÷min_qty`로 장당가 환산 후 ×qty.

- 명함: 3500 ÷ 100 × 100 = 3,500 ✅ / 다른 수량 주문 시 35원/장 × qty (장당 누적)
- 엽서북: 브래킷 선택(min_qty 매칭) 후 (해당총액 ÷ 브래킷) × qty. 2매=11000÷2×2=11,000 ✅ / 20매=5200... ※주의 아래
- BULK: 9500 ÷ 100 × 100 = 9,500 ✅
- FOIL: 24800 ÷ 300 × 300 = 24,800 ✅

### 교정안 B — qty=100 고정 + 합가형(또는 현 단가형 유지)

주문 UX가 "100매 1세트 고정"이면 견적 qty를 항상 min_qty(100)로 고정 입력. 이 경우 현 단가형도 결과는 우연 정합(3500×... 아니라 엔진엔 qty=100이 안 들어가고 세트=1로 들어가야 함). 실무적으로는 **합가형 + UX가 "세트 수" 입력**.

### 트레이드오프 표

| 축 | 교정안 A (합가형 ÷min_qty) | 교정안 B (qty 고정세트 + 합가형) |
|----|---------------------------|--------------------------------|
| **전제(단가 의미)** | 단가=구간총액·고객은 **장당 누적 주문**(임의 수량) | 단가=세트총액·고객은 **고정 세트단위 주문** |
| **단가행 변경** | prc_typ .01→.02 (값 불변) | prc_typ .01→.02 + 주문 UX/엔진 qty 정규화(세트수) |
| **min_qty=NULL 리스크** | 🔴 SETUP·일부 행 min_qty=NULL이면 **ValueError**(견적 폭발 아닌 견적 불가) → R-4 동반 필수 | 동일(합가형이면 동일 가드 필요) |
| **다구간(ⓑ) 적합성** | ✅ 브래킷별 ÷min_qty가 볼륨디스카운트를 자연 표현(PCB/BULK/FOIL) | ⚠ "세트 고정"과 다구간 볼륨 하향이 충돌(고정세트는 단일가 가정) |
| **단일구간(ⓐ) 적합성** | ⚠ min_qty=100 단일이라 100매 미만 주문 시 35원/장 = 100매 미만도 견적 성립(가격표는 100매 단위 의도일 수 있음) | ✅ "100매 1세트"가 가격표 정신과 일치 |
| **주문 UX 함의** | 위젯 수량 슬라이더가 임의 수량 → 장당 누적 청구 | 위젯이 세트 수(100/200/...) 또는 수량 브래킷 택1 |
| **돈 안전성** | ×qty 폭발 차단·장당가 환산으로 항상 정합 | 폭발 차단·단 qty 정규화 누락 시 재발 위험(엔진 밖 의존) |
| **후니 선례** | ✅ COMP_STK_PACK(54장세트)·TATTOO(3장세트) 이미 합가형 ÷min_qty(round-23) | 선례 없음(세트수 입력 UX 미구현) |
| **codex 권고** | "prc_typ를 총액 의미에 맞게 재설계" = 합가형 = A | "qty를 엔진 전에 세트수량으로 정규화" = B (codex가 둘 다 제시) |

### ⓐ vs ⓑ 결정적 차이 (왜 단일 답이 아닌가)

- **ⓑ 다구간(PCB·BULK·FOIL)**: min_qty가 **주문수량 브래킷**이고 unit이 그 브래킷의 묶음총액 → **합가형 ÷min_qty가 명백한 정답**(볼륨디스카운트가 자연 표현됨·UX는 임의 수량). 교정안 A 단독으로 안전. **단, P3 tier 매칭이 "주문수량 → 해당 브래킷 행 선택"으로 동작해야** ÷min_qty가 맞는 장당가를 줌(브래킷 매칭 + ÷min_qty의 합성이 핵심). qty=20 주문 → min_qty=20 행(5200) 선택 → 5200÷20×20=5,200 ✅.
- **ⓐ 단일구간(명함·SET)**: min_qty=100/1 단일이라 "100매 단위 주문"인지 "장당 누적"인지 가격표만으로 불확정 → **CV-2 컨펌이 갈림길**. 100매 고정이면 A·B 결과 동일(qty=100 고정 시); 임의 수량 허용이면 A(÷100=35원/장)가 100매 미만도 견적 성립시키는데 **그게 가격표 의도인지** 사용자 확인 필요.

## 4. R-4/CV-4 — 박 SETUP(동판비) 정액 처리

**실측**: `COMP_NAMECARD_FOIL_SETUP_S1/S2_STD` — prc_typ=.01·**use_dims=[]**·단가행 1행·**min_qty=NULL**·unit=5000.

- 현 단가형: 5000 × qty → 300매 시 1,500,000 폭발(동판은 수량 무관 1회여야).
- 합가형 전환 시도: min_qty=NULL → **ValueError(견적 불가)**. 즉 단순 .02 전환만으론 SETUP은 안 됨.

**처리 옵션**:
1. **합가형 min_qty=1** — (5000÷1)×qty = 5000×qty → ×qty 폭발 그대로. ❌ 부적합(동판이 정액이 아님).
2. **정액 1회 보장** — 동판비는 "수량 무관 1회 정액 5000"이 의도면, 엔진이 qty를 안 곱하는 경로가 필요. 현 엔진(pricing.py)은 단가형·합가형 모두 ×qty/×qty(÷min_qty 후) → **순수 정액 표현 그릇이 엔진에 부재**.
   - 차선 a: **합가형 + 단가행 min_qty = "예상 주문수량과 동일"**은 불가(수량 가변).
   - 차선 b: **별도 처리 — SETUP을 가격엔진 comp가 아닌 "주문당 1회 고정비"로 분리**(엔진 밖 또는 엔진에 정액 prc_typ 신설). 단 정액 prc_typ는 라이브 미존재 → 그릇 부재.
   - 차선 c(현실적): **합가형 + min_qty=1 + addtn_yn 처리에서 qty=1 강제**(동판은 "수량" 차원을 안 받게 selections에서 동판 qty를 1로 고정). 엔진 변경 없이 selections 레벨에서 격리.

**★컨펌 선결(CV-4)**: 동판비 = 수량 무관 1회 정액 5000이 **가격표·실무 확정 의도인가**? 확정이면 위 차선 b/c 중 택. **미확정이면 추측 적재 금지** — designer/사용자 컨펌 큐로 분리.

**라우팅**: 정액 그릇 부재는 가격엔진 차원 신설 후보 → 확정 시 dbm-ddl-proposer(정액 prc_typ 또는 별도 fixed-fee 메커니즘) 검토. **단 webadmin pricing.py는 개발자 GitHub 배포·read-only — 엔진 변경은 개발자 백로그**.

## 5. CV-3 — 인쇄면(단/양면) 차원 통합 vs comp 분리

**실측 V-DGP-1**: PRF_NAMECARD_FIXED에 S1(단면)·S2(양면)가 둘 다 disp_seq·addtn_yn=Y로 배선·**print_opt_cd 판별차원 부재** → 단면 선택해도 S1+S2 둘 다 매칭·silent 합산(350,000+450,000=800,000). 엽서북 S1_20P+S2_20P·포토카드 SET+CLEAR_SET 동형.

| 방식 | 설명 | 가격 관점 평가 |
|------|------|----------------|
| **통합(comp 1개 + print_opt_cd 차원)** | S1/S2를 한 comp로 합치고 단가행에 print_opt_cd(단/양면) 충전·use_dims에 print_opt_cd 포함 → 인쇄면 선택이 정확히 1행 매칭 | ✅ **silent 이중합산 원천 차단**(comp 1개라 합산 대상이 1개). codex D-2 가드: 단순 컬럼 충전만으론 부족·**use_dims 배열에 print_opt_cd 추가**가 필수(매칭은 use_dims 기준) |
| **분리 유지(comp 2개) + 차원 충전** | S1/S2 별 comp 유지하되 각 단가행에 print_opt_cd 충전·use_dims 포함 → 선택 시 1개만 매칭 | ⚠ 충전이 완벽하면 동작하나, **둘 다 배선된 상태에서 한쪽이 차원 미충전이면 재발**. comp 2개라 silent 합산 표면적이 남음 |

**가격 관점 권고: 통합(comp 1개)** — 이중합산은 "둘 다 배선됨 + 판별 부재"의 곱이다. comp를 1개로 만들면 곱의 한 항(둘 다 배선)이 사라져 **구조적으로** 이중합산 불가능. 분리 유지는 차원 충전의 정확성에 의존(운영 중 한 행 누락 시 재발). **단 통합은 단가행 재구성(S1·S2 행을 print_opt_cd 차원으로 병합)이 필요**해 변경량이 큼 → designer 폐루프에서 단가행 병합 명세 + use_dims 포함 라이브 재확인(codex D-2 미검증 가설 해소).

**주의**: 엽서북은 인쇄면(S1/S2)뿐 아니라 **페이지수(20P/30P)** 차원도 동시매칭 위험(현재 20P만 배선이라 잠복). 통합 시 print_opt_cd + page 두 차원 모두 충전해야. 포토카드 SET/CLEAR_SET은 인쇄면이 아니라 **소재(투명 여부) 판별** → mat_cd 또는 별 차원으로 분기 필요(SET+CLEAR_SET 동시 배선도 이중합산).

## 6. 사용자가 컨펌해야 할 결정점 (CV-1~CV-4 — 권고 방향 포함)

> 아래는 **추측 적재 금지** 항목. 권위 = 상품마스터260610·가격표260527·실무 주문정책. 미확정 시 컨펌 큐 유지.

### CV-1 — 고정가형 단가 의미 = "묶음/구간 총액"인가 "장당가"인가?
- **권고 방향: 묶음/구간 총액으로 확정 → 전 고정가형 합가형(.02) 교정.**
- 근거: 명함 3500=100매 세트 총액(장당 3500이면 100매 35만원=시장 비현실)·PCB 다구간 하향(2매 11000 > 100매 4500 = 장당가가 아니라 묶음총액의 볼륨디스카운트)·후니 선례(STK_PACK 54장세트 합가형). **단가형 가정(장당 3500)은 ×qty로 350,000을 내므로 그 자체가 반증.**
- 영향: 명함/엽서북/포토카드BULK/박 전 고정가형 견적 정합. **이 한 칸이 NO-GO의 핵심**.

### CV-2 — 명함 주문 UX = "100매 고정세트"인가 "장당 누적 주문"인가?
- **권고 방향: 가격표가 100매 단위 단일구간이므로 "수량 브래킷(100/200/...) 택1" UX 권장. 100매 미만 견적 허용 여부만 명시 확정.**
- 합가형(A)이면: 100매 주문=3500 정합·임의 수량(예 150매)=35원×150=5,250 자동 산출. 이게 허용이면 A 단독; 100매 배수만 허용이면 위젯 수량 제약(CPQ) 추가.
- 영향: prc_typ 교정 방향(A 확정) + 위젯 수량 입력 제약 설계.

### CV-3 — 인쇄면(단/양면) = comp 통합인가 분리+차원 충전인가?
- **권고 방향: 통합(comp 1개 + print_opt_cd를 use_dims에 포함) — silent 이중합산 원천 차단.**
- 동반: 엽서북 페이지수(20P/30P)·포토카드 소재(SET/CLEAR_SET)도 같은 통합/판별 원칙 적용. 단가행 병합 + use_dims 라이브 재확인(codex D-2).
- 영향: V-DGP-1 해소법·단가행 재구성 변경량.

### CV-4 — 박 동판비 = 수량 무관 1회 정액 확정인가?
- **권고 방향: 정액 5000 1회가 가격표·실무 의도이면 확정 표기. 단 현 엔진에 정액 그릇 부재 → selections 레벨 qty=1 격리(차선 c) 또는 정액 prc_typ 신설(개발자 백로그) 중 택.**
- 미확정이면 적재 보류(추측 금지). 가격표 후가공_박 시트 동판비 셀 + 실무 주문정책 확인 필요.
- 영향: R-4 처리·박 견적 폭발(1.5M) 차단.

### 보조 컨펌(designer 큐 유지·이번 심의 범위 밖)
- 형압명함 comp 부재(FOIL 공유 vs 별 comp)·봉투류 추가상품/완제품 경계(G-6b)·엽서북 내지/표지 BOM 바인딩 불요 확정(S-1)·option_items→print_opt_cd 자동주입 미연결(G-7).

- 정액 의도는 CV-4로 확정(보류 해제)·차선 택은 개발자 협의.

### 10-3. 인쇄면/페이지/소재 = 통합 판별차원 `CV-3 확정`

| 공식 | 현재(결함) | 교정 |
|------|-----------|------|
| PRF_NAMECARD_FIXED 등 | S1+S2 둘 다 배선·print_opt_cd 부재→silent 합산 | **comp 1개 통합**·print_opt_cd 충전·**use_dims에 print_opt_cd 추가**(codex D-2: 매칭=use_dims 기준) |

### 10-4. 주문 UX = 수량 브래킷 택1 `CV-2 확정`

명함=제작수량 100 단일·엽서북/BULK/FOIL=다구간 브래킷. 위젯 수량=**가격표 제작수량 브래킷 택1**(임의 자유입력 아님)·합가형 ÷min_qty가 브래킷 단가→장당가→×qty. off-grid 정책(브래킷 사이=다음 ceiling vs 보간)은 위젯/엔진 설계·브래킷 택1이 가장 안전. CPQ 수량 제약(enum/step) 표현.
