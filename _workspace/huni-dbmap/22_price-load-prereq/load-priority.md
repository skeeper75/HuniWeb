## 3. 우선순위 권고 (착수 순서)

가격사슬을 "고아 없이 + 최소 신규로" 완결하는 순서. 위험·선행조건이 가벼운 것부터.

1. **[즉시·무위험] 배선/바인딩 단절 닫기 — 단가행이 이미 멀쩡한 것**
   - 떡메 바인딩 +1행(PRD_000097→PRF_TTEOKME_FIXED): 단가행 112·공식·comp 다 있고 상품도 있음. **INSERT 1행으로 전건 사장 해소**.
   - 엽서북 30P 배선 +2행(S1_30P seq3·S2_30P seq4): 단가행 234행 사장 해소. **INSERT 2행**.
   - → 둘 다 상품/차원/단가행 추가 적재 **불요**. 가장 낮은 위험·높은 효과.

2. **[교정 묶음] 합판 prc_typ .02 정정 + mat 6분해 단가행** — P-4 안전성 컨펌 후 한 트랜잭션.

3. **[배선+교정] 커팅 타공합가** — P-5(prc_typ .02 + 배선 대상 상품) 결정 후.

4. **[siz 선적재 의존] 스티커 타투/팩·아크릴 본체·포스터 27상품** — P-1/P-2/P-3 결정 + siz 채번(단계00) 선행 후. 가장 무거움(공식+comp+배선+바인딩+siz 다단계).

## 4. 비전문가용 한 단락 요약 (쉬운 한국어)

**가격을 DB에 넣기 전에 상품마스터를 통째로 먼저 넣어야 하느냐? — 대부분 아니다.** 가격이 가리키는 상품(아크릴키링·합판스티커·엽서북·떡메·포스터 28종 등)과 크기/소재 값은 이미 거의 다 DB에 들어 있다. 진짜로 빠진 건 "이 상품은 이 가격공식을 쓴다"는 **연결고리**(공식 정의·배선·상품-공식 묶음)다. 그래서 ① 떡메·엽서북처럼 연결고리 한두 줄만 이으면 바로 가격이 살아나는 것(가장 먼저·위험 없음), ② 합판·커팅처럼 단가/합가 표시가 틀려 먼저 고쳐야 하는 것, ③ 아크릴·스티커 일부·포스터처럼 새 크기(siz)부터 등록하고 공식 전체를 새로 만들어 연결해야 하는 것 순서로 진행하면 된다. 즉 **순서는 "새 크기 등록 → 새 구성요소 → 새 공식 → 단가표 → 배선 → 상품 연결"이고, 대부분의 상품과 크기는 이미 있으니 새로 넣을 건 그 사이를 잇는 연결고리뿐**이다. 실제 DB 쓰기는 모두 사람 승인 후에 한다.

## 5. 한 줄 현황

가격 적재 우선순위 정리 완료 — ⓐ선결0(디지털·코팅 재현) / ⓑ차원·comp 선적재(siz 미채번 3시트·comp 신규 3종) / ⓒ교정선행(합판·커팅 prc_typ·실사매칭) / ⓓ상품미적재 0(GO 8시트 상품 전부 실재). FK 위상정렬 7단계(siz·comp → 공식 → 단가행/교정 → 배선 → 바인딩). **착수 권고: 떡메 바인딩+엽서북30P 배선(무위험·즉효) → 합판/커팅 교정 → siz의존 신규(아크릴·스티커타투·포스터).** 컨펌 6건(P-1~6) 인간 결정 대기. **다음 = dbm-validator G1~G9 검증.**
