## 1. codex 신규 발굴 (inspector 놓친 gap) — Claude 재실측 확정

### NEW-1 [확정] 누락 셀 수 산술 불일치 — "전부 하삼각" 성립 불가
- **codex 적발:** 권위 CSV 기준 w>h 셀 = B01 **91**·B02 **36**·B03 **36**·B05 **15**. inspector 보고 누락 = B01 83·B02 29·B03 29·B05 15.
- **Claude 재실측(권위 CSV·확정):** 삼각분포 = B01(w<h 91/w=h 14/w>h 91)·B02(36/9/36)·B03(36/9/36)·B05(15/6/15). **codex 수치 정확.**
- **결론:** "미적재 83=전부 하삼각(w>h)"은 **거짓**. w>h가 91개인데 83개만 누락 → 8개 차이가 어디인지 불명(하삼각 일부 적재됐거나, 누락이 하삼각 외에도 분포). B02/B03도 36 vs 29 = 7개 차이. **inspector의 "라이브는 상삼각만, 하삼각 대칭쌍 83/83 확인"은 산술적으로 불가능** — 라이브 113셀의 실제 (w,h) 좌표를 재추출해 정확한 missing diff를 내야 함.
- **잔여 리스크:** 누락 셀 좌표가 부정확하면 교정 적재 SQL이 틀린 셀을 INSERT. **게이트 필수 재실측 항목.**

### NEW-2 [확정] B01/B02/B03 완전 대칭 아님 — 비대칭쌍 3건
- **codex 적발:** CSV에 w30h50≠w50h30: B01 3700↔3800·B02 2960↔3040·B03 7400↔7600(B01×2 전파).
- **Claude 재실측(권위 CSV·확정):** 정확히 3쌍 일치. **codex 정확.**
- **결론:** inspector·권위 dims 문서의 "가로↔세로 완전 대칭(w20h30=w30h20)" 전제가 **부분 거짓**. 1쌍(w30/w50×h30/h50)은 권위값이 다름(8500 vs 8700 류 80~200원 차). **대칭 전개로 누락을 메우면 이 좌표는 오가격**(정합불일치 결함 유발). 교정 적재는 대칭 추정 금지·권위 CSV verbatim 전개해야 함.
- **잔여 리스크:** 만약 라이브가 이미 대칭 가정으로 적재됐다면 이 3좌표(×2 두께/미러 = 최대 9셀)는 **정합불일치(오적재)** — inspector "정합불일치 0" 판정도 재검 필요.

### NEW-3 [보강·게이트 검사] chain/link audit 2건
- **B04b 상품그룹 귀속:** 엔진(`pricing.py:651,93`)은 단가행에서 상품그룹을 안 보고 frm_cd/opt_cd 매칭으로 흡수. → 적재된 4옵션이 **올바른 상품 공식에만** 붙었는지 별도 chain audit 필요(키링 고리가 뱃지에 안 붙는지). inspector는 개수(4/26)만 봄.
- **B07 카라비너 할인 링크:** 할인은 상품별 `TPrdProductDiscountTables` 링크(`pricing.py:703`)로 적용. B07 3구간 적재만으론 부족 — PRD_000166에 실제 링크됐는지(안 되면 할인 0)·B04a가 잘못 링크됐는지(과할인) 게이트가 SELECT 확인.

## 2. codex 과적발 적발 (inspector가 결함으로 본 것 중 정당한 차이) — 0건

codex는 inspector의 결함 판정 중 **false-positive(과적발)를 찾지 못함** — 6 돈크리티컬 전부 CONFIRM. 오히려 inspector가 false-positive로 미리 가드한 3건(할인 스케일·도수 통용·B07 분리)을 codex가 독립적으로 **동일하게 REFUTE=정상** 판정 → inspector 과적발 0 확증.

단, codex는 inspector의 **기술(서술) 오류 1건**을 적발: D-04 코롯토를 inspector가 "비대칭 격자"로 기술했으나 codex 실측상 **B05는 대칭**(w<h=w>h=15·비대칭쌍 0). 결함 유무(15셀 미적재)는 동일하나 원인 기술이 틀림 — 교정 시 대칭 전개 가능(D-01류와 동일 패턴).
