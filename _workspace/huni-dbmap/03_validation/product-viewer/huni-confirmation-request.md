## D-A. 색상/마감 자재(mat_cd) 등록 — AWK-6 확장 (105건)

> 105 = MAT_COLOR_IN_NAME 68 + MAT_COLOR_ONLY 27 + MAT_ATTR_COMBO 10(색상+크기 복합, 의류 — 색상 축이 mat_cd 등록 대상이므로 D-A에 포함; 크기 축은 D-B와 교차).

**필요한 결정**: 색상/마감별 가격이 다른 variant를 `t_prc_component_prices.mat_cd` 차원으로 적재하려면, 해당 색상/마감을 `t_mat_materials`에 자재로 등록할 것인가, 아니면 색상별 별도 SKU(prd_cd)로 분리할 것인가.

**가격매핑을 막는 이유**: 규칙③(variant 색상/재질→mat_cd)에 따라 색상 variant 단가를 적재하려면 mat_cd가 부모 테이블에 선존재해야 한다(FK). 현재 색상/마감이 자재명에 텍스트로만 박혀 있거나 자재 마스터에 없어 variant 단가행의 차원 키를 만들 수 없다.

**영향 상품/자재(대표)**:
- **머그컵(PRD_000193)** — 색상(화이트/반투명/투명). **AWK-6 원본 블로커**. 자재 마스터에 머그바디 색상 자재 부재.
- **아크릴 30+종(PRD_000146~169, 226)** — 투명/홀로그램/유광/미러/골드실버. 면적+색상 이중 variant.
- **스티커 5종** — 반칼/낱장 투명·홀로그램(PRD_000053/054/056/063/066).
- **책자/다이어리 6종** — 레더 등 커버 색상/소재(PRD_000071/072/077/082/088/174/175).
- **캘린더 8종(PRD_000108~117)**, **미니매트/피크닉매트(PRD_000197/198)**, **만년스탬프(PRD_000217)·리필잉크(PRD_000015)**, **와이어링(PRD_000007)·미니우치와키링(PRD_000227)** 등.

**옵션/권고**:
- (권고) 색상/마감이 **가격에만 영향**하고 디자인 동일 → `t_mat_materials`에 색상자재 등록 후 mat_cd variant. 스키마 무변경(데이터 추가)이며 product_prices PK 충돌 회피(규칙③ 근거).
- (대안) 색상이 **별도 상품 정체성** → 색상별 prd_cd 분리(SKU). 단 상품 수 급증.
- **별색(화이트/클리어/핑크/금은)은 D-A 아님** — 별색은 색상자재가 아니라 **공정**(PROC_000007~012, 규칙①)이며 단가는 디지털인쇄비 F~O 시트. 'PDF(+W/+WC/+P/+GS)' 표기 상품(엽서/명함/스티커 PRD_000019~022 등)은 별색 공정으로 처리 — 자재 등록 불요.

## D-B. 용량·개수·사이즈 variant 귀속 규칙 — AWK-5 확장 (55건)

**필요한 결정**: "사이즈(필수)" 칸 또는 자재 슬롯에 들어간 **용량(11온스/350ml/500ml)·개수(2구/3구/4구/2개1세트/2개1팩)·치수(S/M/L/XL)·형태(원형/사각)**를 각각 어느 차원으로 귀속할지. siz_cd(치수)인가, mat_cd(자재)인가, 묶음수 bdl_qty인가, 별도 옵션인가.

**가격매핑을 막는 이유**: 이 값들이 variant 가격축(예: 11온스 vs 다른 용량 단가 상이)인데 자재 슬롯에 혼재해 있어, component_prices의 어느 차원(siz_cd/mat_cd/bdl_qty)에 매핑할지 미정이면 단가행 키를 확정할 수 없다. 차원이 잘못되면 자연키 UNIQUE8 충돌 또는 잘못된 룩업.

**영향 상품/사이즈(코드)**:
- **머그컵(PRD_000193)** '11온스', **워터북보틀(PRD_000194)** '350ml/500ml' — 용량.
- **키캡키링(PRD_000202)·LED투명키캡키링(PRD_000203)** '1구/2구/3구/4구' — 개수(묶음 bdl_qty 후보).
- **천정고리(PRD_000008)** '2개1세트', **자석북마크(PRD_000214)** '2개1팩' — 세트수.
- **파우치 14종(PRD_000230~274)** 'S/M/L/XL' — 치수. round-1이 `t_prd_product_sizes`로 이미 링크 → siz_cd 귀속이 자연스러움.
- **레더 아이패드/노트북 파우치(PRD_000238)** '11/13/15인치' — 치수.
- **말랑키링(PRD_000221)·레더 미니파우치 3종(PRD_000252/254/255)** '원형/사각' — 형태(MAT_SHAPE_ONLY).

**옵션/권고**:
- (권고) **용량/치수(온스/ml/인치/S·M·L)→siz_cd**(규칙③: 사이즈 variant→siz_cd, 파우치는 product_sizes 기존 링크 활용), **개수/세트(2구·2개1세트)→bdl_qty**(묶음수 차원, t_prd_product_bundle_qtys), **형태(원형/사각)→옵션 또는 mat_cd**. 자재 슬롯에서 분리.
- (대안) 단일 차원 일괄 처리는 자연키 의미 왜곡 위험 → 비권고.
- 결정 시 **파우치 S/M/L은 round-2에서 신규 매핑하지 말 것** — round-1 product_sizes 링크 재사용, base 단가만 적재(규칙⑨, AWK-9).

## D-C. 면적상품 작업·재단 치수 등록 — NEW(N-1) (86건 / 29종)

**필요한 결정**: 면적기반 가격(면적매트릭스형) 상품 29종의 비어있는 작업/재단 치수(work_width·work_height·cut_width·cut_height)를 후니가 등록할 것인가. 또는 면적이 아닌 다른 단가 방식(고정가/사이즈룩업)으로 가격을 매기는가.

**가격매핑을 막는 이유**: 규칙⑤(면적형 곱셈=공식외부, 매트릭스 셀=룩업)에서 단가는 siz_cd 차원으로 component_prices에 적재하고 면적은 치수로 계산한다. work/cut 치수가 모두 NULL이면 면적을 산출할 수 없고 siz_cd 룩업 키도 불완전 → 면적매트릭스형 매핑 착수 불가.

**영향 상품(29종, prd_cd)**:

**옵션/권고**:

## D-D. 완제품가 comp_typ_cd 유형 신설 여부 — AWK-7 (구조 결정)

**필요한 결정**: 굿즈/완제품의 "상품가/완제품가"가 `PRC_COMPONENT_TYPE` 5종(인쇄비/코팅비/용지비/후가공비/박형압비) 어디에도 맞지 않는다. (가) comp_typ_cd를 **NULL로 적재**(스키마 무변경)할 것인가, (나) "상품가/완제품가" 코드를 **신설**(데이터 추가)할 것인가.

**가격매핑을 막는 이유(약차단)**: 완제품가를 component로 표현할 때 유형 분류가 없으면 comp_typ_cd가 빈다. nullable이라 NULL 적재는 가능하나, 다수 DIRECT 상품(D-A/B/C 상품 대부분이 완제품가)에 일관 규칙이 없으면 분류 일관성·후속 운영에 혼선.

**영향**: DIRECT 246건 상품 다수(굿즈/파우치/아크릴/완제품 전반). 단가/합가를 component로 적재하는 모든 케이스.

**옵션/권고**:
- (권고) comp_typ_cd **NULL 적재**(스키마 무변경, 규칙 즉시 적용 가능). 완제품가는 대부분 `t_prd_product_prices`(상품단가)로 가므로 component 분류 자체가 불필요한 경우 많음.
- (대안) 운영상 분류가 필요하면 `PRC_COMPONENT_TYPE`에 "상품가/완제품가" 코드 1종 추가(데이터 추가, 코드 마스터만, 스키마 무변경).

## D-E. apply_ymd 전역 일자 형식 표준 — AWK-8 (go-live 전 결정)

**필요한 결정**: round-1 `t_prd_product_discount_tables`는 `20260601`(yyyyMMdd 8자리), DDL/스펙 comment는 `yyyy-MM-dd`(10자리). 프로젝트 전역 가격 효력일 형식을 **1개로 확정**.

**가격매핑을 막는 이유**: `t_prc_component_prices.apply_ymd`·`t_prd_product_prices.apply_ymd`는 varchar(10) NOT NULL이며 자연키 UNIQUE8의 구성요소. round-1과 형식이 다르면 라운드 간 가격 효력일 비교·정합이 깨지고, 잘못된 형식은 길이/정합 오류.

**영향**: round-2 가격 적재 전체(component_prices·product_prices 모든 행).

**옵션/권고**:
- (권고) **go-live 시점에 실제 일자 일괄 주입** + round-1과 동일 형식으로 통일. DDL이 varchar(10)이므로 `'2026-06-01'`(yyyy-MM-dd) 채택 시 round-1(`20260601`)도 같은 형식으로 정정 권장. 파일럿은 잠정 round-1 정합값(20260601) 사용 중.
- 결정 1개만 하면 됨(형식 + go-live 일자).

## D-F. (참고) INDIRECT·옵션 정규화 확인 항목 — 차단 아님

가격 fallback이 있어 매핑을 막지는 않으나 옵션 정규화 시 후니 확인이 도움되는 항목:
- **N-2 인쇄면 비표준 모드 63건**(PRINTSIDE_INVALID, 아크릴 '배면양면·투명테두리·풀빼다'): 이 값들이 가격에 영향하는 인쇄옵션인지(별도 comp_cd 필요) 단순 표기인지 확인.
- **N-4 단면/양면이 자재로 등록 19건**(MAT_PRINTSIDE_ONLY): print_options로 회수하되, 단/양면 단가차이 존재 여부 확인(규칙②).

이들은 결정 없이도 매핑 로직(공정/옵션 정규화)으로 진행 가능하므로 우선순위 Low.
