## 1. "완제품가 .06이라 비교 불가" 의심을 먼저 반증 (돈크리티컬 가드)

지시문이 경고한 함정: 라이브 명함박 본체 comp `COMP_NAMECARD_FOIL_S1_STD`는 **comp_typ_cd=PRC_COMPONENT_TYPE.06**, comp_nm="오리지널박명함 완제품가 단면·일반박(**종이+동판+박**)"이다. 이름만 보면 종이+동판이 합산된 **복합 완제품가**라서, 박가공비 단독 권위 64,000과 직접 비교하면 안 되는 것처럼 보인다. → 표면 숫자 일치로 자동교정하지 말 것.

**반증(라이브 단가행 전수 대조·2026-06-30 SELECT):** S1_STD의 9개 수량구간 단가가 가격표 박(소형) B03 **일반박 E등급(M열)** 박가공비 단독값과 **글자 그대로 일치**한다(이름과 달리 실제 숫자는 박가공비 단독).

| min_qty | 라이브 S1_STD | 권위 small 일반박 E열(M열) cell | 일치? |
|---|---|---|---|
| 200 | 19,200 | M10 = 19,200 | ✅ |
| 300 | 24,800 | M11 = 24,800 | ✅ |
| 400 | 30,400 | M12 = 30,400 | ✅ |
| 500 | 36,000 | M13 = 36,000 | ✅ |
| 600 | 41,600 | M14 = 41,600 | ✅ |
| 700 | 47,200 | M15 = 47,200 | ✅ |
| 800 | 52,800 | M16 = 52,800 | ✅ |
| 900 | 58,400 | M17 = 58,400 | ✅ |
| **1000** | **63,000** | **M18 = 64,000** | **✗ (이 1셀만 불일치)** |

9셀 중 8셀이 권위 E등급 박가공비와 verbatim 일치 → **이름표("종이+동판 합가")는 마케팅/설명 텍스트일 뿐, 적재된 실제 숫자는 박가공비 E등급 단독**임이 통계적으로 확정. 종이비·동판비가 더해진 복합값이라면 8셀이 우연히 권위 박가공비와 정확히 일치할 수 없다.

**동판비는 별도 comp로 실재**: `COMP_NAMECARD_FOIL_SETUP_S1_STD`(comp_typ .05·prc_typ .03·unit_price=5,000) = 권위 소형 동판비 5,000(small B01 B3) verbatim. PRF_NAMECARD_FOIL = S1_STD(박가공비) + SETUP_S1_STD(동판비 5,000) 2행 합산 → 골든 G-F4의 "5,000 + 64,000 = 69,000" 구조와 정확히 동형. 즉 동판비는 본체에 안 들어있고 별도 합산이므로, 본체 = 박가공비 단독이 맞다.

∴ **"완제품가라 비교 불가" 의심은 반증됨 — 본체 comp = 박가공비 E등급 단독이고, 권위 64,000과 동일 차원이다.** 따라서 63,000 vs 64,000은 같은 셀 대조이며, 1셀 오적재가 성립한다(거짓양성 아님).

## 2. 판정 = REAL 1-cell mis-load (근거 + 적재로직 원인)

- **무엇이 틀렸나**: `COMP_NAMECARD_FOIL_S1_STD` min_qty=1000 행 unit_price=**63,000** (comp_price_id=3385). 권위 = small-l1.csv B03 M18 = **64,000**.
- **왜(oracle)**: 권위 가격표 박(소형) 일반박 E등급 1000구간 = 64,000. 같은 comp의 다른 8개 수량구간이 전부 권위 verbatim → 1000구간만 1자리 전사 누락(64→63). 설계 `design-decisions-foil-rev3.md` §4·codex 합의로 "1셀 오적재" 불변 확정.
- **적재로직 원인(추정·재구성)**: load 시점에 E열 1000구간 값 64,000을 63,000으로 1자리 잘못 입력한 **단순 전사오류**(systematic 변환 결함 아님 — 다른 셀·다른 등급·HOLO·SETUP 전부 정확하므로 변환 규칙 자체는 건전). load_master/가격뷰어 그리드 수기 입력 단계의 오타에 가까운 sparse 1-cell 결함.
- **반증된 설계자 추정**: 초기 designer는 "동판비 미합산(63,000=64,000−1,000은 우연)"을 의심했으나, SETUP 5,000이 별도 comp로 실재(동판비는 합산돼 있음)하므로 **그 추정은 반증**. 순수 1셀 단가 오타가 정답.

### 동일 결함 2건 (범위)
| comp | 1000구간 라이브 | 권위 | 바인딩 | 위상 |
|---|---|---|---|---|
| **COMP_NAMECARD_FOIL_S1_STD** | 63,000 | 64,000 | PRF_NAMECARD_FOIL(✅ 바인딩) | **MIS-LOADED·실 견적 영향** |
| **COMP_NAMECARD_FOIL_S2_STD** | 63,000 | 64,000 | 미바인딩(어느 공식에도 없음) | MIS-LOADED·잠재(양면 바인딩 시 영향)·전 9셀 동일 E열·1000만 오타 |
| COMP_NAMECARD_FOIL_S1_HOLO / S2_HOLO 1000 | 92,000 | small 특수박 E등급 1000(M41)=92,000 | — | ✅ CORRECT(미터치) |
| COMP_NAMECARD_FOIL_SETUP_* | 5,000 | small 동판비 5,000 | — | ✅ CORRECT(미터치) |

두 결함 모두 64,000으로 교정(권위 verbatim). HOLO·SETUP은 권위 일치이므로 건드리지 않는다.

## 3. 돈 영향 (정량)

- **S1_STD(바인딩)**: 1000매 단면 일반박 명함 주문 = 박가공비 63,000 + 동판비 5,000 = **68,000 청구**(라이브). 권위 = 64,000 + 5,000 = **69,000**. → **건당 1,000원 저청구**. 1000구간 1셀에 국한(200~900구간은 정확). 박가공비는 작업 1건 고정금액(×qty 아님)이라 수량 폭증 없음·정확히 1,000원 차이.
- **S2_STD(미바인딩)**: 현재 견적 영향 0(어느 공식에도 안 걸림). 양면 명함박 공식이 추후 연결되면 동일 1,000원 저청구가 발생하므로 선제 교정 권장.

심각도: **Low~Medium**(건당 1,000원·1구간 한정·저청구). 돈크리티컬 폭주(×qty)는 아님. 단 권위 불일치이므로 교정 대상.

## 4. MAT_000260 중복 진단 (삭제 금지 [HARD] — 진단만)

| mat_cd | mat_nm | mat_typ | use_yn | del_yn | 바인딩 |
|---|---|---|---|---|---|
| **MAT_000250** | `아트250+무광코팅` | .01 | Y | **N**(활성·canonical) | product_materials 8건·정상 |
| **MAT_000260** | `아트250 + 무광코팅`(공백만 다름) | .01 | Y | **Y**(논리삭제됨) | product_materials 0·component_prices 0·**완전 고아** |

- **어느 두 mat_cd인가**: MAT_000250(생존) ↔ MAT_000260(논리삭제). 차이 = "+" 양옆 **공백**뿐("아트250+무광코팅" vs "아트250 + 무광코팅") = 동일 자재의 표기 중복.
- **어디에 묶여 있나**: 모든 바인딩(8 product_materials)이 **MAT_000250**에 있고, MAT_000260은 product_materials·component_prices 어디에도 참조 0.
- **merge 필요한가**: **불필요.** 중복은 **이미 정상적으로 해소된 상태** — 중복본(MAT_000260)이 del_yn=Y로 논리삭제되고, 모든 참조가 canonical(MAT_000250)로 모여 있다. 이것이 SOT 권고(통합=신규/canonical 유지 + 중복 use/del 토글, 삭제 아님)와 정확히 일치하는 종착 상태다.
- **조치**: **없음(NO-OP).** MAT_000260을 **삭제하지 말 것**([HARD]·기초마스터 코드 DELETE 금지). 이미 del_yn=Y로 무력화돼 조회/BOM/가격에서 빠져 있어 추가 작업 불요. (별 merge load-set 미빌드 — 만들 것이 없음.)

## 5. 교정 load-set (빌드됨·인간 승인 대기)

| 파일 | 역할 | 종결 |
|---|---|---|
| `namecard-foil-g6-load.sql` | 멱등 UPDATE(63000→64000·가드: unit_price=63000인 행만) · 2 comp 1000구간 | `COMMIT` 주석·`ROLLBACK` 종결(승인 후 COMMIT 주석 해제) |
| `namecard-foil-g6-dryrun.sql` | BEFORE/AFTER + 멱등 2회차 확인 후 무조건 `ROLLBACK` | `ROLLBACK`(라이브 무변경) |
| `namecard-foil-g6-undo.sql` | 역연산 UPDATE(64000→63000) | `COMMIT` 주석·`ROLLBACK` 종결 |

**DRY-RUN 실행 결과(2026-06-30·ROLLBACK 안전)**:
```
BEFORE: S1_STD=63000  S2_STD=63000
UPDATE 2          ← 1차: 2행 교정(63000→64000)
AFTER : S1_STD=64000  S2_STD=64000
UPDATE 0          ← 2차(멱등 재실행): 0행 영향
ROLLBACK          ← 라이브 무변경
```
사후 SELECT로 라이브 두 행이 여전히 63000임을 확인 → ROLLBACK이 읽기전용 안전을 보장(실 COMMIT 0).

- **멱등성**: ✅ `AND unit_price=63000` 가드로 재실행 시 0행(이미 64000이면 NO-OP).
- **단가 verbatim**: ✅ 64,000은 권위 small-l1.csv M18 그대로(날조 0).
- **비파괴**: ✅ UPDATE만(DELETE/DDL 없음)·COMMIT 주석·동판비/HOLO 미터치.
