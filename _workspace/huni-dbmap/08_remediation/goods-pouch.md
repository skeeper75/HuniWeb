# goods-pouch(굿즈파우치) 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 **누락·오매칭·모호**가 생겼다.
> 본 문서는 그 결함을 *올바로 처리(수정)*하기 위해 **라이브 Railway DB 직접 read-only SELECT를 권위**로
> goods-pouch 상품군(103상품, 시트 최대)을 종단 규명한다. 도메인 지식(07_domain KB)은 결함을 옳게 분류하는 렌즈다.
> 정제된 DB가 인쇄 자동견적 위젯의 기반이 된다.
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본 ref-CSV·v2는 stale 가능). 추정·임의판단 0 —
> 라이브·엑셀 L1·도메인 KB 실재 근거만. 미지/추정은 가설+컨펌 플래그. INSERT/UPDATE/DDL 0. DB 미적재 유지.
> `db railway`·비표준포트·JOIN KEY=`prd_nm`.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. **라이브 SELECT 수행 약 14회**(전부 read-only).

## ⚠️ 머리말 — 본 검증이 v2 대비 뒤집은 / 새로 확정한 가장 큰 사실 (권위 반전)

라이브 직접 확인 결과, 추출본·v2만 보면 알 수 없던 두 가지를 확정했다:

1. **그레이밴딩의 정체 = "DB 미적재 마킹"이며 정확히 폰케이스 5상품에만 존재한다.**
   엑셀 회색배경(`fill_rgb=FFD9D9D9`) 셀 3193개를 추적하니 **슬림하드 폰케이스·블랙젤리·임팩트 젤하드·
   에어팟케이스★·버즈케이스★ 5상품에만 집중**(1240/961/775/155/62). 이 5상품은 **라이브 `t_prd_products`에
   어떤 표기로도 미등록**(fuzzy `%케이스%/%젤리%/%하드%/%폰%` 전수 검색해도 부재).
   → **회색밴딩 = 적재자가 "아직 안 넣음/준비중"으로 표시하고 건너뛴 상품**. 사용자가 지목한 "그레이밴딩 비활성"의
   정체가 **품절이 아니라 상품 자체 결손(5상품 미생성)**임을 라이브로 확정. 이는 가장 큰 BLOCKER다(G-GP-1).
   행숨김(`_row_hidden`)은 0행 — 비활성은 행숨김이 아니라 **셀 회색배경**으로 표시됨.

2. **size(재단치수) 적재가 거의 전무하다 — plate(작업사이즈)만 적재됐다.**
   엑셀에 size값을 가진 상품 70개 중 라이브 `t_prd_product_sizes`에 size가 적재된 상품은 **11개뿐**.
   plate>0·size=0 패턴이 **77상품**. 적재자가 "작업사이즈(파일사양)"만 `t_prd_product_plate_sizes`에 넣고
   고객 선택용 **재단치수(size)를 통째로 누락**했다. 이는 위젯 옵션의 사이즈 선택지가 사라지는 치명 결함이다(G-GP-2).

**교훈(HARD 재확인):** 결함은 라이브로 재확인해야 한다. 추출본만 보면 (a) 회색밴딩을 "품절"로 오판, (b) plate 적재를
size 적재로 착각 — 둘 다 발생. 본 문서 모든 수치는 라이브 SELECT 재산출이다.

## ① goods-pouch 상품군 — 도메인 이해 (라이브 권위)

**라이브 SELECT:** 엑셀 L1 103상품명 → `t_prd_products` 매칭 = **98 matched / 5 unmatched**.
matched 98상품 전부 `prd_typ_cd = PRD_TYPE.03`(부자재/굿즈 계열), `prd_cd` **PRD_000183~280 연속 블록**(이 범위에 정확히 98상품 존재).

## ② 엑셀 옵션 구조 (goods-pouch L1, 무손실 추출 권위)

L1(`06_extract/goods-pouch-l1.csv`, 303 데이터행·40+컬럼)이 표현하는 옵션 캐스케이드:

| 엑셀 컬럼(그룹헤더) | 의미 | DB 정규화 대상 |
|---------------------|------|----------------|
| `사이즈(필수)` | 재단치수/형태(M·L)/폰모델★ | **size** (`t_prd_product_sizes`) |
| `파일사양_작업사이즈`·`재단사이즈`·`출력용지규격` | 작업/전지 판형 | **plate_size** (`t_prd_product_plate_sizes`) |
| `선택(옵션)_선택` | 색상/규격 **variant**(머그 화이트/반투명/투명, 티셔츠 화이트M…) | **material**(자재 variant) 또는 size variant — round-2 component_prices 차원 교차 |
| `선택(옵션)_가격`·`가격`·`가공(옵션)_가격`·`추가상품(옵션)_추가가격` | 가격축 | (round-2 t_prc_*, 본 라운드 이연) |
| `가공(옵션)_가공` | 후가공/에폭시/부착 공정 | **process**(`t_prd_product_processes`) |
| `추가상품(옵션)_추가상품` | addon(말랑=볼체인 등) | **addon**(`t_prd_product_addons`) |
| `제작수량(필수)_최소/최대/증가` | 수량 규칙 | (products.qty 필드) |
| `구간할인적용테이블` | 수량구간 할인 포인터(굿즈A/B·말랑·파우치/에코백) | **discount_table**(round-1 완료, `t_prd_product_discount_tables`) |

**핵심 도메인 캐스케이드(반영):**
- **variant 자재 분기(R-MAT-5):** 머그(색상 화이트/반투명/투명)·거울(S/M/L)은 `선택` 컬럼의 variant다. 라이브에서
  머그컵(193) material=4행(MAT_000268/255/146/143, 전부 USAGE.07)로 적재 — **variant 자재가 material 차원으로 정상 적재**됨.
  단 사이즈성 variant(거울 S/M/L)는 size로 가야 하는데 size=0 — variant 축↔size/material 분기에서 size 쪽이 누락(G-GP-2).
- **에폭시(PROC_000083) / 부착(PROC_000081):** 라이브 마스터 `t_proc_processes` 확인 — `PROC_000083=에폭시`,
  `PROC_000081=부착` 실재. 캔버스 계열 6상품에 PROC_000081(부착) 적재됨. 에폭시는 아직 어느 상품에도 미적재(G-GP-4).
- **폰모델★ 비치수 사이즈:** 폰케이스 size칸은 치수가 아닌 기종명(삼성 갤럭시S21~, 아이폰11~15). ★=신규 기종.

## ④ 결함 표 — 누락(MISSING)·오매칭(MISMATCH)·모호(CONFIRM) (도메인 렌즈, 라이브 재확인)

> MISSING = 정규화 규칙대로면 적재돼야 하는데 라이브에 없는 행. MISMATCH = 잘못 연결. CONFIRM = 도메인 의미 미확정.

### G-GP-1 [MISSING · BLOCKER] 폰케이스 5상품 자체 미등록 (그레이밴딩)
- **도메인 근거:** 엑셀 회색(`FFD9D9D9`) 셀이 정확히 슬림하드 폰케이스·블랙젤리·임팩트 젤하드·에어팟★·버즈★ 5상품에만 집중.
- **라이브 재확인:** 5상품 `t_prd_products` 미등록(fuzzy 전수 검색 부재). 이들의 폰모델★ 사이즈(블랙젤리 30종,
  슬림하드 39종 등)·material·process 전량 부재.
- **MISSING:** 상품 5개 + 그에 딸린 size 100+종 + 자재/가공/가격 전부. → **사용자가 지목한 그레이밴딩 비활성의 정체**.
- **판정:** 회색밴딩 = 비활성(품절)이 아니라 **적재 미완료 마킹**. 출시 여부는 컨펌(미출시면 보류, 출시면 BLOCKER).
- **처리방향(R1):** 5상품을 `t_prd_products` 등록 후 폰모델★ size·material(케이스 소재)·process 적재. **출시 여부 선확인.**

### G-GP-2 [MISSING · BLOCKER] size(재단치수) 거의 전량 미적재 — plate만 적재
- **도메인 근거:** 엑셀 `사이즈(필수)`칸 값(거울 무광75mm, 파우치 M/L, 코스터 치수 등)은 고객 선택용 재단 size.
  `파일사양_작업사이즈`는 별개의 작업판(plate).
- **라이브 재확인:** size 적재 11상품(사각손거울·블랙사각손거울·미니매트·피크닉매트·키캡키링·클립보드·만년스탬프·
  타이벡북커버·캔버스스트랩라벨파우치·광목스트링라벨파우치·레더라벨제작)뿐. plate>0·size=0 = **77상품**.
  사각손거울(186) 검증: size(SIZ_384/386/388)·plate(SIZ_385/387/389)가 짝지어 정상 — **size 적재 가능한데 77상품은 누락**.
- **MISSING:** 엑셀 size값 보유 70상품 중 59상품 size 전량 누락(폰케이스 5 제외 시에도 거대). size행 합계 224 중 다수 결손.
- **처리방향(R2):** 각 상품 엑셀 `사이즈(필수)` 행을 `t_siz_sizes` 매치 후 `t_prd_product_sizes`에 다행 적재(dflt_yn·disp_seq).

### G-GP-3 [CONFIRM] variant 자재/사이즈 분기 (머그·거울·티셔츠)
- **도메인 근거:** `선택` 컬럼 variant는 **자재성(머그 색상→material)** vs **사이즈성(거울 S/M/L·티셔츠 M/L/XL→size)**로 갈린다.
- **라이브:** 머그컵(193) material=4(색상 variant 정상). 그러나 티셔츠(206) material=8(화이트/블랙×M~XXL)인데 size=0
  → 색상×사이즈 복합 variant를 **전부 material로 평면화**, size 차원 분리 안 됨.

  (round-2 component_prices 차원과 교차 — 자재 매핑 분기 R-MAT-5 주의.)

## ⑦ 큰 그림 적용 (도메인 → 엑셀 → DB결함 → 처리 → 위젯)

- **도메인 KB(07_domain):** 그레이밴딩=미적재 마킹·variant 자재/사이즈 분기·에폭시 굿즈전용 공정이
  "회색셀/선택칸/가공칸"을 옳게 해석하는 렌즈 → 원적재의 오해(회색=건너뜀, 작업사이즈=size, variant 평면화)를 결함으로 규명.
- **엑셀 L1(무손실):** 전 컬럼·셀배경·신규색 보존 덕에 회색밴딩 5상품·size값 70상품·variant 10상품을 정확 추출(추정 0).
- **DB 라이브 권위:** 추출본만으론 그레이밴딩=품절 오판, plate=size 착각이 생긴다. 라이브로 (a) 회색=미등록 5상품,
  (b) size 11 vs plate 85, (c) variant material 정상 적재를 확정.
- **처리방향:** 마스터 무변경 + 상품 등록(R1)·연결 테이블 적재(R2~R4). 사각손거울을 size/plate 짝 적재 레퍼런스로.
- **위젯 기반:** 정제된 size·variant·공정이 굿즈 위젯 옵션 캐스케이드(사이즈→재단, 선택→색상/규격, 가공→에폭시,
  추가상품→볼체인) 데이터 소스가 됨. size 결손(R2)·폰케이스 미등록(R1) 잔존 시 위젯 사이즈 선택 불가 → 위젯 선결 조건.
