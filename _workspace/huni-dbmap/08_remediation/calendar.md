# calendar(캘린더) 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 **누락·오매칭·모호**가 생겼다.
> 본 문서는 그 결함을 *올바로 처리(수정)*하기 위해, **라이브 Railway DB 직접 read-only SELECT를 권위**로
> calendar(캘린더) 상품군을 종단 규명한다. 도메인 지식(07_domain KB)은 결함을 옳게 분류하는 렌즈다.
> 정제된 DB가 인쇄 자동견적 위젯의 기반이 된다.
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본·v2 2026-06-04은 stale 가능).
> 추정·임의판단 0 — 라이브·엑셀 L1·도메인 KB·PDF 실재 근거만. 미지/추정은 가설+컨펌 플래그.
> INSERT/UPDATE/DDL 0. DB 미적재 유지(본 문서는 처리 *방향* 설계). `db railway`·비표준포트·JOIN KEY=`prd_nm`.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. 라이브 SELECT 3시트 공통 9회(전부 read-only).

## ⚠️ 머리말 — 본 검증이 v2 대비 뒤집은 가장 큰 사실 (권위 반전)

**v2는 공정택일그룹(process_select_group)을 "N/A 보류(엑셀 원천 부재)"로 닫았으나, 캘린더는 택일 구조가 실재하며 라이브 적재가 불완전하다.**

라이브 직접 확인 결과(가장 큰 반전 3건):

- **캘린더 가공 택일그룹(GRP-CAL-가공)이 110·111·112에만 적재, 108·109엔 0행.**
  그런데 적재된 택일그룹조차 **멤버 공정과 연결이 끊겼다** — `t_prd_product_processes.excl_grp_cd`가 모든 캘린더 공정행에서 **NULL('-')**.
  즉 excl_groups 테이블엔 헤더(택일그룹)만 있고, **어떤 공정이 그 택일그룹의 멤버인지 연결 FK가 미적재**. UI 택일이 구성 불가.
- **택일 멤버 다수 누락.** 엑셀 엽서캘린더(110)는 `가공없음(재단만) / 우드거치대 / 1구타공+끈` 3택1, 벽걸이(111)는
  `고리형트윈링제본 / 가공없음 / 2구타공+끈` 다택인데, excl_group note의 trigger는 **1개만**(110=우드거치대, 111·112=고리형트윈링).
  → 택일 후보 공정 중 1개만 흔적, 나머지 미적재.
- **우드거치대가 공정(excl_group trigger)이면서 동시에 추가상품(addon)으로도 적재** — PRD_000110 addon에 `우드거치대`(PRD_000012) 존재.
  **같은 옵션이 두 축(공정 택일 vs 추가상품)에 이중 분류 = 도메인 분해 결함.** 캘린더가공 컬럼의 다축 혼재가 그대로 DB에 새었다.

**교훈(HARD 재확인):** "엑셀 원천 부재 → N/A 보류"는 택일 구조가 *실재*하는 캘린더에선 결함을 은폐한다.
라이브 excl_group 13행과 process.excl_grp_cd 연결을 직접 봐야 "헤더만 있고 멤버 연결 없음"이 드러난다.

## ① calendar 상품군 — 도메인 이해 (라이브 권위)

**라이브 SELECT #1:** 캘린더 시트 5 prd_nm 전수 매칭.

캘린더 시트 = **탁상형·미니탁상형·엽서·벽걸이·와이드벽걸이** 5상품. 라이브 `prd_cd` **PRD_000108~112 연속**,
전부 **PRD_TYPE.04 디자인상품**, use_yn=Y.

| 상품(prd_nm) → prd_cd | 인쇄 도메인 의미 | 캘린더가공 형태(엑셀) |
|----------------------|------------------|----------------------|
| 탁상형캘린더 108 | 삼각대 거치 탁상달력 | **삼각대(그레이)** 단일 |
| 미니탁상형캘린더 109 | 소형 삼각대 달력 | **삼각대(블랙)** 단일 |
| 엽서캘린더 110 | 낱장/거치 엽서형 | **가공없음 / 우드거치대 / 1구타공+끈** (택일) |
| 벽걸이캘린더 111 | 벽걸이 제본형 | **고리형트윈링제본 / 가공없음 / 2구타공+끈** (택일) |
| 와이드벽걸이캘린더 112 | 대형(3절) 벽걸이 | **고리형트윈링제본 / 제본없음** (택일) |

- **캘린더가공 = 거치/제본/타공 다축.** 삼각대(거치대)·우드거치대=거치, 고리형트윈링제본=제본, N구타공+끈=타공+부착.
  도메인상 **거치·제본·타공이 한 컬럼에 혼재** → DB에선 공정(타공079·트윈링021)·추가상품(거치대)·variant로 분해 필요(B R-PROC-7 비명칭 배제 신중).
- 라이브 확인: 5 prd_nm 전부 등록(unmatched 0). use_yn=N 없음.
- **qty_unit_typ_cd = NULL (전 5상품):** 272 전상품 NULL — 캘린더 고유 결함 아님(§4 G-CL-6).

## ② 엑셀 옵션 구조 (calendar L1, 무손실 추출 권위)

L1(`06_extract/calendar-l1.csv`, 19행)이 표현하는 옵션 캐스케이드:

| 엑셀 컬럼(그룹헤더) | 의미 | DB 정규화 대상(9속성) |
|---------------------|------|----------------------|
| `사이즈(필수)` | 완제품 치수(`220 x 145 mm`) | size |
| `파일사양_작업/재단사이즈`·`_출력용지규격`(col_hidden) | 작업 판형·전지(`316x467`·`330x660`) | plate_size (전지=출력용지규격) |

| `인쇄(필수)` = 단면/양면 | 인쇄면 | print_option |
| `장수(필수)` = `4(8P) / 8(16P) / 12(24P)…` | **장수↔페이지 매핑**(달력 월수) | page_rule(?) / variant |
| `캘린더가공(필수)_삼각대컬러` = 삼각대(그레이/블랙) | **거치대 색상 variant** | (거치대 공정·추가상품) |
| `캘린더가공(필수)_캘린더가공` = 우드거치대·고리형트윈링제본·N구타공+끈·가공없음 | **공정 택일(다축 혼재)** | **process(택일그룹)·addl_product** |
| `캘린더가공(필수)_추가가격` = 0·1000·1500·2000·4000 | 가공 추가가 | (가격, 이연) |
| `캘린더가공(필수)_링칼라` = 블랙 / `★고리형트윈링제본선택시만` | **링색상(조건부 의존)** | process param(고리형 선택시만) |
| `제작수량_최소/최대/증가` | 수량 규칙 | (products.qty) |
| `개별포장(옵션)`·`추가상품(옵션)_추가상품` = 캘린더봉투·우드거치대 | 포장·부속 | process(포장)·addl_product |

**핵심 도메인 캐스케이드:** `캘린더가공` 컬럼이 **거치(삼각대/우드거치대)·제본(고리형트윈링)·타공(N구타공+끈)·가공없음**을
한 칸에 담은 **공정 택일그룹**이다(SEL_TYPE.01 단일·max_sel=1). `링칼라`는 "★고리형트윈링제본선택시만" =
**고리형트윈링 선택을 부모로 하는 조건부 자식**. `삼각대컬러`는 삼각대 선택의 variant. 이 다축 혼재를
원적재가 평면화·부분적재하면서 택일 구조가 깨졌다.

## ④ 결함 표 — 누락(MISSING)·오매칭(MISMATCH)·모호(CONFIRM) (도메인 렌즈, 라이브 재확인)

### G-CL-1 [MISMATCH+MISSING · BLOCKER] 캘린더가공 택일그룹 멤버 연결 끊김 (R-PSG)

- **도메인 근거:** `캘린더가공` 컬럼 = 공정 택일(SEL_TYPE.01 단일). 택일 멤버 공정들이 `t_prd_product_processes`에
  **`excl_grp_cd=GRP-CAL-가공`으로 연결**돼야 UI 택일 구성 가능(R-COD-3·R-PSG-1).
- **라이브 재확인(SELECT #4,5):** excl_groups에 110·111·112 헤더는 있으나, 그 상품들 process행의 `excl_grp_cd`는 전부 NULL.
  **헤더(택일그룹)만 적재, 멤버 공정 연결 0건.** → 택일 UI가 멤버 없는 빈 그룹.
- **MISMATCH(연결 끊김) + MISSING(멤버):** 111(트윈링021·타공079)은 GRP-CAL-가공 멤버여야 하나 excl_grp_cd NULL.
- **처리방향(§5 R1):** 택일 멤버 공정 적재 후 `excl_grp_cd=GRP-CAL-가공` 연결. 택일 후보(가공없음·거치대·타공·트윈링) 전종 적재.

### G-CL-2 [MISSING] 거치대(삼각대/우드) 공정 미적재 (108·109)

- **도메인 근거:** 탁상형(108)·미니탁상형(109)은 `삼각대(그레이/블랙)` 거치 가공이 필수(`캘린더가공(필수)`).
- **라이브 재확인:** 108·109 process = 수축포장만. **삼각대 거치 공정·excl_group 0행.** 거치대를 어떤 proc로 표현하는지 부재.
- **MISSING + CONFIRM:** 삼각대 거치를 공정(신규?)·추가상품·variant 중 무엇으로 적재할지 마스터에 대응 공정 부재(컨펌).
  우드거치대는 addon(PRD_000012)으로 적재된 전례 있음 → 삼각대도 addon인지, excl_group 공정인지 불일치.

### G-CL-3 [MISMATCH] 우드거치대 이중 분류 (공정 택일 trigger ↔ addon)

- **라이브:** 110 excl_group note `trigger=우드거치대`(공정 택일 멤버 신호) ↔ 110 addon에 `우드거치대`(PRD_000012) 적재.
- **판정:** **동일 옵션이 공정 택일과 추가상품 두 축에 이중 분류.** 캘린더가공 컬럼(공정)과 추가상품 컬럼이 겹친 결과.
  하나로 정해야 함 — 거치대는 **추가상품(addon)**이 도메인상 자연스러움(부속 거치대 판매). excl_group trigger에서 제거 필요. **CONFIRM.**

### G-CL-4 [MISSING] 자재 `*별도설정` → IMPORT 종이 미적재

- **도메인 근거(R-MAT-2 HARD):** `종이사양 = *별도설정`은 가격표 `출력소재(IMPORT)` 포인터.
- **라이브 재확인:** material 적재 = 108(3행)·109(3행)·110(1행)·111(2행)·112(4행). 일부 적재됐으나
  `seoljeong-import-map.csv` 기준 IMPORT 종이수(탁상형 8·미니 7·엽서 10·벽걸이 22종) 대비 **대폭 부족**.
- **MISSING:** IMPORT 종이 다수 미적재(탁상형 8종 중 3행만). 와이드벽걸이는 `*별도설정` 아닌 직접명(스노우지/몽블랑 3절) — 직접 적재.
- **처리방향(§5 R3):** IMPORT 컬럼↔상품 매핑 확정 후 `t_mat.mat_nm` 매치 다행 적재.

### G-CL-5 [CONFIRM] 장수↔페이지(page_rule) / 링칼라 조건부 의존

- **라이브:** 캘린더 5상품 page_rule **0행.** 엑셀 `장수(필수) = 4(8P)/8(16P)/12(24P)…`는 달력 월수↔페이지 매핑.
- **판정:** 장수는 page_rule(min/max)인가, variant(선택 옵션)인가 미확정. `4(8P)`형은 장수4=8페이지 = **변형 옵션**에 가까움(컨펌).
- **링칼라:** `★고리형트윈링제본선택시만 : 링칼라선택` = 고리형트윈링(proc) 선택 시만 노출되는 **조건부 자식 옵션**.
  라이브엔 링칼라가 별도 적재 안 됨(트윈링제본 proc의 param/note 부재). **CONFIRM**(조건부 의존을 어디에 인코딩).

### G-CL-6 [글로벌 갭] qty_unit_typ_cd 전 NULL

- **라이브:** 캘린더 5 전부 NULL(272 전상품). 캘린더 고유 결함 아님 — 글로벌 정책(§ digital-print G-DP-9 동일).

## ⑦ 큰 그림 적용 (도메인 → 엑셀 → DB결함 → 처리 → 위젯)

- **도메인 KB:** GRP-CAL-가공 택일그룹(db-domain §4)·캘린더가공 다축(거치/제본/타공)이 "엑셀 캘린더가공 컬럼"을 옳게 분해하는 렌즈.
- **엑셀 L1(무손실):** 캘린더가공 4컬럼(삼각대컬러/캘린더가공/추가가격/링칼라) 분리 보존 → 택일 후보·조건부 의존(링칼라)을 정확 추출.
- **DB 라이브 권위:** v2 "택일그룹 N/A 보류"가 은폐한 "헤더만 있고 멤버 연결 0건"·"우드거치대 이중분류"가 라이브 excl_grp_cd 연결 직접 확인으로 드러남.
- **처리방향:** 마스터 무변경 + 택일그룹 멤버 연결·거치대 단일축화·IMPORT 자재 적재.
- **위젯 기반:** 캘린더가공 택일이 위젯의 대표 택일 UI(사용자 지적). 멤버 연결 끊김 잔존 시 위젯 택일 옵션이 빈 그룹 → R1이 위젯 선결 조건.
