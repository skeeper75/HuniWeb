# 합판도무송스티커(PRD_000066) 가격구성요소 골든 정밀검증 (§29 후속 · §26 셀단위)

> 2026-06-30. 권위=인쇄상품 가격표 260527 「합판도무송스티커」 시트(절대)·라이브=감사 대상(읽기전용 SELECT+시뮬레이터 실호출).
> 트리거: §29 준비도 평가에서 066만 L2+(62.5%·calc=OFFGRID-격자밖·"골든 불일치 조사(판수/단가)"). 스티커 시트 가격구성요소 진단의 유일 잔여 의심.
> 방식: ① 권위 추출 CSV(`huni-dbmap/06_extract/price-gangpan-sticker-l1.csv`) 셀단위 전수 → 라이브 단가행 결정론 대조 ② 라이브 시뮬레이터(price-viewer/simulate) 실호출 ③ 예전사이트 골든 가용성.

## 결론: **GO — 실 결함 0건. §29 "골든 불일치/OFFGRID"는 가짜신호.**

데이터는 권위와 verbatim 일치하고, 전 유효수량에서 엔진이 권위가를 정확 산출. §29 플래그는 평가 대표케이스의 수량 오설정 + 예전사이트 골든 부재가 원인이었음.

## 1. 데이터 무결성 — 라이브 격자 = 권위엑셀 verbatim (셀단위)

- 권위 데이터셀 **370** (37사이즈 × 2자재그룹 × 5수량밴드) × 자재3 전개 = **라이브 1110 단가행**.
- **결정론 대조: 일치 1110 / 1110 · 불일치 0 · 권위누락 0 · 라이브잉여 0 (gap-0).**
- 사이즈 매핑 37/37 정확(원형11+정사각12+직사각14). 자재그룹: {비코팅/무광/유광}·{유포/투명데드롱/은데드롱} → 라이브 6 mat_cd 정합.
- ★§26 verdict.md(B01~B07 반칼/낱장/타투/팩)가 다루지 않은 **도무송 시트 셀단위를 본 검증에서 확정** — §26 스티커 트랙 100% 종결.

## 2. 엔진 실호출 — 전 유효수량 권위 정확 (시뮬레이터)

| 입력(사이즈·자재·수량) | 라이브 PRICE | 권위 | 판정 |
|---|---|---|---|
| 정사각10x10 유포 1000 | 26100 | 26100 | ✅ |
| 정사각10x10 유포 2000 | 39200 | 39200 | ✅ |
| 정사각10x10 유포 5000 | 78300 | 78300 | ✅ |
| 정사각10x10 비코팅 1000 | 20000 | 20000 | ✅ |
| 정사각90x90 유포 3000 | 120800 | 120800 | ✅ |

- 공식 PRF_GANGPAN_FIXED · 단일 COMP_GANGPAN_PRINT · prc_typ **.02 합가형(정액 룩업)** · use_dims=[siz_cd,mat_cd,min_qty].
- qty_rule = **{min:1000, max:5000, incr:1000}** → 유효수량 = {1000,2000,3000,4000,5000}뿐.

## 3. §29 가짜플래그 원인 규명

- **OFFGRID-격자밖**: 격자엔 구멍 0(37×6×5=1110 완전 카르테시안). §29 평가가 **유효수량(≥1000·incr 1000) 미사용**(qty_rule.dflt=null) → qty<1000 → PRICE=0 → OFFGRID에 멈춰 골든 미수행. 데이터 결함 아님.
- **예전사이트 골든 N/A**: 도무송은 예전사이트(huniprinting.com)에 **단독 미존재 신규/통합 상품**(`sticker-live-crosscheck-matrix.md`: "반칼스티커 사각분할/라벨택과 형태 유사하나 단독 부재"). product-spine 이전사이트칸이 빈 것이 정상. 비교 대상 자체가 없어 골든 불일치는 성립 불가.
- **선행 교정 이력**: 도무송은 이전에 4소재→6소재 견적불가 교정 완료(`sticker-simulator-golden.md`).

## 4. 판수(C트랙 코드버그) 우려 — 도무송 비해당

- 도무송 가격 = (사이즈·자재·수량밴드) 정액 룩업. **제작수량 밴드축이 곧 가격축** — fn_calc_pansu 기하 판수계산 미개입(단일 COMP·정액형). [[pansu-authority-fn-calc-pansu-260628]]의 각판/판걸이 기하 과다 결함과 **다른 모델**. 판수 정확성 이슈 없음.

## 5. 부수 관찰 (위젯 트랙 · 가격계산 비차단)

- 엔진이 **밴드 사이 수량**(예 1500)에서 정액(step)이 아니라 **단위환산 프로레이팅**(26100×1500/1000=39150)을 반환. qty_rule.incr=1000이라 정상경로(유효수량 1000단위)에서는 무해(전부 권위 정확). 
- **권고**: 위젯이 수량을 {1000,2000,3000,4000,5000} 드롭다운/step=1000으로 강제(자유입력 차단)해야 비권위가(off-band 프로레이팅) 노출 방지. → §6 위젯트랙·비차단. 단 1000 미만은 엔진이 PRICE=0(견적불가)로 안전 거동.

## 요약

**합판도무송스티커 가격구성요소 = 데이터·계산 모두 무결.** 라이브 1110 단가행이 권위엑셀과 verbatim 일치(셀단위 gap-0)하고, 전 유효수량(1000~5000·1000단위)에서 시뮬레이터가 권위가를 오차0 산출. §29의 L2+/OFFGRID/골든불일치는 (a) 평가 대표케이스의 유효수량 미사용, (b) 예전사이트 골든 부재(신규/통합 상품)에서 비롯한 **가짜신호**였으며 dbmap 라우팅할 실 결함 0건. 판수 C트랙 우려는 정액 합가형이라 비해당. 잔여=위젯 수량입력 step 강제(§6·비차단).
