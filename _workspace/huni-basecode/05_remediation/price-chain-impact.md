## 0. 가격사슬 구조 (라이브 실측 그라운딩)

```
t_prc_price_formulas (frm_cd)               ← 상품↔공식 바인딩(PRF_*)
   └ t_prc_formula_components (frm_cd, comp_cd, addtn_yn)   ← 공식 = 구성요소(comp_cd) 합산
        └ t_prc_component_prices (comp_cd + 차원키)          ← 단가행(unit_price)
```

**★결정적 사실 — 가격사슬 배선 키 (라이브 information_schema 실측):**
- `t_prc_formula_components` 컬럼 = **`frm_cd, comp_cd, disp_seq, addtn_yn, reg_dt, upd_dt`** — **proc_cd·mat_cd·siz_cd 직접 컬럼 없음.** 공식은 **comp_cd로만** 구성요소를 배선한다.
- `t_prc_component_prices` 차원키 = `comp_cd` + `siz_cd, clr_cd, mat_cd, proc_cd, opt_cd, print_opt_cd, plt_siz_cd, coat_side_cnt, bdl_qty, siz_width, siz_height, dim_vals(jsonb)`. **단가는 comp_cd로 식별된 행에 mat_cd/siz_cd/proc_cd 등 차원이 박혀 매칭.**
- **함의:** 기초코드(mat_cd/siz_cd/proc_cd) 교정이 가격에 닿는 경로는 **두 갈래** — ① 그 코드가 `component_prices`의 차원키 셀에 박혀 있으면 **직접 파손** ② 박혀 있지 않아도 그 코드가 BOM/공정 링크로 상품에 묶여 공식(PRF_*)이 그 상품을 가격화하면 **간접 영향**.

**전역 단가행 차원키 채움(baseline·라이브):**

| 차원키 | 채운 행수 / 전체 7293 |
|--------|----------------------|
| `mat_cd` | **3342** |
| `siz_cd` | 4157 |
| `proc_cd` | 1901 |
| `clr_cd` | **0** (도수는 단가행 차원키 미사용) |
| `print_opt_cd` | 1431 (POPT_000001 단면 689 / POPT_000002 양면 742) |

> mat_cd가 3342행에 박혀 있다 = 자재 코드는 가격에 광범위하게 닿는다(단, 오염 .08/.09/.10는 아래대로 0). clr_cd 0 = 도수 교정은 단가행에 닿을 수 없음(가격 무관 확정).

**★간접 가격 경로 (실측 규명):**
- `t_prd_product_print_options`는 **2개 컬럼이 공존** — `print_side`(UV 변형 텍스트·가격축 아님) + `print_opt_cd`(단면/양면 POPT·**가격축**).

- `component_prices`는 `print_opt_cd`(POPT_000001/002)로 단가를 매칭(전역 1431행) → **단면/양면 = 가격에 닿는 축.** UV 변형(풀빼다 등)은 별도 `print_side` 텍스트 → 단가행에 박히지 않음(comp 단가는 print_opt_cd로만).

**돈 크리티컬 판정 = 간접·주의 요(★).** PO-1은 UV 변형값을 `print_side`에서 빼서 `option_items.dtl_opt`로 옮기고 `print_side`를 단면/양면으로 정규화하는데, **그 정규화가 같은 행의 `print_opt_cd`(가격축)를 건드리면 아크릴 골든(186행·최대 32700) 파손.** 단가행 자체는 comp_cd로 매칭되어 print_side 텍스트 무관이지만, **print_opt_cd는 가격축이므로 무접촉 보장이 명세 가드로 필수.** → **dbm-price-arbiter 협업 권고**(아크릴 가격사슬 트랙 권위·골든 byte 불변 3중 보장 검증).
