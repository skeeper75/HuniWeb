## 0. 엔진 재구현 동치 입증 (pricing.py 코드 직접 검증)

골든 재계산에 쓰는 산식이 라이브 엔진과 동치임을 코드 라인으로 고정:

| 산식 | pricing.py 라인 | 재구현 |
|------|-----------------|--------|
| 비수량 차원 정확매칭(siz_cd·mat_cd·opt_cd·bdl_qty) | `NON_QTY_DIMS`(:38-39) + `_row_matches`(:78-90) | 선택 siz/mat 정확히 같은 행만 후보·행 NULL=와일드카드 |

| 동시매칭 2조합↑ = ERR_AMBIGUOUS | `:136-138` | 비수량 조합 1개만 정상 |
| 후보 0 = no_match | `:129-130` | 가격 0/오류(lenient skip / strict 실패) |
| 단가형(.01) = unit×qty | `component_subtotal`(:191-192) | `up * q` |
| 합가형(.02) = (unit÷tier_min_qty)×qty | `:185-190` | `(up/base) * q`·`base≤0`이면 ValueError(:187-188) |

## 1. 정상 케이스 재계산 (GC-STK1~4)

### GC-STK1. 124x186 유포 반칼 qty3 (PRF_STK_FIXED·.01)

라이브 단가행 verbatim (`SELECT … WHERE comp_cd='COMP_STK_PRINT' AND siz_cd='SIZ_000059' AND mat_cd='MAT_000153' ORDER BY (min_qty)::int`):
```
min_qty=1 → 6000 / min_qty=2 → 6000 / min_qty=3 → 5900 / min_qty=4 → 5800 …
```
- 재계산: 매칭행 siz_059·mat_153·min_qty=3(qty3 이하 최대 구간) → unit=5900 → `.01` → **5900 × ... ?** 

  ★주의: `.01` 산식은 `unit × qty`. 그러나 GC-STK1 골든 **5,900 = qty3 구간 단가**(개당가 아닌 "3개 주문 시 장당단가 5900"의 의미인지, "3개 총액 5900"인지). 단가형 정의상 `unit×qty = 5900×3 = 17,700`이 엔진 산출. 골든표는 "5,900원"을 **min_qty=3 구간의 장당 단가**로 인용(가격표 셀). **즉 GC-STK1의 "5,900"은 unit_price(장당 단가행 값)이지 evaluate_price 최종가가 아니다.** evaluate_price(qty3) = 5900×3 = 17,700.

| 항목 | 값 |
|------|-----|
| 매칭 단가행 unit_price | **5,900**(min_qty=3 구간·라이브 verbatim) |
| evaluate_price(.01·qty3) | **17,700** (5900×3) |
| evaluate_price(.01·qty1) | **6,000** (6000×1·min_qty=1 구간) |

→ **판정**: 단가행 unit=5,900·6,000은 라이브 verbatim **일치(허용오차 0)**. 골든표 "5,900/6,000"은 **단가행 값**(가격표 셀)을 가리키며 dbmap `bankal-058-064-deepcheck` 재현과 정합. evaluate_price 최종가(×qty)는 골든표에 미명시 → designer가 "단가행 verbatim"으로 표기한 것은 정확하나 **"기대 골든=최종 견적가"로 오독 위험**(검증 정정 CV-STK-G1·LOW·산식은 옳음). 단가행 값 자체는 100% 일치.

## 3. ★검증 추가 발굴 — 053/054 siz↔mat 교차바인딩 비정합 (design 미카탈로그)

designer G-STK-1~4가 닫지 못한 **추가 정합 결함**(독립 라이브 실측 발굴·E1 누락 항목):

라이브 실측(`t_prd_product_sizes` × `COMP_STK_PRINT` mat 분포 교차):

| 상품 | 바인딩 siz | 바인딩 mat | 그 mat의 단가행이 실존하는 siz | 교차 결과 |
|------|-----------|-----------|-------------------------------|-----------|
| **PRD_053**(반칼투명·active) | {170, 172, 196} | {162} | mat162 rows @ {059,060,172,174,197,514,515,518,519} | **170→0행·196→0행·172만 6행(낱장)** |
| **PRD_054**(홀로그램·active) | {170, 172, 196} | {163} | mat163 rows @ {059,060,518,519} | **170/172/196 전건 0행 = 모든 조합 no_match** |

★ **핵심**: 053(투명)·054(홀로그램)의 단가행은 **반칼정사각 사이즈(059/060)에 적재**돼 있는데, 053/054 상품은 **A5/A4/A6(170/172/196) 사이즈에 바인딩**돼 있다. 교집합이:
- 053: SIZ_172만 매칭(그나마 낱장 4000·반칼 5000 SIZ_520 아님 = G-STK-2 동근 + 170/196 no_match).
- 054: **교집합 0 → 054는 어느 바인딩 사이즈로도 가격 산출 불가**(전건 no_match).

→ designer G-STK-2는 "052/053/054 SIZ_172→SIZ_520"으로, G-STK-3은 "052~054 SIZ_196 0행"으로 표현했으나, **053/054가 mat162/163이라 SIZ_520(반칼)·SIZ_170(A5) 단가행조차 부재**라는 점(투명/홀로 소재는 059/060에만 적재)을 명시하지 않았다. **G-STK-2 교정안(052/053/054 SIZ_172→SIZ_520)을 053/054에 그대로 적용하면 SIZ_520×mat162/163 단가행이 없어 여전히 no_match.** 053/054는 siz 재바인딩(170/172/196 → 059/060) 또는 mat162/163 단가행을 520/170에 적재해야 함 — **G-STK-2와 다른 교정 경로**. 이는 결함 카탈로그 누락(E1 감점)이며, 053/054 교정안이 052와 동일하지 않음(돈크리티컬 정합).

