# 후니프린팅 리뉴얼 — Shopby 기능 갭 + 레드프린팅 1차 런칭 벤치마크

> 일정관리 IA 보강용 리서치. 입력 IA 144기능(`02_IA마스터`)을 기준으로,
> ① Shopby 표준 커머스 기능 대비 갭, ② 레드프린팅 1차 노출 기준 단계배치, ③ 외부연동/계약 체크리스트를 산출.
>
> **Shopby 사용 원칙:** Shopby에 직접 종속하지 않는다. Shopby의 **API·표준기능을 "쇼핑몰이 갖춰야 할 표준 체크리스트"이자 외부연동 후보**로만 활용한다(베스트프랙티스 모델). 후니는 자체 빌더 위에 필요한 기능을 직접 구현하되, Shopby가 표준으로 무엇을 제공하는지로 누락을 점검한다.
>
> 근거: Shopby 공개 OpenAPI 스펙(shop/server/admin/internal 4레이어, 실제 엔드포인트)에서 도메인·기능을 직접 추출. 레드프린팅은 공개 메인 구조(읽기 탐색)로 1차 노출 감각 확인. 상대 서비스에 주문·결제·폼제출 등 영향 행위 없음(읽기 전용).

## 0. Shopby 표준 커머스 도메인 지도 (근거)

Shopby는 커머스 기능을 다음 도메인으로 모듈화해 API로 제공한다(공개 스펙 실측). 후니가 "쇼핑몰 일반 기능"으로 갖춰야 할 표준 목록의 권위로 사용.

| Shopby 도메인 | 표준 제공 기능(실측 엔드포인트 요약) |
|---|---|
| **member**(회원) | 회원가입항목 config, 간편회원가입(소셜 provider), 회원등급, 탈퇴/탈퇴철회/복수탈퇴, 휴면 전환/해제 |
| **order**(주문) | 주문조회(v1.1·배송번호기준), 이전주문 등록, 배송지 CRUD, 무통장 입금확인/미입금 리스트, 결제완료 처리, **현금영수증 발행결과 업데이트**, **세금계산서 발행결과 업데이트**, 주문상태 일괄변경, 송장번호 변경, 정기결제(배송) |
| **pg / 결제키**(order 내장) | inicis(이니시스)·kcp·nicepay·kakaopay·naverpay·payco·toss·갤럭시아·이지페이 등 **PG/간편결제 키 등록** 표준 지원 |
| **claim**(취소·반품·교환) | 주문취소, 옵션취소, 품절취소, 반품신청/수거완료, 반품교환 신청/수거/추가결제 입금확인, 클레임 승인/철회, 교환 전후 정보 |
| **delivery**(배송) | **지역별 추가배송비(도서산간) 설정**, 배송비 템플릿/그룹 CRUD, 입출고 주소(창고) |
| **promotion**(쿠폰) | 쿠폰 생성/수정/검색, 회원에게 쿠폰 발급/철회, 쿠폰 사용/취소, 사용중지·재개, 대상/제외대상 조회 |
| **manage**(적립금·문의·메시징) | 적립금 조회/지급/차감/만료/사용처추적, **1:1문의 유형·답변**, **카카오 알림톡 설정·수동전송**, 080 수신거부, 약관 동의 관리 |
| **display**(진열·후기·문의) | 전시 카테고리 트리 CRUD, **상품평(리뷰) CRUD·베스트·전시상태**, **상품문의(상품Q&A) 답변** |
| **product**(상품) | 상품 CRUD, 브랜드, 상품정보고시(Duty Category), 상품 추가항목(Custom Property), 구매권한, 상품 이용안내 |
| **marketing** | 네이버쇼핑 연동, 마케팅 상품 노출 일괄변경 |

> 핵심 시사점: 후니 IA가 P2/대기로 미뤄둔 **쿠폰·적립금(프린팅머니)·증빙(현금영수증/세금계산서)·리뷰·1:1문의·알림톡·도서산간 배송비**는 전부 Shopby가 **표준 모듈**로 제공한다. 즉 "특수 개발"이 아니라 커머스 솔루션의 기본 묶음이며, 외부연동 시 자체 구현 공수를 크게 줄일 수 있는 후보다.

## 2. 레드프린팅 1차 런칭 벤치마크

### 2-1. 레드프린팅이 "전면 노출"하는 1차 기능(공개 메인 실측)

- **상단/메인:** 상품 카테고리(디지털인쇄·옵셋·쉬운주문·대형실사·사진관·문방구), 검색, 로그인/회원가입, 장바구니, **제작가이드**, 언어선택
- **고객지원:** 1:1상담, 자주묻는질문(FAQ), 공지사항, 전화 고객센터
- **주문 흐름:** 메인은 카테고리·배너 중심. **옵션선택→실시간가격→파일업로드**는 상품 상세페이지에서 작동(인쇄몰 공통 패턴)
- 브랜드몰/B2B(배민·롯데렌탈 등) 채널 존재 → **B2B/대량주문 동선**을 초기부터 노출

### 2-2. 인쇄 도메인 1차(P0) 필수 — "이게 없으면 매출이 안 일어남"

레드 기준 + 인쇄 베스트프랙티스로, 후니 1차에 반드시 들어가야 하는 묶음:

| 묶음 | 후니 IA 매핑 | 사유(레드 근거) |
|---|---|---|
| 상품 카테고리·검색·상세 진입 | No.31,32,66 | 레드 메인의 1순위 동선 |
| **옵션위젯(규격·수량·종이·도수·제본)** | No.34~43, B1~B10 | 인쇄몰 핵심. 상세페이지 = 견적기 |
| **실시간 가격 재계산(C1)** | No.49 | 레드 상세의 심장. XL이지만 P0 절대 |
| **파일 업로드(PDF) + 미리보기** | No.52,53 D1·D2 | 업로드 없으면 주문 불성립 |
| **작업/재단/블리드 안내(E4)** | No.60 | 파일 사고 방지, 레드 "제작가이드" 상시 노출 |
| 주문 전 검증 + 주문전환 + 플로팅 주문박스 | No.61~64 F1~F4 | 옵션→장바구니→주문 |
| 로그인/회원가입/회원정보 | No.1,4~8,21~24 | 주문 귀속 |
| 결제(이니시스) + 주문완료 | No.80,81 | 매출 확정 |
| 배송정보 입력 + 도서산간 배송비 | No.77~79 | 정확한 청구 |
| 관리자 상품/가격 셋업 + 주문관리 | No.100~108,136~140 | 운영 백본 |
| 약관/개인정보 | No.84,85 | 법적 필수 |

> **C1(실시간가격)·D1(파일업로드)·B9/B10(제본)·B15(옵션제약)**은 IA에서 이미 P0. 레드 벤치마크상 **이 묶음이 곧 1차 런칭의 정의**임을 재확인 — 절대 P1로 미루지 말 것.

### 2-3. 2차(P1)로 미뤄도 되는 것 — "런칭 직후 안정화"

- 쿠폰·적립금/프린팅머니(No.12~15) — 재구매 유도는 트래픽 쌓인 뒤
- 증빙(현금영수증/세금계산서, No.25~28) — B2B 본격화 시점. 단 **세금계산서는 B2B 비중 크면 1차 당김 검토**
- 코팅·커팅·박/형압·캘린더가공(B6~B13) — 핵심 품목 P0 옵션 먼저, 부가옵션은 순차
- 견적서 다운로드·수량별 가격표(C2·C3) — 실시간가격(C1) 안정화 후
- 옵션보관함(F5)·재주문 — 재구매 동선
- 고객센터 게시판류·알림톡(No.67~72,138,144) — 계약 선행 의존
- 거래처/원장/통계(No.95~99,129~135) — 운영 정착 후

### 2-4. 3차(P2)로 미뤄도 되는 것 — "확장·미결정 의존"

- 온라인 에디터·VDP(D3·D4) — 외부도입 결정 의존(협업)
- 리뷰/체험단/마케팅 랜딩(No.17~19,88~90,121,122)
- 수작/디자인 브랜드(No.82,93,111,113) — 브랜드 존치 결정 의존
- 고급통계(No.135)
