# acrylic(아크릴) 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 아크릴 상품군에
> **누락·오매칭·모호**가 다수 생겼다. 본 문서는 그 결함을 *올바로 처리(수정)* 하기 위해,
> **라이브 Railway DB 직접 read-only SELECT를 권위**로 acrylic 25상품을 종단 규명한다.
> digital-print 파일럿(`08_remediation/digital-print.md`)의 종단 흐름·원칙·산출형식을 템플릿으로 삼았다.
> 정제된 DB가 인쇄 자동견적 위젯의 기반이 된다.
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본 `ref-*.csv` stale 가능).
> 추정·임의판단 0 — 라이브·엑셀 L1·도메인 KB·PDF 실재 근거만. 미지/추정은 가설+컨펌 플래그.
> INSERT/UPDATE/DDL 0. DB 미적재 유지(본 문서는 처리 *방향* 설계). `db railway`·비표준포트·JOIN KEY=`prd_nm`.
> EXTRA 삭제단정 금지. 숨김/미출시(`use_yn=N`)·행숨김=비활성 분류.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. **라이브 SELECT 수행 약 13회**(전부 read-only).

## ⚠️ 머리말 — 본 검증이 v2/직관 대비 뒤집은 가장 큰 사실 (권위 반전)

**아크릴 시트의 핵심 결함은 "공정(가공·완칼·조각수) 전무"이며, 이는 마스터가 멀쩡한데 상품 연결만 안 된 적재 결손이다.**

라이브 직접 확인 결과:
- **아크릴 23 등록상품 중 process 행을 가진 상품은 단 1개(맥세이프 스마트톡, PROC_000081 부착)뿐.** 나머지 22상품 = process **0행**.
  → 모든 아크릴 굿즈는 도메인상 **완칼(die-cut, PROC_000053)** 이 필수(아크릴을 모양대로 자름)인데 **전무**. 조각수형(자유형스탠드·미니파츠)의 **완칼/스티커완칼 조각수 param**도 전무.
- **마스터 `t_mat_materials`에는 이미 두께 변형(`아크릴 투명 1.5mm`=MAT_000042·`3mm`=043·`8mm`=044)과 부자재(`아크릴부속 일자핀/자석/고리/투명집게…`=MAT_TYPE.07 MAT_000045~057)가 전부 정의돼 있다.**
  그러나 적재된 23상품은 **두께 없는 `투명아크릴`(MAT_000192) 한 코드로 일괄 매핑** — L1의 `투명아크릴 3mm`/`8mm`/`1.5mm` 두께 구분이 **소실**됐고, 가공(옵션) 부자재(자석/핀/고리 등)는 **단 1상품(맥세이프 부착)만 연결**.
- **L1 인쇄사양(`배면양면`/`풀빼다`/`투명테두리`/`단면인쇄`)은 `t_prd_product_print_options.print_side`로 적재**됐다.
  그러나 도메인 KB(`db-domain-structure-live.md` §2-4)상 이 4값은 **UV 인쇄방식(PROC_000002)의 `변형` enum**(일반/배면양면/풀빼다/투명테두리/단면)이다.
  → **인쇄"면" 컬럼에 UV "변형"을 욱여넣은 의미축 오매칭**(CONFIRM, 후술 G-AC-5).

**교훈(HARD 재확인):** 마스터가 풍부한데(두께·부자재·완칼·조각수 코드 전부 존재) **상품 연결 테이블만 비어** 있다.
즉 아크릴 결함은 "코드 부재"가 아니라 **"엑셀 옵션→상품 연결 매핑 미수행/오매핑"**이다. 라이브로 봐야만 드러난다.

## ① acrylic 상품군 — 도메인 이해 (라이브 권위)

**라이브 SELECT:** `t_prd_products` 전수 272, 아크릴 25 `prd_nm` 매칭.

acrylic 시트 = **단품형(14)·조합형(11)** 2개 `구분`의 아크릴 굿즈. 라이브에서 등록상품은
`prd_cd` **PRD_000146 ~ PRD_000169**(연속, PRD_000167 결번), 전부 **`prd_typ_cd = PRD_TYPE.04 디자인상품`**.

> **라이브 등록 실태(SELECT #2,3):** L1 25상품 중 **23상품 등록 / 2상품 미등록**.
> 미등록 = **아크릴쉐이커(★)·지비츠(★)** — 둘 다 L1에서 **전행 숨김(`_row_hidden=true`)+★제약**.
> → 숨김/미출시 = 비활성(MISSING 아님). 단 "★제약 상품은 등록 보류"가 의도된 정책인지 컨펌(G-AC-8).

| 구분 | 상품(prd_nm) → prd_cd | 인쇄 도메인 의미 |
|------|----------------------|------------------|
| **단품형**(14) | 아크릴키링(146)·마그넷(147)·뱃지(148)·집게(149)·스마트톡(150)·맥세이프스마트톡(151)·명찰(152)·명찰골드실버(153)·머리끈(154)·볼펜(155)·지비츠(156)·네임택(157)·포카키링(158)·코스터(159) | 단일 아크릴 조각 굿즈. 완칼(die-cut)로 모양 자름. 부자재(고리/자석/핀)는 가공옵션 |
| **조합형**(11) | 자유형스탠드(160)·판아크릴(161)·포카스탠드(162)·미니파츠(163)·코롯토(164)·포카코롯토(165)·카라비너(166)·입체코롯토(168)·입체블럭(169)·**아크릴쉐이커★(미등록)**·**지비츠★(미등록)** | 다조각 결합 굿즈. **조각수**(자유형스탠드 2~6조각·미니파츠 10조각)·접합/라미(카라비너 3mm+3mm·입체블럭 10T) |

- **use_yn=N (비활성/미출시, MISSING 아님):** 명찰골드실버(153)·지비츠(156)·코스터(159)·코롯토(164)·포카코롯토(165)·카라비너(166).
  → 결함 집계에서 분리. **단 153/156은 use_yn=N인데 자재·인쇄옵션은 적재됨**(부분 활성) — 출시 의도 컨펌.
- **nonspec_yn (라이브):** 입체코롯토(168)·입체블럭(169)만 `Y`. 그러나 **전 25상품 `nonspec_width/height_min/max` = NULL**.
  L1에서 11상품이 `사용자입력`(비치수, R-SIZE-5) 옵션을 가지는데 **nonspec 범위가 DB에 전무** → G-AC-6.
- **qty_unit_typ_cd = NULL (전 23상품):** 라이브 확인. digital-print와 동일한 **글로벌 갭**(272 전상품 NULL) → §4 G-AC-9.

L1(`06_extract/acrylic-l1.csv`, 117 데이터행·45컬럼)이 표현하는 옵션 캐스케이드:

| 엑셀 컬럼(그룹헤더) | 의미 | DB 정규화 대상(9속성) |
|---------------------|------|----------------------|
| `사이즈(필수)` | 재단치수 리스트 + `사용자입력`(비치수) | size + (nonspec) |

| `가공(옵션)_가격` = `*가격표참고`·`0`·`500` | 가격연동 노이즈(R-PROC-7 배제) | (매핑 금지) |
| `추가상품(옵션)_추가상품` = 볼체인 6색 | 부속 addon | addl_product |

**핵심 도메인 캐스케이드(아크릴 특화):**
1. **완칼은 묵시 필수** — 아크릴 굿즈는 전부 시트에서 모양대로 잘라낸다(die-cut). 엑셀에 명시 컬럼이 없어도 도메인상 PROC_000053 적재 대상. **현 DB 전무 = 도메인 누락(가설, 컨펌 G-AC-1).**
2. **인쇄사양 ≠ 인쇄면** — `배면양면/풀빼다/투명테두리`는 단/양면 도수가 아니라 **UV 출력 변형**. DB가 print_side로 적재한 것은 의미축 오매칭.
3. **가공 = 자재(부속) vs 공정 양분** — `자석/핀/고리/투명집게/헤어끈/바디` = MAT_TYPE.07 부속 자재. `맥세이프부착` = 부착공정. **현 DB는 맥세이프(부착)만 연결, 나머지 부속 전무.**
4. **두께가 자재 식별자** — `투명아크릴 3mm`↔`8mm`(코롯토)↔`1.5mm`(미니파츠)는 서로 다른 자재. 마스터에 MAT_000042/043/044로 존재하나 **적재는 두께없는 192로 일괄**.

## ④ 결함 표 — 누락(MISSING)·오매칭(MISMATCH)·모호(CONFIRM) (도메인 렌즈, 라이브 재확인)

> 분류 기준: **MISSING** = 정규화 규칙(B)대로 엑셀 변환 시 적재돼야 하는데 라이브에 없는 행.
> **MISMATCH** = 잘못된 축/코드로 연결. **CONFIRM** = 도메인 의미 미확정(실무 컨펌).

### G-AC-1 [MISSING · BLOCKER급] 완칼(die-cut) 공정 전무

- **도메인 근거:** 아크릴 굿즈는 전부 시트에서 모양대로 잘라낸다 = **완칼 PROC_000053**(종이+후지 자름, KB §2-1). 코스터 `원형/사각`, 모든 키링/뱃지/스탠드가 die-cut 필수.
- **라이브 재확인(SELECT #6):** 23상품 중 완칼/반칼/스티커완칼(053/054/055) **적재 0건.**
- **MISSING ≈ 21건**(입체코롯토 제외 전 단품·조합형). 위젯 옵션에서 "모양/완칼"이 통째로 사라짐.
- **처리방향(§5 R1):** 각 아크릴 상품에 PROC_000053 적재. 사용자입력형(뱃지·집게 등 `사용자입력` 사이즈 보유) = 사용자 모양 → 완칼 `모양` param.
- **가설 플래그:** 엑셀에 완칼 명시 컬럼 부재 → "묵시 필수"는 도메인 추론. **실무 컨펌 필수**(아크릴 절단이 완칼 공정으로 모델링되는지, 아니면 사이즈/가격에 내포되는지).

### G-AC-2 [MISSING] 조각수 공정 미적재 (조합형)

- **도메인 근거:** L1 `조각수(옵션)` = 자유형스탠드 `2~6조각`·미니파츠 `10조각`. KB §2-4: **완칼/스티커완칼 `조각수` integer(≥1) param**. 조각수 = 결합 파츠 개수.
- **라이브 재확인:** 자유형스탠드(160)·미니파츠(163) process **0행.** 조각수 신호 미반영.
- **MISSING 2상품**(+조각수 param). 위젯에서 "조각수 선택" 누락.
- **처리방향(§5 R2):** PROC_000053(또는 055) 적재 + `조각수` param(자유형=2~6, 미니파츠=10).

### G-AC-3 [MISMATCH·MISSING] 자재 두께 소실 + 부자재(부속) 미연결

- **도메인 근거:** 자재 식별자 = 아크릴 종류 + **두께**. L1 `투명아크릴 3mm`(대부분)·`8mm`(코롯토·포카코롯토)·`1.5mm`(미니파츠)·`라미 10T`(입체블럭)·`3mm+3mm 접합`(카라비너). 마스터에 **MAT_000042(1.5mm)·043(3mm)·044(8mm) 존재**.
- **라이브 재확인(SELECT #7):** 22상품 자재 = **전부 MAT_000192 `투명아크릴`(두께 없음)** 일괄. 코롯토(8mm)·미니파츠(1.5mm)도 192 → **두께 MISMATCH**(잘못된 자재코드).
- **부자재(MAT_TYPE.07):** L1 가공(옵션) `자석/일자핀/원형핀/투명집게/블랙헤어끈/투명바디/화이트바디/은색·금색고리` = 마스터 MAT_000045~057에 **전부 존재**. 그러나 상품 연결 **전무**(맥세이프 부착 제외).
- **판정:** 두께 = **MISMATCH**(192→042/043/044 교정). 부자재 = **MISSING**(부속 자재행 다수 미연결).
- **처리방향(§5 R3):** ① 코롯토/포카코롯토→MAT_000044(8mm), 미니파츠→042(1.5mm), 나머지 3mm→043으로 두께 교정(또는 192 유지+두께 속성축 컨펌). ② 가공 부자재를 MAT_TYPE.07 부속으로 `t_prd_product_materials`(usage 부속) 다행 적재.

### G-AC-4 [MISSING] 가공 부착공정 부분 적재

- **도메인 근거:** 가공 중 `맥세이프스마트톡`·`자석부착` 류는 **부착공정 PROC_000081**(KB: 부착 `대상` enum 라벨/맥세이프/끈/테입). 핀/고리도 부착 성격.
- **라이브 재확인(SELECT #6):** 맥세이프(151)만 PROC_000081 적재. 자석부착(마그넷147)·핀류·고리류 **부착공정 미적재.**
- **판정:** **MISSING**(단 부속을 자재로 볼지 공정으로 볼지 분기 — G-AC-3과 한 쌍). 자석/핀/고리를 "부착 대상 자재"로 모델링 시 자재축, "부착 행위"로 모델링 시 공정축.
- **처리방향:** 부속=자재(MAT_TYPE.07) + 부착=공정(PROC_000081) **2축 동시**가 KB 정합. 컨펌(G-AC-3/4 통합).

### G-AC-5 [MISMATCH·CONFIRM] 인쇄사양(UV 변형) → print_side 오적재

- **도메인 근거:** L1 `인쇄사양` = `배면양면\|풀빼다\|투명테두리★\|단면인쇄`. KB §2-4: **UV(PROC_000002)의 `변형` enum = 일반/배면양면/풀빼다/투명테두리/단면.** 이는 인쇄"면"이 아니라 **UV 출력 변형(인쇄방식 옵션)**.
- **라이브 재확인(SELECT #5):** 아크릴키링·마그넷 `t_prd_product_print_options.print_side` = `배면양면`·`풀빼다`·`투명테두리` 3행으로 적재(front/back 도수쌍 부여).
- **판정:** **MISMATCH(의미축)** — UV 변형을 인쇄면 컬럼에 적재. 단, print_side를 "변형 슬롯"으로 운영하는 게 의도일 수도 → **CONFIRM.**
- **처리방향(§5 R5):** ① UV 변형을 PROC_000002 process + `변형` param으로 적재하고 print_side는 실제 단/양면으로 정정, 또는 ② print_side를 변형축으로 공식 인정. 인쇄방식(PROC_000002~6)이 **272 전상품 미연결**(KB §5-1)인 글로벌 갭과 연동 — 정책 컨펌.

### G-AC-6 [MISSING] nonspec(사용자입력) 범위 전무

- **도메인 근거:** L1 11상품이 `사용자입력` 사이즈(뱃지·집게·스마트톡·명찰·머리끈·볼펜·지비츠 등). R-SIZE-5: 사용자입력 = nonspec 가로/세로 범위.
- **라이브 재확인(SELECT #2):** 입체코롯토/블럭만 `nonspec_yn=Y`, **그나마 width/height_min/max = 전 23상품 NULL.** 사용자입력 상품도 nonspec_yn=N.
- **판정:** **MISSING**(사용자입력 옵션의 치수 범위 미적재). 위젯에서 사용자입력 가로/세로 입력 범위 검증 불가.
- **처리방향(§5 R6):** L1 비규격 가로/세로 범위(또는 가격표 면적축)를 `nonspec_*_min/max`에 적재 + `nonspec_yn=Y` 정정. (round-2 면적가격과 연동 가능).

### G-AC-7 [MISSING] 추가상품(addon) 볼체인 부분 적재

- **도메인 근거:** L1 키링/포카키링 추가상품 = `볼체인` **6색**(블랙/핑크/오렌지/민트그린/핫핑크/화이트).
- **라이브 재확인(SELECT #10):** 키링(146)·포카키링(158) addon = `PRD_000006 볼체인(선택안함)` **1행씩**. 6색 미반영.
- **판정:** **MISSING/모호** — 6색이 addon 1상품(볼체인)의 색상 variant인지, 6개 addon인지 미확정. variant면 색상축 컨펌.
- **처리방향(§5 R7):** 볼체인 색상축 결정 후 적재. 색상=variant면 addon 1+색상 옵션, 6 addon이면 6행.

### G-AC-8 [CONFIRM] ★제약 상품 미등록 (쉐이커·지비츠)

- **라이브:** 아크릴쉐이커·지비츠(★) = `t_prd_products` **미등록**(prd_cd 부재). L1에서 전행 숨김+★.
- **판정:** 숨김/미출시 = 비활성(MISSING 아님). **단 "★제약=등록 보류"가 의도된 정책인지, 출시 예정인지 컨펌.** EXTRA 삭제단정 금지 원칙상 보류로 분류.

### G-AC-9 [글로벌 갭, acrylic 한정 아님] qty_unit_typ_cd 전 NULL

- **라이브:** `t_prd_products.qty_unit_typ_cd` = 272 전상품 NULL(acrylic 23 포함). digital-print와 동일.
- **판정:** acrylic 고유 결함 아님 → 글로벌 갭(QTY_UNIT 코드 부여 정책 컨펌).

## ⑦ 큰 그림 적용 (도메인 → 엑셀 → DB결함 → 처리 → 위젯)

- **도메인 KB(07_domain):** UV `변형` enum·완칼 `조각수`·MAT_TYPE.03 두께·MAT_TYPE.07 부속·부착 `대상`이
  "엑셀 인쇄사양/조각수/가공/3mm"를 옳게 해석하는 렌즈 → 원적재의 오해(인쇄면 욱여넣기·두께 평면화·부속 누락)를 결함으로 규명.
- **엑셀 L1(무손실):** 전 컬럼 보존 덕에 인쇄사양 4변형·조각수·가공 부속명·두께(3mm/8mm/1.5mm)·볼체인 6색·★숨김을 상품별 정확 추출(추정 0).
- **DB 라이브 권위:** 추출본으로는 안 보이는 "process 0행(맥세이프만)·두께 192 일괄·부속 미연결·UV→print_side"를 라이브 13 SELECT로 확정. 마스터는 풍부한데 **연결만 빈** 구조 노출.
- **처리방향:** 마스터 무변경 + 상품 연결 테이블만 적재(FK 안전). 완칼·UV변형은 모델링 컨펌 선행.
- **위젯 기반:** 정제된 9속성이 자동견적 위젯의 옵션 캐스케이드(모양→완칼, 조각수, 두께→자재, 부속→가공, UV변형)
  데이터 소스. **process 전무가 잔존하면 아크릴 위젯은 "모양/가공/조각수" 옵션이 통째로 비어** 견적 불가 → R1~R4가 위젯 선결.
