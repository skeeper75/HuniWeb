## 3. 코팅 UX 패턴 오라클 — 일반 명함(BCSPDFT) 비교

후니 040은 가격표에 코팅/무코팅 단가행을 보유하나, RedPrinting **화이트 명함은 코팅이 없음**. 따라서 코팅 UX 패턴은 **일반 명함(BCSPDFT)**에서 가져온다(동일 BC 카테고리·동일 위젯 엔진).

BCSPDFT 코팅 공정(COT_DFT 그룹, WEB_PCS_DTL_GRP=`COT_DFT_TC` 단일 토글 그룹):
| 상세 | PCS_DTL_CD | ESN(필수) | VIEW |
|---|---|---|---|
| 무광코팅단면 | TCMAS | **N** | Y |
| 무광코팅양면 | TCMAD | **N** | Y |
| 유광코팅단면 | TCGLS | **N** | Y |
| 유광코팅양면 | TCGLD | **N** | Y |

패턴 결론:
1. **코팅 = 선택(optional) 부가공정**: ESN_YN=**N**(전부). 손님이 코팅을 안 골라도 주문 진행 가능.
2. **기본값 = 무코팅**: 코팅 행에 DFT/기본 플래그 없음 → 사전선택 0 → 미선택 상태 = **코팅 안 함(무코팅)**. "무코팅" 라디오가 따로 있는 게 아니라, 코팅 공정을 추가하지 않으면 코팅 없음 = base 가격.
3. **코팅은 단면/양면 구분**: 코팅 항목 자체가 단면/양면으로 분리(무광단면/무광양면/유광단면/유광양면). 즉 코팅 면 선택이 코팅 옵션 안에 내장.
4. **코팅은 자재 종속**: disable 규칙 `{MTRL_CD: RXETP226, PCS_CD: COT_DFT}` — 특정 용지에서 코팅 공정 비활성. 코팅이 자재에 따라 노출/숨김(제약축).

## 4. 후니 040 결정 권고 (직접 오라클)

후니 040 라이브 단가행 4변형: 단면·무코팅 14,500 / 단면·코팅 16,000 / 양면·무코팅 16,000 / 양면·코팅 19,000.

### 결정 1 — 코팅 기본값 = **무코팅 (싼쪽)**
근거: RedPrinting 코팅은 사전선택 없는 부가공정 → 미선택 base = 무코팅. 손님이 코팅을 안 고르면 **무코팅(단면 14,500 / 양면 16,000)**으로 진행. (RedPrinting 화이트명함은 코팅 자체가 없어, 화이트명함의 자연 상태 = 무코팅임을 한 번 더 확증.)

### 결정 2 — 코팅 = **선택(필수 아님), 기본값 있는 토글**
근거: RedPrinting 코팅 ESN_YN=N = 필수 아님. 후니 040 코팅은 **기본 무코팅 토글**로 배선하고 손님이 코팅을 추가로 선택하면 코팅 단가행으로 환원. 코팅을 반드시 고르게 강제(required)하지 말 것.

### 배선 권고
- opt_cd 판별차원 = **코팅 유/무**(2값: 무코팅=기본 / 코팅). 단/양면은 별도 도수축.
- 단가행 매칭: (도수 단/양면) × (코팅 유/무) = 4변형 그대로. 후니 040 가격표가 단/양면 × 코팅을 이미 커플링했으므로 코팅 토글 + 도수축 2개면 4행 정확 환원.
- 코팅 미선택 시 무코팅 단가행으로(silent 누락 가드: 코팅축 미선언이면 무코팅 base가 default여야 함).
- RedPrinting naming/codes(COT_DFT·TCMAS 등)는 **후니로 유입 금지**(구조 흡수만). 후니 권위 = 상품마스터/가격표.

## 5. Ambiguous fragments (아키텍트 판단 필요)
- **화이트인쇄(PRT_WHT) 공정의 버킷**: ESN_YN=Y 필수·자동·VIEW=N. 자재(흰 토너)+공정(인쇄) 번들 성격 → 후니에선 040의 상품정체(고정 공정)로 흡수할지, 별도 공정코드로 둘지 모호. (후니 040은 큐리어스스킨 자재 고정이므로 상품 정체로 흡수가 자연스러움.)
- **코팅 단면/양면 vs 인쇄 단면/양면 독립성**: RedPrinting은 코팅면을 코팅옵션 안에서 별도 선택(인쇄 단면이어도 양면코팅 가능). 후니 040 가격표는 둘을 커플링(코팅면=인쇄면). 후니는 커플링 모델 유지 권고(가격표 차원이 그렇게 적재됨) — 단, 손님에게 코팅면 독립선택을 줄지는 상품기획 결정.
