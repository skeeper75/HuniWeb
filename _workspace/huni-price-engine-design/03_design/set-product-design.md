## 1. ★엽서북 — G-5 재정의 (세트조합 아님 = 완제품 고정가)

### 1.1 라이브 실측이 cartographer 가설(세트조합)을 반증

cartographer/gap-board G-5는 엽서북을 "내지(몽블랑240)+표지(스노우300) **별 SKU 합산 세트**"로 가설. **라이브 실측은 다른 그림**:

| 실측 | 내용 |
|------|------|
| **본체 PRD_000094 엽서북** | **PRF_PCB_FIXED 바인딩 실재**(미바인딩 아님) |
| PRF_PCB_FIXED 배선 | COMP_PCB_S1_20P·COMP_PCB_S2_20P (+ S1/S2_30P comp 실재) |
| comp 구조 | **"엽서북 완제품가 단면/양면 × 20p/30p"** — `use_dims=[siz_cd, min_qty]`·단가행 verbatim(SIZ_000003 2매=11000 등) |
| 내지/표지 별 comp | **부재** — COMP_PCB_내지·COMP_PCB_표지 같은 부품 comp 없음 |

★ **후니는 엽서북을 "내지단가+표지단가 합산"이 아니라 "페이지수(20p/30p)×인쇄면(S1/S2)×사이즈 완제품 고정단가"로 설계했다.** 상품마스터의 `엽서북-내지(몽블랑240)`·`엽서북-표지(스노우300)` 별 행은 **webadmin BOM(자재 구성)**일 뿐 — 가격은 완제품 통합단가에 이미 포함(용지 포함 고정가 패턴·명함과 동일).

### 1.2 설계 결정 — 엽서북 = 고정가형 (세트 조합 레이어 불요)

> **[보정 R-3 — 2026-06-20] "이중계상 0" 판정 철회.** 검증가 라이브 재계산(recompute §4)이 반증:
> 엽서북은 **BOM(내지+표지) 의미의 이중계상은 0이 맞지만**, 가격엔진 축에서 명함과 **동일한 두 결함**을 보유한다 —
> ① 인쇄면 S1/S2 silent 이중합산(V-DGP-1) ② prc_typ 단가형 ×qty 과대청구(D-10). 명함과 동일 결함군으로 재분류.

| 항목 | 설계 |
|------|------|
| 공식 | PRF_PCB_FIXED **유지**(신설 불요) |
| 계산방식 | 고정가형 — 완제품 통합단가 comp(PCB_S1/S2 × 20P/30P) |
| 페이지수(20p/30p) | **별 comp로 인코딩**(COMP_PCB_*_20P vs _30P) — 페이지수=공식 라우팅 or 옵션→comp 분기 |
| 인쇄면 S1/S2 | print_opt_cd 차원 충전 + use_dims 등재(명함 3.2/D-3와 동형) — **★결함 V-DGP-1: 현재 print_opt_cd=NULL → S1_20P+S2_20P silent 이중합산**(GC-7 q2: 22,000+23,000=45,000) |
| prc_typ | ★결함 D-10 동형: 전부 PRICE_TYPE.01(단가형)·단가=묶음/구간총액 → **×qty 과대청구**(GC-7 q2 라이브 45,000 vs 가격표 11,000). 교정방향 컨펌 대기 |
| 내지/표지 분리 | **가격 분리 안 함** — webadmin BOM·MES 생산정보일 뿐. 가격은 완제품 통합(BOM 의미 이중계상 0 — codex도 합의) |
| 미바인딩 PRD_000095/096 | **별도 가격 바인딩 불요**(내지·표지는 독립 판매상품 아님·BOM 행)·컨펌큐 |

★ **두 종류의 "이중계상"을 구분**(R-3 핵심):
- **BOM(부품) 이중계상 = 0** ✅ — 엽서북은 내지단가+표지단가를 합산하지 않는다(완제품 통합단가가 이미 둘 다 포함). 내지·표지를 별 comp로 합산하면 이중계상이 되므로 합산 금지 가드는 유효. codex도 "내지+표지 별도합산 안 함은 맞다"로 합의.
- **가격엔진 축 이중합산/과청구 ≠ 0** ❌ — 그러나 인쇄면(S1/S2)이 별 comp로 둘 다 배선·print_opt_cd NULL이라 **인쇄면 축에서 실제 이중합산이 발생**하고, prc_typ 단가형이라 ×qty 과청구가 겹친다. **이전 설계는 이 둘을 "이중계상 0"으로 뭉뚱그려 누락** → R-3로 정정. 엽서북은 명함과 동일 결함군(D-10 범위·V-DGP-1 동형).

### 1.3 엽서북 인쇄면 silent 이중합산 점검 (명함 D-2b 동형·R-2 정정)

> **[보정 R-2 동형 — 2026-06-20]** "ERR_AMBIGUOUS"가 아니라 **silent 이중합산**. PCB_S1_20P·S2_20P는 서로 다른 comp_cd라 `match_component` 내부에서 만나지 않으므로 ERR_AMBIGUOUS는 발생하지 않고, print_opt_cd=NULL 와일드카드로 둘 다 매칭→silent 합산된다.

PCB_S1_20P·S2_20P 둘 다 `use_dims=[siz_cd,min_qty]`·단가행 print_opt_cd=NULL → 명함 S1/S2와 같은 **silent 이중합산**(단면+양면 둘 다 합산·경고 없음). 또 20P·30P가 같은 siz_cd+min_qty면 페이지수 축에서도 동시매칭 위험. **해법(명함 D-3 동형)**: ① 인쇄면=print_opt_cd 컬럼 충전 + use_dims 등재 ② 페이지수=별 옵션→공식 라우팅 or 판별차원(예 opt_cd) 충전. dbm-price-arbiter 심의(명함 3.2 해법과 일괄). prc_typ ×qty(D-10)는 별 결함으로 컨펌 대기.

## 2. 포토카드 — P-2 묶음수 단가 (세트=bdl_qty) ★ 후니 보유

### 2.1 라이브 실측 = benchmark P-2 정합

| comp | 주문모드 | use_dims | 단가행 | 의미 |
|------|---------|----------|--------|------|
| COMP_PHOTOCARD_SET | **세트** | siz_cd·**bdl_qty**·min_qty | 1 | 묶음(세트) 단위 단가 |
| COMP_PHOTOCARD_CLEAR_SET | 투명세트 | siz_cd·bdl_qty·min_qty | 1 | 투명 세트 단가 |
| COMP_PHOTOCARD_BULK | **대량(낱장)** | min_qty | 50 | 낱장 대량 단가(bdl_qty 없음) |

★ 포토카드는 **두 주문모드**(세트·대량)를 각자 comp로 보유. benchmark P-2(bdl_qty + bundle_qtys QTY_UNIT)와 정합. **세트 가격 모델 신설 불요** — 후니가 이미 명시적.

### 2.2 설계 결정 — SET/BULK 동시매칭 방지

PRF_PHOTOCARD_FIXED에 SET 2개 + BULK 1개 전부 배선하되 **판별차원으로 동시매칭 회피**:
- SET/CLEAR_SET = `bdl_qty` 차원 보유 → 손님이 세트수 선택 → bdl_qty 매칭.
- BULK = `min_qty`만 → bdl_qty 없음 → `_combo_key`가 SET과 다름(P3-8 회피).

→ **설계: BULK를 PRF_PHOTOCARD_FIXED에 addtn_yn=Y 배선**(현재 orphan). 단, **SET 주문과 BULK 주문이 같은 selections로 동시 매칭되지 않는지** 검증 필요(세트수 미선택 시 SET combo_key=NULL+값 충돌 위험·P3-8). → 컨펌큐: 포토카드 주문 UX가 세트/대량 택1(옵션 분기)이면 안전·둘 다 노출이면 제약 필요. dbm-price-arbiter 심의.

### 2.3 bundle_qtys 그릇

benchmark: `t_prd_product_bundle_qtys(prd_cd, bdl_qty)` + `bdl_unit_typ_cd`(QTY_UNIT: EA/매/권/세트). 포토카드 세트수 옵션은 이 그릇에 충전(라이브 실재 점검 → 미적재면 dbmap 큐). 가격은 component_prices.bdl_qty 단가행이 권위.

## 3. 접지 — P-3 변형 SKU (후니 보유·세트 아님)

접지카드(PRF_DGP_E)·접지리플렛(PRF_FOLD_SUM)은 **원자합산형 완제품**(§engine-design 2-E). 접지=siz_cd SKU + 접지 comp(COMP_FOLD_*) 합산. **세트조합 아님** — 단일 완제품 공식. benchmark P-3 정합(siz_cd SKU + 후가공 comp).

- **와이드접지리플렛 PRD_000049 미바인딩** → PRF_FOLD_SUM(접지리플렛 PRD_000048과 동형) 바인딩. 단 PRF_FOLD_SUM은 현재 COMP_FOLD_CARD_2H 1 comp만 — 와이드접지=4단게이트/아코 등 접지타입 다를 수 있음(COMP_FOLD_LEAF_4GATE/4ACC 실재). 컨펌: 와이드접지 접지타입→적정 comp 바인딩(신규 mint 불요·접지 comp 7종 실재).

## 4. 세트 "구성"(template) vs 세트 "가격"(공식) 분리 (P-1b)

benchmark P-1b 핵심: 세트가 **하나의 완제 상품**으로 팔릴 때 `t_prd_templates`/template_selections가 **부품 구성**(주문/생산 단위)을 묶고, 가격은 **여전히 공식**으로 계산.

디지털인쇄 적용:
- 엽서북·포토카드는 **단일 완제품 가격(고정단가)** — template 가격 불요(엔진 우선순위 1=TEMPLATE_PRICE 미사용·라이브 template_prices 0행).
- template은 **구성**(엽서북=내지+표지 BOM 묶음·MES 생산정보)에만 쓰이고 **가격에 관여 안 함**(가격=PCB 완제품 comp).
- ★ **설계 가드**: template_selections(구성)을 가격 합산에 끌어들이지 말 것 — 이중계상. 가격=완제품 공식 단일.

## 7. 아크릴 절 — 세트/입체 합성 (면적매트릭스 종단·2026-06-20 라이브 실측)

> 입력: `formula-map-acrylic.md` §5·`set-pricing-patterns.md` P-4(P-4a/b/c)·라이브 실측. 단가=verbatim.

### 7-0. ★아크릴은 명시적 반제품(세트조합) 구조가 없다 [HARD]

calc-draft·가격표·라이브 단가행 전부 아크릴을 **단일 면적매트릭스 1 comp**(인쇄+레이저커팅+소재 통합단가)로 둔다. 디지털인쇄 엽서북 같은 "내지+표지 별 SKU 합산" 분해가 **없다**. 따라서 아크릴은 **세트조합 레이어(template 묶음·다부품 합산) 설계 불요** — 본체 면적 공식 1개 + (후가공 addtn_yn 합산)뿐.

### 7-1. benchmark P-4 3합성형 vs 라이브 실측

| 패턴(benchmark) | 정의 | 후니 그릇 | 라이브 실측·설계 판정 |
|-----------------|------|----------|----------------------|
| **P-4a 본체+부착물 BUNDLE** (키링·명찰) | 본체 면적단가 + Σ 부착물(고리/핀/자석) 가산 | 면적 본체 comp + 후가공 comp(addtn_yn=Y) Σ | ✅ 합산형(흡수 불요)·후가공 COMP_ACRYL_FINISH 신설(§6 engine-design)·고리=자재+공정 BUNDLE([[dbmap-option-material-process-bundle]]). 키링 BOM에 은색/금색고리 자재(MAT_000051/052) 실재 확인 |
| **P-4b 접합 완제 = 고정가형** (카라비너) | 형상별 통가격(면적 아님·부품 합산 아님) | 고정가 comp(.06·use_dims=[opt_cd]) | ★**합산 금지** — 카라비너(투명3T+3T 접합)는 "본체×2+접합비 합산"이 아니라 **형상 고정가 단일**(B07 5,800~6,900). "접합=세트"로 오해해 합산하면 오모델(가격표가 형상 고정가로 정의). §5-C engine-design·GC-A7 |

### 7-2. ★입체블럭·쉐이커·입체코롯토 — 다층 접합 완제품이나 가격은 단일 면적단가 [HARD]

### 7-3. 아크릴 세트 종합 판정

1. **세트조합(다부품 합성) 신규 그릇 0건** — 아크릴은 본체 면적 1 comp가 완제품가. 디지털인쇄 책자/엽서북 같은 부품 합산 부재.
2. **합성처럼 보이는 3형태 명확 분리**: 본체+부착물=합산형(addtn_yn)·접합완제=고정가형(합산 금지)·입체/코롯토=면적매트릭스 단일. 셋을 한 공식으로 강제 금지.
3. **부자재 카탈로그(고리/핀/자석)는 굿즈/스티커 횡단 공유**(benchmark C-A4·basecode 거버넌스 입력·여기선 권고).
