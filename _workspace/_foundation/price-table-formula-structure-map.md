## ★3계층 아키텍처 (최종가가 만들어지는 방식)
가격표는 "상품 1개=1가격표"가 아니라 **가격 구성요소(자재·공정·후가공) 단가 블록**들의 집합이다.
최종 상품가 = **합가(여러 단가 블록 합산)** 또는 **단가형(단일 조회)** 또는 **선조립 합가표(블록이 이미 합쳐진 상품표)**.

| 계층 | 시트 | 역할 |
|---|---|---|
| **L1 단가 블록**(atomic 단가·합산 대상) | 출력소재(IMPORT)·디지털인쇄비·코팅·접지옵션·인쇄후가공(모서리)·커팅타공·후가공_박(소형/대형/백업)·제본·아크릴·포스터사인 | 차원으로 단가 1개 조회 → 합가에 더해짐 |
| **L2 선조립 합가표**(상품표·블록이 이미 합쳐짐) | 스티커·합판도무송스티커·봉투제작·명함포토카드·엽서북떡메 | 사이즈×소재×수량 등으로 **완성가** 직접 조회(인쇄비+소재+커팅 등 내포) |
| **L3 수량·판수 modifier** | 판걸이수(임포지션 판수)·굿즈파우치(구간할인율) | 수량→출력장수 환산·구간할인율(곱) |

## Part 3 — 단가/합가 오차 위험 유형 (라이브 교차검증 타깃)
오차는 **단가 블록을 어떻게 합치느냐**에서 발생. 라이브로 잡아야 할 3종:
1. **이중합산(과대)** — L2 선조립표(이미 합가)인데 L1 단가를 또 더함(명함 S1/S2·책자 제본 다종·always-add opt_cd 와일드카드). [[catalog-conformance-foldcard-overcharge-260623]].
2. **미바인딩/부분합산(과소·저청구)** — 합가 사슬 중 일부 블록만 배선(아크릴 키링 고리 미배선·면적+등록사이즈 최소셀 고정·055 option_items 단가행 결손).
3. **차원 미스매치(견적불가/오가격)** — 도수×면·소재·판수·page 키가 단가 블록 차원과 안 맞음(053/054 견적불가·엽서북 page).

→ **라이브 교차검증 절차**: 가족 대표상품을 라이브 goods.asp에서 동일 옵션 구동 → 합계 공급가 ↔ 우리 조립 사슬 결과 대조. 차이=조사신호(권위 엑셀 재확인+무결성 재실측). 단가 블록(L1) 자체는 라이브에 직접 노출 안 되므로 **L2 선조립표·면적격자가 1차 교차검증 대상**, 합가형(디지털·책자)은 옵션 조합별 총액 대조.

## Part 4 — ★계산공식 라이브 확정 (off-grid probe 2026-06-28 · `remediation/price-formula-live-confirmation.md`)
가격표에 없는 수량(95·97)·임의 사이즈를 라이브에 입력해 실제 공식을 확정:
1. **자유수량 단가형**(아크릴 굿즈·실사 규격) = 공급가 = **단가 × 수량**(완전 선형·95/97 정확 배수·구간/할인 0).
2. **per-unit 합가** = 면적/preset base + 옵션 가산(additive·이중합산 0) → 그 뒤 ×수량.
3. **면적 사이즈 = 격자 lookup**(preset만 즉시가)·**off-grid = 견적문의**(연속 보간 아님). → 면적격자+off-grid ceiling/견적 모델 정합·임의단가 금지.
4. **밴드수량**(디지털·명함·봉투·스티커·엽서) = qty SELECT 밴드 lookup(×qty 아님·off-grid 입력 불가).
→ ★합가 오류 3근원(라이브가 정답 제시): ① 밴드수량을 ×qty로 모델 ② L2 선조립표인데 L1 단가 또 더함(이중합산) ③ 면적 연속 보간. **자유수량/밴드수량 구분 + L2 verbatim + 면적격자(off-grid=ceiling/견적)** 지키면 오차 0.
