## 3. verbatim(L1) 독립 재현 — 엑셀 byte 대조

상품마스터 `후니프린팅_상품마스터_260610.xlsx` `굿즈파우치(가격포함)` 시트 직접 추출:

| 케이스 | 생성자 주장 | 엑셀 L1 실측(행·셀) | byte 일치 |
|--------|:--:|------|:--:|
| G1 캔버스삼각파우치·M·라벨없음 | 9,800 | 행157: `14788\|캔버스 삼각 파우치\|M\|...\|9800\|라벨없음\|0` | ✅ |
| G2 캔버스삼각파우치·L·라벨부착 | 11,500 + 라벨 300 | 행158(L 병합행): `L\|...\|11500\|라벨부착\|300` | ✅ |
| G3 캔버스에코백·가로L·라벨없음 | 35,200 | 행210: `14706\|캔버스에코백\|가로L\|...\|35200\|라벨없음\|0` | ✅ |
| G4 라벨부착 추가가 = +300 (6상품 일관) | {'300': 6} | 가공_가격(col18) 전수 분포 = `{'0.0':7, '300.0':6, '6500.0':1}` → 라벨부착 300이 정확히 6건 | ✅ |

> **verbatim 4/4 엑셀 byte 일치.** 캔버스 삼각 파우치는 M(행157)·L(행158) 2행 병합 상품이며 M=9,800/라벨없음/0, L=11,500/라벨부착/300이 정확. 생성자 "값 결함 0" 주장 독립 확증. (참고: col18의 `6500.0` 1건은 맥세이프추가[행308 폰케이스]로 굿즈파우치 시트 내 별 상품군 — 라벨 +300 분포와 무관, 생성자 G4 산정에 영향 없음.)

## 4. 거짓 GO / 거짓 NO-GO / 날조 적발

**적발 = 0건.** 다음을 능동적으로 의심·검증했으나 모두 정직:

- **거짓 NO-GO 의심(가격포함 시트라 GO여야?)**: L1엔 포함가 9,800/11,500이 있으나 라이브 price_formulas/component_prices/바인딩이 통째로 0행임을 독립 재현 → 견적 엔진 도달 경로 0 = NO-GO가 정직. 생성자의 "가격포함 시트라 가격 있음 = GO 처리하면 거짓 GO"라는 자기-경계가 옳다.
- **과장 의심(삼중 전무가 부풀려진 것?)**: CPQ 0 + 공식 0 + 단가행 0 + 봉제 오적재 + 자재 오염을 각각 독립 쿼리로 전부 재현. "극단 형태" 묘사는 사실과 일치(아크릴은 단가행 ACRYL 121행 실재 vs 굿즈는 단가행도 0 — 한 단계 더 비어있음이 실측 확인).
- **대표 끼움 의심(미출시/이질 상품을 대표로?)**: PRD_000240은 use_yn=Y·editor_yn=Y·PRD_TYPE.03(기성)·정상 카테고리 노드 — 미출시/고아 아님. GP-A(패브릭 봉제류) 클래스 선정 근거(봉제 완성공정·라벨/사이즈 옵션 superset) 타당. 만년스탬프(editor_yn=N·미출시 의심)를 대표에서 정직하게 제외함.

## 5. 용어 정밀화 (minor — 비준 영향 없음)

비준을 좌우하지 않으나 정확성 기록:

1. **"봉제 마스터행 실재·링크만 틀림"(5layer §1①)** — 봉제(PROC_000080)는 마스터 실재가 맞으나, 라이브에서 PROC_000080은 굿즈가 아닌 **PRD_000124~138(현수막 영역) 5행에 링크**돼 있다. 굿즈에는 봉제 0행이 정확. 생성자 표현은 옳으나 "봉제가 어디에 쓰이는가"는 현수막 영역임을 부기(교정 라우팅 GP-Q3 시 봉제 코드 재사용 가능 확인).
2. **G2 "캔버스 삼각 파우치·L·라벨부착"** — L1 구조상 L 사이즈(행158)와 라벨부착(가공옵션)이 같은 병합행에 묶여있다. 즉 라벨부착이 L 사이즈에 종속된 것처럼 표현되나, 실제로는 사이즈등급(M/L)과 가공(라벨없음/라벨부착)이 별개 축일 가능성이 높다(M 행157도 라벨없음 단일). 생성자의 "라벨없음/라벨부착 2종 옵션" 해석은 타당하나, CPQ 신설(GP-Q1) 시 사이즈축과 라벨축의 직교성을 L1 병합구조로 단정하지 말 것(컨펌 Q-GP-1/3 의존) — 이미 생성자 인간승인 큐에 반영됨.
