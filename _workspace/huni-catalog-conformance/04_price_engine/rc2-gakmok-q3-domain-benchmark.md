## 0. 질문 (Q3)

각목(상하/좌우 마감용 나무봉) 가격 구분 "각목 900mm 이하(4,000원) / 900mm 초과(8,000원)"를:
- **(a) 손님이 직접 택1 선택** — 가격표 구조 그대로·데이터만 적재·코드 불요, vs
- **(b) 입력 사이즈에서 자동 판정** — 각목 다는 변 길이를 시스템이 보고 900 이하/초과 자동 결정·위젯 코드+사이즈 슬롯 정합 동반.

확정 맥락(변경 금지): 권위 가격표 260527 포스터사인 r249-250 = 각목(900이하)+끈=4,000 / 각목(900초과)+끈=8,000.
단가 2종뿐(양쪽·개수 별가 없음). Q1=손님 택1 확정·Q2=세로/가로는 가격 무관 생산정보로 분리 확정.

## 1. 인쇄 도메인 관행 결론

### 결론: 각목/마감봉은 **손님이 직접 고르는 후가공·부자재 옵션**이며, 길이 구분도 **손님 선택**이 업계 관행. 자동 사이즈→가격 판정은 표준 사례를 한 건도 찾지 못함.

근거(도메인 일반 — WebSearch·인쇄 가이드):

1. **각목 = 후가공/부자재 택일 옵션.** "후가공에서 '각목+끈'을 선택하시면…봉미싱 후 타공처리된 각목을 넣어
   고정"(eprinting/printagency 가이드). 각목은 출력 본체와 별개로 **손님이 거치 방식에 따라 더하는 마감
   부자재**다. 거치 방식(담장/펜스=그로밋, 거치대=봉, 유리창=큐방, 야외장기=미싱)을 **손님이 용도에 맞게
   고른다**는 게 도메인 기본 구조(publog 현수막 가이드).
   - 근거: [퍼블로그 현수막 제작 가이드](https://www.publog.co.kr/blog/banner-guide),
     [프린트에이전시 현수막 5000-900mm](https://printagency.net/shop/실사출력/현수막/현수막5000-900mm/)

2. **각목 길이 구분도 손님 선택 enum으로 제시.** 각목 길이는 두 갈래로 제시되며 손님이 고른다 —
   "① 일반형: 출력물보다 50mm 이상 외부 노출 / ② 출력물 길이에 맞게: 각목 길이=출력물 길이(외부 노출 0)".
   즉 길이는 **시스템 자동 산식이 아니라 마감 방식 선택지**로 제공된다.
   - 근거: [퍼블로그 현수막 제작 가이드](https://www.publog.co.kr/blog/banner-guide)

3. **각목 부착 변과 길이의 도메인 정의(생산 사실).** 각목(나무봉)은 현수막의 **마감 변(상하 가로 또는
   좌우 세로)에 끼우는 봉**으로, 봉 길이 = 각목이 달리는 변의 길이다. 권위 가격표의 900mm 임계는 이 "봉이
   달리는 변의 길이"가 900 이하/초과인지로 갈린다(모델 문서 §1-4 도메인 정합과 일치). 다만 **어느 변에
   다는지(가로/세로)는 가격에 영향 없음**(가격표에 세로/가로 컬럼 없음 — Q2 확정).

4. **자동 길이→가격 판정 사례 부재.** 조사한 어느 인쇄소(레드/와우/eprinting/printagency/pinot/bizhows)도
   "입력 사이즈에서 각목 길이를 읽어 900 임계를 자동 적용"하는 UX를 노출하지 않았다. 모두 **손님이 각목
   옵션(및 길이/위치)을 직접 고르는 드롭다운/체크박스** 구조였다. (자동 판정이 존재한다는 증거를 못 찾음 —
   "없음"이 아니라 "표준 관행은 아니며 사례 미발견"으로 정직 표기.)

## 3. 자동 판정(b) 시 비용 — §6 코드 트랙 범위

후보 (b)를 택하면 다음이 동반 필요(전부 §6 위젯 코드 트랙·BLOCKED 요인):

1. **사이즈 슬롯 정합 선결(돈크리티컬 함정).** 라이브 `nonspec_width`(짧은변 500~1,750)가 권위 엑셀
   가로(500~5,000)와 **뒤바뀌어** 등록됨(모델 문서 §1-4). 자동 판정은 "어느 슬롯 길이를 900과 비교할지"가
   슬롯 정의에 의존하는데, 그 정의 자체가 권위와 어긋난 상태 → **슬롯을 먼저 권위(가로/세로)에 맞춰
   정리하지 않으면 자동 판정이 엉뚱한 변을 보고 4,000/8,000을 오판정**한다(과대/과소청구 위험).

2. **위젯 비교 로직 신규 구현.** "각목 다는 변의 길이"를 위젯이 정해 `selections.siz_width`로 주입해야 함
   (`pricing.py` `_opt_maps`는 width/height를 환원하지 않음 — 프론트 주입 의존, 모델 문서 §2). 부착변
   (세로/가로) 선택 → 해당 변 길이 → 900 비교 → opt_cd 결정의 **새 코드 경로**가 필요.

3. **권위 구조 변형.** 권위 가격표(r249-250)는 각목을 **"각목 900이하 / 각목 900초과"라는 손님 선택 enum**
   으로 못박았다(모델 문서 §1-7). 자동 판정은 이 "손님 선택 enum"을 "엔진 자동 길이 티어"로 재해석하는
   것 = **권위 구조 변형**. 권위 우선 원칙 위반 소지.

→ (b)는 **데이터만으로 닫히지 않고**, ① 슬롯 정합 ② 위젯 코드 ③ 권위 변형 승인 3건이 선결돼야 하므로
**BLOCKED**(코드 동반·인간 권위 변형 승인 필요).

## 4. Q3 권고

### 권고 = **(a) 손님이 직접 택1 선택** — 도메인 관행·경쟁사 전건·권위 가격표 구조 3중 정합

**왜 (a)인가:**

1. **권위 가격표가 이미 손님 선택 enum 구조.** r249-250이 "각목(900이하)+끈 / 각목(900초과)+끈"을
   **두 개의 옵션명**으로 둠 → (a)는 가격표 구조 그대로(무변형). (b)는 이를 자동 티어로 재해석=권위 변형.
2. **인쇄 도메인 관행 정합.** 각목은 손님이 거치 방식·길이를 고르는 후가공/부자재 옵션(§1). 자동 길이
   판정은 도메인 표준이 아님(사례 미발견).
3. **경쟁사 전건 정합.** 레드·와우·eprinting·printagency 모두 손님 직접 선택(§2). 자동 판정 0건.
4. **데이터만으로 즉시 적재 가능.** 옵션 선택→opt_cd 전송(엔진 기본 경로)·**위젯 코드 변경 불요·슬롯
   정합 불요**. 모델 문서의 후보 C와 동일 귀결.
5. **돈크리티컬 함정 회피.** (b)의 슬롯 뒤바뀜 오판정 위험(§3-1)을 원천 차단.

**트레이드오프(정직 표기):**
- (a)의 단점 = 손님이 "내 각목 봉 길이가 900 넘는지" **스스로 판단**해야 함. 단, 각목은 출력 변 길이에
  끼우는 봉이라 손님이 자기 현수막 변 길이를 알고, 경쟁사도 동일하게 손님 판단에 맡기므로 도메인상
  수용 가능한 부담. (편의를 위해 "각목 다는 변이 900mm 넘으면 초과를 고르세요" 안내 텍스트 보강 권장 —
  코드 불요.)
- (b)의 장점 = 손님 판단 부담 제거(편의). 단 §3의 3대 비용(슬롯 정합·위젯 코드·권위 변형 승인)을 치러야
  하며, 본 §21 정합 트랙 범위 밖(코드 동반).

### 적재 가능성

- **(a) 채택 시: 즉시 데이터 적재 가능.** 코드 불요. 모델 문서 §6 "후보 C 데이터 트랙"
  (comp 통합 1개·use_dims=`[opt_cd]`·단가행 opt_cd=각목≤900·4,000 / 각목>900·8,000 verbatim 2행·
  공식 바인딩 addtn_yn=Y) 그대로 실행 가능. **단가 verbatim·날조 0.**
- **(b) 채택 시: BLOCKED.** §6 코드 트랙(위젯 비교 로직)+사이즈 슬롯 정합 선결+권위 변형 인간 승인이
  완료돼야 적재 가능. 본 정합 트랙(§21)에서 즉시 적재 불가.

## 5. 모름/미확인 (정직 표기·추측 0)

1. **경쟁사 실제 가격 매트릭스 비노출.** 레드·와우 모두 각목 옵션의 **구체 단가/길이별·위치별 가격을
   화면에 노출하지 않음**("파일분석에 따라 변동"·"0원" 표시·주문 시 합산). 따라서 "경쟁사도 길이별 2단
   단가를 쓰는가"는 화면만으로 확정 불가 — **확인된 것은 '손님 직접 선택 UX'까지**이고, 그들의 내부 가격
   산식이 후니의 900임계 2단과 동일한지는 미확인(주문/결제 진행 금지 원칙상 더 못 봄).
2. **레드프린팅 "상하 가로/좌우 양끝 세로" 위치 옵션의 가격 영향 여부 미확인.** 위치 선택은 노출됐으나
   그것이 가격을 바꾸는지는 화면에서 확정 못 함. (후니는 위치=가격 무관 확정이나, 레드 내부는 미확인.)
3. **자동 판정 부재 = "사례 미발견"이지 "업계에 절대 없음"이 아님.** 조사 범위(대표 5개 인쇄소·읽기전용
   화면)에서 자동 판정을 못 찾았다는 사실이지, 전 업계 전수 부정은 아님(정직 한계 표기).

## 6. 판정

- **인쇄 도메인 관행 + 경쟁사 전건 + 권위 가격표 구조 = 모두 (a) 손님 직접 택1과 정합.**
- **Q3 권고 = (a).** 데이터만으로 즉시 적재 가능(모델 문서 후보 C 데이터 트랙)·코드 불요·돈크리티컬 함정
  회피. (b)는 코드 동반·슬롯 정합·권위 변형 승인 필요로 **BLOCKED**(편의 외 이점이 비용을 정당화 못 함).
- **본 리서치는 인간 최종 결정 입력.** 경쟁사 naming/codes 후니 유입 0·권위 가격표 단가(4,000/8,000)
  덮어쓰기 0·추측 적재 0·모름은 §5에 정직 표기.
