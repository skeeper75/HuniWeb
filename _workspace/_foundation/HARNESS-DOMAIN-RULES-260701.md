## 판형·판걸이수 엔진 배선 (이번 세션 라이브 실증 — 권위 사실)

가격 설계·진단 전에 이 배선을 반드시 전제로 삼는다. (근거: `raw/webadmin/webadmin/catalog/pricing.py`·`price_views.py` 라이브 실측)

- **판형(plt_siz_cd)은 고객이 안 고른다.** `price_views.py:1757` — 완제품사이즈(siz_cd) 선택 시 `fn_best_plate(prd_cd, item_siz_cd)`가 자동 도출한다.
- **`fn_best_plate` 후보 = `t_prd_product_plate_sizes`(상품↔판형 연결, prd_cd별·del_yn='N').** 그중 **판수>0**인 것만, "1장당 판면적 최소"로 1개 선택. → **상품↔판형 연결이 이 테이블이다.** 연결이 다 판수0이면 `NULL` 반환 → 판형 미결정 → 가격 0.
- **판걸이수 = `fn_calc_pansu(plt_siz_cd, item_siz_cd)`** = `t_siz_pansu` lookup(prd_cd IS NULL generic) 우선 → 없으면 기하 폴백((판형 work−여백)/(아이템 work), 회전 GREATEST). 2인자 시그니처 고정(pricing.py 무수정).
- **판형기준 구성요소**(use_dims에 `plt_siz_cd` 포함: 인쇄비·용지비·코팅비 등)는 `plate_qty = ⌈주문수량 ÷ 판걸이수⌉`(pricing.py:215)로 단가 조회. 단가행은 선택된 plt_siz_cd로 매칭돼야 함(안 맞으면 "제외·해당없음"으로 저청구/견적불가).
- **판형 후보 플래그 = `t_siz_sizes.impos_yn='Y'`**(조판판형여부). **여백 = `margin_top/bot/lft/rgt`**(그리퍼·물림·칼선). 출력용지규격(전지)의 실제 인쇄영역 = work − 여백.

## 12 규칙

### 1) 판걸이수 = 상품↔판형 매칭·판형종류 확인·여백 있는 출력용지규격 (§26·§18) [수정]
- **규칙:** 판걸이수는 (a) 상품이 어떤 판형(전지)에 걸리는지 매칭(`t_prd_product_plate_sizes`), (b) 판형종류 확인(국4절/3절/46전지…·`impos_yn='Y'`), (c) 여백(그리퍼·물림) 있는 실제 출력용지규격 위에서 산출된다.
- **왜:** 순수 기하계산은 여백·칼선 간격을 반영해도 실무 판걸이수보다 과다 → 판수 과소 → 저청구. 실무진 판걸이수가 권위.
- **적용:** 기하 재현 불가 사이즈는 `t_siz_pansu`(권위 판걸이수 저장소)에 lookup 적재. 자재종속(예 투명엽서 6 vs 일반 8)은 prd_cd 인자 필요 = pricing.py 수정(C트랙).
- **검증:** webadmin 가격시뮬레이터가 "주문 N장 ÷ 판걸이수 K = M판" 표시로 실측.

### 2) 출력소재(IMPORT): 판형 없는 소재 제외·하드커버 전용지 복합식 (§26·§29·§18) [확장]
- **규칙:** 판형(plate)은 **종이류 출력소재에만 유효**. 판형 없는 소재(아크릴·실사천·굿즈·전용지 등)는 판형 매칭에서 제외하고 가격확인. 하드커버 전용지처럼 **여러 요소를 합쳐야 하는 계산식**은 복합식으로 설계.
- **왜:** 비종이류에 판형 강제하면 오적재/견적불가. 하드커버 표지=전용지 단가+제본이 얽힘.
- **적용:** [[platesize-paper-only-diagnosis-260630]] 종이류 1차 판정 우선. 전용지 복합식은 §18에서 구성요소 분해.

### 3) 면지 = 색상(화이트/블랙/그레이) 구분·제본비 포함 무가격 (§23·§18) [보완]
- **규칙:** 면지(end-paper)는 색상으로만 구분하고, 가격은 **제본비에 포함**돼 별도 가격 없음(0).
- **왜:** 면지 단독 단가를 매기면 이중청구. [[set-semifinished-3tier-model-260629]] 면지=제본비 포함.
- **적용:** 셋트 구성원으로 면지 mint 시 unit_price=0·제본 공정에 흡수. 색상 = 옵션/자재값.

### 4) 계산식이 2배/시장가 초과 = 가격테이블 lookup 산출 (§18·§7) [추가]
- **규칙:** 계산식(원자합산)으로 산출 시 **2배 가격**이 나오거나 **시장가보다 높게** 나오는 상품은, 계산식 대신 **가격테이블을 만들어** 해당 정보(사이즈·옵션 등)와 매칭해 가격을 뽑는다.
- **왜:** 이중합산·판형 과다·복합옵션 등으로 공식이 실가를 못 맞추는 상품군 존재. 룩업이 정답.
- **적용:** §18 설계 시 "공식형 vs 테이블형" 결정. 테이블형 = `t_prc_component_prices`에 (차원)×단가 직접 적재(prc_typ 고정가/완제품가 활용). 경쟁사 벤치마크가 합리성 오라클.

### 5) 상품마스터 권위·(가격포함) 표기 확인·가격표 참조 (§1 core) [보완]
- **규칙:** 상품의 모든 정보는 **상품마스터(260610) 해당 시트**에 기록. 상품마스터가 권위, 시트의 **"(가격포함)"** 표기 부분 반드시 확인, **참조는 인쇄상품 가격표(260527)**.
- **왜:** 권위 이원(상품마스터=상품·가격 정본 / 가격표=참조). "(가격포함)"=직접단가 상품 신호. [[price-authority-model-reframe]]·[[catalog-price-3model-coverage-260629]].
- **적용:** 시트 (가격포함) 상품은 공식형 아닌 직접단가로 취급.

### 6) 실사시트 전 가격 = 인쇄상품가격표 포스터/사인 (§26·§7) [보완]
- **규칙:** 실사시트의 모든 가격은 인쇄상품 가격표의 **포스터/사인**에서 확인.
- **왜:** 실사=[가로×세로] 면적매트릭스, 포스터/사인 단가가 원천. [[dbmap-silsa-price-via-poster-sign]].
- **적용:** 실사 상품 가격진단·설계는 포스터/사인 매트릭스로 환원.

### 7) 차원 필요 테이블 vs 수량×단가 테이블 = 등록방식 구분 (§26·§7·§18) [추가]
- **규칙:** 가격테이블 중 (a) **차원이 필요한 테이블**(사이즈·자재·도수·판형 등 use_dims 매칭)과 (b) **수량×단가만 계산**하는 부분이 공존. 각각 어떻게 등록할지 구분해 설계.
- **왜:** (a)는 use_dims+component_prices 다차원, (b)는 min_qty 구간 단순. 혼동 시 오적재.
- **적용:** (a) use_dims 선언+차원행 충전 / (b) min_qty만 있는 단가구간. prc_typ으로 단가형/합가형/고정가 구분.

### 8) 상품별 출력용지규격(판형) 상이 → 설계 반영 (§26·§18·§29) [수정]
- **규칙:** 각 상품은 출력용지규격(판형사이즈)이 **다르므로** 이를 고려해 설계. 상품마다 맞는 판형을 `t_prd_product_plate_sizes`에 연결.
- **왜:** 이번 세션 실증 — 썬캡은 국4절 연결(판수0)이라 견적0. 3절 330×540 연결+단가행+`fn_best_plate` 자동선택으로 정상화. [[pansu-authority-fn-calc-pansu-260628]].
- **적용:** 상품 가격설계 시 판형 연결(`t_prd_product_plate_sizes`)·해당 판형 단가행·판걸이수 3종을 세트로 확인.

### 9) 충분조건 미달 시 역산 준비 확인 → 시뮬레이터 입증 (§26·§13·§15·§27) [추가]
- **규칙:** 가격계산의 **충분조건이 안 됐을 때** 반드시 "역으로 계산하려면 어떤 준비(판형연결·차원행·공정바인딩·단가행)가 돼야 하는지"를 확인하고, **가격시뮬레이터로 입증**한다.
- **왜:** 결함을 "안 됨"으로 끝내지 말고, 무엇이 갖춰지면 되는지 정지척도로. [[live-measure-batch-builder-260629]]·[[price-model-decision-tree-sim-gate]].
- **적용:** 견적0/저청구 발견 시 → 필요 선행조건 목록화 → 하나씩 갖춤 → 시뮬레이터 "제외 0·PRICE≠0" 확인.

### 10) 도메인·이전 후니사이트·레드프린팅 = 상품구성요소 참조 (core) [보완]
- **규칙:** 인쇄지식 도메인, **이전 후니프린팅 사이트(huniprinting.com)**, 레드프린팅을 상품 구성요소 참조로 쓴다.
- **왜:** 구성요소·옵션·가격 골든의 교차 오라클. 권위(엑셀) 덮어쓰기는 금지(갭헌팅 보강용).
- **적용:** 구성요소 누락/모호 시 이전사이트 goods.asp·레드 상품페이지 대조.

### 11) 위젯 재료 = wowpress+widget_monitor+reversing / 레드 역공학 2문서 필독 (§6·§19·§22) [확장]
