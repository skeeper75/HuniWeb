당신은 인쇄 자동견적 카탈로그의 "독립 2차 교차검증자"입니다. 한 인스펙터 팀이 작성한 결함 보드를
독립으로 재검토합니다. 그들의 GO/NO-GO 판정은 제공되지 않습니다(독립성). 인쇄 도메인 지식과
제공된 권위·라이브 사실만으로 스스로 판단하세요. 당신은 라이브 DB에 직접 접속할 수 없으니
"제공된 라이브 실측 사실 + 권위 + 도메인 논리"의 정합만 판정하고, 불확실하면 CONFIRM/추정으로 표기하세요.

== 배경 ==
대상: 후니프린팅 라이브 DB의 책자10·문구9·상품악세사리15 = 34상품(배치2). 권위=상품마스터 엑셀(260610)+인쇄상품 가격표(260527).
가격엔진=evaluate_price(단일 권위 알고리즘). 공식→formula_components→component_prices→단가행.
삭제 권위=del_yn='Y'(논리삭제). 옵션→차원 연결=option_item.ref_key1→siz/proc/mat 마스터(polymorphic).
addon→템플릿 연결=t_prd_product_addons.tmpl_cd→t_prd_templates.base_prd_cd.

== 도메인 렌즈(권위 발췌) ==
- [HARD] 자재(materials)=종이/소재 사양만. 링·투명커버·삼각대 등 부속물은 제본옵션/공정/표지옵션 컬럼(자재 아님)이 원칙.
- [HARD] 부속물(악세사리)=완제 부속품. 사이즈(필수) 컬럼에 변형(색/길이)이 들어감. 가격=자기 고정가.
- [HARD] PRICE=0은 항상 우리측 결함(RedPrinting은 0 반환 안 함). 미바인딩/미가격=견적 차단.
- 인쇄면(단면/양면)은 가격 판별차원이어야 함. 같은 use_dims·print_opt_cd=NULL인 단/양면 comp 둘을 동시 배선하면
  단/양면 선택과 무관하게 둘 다 와일드카드 매칭되어 "silent 이중합산"(경고 없이 틀린 합계로 성립).
- pricing.py(evaluate_price)는 comp/단가행 조회 시 del_yn 필터를 적용하지 않는다(아래 S-CODE 참조).

== 라이브 실측 사실(읽기전용 SELECT, 검증자가 독립 재실측 완료 2026-06-23) ==
S-CODE. raw/webadmin/.../pricing.py 실코드 확인:
  - 공식 구성요소 조회 = TPrcFormulaComponents.objects.filter(frm_cd=...)  ← del_yn 필터 없음
  - 단가행 조회(_component_rows) = TPrcComponentPrices.objects.filter(comp_cd=...)  ← del_yn 필터 없음
  - 즉 del_yn='Y'(논리삭제) comp·단가행도 가격 합산에 그대로 포함됨(코드 전체에 'del_yn' 문자열 0회).

[쟁점1 — 엽서북 094 silent 이중합산]
S1a. PRD_000094 엽서북 바인딩 공식=PRF_PCB_FIXED. 그 공식에 배선된 comp = COMP_PCB_S1_20P, COMP_PCB_S2_20P (2개).
S1b. COMP_PCB_S1_20P·S2_20P·S1_30P·S2_30P 각 117행, **전 행 print_opt_cd=NULL**.
S1c. S1=단면 의미, S2=양면 의미. use_dims는 둘 다 [siz_cd,min_qty](인쇄면 판별차원 부재).
S1d. 30p variant(COMP_PCB_S1_30P/S2_30P)는 단가행 실재하나 공식에 미배선(orphan).
S1e. 단가값: SIZ_000003·min_qty=2에서 S1=11,000 / S2=11,500(가격표 verbatim).

[쟁점2 — DEAD_LINK 책자 사이즈 옵션]
S2a. 책자 사이즈 옵션항목(option_item, ref_dim_cd=OPT_REF_DIM.01)이 가리키는 siz 마스터의 del_yn:
  - PRD_000068 중철책자: SIZ_000172(N 활성) + SIZ_000170(Y 삭제)
  - PRD_000069 무선책자: SIZ_000172(N) + SIZ_000170(Y 삭제)
  - PRD_000071 트윈링책자: SIZ_000172(N) + SIZ_000170(Y) + SIZ_000253(Y) + SIZ_000255(Y) = 3건 삭제
S2b. 삭제된 siz 마스터: SIZ_000170='A5(148x210mm)'·SIZ_000253='A5(210x148mm)'·SIZ_000255='A4(297x210mm)' 전부 del_yn=Y.
  → 고객이 A5/A4를 옵션으로 고르면 ref_key1이 활성 siz로 환원 안 됨(dead link).

[쟁점3 — del_yn='Y' comp 합산 포함]
S3a. PRF_BIND_SUM(책자 068~071 바인딩)에 배선된 제본 comp = COMP_BIND_JUNGCHEOL(중철) 1개뿐. 그 comp del_yn='Y'(삭제).
S3b. 제본 comp del_yn 전수: 활성(N)=COMP_BIND_TWINRING·SSABARI·CAL_WALL만. 나머지 삭제(Y)=JUNGCHEOL·MUSEON·PUR·HC_MUSEON·HC_TWINRING·CAL_DESK130/220/MINI.
S3c. S-CODE에 의해 del_yn=Y인 JUNGCHEOL도 평가에 포함됨 → 068(중철)은 우연히 맞지만 069(무선)·070(PUR)·071(트윈링)도
  자기 제본 comp(MUSEON/PUR/TWINRING)가 아닌 중철값(JUNGCHEOL)으로 매겨짐(misfire). 표지/내지 인쇄·용지 comp는 배선 0(미완성가).

[쟁점4 — 악세 부속물 변형 축귀속(sizes vs materials)]
S4a. PRD_000006 볼체인: t_prd_product_sizes 0행. materials 8행(전부 usage_cd=USAGE.07), 행 이름=
  '볼체인 오렌지(3개1팩)'·'볼체인 핑크(3개1팩)'·…·'볼체인 화이트(3개1팩)' = 색상 변형 8종.
S4b. 권위 상품마스터는 이 변형을 "사이즈(필수)" 컬럼에 8변형으로 둠. 라이브는 sizes 0·materials 8(색상 변형을 자재로 등록).
S4c. 동형: 와이어링007(3변형)·천정고리008(1)·행택끈010(3)·우드거치대012(1)·우드봉013(3)·우드행거014(3)·리필잉크015(7색)
  모두 sizes 0·materials N(USAGE.07). 봉투류(001~005·009·011)는 치수형이라 sizes 정상 등록(MATCH).
S4d. 책자 071·082·088: 바인더링(화이트/블랙/메탈)·투명커버(유광/무광)를 materials(USAGE.05/07)에 등록(종이 외).

[쟁점5 — Q-PA-ADDON 악세 이중역할]
S5a. 엽서 PRD_000016의 t_prd_product_addons = 5행. 각 tmpl→base_prd_cd:
  TMPL-000005→PRD_000001(OPP접착봉투)·TMPL-000006→PRD_000002(OPP비접착봉투)·TMPL-000009→PRD_000283(트레싱지봉투)·
  TMPL-000010→PRD_000281(카드봉투화이트)·TMPL-000011→PRD_000282(카드봉투블랙).
S5b. 악세 001/002의 자체 t_prd_product_prices=0행. t_prd_template_prices 전체=0행.
  → 악세는 "자기 완제품가(product_prices)"와 "본상품 addon가(template_prices)" 양쪽 다 0(가격 전무).
S5c. 악세 003~015는 어느 본상품 addon에도 base로 안 붙음(연결 자체 부재).
