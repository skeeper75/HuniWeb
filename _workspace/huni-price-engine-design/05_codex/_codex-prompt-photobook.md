# codex 독립 2차 교차검증 프롬프트 — 포토북 가격엔진 설계 (Phase 5.5)

> ★우리(Claude) 게이트 판정·E1~E7 결론 일절 비노출. codex는 설계·지도·골든·권위·엔진계약만 보고 자기 독립 판정을 낸다.

## 프롬프트 (codex exec stdin)

당신은 인쇄 자동견적 가격엔진 설계의 독립 외부 심사관입니다. 한 한국 인쇄회사(후니프린팅)의 "포토북"(사진북·반제품 세트상품·8x8/10x10/A5/A4 사이즈 × 하드커버/레더하드커버/소프트커버 표지 × 페이지수 4~150P) 가격계산 엔진 설계를 받았습니다. 이 설계가 실제로 가격계산을 성립시키는지 독립적으로 판정하세요. 다른 사람의 판정은 받지 않았습니다 — 당신의 독립 의견이 필요합니다. 가격은 돈 크리티컬입니다. 환각 금지 — 데이터에 없으면 "데이터 부족"이라고 하세요.

### 배경 (엔진 계약 = 라이브 pricing.py, 고정·변경 불가)
가격엔진 evaluate_price는 가격 소스를 우선순위로 선택합니다:
1. TEMPLATE_PRICE: 타깃이 template(tmpl_cd)이고 t_prd_template_prices.unit_price 존재 → unit_price × qty
2. PRODUCT_PRICE: t_prd_product_prices.unit_price 존재 → unit_price × qty (FORMULA보다 선점)
3. FORMULA: 위가 없으면 상품-공식 바인딩(t_prd_product_price_formulas)으로 공식 평가(구성요소 합산 Σ)
4. NONE: 전부 없으면 0원(strict는 error)
- 엔진은 frm_typ_cd(공식유형)를 참조하지 않음 — 공식은 항상 구성요소 합산(addtn_yn='Y' 항의 Σ).
- 구성요소 소계: prc_typ가 단가형(.01)이면 `unit_price × qty`. 합가형(.02)이면 `unit_price ÷ tier_min_qty × qty` (tier min_qty 없으면 ValueError).
- 차원 매칭: siz_cd·mat_cd·proc_cd·opt_cd 등 비수량 차원은 정확매칭(NON_QTY_DIMS) — 한 구성요소 안에서 후보가 2개 이상이면 ERR_AMBIGUOUS. 단가행에 차원값이 NULL이면 와일드카드(아무 값에나 매칭). min_qty는 수량구간 tier 룩업 키(단가형일 때 분모 아님).
- 서로 다른 comp_cd는 각자 독립 매칭·합산되며 서로 ERR_AMBIGUOUS를 일으키지 않음.

- 포토북 세트 부모 PRD_000100: t_prd_product_prices 0행 / 공식 바인딩(t_prd_product_price_formulas) 0행 / 포토북 전용 공식(frm_cd ILIKE %PHOTOBOOK%) 0행 / 수량구간할인(t_prd_product_discount_tables) 바인딩 0행. → 포토북 가격사슬 사실상 전무(견적 불가·source=NONE).
- 세트 그릇 t_prd_product_sets(PRD_000100): 7행 실재(reg_dt 2026-06-03·전건 sub_prd_qty=1). 구성 = 내지(몽블랑130, PRD_000101)/표지 하드커버(102)/표지 아트250+무광코팅(103)/면지 그레이(104)/표지 레더하드커버(105)/표지 레더(106)/표지 소프트커버(107).

- 라이브에 실재하는 재사용 구성요소: 제본비 COMP_BIND_PUR(8행·prc_typ .01·proc_cd PROC_000020·5000~1500/부 수량tier)·용지비 COMP_PAPER(내지 몽블랑130 MAT_000105 SIZ_499=77.03/판·표지 아트250 MAT_000081 SIZ_499=77.75/판)·표지 무광코팅 COMP_COAT_MATTE(prc_typ .01·92행 단가행)·인쇄비 COMP_PRINT_DIGITAL_S1.
- 포토북 전용 신규 구성요소 COMP_PHOTOBOOK_BASE/COMP_PHOTOBOOK_PAGE는 라이브 0행(부존재).

### 권위 가격값 (상품마스터 "포토북(가격포함)" 시트 inline verbatim — 날조 금지)
**base24 = 기본가(24P 기준 완제 상품단가). 차원 = [사이즈 × 표지타입]:**
- 8x8: 하드커버 15,000 / 레더하드커버 23,000 / 소프트커버 12,000
- 10x10: 하드커버 22,000 / 레더하드커버 32,000 / 소프트커버 **(시트 공란)**
- A5: 하드커버 12,000 / 레더하드커버 19,000 / 소프트커버 10,000
- A4: 하드커버 16,000 / 레더하드커버 26,000 / 소프트커버 13,000

**per2p = 추가단가(2페이지당). 차원 = [사이즈]만(표지타입 무관·같은 사이즈면 하드/레더/소프트 전부 동일):**
- 8x8 = 500 / 10x10 = 1,000 / A5 = 300 / A4 = 600

**상품마스터 row17 명문 산식(verbatim):** `상품단가+페이지당단가적용` + 비고 `그레이칼라는 노출안되는 정해진 사양`·`핑크는 편집기를 통해 나오는 사양이므로 옵션노출 없음`.
**포토북 전용 가격표 시트는 부재**(가격표 260527 19시트 중 포토북 없음·제본/디지털인쇄 시트가 부품 단가 보조). 표지/내지 부품별 단가표는 권위에 없음(완제 base24만 제공).

### 설계 요지 (판정 대상)
- 계산방식 = "부품합산 세트형 + 페이지 선형". 판매가 산식:
  `포토북가격(N pages, Q부) = base24[siz, 표지mat] × Q + per2p[siz] × ceil((N − base_min) / 2) × Q`
  - base_min = 하드/레더하드 = 24, 소프트 = 4(추정·page_rule 권위 주입).
  - 증분횟수 = ceil((N − base_min)/2) = 앱 런타임 계산(off-grid ceiling 동류·DB 미저장). DB는 base24·per2p 단가만 룩업.
- 신규 공식 1개(PRF_PHOTOBOOK_SUM·합산형). 구성요소 2개 배선:
  - COMP_PHOTOBOOK_BASE: prc_typ .01 단가형·use_dims=[siz_cd, mat_cd]·base24 완제 통합단가(표지/내지/제본/면지 부품을 이 안에 묶음=internalize·외부 미배선).
  - COMP_PHOTOBOOK_PAGE: prc_typ .01 단가형·use_dims=[siz_cd]·per2p 페이지당단가(표지 무관 1차원).
- 책자(다른 상품군) 공식 PRF_BIND_SUM 공유는 부결하고 PRF_PHOTOBOOK_SUM 신설. 사유 = 책자는 제본비+표지+내지를 각자 별 comp로 full 분해 합산하고 그 공식의 comp는 COMP_BIND_JUNGCHEOL(중철) 단일인데, 포토북은 base24(묶음)+per2p 2 comp라 comp 집합이 근본적으로 상이.
- base24를 부품(제본+용지+인쇄+코팅) Σ로 분해하지 않고 완제 통합단가 1단가행으로 적재. 사유 = 권위 가격표가 부품별 단가를 안 주고 완제 base24만 줌(부품 역산은 추측).
- 표지타입 mat_cd = MAT_000005 하드커버 / MAT_000006 레더하드커버 / MAT_000007 소프트커버. base24는 표지타입(mat_cd) 3종으로 분기. (sub_prd 106 "레더"는 레더하드커버 표지 소재 variant로 보고 별 base24 행으로 두지 않음 — 인간 컨펌 남김.)
- 부모 PRD_000100 → PRF_PHOTOBOOK_SUM 바인딩(FORMULA). sub_prd 101~107은 가격 비기여(생산 BOM·표지/면지 택1 선택지). PRODUCT_PRICE INSERT 금지(product_prices 0행 유지·본체에 값 넣으면 FORMULA 통째 우회 silent).
- 신규 테이블/가격축 0. 10x10 소프트커버(시트 공란)·base24 부품 정확 역산은 추측 금지로 BLOCKED 표기.

### 당신이 독립 판정할 열린 질문 (우리 결론 없음 — 당신의 답을 직접 산술로 주세요)
각 질문에 SOUND/RISK/FLAW 명확한 판정 + 근거. 데이터로 직접 계산하세요.

1. **★페이지 곱 산출(GC-PB-7)**: 8x8 하드커버 150페이지 1부 가격을 위 산식으로 **직접 계산**하세요. base24(8x8 하드)와 per2p(8x8), base_min(하드=24)을 대입해 증분횟수와 최종가를 산출하세요. 그리고 만약 설계가 페이지 곱 항(per2p × 증분횟수)을 누락하면 얼마가 나오는지, 그 과소청구 배수가 몇 배인지 정량화하세요.

2. **★per2p 차원(GC-PB-8 vs GC-PB-5)**: per2p가 사이즈(siz_cd)에만 종속하고 표지타입(mat_cd)과 무관하다는 설계가 권위 데이터와 정합하는가? 10x10 하드 40P 1부와 8x8 하드 40P 1부를 각각 계산해 per2p 차이(1,000 vs 500)가 가격에 어떻게 반영되는지 보이세요. per2p에 표지(mat_cd) 차원을 잘못 넣으면 무슨 문제가 생기는가?

3. **★부수 곱 양방향(GC-PB-10)**: 8x8 하드 40P를 10부 주문하면 가격은? base24와 per2p가 둘 다 ×부수가 되는지, 증분횟수(페이지 기반)는 부수와 독립인지 직접 계산해 보이세요. base만 ×부수 하고 per2p는 1배로 두는 오류 시 얼마가 나오는가?

4. **★소프트 base_min 결판**: 하드/레더하드는 base24가 24P 기준(증분 시작 24), 소프트는 base24가 4P 기준(증분 시작 4)이라는 설계가 위험을 가지는가? page_rule이 라이브에 소프트 전용으로 미저장(부모 24/150/2만)인 상태에서 소프트 base_min을 24로 오적용하면 무슨 일이 생기는가? A5 소프트 24P 1부를 base_min=4와 base_min=24 두 경우로 계산해 차이를 정량화하세요.

5. **PRF_BIND_SUM 공유 부결**: 책자 기존 공식을 재사용하지 않고 포토북 전용 공식을 신설하는 게 정당한가, 아니면 공식 중복(과적재)인가? 두 공식의 구성요소 집합이 다르면 공유 시 무슨 일이 생기는가(같은 공식에 서로 다른 comp가 배선됐을 때의 매칭 거동)?

6. **base24 통합 vs 부품 분해**: 권위 가격표가 완제 base24만 주고 표지/내지/제본 부품 단가표를 안 줄 때, base24를 완제 통합단가 1행으로 적재하는 것과 부품(제본+용지+인쇄+코팅) 단가행 합산으로 분해하는 것 중 어느 쪽이 옳은가? 부품 분해 시 위험(추측 단가·이중 배선·골든 불일치)이 있는가?

7. **돈크리티컬 가드(놓친 것 포함)**: 다음 위험이 실재인가 과잉설계인가? (a) base24를 product_prices에 INSERT하면 FORMULA(per2p 페이지 가산)가 통째 우회되는가? (b) base24 단가행에 mat_cd 판별차원이 NULL이면 무슨 일(소프트 12,000 주문에 하드 15,000 오청구 등)? (c) base24/per2p를 .02 합가형으로 오적용하면? 그리고 설계가 추가로 놓친 가격축/구성요소/세트 이중계상(표지 5 variant 합산 등)이 있는가?

각 항목 SOUND/RISK/FLAW 명시. 근거는 위 데이터로 직접 산술. 환각 금지 — 데이터에 없으면 "데이터 부족"이라고 하세요.
