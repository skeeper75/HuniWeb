# 책자(booklet) — CPQ 옵션/템플릿/제약 반영 설계 (cpq-plan)

> **작성** 2026-06-14 · round-13. 사용자 기준 5번 = "기초데이터(size/자재/공정/도수/판형/묶음수)만이 아니라 option/template/constraint까지 묶어 설계".
> **라이브 CPQ 스키마(2026-06-14 실측):** `t_prd_product_option_groups`(opt_grp_cd·sel_typ_cd·min/max_sel_cnt·mand_yn) → `t_prd_product_option_items`(opt_cd·ref_dim_cd polymorphic·ref_key1/2·qty) → `t_prd_product_constraints`(rule_typ_cd·logic JSONLogic·err_msg). 검증 트리거 `fn_chk_opt_item_ref` 실재.
> **라이브 책자 CPQ 실측:** option_groups **0행**·constraints **0행**·sets(반제품 바인딩) **21행**. CPQ 옵션 레이어 전면 미적재(OM-6 정합).
> **[HARD] 설계까지** — 실제 CPQ 적재는 인간 승인. 차원행 미적재면 "L1 선적재 BLOCKED" 정직 표기.
> **책자 특수성:** 반제품=표지+내지 결합 전체관점(Q3). 표지/면지/링/D링은 sets(반제품 바인딩)으로 표현됨 → CPQ template 역할을 sets가 이미 수행.

## 2. 제본방식 — 상품=제본 1:1 (BK-9 CORRECT·CPQ 불요)

**핵심:** 책자는 **상품명 자체가 제본방식**(중철책자→중철제본·무선책자→무선제본·PUR책자→PUR제본·트윈링책자→트윈링제본·하드커버→하드커버무선·레더→레더표지·레더링바인더→D링). 한 상품이 한 제본이라 **제본 택일그룹(GRP-BOOK-제본)이 불필요**.

- 라이브 option_groups 0행 = 정합(택일 불요). RedPrinting류 "한 위젯에서 제본 택1"과 다른 모델 — 후니는 제본별 SKU 분리.
- **CPQ 레이어 전면 적재(round-6) 시에도 책자 제본은 option 아님.** 단, "책자" 상위에서 제본방식을 1차 선택 → 해당 SKU로 라우팅하는 **상위 내비게이션(카테고리 트리)**가 그 역할 = BK-CAT 전용 잎노드 재연결이 곧 그 내비게이션 정합화.

> **결론:** 제본방식은 CPQ option_group 아닌 **카테고리 잎노드(BK-CAT)** + 상품 SKU로 표현. 본 정정(전용 잎노드 재연결)이 제본 선택 UX의 토대.

## 3. 표지/면지/링 결합(반제품) — sets = template (Q3 전체관점)

**문제:** B 셋트(하드커버/레더/하드커버링/레더링바인더)는 표지(반제품)+면지+링/D링을 결합한 완성품. 이 "결합"이 CPQ template 역할.

**라이브 현황:** `t_prd_product_sets` 21행이 완제품↔반제품(sub_prd) 결합을 이미 바인딩(072→073~076·077→078~081·082→083~087·088→089~093·094→095/096·097→098). **CPQ template 신규 설계 불요** — sets가 그 자리.

**정정과의 정합:**
- BK-2(반제품 078 자재 잡음 삭제)는 sets 바인딩을 건드리지 않음(자재행만 논리삭제). 결합 구조(sets 21행) 보존.
- 자재 권위=완제품 usage_cd(표지=USAGE.02·면지=USAGE.03·링=USAGE.07). 반제품은 빈 껍데기 → CPQ가 반제품 BOM 조회 시 완제품 usage 슬롯이 답.

> **결론:** 책자 반제품 결합은 sets로 이미 template화됨. CPQ 신규 적재 불요. 정정은 sets 무손상.

## 4. 박칼라 8종 — option_group(택1·박색 풀) (BK-12 CORRECT)

**문제:** 무선/PUR(069/070)이 박색 자식 8종(PROC_000037 홀로그램~PROC_000044 트윙클)을 개별 공정 행으로 연결, 부모 박(PROC_000033) 미연결. 디지털 박색 패턴과 동형 라이브 전역 일관.

**설계(069 무선 기준):**

```
[기초데이터] t_prd_product_processes: 박색 8종(037~044) 이미 적재 ← 무교정(BK-12)
[option_group] OG-069-BAKCOLOR (박색)
  sel_typ_cd=택1 · mand_yn=N (박 선택 시)
  [option_items] ref_dim_cd=공정(OPT_REF_DIM 공정) · ref_key1=proc_cd
    홀로그램(037)/금유광(038)/은유광(039)/먹유광(040)/동박(041)/적박(042)/청박(043)/트윙클(044)
```

**fn_chk_opt_item_ref 정합:** ref_dim_cd=공정 · ref_key1=PROC_000037~044는 전부 `t_proc_processes`/`t_prd_product_processes`에 실재 → 트리거 무결성 통과(존재하는 공정행 참조). 박칼라=공정(Q2 ★)이라 ref_dim_cd=공정이 정답(자재/도수 아님).

> **권장:** 박색 8종이 이미 자식 공정행으로 존재하므로 option_items로 묶으면 부모 박 추가 없이 CPQ 완결. BK-12=무교정(라이브 자식행 정합). CPQ 적재는 round-6 인간 승인.

## 5. 표지코팅·투명커버·제본방향·면지(옵션) — option_group(택1)

**설계(중철 068 표지코팅 기준):**

```
[option_group] OG-068-COVERCOAT (표지코팅)
  sel_typ_cd=택1 · mand_yn=N
  [option_items] ref_dim_cd=공정 · ref_key1=코팅 proc_cd (또는 "없음")
    무광코팅 / 유광코팅 / 코팅없음
```

- 표지코팅·투명커버·제본방향·면지는 L1에서 "(옵션)" 표기 → 선택성. 공정/자재 차원행 선적재 후 option_items로 참조.
- **BLOCKED 점검:** 코팅 공정행이 상품에 연결돼야 ref 가능. 무선/PUR엔 코팅 공정 일부 실재하나, 옵션 풀("무광/유광/없음")을 다 담으려면 공정행 누락분 점검 선행 → **부분 BLOCKED**(공정행 선적재 의존).

## 6. CPQ 적재 준비 상태 요약

| CPQ 항목 | 기초데이터 선적재 | CPQ 적재 가능? | BLOCKER |
|----------|-------------------|:--:|---------|
| 제본방식(택1) | — | **불요** | 상품=제본 1:1(BK-9)·카테고리 잎노드가 내비게이션 |
| 반제품 결합(template) | sets 21행 실재 | **불요(이미 sets)** | — |
| 박색 8종(택1) | 자식 8행 이미 적재 | ✅ 가능 | round-6 인간 승인 |
| 표지코팅·투명커버(택1) | 공정행 일부 | 부분 | 코팅 공정행 누락분 선적재 |
| 제본방향·면지(택1) | 자재/메타 | 부분 | 옵션 차원행 선적재 |

> **종착(비파괴):** 본 산출은 CPQ **설계까지**. option_groups/items/constraints 실 적재는 ① 기초데이터(코팅·옵션 공정행) 선적재 ② round-6 CPQ 트랙 인간 승인 후. 책자 CPQ 핵심 = **제본=상품 1:1(option 아님)·반제품 결합=sets(이미 template)·박색=공정 자식(이미 적재)** → 신규 CPQ 부담이 가장 낮은 상품군. 본 정정(BK-CAT 잎노드 재연결)이 제본 선택 UX의 카테고리 토대를 정합화.
