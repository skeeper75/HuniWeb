# codex 독립 2차 교차검증 프롬프트 — 상품악세사리 가격엔진 설계 (Phase 5.5)

> ★우리(Claude) 게이트 판정·E1~E7 결론 일절 비노출. codex는 설계·지도·골든·권위·엔진계약만 보고 자기 독립 판정을 낸다.

## 프롬프트 (codex exec stdin)

당신은 인쇄 자동견적 가격엔진 설계의 독립 외부 심사관입니다. 한 한국 인쇄회사(후니프린팅)의 "상품악세사리"(부자재: 봉투·케이스·볼체인·와이어링·리필잉크·우드부속·자석·끈) 가격계산 엔진 설계를 받았습니다. 이 설계가 실제로 가격계산을 성립시키는지 독립적으로 판정하세요. 다른 사람의 판정은 받지 않았습니다 — 당신의 독립 의견이 필요합니다.

### 배경 (엔진 계약 = 라이브 pricing.py, 고정·변경 불가)
가격엔진 evaluate_price는 가격 소스를 우선순위로 선택합니다:
1. TEMPLATE_PRICE: 타깃이 template(tmpl_cd)이고 t_prd_template_prices.unit_price 존재 → unit_price × qty
2. PRODUCT_PRICE: t_prd_product_prices.unit_price 존재 → unit_price × qty
3. FORMULA: 위가 없으면 상품-공식 바인딩(t_prd_product_price_formulas)으로 공식 평가(구성요소 합산)
4. NONE: 전부 없으면 0원(strict는 error)
- 엔진은 frm_typ_cd(공식유형)를 참조하지 않음 — 공식은 항상 구성요소 합산.
- 구성요소 소계: prc_typ가 단가형(.01)이면 unit_price × qty. 합가형(.02)이면 unit_price ÷ tier_min_qty × qty (min_qty 없으면 ValueError).
- 차원 매칭: siz_cd·opt_cd·bdl_qty 등은 정확매칭(NON_QTY_DIMS) — 손님 선택 variant 1행만 후보.
- 수량구간할인(_quantity_discount): 상품에 t_prd_product_discount_tables 링크 없으면 no-op(정가 그대로).

### 후니 라이브 현황 (읽기전용 실측)
- 상품악세사리 18상품: t_prd_product_prices 0행 / 공식 바인딩 0행 / t_prd_template_prices 0행 / CPQ option_items 0(테스트 잔재 2) / 수량구간할인 바인딩 0행.
- 봉투 addon 배선은 실재: t_prd_product_addons 5행 (엽서 PRD_000016 → TMPL-000005/006/009/010/011).
- 즉 가격사슬이 사실상 전무한 상태.

### 권위 가격값 (상품마스터 inline 고정가, verbatim)
- 볼체인(8색 전부): 1,000 (3개1팩) / 와이어링(3색 전부): 500 / 만년스탬프 리필잉크(7색 전부): 2,500 (5cc)
- OPP접착봉투(11규격, 전부 50장 팩): 70x200=1,100 … 100x200=1,200 … 230x350=3,250
- 트래싱지 카드봉투(같은 규격×다른 묶음수): 160x110 20장=6,000 / 40장=12,000 / 100장=28,000 ; 100x100 20장=6,800/40장=13,600/100장=32,000 ; 70x100 30장=3,600/100장=10,000
- 카드봉투: 화이트 165x115(10장)=1,000 / 블랙 165x115(10장)=1,500
- 우드행거: 230mm=16,000 / 320mm=18,000 / 440mm=20,000 ; 투명케이스: 42x57x20=3,000/75x75x15=3,000/75x110x15=3,500

### 설계 요지 (판정 대상)
- AC-1 단일고정가 3상품(볼체인·와이어링·리필잉크): 색상이 가격에 무영향(전 variant 동가) → t_prd_product_prices.unit_price 1행 INSERT (PRODUCT_PRICE 경로). 공식·구성요소 신설 안 함.
- AC-2 변형고정가 11상품(봉투·케이스·우드 등): variant마다 다른 고정가 → 신규 공식 3개 + 구성요소 3개(use_dims = ["siz_cd"] / ["siz_cd","bdl_qty"] / ["opt_cd"]) + variant당 단가행 1행. prc_typ = 단가형(.01). PRODUCT_PRICE INSERT 금지(FORMULA만).
- addon 봉투: 봉투가 엽서 addon으로도 붙음(이중역할). t_prd_template_prices.unit_price 적재 → TEMPLATE_PRICE 경로. 독립판매(PRODUCT_PRICE/FORMULA)와 다른 테이블이라 경로 분기.
- 신규 테이블/가격축 0. 수량구간할인 신설 0(부자재 미해당).
- 묶음수 "(50장)"·"(20장)"은 단가행 식별/판별차원(어느 팩이냐)이지 ÷ 분모 아님 — 1팩당 고정가에 ×qty(팩수).

### 당신이 독립 판정할 열린 질문 (우리 결론 없음 — 당신의 답을 주세요)
각 질문에 ABSORBED/CONDITIONAL/REJECT 또는 SOUND/RISK/FLAW 식으로 명확한 판정 + 근거를 주세요:

1. **부자재 inline 고정가 모델 적정성**: 봉투·고리·끈·잉크 같은 부자재가 인쇄 BOM(자재/공정/도수) 없이 단순 inline 고정가 × 수량으로 가격계산이 성립하는가? 적정한가?
2. **사이즈+묶음 복합("160x110mm 20장")**: 같은 규격이라도 묶음수가 다르면 다른 가격(20장6000 vs 100장28000)인 트래싱지를 어떻게 담아야 하는가? variant 식별축(.01 팩단가)인가, 묶음 환산(÷min_qty .02 합가형)인가? 잘못 선택 시 위험은 구체적으로 무엇인가?
3. **variant별 다른 고정가의 그릇**: OPP 11규격 / 트래싱지 묶음별처럼 variant마다 다른 고정가를 product_prices(unit_price 1개)에 담을 수 있는가? option_items add_price(컬럼 없음)? formula? 어느 것이 맞고 왜인가?
4. **product_prices + formula 동시 존재 위험**: 한 상품에 product_prices 행과 공식 바인딩이 둘 다 있으면 엔진이 어느 것을 먹는가? AC-2를 formula로 가는데 product_prices를 1건이라도 넣으면 어떻게 되는가?
5. **봉투 이중역할(독립판매 + 엽서 addon)**: 같은 봉투가 직접 주문도 되고 엽서 부속(addon)으로도 붙을 때 가격 충돌이 있는가? 두 경로의 단가는 어떻게 정합되어야 하는가?
6. **silent 결함 헌팅**: 이 설계에 숨은 이중합산, 과청구/과소청구, 신규 가격축 오신설, 평탄화(variant를 한 값으로 뭉갬) 위험이 있는가? 있다면 어디서·얼마나(구체 수치)?

### 골든 케이스 (직접 재계산해서 맞는지 확인)
산술로 직접 검산하세요(허용오차 0):
- GC-AC2: 볼체인 1,000 × 50 = ? (수량구간할인 없음)
- GC-AC6: OPP접착봉투 230x350 단가 3,250 × 1 = ?
- GC-AC8: 트래싱지 160x110 100장 단가 28,000 × 1 = ? (100장 주문이 20장가 6,000으로 뭉개지면 안 됨)
- GC-AC11: 우드행거 440mm 20,000 × 2 = ?
- GC-AC12: 투명케이스 75x110x15 단가 3,500 × 10 = ?
