# M1 — 열재단(熱裁斷) 도메인 리서치: 별도 공정인가, 기본 마감 sentinel인가

> **목적:** 후니 CPQ 일반현수막(PRD_000138)의 가공 옵션그룹 `[택1·필수·기본값=열재단]` (열재단 / 타공 / 양면테입 / 봉미싱)을 적재할 때,
> **열재단**을 (①) 마스터 공정으로 신규 등록할지, (②) "추가 가공 없음(기본 재단 마감)" sentinel로 다룰지 결정하기 위한 **도메인 증거 수집**.
> 본 문서는 **결정하지 않는다** — 도메인 evidence + 권고만 제시하고, lead가 경쟁사 매핑 findings와 결합해 최종 결정한다.
> **read-only 리서치. DB 미적재·git 무변경.**

## 0. 요약 (TL;DR)

- **열재단 정체:** 실사·현수막(플렉스/패브릭) 원단의 **가장자리를 뜨거운 인두/열칼(hot knife)로 재단하는 행위 그 자체**. 합성섬유가 열에 녹아 융착(cauterize)되어 **올풀림(fraying)을 막는 sealed edge**가 단 한 번의 패스로 만들어진다. = "자르는 동작 = 마감"이 합쳐진 형태.
- **별도 가공 vs 기본 마감:** 산업 표준·후니 공정관리 doc·후니 DB 공정 마스터 **3개 증거선이 모두 "②기본 마감(default sentinel)"으로 수렴**.
- **도메인 verdict(권고):** **② default sentinel** (별도 공정 신설 불요). 가장 강한 단일 근거 → 경쟁사(비즈하우스/RedPrinting 계열)가 열재단을 **명시적으로 "(기본 마감)·0원(무료)"** 으로 표기하고, 미싱/봉미싱은 "시접을 접어 재봉하는 별도 천 처리"로 구분.
- **후니 공정관리 doc finding(실사):** 실사 finishing은 `실사출력 → 쿨코팅 → 조립/후가공 → 1차+박스포장`(Case7)으로만 표기되며, **열재단·재단을 별도 추적 공정으로 나열하지 않는다**. 현수막은 아예 **외주(Case17 "실사현수막")** 로 처리.

## 1. 열재단 정체 (what it physically is)

### 1.1 물리적 정의
열재단은 현수막/배너용 합성 원단(플렉스·패브릭·메쉬 등)의 가장자리를 **뜨거운 인두(hot iron) 또는 열칼(hot knife)로 재단**하는 마감 방식이다.

- 한국 업계 문구(비즈하우스): "현수막을 **뜨거운 인두로 재단마무리**하여 **올이 풀리지 않아**, 각종 전시회 및 실내에서 압정, 스템플러 또는 양면테이프로 깔끔하게 부착할 수 있습니다." (열재단 = **(기본 마감)·0원(무료)**)
- 한국 업계 문구(printn): "현수막의 양 끝을 **뜨거운 인두로 재단**하는 방법으로, **추가 천 처리 없이** 직접 열로 마감합니다. 올풀림이 적어 마감이 깔끔한 **기본 방식**."
- 영문 도메인(Carolina Knife): hot knife는 "as the blade slices through the material, the thermal energy simultaneously **melts the fibers along the cut line, cauterizing the edge and fusing the strands together** … in a single pass." → **자르는 패스 = 봉합 패스**. "using a heated knife to seal the edges **may allow you to skip hemming**."

→ 핵심: **열재단은 "잘라내는 동작"과 "가장자리 봉합 마감"이 한 패스로 합쳐진 것**. 별도의 천 추가·접기·재봉이 없다.

### 1.2 미싱(봉제)/타공과의 관계
| 가공 | 물리 동작 | 추가 천 처리 | 후니 공정 대응 |
|---|---|---|---|
| **열재단** | 열칼로 자르며 가장자리 융착 | **없음**(자름=마감) | (마스터에 없음 — sentinel 후보) |
| 일반미싱 | 시접(30mm) 뒤로 접어 재봉 | 있음(접기+재봉) | PROC_000080 봉제(말아박기/오버로크) |
| 봉미싱(나무미싱) | 시접(100mm) 접어 봉/각목 삽입공간 재봉 | 있음(접기+재봉) | PROC_000080 봉제(봉미싱 enum) |
| 줄미싱 | 줄 삽입 후 재봉 | 있음 | PROC_000080 봉제 계열 |
| 타공(아일렛) | 금속 아일렛 펀칭·부착 | 있음(부착) | PROC_000079 타공 |
| 양면테이프 | 테이프 부착 | 있음(부착) | PROC_000081 부착(테입) |

→ **미싱·타공·양면테이프는 전부 "원단에 무언가를 추가/접기/부착"하는 별도 노동**. 반면 **열재단은 원단을 그냥 자르면 자동으로 되는 가장자리 상태**다. 이 구조가 곧 "열재단 = 추가 가공 없는 기본"을 뜻한다.

## 2. 별도 공정 vs 기본 마감 — 양쪽 읽기 정직 평가

### 2.1 후니 공정관리 doc 증거 (`docs/huni/후니프린팅_공정관리_시행초안_20260210.pdf`)
PDF 전문(pdftotext -layout, 127행) 정밀 grep 결과:

- **"열재단"·"천재단"·"용단"·"heat" 단어 0건** — 공정 플로우/팀별 현황표/To Do List 어디에도 없음.
- **실사 finishing (Case7 실사/사인):** `실사출력 → 쿨코팅 → 조립/후가공 → 1차포장+박스포장`. → 재단·열재단을 **별도 단계로 나열하지 않음**. finishing은 뭉뚱그린 "조립/후가공"뿐.
- **현수막 (Case17 외주제작):** `책자제작 → 실사현수막 → 합판인쇄 → 고주파 → 1차포장`. → 현수막은 **외주(현수막(외주) 팀)** 로 들어왔다 나가며, 내부 재단/열재단 공정이 **추적 대상 자체가 아님**.
- **To Do List 공정명(추적되는 공정):** 출력완료 / 커팅완료 / 봉제미싱입고 / 아크릴가공입고 / 제본입고 / 박작업입고. → **재단·열재단은 독립 추적 공정이 아니며**, "출력완료"에 흡수되거나 외주 합류로 처리.
- 도메인 원칙(p1): "앞공정 안찍으면 다음공정 못찍히게" = **노동/팀 단위 공정만 추적**. 열재단처럼 출력물을 자르는 default 마감은 추적 단위가 아님.

→ **후니 자체 doc는 열재단을 distinct 공정 step으로 취급하지 않는다.** (실사 finishing에 열재단 명칭 부재 + 현수막 외주 처리)

### 2.2 후니 DB 공정 마스터 증거 (`00_schema/ref-processes.csv`, 83행 전수)
- **"열재단"·"천재단"·"heat"·"용단" 공정 0건.**
- 재단/cut family 실재: `PROC_000027 직각`(note="default 재단") · `PROC_000053 완칼`(Die Cut 종이+후지) · `PROC_000054 반칼`(Kiss Cut 스티커) · `PROC_000055 스티커완칼`.
- **현수막 가공으로 실제 적재된 것:** PROC_000079 타공 / PROC_000080 봉제(오버로크·말아박기·봉미싱) / PROC_000081 부착(라벨·맥세이프·끈·테입). (과업 명시: PRD_000138에 타공079·봉제080·부착081만 로드, 재단/cut 공정 없음.)
- 주목: **`PROC_000027 직각`의 note가 "default 재단"** — 후니 스키마는 이미 "기본 재단 = 별도 가공 없음"을 **default sentinel 패턴으로 인코딩하는 전례**를 갖고 있다(귀돌이 그룹의 직각=기본, 둥근=가공). 열재단도 동형(현수막 가공그룹의 "기본 자름").
- **[추정]** 현수막의 양면테이프 = PROC_000081 부착(대상=테입)에 흡수 가능. 봉미싱 = PROC_000080 봉제(유형=봉미싱)에 흡수 가능. 타공 = PROC_000079. → **열재단을 뺀 3개 옵션은 이미 기존 공정에 1:1 대응**. 열재단만 마스터에 대응이 없다는 사실 자체가, 열재단이 "공정"이 아니라 "기본값(가공 안 함)"임을 시사.

### 2.3 산업 표준 증거 (WebSearch/WebFetch 검증)
- **비즈하우스(bizhows) 옵션 설정:** 열재단 = **"(기본 마감)"**, 가격 **"0 원 (무료)"** 로 명시 표기. (WebFetch 직접 인용)
- **printn 현수막 가공 팝업:** 열재단 = "기본 방식 / 추가 천 처리 없음", 일반미싱·나무미싱·줄미싱 = "시접 접어 재봉하는 별도 처리(시접 30~100mm)".
- **RedPrinting (BN/BNBNFBL):** 현수막 상품에서 동일 4-옵션 가공그룹 운영(검색 결과 노출) — 후니 옵션그룹이 RP 패턴과 동축임을 시사. **[추정]** RP도 열재단을 기본값으로 둠(WebFetch 미검증, 검색 노출만).
- **영문 도메인:** "heat/hot-knife seals the edge in the cutting pass → may skip hemming." 즉 **hemming(미싱)은 열재단 위에 더해지는 별도 보강**, 열재단은 자르면 기본으로 얻는 가장자리.

### 2.4 "①별도 공정"으로 읽을 여지 (정직한 반대 증거)
- 열재단도 **노동(열칼 작업)이 실제로 들어가며**, 일반 기계재단(직각 종이 재단)과는 장비·인건이 다르다. "공정"으로 본다면 틀린 말은 아니다.
- 가격엔진 관점: 열재단이 **0원이 아닌 면적당 단가**를 갖는 업체도 존재할 수 있다 **[추정]**. 후니 가격표에 현수막 열재단 단가행이 있다면 "공정"으로 볼 근거가 생긴다(→ **lead가 가격표를 교차 확인 필요**).
- 그러나 위 2.1~2.3 증거는 모두 "열재단 = 무료·기본·자름=마감"으로 수렴하므로, 반대 읽기의 weight는 낮다.

## 3. "기본 선택" 관점 — 기본값=열재단의 의미

- 옵션그룹 `[택1·필수·기본값=열재단]`에서 **기본값=열재단**은 도메인상 **"표준 재단, 추가 가공 없음"** 을 뜻한다.
- 즉 고객이 아무 가공도 추가하지 않으면 → 현수막은 **열재단된 깔끔한 가장자리 상태로 그냥 나간다**. 타공·양면테이프·봉미싱을 **고를 때만** 그 위에 별도 노동이 추가된다.
- 이는 후니 스키마의 기존 sentinel 패턴(`PROC_000027 직각 = "default 재단"`, 귀돌이 그룹)과 **정확히 동형**: 그룹의 첫 멤버 = "기본(가공 없음)", 나머지 = "추가 가공".
- **함의:** 열재단을 마스터 공정으로 신설하면, 가격 0·노동 추적 없는 "no-op 공정"이 생겨 공정 추적 모델(2.1의 노동단위 추적 원칙)과 충돌한다. sentinel로 두면 모델 일관성이 유지된다.

## 4. 도메인 verdict (lead용 권고)

> **② default sentinel (기본 마감 = "추가 가공 없음")** 로 다루는 것을 권고한다. 별도 마스터 공정 신설 불요.

**증거 weight 종합:**
| 증거선 | 판정 | weight |
|---|---|---|
| 후니 공정관리 doc (실사 finishing에 열재단 부재, 현수막 외주, 추적 공정명에 재단 없음) | ② | 高 |
| 후니 DB 공정 마스터 (열재단 0건 + 직각="default 재단" sentinel 전례 + 나머지 3옵션은 기존 공정 대응) | ② | 高 |
| 산업 표준 (비즈하우스 "(기본 마감)·0원", printn "추가 천 처리 없음", 영문 "cut=seal, skip hemming") | ② | 高 |
| 반대 읽기 (열칼 노동 실재, 가격표에 단가행 있을 가능성 [추정]) | ① 여지 | 低 |

**lead가 결합/확인할 것:**
1. **경쟁사 매핑 findings** (RP BNBNFBL의 가공그룹 인코딩 — 열재단을 옵션 멤버로 두는지 vs 기본 sentinel로 두는지)와 본 도메인 verdict를 대조.
2. **후니 인쇄상품 가격표**에 현수막 "열재단" 단가행이 실재하는지 교차 확인 — **있으면 ①(가격 갖는 공정) 재검토, 없으면 ② 확정**. (가격엔진=권위, note 추정 금지 원칙)
3. sentinel 구현 형태 결정: (a) 가공그룹 옵션 멤버로 "열재단(기본·0원)" 1행을 두되 process 참조 없이 → 가격 0 sentinel, 또는 (b) `PROC_000027 직각` 패턴처럼 "기본 마감" 멤버를 그룹 root 자식으로. **[lead 결정 영역 — 본 문서는 권고만]**
