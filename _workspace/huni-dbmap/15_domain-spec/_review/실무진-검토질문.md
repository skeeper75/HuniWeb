> **목적:** 위 5개 상품군의 엑셀 데이터를 시스템 데이터로 정리하면서 **사람이 결정해야 할 항목**을 모았습니다. 데이터 분석으로 "이렇게 하면 될 것 같다"는 **권장안**까지 냈으니, 실무진께서 **맞다/다르다/보류**만 표시해 주시면 됩니다.

## 이 문서 읽는 법

- 각 질문은 **무엇을 / 왜 / 어떻게(선택지)** 순서입니다. 어려운 시스템 용어 대신 인쇄 작업 용어로 적었습니다.

## 📐 Q7 제안 — 합판도무송 형상의 DB 관리방안 (실무진 요청)

> **전제(실무진 확정):** 도무송 칼틀이 **형상(정사각/직사각/원형) × 치수별로 물리적으로 존재** → **사이즈 1개 = 칼틀 1개 = 칼선 1개(1:1)**. 사이즈를 선택하면 칼선이 자동 결정된다. 사이즈에 형상을 넣은 것은 고객이 빠르게 고르게 하려는 의도이고, **그래서 커팅(옵션)에 같은 것을 중복으로 넣지 않았다.**

**현재 관련 스키마:**
- `t_siz_sizes`(사이즈 마스터): `siz_nm`·재단/작업 치수·여백·`impos_yn`(조판가능). 합판도무송은 `정사각 30×30mm (2EA)`처럼 **형상+치수+조각수(EA)**를 `siz_nm`에 인코딩.
- `t_prd_product_sizes`(상품별 사이즈): `prd_cd + siz_cd` 연결.
- `t_proc_processes`(공정): 도무송/완칼 계열(예 `PROC_000055`) + `prcs_dtl_opt`(JSON, `모양` 등 파라미터).
- `t_prd_product_processes`(상품별 공정): `prd_cd + proc_cd` + `mand_proc_yn`(필수공정).

**제안 (권장 — 실무 의도 정합):**

1. **형상-사이즈는 `t_siz_sizes`에 그대로 유지.** `siz_nm`=형상+치수, EA=조각수. → 고객이 이 사이즈를 고르는 순간 칼틀이 확정된다(실무 의도 = 빠른 선택).
2. **커팅 공정은 상품에 "도무송" 1줄만 연결**(`t_prd_product_processes`, `mand_proc_yn=Y`). **구체 칼틀(모양)은 `prcs_dtl_opt`에 중복 입력하지 않는다** — `siz_cd`가 곧 칼틀 식별자이기 때문(실무 "커팅에 중복 안 넣음"과 정확히 일치).
3. **"사이즈 → 칼선" 연결은 앱 런타임으로 도출.** 고객이 사이즈를 고르면 그 사이즈에 매인 칼틀로 도무송 작업지시가 생성된다. DB는 사이즈↔공정의 룩업만 보유(앱 계산 철학 — 판걸이수·박 등급과 동일 원리).
4. **`round-10` "커팅옵션 클리어 37셀" = 정상으로 확정.** 합판도무송은 커팅 옵션그룹이 없는 게 **맞다**(칼틀이 사이즈에 1:1로 매여 있으므로). 이전 escalate는 본 모델로 닫힌다.

**다른 스티커와의 차이(상품군별 귀속이 다른 게 정상):**
- 반칼/완칼 **규격형** → 형상을 `prcs_dtl_opt.모양`에 두고 사이즈는 치수만(자유 형상, 칼틀 없음).
- 합판도무송 → 형상=사이즈(물리 칼틀 1:1). **"형상은 무조건 공정"이라는 일률 규칙이 틀렸음** — 칼틀 물리 존재 여부로 귀속이 갈린다.

**최종 권고:** 권장안(커팅 재분류) **철회**. **사이즈 유지 + 칼선=사이즈 종속 자동 도출**이 실무 의도·고객 UX·중복 0을 모두 만족한다. 남은 확인 = C3(합판도무송 가격이 사이즈로 결정되는지 가격표 대조) — 사이즈 기반이면 본 제안 완결.
