## 5. 돈 영향 정량

### 159 (봉투제작)
- 도달 행이 **기본(dflt) 모조봉투 가격** = 봉투 견적의 핵심 단가. 예: 225x193 1,000매 = 96,000 / 510x387 5,000매 = 672,000.
- **159를 단순 삭제(가격행 제거)하면**: 모조봉투 선택 시 COMP_ENV_MAKING no-match → 봉투 완제품가 0 합산 → **견적불가 또는 저청구(0원 봉투).** 돈크리티컬.
- **부활(현 단가 유지)하면**: 현행 청구 그대로 — 영향 0(현 라이브가 이미 이 단가로 청구 중).

### 119 (소량전단지)
- 119 선택 시 용지비 500/판 적용(전지 316x467 절가). off-grid 아님(고정 절가).
- **삭제하면**: 리브스디자인 선택 시 COMP_PAPER no-match → 용지비 미합산 → **저청구**(다른 합산요소만 남음). 단 dflt가 아니라 사용자가 굳이 119 선택해야 발현 → 노출 빈도 낮음.
- **명함/리플렛(31/48) 배선**: 공식이 COMP_PAPER를 안 읽으므로 가격영향 0 — 삭제·부활 어느 쪽이든 청구 불변(UI 옵션 잉여만).

**요지: 단순 "좀비행 정리=삭제"는 159에서 봉투 견적을 깨고(돈크리티컬), 119에서 전단지 용지옵션을 저청구로 만든다. 가격값 자체는 권위 검증 전까지 절대 건드리지 말 것.**

## 6. A/B/C 판정과 근거

### MAT_000159 — 시나리오 C(유효·고유) → 권고: 자재 del_yn 부활 또는 "삭제 유지+가격행 보존" 양자택일(CONFIRM)
- A(073 병합) 기각: 073은 COMP_ENV_MAKING 행 0건 → 이전할 정본 슬롯이 비어 사실상 신규생성. 게다가 073(용지절가용)과 159(봉투완제품가)는 **다른 가격 도메인** — 병합 부적절.
- B(부활) 유력: 159는 라이브 봉투의 기본 모조옵션·유일 가격원. 자재가 삭제(del=Y)인데 가격·배선은 살아 도달 = **자재 삭제가 오삭제(또는 IMPORT 잔재 root 157 평면화 미완)**일 가능성.
- 단, "159=모조 / 073=백색모조지"가 실무상 같은 종이를 두 이름으로 IMPORT한 결과라면, 장기적으로 **159 가격·배선을 073(또는 봉투 전용 모조자재)로 이관 후 159 폐기**가 정도일 수 있음 → CONFIRM-159.

### MAT_000119 — 시나리오 C(유효·고유) → 권고: 자재 del_yn 부활
- A 기각: 활성 리브스디자인 0건 → 병합 대상 부재.
- B 유력: 119는 유일한 리브스디자인 250g·소량전단지의 정상 용지옵션·고유 용지비(500) 보유. 자재만 삭제됐는데 옵션·가격이 살아있음 = **오삭제 정황.** 부활이 자연스러움.
- 명함/리플렛의 119 배선은 죽은 옵션(공식 미사용) → 부활 여부와 무관하게 **UI 정리(잉여 배선 제거) 별건 권고.**

## 8. CONFIRM (인간 확인 — 2건)

- **CONFIRM-159 (실무진/권위 엑셀):** "모조 120g(159)"과 "백색모조지 120g(073)"이 봉투용으로 동일 종이인가? 동일하면 장기적으로 159 가격·봉투배선을 073(혹은 봉투전용 모조자재)로 통합하고 159 폐기가 정도. 다르면(모조≠백색모조) 부활 확정. → 권위=상품마스터 자재시트의 봉투 소재 표기.
- **CONFIRM-119-root:** 119의 upr_mat_cd=NULL(고아). 리브스디자인을 활성 디자인지 root 아래로 재지정할지(자재 트리 정합) — 가격 무관·구조 정리 항목.
- (참고·범위 밖) GAP: PRD_000050 봉투의 169(레자크줄무늬)는 COMP_ENV_MAKING 행 0건 → 선택 시 봉투가 미합산. 별도 가격행 등록 필요(159 심의와 독립).
