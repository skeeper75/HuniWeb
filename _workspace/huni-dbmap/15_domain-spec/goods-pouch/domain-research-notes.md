## 1. 발견한 충돌 — 사이즈 재분류 + 가공 택일그룹 + 인쇄방식 PROC 트리 (굿즈 핵심 3건)

엑셀 실측 × 07_domain × round-10 변경추적 3자 대조.

### 1-1. 사이즈 컬럼 = 치수 OR 옵션 variant [핵심 재확인 — round-10]
- C5 사이즈(필수) **224 비빈값 중 202행(90%)이 비치수 옵션성** — 폰모델명(아이폰12~15·갤럭시S22~24 각 변형)·사이즈등급(M/L/XL/XXL)·방향(세로/가로형)·구수(2/3/4구)·면(단/양면)·형상(원형/사각). 진짜 치수(NxN)는 22행.
- **round-10 변경추적이 굿즈파우치 448셀을 `사이즈(필수)`→`상품(옵션)`으로 재분류**(224쌍·58상품, [[dbmap-change-tracking-round10]]). 즉 데이터모델 의도가 size→option으로 전환됨.
- **올바른 모델:** 치수형(22)→size(`t_siz_sizes.cut_*`)·옵션형(202)→CPQ option(`option_items` ref_dim_cd). 단 폰기종은 별도 옵션 차원(siz 아님)일 가능성·사이즈등급(M/L)은 비치수 siz vs option 양론. **기계적 size 삭제 금지**(적재된 size/price 사슬 파손, round-10 교훈). → **CONFIRM-GP-1.**

### 1-2. 굿즈 가공 택일그룹 미적재 (G-GP)
- 가공(C17)=라벨없음/라벨부착(파우치/에코백/필통)·에폭시(말랑)·맥세이프추가(폰케이스). UI상 상품별 택일.
- **라이브 excl_groups에 `GRP-BOOK-제본`·`GRP-CAL-가공` 2종뿐** — 굿즈/현수막 가공 택일그룹 부재(`db-domain-structure-live §330`). 실사 G-SL-3와 동일 구조 문제.
- **세 후보:** ① GRP-GP-가공 신규 excl_group 제안(ddl-proposer·캘린더 GRP-CAL-가공 동형) ② 상품별 1가공 고정(비택일) ③ round-6 옵션그룹. → **CONFIRM-GP-3.**

### 1-3. 인쇄방식 7종 → PROC 트리 매핑 [굿즈 고유]
- 폴더(C12)=패브릭인쇄(86)·UV인쇄(18)·전사인쇄(17)·이지굿즈(13)·디지털인쇄(7)·만년도장(7)·실사출력(2).
- **라이브 PROC 트리는 인쇄방식 코드그룹 미존재**(`db-domain-structure-live §204` — 인쇄방식은 PROC_000001 자식 트리로만 표현). UV(000007)·디지털(000003)·실사(000006)는 PROC 매핑되나, **이지굿즈(PVC고주파)·만년도장·전사·패브릭인쇄는 명시 PROC 코드 부재**(process-recipe §1 L2-4 외주 인쇄출력 표현 미정 🔴).
- **가설:** 패브릭인쇄=봉제미싱 라인(§9)·전사=외주(§13 코드005 쿠마샵)·이지굿즈=PVC고주파 신규 PROC·만년도장=고무도장 신규 PROC. → **CONFIRM-GP-7.**

**검증된 도메인 정합(07_domain 예측 ↔ 엑셀 실측 일치):**
- 파우치/에코백 봉제 = PROC_000080 봉제(유형/폭) + process-recipe §9 봉제미싱 ✅
- 말랑 에폭시 = PROC_000083 에폭시(b.12 굿즈파우치 전용) ✅
- 의류 전사 = process-recipe §13 전사 ✅
- 핀버튼 형상(원형/하트/정사각) = 규격 융합(wowpress 1행=1 sizeno) ✅
- 머그 본체색+용량 = 본체색 재질행·용량 규격(huni-goods §2.3) ✅
- 캔버스에코백 사이즈+방향 = huni-goods §2.5 ✅
- 본체색=재질행 합성 = wowpress 규칙B·huni-goods §2.1 ✅

> **포토북 G-PB-2·스티커 G-SK-2 같은 오모델은 굿즈파우치에 없음** — 후니/wowpress가 이미 본체색 합성·형상 융합으로 올바른 모델. 굿즈 고유 미확정=사이즈 재분류(GP-1·round-10)·가공 택일그룹(GP-3)·인쇄방식 PROC 트리(GP-7·외주 표현).

## 2. 🔴/🟡 컨펌 질문 (인간 결정 대기)

### CONFIRM-GP-1 [🟡·핵심] 사이즈 컬럼 귀속 — 폰기종/사이즈등급 = size vs option?
- **상황:** C5 224행 중 202 옵션성. round-10이 굿즈파우치 448셀을 size→option 재분류. 치수형 22행만 진짜 치수.
- **가설:** 치수형→size·옵션형→CPQ option. 폰기종(아이폰/갤럭시)=별도 옵션 차원(siz 아님). 사이즈등급(M/L/XL)=비치수 siz 또는 option.
- **질문:** 폰기종(아이폰15프로맥스 등)을 size로 둘까요 별도 옵션 차원으로 둘까요? 사이즈등급(M/L/XL)은 비치수 siz vs CPQ option 중 무엇? (round-10 재분류 정합 — 기계적 size 삭제는 가격사슬 파손)

### CONFIRM-GP-2 [🟡] 본체색 합성 입도 — "블랙 XL" 2축 분리 + 글리터/멜란지?
- **상황:** 선택옵션(C15)에 "블랙 XL"(색+사이즈 복합)·글리터(핑크/화이트/블루/블랙)·멜란지 XL 존재.
- **가설:** "블랙 XL"=재질(블랙 본체색)×규격(XL 사이즈등급) 2축. 글리터/멜란지=본체색(재질행 합성). OTC C-6 화이트M/L=한덩어리 규칙 정합.
- **질문:** "블랙 XL"을 본체색×규격 2축으로 분리할까요? 글리터/멜란지=본체색(재질행)으로 둘까요?

### CONFIRM-GP-3 [🟡·핵심] 굿즈 가공 택일그룹 — GRP-GP-가공 신규 vs 단순공정? (G-GP)
- **상황:** 가공(라벨없음/라벨부착·에폭시·맥세이프) UI 택일이나 라이브 excl_group 부재(GRP-BOOK/CAL만). 실사 G-SL-3 동형.
- **가설:** ① GRP-GP-가공 신규 excl_group(캘린더 GRP-CAL-가공 동형) ② 상품별 1가공 고정(비택일) ③ round-6 옵션그룹.
- **질문:** 굿즈 가공을 신규 택일그룹(GRP-GP-가공)으로 둘까요, 상품별 단순공정으로 둘까요? (실사 SL-3과 일괄)

### CONFIRM-GP-4 [🟡] 부속 귀속 — 볼체인·맥세이프·보냉내피·거울유리 = 자재 vs 공정 vs 추가상품?
- **상황:** 볼체인(키링 9색)·맥세이프 자석링(폰케이스)·보냉 내피(보냉백)·거울 유리(거울).
- **가설:** 볼체인=추가상품(별매 SKU 색 variant)·맥세이프=가공(부속+부착 공정)·보냉 내피=자재(부속)·거울 유리=자재(본체). booklet BK-3·calendar CL-5·실사 SL-4와 동류.
- **질문:** 볼체인을 추가상품(별매), 맥세이프를 가공(부착), 보냉/유리를 자재(부속)로 둘까요?
