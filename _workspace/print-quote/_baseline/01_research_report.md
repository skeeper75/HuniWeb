## 1. 제품 도메인

### 1.1 표준 제품 분류 (3-tier)

한국 인쇄업계는 일반적으로 **대분류 → 중분류 → 소분류** 3단계 트리를 사용한다. 카테고리는 평균 6~8개 대분류, 30~50개 중분류, 100~200개 소분류 규모.

| 대분류 | 중분류 | 소분류 예시 | 핵심 사양 변수 |
|--------|--------|-------------|----------------|
| 명함류 | 일반명함, 고급명함, 투명명함, 특수명함 | 표준 일반명함(90×50), 미니명함(86×52), 양각명함 | 용지, 코팅, 도수, 후가공 |
| 전단/리플렛 | 전단지, 리플렛, 팜플렛, 카탈로그 | A4 양면전단, 3단접지 리플렛 | 사이즈, 용지, 접지, 도수 |
| 봉투류 | 소봉투, 중봉투, 대봉투, 창봉투 | 110×220 양식봉투, 가로봉투 | 봉투규격, 창유무, 인쇄위치 |
| 배너/사인 | X배너, L배너, 현수막, 롤업 | 600×1800 X배너, 현수막 자유사이즈 | 사이즈(자유), 소재, 거치대 |
| 스티커/라벨 | 일반스티커, 투명스티커, 칼선스티커, 도무송 | 원형 50mm, 사각 80×80, 자유형 칼선 | 칼선형태, 소재, 점착력 |
| 책자/제본 | 중철, 무선제본, 링제본, 양장제본 | 16P 중철 카탈로그, 100P 무선제본 | 페이지수, 표지/내지 분리, 제본방식 |
| 포스터/달력 | 포스터, 벽걸이달력, 탁상달력 | A2 포스터, 7매 탁상달력 | 사이즈, 페이지수, 거치형식 |
| 굿즈/패키지 | 박스, 쇼핑백, 종이컵 슬리브 | 딱지박스, 끈손잡이 쇼핑백 | 전개도, 사이즈, 후가공 |

### 1.2 사양(Specification) 유형과 선택지

| 사양 카테고리 | 선택지 | 특성 / 가격 영향 |
|--------------|--------|-----------------|
| **사이즈** | A4(210×297), A5, A6, B4, B5, 명함규격(90×50/86×54), 자유사이즈(mm) | 자유사이즈는 면적 기반 가산. 규격품은 단가표 직매핑 |
| **용지(원지)** | 아트지(스노우/매트), 스노우화이트, 랑데뷰, 미색모조, 크라프트, 수입지, 합성지, 친환경지 | 평량(g/m²) 별 단가 차등: 150g/200g/250g/300g 등. 평균 5~8종 노출 |
| **코팅/표면처리** | 무코팅, 유광라미, 무광라미, 소프트라미, UV코팅, 부분UV, 형압, 박(금박/은박) | 단면/양면 옵션 분리. 부분코팅은 칼선 데이터 필요 |
| **인쇄도수** | 단면4도(CMYK), 양면4도, 단면1도(흑백), 양면1도, 별색(팬톤), 5도이상(별색추가) | 옵셋은 판비용 발생, 디지털은 도수 무관 |
| **수량** | 50/100/200/300/500/1000/2000/3000/5000/10000/20000+ | 수량 구간별 단가 급격히 하락(규모의 경제). 슬라이더+직접입력 병용 |
| **후가공** | 미싱(접지), 오시(접힘선), 도무송(칼선), 타공, 넘버링, 제본, 박, 형압, 귀도리(라운딩) | 옵션 다중 선택 가능, 일부는 상호 배타 |
| **납기** | 당일, 익일, D+2, D+3, 일반(D+5), 긴급(당일오후) | 납기 단축 = 할증률 적용 |

### 1.3 제품별 특이 사양

- **명함**: 규격형 우세(90×50, 86×54mm). 자유사이즈는 옵션. 양면 인쇄 기본. 미니/대형명함은 별도 제품.
- **현수막/배너**: 사이즈 자유 입력(mm). 면적(㎡) × 단가 방식. 거치대(X배너 거치대, 롤업 거치대) 옵션화.
- **스티커**: 칼선(다이컷) 데이터 별도 업로드. 자유형은 칼선 제작비 추가.
- **책자**: 표지(외지)/내지 사양 분리 입력. 페이지수는 4의 배수(중철) 또는 2의 배수(무선) 강제.
- **봉투**: 규격이 강한 도메인. 창봉투는 창사이즈/위치 별도 사양.
- **투명명함/투명스티커**: 코팅 옵션 무의미 → UI에서 비활성/숨김 처리 필요.
- **박/형압**: 추가 도판비(판비) 발생. 디자인 영역 제약(최소 폰트 크기 등) 검수 기준 별도.

### 1.4 SKU 폭발 문제

명함 1개 제품 = 용지 8종 × 코팅 5종 × 도수 4종 × 수량 10구간 = 1,600 SKU. 전 제품 통합 시 수십만 SKU. **사양 조합 기반 가상 SKU + 가격규칙 엔진**으로 운영하는 것이 표준.

## 2. 가격 도메인

### 2.1 가격 계산 공식 (한국 온라인 인쇄 표준)

```
최종금액(공급가) =
    [Σ (기본단가 × 수량)]                         # 본 인쇄비
  + [Σ (후가공 단가 × 수량 또는 고정비)]          # 후가공비
  + [셋업비/판비]                                  # 옵셋 한정
  + [할증액 (긴급납기/특수규격/주말작업 등)]
  - [할인액 (회원등급/쿠폰/대량주문/프로모션)]
  + [배송비]

부가세(VAT) = 공급가 × 0.10
최종 결제금액 = 공급가 + 부가세
```

### 2.2 수량 구간별 단가 예시 (일반명함, 양면4도, 아트지 250g, 무광코팅 기준)

| 수량 | 장당 단가 | 본 인쇄비 |
|------|-----------|-----------|
| 100매 | 80원 | 8,000원 |
| 200매 | 50원 | 10,000원 |
| 500매 | 30원 | 15,000원 |
| 1,000매 | 18원 | 18,000원 |
| 2,000매 | 12원 | 24,000원 |
| 5,000매 | 7원 | 35,000원 |
| 10,000매 | 5원 | 50,000원 |

특징:
- 구간 진입 시 **계단형(step) 단가**가 표준. 일부 업체는 보간(interpolation) 사용.
- 최저 수량(MOQ) 존재: 명함 100매, 책자 50권, 현수막 1장.
- 최대 수량 초과 시 별도 견적(B2B) 라우팅.

### 2.3 후가공 단가 구조

| 후가공 유형 | 단가 모델 | 예시 |
|-------------|-----------|------|
| 코팅(라미네이팅) | 장당 단가 | 무광 +2원/장, UV +5원/장 |
| 박(금/은박) | 면적당 + 판비 | 5,000원 판비 + 면적당 30원 |
| 형압(엠보싱) | 면적당 + 판비 | 7,000원 판비 + 면적당 25원 |
| 도무송(칼선) | 칼선 1개당 + 판비 | 칼판 30,000원 + 장당 10원 |
| 타공 | 타공 1개당 | 1구당 5원 |
| 미싱/오시 | 라인당 | 접지 1라인 +3원/장 |
| 제본 | 권당 고정 + 페이지수 가변 | 무선제본 권당 800원 |
| 넘버링 | 자릿수 × 수량 | 4자리 +10원/장 |

### 2.4 할증/할인 유형

**할증(Surcharge):**
- 긴급납기: 당일 +50%, 익일 +20%, D+2 +10%
- 특수사이즈(자유재단): +10~15%
- 토/일/공휴일 작업: +15%
- 별색 추가(팬톤): 별색 1도당 +10,000원 판비
- 소량 할증: MOQ 미만은 별도 가산
- 친환경/특수지: 자재비율 +30~100%

**할인(Discount):**
- 회원등급(VIP/실버/골드): 3~10%
- 누적 주문 할인(LTV 기반)
- 쿠폰: 정액/정률
- 대량 주문: 10,000매 초과 추가 5%
- 첫 구매 프로모션
- 동일 디자인 재주문 할인(re-order discount)

### 2.5 VAT 처리

- **공급가액 + 부가세 10% 별도** 표기가 표준 (B2B 거래 다수)
- 카드결제: 자동 부가세 합산
- 세금계산서/현금영수증 선택 가능
- 면세 품목은 사실상 없음(인쇄물은 과세)
- 가격표 노출 정책: 부가세 포함/별도 둘 다 운영. 메인 화면은 보통 "VAT 별도" 표기

### 2.6 가격 산출 방식 분류

1. **가격표 직매핑(Lookup Table)**: 명함, 전단 등 규격품. (제품, 사양조합, 수량) → 가격
2. **공식 기반(Formula)**: 현수막, 자유사이즈 — 면적/길이 기반
3. **하이브리드**: 기본가는 가격표 + 후가공/할증은 공식

설계 시 두 방식을 모두 지원하는 가격 엔진 추상화가 필요.

## 3. 주문 도메인

### 3.1 완전한 주문 상태 머신

```
[CART] 장바구니
  ↓ (견적 확정)
[DRAFT] 견적임시저장 ←─┐
  ↓ (재방문/공유)        │
[QUOTE_CONFIRMED] 견적확정
  ↓ (결제 시도)
[PAYMENT_PENDING] 결제대기 ──→ [PAYMENT_FAILED] 결제실패 ──→ (재시도)
  ↓ (결제 성공)
[PAYMENT_DONE] 결제완료
  ↓ (자동 진행)
[FILE_PENDING] 파일업로드대기 ──→ [ORDER_CANCELED] 취소(고객요청)
  ↓ (파일 업로드)
[FILE_REVIEW] 파일검수중
  ├──(반려)─→ [FILE_REJECTED] 파일반려 ──→ (재업로드) → FILE_REVIEW
  └──(승인)─→ [FILE_APPROVED] 파일승인
                ↓
[PREFLIGHT_DONE] 프리플라이트완료(CTP출력)
  ↓
[IN_PRODUCTION] 생산중
  ↓
[POST_PROCESSING] 후가공중
  ↓
[QC] 품질검사
  ├──(불량)─→ [REPRINT] 재인쇄 ──→ IN_PRODUCTION
  └──(통과)─→ [QC_DONE] 검사완료
                ↓
[PACKAGING] 패키징
  ↓
[SHIPPED] 출고(송장발행)
  ↓
[IN_TRANSIT] 배송중
  ↓
[DELIVERED] 배송완료
  ↓ (구매확정 또는 자동확정 7일)
[COMPLETED] 완료

특수 분기:
[ORDER_CANCELED] - PAYMENT_DONE 이전 자유 취소
[REFUND_REQUESTED] → [REFUND_APPROVED]/[REFUND_REJECTED] → [REFUNDED]
[ON_HOLD] - 자재 결품/장비 이슈로 보류 (어떤 상태에서도 진입 가능)
```

### 3.2 파일 업로드 및 검수 프로세스

**업로드 단계:**
1. 결제 완료 후 마이페이지 또는 이메일 링크로 업로드 페이지 진입
2. 지원 포맷: PDF(권장), AI, PSD, JPG/PNG(보조)
3. 파일 크기 제한: 일반적으로 200MB ~ 2GB
4. 다중 파일 업로드(앞면/뒷면, 표지/내지 분리)

**자동 검수(Preflight) 항목:**
- 해상도: 인쇄 해상도 300 DPI 이상
- 컬러모드: CMYK (RGB 입고 시 자동변환 또는 반려)
- 블리드(재단여백): 상하좌우 3mm
- 안전여백(safety margin): 재단선에서 추가 3mm 안쪽
- 폰트 임베딩 또는 아웃라인 변환 여부
- 사이즈 일치(주문 사이즈 vs 파일 사이즈)
- 별색(spot color) 처리 정의
- 오버프린트(overprint) 설정

**수동 검수(육안 검수):**
- 디자인 영역의 문제(이미지 깨짐, 누락 텍스트)
- 칼선/박/형압 데이터 별도 레이어 확인
- 검수 SLA: 평균 영업일 1~4시간 내 결과 통보

### 3.3 주문 취소/환불 정책

| 단계 | 취소 가능 | 환불 정책 |
|------|-----------|-----------|
| DRAFT/QUOTE | 즉시 가능 | 결제 전 |
| PAYMENT_DONE ~ FILE_PENDING | 자유 취소 | 100% 환불 |
| FILE_REVIEW ~ FILE_APPROVED | 가능(검수비 차감) | 5,000~10,000원 검수비 차감 |
| PREFLIGHT_DONE 이후 | 불가 또는 부분 | CTP/판비 차감, 본 인쇄 시작 후 환불 불가 |
| IN_PRODUCTION 이후 | 불가 | 환불 불가 (제작 손실) |
| 배송 후 불량 | 재인쇄 또는 환불 | 사진 증빙 + 7일 이내 신청 |

### 3.4 비회원 주문 지원

- 한국 시장 표준은 **비회원 주문 지원**(이메일 + 연락처 + 주문번호로 조회)
- 다만 파일 업로드 보안 및 재주문 편의로 회원 전환 유도 강함
- 비회원도 세금계산서 발행 가능(사업자번호 입력)
- 현실적 제약: 기업 고객 대량 주문은 회원/사업자 인증 필수가 많음

## 4. 위젯(UI) 도메인

### 4.1 인쇄 견적 계산기 표준 단계 구성

전형적 단계는 5~7 step. 좌측에 단계 위저드, 우측에 실시간 견적 패널 레이아웃이 표준.

```
[Step 1] 제품 선택        - 카테고리 트리/타일 그리드, 검색
[Step 2] 사이즈 선택      - 규격 라디오 + 자유사이즈 mm 입력
[Step 3] 용지 선택        - 카드형 라디오(이미지+이름+평량)
[Step 4] 인쇄 사양        - 도수, 코팅 (도수: 라디오, 코팅: 드롭다운)
[Step 5] 후가공 선택      - 다중 체크박스, 일부 옵션은 상세 입력
[Step 6] 수량 선택        - 슬라이더 + 직접입력 + 수량별 단가표
[Step 7] 납기/배송        - 캘린더, 배송지 입력
─────────────────────────────────────────
[우측 사이드바] 실시간 견적 패널
  - 제품 미리보기 썸네일
  - 사양 요약
  - 가격 분해 (본 인쇄비, 후가공, 할증, VAT)
  - "장바구니 담기" / "바로 결제"
```

### 4.2 단계별 입력 필드 유형

| Step | 필드 유형 | UI 컴포넌트 |
|------|-----------|-------------|
| 제품 | 카테고리 선택 | 타일 그리드, 트리 메뉴, 자동완성 검색 |
| 사이즈 | 단일선택 + 조건부 입력 | 라디오 그룹 + (자유사이즈) Number input(mm) ×2 |
| 용지 | 단일선택 | 이미지 카드 라디오, 평량 토글 |
| 코팅 | 단일선택 | 드롭다운 또는 라디오, 단/양면 별 |
| 도수 | 단일선택 | 라디오(단면4도/양면4도/...) |
| 후가공 | 다중선택 + 조건부 상세 | 체크박스 + 칼선 업로드, 박 위치 지정 |
| 수량 | 단일선택 또는 직접입력 | Slider + Number input + 단가표 툴팁 |
| 납기 | 단일선택 | 캘린더 또는 옵션 카드(가격 차이 표시) |

### 4.3 실시간 가격 업데이트 패턴

- **debounce 200~400ms**: 사용자 입력 후 디바운스 후 가격 재계산
- **Optimistic UI**: 일부 사양은 클라이언트 캐시 가격표로 즉시 표시 → 서버 최종 검증
- **계산 위치**:
  - 단순 룩업(명함 등): 클라이언트 측 가격표 다운로드 후 계산 (오프라인 가능)
  - 복잡 공식(현수막/책자): 서버 API 호출
- **표시 패턴**:
  - 가격 변동 시 숫자 카운팅 애니메이션
  - 직전 가격과 차이(+1,200원) 일시 표시
  - 가격 분해(breakdown) 토글 펼침

### 4.4 조건부 필드 표시 규칙(예시)

| 조건 | 결과 |
|------|------|
| 제품 = 투명명함 | 코팅 옵션 비활성/숨김 (이미 PET 자체 표면) |
| 제품 = 투명스티커 | "백색인쇄" 옵션 추가 노출 |
| 사이즈 = 자유사이즈 | 자유사이즈 할증 알림 표시, mm 입력 활성 |
| 후가공 ⊃ 박 | 박 색상/위치 입력 컴포넌트 노출 + 판비 알림 |
| 도수 = 별색 | 팬톤 코드 입력 필드 노출 |
| 제본 = 무선제본 | 페이지수 입력(2의 배수 강제), 표지/내지 분리 입력 |
| 책자 페이지수 변경 | 단가/납기 재계산, 표지 사양 별도 노출 |
| 수량 < MOQ | 가격 비활성, "최소 수량" 알림 표시 |
| 결제 직전 사이즈 변경 | "변경 시 가격이 재계산됩니다" 모달 |

이런 규칙은 위젯 도메인에 **조건부 노출 룰셋(Visibility Rules)**으로 별도 모델링 필요.

### 4.5 미리보기 이미지 처리 방식

3가지 단계로 운영:
1. **카탈로그 이미지(Catalog Mockup)**: 제품 선택 시 정적 이미지 — 가장 단순
2. **사양 반영 목업(Spec Mockup)**: 선택 사양에 따라 사전 렌더링된 이미지 매핑(예: 코팅별 광택 표현)
3. **사용자 디자인 미리보기(Live Preview)**: 파일 업로드 후 PDF 썸네일 + 재단선 오버레이
4. **3D 미리보기(고급)**: WebGL 기반 책자 펼침/명함 들기 등 (대형 업체)

미리보기는 결제 전 단계에서 핵심 전환율 요소. 자산 관리(asset management) 별도 도메인 필요.

## 5. 생산 도메인

### 5.1 인쇄 생산 단계

```
[1] 프리프레스(Prepress)
    - 파일 수신, 자동 검수 결과 반영
    - 색상 교정(컬러 프로파일 매칭, ICC)
    - 면 부착(임포지션, gang plate)
    - CTP(Computer-to-Plate) 판 출력 (옵셋 한정)
    - 디지털은 RIP만 처리

[2] 인쇄(Press)
    - 옵셋: 4도 인쇄기, 5도+ 별색기
    - 디지털: 토너/UV 잉크젯
    - 대형: 솔벤트/UV 평판/롤투롤(현수막)

[3] 후가공(Finishing)
    - 코팅(라미네이팅): 유광/무광/소프트
    - 박(Foil), 형압(Embossing/Debossing)
    - UV 코팅, 부분 UV
    - 도무송(Die-cutting): 칼판 제작 → 컷팅
    - 미싱, 오시, 타공, 넘버링
    - 제본: 중철, 무선, 링, 양장

[4] 재단(Cutting)
    - 정밀 재단(±1mm)
    - 자동 재단기(폴라 등) 사용

[5] 패키징(Packaging)
    - 묶음 단위(100매/500매) 포장
    - 박스 적재, 송장 부착

[6] 출고(Shipping)
    - 택배사 연계(CJ대한통운, 한진 등)
    - 퀵/자체배송(서울 권역)
    - 송장번호 ERP 자동 등록
```

### 5.2 납기 계산 방식 (영업일 기준)

```
주문일 D
  ├ 결제 + 파일 검수 완료시각 ≤ 14:00 → D 작업 시작
  └ 14:00 초과 → D+1 작업 시작 (마감 컷오프)

기본 생산 일수(영업일):
  - 명함: 1일
  - 전단지/리플렛: 1~2일
  - 책자(중철): 2~3일
  - 책자(무선제본): 3~5일
  - 현수막/배너: 1~2일
  - 박/형압 후가공: +1~2일
  - 도무송(칼판제작): +1일

배송 일수: D+1 (서울/경기), D+2 (지방), D+3 (제주/도서산간)

영업일 계산 시 제외:
  - 토요일, 일요일, 법정 공휴일
  - 명절 연휴(설/추석 ±2일)
  - 회사별 휴무일(여름휴가 등)
```

### 5.3 배치 인쇄(Gang Print) 개념

**Gang Run Printing**은 같은 용지/사양을 가진 여러 주문을 1장의 큰 판에 모아 동시 인쇄하는 방식.

- 비용 절감 핵심 (옵셋 판비 / 셋업 시간 분할)
- "당일 마감 컷오프"가 존재하는 이유 = gang plate 묶음 시각
- 단점: 색상 미세 차이 발생 가능, 개별 색맞춤 불가
- 시스템 영향:
  - 주문은 (용지 × 코팅 × 도수 × 기계) 단위로 묶임
  - **생산 작업(Production Job)** 엔티티가 여러 주문을 참조하는 N:M 관계 필요
  - 마감 시각 이후는 다음 배치로 자동 이월

### 5.4 품질 검사 체크포인트

| 단계 | 검사 항목 |
|------|-----------|
| 프리프레스 후 | CTP 판 정확도(레지스터마크), 임포지션 |
| 인쇄 직후 | 색상 일치(컬러 프루프 vs 인쇄물), 핀홀/줄무늬 |
| 후가공 후 | 코팅 균일도, 기포, 박/형압 위치 정확도 |
| 재단 후 | 사이즈 ±1mm, 직각도, 재단선 |
| 패키징 전 | 수량 정확도, 묶음 정확도, 오염/스크래치 |
| 출고 직전 | 송장 매칭, 박스 손상 |

불량 발견 시 → 재인쇄(Reprint) 분기. 재인쇄 비용은 내부 손실 처리.

### 5.5 자재 재고 관리 필요성

- 용지 재고: 평량/사이즈/색상 SKU 단위. 재고 부족 시 주문 받기 전 차단 또는 대체 제안
- 잉크/토너: 정기 발주
- 후가공 자재: 라미네이팅 필름, 박 호일 등
- 칼판: 주문별 1회성 제작. 반복 주문 시 재사용(고객별 보관)
- **자재 부족 → 납기 지연**의 주된 원인
- 시스템 측면: ERP/재고 API 연계, 주문 시점 가용 재고 검증, 안전 재고 알림

## 6. 스키마 설계 시 고려사항

### 6.1 도메인 간 의존성 맵

```
                    [Product]
                       │
                       ├── [Specification] (제품별 가능 사양)
                       │
                       ▼
         [Pricing] ←──── [Quote/Cart]
            │              │
            │              ▼
            │           [Order]
            │              │
            │              ├── [File] (검수 데이터)
            │              ├── [Payment]
            │              ├── [Shipping]
            │              └── [Production Job] ←─── [Material/Inventory]
            │                       │                       
            │                       └─── [QC Record]
            │
            └────── [Discount/Surcharge Rules]

[Widget Config] ──→ 제품/사양/가격 메타데이터를 읽어 UI 렌더링
                    조건부 노출 룰셋(Visibility Rules) 별도 보관
```

핵심 규칙:
- Product는 Spec/Price의 정의 마스터, Order는 그 시점의 스냅샷
- Production Job은 Order N:M (gang printing)
- Widget은 읽기 전용 소비자 (마스터 데이터 변경 영향)

### 6.2 스냅샷 저장이 필요한 데이터 (불변성)

주문 시점에 **반드시 스냅샷으로 복사**해야 하는 데이터:

| 데이터 | 이유 |
|--------|------|
| 제품명/사양 라벨 | 마스터 변경 시 과거 주문 표시 깨짐 |
| 단가 / 가격표 | 가격 정책 변경 후에도 영수증 재발행 가능해야 함 |
| 적용 할증/할인율 | 정산/감사 추적 |
| VAT율 | 세율 변경 가능성 |
| 주소/연락처 | 고객 정보 수정 후에도 송장 무결 |
| 세금계산서 사업자정보 | 회계 감사 |
| 납기 계산 근거 (영업일/공휴일 캘린더) | 분쟁 시 증빙 |

설계 패턴: `order_items.spec_snapshot JSONB`, `order_items.price_breakdown JSONB`, `orders.billing_snapshot JSONB`.

### 6.3 성능 이슈가 예상되는 쿼리 패턴

| 쿼리 패턴 | 위험도 | 대응 |
|-----------|--------|------|
| 사양조합 → 가격 룩업 (위젯 실시간) | 높음 | 가격표 캐시(Redis), 클라이언트 가격표 prefetch |
| 제품 카탈로그 트리 + 인기/추천 | 중 | 카테고리 트리 캐시, 머티리얼라이즈드 뷰 |
| Gang Print 묶음 후보 조회 (생산) | 높음 | (용지×코팅×도수×마감일) 복합 인덱스, 파티셔닝 |
| 주문 상태별 대시보드 (관리자) | 중 | status 인덱스, 최근 30일 파티션 |
| 파일 업로드/검수 큐 | 중 | status='FILE_REVIEW' 부분 인덱스 |
| 재고 부족 사전 검증 (모든 주문 진입 시) | 높음 | 재고 캐시, 감산 비동기 + 사전체크 |
| 영업일 납기 계산 | 낮음 | 공휴일 테이블 캐시, 함수화 |
| 가격표 변경 영향 시뮬레이션 | 낮음(배치) | 별도 분석 DB |

### 6.4 확장성 고려사항

**1) 새 제품 추가**
- 제품 = (분류, 사양 스키마, 가격 규칙, 위젯 단계 정의) 4쌍 추가만으로 추가 가능해야
- 사양은 정형(`paper_id`, `coating_id`)과 비정형(`extra_options JSONB`) 혼합
- 가격 규칙 엔진은 lookup-table / formula / hybrid 모두 지원

**2) 국제화(i18n)**
- 다국어: 제품명/설명/사양 라벨/위젯 문구 — `translations` 테이블 분리 또는 JSONB
- 통화: 기본 KRW. USD/JPY 추가 시 환율 스냅샷 저장
- 사이즈 단위: mm/inch 토글
- 세제: 한국 VAT 10% / 일본 소비세 / 미국 sales tax(주별) — 세금 규칙 별도 도메인
- 주소 포맷: 국가별 다르므로 `address JSONB` 권장

**3) 멀티 공장/멀티 채널**
- 같은 제품을 여러 인쇄소가 생산 → Production capability 별도 매핑
- B2C 자동견적 + B2B 수동견적 + 화이트라벨 채널 — `channel_id` 분리

**4) 가격 변경 이력**
- 가격표는 시계열(`effective_from`, `effective_to`) 관리
- 과거 주문 재계산 가능해야 함

**5) 위젯 변경**
- UI 단계/노출 규칙은 코드가 아닌 데이터로 관리 → A/B 테스트 가능
- `widget_config` 버저닝, 단계별 필드 정의, 조건부 표시 룰을 JSON 스키마로

### 6.5 핵심 아키텍처 권장사항

1. **가격은 엔진으로 분리**: PricingEngine은 (Product, Specs, Quantity, Date, Customer) → PriceBreakdown 함수형 모듈
2. **주문은 이벤트 소싱 친화적으로**: 상태 머신이 복잡 → state transition을 audit log에 기록
3. **사양은 스키마 + 값 분리**: 제품마다 사양 스키마 다름 → JSON Schema 정의 + 값 검증
4. **파일/생산/QC는 Order의 1:N 자식**: 같은 Order가 여러 production job에 묶일 수 있어 N:M 중간 테이블
5. **위젯 메타데이터는 Product의 'view'**: 같은 마스터를 다양한 UI로 노출
