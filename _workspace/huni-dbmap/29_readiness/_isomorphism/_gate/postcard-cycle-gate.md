## V3 — 가격 골든 C1=44,330 / C2=141,650 (REFUTED)

**라이브 단가행은 재현됨** (보정 하드코딩 없음 확인):
- `COMP_PRINT_DIGITAL_S1` SIZ_000499(국4절)·CLR_000005(4도)·단면: min_qty=100→**350**, min_qty=500→**190** (실측).
- `COMP_PAPER` SIZ_000499·MAT_000082(아트지300)→**93.30** (실측).
- 생성자 산식대로면 C1 = 350×100 + 93.30×100 = 44,330 / C2 = 190×500 + 93.30×500 = 141,650 — 산수는 맞음.

**그러나 골든이 순환(circular)이고 공식을 위반한다** — 이것이 REFUTED 사유:

1. **라이브 min_qty 축 = 출력매수, 주문수량 아님.** `COMP_PRINT_DIGITAL_S1` 단가행 note 실측:
   ```
   디지털인쇄 출력비(국4절)/칼라(CMYK)/단면 출력매수≥100 ...
   ```
   = min_qty가 **출력매수(판 임포지션 수)** 임을 명시. 가격표 `디지털인쇄비` 시트도 행축이 출력매수(칼라단면 출력매수=100→350·90→350·15→900·10→1000).

2. **문서화된 공식을 건너뜀.** `계산공식집초안` R5 `출력매수 = 주문수량 / 판걸이수`, R6 `인쇄비 = [수량행단가] × 출력매수`. `판걸이수`(100×150mm)=**8**(가격표 판걸이수 시트 R4 실측). 주문 100매면 출력매수 = ceil(100/8) = **13**, 수량행단가는 출력매수=13행(≈900~1000원), 인쇄비 ≈ 900×13 ≈ **11,700원** — 생성자의 35,000원과 **약 3배 차이**.
   - 생성자는 "주문 100매"의 100을 (a) 수량행 lookup 키 AND (b) ×출력매수 곱수 양쪽에 동시 사용 = 판걸이수 변환 단계를 통째로 생략. 35,000원은 출력매수=100(주문≈800매)일 때만 나오는 값.

3. **골든이 가격표 추적이 아님.** 생성자 표는 "골든(가격표)"라 라벨했으나, 그 골든값(44,330/141,650)은 round-18 `recompute-cases.md`의 손계산(라이브行×주문수량)을 그대로 운반한 것 = **검증값과 골든값이 같은 라이브行에서 파생** = 순환. 가격표에서 주문 100매 엽서의 최종가를 독립 추적한 흔적 없음.

> **결론 REFUTED.** C1/C2는 (a) 순환 골든(live==golden), (b) 판걸이수/출력매수 변환 누락으로 공식 위반. 따라서 "기본·도수 견적가능 GO·보정0"의 가격 근거가 무너진다. **단** 이 결함은 round-18/19 전 사슬이 공유한 것이고(생성자 단독 날조 아님), 라이브 단가행 자체는 정확. 라우팅 = `dbm-price-engine-verify`/엔진 명세(출력매수 변환 단계 명문화) + 생성자 재판정.
> **주의:** "수량(국4절)" 헤더 vs note "출력매수" 라벨 충돌은 도메인 컨펌 여지가 있으나, note·공식·판걸이수 시트 3중 증거가 출력매수 해석을 가리킴 → 입증책임은 "min_qty=주문수량"을 주장하는 쪽에 있으며 생성자는 그 입증을 안 함.

## 게이트 종합

생성자의 자율 파이프라인은 동형 분류(V1)·대표 선정(V2)·라이브 행수(V4)·D-1b(V5)에서 **정직하고 정확**하다. 그러나 가격 정합(V3)에서 순환 골든 + 공식 변환 누락으로 **"기본 견적가능 GO"가 거짓 GO**다. 5레이어 중 ①②(BOM·CPQ)는 GO, ⑤가격은 D-1b 외에 **출력매수 변환 결함이 추가로 존재**하므로 생성자의 "기본·도수 케이스 GO·후가공만 막힘" 보다 가격 미달 범위가 넓다.

## V3 재판정 = **REPRODUCED** (1차 REFUTED → 보정 후 재현됨)

### (a) 출력매수 변환 재실측 — 재현됨
- **판걸이수=8**: 가격표 `판걸이수` 시트 R4 C6=**8.0** 실측(C1=100×150mm·C5=엽서). 실제 셀 확정. (생성자 인용 73×98=18·98×98=12·135×135=6·148×210=4도 R2/R3/R5/R8 실측 일치.)
- **min_qty 축 = 출력매수**: 라이브 `COMP_PRINT_DIGITAL_S1` note "디지털인쇄 출력비(국4절)/칼라(CMYK)/단면 **출력매수≥N**" 재확인(1차 실측과 동일).
- **출력매수별 단가**: 라이브 component_prices(siz499·CLR_000005) min_qty 10→**1000**·35→**700**·60→**500**·100→**350**·500→190 실측 = 생성자 인용 일치.
- **산수 독립 검산**(검증자 자체 계산, ceil + step-down 룩업 + 손지율 +5):

  | 케이스 | 주문 | 출력매수 ceil | 단가행 | 인쇄비 | 용지비 93.3×(출+5) | 합계 | 생성자값 |
  |------|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
  | C1 | 100 | 13 | ≥10=1000 | 13,000 | ×18=1,679 | **14,679** | 14,679 ✅ |
  | C5 | 300 | 38 | ≥35=700 | 26,600 | ×43=4,012 | **30,612** | 30,612 ✅ |
  | C2 | 500 | 63 | ≥60=500 | 31,500 | ×68=6,344 | **37,844** | 37,844 ✅ |
  | C6 | 1000 | 125 | ≥100=350 | 43,750 | ×130=12,129 | **55,879** | 55,879 ✅ |

  → 전건 일치. 생성자가 제시한 14,679/37,844도 검증자 독립 산수와 동일. **출력매수 변환 정확 적용**.

### (b) 골든 독립성(순환 해소) — 재현됨
- **인쇄단가 출처**: 가격표 `디지털인쇄비` 칼라단면(C4) 출력매수→단가(10→1000·35→700·60→500·100→350) = **엑셀 파일**. 라이브 `COMP_PRINT_DIGITAL_S1` 단가행 = **DB**. **두 다른 출처가 동일 출력매수에서 동일 값** = 교차 일치.
- **용지단가 출처**: 가격표 `출력소재(IMPORT)` R13 아트지300g C9(가격 국4절)=**93.3** = 엑셀 셀. 라이브 `COMP_PAPER` MAT_000082=**93.30** = DB. 다른 출처 일치.
- → 1차의 순환(live==golden, 같은 行 자기참조)이 **해소됨**. 검증 = (공식 정합) + (excel↔live 교차 일치), 자기참조 아님.

### (c) 최종가 골든 정직 부재 — 타당
- 가격표 `디지털인쇄비` 시트 전수 스캔: "최종가/판매가/엽서가격" 류 행 **부재**(component 단가표만 존재) — 실측 확정.
- 원자합산형(build-up)은 가격표가 component 단가만 제공·최종가 known값 미보유 → "최종가 골든 부재로 인간 승인 큐 분리"는 **못 찾은 게 아니라 실제 없음**. 정직 분리 타당.

### (d) 보정 하드코딩 0 / 후가공 분리 — 재현됨
- 후가공 축 정정 재확인: 라이브 `COMP_PP_CREASE_1L` note "오시/1줄 **제작수량≥N** (합가)" 실측 → 후가공=제작수량축(주문수량)·합가, 인쇄/용지=출력매수축. 생성자 §5 "축이 갈린다(혼동 금지)" 정정 **정확**(계산공식집초안 R10 `후가공비=[제작수량행]`과 정합).
- 후가공(C3/C4)은 D-1b로 "계산불가" 분리, 임의 0원 대체 0. 라이브에 없는 매핑을 코드로 메운 흔적 0(인쇄·용지·후가공 단가 전부 라이브 실재 行 인용).

## 새로 발견한 결함 (재게이트)

| # | 종류 | 내용 | 심각도 |
|---|------|------|:--:|
| (없음) | — | C1/C5/C2/C6 산수·판걸이수·교차 일치·후가공 축·최종가 부재 전건 정확 | — |
| N-1 | 잔존 미검증(정직 표기됨) | C-DUP(양면)·C-COAT(코팅)은 "구조 정합"만(수치 골든 미제시) — 생성자가 ✅(구조)로 정직 표기. 별색 4상품(019~022)·코팅 수치 케이스는 Sp 단계 과제로 분리 | INFO(거짓 아님) |

> N-1은 결함이 아니라 정직 표기 확인 — 생성자가 양면/코팅을 "수치 일치"라 과장하지 않고 "구조 정합 ✅(구조)"로 한정했다. 거짓 GO 아님.

## 전체 게이트 재판정 = **CONDITIONAL-GO** (보정 정직·V3 해소)

- V1·V2·V4·V5 carry-forward GO + **V3 REPRODUCED**(보정 후) → 1차 BLOCKER 해소.
- 생성자 자기판정 **"기본·도수·코팅 GO(교차검증) + 최종가 골든 부재(분리) + 후가공 D-1b 잔존" = PARTIAL-REPRODUCED**에 **동의**. 거짓 GO 잔존 없음:
  - 기본·수량구간 견적(C1/C2/C5/C6) = 출력매수 변환 적용·excel↔live 교차 일치로 **실증 GO**.
  - 도수(양면)·코팅 = 구조 정합만(수치 미제시) — 정직하게 "구조 ✅"로 한정, GO 과장 아님.
  - 후가공 = D-1b 계산불가로 분리(임의값 0).
  - 최종가 골든 = 가격표 실제 부재로 인간 승인 큐 분리(정직).
- **조건부 GO 사유**: 견적 사슬은 검증됐으나 ① D-1b prc_typ 정정(후가공) ② 최종가 known값 도메인 컨펌 ③ 019~022 CPQ 적재(Sp) 가 인간 승인 큐로 잔존. 실 COMMIT 금지 원칙 유지.

## 인간 승인 큐 정직성 = **정직**
- D-1b 정정(prc_typ .03), 최종가 골든 부재(가격표 실측 확인), 019~022 CPQ 적재 — 셋 다 검증자 재실측으로 실재 확인된 미해소 항. 자율 판정을 인간 승인 큐로 정직 수렴(자율 실패 위장 아님).

**재게이트 종합:** F-1 BLOCKER는 보정으로 해소됐고, 보정값은 1차 권위에서 독립 재현된다(산수 검산·다른 출처 교차 일치·최종가 부재 실측). 전체 = **CONDITIONAL-GO**. 거짓 GO 잔존 없음.
