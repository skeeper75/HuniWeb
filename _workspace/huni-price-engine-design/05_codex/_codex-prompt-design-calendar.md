# codex 독립 2차 교차검증 프롬프트 — 디자인캘린더(가격포함) 가격엔진 설계 (Phase 5.5·11번째·최종 종단)

> ★우리(Claude) 게이트 판정·E1~E7 결론 일절 비노출. codex는 설계·지도·골든·권위·엔진계약만 보고 자기 독립 판정을 낸다.

## 프롬프트 (codex exec stdin)

당신은 인쇄 자동견적 가격엔진 설계의 독립 외부 심사관입니다. 한 한국 인쇄회사(후니프린팅)의 "디자인캘린더(가격포함)" 가격계산 엔진 설계를 받았습니다. 이 설계가 실제로 가격계산을 성립시키는지 독립적으로 판정하세요. 다른 사람의 판정은 받지 않았습니다 — 당신의 독립 의견이 필요합니다. 가격은 돈 크리티컬입니다. 환각 금지 — 데이터에 없으면 "데이터 부족"이라고 하세요.

"디자인캘린더(가격포함)"는 후니가 디자인을 제공하는 편집기형 완제 캘린더 5종(탁상형/미니탁상형/엽서/벽걸이/와이드벽걸이)이며, 일반 "캘린더" 시트와 **동일 5 prd_cd**(라이브 실측·신규 상품군 아님)이되 가격을 1부 정찰가(inline)로 표기한 변종입니다.

### 배경 (엔진 계약 = 라이브 pricing.py, 고정·변경 불가)
가격엔진 evaluate_price는 가격 소스를 우선순위로 선택합니다:
1. TEMPLATE_PRICE: 타깃이 template이고 unit_price 존재 → unit_price × qty
2. PRODUCT_PRICE: t_prd_product_prices.unit_price 존재 → unit_price × qty (FORMULA보다 선점)
3. FORMULA: 위가 없으면 상품-공식 바인딩으로 공식 평가(구성요소 합산 Σ)
4. NONE: 전부 없으면 0원(strict는 error)
- 엔진은 frm_typ_cd(공식유형)를 참조하지 않음 — 공식은 항상 구성요소 합산(addtn_yn='Y' 항의 Σ).
- 구성요소 소계: prc_typ가 단가형(.01)이면 `unit_price × qty`. 합가형(.02)이면 `unit_price ÷ tier_min_qty × qty`. min_qty는 단가형일 때 수량구간 tier 룩업 키(분모 아님).
- ★라이브 PRICE_TYPE enum = `.01(단가형)·.02(합가형)` 둘뿐이고 **`.03(고정가형) 그릇 부존재**. 따라서 "qty 무관 단일 정찰가"는 `.01 단가형 + min_qty=1`(qty=1 tier 1행)로 표현해야 합니다. qty 무관 단일 정찰가를 .01 단가형 min_qty=1로 담으면 결과가 정찰가 그대로인지 당신이 판정하세요.
- 차원 매칭: siz_cd·opt_cd 등 비수량 차원은 정확매칭. 한 구성요소 안에서 후보가 2개 이상이면 ERR_AMBIGUOUS. 단가행 차원값 NULL이면 와일드카드(아무 값에나 매칭). 서로 다른 comp_cd는 각자 독립 매칭·합산.

### 후니 라이브 현황 (읽기전용 실측 2026-06-22)
- 디자인캘린더 5 prd_cd = PRD_000108(탁상형)/109(미니탁상형)/110(엽서)/111(벽걸이)/112(와이드벽걸이). 전부 use_yn=Y·del_yn=N·editor_yn: 108/109/111/112=Y(편집기)·110(엽서)=N.
- t_prd_product_prices(108~112): **0행**(전체 테이블도 0행). 정찰가를 담을 그릇조차 비어 있음.
- 공식 바인딩(t_prd_product_price_formulas, 108~112): **0건**. 디자인 정찰가도, 일반 캘린더 산식도 둘 다 미배선(견적 불가·source=NONE).
- 일반 캘린더 전용 공식(이름에 캘린더/달력 포함된 frm_cd): 라이브 **부존재**(미적재). 즉 디자인 정찰가 공식도 일반 캘린더 산식 공식도 현 라이브엔 둘 다 없음.
- 재사용 가능 실재 구성요소: 인쇄비 COMP_PRINT_DIGITAL_S1(212 단가행)·용지비 COMP_PAPER(56)·제본비 COMP_BIND_CAL_DESK220/DESKMINI/WALL 등(단가행 실재). ★단 디자인캘린더 정찰가가 이들 단가행 합산으로 재현 안 됨(아래 권위/역산 참고).
- 우드거치대용 구성요소 COMP_CALOPT_STAND: 라이브 component·단가행 **0행**(부존재) — 별도 "일반 캘린더" 설계가 신규 mint하기로 한 미적재 그릇.
- 캘린더봉투: 독립 상품 PRD_000005(MES 012-0008)로 라이브 실재.
- option_groups(108~112): 0행(옵션 레이어 전면 미적재).

### 핵심 권위 사실 — inline 정찰가가 단가행 산식으로 재현되는가 (독립 역산 입력 verbatim)
설계는 inline을 "인쇄비+용지비+제본비 단가행 합산"으로 재구성 시도했고 재현 실패라 결론냈습니다. 당신이 직접 검산하세요. 산식 = `유효판수 = (inline − 제본비) / (인쇄단가 + 용지단가)`. 라이브 단가 verbatim:
- 디지털인쇄 양면 q1 = 4,000/판 · 단면 q1 = 3,000/판
- 용지(몽블랑190g 국4절) = 112.58/판
- 제본 탁상220 = 5,000/부 · 미니 = 4,500/부 · 벽걸이/와이드 = 5,000/부 · 엽서 = 제본 없음
- 페이지: 탁상=30P · 미니=26P · 엽서=12P · 벽걸이/와이드=13P

| 상품 | inline | 제본 | 잔여(인쇄+용지) | per_plate(인쇄+용지) |
|------|-------:|-----:|---------------:|---------------------:|
| 탁상220 양면 | 10,400 | 5,000 | 5,400 | 4,112.58 |
| 미니 양면 | 6,500 | 4,500 | 2,000 | 4,112.58 |
| 엽서 단면 | 4,000 | 0 | 4,000 | 3,112.58 |
| 벽걸이 단면 | 9,900 | 5,000 | 4,900 | 3,112.58 |
| 와이드 단면 | 24,000 | 5,000 | 19,000 | 3,112.58 |

각 행의 유효판수를 직접 나눠 계산하고, (a) 정수인가 (b) 페이지수와 정수 배수 관계가 있는가를 판정하세요. (비교 참조: 다른 "포토북" 시트는 row17에 `상품단가+페이지당단가적용` 명문 산식이 있고 per2p가 imposition 비용추세(4-up=300<2-up=500/600<1-up=1000 단조)를 따라 공식화에 성공했습니다. 디자인캘린더에는 그런 명문 산식이 없습니다.)

### 설계 요지 (판정 대상)
- 계산방식 = **정찰가 스냅샷 [BLOCKED·공식화 불가]**. inline이 단가행 산식으로 정수·일관 재현 안 됨 → "산식으로 inline을 맞추는 골든"은 존재하지 않고, 추측 단가 INSERT 금지(BLOCKED 정직).
- 적재 경로(인간 컨펌 후): inline 정찰가를 **고정가형 완제 SKU**로 채택. 동일 prd_cd가 일반 캘린더 산식(PRF_CAL_*·업로드 주문)과 디자인 정찰가(PRF_DCAL_*·편집기 주문) 두 가격을 가질 수 있으므로, **주문방법(편집기 vs 업로드)을 판별 차원으로 별 공식 분기**(같은 prd 동시 바인딩 금지·비결정 회피). 별 prd 분리는 부결(라이브 동일 prd_cd 역행).
- 본체 정찰가 그릇: 신규 comp COMP_DCAL_FIXED · prc_typ **.01 단가형 + min_qty=1**(라이브 .03 부재) · use_dims=[siz_cd] · addtn_yn='Y'. 단가행 7행 = siz_cd별 inline verbatim(220x145→10400 등). 디자인비는 별 comp 분리 금지(정찰가에 baked).
- 신규 공식 5개(상품×사이즈 정찰가가 달라 동시매칭 회피 위해 상품별 전용 공식): PRF_DCAL_DESK / DESKMINI / POSTCARD / WALL / WALLWIDE. 부모 5상품 → 각 PRF_DCAL_* 바인딩(FORMULA).
- add-on: 엽서캘린더 우드거치대(4,000)는 별 comp COMP_CALOPT_STAND(opt_cd 차원·.01 단가형·×qty)로 본체 공식에 가산 — 단 이 comp는 라이브 0행이라 "일반 캘린더" 설계의 신규 mint에 선행 의존(디자인캘린더 독자 mint 금지). 캘린더봉투(2500/2400)는 독립 PRD_000005 + 본체 addon 이중역할이며 **봉투제작 트랙에 위임**(본체 가격공식 외부·본체에 합산하면 오과금).
- ★G-PRODPRICE 가드: 본체 정찰가를 product_prices에 INSERT 금지(.01 단가형 comp로 formula 바인딩만). product_prices 1건이라도 있으면 엔진이 PRODUCT_PRICE를 선점해 FORMULA(우드거치대 add-on 가산)를 통째 우회 silent → 엽서캘린더+우드거치대가 본체값만 출력. 정찰가형이라도 add-on이 있으면 formula 합산 구조 유지.
- 신규 테이블/가격축 0. 신규 mint = 공식 5 + 정찰가 comp 1(COMP_DCAL_FIXED)뿐(우드거치대 comp는 일반 캘린더 소관). 전부 인간 컨펌 후.

### 골든 기대값 (설계가 재현해야 하는 권위값 — inline verbatim)
- 탁상형 220x145 = 10,400 / 130x220 = 9,700 (siz_cd 분기 입증·단일가로 뭉개면 700원 오차)
- 미니 90x100 = 6,500 / 148x60 = 6,500
- 엽서 145x145 = 4,000 (본체만)
- 벽걸이 210x297 = 9,900 / 와이드 300x625 = 24,000
- 엽서 + 우드거치대 qty=1 = **8,000** (본체 4,000 + 우드 4,000·formula 합산). product_prices에 본체를 박으면 add-on이 우회돼 4,000만 출력(가드 입증).
- 엽서 + 우드거치대 qty=10 = 44,000 (개당 가산 가설·본체 4,000 + 우드 4,000×10) — 단 개당 vs 주문당 정액은 상품마스터 명시 부족(컨펌 필요).

### 당신이 독립 판정할 열린 질문 (우리 결론 없음 — 당신의 답을 직접 산술로 주세요)
각 질문에 SOUND/RISK/FLAW 명확한 판정 + 근거. 데이터로 직접 계산하세요.

1. **★inline 정찰가 산식 재현 (핵심 directive)**: 위 7개 정찰가(탁상 10400/9700·미니 6500·엽서 4000·벽걸이 9900·와이드 24000)가 라이브 단가행(인쇄·용지·제본)의 산식으로 정수·일관 재현되는가? 각 행 유효판수를 직접 나눠 계산하고, 정수·페이지수 정수배수 여부를 보이세요. 결론: 이 시트는 (가) 단가행 합산형(→FORMULA로 공식화 가능) 인가 (나) 정찰가 스냅샷(→inline=권위·산식 골든 부재·추측 단가 INSERT 금지·BLOCKED) 인가? 포토북(per2p cost-driven 정수재현→공식화)과 대비해 판정하세요.

2. **★고정가형 그릇 매핑**: 라이브에 .03(고정가형) 그릇이 없을 때, qty 무관 단일 정찰가(예 탁상 10400)를 `.01 단가형 + min_qty=1` + use_dims=[siz_cd]로 담는 설계가 타당한가? 탁상형 1부와 10부 주문 시 가격이 각각 얼마로 나오는지 직접 계산해(정찰가는 qty 무관 단일가인지 ×qty인지 당신이 판정), 이 그릇이 정찰가를 정확히 표현하는지 보이세요. siz_cd 차원이 없으면(NULL 와일드카드) 탁상형 220과 130 두 정찰가가 한 공식 안에서 무슨 문제(ERR_AMBIGUOUS 등)를 일으키는가?

3. **★add-on 귀속 + G-PRODPRICE 가드**: (a) 우드거치대 4,000을 별 comp(.01·×qty)로 본체 공식에 가산하는 설계로 "엽서 + 우드거치대 1개" 가격을 직접 계산하세요(기대 8,000). (b) 만약 본체 정찰가 4,000을 product_prices에 INSERT하면 엔진 우선순위상 무슨 일이 생기고 그 주문 가격이 얼마가 되는가(add-on silent 우회)? (c) 캘린더봉투(2500/2400)를 본체 공식에 합산하지 않고 독립 PRD/봉투트랙으로 분리하는 게 옳은가, 아니면 본체 add-on으로 합산해야 하는가? 봉투가 사이즈별 변형가(2500 vs 2400)인데 단일값 평탄 적재 시 위험은?

4. **세트 불요 판정**: 디자인캘린더를 부품합산 세트(내지+표지+제본 등)로 분해하지 않고 정찰가 완제품 + 외부 add-on으로 두는 설계가 정당한가, 아니면 세트 분해가 필요한가? 페이지수(30/26/12/13)를 책자/포토북처럼 내지 부품 N장 곱으로 오적용하면 무슨 일이 생기는가?

5. **돈크리티컬 가드(놓친 것 포함)**: 다음이 실재 위험인가 과잉설계인가, 그리고 설계가 **놓친** 결함/엣지케이스(과청구 또는 저청구)가 있는가? (a) 동일 prd_cd가 PRF_CAL_*(산식)와 PRF_DCAL_*(정찰가) 둘 다 바인딩되면(주문방법 분기 없으면) 무슨 일? (b) 미니탁상형 두 사이즈 동일가(6500=6500)는 결함인가 정상인가? (c) 벽걸이/와이드의 트윈링제본·삼각대가 정찰가에 baked인데 별도 가공 comp로 또 가산하면? (d) inline을 비정수 역산해 산식 단가로 component_prices에 박으면? 그 외 당신이 보기에 설계가 빠뜨린 가격축/구성요소/이중계상이 있으면 지적하세요.

각 항목 SOUND/RISK/FLAW 명시. 근거는 위 데이터로 직접 산술. 환각 금지 — 데이터에 없으면 "데이터 부족"이라고 하세요. 우리 결론을 모방하지 말고 당신의 독립 계산으로 판정하세요.
