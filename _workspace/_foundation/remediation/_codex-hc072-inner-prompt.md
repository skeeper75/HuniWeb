# 독립 분석 요청 — 하드커버책자 내지 가격모델 판정 (읽기전용·2nd opinion)

당신은 인쇄 가격엔진 독립 검증자다. 아래 데이터만으로 **하드커버책자(072) 내지(PRD_000284) 디지털인쇄 페이지가**가 어떤 가격모델인지 판정하라. 우리 1차 판정을 미리 보지 말고 스스로 결론을 내라.

## 컨텍스트
- 후니 가격엔진 evaluate_price: 공식(formula) = Σ 가격구성요소(component). 단가형(.01)=unit_price × qty.
- 하드커버책자072 = 세트(부모) ← 표지(073)+내지(284)+면지(074/075/076). 표지+제본은 이미 라이브 적재됨(per-book 밴드룩업·사이즈독립·수량밴드 6단).
- 내지(284)는 가격공식 없음(0원). 내지종이=*별도설정(여러 종이 선택), 내지인쇄=단면, 페이지 24~300 증가2.

## 참조 동형: 포토북 내지(101) 모델 — 이번에 우리가 설계/COMMIT한 단순 단가형
- 포토북 내지 = "사이즈별 추가2P당 고정단가" (8x8=500/10x10=1000/A5=300/A4=600). prc_typ=.01(단가형).
- 위젯이 내지qty = 부수 × ⌈(page−24)/2⌉ 전달 → 추가2P당 × 내지qty = 페이지 선형증가.
- 포토북은 내지종이가 1종 고정(몽블랑130), 수량할인 없음(per-2page 고정), 표지타입 무관.

## 라이브 이전사이트 실측 — 하드커버072 내지(p01) 격자 (A4세로·백색모조지100g·읽기전용 gstack)
출력: total_inner_price (책 qty권 전체 내지가, 단위 원)
```

```

## 라이브 디지털인쇄 단가 구조(참조)
- 용지비 COMP_PAPER: 백색모조100g 국4절(316x467) 절가 = 30.73원/장. "실제 청구는 출력매수만큼 자동계산"(앱이 출력매수 곱).
- 디지털인쇄비 COMP_PRINT_DIGITAL_S1: 국4절 단면 판수밴드(min_qty=판수): 1판=4000, 2=2800, 3=2500, 5=1700, 10=1000, 30=750, 60=500, 100=350, 300=270 ... (판수 많을수록 단가↓). 인쇄비 = 밴드단가 × 판수.

## 판정 질문 (각각 근거와 함께)
1. **내지 per-book가 page와 qty 둘 다에 비선형 의존하는가?** page구간별 기울기(303 vs 357/page at q1), qty밴드별 기울기 변화를 보고 판단하라.
2. **포토북식 "사이즈별 추가2P당 고정단가(단가형 .01)"로 이 격자를 verbatim 재현 가능한가?** (수량할인이 내지에 들어있으면 고정단가형으로는 불가)
3. 불가라면, **디지털인쇄 합가형(용지비 절가×출력매수 + 인쇄비 판수밴드×판수, 출력매수=페이지×부수/임포지션 앱계산)**이 맞는 모델인가? 출력매수/판수 임포지션 규칙(A4 2up 국4절 등)을 추정해 격자 재현을 시도하라.
4. 만약 디지털 합가형이 맞다면, **포토북 동형 단순 단가형 설계는 부적절**이고 무엇이 추가로 필요한가(plt_siz 차원, 출력매수 앱계산 엔진로직, 판수밴드)?
5. 이 격자가 표지+제본처럼 **라이브가 묶음으로 노출한 밴드룩업(per-book 수량밴드)** 일 가능성 — page별로 per-book 6수량밴드 테이블을 그대로 단가행으로 적재하는 모델(use_dims=[page구간, min_qty])은 어떤가? 장단점은?

결론: 4가지 후보(A=포토북식 단순단가형 / B=디지털 합가형 재구성 / C=per-book 밴드룩업 테이블 직접적재 / D=데이터 불충분) 중 어느 것이 가장 타당한지, 그리고 verbatim 정확성을 위해 무엇이 더 필요한지 명시하라.
