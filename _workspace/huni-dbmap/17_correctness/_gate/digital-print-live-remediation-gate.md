## 3. 즉시적용 4건 승격 정당성 평결 [관건] — **정당(GO)**

**쟁점:** 원본 correction-manifest(C-09/C-14)는 카테고리 오연결을 🟡(중)·"컨펌 Q-ID-B"로 분류했는데, 새 worksheet는 043~046을 **즉시적용으로 승격**했다.

**승격이 정당한 이유:**
1. **원본도 "정상노드 재연결"을 권장했다** — product-identity L71 원문: "교정은 296 upr UPDATE(차선)가 **아니라**, 이미 정상인 273/274/275/283으로 상품을 **재연결**(search-before-mint)". Q-ID-B는 *두 방안 중 선택* 형식의 컨펌이었고, worksheet는 그 권장안을 채택. 권장안 채택 = 사람 결정 불요 수준.
2. **노드명-상품명 결정적 매칭** — 043→CAT_000273은 "인쇄배경지(OPP봉투타입)" **완전 동일 문자열**(044/045 동일). 046만 "라벨/택"→"라벨/포장스티커" 의미매칭이나 012 포장 하위에서 명백. 임의 추측 여지 0 → 즉시적용 자격 충족.
3. **search-before-mint 100% 준수** — 273/274/275/283 라이브 실재 + 비어있음(다른 상품 0). 신규 mint 0·hard-delete 0.
4. **멱등·비파괴 라이브 실증** — 2-pass DRY-RUN 통과·ROLLBACK 원상복구.
5. **분류 정밀성** — 041/042(상품권)는 MES 003·노드명 매칭 불가라 즉시적용에서 **정확히 제외**(컨펌대기 W-05 유지). 노드명 결정성이 있는 4건만 승격 = 경계가 옳다.

**결론: 즉시적용 4건 승격은 부당하지 않다 → K4 PASS 유지(컨펌대기 환원 권고 안 함).**

## 4. 뒤집힘 / 보정 / 누락 / MINOR

- **뒤집힘: 0** — 매니페스트 주장이 라이브 독립 재측정으로 뒤집힌 건 없음.
- **보정: 0** — 즉시적용분에 정합 결함 없음.
- **MINOR-1 (정보·non-blocking)** — apply.sql 1b UPDATE는 296행 main_cat_yn만 'N'으로 강등(del_yn 컬럼 부재로 완전해제 불가). 고아 296 노드 자체는 잔존 → 카탈로그 트리에 빈 고아 노드가 남음. apply.sql이 "완전 해제는 컨펌 후(Q-ID-B)"로 정직 표기 → 결함 아님·후속 정리 권고로 충분.
- **MINOR-2 (정보)** — rollback.sql 2번 UPDATE가 296행 note=NULL로 일괄 복원. 296행이 원래 note를 갖고 있었다면 손실 가능하나, 독립 측정상 296 연결행은 정정 대상 신규행이라 원래 note 無 → 실손실 0. apply→rollback 왕복 시 무해.

## 5. 쉬운 한국어 결론

- **즉시적용 4건(배경지OPP·케이스·헤더택·라벨택의 카테고리 재연결)은 바로 적용해도 안전합니다.** 라이브를 직접 다시 측정한 결과, 이 4상품이 정말로 "어디에도 안 붙은 떠 있는 칸(배경지 296)"에 묶여 있고, 진짜 자리(포장 밑 인쇄배경지/헤더택/라벨 칸)는 이미 만들어져 비어 있는 게 확인됐습니다. 게다가 그 진짜 칸의 이름이 상품 이름과 **글자 그대로 똑같아** 사람이 따로 판단할 게 없습니다.
- **새로 만드는 카테고리 0개·삭제 0개**입니다. 기존 빈 칸에 연결만 옮기고, 가짜 칸 연결은 끄기(논리정리)만 합니다. 두 번 실행해도 한 번과 결과가 같음(멱등)을 라이브에서 롤백 모드로 직접 확인했고, 실제 저장(COMMIT)은 하지 않았습니다.
- **상품권 2건은 즉시적용에서 정확히 빠졌습니다.** 상품권은 포장(012)이 아니라 인쇄홍보물(003)이라 어느 칸에 둘지가 아직 안 정해졌기 때문입니다 — 이건 사람 결정(컨펌)이 맞습니다.
- **나머지(봉투 세트·형상 커팅·박색·종이 누락)는 설계만 해두고 보류**한 것이 옳습니다. 형상값을 담을 칸이 DB에 아직 없거나, 세트로 묶을지 추가상품으로 묶을지 결정이 필요해서입니다. 억지로 엉뚱한 칸에 넣지 않았습니다.
