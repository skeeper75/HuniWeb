가격 정합 심의(dbm-price-arbiter). **라이브 읽기전용·DB 미적재·DRY-RUN까지**(실 COMMIT 인간 승인).
권위[HARD] = 인쇄상품 가격표 260527 명함포토카드 시트 B01(스탠다드명함). 라이브·v03·경쟁사는 검증 오라클이지 가격 권위 아님.

## 0. 결함 요지 (A2 C1 / D-C1-04)

`COMP_NAMECARD_STD_S1/S2`(PRF_NAMECARD_FIXED 바인딩)가 **동일가 자재군을 단일 mat_cd로 collapse**(각 comp 2행만)
→ 스탠다드명함이 제공하는 종이 5종 중 **3종(아트250·스노우250·스노우300) 선택 시 단가행 0건 = 견적불가**.
★적재된 단가값 자체(3500/3800/4500/4800)는 권위와 verbatim 정합 — 결함은 **자재 차원 collapse(필요한 mat_cd 단가행 누락)**.

## 2. 권위 대조 (가격표 260527 명함포토카드 B01 스탠다드명함)

추출 캐시 `06_extract/price-namecard-photocard-l1.csv` B01 블록 — **열 헤더가 종이를 묶는다**:

| 면 | 가격표 컬럼 헤더 | 단가(min_qty 100) |
|---|---|---|
| 단면 | 백모조220 / 아트250 / 스노우250 | 3500 |
| 단면 | 아트300 / 스노우300 | 3800 |
| 양면 | 백모조220 / 아트250 / 스노우250 | 4500 |
| 양면 | 아트300 / 스노우300 | 4800 |

**판정: 종이는 "부분적으로만" 가격축이다.** 5종 종이는 전부 같지도 전부 다르지도 않고 **2개 가격 그룹**으로 묶인다:
- **그룹1** (단면 3500 / 양면 4500): 백모조220 · 아트250 · 스노우250 (3종 동일가)
- **그룹2** (단면 3800 / 양면 4800): 아트300 · 스노우300 (2종 동일가)

교차 검증(같은 시트 다른 블록):
- **코팅명함 B03**: 종이가 개별 컬럼("아트250" 단독 / "아트300" 단독) → 단가 다름(5500/5800) → 라이브 COAT_S1도 mat_cd별 개별 전개(정합). ⇒ 가격표가 "묶으면 묶고, 안 묶으면 안 묶는다"는 의미축이 진짜임을 입증.
- **펄명함 B04 / 라이브 PEARL_S1**: "다이아240 / 실버240 / 골드240"이 한 mat_cd 행에 collapse → **동형 결함**(본 교정 스코프 밖, §6 전파 대상).

⇒ collapse 자체는 가격표 의도(같은 칸=동일가)지만, 라이브는 **묶인 그룹의 대표 mat_cd 1개만 적재하고 나머지 4종 mat_cd 행을 누락**한 것이 결함.

## 3. 근본원인 (root-cause)

webadmin `load_master`는 가격표 셀을 무변환 전파한다([[dbmap-axis-staged-load]]). 가격표 B01의 단면 그룹1 셀(3500)은 **"백모조220 / 아트250 / 스노우250"이라는 3-종이 묶음 헤더 1셀** → 적재 시 셀 1개당 단가행 1개 생성 + mat_cd는 헤더 토큰의 **첫 종이(백모조220=MAT_000074)만** 파싱·바인딩.
- 가격축 의미("이 단가는 3종 모두에 적용")가 mat_cd 단일값으로 좁혀짐 → 나머지 2종(아트250·스노우250)은 어떤 단가행도 가리키지 않게 됨.
- use_dims에 mat_cd가 있어 엔진은 mat_cd 정확매칭을 요구(`_row_matches`: 행 mat_cd가 NULL이 아니면 선택값과 일치해야 통과) → 누락 mat 선택 시 `no_match` → 견적불가.

**가설 반증**: "혹시 종이가 가격축이 아니라서 mat_cd=NULL로 둬야 했나?" → 반증: COAT_S1은 종이별로 단가가 다르고(B03 5500/5800) 라이브도 mat_cd별 전개되어 있음. 가격표가 명백히 종이를 가격축으로 다룸. mat_cd=NULL 완전 와일드카드는 그룹2 종이(3800)도 그룹1 행(3500)에 매칭시켜 저청구를 일으킴 → 부적격(§4).

## 4. 교정 구조 심의 — (a) NULL 와일드카드 vs (b) 종이별 전개

엔진 사실(`raw/webadmin/.../pricing.py`):
- `_row_matches` L83-88: **행 차원이 NULL이면 와일드카드(어떤 선택값이든 통과)**.
- `match_component` L137-142: 비수량 차원조합(`_combo_key`)이 2개 이상 동시 매칭되면 `ERR_AMBIGUOUS`(데이터 오류, 매칭 거부).

### 옵션 (a) — mat_cd=NULL 완전 와일드카드 (자재 무관·동일가일 때)
- 적용 조건: 5종 단가가 **전부 동일**일 때만 안전.
- ❌ **부적격**. 5종은 2개 가격 그룹(3500/3800)으로 갈린다. NULL 단일행이면:
  - 그룹2 종이(아트300/스노우300, 3800) 선택 시에도 NULL 행(3500)에 매칭 → **장당 300원 저청구**(돈 결함).
  - 또는 그룹1·그룹2 두 NULL 행 공존 시 양쪽 다 와일드카드로 동시 매칭 → `ERR_AMBIGUOUS` → 견적불가.
- → 동일가가 아니므로 NULL 와일드카드는 가격축을 뭉갠다.

### 옵션 (b) — 종이별 단가행 전개 (다른가일 때) ★권고
- 누락 4종의 mat_cd 행을 **각 그룹 단가로** 추가. 기존 2행은 verbatim 보존.
- 각 종이는 고유 mat_cd → 한 선택값당 정확히 1행 매칭(`_combo_key`가 mat_cd 포함). 동시매칭 0.
- 권위 그룹가(3500/3800·4500/4800)를 종이마다 그대로 복제 → 저청구·과청구 0, 권위 verbatim.

### 권고: **(b) 종이별 단가행 전개**
근거: 가격표가 종이를 가격축으로 취급(2그룹·동일가 아님)하고, 엔진 use_dims에 mat_cd가 이미 선언돼 있으며, 동일 시트 COAT가 이미 mat_cd별 전개 선례. (a)는 동일가 전제가 깨져 저청구/동시매칭을 유발하므로 배제.

> **하이브리드 대안(b')은 채택하지 않음**: "그룹1은 mat_cd=NULL(catch-all), 그룹2만 mat_cd 명시" — 이러면 그룹2 종이가 NULL행+명시행 양쪽 매칭 → ERR_AMBIGUOUS. NULL 와일드카드와 명시행 혼용은 동시매칭을 부른다. 전 종이 명시 전개(b)가 유일하게 동시매칭 0·저청구 0.

## 7. 미해소·컨펌 분리 / 동형 전파

- **[전파·스코프 밖]** PEARL_S1/S2(다이아240/실버240/골드240 collapse), 그리고 STD에 동일 패턴이 있을 다른 명함 comp는 본 교정 스코프 아님. 동형 결함이므로 별도 동형 전파 교정 필요(권위 = 각 명함 블록 헤더 묶음). 본 산출은 STD만.
- **[주의·별도 트랙]** formula_components에서 STD_S1·STD_S2 **둘 다 addtn_yn=Y**. 단면 주문 시 S1·S2가 동시 합산되면 이중청구 위험(셋트상품 하네스 §23 S1/S2 이중합산 코드결함과 동류). 단, 본 결함(자재 결손)과 독립 — S1/S2 분기(print_opt_cd 매칭으로 자연 배제되는지)는 별도 검증 필요. **본 교정은 이중합산을 새로 만들지 않음**(기존 행과 동일 print_opt_cd·mat 차원만 추가).
- **[권위 확정]** 단가·그룹 묶음은 가격표 B01에서 verbatim 인용(추측 0). min_qty=100 단일 구간만 존재(가격표·라이브 일치) — 100 미만 주문은 ERR_BELOW_MIN(가격표 정책, 결함 아님).

## 8. 권고 한 줄

**교정 구조 = (b) 종이별 단가행 전개** (자재 무관 NULL 와일드카드 (a) 아님). 왜: 가격표 B01이 5종 종이를 **2개 동일가 그룹(3500/3800)**으로 묶어 "전부 동일가"가 아니므로, NULL 와일드카드는 그룹2를 그룹1가로 저청구시키고, 두 NULL 공존은 ERR_AMBIGUOUS를 부른다. 누락 4종 mat_cd 행을 각 그룹가로 명시 전개하면 전 종이가 정확히 1행 매칭(동시매칭 0)된다.
