# 후니프린팅 어색데이터 정정 검토요약 — SIZE_NAME_NOISE

실무진(후니)이 "어느 어색항목이 어떻게 정정됐는지"를 before/after로 빠르게 확인하기 위한 요약입니다.
함께 제공되는 `product-viewer-정정반영_v2.html`(사본)을 브라우저에서 열어 항목별로 확인할 수 있습니다.
**원본(`docs/huni/...`)은 수정하지 않았으며, 본 사본만 정정 정보를 덧입혔습니다.**

---

## 1. 정정 개요

이번 정정의 핵심은 어색데이터를 단순 "오류(noise)"가 아니라 **"차원이 미분리된 데이터"**로 재해석한 것입니다.

- 대표 예: `70x200mm(50장)`
  - 치수 `70×200mm` → **라이브 DB `t_siz_sizes`에 이미 정상 등록되어 있음**(후니 D-C 등록분 반영, 적재 불요)
  - 장수 `50` → **묶음수 `bdl_qty`로 신규 분리**(`t_prd_product_bundle_qtys`)

> **권위 반전 주의**: 초기 가설("치수 슬롯이 비어 있어 채워야 한다")은 라이브 DB 검증 결과 **거짓**으로 판명됐습니다. 라이브 DB의 치수(work/cut, `numeric(8,2)`)는 이미 77/77 채워져 있습니다. 따라서 **치수축 정정은 불요**이고, 이번 정정의 실제 적재 산출물은 **묶음수 18행**입니다.

---

## 2. 전체 통계

| 구분 | 건수 | 의미 |
|------|:----:|------|
| 전체 어색항목 (findings) | 366 | product-viewer가 검출한 전체 |
| **SIZE_NAME_NOISE (이번 범위)** | **81** | 사이즈 합성명 = 치수 + 묶음수(+모양/접두) |
| └ 정정완료 (corrected) | 63 | 우리측 차원 분해 완결 |
| └ 후니확인 (huni_pending) | 18 | 분리는 완료, 후니 원천 확정/등록 대기 |
| 미착수 (untouched) | 285 | SIZE_NAME_NOISE 외 다른 유형(이번 범위 밖) |

검증: 정정완료 63 + 후니확인 18 = **81** (SIZE_NAME_NOISE 전수 일치).

---

## 3. status 3종 정의

| status | 뱃지 | 정의 |
|--------|------|------|
| **corrected (정정완료)** | 🟢 초록 | 순수 mm 치수 + 묶음수(또는 모양 동반 mm). 치수=라이브 기정상, 묶음수=신규 분리 → 우리측 분해 완결 |
| **huni_pending (후니확인)** | 🟡 호박 | cm 명칭(2)·단위 미표기(10)·3D 두께(3)·색상/자재 접두(5) 등. 분리는 됐으나 후니 원천 확정/등록 대기 |
| **untouched (미착수)** | ⚪ 회색 | `MAT_*`·`PRINTSIDE_INVALID`·`PLATE_FILETYPE` 등 다른 유형 285건. 이번 정정 범위 밖 |

> `untouched`(285)는 정정 실패가 아니라 **이번 작업 범위(SIZE_NAME_NOISE)에 포함되지 않은 다른 유형**입니다.

## 4. 후니확인 18건 내역

| 분류 | 건수 | 대상 예시 | 확인 사항 |
|------|:----:|-----------|-----------|
| cm 표기 사이즈명 정돈 | 2 | 피크닉매트 `100x70cm(1인용)` (SIZ_000399/402) | 치수값은 라이브가 mm 환산(1000×700) 보유=정확. 사이즈**명**의 'cm' 표기 정돈만 후니 결정 |
| 단위 미표기 mm 타당성 | 10 | 만년스탬프 `원형13x13`, 레더라벨 `레더15x30` 등 | 라이브 동일 값 보유. 실제 단위가 mm인지 후니 원천 확인 |
| 3D 두께(depth) 슬롯 부재 | 3 | 투명케이스 `PP투명케이스42x57x20mm` (SIZ_000108~110) | `t_siz_sizes`는 W×H 2축만 보유. depth(20/15mm) 보존 방법 후니 결정 |
| 색상/자재 접두 → mat_cd | 5 | 카드봉투 `화이트/블랙165x115mm`, 레더라벨 `레더…` | 접두는 사이즈가 아닌 자재 차원. `t_mat_materials` variant 별도 등록(MAT_* 라운드) |

> 일부 항목은 분류가 겹칩니다(예: 레더 3건은 단위 미표기이면서 자재 접두). 합계는 finding 단위 18건 기준입니다.

상세 근거: `huni-source-fix-request.md`(후니 요청서), `correction-validation-report.md`(검증 보고서).

## 5. 후니 원천 확정이 필요한 6항목 (요청서 연계)

`huni-source-fix-request.md`에 정리된 후니 결정 대기 항목입니다.

1. **cm 단위 확정(2건)** — 치수값은 라이브 정확. 사이즈명 'cm' 표기 정돈 여부.
2. **단위 미표기 mm 타당성(10건)** — 라이브 동일 값 보유. 실제 mm 여부 확인.
3. **3D depth 슬롯(3건)** — PP케이스 두께 보존 방법(siz_nm 보존/옵션/별 컬럼).
4. **'인용=세트' 해석(2건)** — 피크닉매트 1인용/2인용을 `QTY_UNIT.04(세트)`로 분리한 해석 확인.
5. **모양 귀속(원형/사각/하트)** — 옵션값인지 siz 규격인지 결정.
6. **색상/자재 접두 mat_cd(5건)** — 화이트/블랙/레더 접두를 자재 variant로 등록(MAT_* 라운드).

추가로 **도무송 사이즈-EA 종속(PRD_000066)**: 사이즈마다 구성 EA가 달라(`(prd_cd, bdl_qty)` PK 단독 표현 불가) 옵션 캐스케이드/`constraint_json` 모델링 결정 필요.

> **적재 권한**: 묶음수 18행(`t_prd_product_bundle_qtys.csv`)은 검증 전 항목 PASS(GO). 후니는 INSERT 실행만 하면 됩니다. (현재 DB 미적재 상태 유지, CSV 핸드오프.)

## 6. HTML 사용법 (`product-viewer-정정반영_v2.html`)

1. **좌측 제품 목록**: 각 제품 행에 정정 뱃지가 표시됩니다.
   - `✓N` (초록) = 해당 제품의 정정완료 SIZE_NAME_NOISE 건수
   - `!N` (호박) = 해당 제품의 후니확인 건수
   - 빨간 숫자 = 기존 어색항목 전체 건수
2. **status 필터(목록 상단)**: `전체 / 정정완료 / 후니확인` 라디오로 해당 status를 가진 제품만 추립니다.
3. **제품 클릭 → 우측 상세**:
   - 상단 **정정 반영 요약 패널**: 전체 366 / SIZE_NAME_NOISE 81(정정완료 63·후니확인 18) / 미착수 285 + 권위 반전 주석.
   - **어색함 목록**: 각 SIZE_NAME_NOISE 항목에 status 뱃지 + `→ 분리결과`(치수/묶음수/모양/접두 분해 내역).
   - **사이즈 테이블**: 정정 대상 행을 status 색(🟢/🟡)으로 음영 처리하고, Description 열에 `→ 분리결과`를 표시.
4. **뱃지 색 의미**: 🟢 정정완료 / 🟡 후니확인 / ⚪ 미착수.

> 사본은 외부 의존 없는 정적 HTML입니다. 파일을 더블클릭(또는 브라우저로 열기)하면 바로 동작합니다.
