# 가격 적재 전 선결 의존성 진단 — 시트별 FK 의존성 맵 (round-17 후속)

> **작성** 2026-06-13 · round-17 후속. 입력 = round-16 GO 8시트 decomposition + 라이브 `t_*` 실측(읽기전용 SELECT, 2026-06-13). **DB 미적재 — 분석 전용.** 생성자(dbm-load-builder) 산출이며 자기 GO 판정 없음(검증은 dbm-validator).
>
> **목적**: "가격(가격구성요소·단가행·가격사슬)을 라이브에 적재하기 전, 그 가격이 참조하는 상위 엔티티(상품 prd_cd·차원 siz/mat/proc/opt·구성요소 comp_cd·공식 frm_cd)가 먼저 적재돼야 하는가"를 시트별로 진단해, 고아(orphan FK) 없이 가격이 들어가는 순서를 정한다.

## 0. 가격사슬 FK 구조 (라이브 실측 = 권위)

가격이 "엔진 조회 가능"하려면 아래 5단계가 모두 연결돼야 한다. 한 단계만 끊겨도 단가행은 적재돼 있어도 엔진이 못 찾는다(= **가격사슬 단절**).

```
[공식정의]   t_prc_price_formulas(frm_cd)              ← 상위 0 (루트)
[배선]       t_prc_formula_components(frm_cd → comp_cd) ← frm_cd FK + comp_cd FK
[구성요소]   t_prc_price_components(comp_cd)            ← 상위 0 (루트)
[단가행]     t_prc_component_prices(comp_cd, siz_cd, mat_cd, proc_cd, opt_cd, clr_cd, …)
                                     ↑comp FK  ↑차원 마스터 FK(siz/mat/proc/opt/clr/cod)
[상품바인딩] t_prd_product_price_formulas(prd_cd → frm_cd) ← prd_cd FK + frm_cd FK
```

**FK 위상정렬(부모→자식)**: ① 차원 마스터(siz/mat/proc/opt/clr·`t_cod` PRICE_TYPE) + 상품(prd_cd) → ② `price_formulas`(공식정의) + `price_components`(구성요소) → ③ `formula_components`(배선) + `component_prices`(단가행) + `product_price_formulas`(바인딩).

**핵심 통찰**: 단가행(`component_prices`)이 참조하는 차원(siz/mat)과 상품(prd_cd)은 **대부분 이미 라이브 실재**한다. 진짜 선결 부족분은 차원이 아니라 **상위 사슬 자체**(공식정의·배선·바인딩)다. 즉 "가격을 적재하려면 상품마스터를 먼저 적재해야 하나?"의 답은 대부분 **"상품/차원은 이미 있다 — 빠진 건 그 사이를 잇는 공식·배선·바인딩"**이다.

## 5. 한 줄 현황

GO 8시트 가격사슬 FK 의존성 진단 완료 — **차원/상품은 대부분 라이브 실재**(합판 siz 37·mat 6·엽서북떡메 siz 5·아크릴본체 상품 146~150·포스터 28상품 전부 ✅). 진짜 선결 부족분은 차원이 아니라 **가격사슬 상위**(공식정의·배선·바인딩): 배선단절 3(엽서북30P·커팅타공합가·포스터27상품)·바인딩단절 1(떡메)·전건신규 2(아크릴본체·스티커타투)·siz미채번 3(스티커240·포스터667·아크릴117)·prc_typ오적재 2(합판·커팅). round-13/15 교정과 직접 충돌=합판·실사뿐. **다음 = load-priority.md(FK 위상정렬 우선순위) → dbm-validator 검증.**
