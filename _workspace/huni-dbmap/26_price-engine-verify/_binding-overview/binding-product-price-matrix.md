## 2. A/B/C/D 분류 + 처리방안 초안

### A 상품O·가격O (9종) — 연결만 하면 됨
- **데이터 준비완료**: comp·단가행·prc_typ(.01)·값(74/74 정합) 전부 정상([[hardcover-calendar-value-check]]).
- **잔여 = 공식 배선 + 상품 바인딩**(중철만 배선·나머지 8 미배선).
- **처리방안**: [[PRF_BIND_SUM]] BIND-C1 A안(제본방식별 공식 분리 PRF_BIND_<방식> + 상품 1:1 바인딩) 그대로 확장 적용.
  - 책자4(중철/무선/PUR/트윈링) → A안 공식분리·comp 배선.
  - 하드커버3(무선/트윈링) → 하드커버책자류 상품에 PRF_BIND_<HC방식> 바인딩.
  - 캘린더4 → 캘린더 완제품 공식의 제본 항으로 CAL_* comp 편입(트윈링 아님·BC-3 확정).
- **comp 신설 0·값 수정 0** — 순수 배선/바인딩(인간 승인).

### B 상품O·가격X (1종: 떡) — 가격 신규 구성
- **상품 3종 실재**(엽서북094·떡메모지097·메모패드179) but **떡 제본비 단독 단가표·comp 없음**.
- 엽서북(094)은 완제품가(PRF_PCB_FIXED·엽서북떡메 시트)로 구성됨 → 떡 제본을 별 항으로 안 뽑음(완제품가 내포).
- **떡메모지(097)·메모패드(179)는 가격공식 ❌미구성** = 가격 자체 없음.
- **처리방안**: 떡 가격은 두 갈래 —
  - ① 떡메모지/메모패드도 엽서북처럼 **완제품 고정가**(상품마스터 가격포함 시트·문구 시트 가격)로 구성 → round-2/16 가격 적재(PRF_PCB_FIXED 동형 or 신규).
  - ② 떡 제본비를 별 comp로 뽑을지는 가격표 권위에 떡 제본비 단독 표가 없으므로 **불요**(완제품가 방식 권장).
  - 컨펌 BC-7: 떡메모지/메모패드 가격 출처(상품마스터 문구 가격포함 시트에 단가 있나).

### C 가격O·상품X (1종: 싸바리) — ~~고아 부품~~ → 실재(다른 이름)로 교정
> **[교정 2026-06-15·[[sabari-layflat-research]]] 고아 아님.** 싸바리=합판지를 인쇄표지로 싸 발라 하드커버(양장) 표지를 만드는 현장 용어. 후니는 실제 판매하며 라이브에 **`PRD_000088 레더 링바인더`(+표지/면지 5상품)·`CAT_000131 레더링바인더`·후니 사이트 카테고리 "싸바리바인더"**로 실재. 미연결 원인 = "쓰는 상품 없음"이 아니라 **명칭 불일치(싸바리바인더↔레더 링바인더) + BOM/공정 미적재(088~093 공정 0)**. 처리 = 폐기 금지·**명칭 매핑 + 레더 링바인더 5상품 BOM/공정 적재 + COMP_BIND_SSABARI 가격 배선**(연결 가능·인간 승인). 컨펌 SB-1(싸바리바인더=레더 링바인더 동일 확인)·SB-2(제본 공정: 싸바리 신규 vs 하드커버 계열).

- **싸바리 comp·단가행 실재**(COMP_BIND_SSABARI 6행·값 정합) — 단가는 준비완료. (아래 "쓰는 상품 0" 판정은 위 교정으로 무효: 레더 링바인더로 실재.)
- **처리방안 초안(택1·인간 승인)**:
  - ① **보류 유지**(권장 1차): 싸바리바인더 상품을 향후 출시 예정이면 comp 보존. 가격표 B2에 싸바리 단가가 명시돼 있어 후니가 의도적으로 준비한 부품일 수 있음 → 폐기 신중.
  - ② **부품 비활성**: 출시 계획 없으면 `use_yn='N'`(논리 비활성·제안만). 단가행 보존(가격표 권위 존재).
  - **폐기(hard-delete) 금지** — 가격표에 권위 있음. 실무진 컨펌 BC-1(싸바리 상품 출시 계획) 후 결정.

### D 상품X·가격X (1종: 레이플랫) — 폐기/보류 후보
- **레이플랫 = 공정(PROC_000025)만 존재**, 쓰는 상품 0·가격표 0·comp 0.
- **처리방안 초안(인간 승인)**:
  - ① **보류**(권장 1차): 포토북 등에 레이플랫 도입 예정이면 공정 코드 보존(코드만·데이터 0이라 부담 없음).
  - ② **공정 논리 비활성**(`use_yn='N'`): 도입 계획 없으면. 단 PROC는 코드 마스터라 영향 최소.
  - **폐기 금지** — 공정 코드는 향후 확장 여지. 실무진 컨펌 BC-2(레이플랫 도입 계획) 후.
