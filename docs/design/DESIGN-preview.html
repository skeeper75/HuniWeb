1.Overview — 디자인 컨셉

후니프린팅의 디자인은 보라(Purple) 브랜드 컬러를 중심으로 한 절제된 인쇄 주문 UI입니다. 핵심은 "선택의 명확성". 사용자는 사이즈·종이·수량·후가공 등 수십 개의 옵션을 빠르게 탐색·선택해야 하므로, 모든 선택 컴포넌트는 흰 배경 + 보라 테두리로 선택 상태를 표현하여(채워진 배경 금지) 시각적 소음을 최소화하고 어떤 항목이 선택되었는지 한눈에 구분되게 합니다. 타이포는 Noto Sans 단일 패밀리에 -5% 자간을 일관 적용, 모든 조작 컴포넌트는 50px 고정 높이와 4px(컴포넌트)/5px(CTA) 라운드로 통일된 리듬을 만듭니다. 옵션 라벨·값은 절대 하드코딩하지 않고 DB/API에서 동적 주입됩니다.

3.Typography — 타이포그래피

Font Family: Noto Sans ONLY (400/500/600) · Letter Spacing: -5% 범용 (letterSpacing = fontSize × -0.05)

[CRITICAL] 폰트는 반드시 Noto Sans. Lucide 등 외부 아이콘 폰트로 캐럿(▼) 대체 금지 — 캐럿은 텍스트 문자.

8.Do's and Don'ts — Critical Rules

후니 디자인의 8개 핵심 규칙. ✅ 좋은 예 vs ❌ 나쁜 예.

RULE-1SelectBox: native <select> 금지
✅ DO
custom div 드롭다운(HuniCustomSelect), 캐럿은 ▼ 텍스트 문자 오버레이, 외부 클릭 감지 + 키보드 접근성.
아트지 250g▼
❌ DON'T
<select><option>...</option></select> 사용 — 크로스브라우저 스타일 불가, Figma 스펙 위반.
RULE-2선택 상태 = 흰 배경 + 보라 테두리
✅ DO
선택 시 bg-white border-2 border-[#553886] text-[#553886]
선택됨
❌ DON'T
선택 시 컬러 배경 채움 bg-[#553886] text-white 또는 #EEE8FF 채움
채운 배경 (금지)
RULE-3CounterInput: 직사각형 3-part
✅ DO
[34px −] [155px 값] [34px +] = 223×50px, divider 1px #CACACA
−
1,000
+
❌ DON'T
원형 버튼(rounded-full)·native number input·증감 화살표 스피너
−+
RULE-4ColorChip: 50×50 원형
✅ DO
50×50px ellipse, 선택 시 흰 채움 + #553886 ring 2px
❌ DON'T
32×32 등 다른 크기로 ColorChip 렌더 (32px는 MiniColorChip 별도)
RULE-5옵션 라벨 하드코딩 금지 (동적 데이터)
✅ DO
모든 옵션 라벨/값/구조를 DB/API에서 동적 로드 후 .map() 렌더.
우선순위: Option Schema API → Zustand store → props.
❌ DON'T
<OptionButton>무광코팅(단면)</OptionButton> 같이 한국어 옵션 라벨을 JSX에 직접 작성.
RULE-5/6/7/8-EXTSlider Radix 필수 · ImageChip 50×50 · MiniChip 32×32 · LargeChip grid
✅ DO
@radix-ui/react-slider 사용 · ImageChip 50×50 원형 · MiniColorChip 정확히 w-8 h-8(32px) · LargeColorChip grid-cols-5 DB 동적 로드(상한 없음).
❌ DON'T
native input[type=range] 사용 · ImageChip 사각 큰 썸네일 · MiniChip을 w-12 h-12(48px)와 혼동 · 색상 개수 코드 하드코딩.
공통타이포 / 색상
✅ DO
Noto Sans only, 자간 -5%, 본문 텍스트 색상은 #424242/#553886/#979797/#000000.
❌ DON'T
다른 폰트 패밀리, 컬러 폰트(#FF00FF 등) 콘텐츠 텍스트 사용, 흰 텍스트를 컬러 배경 없이 사용.
