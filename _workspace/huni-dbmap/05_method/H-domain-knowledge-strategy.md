## 1. 문제 진단 — 인과 체인의 구조화

### 1-1. 결함 인과 체인 (사용자 통찰의 형식화)

```
[근본] 도메인 지식이 1급 입력이 아님 (데이터에서 역추론)
   │
   ▼
[1차] 용어 의미 오해   — "작업사이즈=재단+블리드" 모름 / "별색=공정" 모름 / "전지≠작업사이즈" 모름
   │
   ▼
[2차] 변환규칙 오류    — B 28규칙이 *추정된* 도메인 의미 위에 세워짐 (R-SIZE·R-MAT·R-PROC·R-CLR…)
   │
   ▼
[3차] 매핑/검증 결함   — false MISSING / 오분류 / N/A로 닫음 / 상쇄형 누락 은폐
```

핵심: **B 규칙 전체가 인쇄 도메인 의미 해석에 의존**한다(B 문서 자체가 "각 규칙 형식: [엑셀 칸] → [DB 코드] + **도메인 의미**"로 명시). 그런데 그 도메인 의미가 *데이터에서 역추론한 추정*이라, 추정이 맞으면(R-PROC-2 줄수=공정) 진짜 결함을 잡고, 틀리면(직각/둥근·별색·작업사이즈) 거짓 결함을 양산한다. 규칙이 진동하는 이유 = **도메인 근거가 공리(axiom)가 아니라 추측(guess)**이기 때문.

### 1-2. 결함별 인과 추적 (전부 "도메인 의미를 몰라서")

| # | 결함 (증거) | 몰랐던 도메인 의미 | 잘못된 규칙/판정 | 검증 결과 |
|---|------------|-------------------|-----------------|----------|
| 1 | 포맥스보드 A1 false MISSING | 작업사이즈=재단+블리드, **작업사이즈 공백=생산 미정의 신호** | L1이 사이즈를 E단일컬럼만 평면화(작업사이즈 I/J drop) | 1차 false MISSING → L1 2계층 분리로 해소(`HANDOFF §5-3`) |
| 2 | 별색=도수로 오분류 | 별색인쇄(화이트/클리어/금/은)=별도 **후가공 공정**, color count 아님 | 인쇄옵션 컬럼 위치만 보고 도수축으로 라우팅 | v2 별색 공정분기로 정정, 잔여 CONFIRM 7(`audit-summary-v2 §5`) |
| 3 | 판형 의미축 충돌 | 엑셀 **전지**(press sheet, 원사이즈)≠DB **작업사이즈**(cut+bleed) — 같은 "판형"이 다른 물리 개념 | plate_size 행대조 무의미 | CONDITIONAL, 자동대조 보류(`audit-summary-v2 §1·5`) |
| 4 | 공정택일그룹 N/A로 닫음 | 캘린더·현수막류 **여러 공정 UI 택일** 구조(인쇄방식별 공정 구성 지식 필요) | 엑셀 명시 컬럼 부재 → "원천부재 N/A" | BLOCK-3 N/A 보류, **닫지 말 것**(`HANDOFF 시작점`) |
| 5 | 묶음수 도메인 미확정 | 명함 Y=플래그 vs 수량 / 조각수 / 페이지 의미 | 묶음수=정수 가정 | MAJOR, 의미 미확정(`audit-summary-v2 §1`) |
| 6 | 박/형압/오시/미싱/VDP 부모-자식·leaf | 박=색상별 자식 leaf, 형압=양각/음각, 오시(누름선)≠미싱(절취선), 가변텍스트/이미지=VDP | 부모코드 오매칭 false MISSING 12 | v2 부모규칙으로 정정, 형압자식 6 신규 진짜 MISSING(`audit-summary-v2 §3`) |

→ **6/6 모두 도메인 의미가 공리로 깔려 있었으면 사전 차단됐을 결함**. 결함의 종류가 일관됨 = 우연이 아니라 구조.

### 1-3. 현재 방식("데이터→도메인 역추론")이 구조적으로 틀리는 이유

- **역추론은 표본 의존**: 4종 표본(프리미엄엽서 외)에서 규칙을 뽑아 260상품에 일반화. 표본에 없는 의미축(전지/택일그룹/substrate)은 규칙에 안 잡힘.
- **데이터는 의미를 말하지 않음**: `1줄`이 "공정 존재 신호"인지 "옵션값"인지 데이터만으로는 판정 불가(1차가 후자로 처리→누락 은폐). 의미는 *외부 도메인 권위*에서만 온다.
- **상쇄형 은폐**: 거짓 정합(연결만 맞고 의미 틀림)은 데이터 대조로 안 보인다. 도메인 의미 검증 없이는 "GO"가 거짓 GO일 수 있음(1차 검증 결함의 본질, `HANDOFF §2`).
- **옳은 방향**: 도메인 지식 먼저 확립 → 그 공리로 데이터 해석. 규칙은 데이터에서 *발견*하는 게 아니라 도메인에서 *연역*하고 데이터로 *검증*.

### 1-4. 재발견된 사실 — 도메인 지식은 이미 DB에 인코딩돼 있다 (전략 전환점)

`ref-processes.csv` 결정적 증거: `PROC_000001 인쇄`(부모) → 자식 `옵셋(003)·디지털(004, note "토너 후니 현재 운영")·실크(005)·실사(006)·UV(002)`, note **"Decision 14 v3 — 1상품=1인쇄방식"**. 즉 **DB가 이미 "인쇄방식=최상위 축, 1상품=1방식" 분류를 담고 있다**(사용자가 독립적으로 명명한 "옵셋/실사/윤전/디지털 요리 레시피" 축과 정확히 일치 → 2중 독립 도달, 최상위 축 확신 高).

note 컬럼 곳곳에 `Decision 12`·`D-24`·`D-25`·`D-26`·`R4-3`·`b.9`·`b.12` 등 **설계 결정 트레일**이 박혀 있다. 누군가 의도적 정규화 결정을 내렸고 그 근거가 note에 화석화돼 있다. → **도메인 지식은 "부재"가 아니라 "재구성 안 됨"**. 전략은 지식을 *발명*하는 게 아니라 흩어진 4곳(①DB 구조+note ②한국 인쇄 표준 ③Decision 문서(존재 시) ④실무진)에서 **재구성·통합**하는 것.

## 3. 인쇄 도메인 지식 아키텍처 — 4계층 모델

> 설계 판단: 단순 glossary로는 결함 차단 불가. 결함은 *용어 정의*가 아니라 *관계*(방식→레시피, 용어→의미축, 부모→leaf)에서 났다. glossary는 "오시=누름선"을 주지만 "오시는 digital-print 시트에서 줄수 컬럼으로 표출되고 공정 존재를 신호한다"를 주지 못한다. → 관계를 담는 4계층이 결함 차단의 *최소* 구조.

### L0 — 인쇄방식 Taxonomy (최상위 축)

**근거**: ref-processes `PROC_000001` 자식 5종 + note "1상품=1인쇄방식". 사용자 "요리 레시피" 비유의 형식화.

```
인쇄방식 (1상품=1방식, 가능 공정·용어·가격구조를 결정하는 최상위 축)
├ 옵셋 (PROC_000003) — 평판, 대량, 판비 발생. 명함/책자/엽서 등 범용지
├ 디지털 (PROC_000004, "토너 후니 현재 운영") — 무판, 소량, 가변(VDP) 가능
├ 실크 (PROC_000005) — 공판, 특수 소재/굿즈
├ 실사 (PROC_000006) — 대형 잉크젯, 비규격 가로/세로 입력, 현수막/배너
└ UV (PROC_000002) — 특수, 변형 enum(투명/배면양면 등)
+ 특수 add-on 공정군(별색·박·형압)은 base 방식 위에 레이어
```

- **시트 ↔ 인쇄방식 매핑**(13 상품마스터 시트): 디지털인쇄·스티커·책자·포토북·캘린더·디자인캘린더·실사·아크릴·문구·굿즈파우치·상품악세사리 → 각 시트 ≈ *인쇄방식으로 경계지어진 상품군*. 이 매핑 확정이 L0의 핵심 산출(현재 🟡 — 시트명에서 추정 가능하나 실무진 확정 필요).
- **방식별 공정 레시피 가능집합**: 각 방식이 허용하는 공정 부분집합. 예) 실사=족자제작(PROC_000082 "b.9 실사 전용")·봉제·부착 / 굿즈파우치=에폭시(PROC_000083 "굿즈파우치 전용") / 디지털=가변텍스트·가변이미지(VDP). note가 이미 일부 명시 → 전수 정리.

### L1 — 용어 브리지 사전 (4자 매핑, 단순 glossary 아님)

각 행 = **[인쇄 표준용어 ↔ 물리적 의미 ↔ 후니 엑셀 표기 ↔ DB 코드]** + 확정도 + 권위출처. 결함을 차단했을 바로 그 산출물. 예시:

| 표준용어 | 물리적 의미 | 후니 엑셀 표기 (시트·컬럼) | DB 코드 | 확정도 | 권위 |
|---------|-----------|--------------------------|---------|:------:|------|
| 오시(Crease/누름선) | 접지 전 종이 갈라짐 방지 압선 | `없음\|1줄\|2줄\|3줄` (digital-print AE) | `PROC_000029 {줄수 max:3}` | 🟡 | 표준+DB note "Crease 누름선" |
| 미싱(Perforation/절취선) | 손으로 뜯도록 점선 천공(티켓/쿠폰) | `없음\|1줄~3줄` (digital-print AF) | `PROC_000030 {줄수 max:3}` | 🟡 | 표준+DB note |
| 별색인쇄(Spot color) | CMYK 외 별도 잉크 후공정(화이트/클리어/금/은) | 별색 5컬럼 (R~V) | `PROC_000007`+자식 008~012 | ✅ | v2 검증+DB note "Decision 12" |
| 전지(Press sheet) | 종이 원사이즈 재단 전 원장(국전 636×969·46전 788×1090) | 출력용지규격 (J/col) | (t_siz 부재) | 🔴 | 표준(samto·bncplan), DB 미보유 |
| 작업사이즈(Work size) | 재단+블리드 포함 작업 영역 | 작업사이즈 (시트별 H~J) | `t_prd_product_plate_sizes.siz_cd` | 🟡 | 판걸이수 브리지 |
| 형압(Embossing 양각/음각) | 압형으로 종이 돋움/눌림 | 박/형압 가공 (AI) | `PROC_000050`+양각051/음각052 | 🟡 | 표준+DB |

### L2 — 공정 레시피 트리 (택일/다중·전후 의존)

### L3 — 엔티티 의미 모델 (속성별 의미축)

9속성 각각이 인쇄에서 *무엇을 모델링*하나 + **의미축**. 전지vs작업사이즈 류 구분의 거처.

- **사이즈 = 3 의미축**: 전지(원사이즈)/작업사이즈(재단+블리드)/재단사이즈(완성). 엑셀·DB가 서로 다른 컬럼에 인코딩 → "판형/사이즈" 단일명이 충돌 유발. 3축을 명시 분리.

## 4. 지식 획득 3원(+1) 전략 — 권위 순서·충돌 해소

사용자 명시 3원에 **Decision 트레일(0순위)** 추가 권고(아래 §4-0 pushback).

### 4-0. [선행 권고 — pushback] Decision 트레일 수확 (0순위)

note 컬럼의 `Decision 1~26 v3`·`D-24`·`b.9` 등이 가리키는 **설계 결정 문서가 존재한다면, 그것은 이미 글로 화석화된 후니 권위**다. 실무진 인터뷰 전에 이를 찾으면(저비용·고권위) 질의 부하가 줄고, *이미 문서화된 걸 다시 묻는* 낭비를 막는다(scope discipline). → **Phase 0에서 Decision 문서 소재 확인 → 있으면 수확, 없으면 실무진 질의로 승계**. 단정 금지: 문서 존재 여부 미확인(🔴).

### 4-1. 권위 순서

| 순위 | 원천 | 역할 | 강점 | 한계 |
|:---:|------|------|------|------|
| 0 | Decision 트레일(존재 시) | 동결된 후니 결정 근거 | 명시적·재논쟁 불요 | 존재 미확인 |
| 1 | **후니 실무진** | 후니 고유 의미·암묵규칙 (최종 권위) | 후니 진실 | 비용·가용성·암묵지 추출 난이도 |
| 2 | 인쇄 산업 표준 | 골격·교차검증 (**보조**) | 일반성·검증성 | 후니 표기와 충돌 가능 |
| 3 | 데이터 역공학 | 검증·교차·후보 생성 | 전수성 | 의미 미보유(추정만) |

### 4-2. 충돌 해소 규칙

- **실무진 > Decision문서 > 표준 > 데이터**. 단, Decision문서가 실무진 현행과 어긋나면(설계 후 운영 변경) 실무진 우선 + 변경 기록.
- 표준이 후니 표기와 충돌: 표준은 `term-bridge`의 *보조 컬럼*으로만 보존, 후니 표기/DB 코드가 권위. 충돌은 플래그(예: 후니 "별색" 범위 ≠ 표준 spot color).
- 데이터 역공학은 **후보만 생성**(MISSING/EXTRA 신호), 의미 단정 금지. 단정은 1~2순위로만.

### 4-3. 실무진 질의 우선순위 (구조화 질의서 — 빈도×모호×블로킹)

각 질의 = **DB 현재 인코딩 + 엑셀 증거 + 후보 해석 2~3안 제시 → 택1/교정 요청**(개방형 금지, AskUserQuestion 형식).

| 우선 | 질의 주제 | 제시할 후보 | 해소 대상 |
|:---:|----------|-----------|----------|
| 1 | **시트↔인쇄방식 매핑** (L0 앵커) | 13시트 각 옵셋/디지털/실사/실크/UV 중 + "1상품=1방식" 확인 | 최상위 축, 전 캐스케이드 |
| 2 | **공정택일그룹 의도** (BLOCK-3) | DB 13행 그룹 = 방식별 UI 택일 구조? 캘린더/현수막 예시 확인 | 결함4 / N/A 해소 |
| 3 | **판형 3축 의미** | 전지/작업/재단 — 엑셀 출력용지규격=전지? DB plate=작업? | 결함3 / CONDITIONAL |
| 4 | **별색 substrate 잔여** (CONFIRM 7) | 포맥스/폼보드 = 별색 공정 대상? 아니면 substrate? | 결함2 잔여 |
| 5 | **묶음수 도메인** | 명함 Y=플래그/수량? 조각수=반칼 파라미터? 페이지 vs page_rule? | 결함5 / MAJOR |
| 6 | **의미코드맵 시트별** | 노랑/그레이/숨김 = 신규/품절/비활성? 시트별 범례 | 메타·라이프사이클 |
| 7 | 자재 명칭약어 표준화 | 백모조=백색모조지? IMPORT 카드 통합컬럼 분기 | 자재 MAJOR 잔여 |

→ **1번이 최우선**: 인쇄방식 축이 확정돼야 2~7의 방식별 해석이 가능(레시피·택일·가능공정 모두 방식 종속).
