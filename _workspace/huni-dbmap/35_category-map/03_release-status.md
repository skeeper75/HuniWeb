# 03 — 출시상태 3분류 v2 (권위 = 원천 데이터: 상품마스터 260610 + 인쇄상품 가격표 260527)

**산출**: `release-status.csv` (MAP product 244건). v1 백업 = `release-status-v1.csv`.
판정 = **원천 가격계산 가능성**(`_reclassify/price-quotability.csv`). 라이브 적재는 판정 근거 아님(참고만).

## v1 → v2 권위 전환

v1은 출시상태를 **라이브 적재 여부**(option_items>0 AND priced=Y)로 판정했다. 그러나 라이브는
옵션(round-7 CPQ 전역 미적재)·가격사슬(round-16/18 단절)이 *아직 안 옮긴* 상태라, 적재 기준으로 판정하면
원천상 정상인 상품이 전부 옵션부족으로 오분류된다. v2는 스킬 §4 갱신에 따라 권위를 **원천 데이터**로
전환 — 핵심 질문 = "이 상품이 가격/견적 계산에 필요한 필수 옵션·요건을 원천 데이터상 갖추었는가."

## 분류 정의 v2 (원천 권위)

| 상태 | 정의 | 판정 조건 (price-quotability) |
|------|------|----------|
| ✅ 정상등록가능 | 가격공식·필수입력이 원천상 완비 → 견적 성립 | verdict = **QUOTABLE** |
| 🟡 옵션부족 | 가격계산 필수입력이 원천상에서도 결손 → 견적 불가 | verdict = **NOT-QUOTABLE** (현재 **0건**) |
| 🟦 보류(NEEDS-DOMAIN) | 옵션부족이 아니라 가격 귀속 자체가 도메인 컨펌 대기 | verdict = **NEEDS-DOMAIN** (11건) |
| ❌ 미출시(부재) | 상품 자체가 원천에 없음 | 데이터시트 0 AND 라이브 0 |

★ **옵션 개수>0은 ✅ 조건이 아니다.** 고정가형·사이즈×수량 면적매트릭스형은 옵션 없이 가격표만으로
견적되는 정상 상품(`[[dbmap-price-formula-types-authority]]`). 🟦보류는 🟡옵션부족과 **구분되는 별도 상태** —
견적 불능이 아니라 가격축 귀속이 미확정(아크릴 완제품 vs 자유형·굿즈 가격공란 추출 불일치)이다.

보조: ◆교차참조(별칭), ➖N/A(섹션). 244 product 행만 본 표 대상.

\* v1 status 컬럼은 ❌11(244행 기준). v1 문서표의 "12"는 별칭 1건 포함 집계 차이로, 행 단위 권위는 11.

- **격상(🟡→✅) 186건** = 원천상 가격계산이 성립하는 정상상품. 라이브 옵션0/가격사슬X는 *적재 미완*일 뿐
  출시 결격이 아님 → `_reclassify/upgraded-green.csv` 전수 목록.
- **재라벨(🟡→🟦) 11건** = v1에서 옵션부족으로 묶였으나 실제로는 가격축 귀속 미확정(도메인 컨펌 대기).
- **강등 0건** — v1의 ✅36은 전부 v2에서도 ✅(QUOTABLE) 유지.
- 각 행 `reason` 컬럼에 `[v2·verdict·변동]` 태그 + price-quotability evidence 기재.

## 주의

- 본 판정은 **원천 가격계산 가능성** 기준. 라이브 적재 현황(`live_opt_n`/`live_priced`)은 참고 컬럼.
- price-quotability는 가격 도메인 판정(dbm-price-arbiter 협업 산출). ✅도 변형 단가행 전수 완전성까지
  입증한 것은 아님 — 2단계/검증에서 심화(생성자≠검증자).
- DB 미적재 — 분류·라우팅까지만. 실 적재/교정은 인간 승인 후 기존 라운드 위임.
