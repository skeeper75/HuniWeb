## §1 상품 모집단 정합 (280 vs 260)

| 측면 | DB (`ref-products.csv`) | 엑셀 (`excel-qty-range.csv` backbone) |
|------|------------------------|--------------------------------------|
| 행수 | **280** | **260** |
| 고유 `prd_nm` | **273** | **253** |
| 중복 이름 (동일 이름 2행) | 7 | 7 |

### 모집단 산식 (양측 정합)
- DB: 280행 = 273 고유 + 7 중복추가행. 273 고유 = 245(양쪽 공통) + 28(DB-only).
- 엑셀: 260행 = 253 고유 + 7 중복추가행.
- **고유 이름 차이 273−253 = 20 = 28(DB-only) − 8(엑셀-only).**

### DB-only 28종 — 성격: **전부 반제품 컴포넌트** (`PRD_TYPE.02`)
근거: `ref-products.csv`에서 28종 전부 `prd_typ_cd=PRD_TYPE.02`이며 `semi_role_cd`가 SEMI_ROLE.01(내지)/.02(표지)/.03(면지). 예:
- `포토북 [디자인명]-내지(몽블랑130)`·`-표지(레더하드커버)`·`-면지(그레이)`
- `하드커버책자-면지(그레이면지)`·`-표지(전용지)`, `레더 링바인더-면지(블랙면지)`, `엽서북-내지(몽블랑240)` 등

→ DB는 책자/포토북류를 **내지·표지·면지 반제품 단위로 분해 저장**, 엑셀은 동일 상품을 **하나의 통합 상품행 + 내지/표지 2축 컬럼**(M=내지종이, X/Z=표지종이)으로 보유. 즉 같은 정보의 모델링 차이 — **누락 아님**. 삭제 금지, "DB 반제품 분해" 플래그.

### 엑셀-only 8종 — 성격: 잡음 1 + 보류/품절/미등록 7 (확인필요)
근거: `excel-qty-range.csv` sheet 컬럼.
- **잡음 1**: `노랑색배경은 신규상품으로...`(문구 시트) = 엑셀 헤더 주석 셀이 상품명으로 오파싱된 **비상품 행**. 모집단 분모에서 제외 권고.
- **보류/품절/신규 7**: `링바인더 (보류중)`(책자), `투명포스터★`(실사), `버즈케이스★`·`에어팟케이스★`·`블랙젤리`·`슬림하드 폰케이스`·`임팩트 젤하드`(굿즈파우치). `★`/`(보류중)`=신규/미등록 신호(엑셀 범례). **DB 미등록 = 정합 위반 아님(향후 등록 대상)** — 확인필요 플래그.

### 7 중복 이름 — 양측 동일, 정합
근거: 양측 모두 동일 7개(`떡메모지`·`탁상형캘린더`·`미니탁상형캘린더`·`엽서캘린더`·`벽걸이캘린더`·`와이드벽걸이캘린더`·`아크릴코롯토`). DB에서 캘린더 5종은 `PRD_TYPE.04` 2행(캘린더 시트 ↔ 디자인캘린더 시트 미러), `떡메모지`/`아크릴코롯토`는 `.03 기성` ↔ `.04 디자인` 듀얼. 엑셀도 동일하게 2시트에 중복 — **양측 일치**.

> **9속성 정합 분모**: 엑셀 유효상품 = 260 − 1(잡음) = **259** (또는 보류 7 제외 시 252). 양쪽 공통 매핑 가능 모집단 = **245 고유 이름**(+7 중복쌍).

## §4 Phase 4(속성별 연결 검증) 인계 주의점

1. **모집단 분모 = 245 공통 고유 이름(+7 중복쌍)**. 엑셀 잡음행 1(노랑배경 주석) 제외, 보류/품절 7은 검증대상 외(확인필요). DB 반제품 28(`PRD_TYPE.02`)은 엑셀 통합상품의 내지/표지/면지 분해 — 연결 검증 시 통합↔분해 매핑 규칙 필요.
2. **사이즈 표기차 정규화 규칙(HARD)**: `\d+ x \d+ mm` → 공백·mm 제거 → `\d+x\d+`로 DB siz_nm 대조. 판걸이수 브리지(A↔B). 범위형(`200~3000`)은 siz 아닌 `t_prd_products.nonspec_*`. 복합형(`원형 10mm (8EA)`)은 사이즈+조각 분리.
3. **자재 표기차 규칙**: `아트250`→`아트지 250g`(지/g 보정). 13 확인필요는 연결 전 마스터 확인 우선. `*별도설정` 제외, 복합(`+`) 분해.
4. **공정 컬럼 과적재 주의**: 엑셀 공정 토큰의 대부분(~280)은 공정명이 아닌 옵션값/파라미터/사이즈 → DB `t_proc_processes`(81 계층)에 직접 대조 금지. 공정명만 추출 후 대조, 옵션값은 별도 옵션축으로 해석.
5. **별색 = 공정축(확정)**: 화이트/클리어/핑크/금색/은색은 인쇄옵션 컬럼 위치이나 DB PROC_000008~12로 연결. 인쇄옵션(`t_prd_product_print_options`) 아닌 공정(`t_prd_product_processes`)으로 매핑.
6. **공정택일그룹은 추론값(확인필요)**: 엑셀 직접 컬럼 부재. `t_prd_product_process_excl_groups`(13행) 실제 sel_typ_cd와 "엑셀 옵션-컬럼=1택일그룹" 추론을 단정대조 금지, 분기 명시.
7. **EXTRA(DB 잉여) 삭제금지**: 사이즈 280 EXTRA·자재 다수 EXTRA는 엑셀 미참조이나 다른상품/사전적재 가능 — 플래그+출처만, 삭제 단정 금지.
