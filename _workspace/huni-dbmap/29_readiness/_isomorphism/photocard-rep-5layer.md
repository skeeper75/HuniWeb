### 대표 종단 = **부분 GO** (세트제작 모드 = 견적가능 GO · 대량제작 모드 = NO-GO[PHOTOCARD-BULK-WIRE])

- **024/025 세트제작** = 완전 견적가능 GO: ①②③ 전부 성립, SET/CLEAR_SET 배선·라이브 단가로 재현(6,000/8,500·보정0·×수량 안 함). **명함의 NAMECARD-WIRE(상품 2/3 단절)와 달리 세트모드는 두 상품 다 배선 완비.**
- **024/025 대량제작** = ①② 성립·③ **단절**(BULK comp 50구간 데이터 실재하나 어느 공식에도 미배선) → 대량주문 견적 도달 불가 → **NO-GO**(배선만·comp/단가 신설 불요).
- **화이트별색 옵션** = optional 그룹의 1옵션이 item 0(BLOCKED) → 선택은 표시되나 차원환원 불가(견적 본체 무영향·optional이므로 NO-GO 사유 아님·정직 갭).
- 거짓 GO 방지: 포토카드를 무조건 "견적가능"이라 하지 않고 **"세트제작 모드 GO + 대량제작 모드 NO-GO(PHOTOCARD-BULK-WIRE) + 화이트별색 BLOCKED"** 로 정직 판정.
- ★ **명함(2차)보다 가볍다** — 명함은 상품군 2/3(031·032)이 가격 완전 단절이었으나, 포토카드는 **양 상품 세트모드 완비**이고 결함은 대량모드 1배선 + optional BLOCKED. 고정가형 결함 스펙트럼에서 "per-variant 배선 완비·모드 누락만" 케이스를 추가.

## 2. Q4·Q5 가격 게이트 상세 — 고정가형 (×수량 안 함·보정 0)

### 2-1. 고정가형 룩업 메커니즘 (라이브 실측·명함과 동형·BULK 모드 추가)

| 항목 | 세트제작(SET·CLEAR_SET) | 대량제작(BULK) |
|------|------|------|
| 공식 구조 | 상품 → SET/CLEAR_SET comp **택1 룩업** | 총제작수량 → 구간 룩업(50밴드) |
| 수량 처리 | **×수량 안 함**(20장1세트 세트당 총액·bdl_qty=20·min_qty=1) | 총제작수량(매) min_qty 구간 단가(이미 총액·×수량 안 함) |
| 룩업 키 | (상품 → comp) × (소재 mat_cd) | (총제작수량 → min_qty 구간) |
| 용지 | comp 단가에 **포함**(고정가형 원리) | comp 단가에 포함 |
| 배선 | ✅ SET disp1·CLEAR_SET disp2 | ❌ 미배선 |

★ **명함과 동형(×수량 안 함)**: 세트제작 SET 6,000은 "20장1세트 세트당 총액"(bdl_qty=20). ×주문수량을 하지 않는다. BULK도 구간 단가가 이미 총제작수량별 총액(예 100매=9,500은 100매 전체 총액). 출력매수 변환(엽서 합산형 함정)은 고정가형엔 구조적으로 없음.

### 2-2. ★ 라이브 신규 확정 — 세트 vs 대량 = 단일 vs 50구간

라이브 실측: `COMP_PHOTOCARD_SET`/`CLEAR_SET` = **각 1행**(min_qty=1·bdl_qty=20·세트당 고정). `COMP_PHOTOCARD_BULK` = **50구간**(min_qty 20~3000). → 세트모드 Q5는 수량구간 교차 불요(룩업 1회)·대량모드는 50구간 단조 룩업.
- 가격표 R98 `300매 → 13,500`은 R97 `280매 → 15,500`보다 낮은 **역전 dip**이나 **라이브 BULK 적재값도 13,500**(엑셀 verbatim) = 적재 결함 아님·가격표 자체의 실무진 입력값(권위·토대 §7). 적재가 골든을 그대로 옮김.

### 2-3. ★★ PHOTOCARD-BULK-WIRE 결함 (라이브 wired_to 실측·재현)

`t_prc_price_components` × `t_prc_formula_components` 조인 실측 — 포토카드 comp 중 SET/CLEAR_SET만 wired_to=PRF_PHOTOCARD_FIXED, BULK는 wired_to=∅:

| comp | 단가행 | 배선됨? | 영향 |
|------|:--:|:--:|------|
| COMP_PHOTOCARD_SET | 1(6,000) | ✅ PRF_PHOTOCARD_FIXED disp1 | 024 세트 룩업 가능 |
| COMP_PHOTOCARD_CLEAR_SET | 1(8,500) | ✅ PRF_PHOTOCARD_FIXED disp2 | 025 세트 룩업 가능 |
| COMP_PHOTOCARD_BULK | 50(5,000~126,000) | ❌ 미배선 | 대량주문 견적 단절(NOT EXISTS unwired=2: BULK + 무관 POSTER_ARTPRINT_PHOTO) |
| COMP_POSTER_ARTPRINT_PHOTO | 1(21,600) | ❌(포스터 comp·무관) | 포토카드 클래스 외 |

★ **명함 대비**: 명함은 동일 모드(세트) 안에서 명함종(STD/PREMIUM/COAT) 분기 24 comp 중 2만 배선(per-variant 결함). 포토카드는 **per-variant(상품별 SET) 배선 완비**, 대신 **per-mode(세트 vs 대량) 누락**. 두 결함은 동형 검사 로직(내 comp가 배선됐나)으로 같이 잡히되 결함 위치가 다름(변형 vs 모드).

★ **거짓 GO 차단(핵심)**: C3에서 대량 100매 주문 시 SET(6,000/세트)으로 떨어지면 100매=5세트=30,000으로 "가격 나옴"처럼 보일 수 있으나(만약 엔진이 ×수량을 한다면), 고정가형 Q4가 "대량모드는 BULK comp가 배선돼야"를 검사해 **세트 단가 강하를 정상가 아닌 단절로 판정** → 거짓 GO 방지. 대량제작 100매 정가 = BULK 9,500(세트환산 30,000 아님)이라 세트강하는 큰 과대청구 함정.
★ **C5(코팅 가산)**: 코팅 옵션이 차원환원(②)은 성립(PROC_000014 유광)하나, 코팅의 **가격 가산분이 SET 포함가인지 별 후가공 comp인지** 라이브 미확정 → Q5 부분(🟡). 세트 기본가는 GO·코팅 가산은 가격사슬 추가 확인 필요(인간 승인 큐 C-COAT).

### 2-5. ★ 고정가형 prc_typ = .01 정합 (명함과 동형·합산형 D-1b와 구분)

라이브 PRICE_TYPE 코드 = **.01 단일값만**(명함 2차 V5 확정과 동일·.02/.03 부재). 포토카드 comp 전부 .01.
- **고정가형(포토카드·명함)**: 단가행이 곧 세트당/구간별 완제품 총액·×수량 안 함 → .01(단가형)로 룩업해 그대로 쓰는 게 **정합**(메타 오적재 아님).
- **합산형(엽서)**: 후가공 13 comp가 .01인데 실제 구간고정총액형(.03 필요) = D-1b 메타 오적재.
- → 포토카드 결함은 **prc_typ 메타가 아니라 순수 배선**(PHOTOCARD-BULK-WIRE). ★ Q4가 두 결함 유형(메타 오적재 vs 배선 누락)을 혼동 없이 구분 = 파이프라인 일반성 재입증.
