## 2. 가짜신호(noise) 원인 — §26 batch가 gap으로 오판한 이유

- **B02/B03**: 권위 사이즈 라벨(A4/B4/A3/B3/A2)과 라이브 siz_cd(SIZ_000172/515/174/514/197)가 **이름은 같으나 코드가 다른 사이즈 인스턴스**(상품 전용 siz_cd). 배치 파서가 bare A4=SIZ_000258 등으로 매칭 시도→불일치로 오판. 실제 상품이 참조하는 siz_cd로 환원하면 30/30 전수 일치.
- **B04**: 단일사이즈+수량 6행 소블록 → 배치가 메인 B01(540셀 대형 격자)에 묻혀 소블록 헤더 인식 실패.
- **B05/B06**: 타투는 **공식형(밴드 enumeration)** — 권위는 "3장마다 4000원" 한 줄 + 기본가 2000 별도 블록(B06)인데, 라이브는 둘을 합쳐 333행으로 펼침. 셀-대-셀 직접 매칭이 불가(권위 2행 vs 라이브 333행)해 batch가 gap으로 오판. 공식 환원하면 오차0.
- **B07**: 권위 "qty 1→4000, qty 1000→4000" 2행을 batch가 수량곡선으로 오인했으나 실제는 수량무관 정액 → 라이브 단일행(min_qty=54 정액)이 정답.

## 요약

**스티커 B02~B07 소블록 전부 gap-0 확정 — 진짜 결함 없음(전건 NOISE).** 6개 소블록 모두 권위 가격표 셀이 라이브에 verbatim 적재돼 있음: B02 낱장 30/30·B03 투명 30/30·B04 대형 6/6·B05 타투 333행 공식 오차0·B07 팩 정액 1/1. B06 "기본가"는 독립 격자가 아니라 B05 타투 공식의 베이스(2000)+증분(4000/3장) 기술자로 라이브 밴드행에 정확히 내장됨. §26 batch의 UNMAPPED/gap 신호는 (a) 사이즈 코드 인스턴스 불일치(권위 라벨 A4 ↔ 상품전용 siz_cd), (b) 공식형 밴드 enumeration(권위 2행 ↔ 라이브 333행), (c) 정액가를 수량곡선으로 오인 — 세 가지 파싱 한계의 false 신호였음. **dbmap으로 라우팅할 실 결함 0건.** 스티커 시트는 B01(반칼 메인 540셀)에 이어 B02~B07까지 100% 적재 무결성 확인 완료.
