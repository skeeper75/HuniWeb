## 0. [보정·독립검증 NO-GO 반영 2026-06-13]

독립검증(`dbm-validator`)이 라이브 실측으로 두 결함을 적발 → 전 binding·매트릭스 보정 완료:

1. **도메인 생산형태 ≠ 라이브 `prd_typ_cd`** (라이브 실측: `.01 완제품 3 · .02 반제품 28 · .03 기성 123 · .04 디자인 118 · .05 추가 3`). 도메인상 완제품인 디지털·스티커·아크릴·실사는 라이브 `.04`(디자인), 굿즈·문구는 `.03`(기성)으로 적재 = **라이브 `prd_typ_cd`가 생산형태 아닌 editor 여부·매입성 혼합기준 오모델.** 악세사리만 도메인=라이브 일치. 각 binding A섹션이 **도메인/라이브 분리 병기**로 정정됨.
2. **"선택·가격 라이브 0"은 부정확** → 라이브 실측 `option_items` **25** · `comp_prices` **3,481** · `product_prices` **0** · `price_formulas` **63**. 아래 §1 종합표의 ❌/🟡는 **"상품군별 충족률"**이지 전역 0이 아님(이전 진단 과장 정정).
3. **실무진 표시용 쉬운 라벨 원칙**(`§1.0-b`) — `note`/`tags`에 `PRD_TYPE.04`→"디자인상품(에디터)" 식 평이 한국어 병기.

> 아래 §1~3은 보정 전 작성분 — ③④ 수치는 "상품군 충족률"로 읽고, 라이브 실측·생산형태 분리는 **본 §0 + 각 binding A섹션이 권위**.

## 2. 지니 핵심 질문 — 완전 답

### Q1. "가격이 없다 = 하나의 엑셀만 본 건가?" → **맞다. 모든 시트의 가격이 두 소스 중 하나에 존재. 라이브 ❌는 전부 미적재.**

| 가격 소스 | 시트 | 가격 모델 |
|----------|------|----------|
| **① 인쇄상품 가격표 엑셀** | 디지털인쇄·스티커·책자·실사·아크릴·**캘린더 제본가** | 원자합산형 / 형상×사이즈 / 면적매트릭스 / 제본 합산 |
| **② 상품마스터 "가격포함"** | 포토북·문구·굿즈파우치·악세사리·디자인캘린더 | 고정가형(base+옵션+가공+추가) |

> **숨은 발견:** 캘린더는 가격표에 독립 시트가 없어 "없다"고 보였으나, 제본가가 **"제본" 시트 B03 블록**에 녹아 있음(`06_extract/price-binding-l1.csv` r25~32, "삼각대 포함" 합가). 표면 관찰의 함정 — round-2가 이미 추출함.

### Q2. "선택→견적까지 매핑이 고려됐나?" → **그릇·정적매핑·가격소스는 준비됨. 끊긴 곳은 ③CPQ 선택 레이어(전 시트 0) + ④가격 라이브 미적재.**

- **공통 단절점 = ③ `option_items` 전 시트 ≈0** (실사 1/28만). 고객이 고를 선택지가 DB에 없음 → round-6 CPQ 적재가 사슬 완성의 열쇠.
- **④가격 = 실사만 라이브 완결**, 나머지는 소스 추출은 됐으나(round-2) 라이브 미적재.
