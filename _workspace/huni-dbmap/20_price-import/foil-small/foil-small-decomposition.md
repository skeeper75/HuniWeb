## 5. 면적 → 등급 매핑표 처리 판정 (②)

**판정: 면적→등급 매핑표는 t_prc_* 가격 그릇에 직접 들어가지 않는다 = 앱(런타임) 책임 + 별 참조 시트 보존.**

- 근거(메모리 권위 `dbmap-compute-in-app-db-stores-lookup`): **"박 면적→등급 = 앱(런타임) 계산, DB는 등급별 가격만 저장."** off-grid ceiling과 동일 철학.
- component_prices는 **등급(opt_cd) × 수량(min_qty) → 가격**만 담는다. 손님 입력(작업 가로/세로) → 앱이 면적→등급 변환 → 등급을 selections.opt_cd로 엔진에 전달.
- **그럼 매핑표는 어디 보존?** → 가격 그릇이 아니므로 import.xlsx에 **별 참조 시트(`A1_area_grade_map`)** 로 무손실 보존(앱 구현 입력·DDL 제안 대상). 침묵 삭제 금지.
- **round-5 DDL GAP "박 2단룩업" 미해소분 = 이 매핑표의 거처**. 선택지:
  - (a) **앱 상수/설정**(권장·메모리 권위 부합) — DB 미저장, 앱이 면적→등급 룩업 보유.
  - (b) DDL 제안: 신규 `t_prc_foil_area_grade`(siz_w·siz_h·grade) 참조 테이블 — DB 관리 원할 시. decomposition은 (a) 권장하되 별 시트로 (b) 입력도 보존.
- **비대칭 격자**(세로 60/80mm 미정의): off-grid → 한 단계 큰 등급/ceiling 또는 미정의 컨펌(앱 정책).

## 6. 등급(A~E) 차원 라이브 실재 판정 (①)

**판정: 등급 전용 차원은 라이브에 부재 → `opt_cd`(옵션 차원)에 등급 코드를 담아 처리. siz_cd=NULL 강제 안 함.**

- 실측: component_prices 10차원(comp_cd·siz_cd·clr_cd·mat_cd·proc_cd·coat_side_cnt·opt_cd·bdl_qty·min_qty·apply_ymd)에 **grade 컬럼 없음**. OPT_REF_DIM 7종(사이즈/판형/자재/공정/묶음수/도수/셋트)에도 등급 없음.
- 명함박 선례는 등급을 **회피**(STD/HOLO를 comp 분리하고 등급 없이 min_qty만 사용) → 소형박은 등급이 명시 5단계라 회피 불가.
- **등급 담을 차원 = `opt_cd`**:
  - opt_cd는 신설 옵션 차원(현 라이브 0행). 등급 = "박 면적 등급 옵션" = opt 매칭 단위로 정당(엔진은 opt_cd로만 매칭).
  - 등급 코드값(`GRADE_A`~`GRADE_E`)을 `t_cod_base_codes`에 신규 등록 필요 → **코드 선적재 제안**(별 시트·인간 승인).
  - **siz_cd=NULL 강제 금지 준수**: 등급은 사이즈가 아니다(면적→등급은 앱 변환). siz_cd는 NULL(박가공은 작업사이즈로 직접 가격 안 정함).
- **BLOCKED 분리**: 등급 코드값 미등록은 별 시트(`B1_grade_codes_proposal`)에 코드 선적재 제안으로 분리. opt_cd 값 자체는 임시 `GRADE_A`~`E` 표기하되 코드 등록 전 적재 차단 플래그.

## 9. "박 = 세트/고정형 신규구조" 최종 평결

- **부분 YES**: 박은 MATRIX/합산형과 구별되는 **"2단 룩업"** 구조 → round-16이 찾던 신규 구조 후보로 **검증됨**.
- 단 DB 그릇 관점에서 신규성의 본질 = **① 등급(opt_cd) 차원의 첫 실사용**(라이브 0행) + **② 면적→등급 앱 계산 분리**(DB 미저장 룩업).
- "세트/고정형"(여러 구성품 1세트 고정가)은 **아님** — 박은 후가공 1요소(동판셋업 + 등급별 가공비 합산). 세트형 신규구조는 여전히 미발견(다음 후보=제본·봉투결합).
- **신규 발견 = 등급 차원 + 면적→등급 앱 룩업 = round-5 DDL GAP "박 2단룩업"의 정체 규명**.
