## 1. ★ 오적재 근본이 단일 병인으로 수렴 (SOT 1+4)

직전 검증(§13)의 D-1/2/3(이중합산)·D-6(현수막 별색)은 별개 결함처럼 보였으나, 7 SOT로 **단일 병인**임이 확정됐다:

- **병인 (SOT 4)**: `t_prc_formula_components`(공식↔구성요소 배선)에 **`prd_cd`가 없다**(라이브 컬럼=frm_cd·comp_cd·disp_seq·addtn_yn·reg_dt·upd_dt, 상품참조 FK 0). 코드 게이트 0(admin.py:931 clean 없음)·DB 게이트 0(가격 6엔티티 유효성 트리거/CHECK 0). 즉 **"이 구성요소가 이 상품군에서 유효한가"를 강제하는 장치가 어디에도 없다.**
- **올바른 경계 (SOT 1)**: 상품마스터 11시트 = 상품군이 허용하는 차원 경계(SOT). 현수막 시트에 별색·접지가 없으면 그건 현수막의 차원이 아니다.
- **증상 (직전 F-3)**: 엔진이 배선된 구성요소를 무차별 합산(addtn_yn 미사용·"판별차원 없음 항상 매칭" pricing.py:415·NULL 와일드카드). 제약이 없으니 광역 공유 comp(`COMP_PP_VARIMG_1EA`=38상품·`COMP_PRINT_SPOT_WHITE_S1`=29공식)가 **silent 합산**된다.
- **부분 선례**: `fn_chk_opt_item_ref`(sql/10:189-236)는 정확히 상품-스코프 유효성을 강제하나 **CPQ option_items 전용 — 가격 공식 배선엔 복제 안 됨.** 후니가 옵션엔 만든 장치를 가격 배선엔 안 만들었다.

→ **U-7 [최우선 신규 미지]: 배선 레벨 제약 장치(comp↔상품 허용 경계 강제)를 어떻게 설계할 것인가.** 오적재 재발 방지의 핵심. fn_chk_opt_item_ref를 가격 배선으로 확장하는 방향이 유력(설계+검증 트랙).

## 2. 구성요소 도출 원리화 (SOT 2+3)

- **결합형 vs 독립형**: 같은 가격테이블 내 **같은 차원** = 결합(디지털인쇄비 = 국4절 1도/4도 + 3절). 차원 비공유 = 독립(별색은 국4절에만 → 별색인쇄비 별도, 색별 결합). 이게 가격구성요소 도출의 권위 규칙.
- **10차원 확정** (SOT 3): 판형사이즈·사이즈·인쇄옵션·묶음수·자재·공정·옵션코드·코팅면수·사이즈가로(구간)·사이즈세로(구간). **clr_cd 없음** → 직전 F-2(도수=print_opt_cd, clr_cd 폐기)를 **SOT 권위로 추인**(U-1 폐기 의도 확정 방향).

## 3. 옵션그릇·추가상품은 가격 장치가 아니다 (SOT 3+6)

- **옵션그릇** = UI 선택 + 생산 BOM(자재/공정/자재+공정/자재+자재). `option_items`에 **add_price 컬럼 부재**·qty=BOM 소요량만 → CPQ→가격 단절은 결함이 아니라 **설계 정합**. opt_cd near-dead(5행)도 정합.
- **수량×단가 (SOT 6) 판정**: 가격구성요소에 "수량×단가" 전용 차원 신설 **불요**. 두 갈래:
  - (a) **추가상품(addon)** = 별도 SKU 재귀평가. `t_prd_product_addons`(가격 없음)→`t_prd_templates`(base_prd_cd+dflt_qty)→시뮬레이터가 개별 `evaluate_price` 호출로 합산. 메인 10차원 **밖**. SKU 단가 그릇 `t_prd_template_prices`=현재 0행.
  - (b) **부자재/악세사리** = 10차원 ⑤자재·⑥공정 + 수량축(min_qty/bdl_qty)으로 흡수 가능 `[추정]`. → U-6(수량=주문수량/BOM소요량/출력매수?) 선결.

## 4. 가격소스 "직접단가/가격공식" 의도 (SOT 5)

- 우선순위 **LOCKED**(11-CONTEXT.md:43): 템플릿단가 → 직접단가 → 공식 → 없음. 직접단가 있으면 공식보다 우선(pricing.py:312 else-skip).
- 직접단가 행 = `(적용일, 단가)` **차원 없는 단일 스칼라** → **완제품 고정가 오버라이드용**(가설 A·확신 높음) + 공식결함 긴급 안전판(가설 B).
- 0행 = "죽은 코드"가 아니라 **완제품/기성품 SKU 적재 시 활성화될 예비 그릇**. → 가격 검증은 데이터 있는 **FORMULA + 수량구간할인**에 집중해야 유효.
