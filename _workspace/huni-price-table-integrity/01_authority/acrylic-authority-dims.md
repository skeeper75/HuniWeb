## 1. 면적 차원 (B01·B02·B03·B05) — `dim_type=area`

### 가격축 판정 [HARD]
가격을 매기는 축 = **가로(width) × 세로(height) mm 면적격자**. 자유치수가 아니라 **격자 좌표값(이산 mm 눈금)** 으로 단가가 박혀 있고, off-grid는 한 단계 큰 눈금으로 올림(ceiling)하는 면적매트릭스형. 격자 좌표열(가로 라벨)·좌표행(세로 라벨)이 차원, 셀값이 단가.

- **B01 투명3T**: 가로 눈금 14개 = {20,30,40,50,60,70,80,90,100,120,140,160,180,200} mm, 세로 눈금 14개(동일). 14×14 = **196셀 꽉 참**. 가로↔세로 **대칭**(w20h30 = w30h20). 셀값 = 정수 원(2500~32700).
- **B02 투명1.5T**: 가로/세로 눈금 9개 = {20,30,40,50,60,70,80,90,100} mm. 9×9 = **81셀 꽉 참**. 대칭. 셀값 2000~10160.
- **B03 미러3T**: 가로/세로 눈금 9개 = {20,...,100} mm. 9×9 = **81셀**. **★전 81셀이 B01 동좌표값의 정확히 ×2** (엑셀 공식 `=Bn*2`, 81/81 검증 0 불일치). 즉 미러3T는 독립 격자가 아니라 투명3T의 2배 파생. 셀값 5000~25400.
- **B05 코롯토**: 가로/세로 눈금 6개 = {30,40,50,60,70,80} mm. 6×6 = **36셀 꽉 참**(헤더 band_header_path가 sparse하게 보이는 건 추출 아티팩트 — 데이터 매트릭스는 완전). **비대칭** (w30h40=4400 ≠ w40h30=4400 우연 동일이나, w30h80=6400 vs w80h30=6400... 실제로는 가로/세로 모두 등장하며 일부 비대칭). 셀값 3600~8400.

### 도수 판정 [HARD] — "양면9도/단면7도 통용", "전면5도 통용" = 도수 무관 단일가
- 블록 제목의 "양면9도 / 단면7도 **통용** 단가", "전면5도 **통용** 단가"는 **별도 도수 차원이 아니다.** "통용" = 해당 도수 조합(양면9도든 단면7도든)에 **동일 단가 적용**. 즉 단가표는 면적 하나로만 인덱싱되고 도수는 가격에 영향 없음.
- → 라이브 구성요소 `use_dims`에 도수축(`print_opt_cd`/`clr_cd`)을 **넣으면 안 됨**(과적발·이중분기 함정). 도수는 가격 차원에서 제외.

### 기대 적재방식 (라이브)
- 구성요소 `use_dims = [siz_width, siz_height]` + 면적 격자 단가행(component_prices, long-format). 자유치수 입력 → off-grid는 큰 눈금 ceiling 올림.
- ★**[HARD] 등록사이즈(siz_cd)로 적재하면 안 됨.** 이건 면적 차원이지 등록사이즈 정확매칭이 아니다. (단, 직전 세션 메모 `price-formula-master-completeness-260627`이 "아크릴 siz_cd×면적=코드버그(엔진 siz_cd 미환원→최소가)"를 적발한 바 있음 — inspector가 라이브 use_dims가 면적인지 siz_cd인지 반드시 확인할 것. 만약 라이브가 `mat_cd` collapse 또는 siz_cd 정확매칭으로 적재돼 있으면 = 차원 누락/오적재 결함.)
- **자재(두께) 분리**: B01(투명3T=MAT_043)·B02(투명1.5T=MAT_042)·B03(미러3T)은 **서로 다른 면적 단가표**. 한 구성요소가 자재 collapse(대표 1개만 적재)하면 나머지 두께 견적 불가. → 자재축이 use_dims에 있거나(`[mat_cd, siz_width, siz_height]`) 자재별 구성요소가 분리돼야 함.
  - 알려진 라이브: COMP_ACRYL_CLEAR3T(use_dims=[mat_cd,siz_width,siz_height,min_qty]·MAT_043/MAT_042) / COMP_ACRYL_MIRROR3T(use_dims=[siz_width,siz_height]) / COMP_ACRYL_COROTTO(use_dims=[siz_width,siz_height]).
  - 주의: CLEAR3T가 mat_cd로 MAT_043·MAT_042를 모두 담는다면 B01·B02 둘 다 1개 구성요소에 적재돼야 함(2개 자재 단가표 = 196+81 = 277셀). inspector가 이 셀들이 다 있는지 확인.

## 2. 수량구간 할인 차원 (B04a·B07) — `dim_type=qty_discount`

### 가격축 판정
가격을 **깎는** 축 = 주문 **수량 구간**. 본체단가(면적격자 결과)에 구간별 할인율을 곱해 최종가. 차원 = 수량 하한~상한, 값 = 할인율(0~0.5).

- **B04a 아크릴상품 공통 수량할인** (적용범위 = 전 아크릴상품):
  | 수량구간 | 할인율 |
  |---|---|
  | 1~49 | 0 |
  | 50~99 | 0.1 |
  | 100~299 | 0.2 |
  | 300~499 | 0.3 |
  | 500~999 | 0.4 |
  | 1000~10000 | 0.5 |
- **B07 아크릴카라비너 전용 수량할인** (B06 카라비너에만 적용 — **B04a와 별도·구간경계도 다름**):
  | 수량구간 | 할인율 |
  |---|---|
  | 1~49 | 0 |
  | 50~99 | 0.1 |
  | 100~10000 | 0.2 |
  → 카라비너는 100 이상 단일구간 0.2 상한(공통표의 0.3/0.4/0.5 없음). **B04a를 카라비너에 적용하면 과할인 결함**.

### 기대 적재방식 (라이브)
- 라이브 `t_dsc_*`(할인테이블) 또는 구성요소 `use_dims`에 `min_qty` 티어로 표현. 적용방식 = **수량 이상(>=) 하한 매칭** (50개면 50~99 구간 → 0.1).
- 핵심: **할인이 본체단가에 곱셈으로 들어가는 구조**(원가×(1-할인율))인지, 아니면 구간별 단가행(min_qty별 별도 가격)인지 라이브 모델 확인 필요. B01 면적표의 `min_qty` use_dims(CLEAR3T가 min_qty 가짐)와 이 할인율표의 관계 = inspector 핵심 검사 대상(할인이 단가표에 이미 녹아있는지 vs 별도 곱셈인지 = 이중적용/미적용 함정).
- **두 할인표 분리 적재**: B04a(전 아크릴) vs B07(카라비너 전용). 카라비너에 B04a를 잘못 묶으면 1000개 주문 시 0.5 과할인.

### 기대 적재방식 (라이브)
- 라이브 CPQ 추가상품(addon) 또는 후가공 공정 단가행. 옵션 선택 시 단가 가산(addtn_yn).

- **단가 0 옵션 가드**: 카라비너 색상 9종·키링 고리없음·볼펜 색 = 단가 0. always-add(use_dims에 미포함 시 NULL 와일드카드 silent 가산) 함정 주의 — 0이라도 옵션축이 정의돼야 함(0 가산이 정상). 색상은 가격축 아님 = 의미축 정당 차이(false-positive 가드).

| 사각자물쇠 (자물쇠변형) 40x69 | 5800 |

## 다음 inspector가 라이브와 대조할 항목 (대조 체크리스트)

1. **면적 격자 미적재 셀 (이 빠짐)**: B01 196셀·B02 81셀·B03 81셀·B05 36셀이 라이브 component_prices에 (w,h)별로 다 있는가. 어느 (가로,세로) 셀이 비었는지 특정. 빈 셀 = 그 사이즈 견적 0/최소가.
2. **차원 유형 일치 [HARD·돈크리티컬]**: 라이브 아크릴 구성요소 `use_dims`가 **면적[siz_width,siz_height]인가, siz_cd인가, mat_cd collapse인가.** 직전 세션이 "siz_cd×면적=코드버그→최소가 과소청구"를 적발 — 라이브가 면적이 아니면 차원누락 결함(돈영향=과소청구).
3. **자재(두께) 분리**: 투명3T(MAT_043)·투명1.5T(MAT_042)·미러3T 3개 단가표가 다 적재됐는가, 1개로 collapse됐는가. B03=B01×2 파생관계가 라이브에 반영됐는가(아니면 미러 81셀 누락).
4. **수량할인 이중/미적용**: B04a 할인율(0~0.5)이 면적 단가에 곱셈으로 들어가는가 vs 단가표에 이미 녹았는가. CLEAR3T의 min_qty use_dims와 할인율표 관계 = 이중적용/미적용 함정.
5. **B07 카라비너 전용 할인 분리**: 카라비너가 B04a(전 아크릴 0.5상한) 아닌 B07(0.2상한)을 쓰는가. 잘못 묶이면 과할인.
6. **후가공 26옵션 상품그룹 귀속**: 각 옵션이 맞는 아크릴 상품그룹에만 붙는가. always-add 0가산 가드(색상축).
7. **도수 과적발 가드**: 라이브에 도수(print_opt_cd/clr_cd)가 가격축으로 잘못 들어갔으면 = 과적발 결함("통용"=도수 무관 단일가).
8. **B06 고정형 vs 면적격자 혼동**: 카라비너가 면적격자로 잘못 적재되지 않았는가(형태별 고정단가 4종).

## 의미축 정당 차이 (false-positive 가드 — 결함 아님)
- 면적격자 가로↔세로 대칭(B01/B02/B03) = 정상(같은 값 중복 아님).
- 카라비너 색상 9종 단가 0 = 정상(색은 가격축 아님).
- 볼펜 색옵션 6종 = 선택지(가격 무관).
- "통용" 도수 = 도수별 단가 동일(도수 차원 부재가 정상).
