## 1. 결함 D-1b — 후가공 prc_typ 메타 오적재 (P1·돈-크리티컬)

### 1-1. 사실 (라이브 실측 + 가격표 권위)

- `prc_typ_cd`는 `t_prc_price_components`(comp 정의 헤더)에 있고 **라이브 전 comp가 `PRICE_TYPE.01` 단일값** = 적재 시 prc_typ를 일괄 .01로 넣음(메타 일괄 오적재). 단가행 값 자체는 가격표 전건 일치(round-16 216/216·d1-plate-conversion §5-3 재확인) = **값 손상 0, 메타데이터만 오적재**.
- 가격표 `인쇄후가공` 시트 헤더에 작성자가 **"합가" 명시**(A1 모서리·A15 오시·F15 미싱) = 셀 값이 장당가 아닌 **수량구간별 고정 총액**. 판별 = **ⓒ 구간고정총액형**(수량 무관 1회 부과). ⓐ단가형·ⓑ명세식 합가형(÷mq×qty) 모두 기각(C4 250매 단조성 위반 입증·[[PRF_DGP_A/d1-plate-conversion]] §5-2).
- 엔진이 .01(단가형)로 해석하면 `unit_price × 주문수량` → 미싱1줄 100매 = 10,000×100 = **1,000,000원**(과대 990,000). ⓒ 정정 시 = 10,000(1회).

### 1-2. D-1b 22 comp 전수 영향표 (라이브 단조성 실측·down_steps=0 = 총액형 시그니처)

### 1-3. 정정 방식 — 대안·트레이드오프·권고

> 정정 대상 = `t_prc_price_components.prc_typ_cd` 메타값(단가행 `unit_price` 불변 — 값은 정확). [[PRF_DGP_A/remediation-plan]] §3.3·[[PRF_DGP_A/d1-plate-conversion]] §5-4 재사용.

| 방식 | 무엇 | 장점 | 단점 | 권고 |
|------|------|------|------|:--:|
| **ⓒ-1** | 그룹① `prc_typ_cd`를 `.02(합가형)`로 변경 + **엔진 합가규칙 "매칭 구간총액 그대로(÷mq×qty 없음)" 정의** + 11-CONTEXT §16 명세에 "구간고정총액형" 케이스 보강 | DDL 0(기존 코드 재사용)·UPDATE만 | 라이브 `.02`의 기존 의미(명세식 ÷mq×qty)와 충돌 → 엔진이 두 합가형을 구분해야(명세 보강 필수). 기존 .02 사용 comp 있으면 회귀 위험 | △(엔진 명세 의존) |
| **ⓒ-2** | 신규 `PRICE_TYPE.03(구간고정총액형)` base_code 1행 + 그룹① prc_typ를 .03으로 변경 + 엔진 .03 분기 | 의미 명확(.01 단가/.02 명세합가/.03 구간총액 3분리)·기존 .02 무회귀 | base_code 1행 추가(경량 DDL)·엔진 분기 1개 추가 | **○ 권장** |

**권고 = ⓒ-2.** 근거: ⓒ-1은 `.02`에 두 거동(명세식·구간총액)을 욱여넣어 엔진이 분기 못 하면 명세식 ÷mq×qty가 적용돼 **새 오류(C4 250매 단조성 위반·d1-plate §5-5)**를 부른다. ⓒ-2는 base_code 1행(라이브 PRICE_TYPE 동형·충돌 0)으로 의미를 명시 분리하고 기존 .02 사용처를 건드리지 않는다(무회귀). **단 ⓒ-1/ⓒ-2 모두 엔진(evaluate_price) 구현이 그 prc_typ를 올바로 해석해야 실효** → webadmin Phase11 범위(우리 밖). DB측 정정(prc_typ 메타 UPDATE)은 round-13 정정 트랙.

> **트레이드오프(큐 적재 vs 즉시 적용):** 엔진(evaluate_price) 미구현·가격뷰어 비활성 = **실청구 위험 0**(데이터는 틀린 채로 남으나 청구 안 됨). → **큐 적재 권고**(적용 시점 유예 안전). 즉시 적용 요구는 전체 정지 유발. 단 적용 시 **prc_typ 정정과 엔진 규칙을 동시 배포**해야(메타만 .03 바꾸고 엔진이 .03 모르면 미정의 동작).

### 1-4. 멱등 UPDATE 제안 형태 (round-13 트랙·실행은 인간 승인)

```sql
-- ⓒ-2 채택 가정. base_code 선적재 후:
-- 1) base_code: cod_cd=PRICE_TYPE.03 · cod_nm=구간고정총액형 · disp_seq=3 (라이브 .01/.02 동형)
-- 2) 그룹① 후가공 .04 comp 13종 prc_typ 정정 (멱등: 이미 .03이면 no-op)

SET prc_typ_cd = 'PRICE_TYPE.03', upd_dt = now()
WHERE comp_cd IN (
  'COMP_PP_CREASE_1L','COMP_PP_CREASE_2L','COMP_PP_CREASE_3L',
  'COMP_PP_PERF_1L','COMP_PP_PERF_2L','COMP_PP_PERF_3L',
  'COMP_PP_VARTEXT_1EA','COMP_PP_VARTEXT_2EA','COMP_PP_VARTEXT_3EA',
  'COMP_PP_VARIMG_1EA','COMP_PP_VARIMG_2EA','COMP_PP_VARIMG_3EA',
  'COMP_PP_CORNER_ROUND'
  -- CUT_PERF_1H6은 가격표 권위 재확인 후 포함(타공=후가공 .04·합가 추정·미확정)
) AND prc_typ_cd <> 'PRICE_TYPE.03';
-- 그룹② .06/.05(박·완칼)은 [CONFIRM:D1b-06] 해소 후 별도 UPDATE
```

> 단가행(`unit_price`) 절대 변경 금지(값은 정확). CORNER_RIGHT(0원) 제외. CUT_PERF_1H6은 가격표 타공 시트 "합가" 라벨 직접 확인 후 포함(현재 미확인 → 컨펌 분리).

## 2. 결함 WIRE — 가격사슬 배선 미완 (P1·돈-크리티컬)

### 2-a. NAMECARD-WIRE — 고정가형 명함종 comp 미배선 (명함 031/032)

**라이브 실측(2026-06-15):** `PRF_NAMECARD_FIXED`에 배선된 comp = **COMP_NAMECARD_STD_S1/S2 단 2개**. 명함종 comp 25개 중 23개(PREMIUM/COAT/FOIL/PEARL/WHITE/CLEAR/SHAPE/MINISHAPE) **전부 MISSING-WIRE**. 단가행은 L1 정합·정확(STD 074=3,500·PREMIUM A=4,500·COAT 아트250=5,500·d1-plate 외 namecard quote-gate §Q4).

| 영향 | 현 거동 | 정정 후 |
|------|--------|--------|
| 031 프리미엄 | PREMIUM 미배선 → STD만 → 031 16소재 ∩ STD{074,082}=∅ → **전 견적 단절** | PREMIUM_S1/S2_MGA/MGB 배선 → 4,500 룩업 |
| 032 코팅 | COAT 미배선 → STD 떨어짐 → 아트300=3,800(코팅 5,800이어야·**2,000 과소 오산**)·아트250 단절 | COAT_S1/S2 배선 → 5,800 룩업 |

**정정 = round-16 배선**(COMP_NAMECARD_PREMIUM/COAT/PEARL 등을 PRF_NAMECARD_FIXED에 formula_components INSERT). **comp/단가 신설 불요·배선만**.

### 3-2. 핵심 트레이드오프 — 명함 FOIL 재사용 가능 여부

명함 FOIL은 **"종이포함 완제품가(합가)"**라 042(종이 별도·박만 add-on)에 **그대로 못 쓴다**. 그러나 **박 가공비 본질 구조(면적등급 A~E × 수량 + 동판셋업 5,000)는 동일**(가격표 후가공_박(소형) 시트 = 명함·042 공통 권위).

| 대안 | 무엇 | 장점 | 단점 | 권고 |
|------|------|------|------|:--:|
| **ⓐ 공용 박가공비 comp 신설** | `COMP_FOIL_AREA_*`(면적등급×수량 순수 박가공비, 종이 미포함) + COMP_FOIL_SETUP(동판 5,000) 신설 → 042 PRF_DGP_A 배선 + (명함은 별도) | 042 박 add-on 의미 정합(종이별도)·면적등급 차원 복원·명함 박과 권위 공유 | 명함 FOIL(종이포함 합가)과 별 comp → 명함은 기존 FOIL 배선·042는 신설 comp = 완전 통일 아님 | **○ 권장(042)** |
| ⓑ 명함 FOIL 그대로 042 재사용 | 명함 COMP_NAMECARD_FOIL_*을 042에도 배선 | 신설 0 | 042=종이별도인데 명함 FOIL=종이포함 → **042 박값에 종이 이중계상(오산)** | ✗ 기각(돈 오류) |
| ⓒ 042 박을 명함처럼 완제품가 통합 | 042도 박포함 완제품 단가표 신설 | — | 042 박은 add-on(택N)이라 완제품 통합 부적합·옵션 구조 위반 | ✗ 기각 |

**권고 = ⓐ.** 박 가공비를 종이와 분리한 공용 comp(`COMP_FOIL_AREA_<색>` 면적등급×수량 2D + `COMP_FOIL_SETUP` 동판 5,000)를 가격표 후가공_박(소형) 시트 권위로 신설 → 042 배선. 명함 031은 기존 FOIL(종이포함 합가) 배선 유지(명함 단가표가 이미 종이포함이라 정합) **또는** 동일 공용 comp로 재모델(명함도 종이/박 분리 — 단 명함 가격표가 종이포함이면 불요).

> **결론 = 부분 동시 해결.** "박 가공비 + 동판셋업 구조"는 명함·042 **공유 권위**로 한 번에 설계하나, **명함 FOIL이 종이포함 완제품가라 042에 단순 재사용은 불가**(종이 이중계상). 042는 종이별도 공용 박가공비 comp 신설이 정답. → **명함 031 박 = 배선만**(기존 comp), **042 박 = 공용 박가공비 comp 신설 + 배선**. 두 작업을 round-2/16에서 **한 박 가격구조 결정**으로 묶어 처리(권위·면적등급·셋업비 일관). `[CONFIRM:WIRE-3]`: 명함 가격표가 박 포함 시 종이를 포함하는지(완제품가 가정 검증) + 042 박 면적등급 단가가 명함과 동일 시트인지.
