# GOAL — round-7 입체 커버리지 검증 (상품마스터 전 시트 × 라이브 t_* 엔티티)

> 작성 2026-06-08. 본 문서는 huni-dbmap 하네스 **round-7**의 권위(Definition of Done + 평가 게이트)다.
> 이전 라운드 권위: `docs/goal-2026-06-06-01.md`(round-4)·`docs/goal-2026-06-06-02.md`(round-5).
> 본 문서는 그것들을 대체하지 않고 **새 검증 축(입체 커버리지)을 추가**한다.

## 1. 문제 정의 (사용자 피드백)

round-1~6은 **시트별 종단**(silsa·postcard·goods 등 개별 상품군 깊이 파기)으로 진행됐다.
그 결과 `03_validation/`에 20여 게이트가 쌓였지만 **전부 특정 트랙/상품군 단위**이고,
**"상품마스터 전 상품군에 걸쳐 각 상품에 필요한 요소가 라이브 DB에 빠짐없이 들어갔는가"**를
한 판에서 조망하는 축이 없다. 시트 하나에 집중할수록 전체 조망이 사라졌다.

> 사용자 원문: "하나의 시트에 집중하니 전체적으로 조망하는 부분들이 빠져있습니다.
> 전체적인 시트를 중심으로 설계를 해서 입체적인 접근이 필요."

## 2. 이번 라운드 목표 (WHAT)

상품마스터 **전 상품군(11시트)** × 라이브 **t_\* 엔티티/속성/관계**의 **3차원 커버리지 매트릭스**를
구축해, 각 상품에 **필요한 요소가 제대로 적재됐는지**와 **무엇이 미적재인지**를 한 판에서 입증한다.

- **축 1 (행)** = 상품마스터 상품군 시트 11종 (디지털인쇄·스티커·책자·포토북·캘린더·디자인캘린더·실사·아크릴·문구·굿즈파우치·상품악세사리)
- **축 2 (열)** = 상품 관련 t_\* 엔티티 (상품정보·사이즈·자재·인쇄옵션·공정·판형·묶음수·페이지룰·셋트·추가상품·카테고리·CPQ 옵션레이어·가격공식바인딩·할인테이블바인딩)
- **축 3 (셀 깊이)** = 그 (상품군 × 엔티티) 교차의 **속성 정합 + 적재 상태 + 관계 무결성**

## 3. 검증 권위 (무엇을 정답으로 보는가)

| 우선순위 | 권위 | 용도 |
|---------|------|------|
| 1 | **상품마스터 엑셀 명시값** | 그 상품이 "필요로 하는 요소"의 정의 = 검증 정답 (추측 금지) |
| 1 | **가격표 엑셀 명시값** | 가격/사이즈/옵션 추가가 권위 ([[dbmap-l2-requires-l1-price-table]]) |
| 2 | **라이브 DB (psql, 읽기전용)** | 실제 적재 상태 = LOADED/PARTIAL/MISSING 판정의 사실 근거 |
| 2 | **라이브 admin product-viewer** | t_\* 엔티티의 **역할 UI ground-truth** (12편집탭=12 t_prd_product_*) ([[dbmap-live-admin-product-viewer]]) |
| 3 | 테이블 명세 `docs/huni/table-spec_260608.html` | 스키마 구조(컬럼·FK·코드값) 참조 |

충돌 시 상위 권위가 이긴다. **추출본/CSV는 권위가 아니다** (stale 주의 — [[dbmap-no-db-load-file-first]]).
DB 조회는 **읽기전용 SELECT만**, 라이브 admin은 **대표상품+미적재 의심분 집중**(gstack browse).

## 4. 완료 기준 (C-Gates) — 평가 게이트

평가자(`evaluator-active`, fresh context)는 아래 C1~C8을 독립 채점한다. **전건 PASS = GO**, 1건 FAIL = NO-GO.

- **C1 매트릭스 완전성** — 11 상품군 × 전 t_\* 엔티티의 **모든 셀이 채워짐**. 해당 없는 교차는 `N/A`로 **명시**(공백 금지). 누락 셀 0.
- **C2 필요요소 도출 정확성** — 각 상품군의 "필요 요소"가 **엑셀 명시값에서** 도출됐고 출처(시트/행/컬럼)가 인용됨. 도메인 추론·번호연속 추측 0 ([[dbmap-print-method-not-absolute-axis]]).
- **C3 적재 상태 실측** — 각 셀의 상태(LOADED/PARTIAL/MISSING/N/A)가 **라이브 DB 실측 행수**로 뒷받침됨. 추출본 단독 판정 0.
- **C4 3원 대조** — 각 상품군 **대표상품 1+ & 미적재 의심분**에 대해 엑셀↔DB↔admin UI **일치 검증**. 불일치는 증거(스크린샷/행)와 함께 기록.
- **C5 관계 무결성** — 엔티티 간 FK 그래프 정합(자식→부모 고아 0), CPQ polymorphic 참조(`ref_dim_cd`+`ref_key`)가 라이브 차원행으로 해소됨([[dbmap-cpq-option-layer-mapping]]).
- **C6 미적재 갭 보드 정직성** — 모든 MISSING/PARTIAL이 **원인분류(차단유형: 코드행 미적재·siz 미등록·자재 mint·DDL 부족·차원 미적재·도메인 미결정 등)** + **라우팅**으로 분리. over-report(허위 누락)·under-report(은폐) 0. 차단은 발명 없이 BLOCKED/GAP-DEFER로.
- **C7 생성-검증 독립성** — 매트릭스 작성(`dbm-coverage-auditor`) ≠ 평가(`evaluator-active`). 평가자가 **실결함 ≥1** 적발(무결함이면 검증이 얕다는 신호).
- **C8 재현성** — 방법론 스킬(`dbm-coverage-matrix`) + 추출 스크립트로 다음 세션 재실행 가능. 산출물 경로·실행법 명시.

## 6. 경계 (NOT in scope)

- **DB 직접 적재(COMMIT) 없음** — 본 라운드는 **검증/조망 전용**. 발견된 미적재의 실제 적재는 인간 승인(기존 원칙 유지).
- 매핑 규칙 **재설계** 없음 — 기존 GO 매핑은 검증 대상이지 재작성 대상 아님. 결함 발견 시 갭 보드에 라우팅만.
- 신규 DDL **적용** 없음 — GAP은 `dbm-ddl-proposer`로 라우팅(제안까지), 적용은 인간 승인.

## 7. 평가 통과의 의미

C1~C8 GO = "상품마스터 전 상품군에 대해, 각 상품에 필요한 요소가 라이브 DB에 무엇이 들어갔고 무엇이
안 들어갔는지가 한 판에 입체적으로·정직하게·재현가능하게 입증됐다"는 뜻. 이것이 사용자가 요구한
"누락 없는 전체 조망"의 Definition of Done이다.
