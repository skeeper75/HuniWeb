## 핵심 발견 한 줄

라이브 plate는 **압도적으로 작업사이즈(cut+bleed)를 담으며**(round-3 결론 유효), 출력용지규격(전지)은 **별도 컬럼/테이블이 아니라 `t_siz_sizes`의 일부 행에 `impos_yn=Y`로 혼재**한다. 그러나 전지가 plate에 실제로 등록된 상품은 **`프리미엄엽서` 단 1건뿐(오등록)**이고, 전지 마스터 자체도 **국4절·반칼전지 2종만 존재**하며 나머지 5종(3절·소형반칼전지·스티커팩·썬캡·투명전지)은 부재다. `output_paper_typ_cd`는 **국전/46 계열 분류 의도가 사실상 미실현**(.02 전무, .01 1건, .03이 임의 1행 표식으로 산재)이다.

> ⚠️ round-3 보정: round-3은 "전지값(316x467 등)이 t_siz에 아예 없다"고 했으나(plate-size-parity.md L16), 라이브 재확증 결과 **316x467·317x440은 `impos_yn=Y`로 존재**한다(SIZ_000499/SIZ_000498). round-3은 작업사이즈 행만 검색해 전지 행을 놓친 것. **전지 = `impos_yn=Y` & work 치수가 전지치수인 행**으로 식별된다.

## A. 라이브 plate가 담는 것 (494행)

| 구분 | 행 수 | 근거 |
|------|:-----:|------|
| **작업사이즈** (cut+bleed, per-item) | **493** | siz_nm/work 치수가 판걸이수 시트 `작업사이즈`(D열)와 정합. 예 접지리플렛 `424x303`·코팅명함 `92x52`·탁상형캘린더 `224x149`. `output_file_typ`(PDF/JPG/AI) 짝 → 출력파일 규격축 |
| **전지(출력용지규격, press sheet)** | **1** | 프리미엄엽서 `SIZ_000499`(316x467, impos_yn=Y) — **유일**. 작업사이즈 행 없이 전지 1행만 보유 = 오등록 |
| 기타(siz 부재 NULL 조인) | 0 | 494행 전부 t_siz 조인 성공 |

**근거(실측):**
- 프리미엄엽서 plate = `[316x467]` 단 1행, `impos_yn=Y`, `output_paper_typ_cd=.01`, **작업사이즈 행 0개**.
- 동일 엽서군 형제 스탠다드엽서 plate = `[75x100, 100x100, 102x152, 137x137, 150x212]` 5행 전부 작업사이즈(impos_yn=N) — **정상**.
- `t_siz_sizes` 구조가 작업사이즈축임을 재확인: `work_width/height`(블리드 포함) + `cut_width/height`(재단) + `margin_*`(여백) + `impos_yn`(임포지션 참여 여부). 즉 plate→siz는 작업사이즈 모델이 설계 정본.

**`impos_yn` 의미 주의(오버로드):** `impos_yn=Y` 79종은 "임포지션에 참여하는 사이즈" 전부를 표시 — 진짜 전지(316x467 등)와 일반 작업사이즈(100x150 등)가 섞여 있다. 진짜 전지의 판별자는 `impos_yn=Y` **AND** work 치수가 전지치수(316x467/317x440 등). 단독으로 전지 마스터 플래그로 쓰기엔 불충분(추후 정의 필요분, 🟡).

## D. output_paper_typ_cd 미분류·산재 진단

분포(라이브): **NULL 394 / .03(기타) 99 / .01(국전계열) 1 / .02(46계열) 0**.

| 코드 | cod_nm | 사용 | 진단 |
|------|------|:----:|------|
| OUTPUT_PAPER_TYPE.01 | 국전계열 | 1행 | 프리미엄엽서 316x467에만. 전지를 국전계열로 분류한 유일 사례(그러나 그 자체가 오등록 행) |
| OUTPUT_PAPER_TYPE.02 | 46계열 | 0행 | **완전 미사용** |
| OUTPUT_PAPER_TYPE.03 | 기타 | 99행 | 임의 산재 |
| NULL | — | 394행 | 미분류 |

**핵심 결함 — `.03`이 의미 있는 분류가 아님(노이즈):**
- MIXED 상품 17종에서 **동일 목적·형제 작업사이즈 행 중 정확히 1행만 `.03`, 나머지는 NULL**. 예:
  - 스탠다드엽서: `75x100`만 .03, 나머지 4행(100x100·102x152·137x137·150x212) NULL
  - 코팅엽서: 1행 .03 / 6행 NULL · 투명엽서 1/3 · 2단접지카드 1/5 · 반칼 자유형 스티커 1/2 …
- 즉 `.03`은 "기타 전지계열"이라는 의미가 아니라 **행 단위로 우발적으로 찍힌 표식**. 출력용지규격(전지) 계열(국전 .01 / 46 .02)을 구분하려는 원래 의도가 **사실상 미실현**.
- NULL 394행은 작업사이즈 행의 정상값에 가까움(전지 분류 대상이 아니므로) — "미분류"라기보다 "분류축이 작업사이즈에 부적합".

**판걸이수 권위로 채울 수 있는가?**
- 판걸이수 시트는 각 작업사이즈가 어느 전지(316x467 국4절 / 330x660 3절 / 330x470·307x430·317x440 반칼계열 / 330x540 썬캡)에 앉는지 단일 권위 보유.
- 계열 매핑 후보: 316x467→국전계열(.01) / 330x660→? (국전 3절도 국전계열일 개연 🟡) / 330x470·317x440·307x430(반칼/스티커전지)→.03 또는 신규계열 / 46계열(.02)은 판걸이수 시트에 해당 전지 부재 → **.02는 채울 데이터 없음**.
- 단, 이는 **작업사이즈 행에 "이 작업물이 어느 전지에 앉는가"를 태깅**하는 것이라 `output_paper_typ_cd`의 현 행단위 의미와 충돌 가능. 정의 확정 선행(🟡).
