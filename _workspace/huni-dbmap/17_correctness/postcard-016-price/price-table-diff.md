### 4-1. 분류 = "축오적재" 이유 (값은 맞는데 왜 문제인가)

- 가격표 오시/미싱/가변 = **`(합가)` 명시 + `제작수량` 축**. 값이 **수량구간별 고정 총액**이다(100매=10,000은 100매 작업 1건의 총액, 장당가 아님).
- 라이브 단가행 unit_price = 이 총액을 **그대로** 넣음(값 정확). note도 `제작수량≥N (합가...)`로 축 정확히 기록.
- **그러나 prc_typ_cd = PRICE_TYPE.01(단가형)** 으로 적재 → 가격엔진이 단가형 규칙(unit_price × 주문수량)을 적용하면 `10,000 × 100 = 1,000,000`으로 **100배 과대**.
- 정답 메타 = 구간고정총액형(`.02` 재사용 또는 신규 `.03` — root-cause §3-1·C-D1b). → correction-manifest M-1.

### 4-2. off-grid 누진룰 + 100000 표지행 (보정 — 재판정)

가격표 `인쇄후가공` 시트는 격자 끝에 off-grid 누진룰을 둔다:
- `A12 = "100장당 1000원씩 올립니다."` (모서리·5000매 초과)
- `A27 / F27 = "1,000장당 20,000원씩 올립니다."` (오시 / 미싱·5000매 초과)
- `A57 = 100000.0`, **B57~K57 = 전부 None** (가변텍스트/이미지) → 값 없는 **누진룰 경계 표지 행**(단가행 아님)

라이브 실측: 모서리/오시/미싱 max min_qty=5000에서 끊김, 가변 max=9500. 누진룰·100000 표지는 라이브에 미반영.
- **100000 행은 미적재가 정당**(값 None을 복제 INSERT하면 잘못된 데이터 mint) → correction-manifest **C-6**(CORRECT 재판정·직전 M-G1 복제 제안 철회).
- 진짜 결함 = **off-grid 누진룰 처리 부재**(5000매 초과 과소청구) → correction-manifest **M-G2**(MISSING). 격자 복제가 아니라 런타임 누진 계산식(토대 §3).

### 4-3. 둥근모서리 / 직각모서리 (참고·우려 8종 외)

- 가격표 `인쇄후가공!A1="모서리 (귀돌이, 합가)"`, C3=2000(둥근)·B3=0(직각).
- 라이브 CORNER_ROUND: 1→2000, 300→4000, 1000→11000, 5000→51000 = 가격표 C열 verbatim ✅. CORNER_RIGHT = 전구간 0.00 ✅(가격표 B열=0).
- 둥근모서리도 오시/미싱과 동일 **축오적재**(prc_typ .01·실제 구간총액형 .02/.03)·동일 off-grid 누진룰(M-G2). 직각은 0원이라 정정 무영향.
