# SIZE_NAME_NOISE 정정 명세 — 사이즈 합성명 → 치수 + 묶음수 차원 분리

product-viewer 어색데이터 **SIZE_NAME_NOISE 81 findings**(고유 siz_cd 77개)를 "치수 + 묶음수(+ 모양/접두) 차원 미분리" 데이터로 재해석하여 분리한 정정 명세. 명칭 파싱은 우리측 기계화로 수행하되, 적재 권한·원천 수정 여부는 후니 영역으로 구분한다. 스키마 무변경.

> **[v2 권위 정정 — validator 라이브 검증 반영]** 1단계 전제 "치수 슬롯 NULL → UPDATE로 채움"은 **라이브 DB 기준 거짓**으로 판명됐다. 라이브 `t_siz_sizes`의 work/cut 치수(`numeric(8,2)`)는 **이미 77/77 채워져 있다(NOT NULL)** — 직전 D-C에서 후니가 등록한 것이 라이브에 반영됨. 1단계가 본 `ref-sizes.csv`(2026-06-04 17:13)는 **stale 스냅샷**이었다. 권위 = **라이브 DB > 추출 시트**.
>
> 결과: **치수축 정정은 불요**(라이브 75행 이미 동일, cm 2건은 라이브가 정확·CSV가 회귀). **실제 정정 성과 = 묶음수 18행 분리**(`t_prd_product_bundle_qtys.csv`). 근거: `03_validation/product-viewer/correction-validation-report.md`.

> 핵심: SIZE_NAME_NOISE는 noise가 아니라 **차원 분리가 안 된 데이터**다. 대표 예 `70x200mm(50장)` = ① 치수 70×200(라이브 `t_siz_sizes.work/cut`에 **이미 반영됨**) + ② 장수 50(=묶음수 `bdl_qty`, **이번 정정의 신규 분리분**).

### stale 추출본 경고 (미래 혼동 방지)
1단계가 본 `00_schema/ref-sizes.csv`(2026-06-04 17:13 추출)의 "work/cut 전부 NULL"은 **stale 스냅샷**이다. 직전 D-C에서 후니가 치수를 등록한 것이 라이브에 반영됐으나 추출본만 옛 상태였다. validator가 라이브 read-only 조회로 77/77 NOT NULL을 확인 → **권위 = 라이브 DB**. crossmap v2의 "후니 전 size 치수 등록"은 라이브 기준 **맞았고**, 1단계의 "NULL 잔존" 판단이 stale 시트 때문에 빗나간 것. 향후 치수 NULL 여부는 ref-sizes.csv가 아니라 라이브 DB로 판정할 것.

## 1. 파싱 규칙 (기계화)

siz_nm 문자열을 좌→우 토큰으로 분해. 발견된 전 패턴 커버(단일 정규식 아님):

1. **접두 분리**: 색상/자재 접두(`화이트|블랙|레더`) 또는 명칭 접두(`PP투명케이스`)를 먼저 떼어냄.
2. **모양 접두**: `정사각|직사각|원형|사각|마카롱`이 앞에 있으면 분리.
3. **묶음수 괄호**: `(N장|N매|N개|N개입|N EA|N인용|N인)` → bdl_qty=N, 단위→QTY_UNIT 매핑.
4. **모양 접미**: 치수 뒤 한글(`원형|사각|하트|하트자물쇠`) 분리.
5. **치수 추출**: `W×H×D` (3D) 또는 `W×H` (2D), 단위 `mm|cm|없음` 판정.

### 단위(mm) 처리 규칙 — 치수축은 적재 불요이므로 참고용(라이브 권위)
- `mm` 명시 → 라이브가 이미 정수 mm 보유(예 70→70.00). 적재 불요.
- `cm` 명시(피크닉매트 2건) → **라이브가 mm 환산값(1000/700·1000/1500)을 이미 보유 = 정답**. 1단계 CSV의 cm 원본 보존(100/70)은 **회귀 오류**로 판명 → 적재 금지(reference.csv에 회귀 경고 표기). cm 표기 사이즈**명** 정돈은 후니 영역.
- 단위 미표기(만년스탬프 7, 레더라벨 3, 자석고무판 1) → 라이브가 동일 값(13×13→13.00 등) 보유. 적재 불요. 단 실제 mm 여부는 후니 확인 항목.

### 묶음 단위(QTY_UNIT) 매핑 근거 (code-values.md)
| 원본 단위 | bdl_unit_typ_cd | 근거 |
|-----------|------------------|------|
| 장 · 매 | `QTY_UNIT.02` (매) | 종이 낱장 단위 = '매' |
| 개 · 개입 · EA | `QTY_UNIT.01` (EA) | 낱개 단위 = EA |
| 인용 · 인 | `QTY_UNIT.04` (세트) | "1인용"=1세트 구성. **확인필요**(인용=세트 해석 가정) |

> "50장=1세트"의 의미 표현: `bdl_qty=50`, `bdl_unit_typ_cd=QTY_UNIT.02(매)` = "50매 묶음"으로 표현한다. round-1 패턴(권=QTY_UNIT.03)과 동일하게 bdl_qty는 묶음당 수량, 단위코드가 수량의 의미(매/EA/세트)를 부여한다.
