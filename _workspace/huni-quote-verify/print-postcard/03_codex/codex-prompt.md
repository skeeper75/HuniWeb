# 독립 가격계산 검증 요청 — 프린트엽서 (디지털인쇄 엽서 group)

당신은 인쇄 자동견적 시스템의 **독립 가격검증 전문가**입니다. 아래는 한 인쇄 상품(엽서)의 가격 산정 구조를 분해한 작업명세입니다. 이 명세만 보고 **독립적으로** 판정해 주세요. 다른 검증자의 결론은 제공되지 않습니다(독립성).

## 핵심 질문 (이것에 답하세요)

**이 상품(프리미엄엽서 · 가격공식 PRF_DGP_A · 합산형)이 자기 가격공식으로 가격계산이 제대로 되는가?**

다음 3축에서 깨지는 곳을 근거와 함께 독립 판정하세요:
1. **축1 (SOT 일치)** — 상품마스터(상품군이 허용하는 차원 경계) ↔ 가격표(단가 매트릭스 구역)가 정합하는가? 상품이 허용하는 차원(사이즈·종이·도수·면·코팅·후가공)이 가격표 구역과 1:1 매핑되는가?
2. **축2 (공식↔구성요소 매핑)** — 공식에 배선된 가격구성요소(component)들이 권위 산식 항목대로, 그리고 상품(엽서) 차원경계 내에서 배선됐는가? 불필요/고아/이중매칭 구성요소는?
3. **축3 (차원 매칭)** — 각 구성요소의 use_dims(선언 차원) ↔ 단가행 충전 차원 ↔ 권위 가격축이 3원 정합하는가? **특히 도수(흑백 1도 / 칼라 CMYK / 별색) × 면(단면/양면) 매트릭스가 제대로 반영되는가?**

**모르면 "모른다"고 명시하세요.** 추정을 사실로 단정하지 마세요. 각 판정에 근거를 박으세요.

## 상품 정체

- **프린트엽서** = 디지털인쇄 시트의 `엽서` 구분(group) = 8개 상품 묶음
- **대표(앵커) 상품** = 프리미엄엽서 (가격공식 PRF_DGP_A 바인딩)
- **가격 산정방식** = **원자합산형**: 판매가 = 인쇄비 + 코팅비 + 용지비 + 후가공비 + 추가상품 (각 항목을 더함)

## 가격공식 PRF_DGP_A 정의

```
공식명 = 디지털인쇄 원자합산형A 엽서·상품권·슬로건
설명   = 엽서·상품권·종이슬로건 가격. 인쇄비+코팅비+용지비+후가공비+추가상품을
         더해 판매가 계산. 별색인쇄비·대형박은 별도 항목.
유형   = 합산형 (전 구성요소 addtn='Y' = 엔진이 무차별 합산. 차감/택일 장치 없음)
```

## 산정 공식 (권위 = 계산공식집)

```
판매가 = 인쇄비 + 코팅비 + 용지비 + 후가공비 + 추가상품
(1) 출력매수 = ceil(주문수량 / 판걸이수)        [판걸이수 = 사이즈별 룩업값, DB 미저장·앱 계산]
(2) 인쇄비 = [디지털인쇄비 수량행단가] × 출력매수
(3) 별색인쇄비 = 별도 항목 (별색 선택 시)
(4) 코팅비 = [코팅 수량행단가 × 코팅타입] × 출력매수
(5) 용지비 = [크기별 기준단가] × (출력매수 + 5장 손지율)
(6) 후가공비 = [제작수량행 단가]
(7) 추가상품 = 목록별 가격적용
```

## 골든 케이스 (대표 선택값 → 기대 가격)

> 단가는 모두 가격표 셀 verbatim. 가공/반올림 임의 금지.

### 케이스 PC-2 — 칼라단면 · 100x150 · 200매
```
사이즈 100x150 → 판걸이수 8
출력매수 = ceil(200 / 8) = 25
인쇄비 단가(칼라단면, 출력매수25행) = 800   [가격표 칼라 열]
인쇄비 = 800 × 25 = 20,000
용지비 = 70.64 × (25 + 5) = 70.64 × 30 = 2,119.2
─────────────────────
기대 인쇄비 소계 = 20,000 (확정 앵커)
기대 final = 20,000 + 2,119.2 = 22,119.2 (반올림 정책 미확정)
```

### 케이스 PC-4 — 흑백단면 · 100x150 · 200매 (도수축 분리 검증)
```
출력매수 = 25 (동일)
인쇄비 단가(흑백단면, 출력매수25행) = 350   [가격표 흑백 열]
인쇄비 = 350 × 25 = 8,750
기대 인쇄비 소계(흑백) = 8,750
```

### ★ PC-2 vs PC-4 = 도수축 검증 쌍
- 권위: 칼라 = 20,000, 흑백 = 8,750 (도수가 가격을 가른다)
- 만약 라이브 엔진이 칼라·흑백 **둘 다 8,750(흑백값)** 을 산출하면 = 도수 차원 드롭(흑백값만 적재) = 결함 확정
- 만약 둘을 구분하면 = 도수 차원이 어디선가 처리되는 것

## 당신이 판정할 것 (요약)

1. **이 상품이 자기 공식(합산형 PRF_DGP_A)으로 가격계산이 성립하는가?** (GO / 조건부 / NO)
2. **3축 각각에서 깨지는 지점**과 근거:
   - 축1 SOT: 사이즈·종이·도수·면·코팅 차원이 가격표 구역과 정합하는가? 도수축이 가격표엔 있는데 상품/구성요소엔 없는 게 SOT 불일치인가?
   - 축2 매핑: 18개 구성요소 배선이 권위 산식대로인가? 유광코팅비 0행(고아)·S1/S2 양키 이중·합산플래그 무차별 합산이 문제인가?
   - 축3 차원: **도수축 드롭(흑백값만 적재·use_dims에 색 없음)이 칼라 선택을 무반영하게 만드는가?** 이게 가장 큰 돈크리티컬 결함인가?
3. **우리가 놓쳤을 추가 결함/위험**이 있는가? (합산형 무차별 합산의 다른 함정, 손지율 +5장, 판걸이수 앱계산의 룩업 위험, 비명시 수량행 구간룩업 등)
4. **도수축 드롭이 결함인가 의도(엽서=흑백 단일가)인가** — 당신 의견과 근거.

근거 없는 단정 금지. 모르는 건 "모른다"로. 산식 검증(20,000 / 8,750 산출 과정)이 맞는지도 독립 점검하세요.
