## 0. 사용자 질문에 대한 직답

> "각 셋트상품의 상품구성요소(옵션수단)와 가격구성요소(component·use_dims·단가행)의 차원이 제대로 갖춰졌는지, 없으면 무엇을 채워야 하는지, 이를 검증→교정할 기반이 마련됐느냐."

**답:**
1. **차원 검증 수단(dim_conformance.py)이 셋트 양 계층에 작동함을 입증했다** — 39 상품 전수 실행, REVIEW 신뢰도가 후가공 분담 잔여 6건을 정확히 격리(false-positive 0 승격).
2. **그러나 셋트의 차원 결함은 MISSING/UNDECLARED로 드러나지 않는다.** 진단기가 행을 만들려면 입력 3면이 다 있어야 하는데(공식 바인딩 + Face C 옵션수단), 셋트 구성원 28개 중 **27개가 공식·옵션수단 자체가 비어있어** 3자 조인 격자가 **성립조차 안 된다**(verdict=NO-FORMULA / NO-FACE-C). 이건 차원 누락보다 **한 단계 상류**의 결함이다.
3. 따라서 "없으면 무엇을 채워야 하는가"의 답은 **단가행 1~2종이 아니라, 구성원 반제품에 옵션수단(sizes/materials/processes) + 가격공식 바인딩 + 단가행을 통째로 채워야** 한다(§2·§3).
4. 검증→교정 기반은 마련됐다 — 진단기가 NO-FORMULA/NO-FACE-C를 결정론으로 분류하고, 각 결함이 dbmap 트랙(search-before-mint·dbm-load-execution·hsp-set-design)으로 라우팅됨을 입증했다(§4).

## 2. ★핵심 발견 — 셋트 차원 결함의 진짜 형태는 "격자 미성립"(상류 결함)

진단 입력 3면이 다 있어야 (상품×component×차원) 격자가 형성된다. 셋트 양 계층 전수 분류:

| 분류 | 상품 | 의미 |
|---|---|---|
| **DIAGNOSABLE** (9) | 068·069·070·071·072(부모)·284(내지)·094·097·100(부모) | 공식+옵션수단 둘 다 보유 → 3자 조인 성립 → **차원 정합 확인됨**(MISSING-HIGH 0) |
| **NO-FORMULA** (3 부모) | 077·082·088 | 부모에 옵션수단은 있으나 **가격공식 미바인딩** → 셋트 견적 불가, 차원격자 미성립 |
| **NO-FACE-C(공식만)** (1) | 101(포토북 내지) | 공식 PRF_PHOTOBOOK_INNER 있으나 **옵션수단 0**(sizes/mats 전멸) → 차원 충전했으나 손님 선택수단 없음 |
| **NO-FORMULA+NO-FACE-C** (27 구성원) | 073~076·078~081·083~087·089~093·095·096·098·102~107·104 | 구성원 반제품에 **공식도 옵션수단도 전무** → 격자 자체 부재 |

**해석:** dim_conformance.py가 0건/REVIEW만 낸 것은 "셋트 차원이 완벽"이 아니라 **27 구성원이 진단 대상조차 못 되는 빈 껍데기**라는 뜻이다. 차원 진단기는 정직하게 "조인할 입력이 없다"를 NO-FORMULA/NO-FACE-C로 드러낸다.

이는 셋트 현황판(set-product-readiness-master.md)의 "5셋트(072/077/082/088/100) 미바인딩·책자류 내지 반제품 부재"와 정확히 일치한다 — 차원 진단이 그 결함을 **독립 재발견**했다(도구 신뢰성 입증).

## 3. "없으면 무엇을 채워야 하는가" — 셋트별 처방

### 구성원 반제품 빈 껍데기 → 옵션수단+공식+단가행 통째 (NO-FORMULA+NO-FACE-C, 27건)
- **책자류 면지(074~076·079~081·084~086·090~092·104)**: 면지=제본비 포함(단가0) 설계 — 옵션수단 불요·공식 불요가 **의도**일 수 있음(현황판: 면지=무료). 단, 면지색 선택을 손님이 한다면 옵션그룹+ref_dim 환원 필요. → §23 hsp-set-design 설계 확인.
- **표지(073·078·083·089·096·102·103·105·106·107)**: 표지는 부모공식의 표지용지 비목으로 흡수되는지(현황판 072 cover_mult), 아니면 구성원 자체 공식인지 §23 설계 권위 확인 후 — 자체공식이면 siz_cd(표지 펼침 390×268 미등록)·mat_cd·단가행 채워야.
- **인쇄면지(087·093)**: 별색/인쇄 면지 → 자체 공식+인쇄 단가행 필요(현황판 B-2 "087/093 연결").
- **포토북 내지 101**: 공식 PRF_PHOTOBOOK_INNER 있으나 옵션수단 0 → **siz_cd·page 선택수단 등록 필요**(NO-FACE-C 단독 케이스 — 차원 충전됐는데 손님이 못 고름).
- **엽서북 내지095·떡메 내지098**: 부모공식 흡수형이면 빈 껍데기 정상 — §23 설계 확인.

★중요: 셋트 면지/표지의 "빈 껍데기"가 **결함인지 설계 의도(부모 흡수·면지무료)인지**는 차원 진단 단독으로 판정 불가. §23 hsp-set-design 권위(set-composition-design)와 대조해야 함. 차원 진단은 "격자 미성립" 신호만 제공.

**입증:** 차원 검증 수단이 셋트 양 계층에 작동했고(39상품 전수), 결함을 4분류로 결정론 격리했으며, 각 분류가 dbmap/§23 트랙으로 라우팅된다. 즉 **"검증 수단으로 교정할 기반"은 마련됐다**. 다만 셋트의 1순위 결함은 단가행 차원 누락이 아니라 **구성원 반제품 빈 껍데기 + 부모 공식 미바인딩**(상류)이므로, 교정 순서는 §23 설계 → 옵션수단/공식 채우기 → 그 후 dim_conformance.py 재진단으로 차원 정합 확인이다.
