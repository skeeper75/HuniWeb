권위: **엑셀=원천**, DB 적재값이 엑셀과 정합하는지 검증. DB 재조회 없음(`00_schema/` 추출본). EXTRA 삭제금지(플래그+출처). JOIN KEY=`prd_nm`(prd_cd→`ref-products.csv`로 해석).

근거 파일:
- DB: `00_schema/ref-product-bundle-qtys.csv` (4행) + `ref-products.csv`(prd_cd↔prd_nm)
- 엑셀: `04_audit/excel-bundle-qty.csv` (260행 파싱, 묶음수 값 보유 22상품)
- 정정: `02_mapping/correction/load/t_prd_product_bundle_qtys.csv` (18행 / 9상품 — SIZE_NAME_NOISE 정정서가 siz_nm에서 장수/개수 분리)
- 불일치: `04_audit/bundle-qty-mismatches.csv`

## §1 3자 관계 정량 (prd_nm 기준)

| 집합 | 상품수 | 행수 | 비고 |
|------|:-----:|:----:|------|
| DB | **1** (떡메모지) | 4 | PRD_000097·PRD_000182 각 (50,100) |
| 엑셀(묶음수 값 보유) | **22** | 22 | 명함10·스티커5·아크릴2·디자인캘린더5 |
| 정정 | **9** | 18 | 봉투/케이스/매트/합판도무송 |

### 핵심: **3자 상호 배타(disjoint)** — 교집합 전부 ∅
- DB ∩ 엑셀(값보유) = **∅**
- 엑셀(값보유) ∩ 정정 = **∅**
- DB ∩ 정정 = **∅**

근거(`bundle-qty-mismatches.csv` + 조인 실측): 세 산출물이 **서로 다른 상품 모집단**을 다룬다.
- **엑셀 묶음수 컬럼 값 보유 22** = 명함(`Y`)·스티커(조각)·아크릴(조각)·디자인캘린더(P) — 묶음수 전용 컬럼에 값이 적힌 것.
- **정정 9** = 봉투/투명케이스/자석고무판/합판도무송스티커/피크닉매트 — **엑셀 묶음수 컬럼은 공란이나** siz_nm(`160x110mm(20장)`)에 장수/개수가 은닉된 것을 정정서가 분리. (9 전부 `excel-bundle-qty.csv`에 행은 있으나 묶음수 값 공란 — 실측 확인)
- **DB 1**(떡메모지) = DB에만 묶음행 존재, 엑셀 묶음수 컬럼 공란(책자·문구 시트 둘 다 공란).

→ "엑셀엔 (묶음수 컬럼) 22 + (사이즈명 은닉) 9 = 사실상 31상품에 묶음 정보가 있는데, DB는 떡메모지 1상품(4행)뿐" = **대량 MISSING** 정량 확증.

## §2 4분류 카운트 + 상품별 명세

| 분류 | 상품수 | 행수 | 의미 |
|------|:-----:|:----:|------|
| **MATCH** | **0** | 0 | DB와 엑셀이 일치하는 상품 없음 |
| **MISSING** (엑셀O DB X) | **22** | 22 | 엑셀 묶음수 값 보유 22 전부 DB 미적재 |
| **EXTRA** (DB O 엑셀 X) | **1** | 4 | 떡메모지 — DB 묶음행 있으나 엑셀 묶음수 컬럼 공란 |

### MISSING 22 명세 (3유형)
1. **명함 `Y` 플래그 10** (`프리미엄명함`·`코팅명함`·`스탠다드명함`·`펄명함`·`모양명함`·`미니모양명함`·`오리지널박명함`·`형압명함`·`투명명함`·`화이트인쇄명함`): 값이 수량이 아닌 `Y`(묶음 여부 표시). **bdl_qty(integer)에 적재 부적합 — 확인필요**(묶음 적용 플래그인지, 별도 묶음수가 다른 곳에 있는지).
2. **조각수 7** (`반칼 자유형 스티커`=`*최대20조각|*최대40조각`, `낱장 자유형 스티커`·`낱장 자유형 투명스티커`·`대형 자유형 스티커`=`5~10조각`, `소량자유형스티커`=`*1조각`, `아크릴자유형스탠드`=`2~6조각`, `아크릴미니파츠`=`10조각`): DB 미적재. `*최대`/`*최소` 안내문 분리 필요, 조각 단위는 §3 참조.
3. **페이지묶음 P 5** (`탁상형캘린더`=30P, `미니탁상형캘린더`=26P, `엽서캘린더`=12P, `벽걸이캘린더`=13P, `와이드벽걸이캘린더`=13P, 전부 디자인캘린더 시트): DB 미적재. P=페이지수 묶음.

### EXTRA 1 명세 (삭제금지)

## §3 단위 코드(QTY_UNIT) 매핑 정합

QTY_UNIT 유효 부분집합(code-values.md): `.01 EA` · `.02 매` · `.03 권` · `.04 세트`. `bdl_unit_typ_cd`는 별도 BDL_UNIT 그룹 없이 QTY_UNIT 재사용(정합).

| 출처 | 사용 코드 | 정합 |
|------|----------|------|
| DB(떡메모지) | `QTY_UNIT.03`(권) | 유효코드. 떡메모지=권 단위 묶음 — 의미 정합 |
| 정정(봉투류) | `QTY_UNIT.02`(매) | 장수→매 매핑 정합 |
| 정정(케이스/고무판/합판도무송) | `QTY_UNIT.01`(EA) | 개/EA→EA 정합 |
| 정정(피크닉매트) | `QTY_UNIT.04`(세트) | 인용→세트 정합 |
| 엑셀(조각수) | 미부여(MISSING) | **「확인 필요」**: `조각`은 QTY_UNIT 4종에 직접 대응어 없음 → `EA(.01)`로 매핑할지 결정 필요 |
| 엑셀(페이지 P) | 미부여(MISSING) | **「확인 필요」**: `P`(페이지)는 묶음수 단위인지 페이지룰(page_rule) 속성인지 분기 — 디자인캘린더 P값은 page_rule과 중복 가능 |

→ DB·정정의 단위 코드는 전부 QTY_UNIT 유효 부분집합으로 정합. 엑셀 조각/페이지 단위는 코드 미부여 상태이며 매핑 규칙 확인 필요.

## §4 판정 + 조치 제안

### 판정: **MAJOR (묶음수 속성 대량 MISSING)**
- MATCH 0 / MISMATCH 0 (마스터 코드·단위는 깨지지 않음 — 적재된 4행·정정 18행은 단위코드 정합).
- 그러나 **엑셀 묶음 정보 보유 ~31상품(컬럼 22 + 은닉 9) 대비 DB 적재 1상품(4행)** = 커버리지 결손이 본질. 거짓 정합 위험은 없음(disjoint이 명확히 드러남).

### MISSING이 정정 18행으로 해소되는 정도: **부분(다른 모집단)**
- 정정 18행/9상품은 **엑셀 묶음수 컬럼 공란분(siz_nm 은닉)**을 보강 — 엑셀 묶음수 컬럼 값 보유 22와 **교집합 ∅**. 즉 정정은 22 MISSING을 **메우지 않는다**(별개 상품군).
- 정정 적재 시 DB 묶음 커버리지는 1상품(4행) → 10상품(22행)으로 확대되나, **엑셀 묶음수 컬럼의 22상품은 여전히 미적재로 남음**.

### 조치 제안 (후니/정정)
1. **정정 18행 적재 권고**: 9상품(봉투/케이스/매트/합판도무송) 단위코드 정합 확인됨 → 적재로 siz_nm 은닉 묶음수 해소. (단 DB 미적재 보류 정책 유지 시 적재용 CSV로만 보존)
2. **엑셀 22 MISSING 후속 매핑 필요**(별도 작업):
   - 명함 `Y` 10: bdl_qty 적재 전 **`Y`의 의미 확정 필요**(묶음 적용 플래그 vs 실수량) — 현 상태로는 묶음수 행 생성 부적합.
   - 스티커/아크릴 조각 7: `*최대/최소` 안내 분리 + 조각→QTY_UNIT 매핑 규칙(EA?) 확정 후 적재.
   - 디자인캘린더 P 5: 묶음수 vs page_rule 속성 분기 판정 후 적재.
3. **떡메모지 EXTRA**: 삭제금지. 50/100권의 엑셀 원천 위치(묶음수 컬럼 아님) 확인 후 정합 판정.

### 후속 인계 주의점
- 묶음수는 **단일 컬럼에 4종 의미 혼재**(명함 플래그·조각·장·페이지) → 속성 정의 자체가 비균일. 적재 전 묶음수의 도메인 정의(무엇이 bdl_qty인가) 합의 필요.
- 조각/페이지 단위는 QTY_UNIT 미대응 → 코드 확장 또는 EA/권 매핑 결정 필요(추정 금지, 확인필요).
