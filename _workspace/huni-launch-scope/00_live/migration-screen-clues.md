# 마이그레이션 화면 관찰 단서 (회원 · 프린팅머니)

> ★위상: 이것은 **화면 관찰값(observed UI)** 이지 DB 권위가 아니다(화면 ≠ DB).
> hls-migration-designer의 원천 단서로만 사용. 실제 컬럼/타입은 백오피스·DB 실측으로 확정.
> ★개인정보 보호: 실제 값은 모두 **마스킹**. 필드 존재·구조만 기록. PII 포함 스크린샷 미저장.

채집: 라이브 huniprinting.com, 로그인 회원 계정, 읽기 탐색만, 2026-06-30.

---

## 1. 회원 (member) — 화면 노출 필드

### 1-1. 대시보드(`/mypage/main.asp`) 노출
| 화면 라벨 | 관찰값(마스킹) | 단서 |
|-----------|----------------|------|
| 회원명 | (○○○ 고객님) — 마스킹 | 회원 표시명 |
| 회원할인율 | 0 % | **회원 등급/할인율 필드** |
| 이메일 | (****@****) — 마스킹 | 이메일 |
| 프린팅머니 | 0 원 | 적립금 잔액 요약 |
| 휴대폰 | (010-****-****) — 마스킹 | 연락처 |

### 1-2. 회원정보수정(`/mypage/modify.asp`) 폼 필드명 = 회원 스키마 단서
(폼 `name` 속성 기준. 값 미수집.)

**개인:**
- `user_pwd` (비밀번호)
- `user_email`, `check_email` (뉴스레터 수신 여부 라디오)
- `user_zip`, `user_zone`(hidden), `user_addr1`, `user_addr2`, `jibun_addr` (주소: 우편번호·도로명·상세·지번 병행)
- `user_tel1/2/3` (일반전화 3분할)
- `user_hp1/2/3` (휴대폰 3분할), `check_sms`(SMS 수신 라디오), `check_bunho`(hidden), `temp_sms`(hidden), `sms_number` (휴대폰 인증번호)

**사업자(세금계산서/회사):**
- `user_job` (직업, select)
- `user_company` (상호/회사명)
- `user_number1/2/3` (사업자등록번호 3분할)
- `user_owner` (대표자명)
- `user_uptae` (업태), `user_upjong` (업종)
- `user_c1zip`, `user_c1zone`(hidden), `user_c1addr1`, `user_c1addr2` (회사주소1)
- `user_ctel1/2/3` (회사전화), `user_cfax1/2/3` (팩스)
- `user_c2zip`, `user_c2zone`(hidden), `user_c2addr1`, `user_c2addr2` (회사주소2 / 추가배송지 추정)

→ **회원 = 개인 + 사업자 + 다중주소(개인주소·회사주소1·회사주소2)**.

### 1-3. 탈퇴(`/mypage/out.asp`) 노출 필드
아이디 · 비밀번호 · 이름 · 이메일 · 휴대전화 · 탈퇴사유.

### 1-4. 로그인 폼(`/login/login.asp`)
- 필드: `user_id`(아이디), `user_pwd`(비밀번호), hidden `where`(리다이렉트 목적지).
- action: `/login/login_ok.asp` (POST). 아이디 기반 로그인.

## 2. 프린팅머니 (적립금/예치금) — 화면 노출

### `/mypage/money.asp`
| 화면 요소 | 관찰값 | 단서 |
|-----------|--------|------|
| 현재 사용가능 금액 | 0 (원) | **잔액(balance)** 단일 값 |
| 내역 목록 | "내역이 없습니다" | **적립/사용 거래 내역(원장)** 테이블 영역 존재 |

- 대시보드에도 "프린팅머니: 0원" 요약 동기 노출.
- 충전/적립/만료 규칙·날짜 컬럼은 내역이 비어 있어 **헤더 미관측(미발견)**.
- 마이그레이션 데이터셋(추정): ① 회원별 잔액 ② 거래 트랜잭션(적립/사용/만료, 일시, 금액, 사유). → DB 실측으로 확정 필요.

## 3. 쿠폰/이용권 (`/mypage/coupon.asp`) — 참고
- 쿠폰 목록 + 사용 정책 안내(유효기간 만료 시 자동 소멸, 1주문 1쿠폰, 회원할인 제외 금액에 추가 적용, 배송비 결제 불가, 환불 시 미반환).
- 마이그레이션 시 쿠폰 = 회원 보유 쿠폰 + 유효기간 + 사용조건. 프린팅머니와 별개 자산.

---

## 4. 적용 순서 단서(가격/할인 체인)
화면 안내 종합: **상품금액 → 회원할인(회원할인율 %) → 쿠폰/이용권 추가할인 → 프린팅머니/결제**.
(쿠폰 안내문 "회원할인 제외 금액에 추가 할인" 근거.)

---

## 5. 미발견/한계
- 회원가입 폼 전체 필드(약관·본인인증·필수/선택): 로그인 상태라 미접근. 로그아웃 후 `/join/intro.asp` 재탐색 필요.
- 프린팅머니 충전 화면·적립 규칙: 화면 미노출(미발견).
- 회원 등급 체계(할인율 외 등급명/혜택): 대시보드 할인율 %만 노출, 등급 라벨 미관측.
- 주문 상세(주문번호→상세 필드): 테스트 계정 주문 0건이라 미도달.
