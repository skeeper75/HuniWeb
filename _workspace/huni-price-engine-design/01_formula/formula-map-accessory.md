## 0. ★결론 먼저 — 상품악세사리는 계산이 가장 단순한 inline 고정가형 + 가격사슬 100% 전무 (5종단·6종단 통틀어 최저 완성도)

| 추정(과업 입력·메모리) | 라이브·권위 실측 결론 |
|----------------|----------------------|
| "사이즈 3축 복합(규격+묶음수)이 가격축인가?" | **거짓.** 사이즈/묶음수/색상은 **가격축이 아니라 variant 식별축**. 각 variant행은 자기 inline 고정가를 가짐(굿즈 GP-2 동형). 면적매트릭스·수량구간·합가형 전부 0건. |
| "OTC 템플릿 이중등록 → 가격 권위 충돌?" | **부분 참.** 부자재는 **이중역할 SKU** — ① 독립 판매(`PRODUCT_PRICE`) ② 본체 상품 addon(`TEMPLATE_PRICE`). 가격엔진이 두 경로를 **우선순위로 분리**(TEMPLATE_PRICE → PRODUCT_PRICE → FORMULA → NONE) → 충돌 아님(경로 분기). 단 **양 경로 단가 모두 라이브 전무**. |
| "굿즈/파우치(GP-1/GP-2)와 동형인가?" | **참 — GP-1 단일고정가 거의 완전 동형 + 변형(BG-2) 보유.** 단 굿즈와 결정적 차이 2: ① **수량구간할인 자체가 없음**(굿즈는 DSC 4타입·82바인딩 실재) ② **addon TEMPLATE_PRICE 경로가 본질**(굿즈엔 없음·부자재 고유). |

"상품악세사리" 또는 "부자재/봉투/케이스" 전용 공식 row는 없음.

## 4. 3층 지도 — (AC-2) 변형고정가 (완제품·이중역할·11상품·★핵심 설계난제)

> calc-draft 공식 row 없음(inline 고정가표가 권위). **고정가가 variant축으로 분기**. ★사이즈 3축 복합(규격×묶음수×색상)이 가격 분기의 핵심.

### 4-A. 규격×묶음수 변형 (봉투류 — OPP접착/비접착·트래싱지카드·캘린더)
```
상품 [완제품·이중역할(독립판매 + 엽서 addon)]:
  OPP접착봉투(PRD_000001·11규격: 70x200=1100 → 230x350=3250)
  OPP비접착봉투(002·11규격·600~1500)
  트래싱지카드봉투(003·★규격3 × 묶음수3 = 8 variant: 160x110 20장6000/40장12000/100장28000…)
  캘린더봉투(005·2규격·2400/2500)
  ├─ variant축①   규격(siz)   = 70x200·80x100… ★cut_width/cut_height로 분해(라이브 정상·SIZ_000078~)
  ├─ variant축②   묶음수      = (50장)·(20장)·(100장) ★현 라이브: siz_nm 텍스트 잔존(F-PA size §4) + bdl_qty 별도(일부)
  ├─ 변형고정가    각 (규격×묶음수) variant 행이 가격 I열에 자기 고정가
  │                ★트래싱지(003): 같은 규격이라도 묶음수가 다르면 다른 고정가(160x110 20장6000 vs 100장28000)
  │                  → 묶음수가 가격 분기축(굿즈 size→option과 동형 — 묶음수=가격 variant)
  ├─ 적용경로      독립=PRODUCT_PRICE / 엽서(PRD_000016) addon=TEMPLATE_PRICE(TMPL-000005/006/009/010/011 실재)
  └─ (구간할인)    없음
```
계산방식: **변형고정가 × 수량** (variant 행별 단가) · ★엔진 그릇 부재(G-AC-1) · baseline: 양 단가 전무 `확신도: 높음(트래싱지 8행 묶음수 가격분기 L1 실측·addon 5행 라이브)`

### 4-B. 색상×규격 변형 (카드봉투 — 화이트/블랙)
```
상품: 카드봉투(PRD_000004·화이트 165x115 1000 / 블랙 165x115 1500)
  ├─ variant축     색상 (화이트/블랙)  ★색상이 가격 분기(화이트1000 vs 블랙1500)
  │                현 라이브: 색상이 siz_nm에 합성("화이트165x115mm"·"블랙165x115mm" 2행·F-PA size §4)
  ├─ 변형고정가    색상별 자기 고정가
  └─ 이중등록      PRD_000281(화이트)·282(블랙)·283(트레싱지) = 별 PRD_TYPE.03(추가상품 아님·기성품)
                   + TMPL-000007~011 봉투 template (엽서 addon)
```
★카드봉투 색상은 굿즈 "면=도수" variant와 달리 **색상=가격 variant**(자재도 도수도 아닌 SKU 분기). 라이브는 siz_nm 합성으로 처리(F-PA size). `확신도: 높음`

### 4-C. 길이/용량/규격 변형 (우드 부속·투명케이스·자석·천정고리·행택끈)
```
상품: 우드봉(PRD_000013·270mm7000/360mm9800/480mm12000)·우드행거(014·230mm16000/320mm18000/440mm20000)·
      우드거치대(012·120mm 4000)·투명케이스(009·42x57x20=3000/75x75x15=3000/75x110x15=3500)·
      자석고무판(011·20x20 20개입 1000)·천정고리(008·2개1세트 6500·use_yn=N)·행택끈(010·3종 3000/3000/4000)
  ├─ variant축     길이(우드)·3D규격(투명케이스 W×D×H)·종류(행택끈)
  ├─ 변형고정가    variant별 자기 고정가
  └─ 적용경로      우드=캘린더/족자/행잉 addon (외주 우드가공·인쇄 BOM 아님)
```
★투명케이스는 **3D 규격(W×D×H 3치수)** — 면적/부피 매트릭스가 아니라 규격당 이산 고정가(42x57x20=3000원). 우드는 외주 가공이나 가격은 동일하게 inline 고정가. `확신도: 높음`

## 5. ★사이즈 3축 복합 — 규명 결과 (과업 핵심 질문 #2)

**질문: 사이즈(필수) "70 x 200 mm (50장)"는 가격축(siz_cd)인가 variant인가 묶음단위(.02 합가형)인가?**

**판정: 3축 모두 variant 식별축(가격 분기) — 합가형(.02) 아님·면적매트릭스 아님.** `확신도: 높음`

| 축 | L1 표현 | 가격 역할 | 라이브 적재 | 설계 함의 |
|----|---------|----------|------------|-----------|
| **①규격(siz)** | "70 x 200 mm" | variant 분기(규격별 다른 고정가) | t_siz_sizes cut_width/cut_height 분해(SIZ_000078~·정상) | siz_cd 이산축 (면적 ceiling 아님) |
| **②묶음수** | "(50장)"·"(20장)"·"(100장)" | **가격 분기축**(같은 규격·다른 묶음=다른 고정가·트래싱지 003) | siz_nm 텍스트 잔존 + bdl_qty 일부(F-PA §3·§4) | ★굿즈 size→option처럼 **묶음수=variant** (가격 동반 분기) |
| **③색상** | "화이트"·"블랙"(카드봉투) | 가격 분기축(화이트1000/블랙1500) | siz_nm 합성("화이트165x115mm") | variant축(자재 아님·도수 아님) |

**★핵심: 묶음수는 ÷min_qty(합가형) 대상이 아니다.** 봉투제작(PRD_000050·별 상품군)은 합가형(.02·구간총액÷min_qty)이지만, **상품악세사리 봉투(PRD_000001~005)는 inline 고정가형(.01 의미)** — "(50장)"은 그 variant의 단위포장 수량이고 고정가 1100원은 그 **1팩 가격**이지 "÷50 = 장당가" 환산 대상이 아님. 주문수량(수량 필수 최소1·최대100·증가1)은 **그 팩을 몇 개 사느냐**(× qty). `확신도: 높음(L1 가격=팩단가·pricing.py:316 PRODUCT_PRICE=unit_price×qty)`

**돈크리티컬 함의(G-AC-2):** 묶음수를 ÷ 처리하면(합가형 오적용) 봉투 1팩 1100원이 22원/장 × 50장으로 환산되어 **묶음수 다른 variant가 가격 붕괴**. AC=고정가형(.01)·합가형 금지 (책자 제본비 .01 정당 단가 교훈 동형 — [[booklet]] G-BK).

## 6. ★OTC 템플릿 이중등록 — 규명 결과 (과업 핵심 질문 #3)

**질문: 거울/키링류가 아크릴 시트와 악세사리 시트에 이중등록인가? 완제품 SKU(addons/templates) 경로인가? 가격 권위 충돌?**

**판정: 아크릴↔악세사리 이중등록 아님 — 봉투류만 "이중역할 SKU"(독립판매 PRD + 본체 addon template). 가격 권위 충돌 없음(엔진 경로 분기).** `확신도: 높음(라이브 addon 5행·template 12행 실측)`

| 발견 | 라이브 실측(2026-06-22) | 판정 |
|------|------------------------|------|
| **거울/키링 아크릴 이중등록?** | 과업 입력의 "아크릴키링 PRD_146·틴거울183·아크릴뱃지148"은 **아크릴/굿즈 시트 상품**(PRD_000146 등)이지 상품악세사리 시트(PRD_000001~015) 상품이 아님. 상품악세사리 시트엔 거울/키링 **없음** — 봉투/케이스 11 + 우드/리필 부속 4뿐. | ★아크릴↔악세사리 이중등록 **사실무근**(시트 혼동·정정). 굿즈 거울(183/184/185)은 굿즈 종단 스코프. |
| **봉투류 이중등록(실재)** | 카드봉투: PRD_000004(기성 PRD_TYPE.03·독립) + PRD_000281/282/283(별 PRD_TYPE.03·★추가상품 아님·기성품) + TMPL-000005~011(봉투 template) | **이중역할 SKU**(독립 PRD + addon template base) — 의도(09_delete_dup 삭제 제외) |

## 7. 완제품 vs 반제품(세트) 판정

**전 18상품 = 완제품 (반제품 세트 0건).** `확신도: 높음(t_prd_product_sets sub_prd_cd 0행 실측)`
