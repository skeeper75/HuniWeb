## 0. 전수 스캔 결과 (70열 가설 → 실측 반증)

- 시트 격자 = `48 × 16`(실데이터). non-empty 셀 = **417개**. max data row 48·max data col 16(P열).
- "70열 = 여러 박종 블록" 가설 **반증**: 실데이터는 K~P(16열)까지뿐. 박종은 **헤더 텍스트에 나열**(A14/A34 괄호)될 뿐 개별 가격 블록 없음. 추가 면적구간/박종 블록 0.
- 데이터 블록 = 4개(수직 스택). 박소형은 동판이 단일 셀이었으나 대형은 **동판이 8×8 면적매트릭스(64셀)**.

## 2. 블록별 정밀 구조

### B1. 동판비 (A1~I10) — **박소형과의 결정적 차이**
```
A1: 동판비
A2: 가로/세로  B2:30mm C2:50mm D2:70mm E2:90mm F2:110mm G2:130mm H2:150mm I2:170mm  (가로 헤더)
A3: 30mm   B3=11000  C3=11000 … I3=12000   (세로 30mm 행)
…
A10: 170mm B10=12000 C10=19000 … I10=64000  (세로 170mm 행)
A11: *아연판       ← 노트(자재 주석)
```
- 의미: **동판(부자재) 가격이 작업 가로×세로 좌표마다 다름**(11,000~64,000원). 작은 면적=11,000 고정, 큰 면적=좌표별 증가.
- **박소형 대비**: 소형 동판은 `80x40mm 단일=5,000`(use_dims=[]·고정). 대형 동판은 **가로×세로 64좌표 직접단가**(use_dims=`["siz_cd"]`·면적매트릭스).
- 8×8 대칭 격자(가로 8 × 세로 8 = 64셀, 빠짐 없음).

### B2a / B3a. 면적 → 등급 정의 매트릭스 (8×8 대칭)
```
가로:  30  50  70  90  110 130 150 170
세로30: A   A   A   A   A   A   A   B
세로50: A   A   A   A   B   B   B   B
…
세로170:B   B   D   D   D   E   E   E
```
- **2단 룩업 1단계**: `(세로, 가로)` 작업사이즈 → 등급 A~E.
- 박소형(3행×5열 비대칭)과 달리 대형은 **8행×8열 대칭 격자**(빠짐·공란 0).
- **일반박/특수박 매핑 완전 동일**(실측 차이 0셀 — 가격만 다름).

### B2b / B3b. 등급별 수량 가격 (밴드 단가표)
```
분류/수량   A        B        C        D        E
10        55,000   65,000   65,000   80,000   120,000   (일반박)
200       65,000   80,000   90,000   120,000  160,000
500       70,000   ...
1000      75,000   ...
2000      150,000  ...
…
10000     750,000  950,000  1,200,000 1,600,000 2,000,000
```
- **2단 룩업 2단계**: `(등급, 수량구간)` → 가격.
- **수량구간 13개**(박소형 18개와 다름): 10·200·500·1000·2000·3000·4000·5000·6000·7000·8000·9000·10000.
- **단위 판별(핵심)**: 값 = 해당 수량의 **총액**(일반박 GRADE_A 1000매=75,000원 = 1000매 전체 가격). 장당가 아님.
  - 근거: 헤더 `분류/수량`이 수량 자체, 2000→150,000 / 10000→750,000(2000 대비 5배=정확 비례·이후 구간은 단순배수) → 구간 총액.
  - → `prc_typ_cd = PRICE_TYPE.02 합가형`(엔진: 구간총액 ÷ min_qty = 장당가 환산).
  - **라이브 충돌**: 명함박(`COMP_NAMECARD_FOIL_*`)도 동형 수량구간 총액인데 라이브에서 `PRICE_TYPE.01`로 등록 → **단가/합가 컨펌 필요**(박소형 P4 동류·decomposition §2 참조).
