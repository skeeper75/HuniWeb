### 2.2 역할 코드 (SEMI_ROLE · 라이브 실측)

`t_cod_base_codes` `cod_cd LIKE 'SEMI_ROLE%'`: **01 내지 · 02 표지 · 03 면지 · 04 간지 · 05 투명커버**.

### 2.3 개수규칙 해석 (sub_prd_qty / min/max/incr)

- 라이브 28행 전부 **sub_prd_qty=1, min_cnt/max_cnt/cnt_incr = NULL** (가변범위 미설정).
- **★권위 엑셀에는 내지페이지 가변범위가 명시** (booklet-l1):
  - 하드커버책자/레더 하드커버책자 내지페이지: **24~300 / +2**
  - 하드커버 링책자: **8~100 / +2**
  - 엽서북 내지페이지: **20~30 / +10**
- → **권위 vs 라이브 갭**: 내지의 페이지 가변범위(min/max/incr)가 라이브 t_prd_product_sets에 미반영(NULL). 이 가변범위는 evaluate_set_price의 구성원 qty(총내지매수=derive_inner_sheets) 산출 입력이므로 **set-designer가 채워야 할 셀** (체크리스트 min/max/incr 열).
- 떡메모지: 제본옵션 "50장1권 / 100장1권" = **묶음수(bundle-qty)** (라이브 `ref-product-bundle-qtys.csv` PRD_000097=50/100/QTY_UNIT.03). 셋트 구성원은 아님 — 셋트공식(떡제본) 차원.

### 2.4 ★ 면지/표지 "택1 옵션"의 평면화 (핵심 해석 · CONFIRM-2)

권위 엑셀에서 `제본(옵션)_면지(옵션)` 컬럼은 **행별 면지 색상 택1 옵션값**(화이트/블랙/그레이/인쇄). 라이브는 이 택1 옵션을 **각각 별개 반제품 sub_prd_cd로 평면화**(예 하드커버책자 면지 3종 = PRD_000074/075/076).
- 해석 후보 A: 셋트는 모든 면지 구성원을 동시 포함(SUM 합산) — **틀림**(면지는 택1).
- 해석 후보 B (권위 정합): 면지/표지 동일 역할의 다수 sub_prd_cd는 **택1 그룹**(손님이 1개 선택). evaluate_set_price `members` 배열에 선택된 1개만 들어감.
- → **CONFIRM-2**: 동일 역할(면지/표지) 다중 구성원 = "택1 그룹"으로 설계해야 하며, set-designer는 always-add(전 구성원 무조건 합산) 함정을 피해야 한다(§21 always-add 가드 교훈 적용).
