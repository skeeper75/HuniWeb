## 0. 포토북이 책자와 갈리는 지점 (한눈 비교)

포토북은 **책자 클래스의 한 갈래**이나, 가격계산 모델에서 결정적으로 세 가지가 다르다:

| 축 | 일반 책자 `[red:book·huni:prior]` | 포토북 `[red:PH·TP·huni:master]` |
|----|----------------------------------|----------------------------------|
| **수량 의미** | 부수(PRN_CNT) = 본 가격 차원(여러 부 발주) | **1부(또는 소량)**·디자인 = 사진 = 1회성·부수 곱은 부차 |
| **페이지 처리** | INN_PAGE 입력 + 내지단가×페이지×부수 곱(런타임) | **기본 N페이지 + 추가 페이지당 단가**(증분 단가)·`[huni:master]` "기본24P + 2P당" |
| **자재 분기** | 표지/내지 용지를 select | **표지타입(하드/레더하드/소프트)이 가격을 가르는 1급 축**·내지종이 1종(몽블랑130) 고정 多 |
| **주문 채널** | 업로드(PDF) | **에디터(KOI/edicus) 중심**·`[red:PH]` PHBK*·`[red:TP]` TPPHSET = 에디터 完·완제 단가 |
| **가격엔진(RP)** | `book2025_price`(부품 분해 매트릭스) | **`digital_price`(PHBK\*) / `tmpl_price`(TPPHSET 사진북)** — 책자와 다른 엔진 라우팅 |

**★한 줄 결론**: 포토북은 **(표지타입×사이즈 고정 기본가) + (추가 페이지당 증분 단가)** 의 *기본가+페이지가산형*이 권위 모델(`[huni:master]` 명시)이다. 경쟁사(RedPrinting)는 ① PHBK*를 `digital_price`(자재×도수×수량 합산)로, ② 사진북(TPPHSET)을 `tmpl_price`(에디터 완제 단가)로 처리하나, **둘 다 후니의 "기본가 + 페이지 증분" 모델로 무왜곡 환원**된다(아래 §3). 새 가격축(테이블) 신설 0 — 기존 그릇(`product_prices` 고정가 + 페이지 증분 단가행 또는 합산 공식)으로 닫힌다.

## 1. RedPrinting — 포토북은 두 엔진으로 갈린다 `[red:PH·TP]`

RedPrinting PH 카테고리 30상품 중 **④ 포토북 PHBK\* 5종**과 TP의 **사진북 TPPHSET**이 포토북류다. 가격엔진(`price_gbn`)이 둘로 갈린다 `[red:PH §0]`:

| 상품군 | pdtCode | `price_gbn`(실측) | 에디터 | 계산 본질 |
|--------|---------|-------------------|--------|----------|
| 포토북(사진책 제본) | PHBK* (5종) | **`digital_price`** | koi=N·pdf=Y | 사진책 제본 = 면수×사이즈×단/양면 합산 `[red:PH §0]` (`[xref:PR]` 책자와 동형) |
| 사진북 | TPPHSET | **`tmpl_price`** | edicus 에디터 | 페이지계층(INN_PAGE)+제본+에디터 템플릿·완제 통합단가 `[red:TP 그룹D]` |
| (참고) 트윈링책자 | PRBKORD | `book2025_price` | pdf | 표지/내지 부품 분해 매트릭스(책자 권위·포토북 아님) |

### 1.1 ★PHBK* = `digital_price` (사진책 = 면수 기반 합산) `[red:PH §0]`

- PHBK*(포토북 5종)는 **`digital_price`** 엔진 — 디지털 명함/포스터와 같은 라우팅. 즉 RedPrinting은 포토북을 "책자 부품 분해(`book2025`)"가 아니라 **"디지털 출력물의 면수 누적"** 으로 계산한다.
- 본질 `[red:PH §0 ④]`: **"사진책 제본(면수×사이즈×단/양면)"** — 면수(페이지)가 가격에 누적되고, 사이즈·단/양면이 단가를 가른다. `digital_price`의 연속 수량곡선(첫N·증N)을 **페이지수에 적용**한 형태.
- 에디터: koi=N·pdf=Y → 업로드형이나 페이지 단위 디자인. PHBKMYB(포토북)만 레거시 SSR로 옵션 select 일부 노출, 나머지 4종 Vue client-render(`[red:PH §0.5]` `unobserved` 부분 있음).

### 1.2 ★사진북 TPPHSET = `tmpl_price` (에디터 완제 단가) `[red:TP §H-8·그룹D]`

- TPPHSET(사진북)은 **`tmpl_price`** + **edicus 에디터**(`edicus_item`) — 페이지(대수) 계층 + 제본 + 에디터 템플릿. 가격은 **에디터 완제 통합단가**(부품 분해 표면 미노출).
- 수량모델#10 실측 `[red:TP §H-8]`: `quantityGroup.title = {"orderCnt":"디자인 수(건수)", "printCnt":"수량"}` = **이중수량(디자인수 × 부수)**. ★"세트 = 서로 다른 N장 assortment" 구조는 **부재**(H-8 REFUTED) — 사진북은 **같은 디자인의 페이지 묶음**이지 이질 N장 세트가 아니다.
- 페이지 계층: INN_PAGE(MIN/MAX/STEP) — 캘린더/북 공통 그릇 `[red:TP §0.4]`. 내지 페이지수가 입력 차원.

### 1.3 책자 가격분해 로그가 보여주는 부품 합산 (포토북 동형 증거) `[red:book]`

PRBKORD(`book2025_price`)의 priceCalc **분해 로그**가 포토북류 부품 합산의 골격을 그대로 드러낸다(실측·`result_log`):

```
표지인쇄(CVR_PRINT_AMT) 400 = 200/부 × 2부
표지자재(CVR_MTRL_AMT)  0
내지인쇄(INN_PRINT_AMT) 288 = 72/면·부 × 2면(PAGE_CNT) × 2부   ← ★페이지수가 곱해진다
내지자재(INN_MTRL_AMT)  0
제본(BIND_AMT)          2000 (트윈링·세로형 좌철)
표지추가(CVR_ADD_AMT)   500 (PVC 추가커버는 별 PCS 1900)
─────────────────────────────────
result_sum.PRICE = 11500 = ETC(9400=PVC1900+링7500) + PRI(2100=인쇄양면)
```

★**결정적 관측**: 내지인쇄 = **단가(72) × 페이지수(2) × 부수(2)** — 페이지수가 내지 단가에 **곱**해진다. 이것이 포토북 "페이지 가산"의 경쟁사 기계 증거(RedPrinting은 페이지×부수 곱, 후니 master는 페이지 증분 단가 — §3에서 동치 입증). 부품(표지인쇄/표지자재/내지인쇄/내지자재/제본/추가커버)이 **각자 단가를 갖고 합산**(result_sum = Σ).

## 3. 두 모델 → 후니 "기본가 + 페이지가산"으로 환원 (핵심 흡수 정리)

후니 권위 `[huni:master]`는 포토북 가격을 **명시적**으로 정의한다(시트 범례 footnote 실측):

> "상품단가 + 페이지당단가적용" / 가격_기본(24P) + 가격_추가(2P)당

| 사이즈 | 표지타입 | 가격_기본(24P) | 가격_추가(2P)당 |
|--------|----------|---------------|-----------------|
| 8×8 (200×200) | 하드커버 | 15,000 | 500 |
| 8×8 | 레더하드커버 | 23,000 | 500 |
| 8×8 | 소프트커버 | 12,000 | 500 |
| 10×10 (250×250) | 하드커버 | 22,000 | 1,000 |
| 10×10 | 레더하드커버 | 32,000 | 1,000 |
| A5 (148×210) | 하드커버 | 12,000 | 300 |
| A5 | 레더하드커버 | 19,000 | 300 |
| A5 | 소프트커버 | 10,000 | 300 |
| A4 (210×297) | 하드커버 | 16,000 | 600 |
| A4 | 레더하드커버 | 26,000 | 600 |
| A4 | 소프트커버 | 13,000 | 600 |

**포토북 가격공식 (후니 권위)**:
```
포토북 가격 = 가격_기본(24P)[사이즈, 표지타입]
            + ceil((페이지수 - 24) / 2) × 가격_추가(2P)당[사이즈]
            (× 부수)
```

★**관측 규칙**:
- **기본가 = (사이즈 × 표지타입) 2차원 룩업** — 표지타입이 가격을 1급으로 가른다(레더하드 > 하드 > 소프트, 8×8 기준 23,000 / 15,000 / 12,000).
- **추가단가(2P당) = 사이즈만 종속** — 표지타입 무관(8×8=500·10×10=1000·A5=300·A4=600). 큰 사이즈일수록 페이지당 비쌈(면적 비례).
- **기본 24P 포함** — 24면까지는 기본가, 이후 2면 증가마다 추가단가. 내지페이지(편집기) 24~150·STEP 2.

### 3.1 경쟁사 모델과의 동치 (흡수 판정)

| 경쟁사 모델 | 후니 환원 | 동치 입증 |
|-------------|----------|----------|
| **RedPrinting `book2025` 내지인쇄 = 72/면 × 페이지 × 부수** `[red:book]` | 후니 "추가단가 × ((페이지-24)/2)" = **페이지에 비례하는 증분** | ★둘 다 **페이지 선형 증분**. RP는 0면부터 면당단가×전체페이지, 후니는 24면 기본가에 흡수+초과분만 증분 — **affine 변환 동치**(기본가 = 24면분 선·내지·표지·제본 묶음·추가분만 노출). |
| **RedPrinting `tmpl_price` 사진북 = 에디터 완제 통합단가** `[red:TP]` | 후니 `product_prices` 고정가(기본24P) + 페이지 증분 | ★완제 통합단가 = 후니 기본가와 동형(부품 미분해). 후니가 **페이지 증분을 명시**(RP 에디터 내부 산식 미노출) = 후니가 더 투명. |
| **RedPrinting `digital_price` PHBK\* = 면수×사이즈×단/양면 합산** `[red:PH]` | 후니 사이즈×표지 기본가 + 페이지 증분 | ★면수 누적 = 페이지 증분과 동형. 단/양면(도수)은 후니 내지인쇄 사양(몽블랑130 양면 고정 多)으로 흡수 또는 표지타입 variant에 베이크. |
| **WowPress `jobqty0→jobcost0` 작업량 2단** `[wow:booklet]` | 후니 단일 evaluate_price + 앱 페이지 환산 | ★흡수 부결(과분화)·DB 파생저장 = 후니 원칙 위배. 단 "페이지→작업량 환산 규칙"은 앱 계산 도메인 단서(가격축 아님). |

**★흡수 종합**: 세 경쟁사 모델(RP 면수합산·RP 에디터완제·WP 작업량2단) 전부 **후니 "기본가(사이즈×표지) + 페이지 증분(사이즈종속)" 모델로 무왜곡 환원**. 신규 가격축 0 — 책자 종단(C-BK1~6) 결론과 일관. 포토북 고유 흡수 nuance = **"페이지 증분 단가"를 후니 그릇으로 어떻게 표현하느냐**(고정가형 product_prices에 페이지 차원 없음 → 흡수후보 §absorption C-PB1).

## 4. 옵션 의존 캐스케이드 (경쟁사 관측)

| 캐스케이드 | RedPrinting `[red:book·PH·TP]` | 후니 표현 |
|------------|-------------------------------|----------|
| **사이즈 → 추가단가 전환** | 사이즈가 면당단가·제본단가를 가름(`book2025` BIND_AMT는 사이즈 종속) | 후니 추가단가(2P당)가 **사이즈 종속** = 동형(가격 룩업 차원). round-6 CPQ 사이즈→가격 캐스케이드 불요(가격 차원이 직접 처리). |
| **표지타입 → 기본가/제본 전환** | 하드/소프트/링이 제본·표지자재·`COV_MIN_WGT` 전환 `[red:book]` | 표지타입이 기본가를 가름(레더하드 > 하드 > 소프트). round-6 CPQ 표지타입 옵션 → 가격 변형. |
| **자재 → 후가공 비활성** | `pdt_disable_pcs_info`(31행·자재별 후가공 차단) `[red:book]` | 후니 미보유 → 흡수후보(booklet과 공통·C-4 제약·round-6 CPQ constraints). 포토북 신규 아님. |

## 5. 종합 판정 (designer 인계 요약)

1. **포토북 권위 가격모델 = "기본가(24P, 사이즈×표지) + 페이지 증분(2P당, 사이즈종속)"** `[huni:master]` 명시 — Q15 실무진 회신 확정.
2. **계산방식 판정 = ① 페이지수 기반 가산**(과업 3개 후보 중 ① 확정). ② 표지×사이즈 고정 매트릭스는 **기본가에만** 적용(전체 가격 아님)·③ 부품 합산(표지+내지+제본)은 **기본가 안에 묶여**(internalized) 외부 노출 안 됨 → 포토북은 책자(B)의 부품 분해 세트와 달리 **완제 기본가 + 페이지 증분**.
3. **경쟁사 모델 전부 후니 모델로 환원·신규 가격축 0**(RP `digital_price`/`tmpl_price`·WP `jobqty` 모두). 흡수 nuance = **페이지 증분 단가의 후니 그릇 표현**(고정가형 product_prices에 페이지 차원 부재 → C-PB1 흡수후보).
