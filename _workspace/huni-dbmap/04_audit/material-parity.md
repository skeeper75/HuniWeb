## 표기차 브리지(마스터 13 확인필요 검증 결과)
마스터 정합서의 "13 확인필요"를 product-materials 연결 수준에서 검증한 결과 — **전부 표기차/모델링 차이로 확인, 진짜 결손 0**:
| 패턴 | 엑셀 | DB | 판정 |
|------|------|-----|------|
| 백모조 | `백모조220`·`120`·`100` | `백색모조220`·`120`·`100` | **표기차 해소**(색 삽입). 평량 부재 아님 |
| 투명아크릴 두께 | `투명아크릴 3mm`·`1.5mm`·`8mm`·`(3T)`·라미`(10T)` | `투명아크릴`(두께 미보유) | **모델링 차이 해소**: 자재는 존재, DB가 두께 미분리 |
| 골드/실버아크릴 | `골드아크릴 3mm`·`실버아크릴 3mm` | `골드아크릴`·`실버아크릴` | 두께 표기차 해소 |
| PET | `투명PET 260g`·`반투명PET 260g`·`투명PET` | `PET`(일반화) | DB 일반화 통합 — 평량/투명도 정보손실 |

→ Phase-3 "13 확인필요" 결론: **자재 마스터 결손 없음**. 단 **두께(mm/T)·평량(g)·투명도 정보가 DB에서 소실**되는 모델링 차이는 **확인 필요**(가격/사양 구분에 두께가 필요하면 DB 자재 세분화 필요).

## §별도설정 IMPORT 해소 (사용자 지시 반영 — 신규)

**[SUPERSEDED — 이전 "별도설정=실자재 아님(DB 미연결 정당)" 단정을 철회]**
사용자 지시: `*별도설정`은 플레이스홀더가 아니라 **출력소재(IMPORT) 시트 참조 신호**. 실자재가 상품마스터엔 비어있고 IMPORT 시트에 정의됨.

### IMPORT 시트 구조 (실측: `출력소재(IMPORT)`)
- A~K = **종이 마스터** 120행: 대분류·중분류·`종이명`(C)·평량(D)·파일명약어(E)·전지(G)·연당가/가격(H~J)·종이사이즈(K).
- L~Z = **상품 크로스탭** 15개 상품 컬럼: 헤더=상품명(프리미엄엽서·스탠다드엽서·`2단접지카드 / 3단접지카드 / 미니접지카드`·프리미엄명함·`소량전단지/접지리플렛`·중철내지/표지·무선내지/표지·트윈링내지/표지·탁상형/미니탁상형/엽서/벽걸이캘린더), 셀=`●`(해당 종이 선택가능). → `*별도설정` 상품의 실제 소재 = IMPORT에서 `●` 표시된 종이 목록(`import-resolution.csv`).

### EXTRA 60 중 MATCH 전환 수: **0**
- IMPORT 해소로 EXTRA 60이 MATCH로 전환될 것이라는 가설(사용자/본인 예측)은 **불성립**. EXTRA 60은 IMPORT 종이와 무관한 사이즈/형태 변형이며, `*별도설정` 상품(IMPORT 대상)은 DB 자재 0행이라 EXTRA로 잡히지도 않았음. IMPORT 해소는 **MISSING 15를 새로 드러냄**(EXTRA 감소 아님).

## 판정: **MAJOR로 하향 (이전 GO 정정) — IMPORT 실자재 대량 미적재**
- **[SUPERSEDED — 이전 "GO" 철회]**: 이전 GO는 `*별도설정`을 실자재 아님으로 오판한 것에 기반. IMPORT가 별도설정 상품의 실자재를 정의하며, DB는 그 종이를 거의 미적재(8 상품 0행, 7 상품 부분) → **MISSING 15, MAJOR**.
- 엑셀 실값 보유 공통 121상품의 MATCH는 유효(브리지 정당). 그러나 IMPORT 의존 별도설정 상품군의 자재 적재가 결손.
