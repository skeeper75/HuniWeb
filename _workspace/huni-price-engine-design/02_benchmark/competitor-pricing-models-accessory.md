**한 줄 결론**: 후니 "상품악세사리" 시트의 67상품은 대부분 **인쇄 없는 물리 완제 부속물**(봉투·케이스·볼체인·와이어링·천정고리·우드거치/봉/행거·자석고무판·행택끈·만년스탬프 리필잉크)이고, 일부만 인쇄 디자인 들어가는 완제품(거울·뱃지·자석 — 후니 시트엔 미수록·경쟁사엔 있음). 경쟁사 두 곳 모두 이 류를 **"개당 고정단가형"**(레드 `tmpl_price`/부속물#8·와우 `jobcost` unit=개 동적견적)으로 계산하고, 평면 인쇄물의 "본품+가산 합산형"과 **다른 패러다임**이다. 후니 악세사리도 시트가 inline 고정가(OPP봉투 1100원·텀블러식 부품 단가)로 이미 같은 패러다임이다. ★**핵심 = 묶음 단위(50장·3개1팩·2개1세트·100개입)를 어떻게 가격 단위로 다루는가** — 경쟁사는 papername/sizename에 융합(와우)하거나 팩 단위(레드), 후니는 현재 `siz_nm`에 융합(`60x90mm(50장)`)하나 전용 `bdl_qty` 그릇을 이미 보유한다.

## 1. 와우프레스 — 악세사리 동적 견적 모델 `[wow:acc]`

### 1.1 가격엔진 = `/std/prod/jobcost` (동적·단가표 없음)
WowPress 악세사리(자석버튼 40187·거울버튼·캠페인버튼·조각자석·각종 케이스)는 전부 `pricing.source=/std/prod/jobcost`·`seltype=M`(매트릭스/조합형)·`unit=개`. 명함/스티커처럼 정적 단가표가 아니라 **옵션코드 묶음을 동적 견적 엔진에 던져 가격을 받는다**(비선형·합판효율 반영). 정적 카탈로그 캡처에는 옵션 축 구조만 있고 실 단가는 없다(라이브 견적 호출 필요분=`unobserved`·읽기전용 가격조회로만 보강 가능).

### 1.2 구성요소 축 (`raw.prod_info` 7축 → 악세사리에서 실제로 쓰이는 축)
| WowPress 축 | 악세사리 실측 | 의미 |
|------------|------------|------|
| `ordqty` (수량) | 뱃지/거울 1,2,3…10,20,30 개 · 케이스 1~10개 · **조각자석 1000~8000개** | discrete tier(연속 아닌 이산 부수)·"개" 단위·tier마다 단가 다름(볼륨할인) |
| `sizeinfo.sizename` | 거울 `원형58`/`원형75` · 뱃지 `원형32`~`하트57x51` · 명함케이스 `100매용`/`200매용` · 틴케이스 `80x160mm` | **형상+치수 융합**(별 형상축 없음)·케이스는 **용량형**(매수 수납량) |
| `paperinfo.papername` | 거울 `유포지(유광) 개별포장있음`/`…없음`(4 variant) · 뱃지 `유포지150g` · 틴케이스 `스틸` · 캔버스 색별 | **소재+질감(유광/무광)+포장유무 합성 SKU**·재질이 본체 정체 |
| `optioninfo` | 자석버튼 = **none**(빈 슬롯) | 악세사리 대부분 추가옵션 없음(순수 본체) |
| `coverinfo` | covercd=0 `통합`(단일) | 다부품 아님(케이스/뱃지=단일 본체) |

### 1.3 ★관측된 계산방식 = **개당단가 동적 견적**
```
WowPress 악세사리 가격 = jobcost( 형상/규격(sizeinfo) × 소재+포장(paperinfo) × 수량tier(ordqty) )
                       = 개당단가(조합별) × 수량   # 동적·비선형·합판효율
```
- **포장유무가 가격 축**: 거울버튼 `개별포장있음/없음`이 papername에 인코딩 = 포장이 단가에 영향(별 옵션 아니라 재질 variant로 융합).
- **유광/무광이 가격 축**: 표면 마감이 papername variant.
- **케이스는 용량형 사이즈**: `100매용/200매용`(수납 매수) = 치수 아닌 용량이 규격축.

### 1.4 WowPress 6축 권위 모델 `[wow:model]`
악세사리도 WowPress의 일반 6축(지종/규격/도수/후가공/수량/포장)으로 환원되며, **"새 축 만들지 말고 6 의미축으로 흡수"** 규칙이 그대로 적용된다(악세사리가 새 가격축을 요구하지 않음).

## 2. 레드프린팅 — 부속물/완제 개당단가 모델 `[red:GS]`·`[red:OT]`·`[red:meta]`

### 2.1 부속물 = 부속물#8(addons) + 완제 개당단가
RedPrinting 17축 메타모델에서 **거치대/끈/링/지퍼 같은 부속물 = #8 부속물 축**(`t_prd_product_addons.tmpl_cd`→독립 SKU 다중귀속) `[red:meta]`. PD(스툴/구조물) 종단에서 등신대 거치대(CDL_DFT)·피켓이 **부속물#8 + 형상#17**으로 흡수됨이 확정. 후니 악세사리의 우드거치대/우드봉/우드행거/천정고리/볼체인/와이어링/행택끈이 정확히 이 류다.

### 2.2 완제 부속물 = `tmpl_price` 개당단가 `[red:GS §0.1]`
완제 굿즈/부속물은 `DIR_MTR`(부자재직접) 항목 하나가 PRICE 주체 = **개당 고정단가**(부품 합산 아님). 소재/색/용량/형상이 완제 SKU 라벨에 융합되고 가격은 그 완제품의 개당단가다.

### 2.3 색·소재·형상 인코딩
- **색** = DTL variant(볼체인 오렌지/핑크/핫핑크/민트그린… = 후니 시트와 동형·색별 동일가 1000원) `[red:GS §3]`.
- **소재** = 코스터 6소재 = 6 별도 pdtCode(소재가 본체 정체) `[red:GS §9]`. 후니 여권케이스(말랑/레더)·카드봉투(화이트/블랙)가 동형.
- **형상** = 완칼 칼틀 DTL(THO_CUT)·형상↔치수 1:1 캐스케이드 `[red:GS §5]`.

### 2.4 봉투 상자(OTPKENV) `[red:OT]`
RedPrinting 봉투/상자류는 **평면 인쇄지(접지 가공)로 출고·입체 조립은 고객 수작업**. distinct 0(17축 재포화)·전개도/dieline조차 새 축 아니라 사이즈#13(work/cut)+공정#2(도무송/오시)+카테고리#7로 무왜곡 흡수. 후니 봉투(OPP봉투·카드봉투·캘린더봉투)는 후니 시트에서 **완제 부속물(50장 묶음 고정가)** 로 단순하나, 만약 후니가 자체 인쇄 봉투를 만들면 이 패턴(접지 공정 + 사이즈)이 흡수처.

### 2.5 관측된 계산방식
```
RedPrinting 부속물 가격 = 완제 부속물 개당단가(tmpl_price·색/소재/형상/용량 SKU)
                        + (부속물#8 addons — 거치/끈/링 추가시)
                        × 수량(PRN_CNT)             # 부품 합산 아님·개당단가
```

## 3. 후니 악세사리 시트 분석 (대조 기준) `[huni:acc]`·`[huni:live]`

### 3.1 시트 구조 (67상품·2 구분)
- **봉투/케이스** 구분: OPP접착봉투·OPP비접착봉투·트래싱지/트레싱지 카드봉투·카드봉투(화이트/블랙)·캘린더봉투·볼체인(8색)·와이어링(3색)·천정고리·투명케이스·행택끈(3종)·자석고정용고무판.
- **상품액세서리** 구분: 우드거치대·우드봉(3길이)·우드행거(3길이)·만년스탬프 리필잉크(7색).
- 각 행 = `사이즈(필수)`(규격+묶음·`70x200mm(50장)`)·`수량(필수)` 최소1/최대100/증가1·`가격`(inline 고정가).

### 3.2 ★현 후니 계산방식 = inline 고정가 (개당/묶음당 단가)
- **OPP봉투**: `70x200mm(50장)` 1100원·`230x350mm(50장)` 3250원 → **사이즈(=면적)별 묶음단가**. 묶음수 50장은 siz_nm에 융합.

- **우드봉/행거**: 길이별 단가(270mm 7000·360mm 9800·480mm 12000) = **길이=사이즈 variant 단가**.
- **만년스탬프 리필잉크**: 색 7종 모두 `(5cc)` 2500원 동일(색=무료 직교·용량 5cc 고정).

### 3.3 ★라이브 가격사슬 = 거의 전무 `[huni:live]`

## 5. 종단 결론

- 경쟁사 악세사리 계산방식 = **개당 고정단가형 한 패러다임**(레드 완제/부속물·와우 jobcost unit=개). 평면 인쇄물의 합산형·면적매트릭스와 다름. 후니 악세사리 시트도 이미 inline 고정가로 같은 패러다임 → **신규 가격축 0건**(6종단 누적 결론 + rpmeta 17축 재포화 + WowPress "6축 흡수" 규칙 정합).
- 후니 실질 갭 = **(a) 가격사슬 미설계**(라이브 product_prices 0행·고정가형 공식/단가행/바인딩 신규) + **(b) 묶음 단위(50장/3개1팩/2개1세트/100개입)를 `siz_nm` 융합에서 전용 `bdl_qty` 그릇으로 정규화할지 결정** + **(c) 부속물(우드거치 등) addons 적재**.
- 흡수 후보·세트 패턴은 동반 파일 `absorption-candidates-accessory.md`·`set-pricing-patterns-accessory.md` 참조. 채택은 designer/validator(E1~E7) 권한.
