## 1. 엽서 가격표 식별 — silsa와 가격권위 구조가 다름

### 1.1 silsa(일반현수막)의 가격권위 = 외부 면적매트릭스 (별도 가격표, 미대조됨)
- silsa 가격권위 = `price-poster-sign-l1.csv` **B26 일반현수막 블록**(면적매트릭스형 5×16, +가공/추가옵션 추가가격 J/K/M/N 컬럼).
- silsa 시트 자체에는 inline 가격이 없고, **포스터/사인 가격표라는 *다른 시트*가 권위**. silsa designer가 이 시트를 대조하지 않아 사이즈를 비치수 연속범위로 오판하고 옵션 추가가격을 누락 — 이것이 `silsa-price-table-gap.md` 결함.

### 1.2 엽서(프리미엄엽서)의 가격권위 = 디지털인쇄 원자합산형 엔진 (별도 면적 가격표 없음)
- 프리미엄엽서는 **디지털인쇄 상품**. `digital-print-l1.csv` 프리미엄엽서 7행의 **가격공식 컬럼**이 권위를 명시:
  - row1(75x100) 가격공식 셀 = `[출력] + [소재] x [제작수량/판걸이수] + [후가공]` = **원자합산형(SUM)**, 면적매트릭스가 아님.
- 즉 엽서 가격권위 = **상품마스터 디지털인쇄 시트 inline 공식 + 디지털인쇄비/코팅/용지/후가공 단가표**이지, silsa 같은 외부 면적 가격표가 아니다.
- 그리고 이 엔진은 **이미 산출·적재됨**: `02_mapping/digital-print-engine/`(설계+검증 GO) → 라이브 308행 COMMIT(메모리 `dbmap-digitalprint-atomic-formula-unbuilt`).
- **엽서용 별도 면적 가격표는 미식별이 아니라 부존재가 정답** — 엽서는 면적과세형 상품이 아니므로(SUM형). (`price-postcard-book-l1.csv`는 "엽서북(떡메)"=별개 상품. 프리미엄엽서 PRD_000016 아님.)

## 2. 파일럿 사이즈·옵션 ↔ 가격엔진 정합 대조

### 2.1 사이즈 — 이산 7행 정합 (오판 없음)

| 점검 | silsa(결함) | 엽서(정합) |
|---|---|---|
| 사이즈 모델 | 비치수 **연속범위** 가로500~1750·세로500~5000 (R-SIZE-NONSPEC) | **이산 7행** SIZ_000001~007 (라이브 차원행 직접 참조) |
| 권위 근거 | 상품마스터 비규격 컬럼만(가격표 미대조) | 상품마스터 디지털인쇄 7행 = 라이브 7 size 차원행과 1:1(73x98·98x98·100x150·135x135·95x210·110x170·148x210) |
| 가격 연동 | 면적매트릭스 미통합 | **판수(15/12/8/6/6/4/4)가 7사이즈 전체에 R-QTY-PANSU constraint로 인코딩** + 공식의 `÷판걸이수` 항으로 가격 연동 |

→ 엽서 파일럿은 사이즈를 라이브 이산 차원행으로 정확히 잡았고, 각 사이즈의 판수(판걸이수)까지 가격공식·제약에 반영. **silsa의 "연속범위 오판"과 정반대.**

### 2.2 옵션 추가가격 — 누락 아님, 가격엔진으로 명시 위임

PRF_DGP_A 구성요소(`t_prc_formula_components_DGP.csv`, **29 components**)가 파일럿 옵션과 **전수 대응**:

| 파일럿 옵션(option layer) | 대응 가격 구성요소(PRF_DGP_A) | 단가 소재 가격표 |
|---|---|---|
| 도수 단면/양면(OG-DOSU) | COMP_PRINT_DIGITAL_S1 / _S2 | 디지털인쇄비(국4절) `price-digital-print-price-l1.csv` |
| 모서리 직각/둥근(OG-MOSEORI) | COMP_PP_CORNER_RIGHT / _ROUND | 후가공 `price-post-process-l1.csv` B01(모서리 귀돌이) |
| 후가공 오시 1~3줄(OG-HUGAGONG) | COMP_PP_CREASE_1L/2L/3L | 후가공 B03(오시 합가) |
| 후가공 미싱 1~3줄 | COMP_PP_PERF_1L/2L/3L | 후가공 B02 |
| 후가공 가변텍스트 1~3개 | COMP_PP_VARTEXT_1EA/2EA/3EA | 후가공 B04(가변텍스트) |
| 후가공 가변이미지 1~3개 | COMP_PP_VARIMG_1EA/2EA/3EA | 후가공 B05(가변이미지) |
| 종이(OG-JONGI) | COMP_PAPER | 용지비(import-paper) |
| (별색 그룹 미인스턴스화) | COMP_PRINT_SPOT_* 10종 | 별색 인쇄비 |

→ 파일럿 R4가 **명시 선언**: "`t_prd_templates.price` 컬럼 라이브 부재 → 템플릿에 가격 없음(**가격엔진 t_prc_* 연계**)". 즉 옵션 추가가격을 **누락한 게 아니라 L1 가격트랙(PRF_DGP)으로 의도적으로 위임**. 이는 L1(가격)/L2(옵션구조) 분리 원칙의 정상 동작(메모리 `dbmap-cpq-option-layer-mapping`).

## 4. 근본 차이 — 왜 엽서는 안전했나

| 축 | silsa | 엽서 |
|---|---|---|
| 가격유형 | 면적매트릭스형(외부 포스터/사인 가격표) | 원자합산형 SUM(디지털인쇄 엔진) |
| 가격권위 위치 | **다른 시트**(포스터/사인 B26) — 대조 누락하면 안 보임 | 상품마스터 inline 가격공식 + 이미 적재된 PRF_DGP 엔진 |
| 파일럿 작성 시 가격엔진 존재 | 가격트랙 미연계(designer가 상품마스터만 봄) | **PRF_DGP 엔진 선존재**(308행 적재) → 파일럿이 명시 참조(R4) |
| 핵심 차이 | L1 가격이 *별도 면적 가격표*에 있는데 L2가 그걸 안 봄 | L1 가격이 *이미 구축된 SUM 엔진*에 있고 L2가 그걸 위임 인지 |

> **방법론 함의(silsa-gap §3 교훈의 반례):** silsa-gap은 "L2 설계는 반드시 L1 가격표(해당 상품군)와 대조해야 하며 엽서도 재점검 필요"라 경고했다. 본 재점검 결과 **엽서는 해당 경고의 위반 사례가 아니다** — 엽서의 L1(PRF_DGP)이 면적 가격표가 아닌 SUM 엔진이고, 파일럿이 이를 R4에서 인지·위임했기 때문. silsa 결함은 "면적매트릭스 상품을 비면적으로 오판"한 **상품유형 오인**이 근인이며, SUM형 엽서엔 그 오인 여지가 없다.

## 5. 잔존 항목 (엽서 — 결함 아닌 기존 GAP, 가격 무관)

silsa 동형 결함은 아니나, 파일럿이 자체 기록한 미해결(가격표 미대조와 무관):
- BLOCKED 5행(종이 mat_cd·후가공 PROC_029~032 차원 0행) = **L1 차원 미선적재**(GAP-DEFER) — 가격 누락이 아니라 옵션 참조대상 차원행 미존재.
- D-1 후가공 줄수 보존처(ref_param_json 라이브 부재), D-4 ★사이즈선택 동적 addon(GAP-B), 봉투 가격 보관처([CONFIRM] R4).
- 이들은 silsa의 "가격표 미대조" 계열이 아니라 스키마/차원 적재 계열.
