# digital-print 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation 파일럿)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 **누락·오매칭·모호**가 생겼다.
> 본 문서는 그 결함을 *올바로 처리(수정)*하기 위해, **라이브 Railway DB 직접 read-only SELECT를 권위**로
> digital-print 상품군을 종단 규명한다. 도메인 지식(07_domain KB)은 결함을 옳게 분류하는 렌즈다.
> 정제된 DB가 인쇄 자동견적 위젯의 기반이 된다.
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본 `ref-*.csv` 2026-06-04은 stale 가능).
> 추정·임의판단 0 — 라이브·엑셀 L1·도메인 KB·PDF 실재 근거만. 미지/추정은 가설+컨펌 플래그.
> INSERT/UPDATE/DDL 0. DB 미적재 유지(본 문서는 처리 *방향* 설계). `db railway`·비표준포트·JOIN KEY=`prd_nm`.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. **라이브 SELECT 수행 12회**(전부 read-only).

## ⚠️ 머리말 — 본 검증이 v2 대비 뒤집은 가장 큰 사실 (권위 반전)

**v2 회귀게이트(`04_audit/v2`)는 stale ref-CSV(2026-06-04) 기반이라, 라이브 현재 상태와 어긋난다.**

라이브 직접 확인 결과:
- **프리미엄엽서(PRD_000016)는 PROC_000029·030·031·032(오시·미싱·가변텍스트·가변이미지)를 *이미* 적재**하고 있다.
  v2는 이를 "MISSING 4건"으로 보고했으나, **라이브에서는 해소됨**.
  → v2의 R-PROC-2 핵심 결함 사례 1건은 **라이브에서 false(이미 처리됨)**.
  → **[사용자 확인 2026-06-05]** 016은 "상품별공정 매핑이 안 돼서 **수동으로 넣어본**" 시험 적재다(자동 매핑·의도된 레퍼런스 아님).
  따라서 R1에서 016을 무비판 레퍼런스로 복제 금지 — **016 자체가 엑셀 의도대로 올바로 들어갔는지 검증 선행** 후 패턴화.
  핵심 문제는 **각 시트 상품의 기초데이터 누락 다수**이며, 그 누락을 제대로 모르면 치명적이라 본 검증·매핑 하네스로 접근 중.
- 그러나 **동일 패밀리 결함은 다른 12개 상품에 그대로 잔존**한다(프리미엄엽서만 단독 처리된 "예시 1건" 상태).

**교훈(HARD 재확인):** 결함 검증은 반드시 라이브로 재확인해야 한다. 추출본을 그대로 신뢰하면
(a) 이미 고친 것을 결함으로 오보, (b) 적재 진행분을 못 봄 — 둘 다 발생. 본 문서의 모든 결함 수치는
**라이브 SELECT로 재산출**한 것이다.

## ④ 결함 표 — 누락(MISSING)·오매칭(MISMATCH)·모호(CONFIRM) (도메인 렌즈, 라이브 재확인)

> 분류 기준: **MISSING** = 정규화 규칙대로 엑셀을 변환하면 적재돼야 하는데 라이브에 없는 행.
> **MISMATCH** = 잘못 연결/모호 매칭. **CONFIRM** = 도메인 의미 미확정(실무 컨펌 대상).
> 모든 판정은 라이브 SELECT로 재확인. v2 수치는 stale → 본 표가 권위.

## ⑦ 큰 그림 적용 (도메인 → 엑셀 → DB결함 → 처리 → 위젯)

- **도메인 KB(07_domain):** `prcs_dtl_opt` 스키마(줄수/개수 max=3)·박/형압 자식 leaf 규칙·별색=공정 분기가
  "엑셀 1줄/2개/박(있음)"을 옳게 해석하는 렌즈가 됨 → 원적재의 오해(옵션값/플레이스홀더)를 결함으로 규명.
- **엑셀 L1(무손실):** 전 컬럼 보존 덕에 줄수/개수 블록·`*별도설정`·봉투 6종을 상품별로 정확 추출(추정 0).
- **DB 라이브 권위:** stale ref-CSV가 만든 false(프리미엄엽서 이미 처리됨·코팅엽서 false EXTRA·박색상 false EXTRA)를
  라이브 12 SELECT로 걷어내고, 진짜 잔존 결함(28+2 공정 MISSING·자재 9상품·addon 3)만 확정.
- **처리방향:** 마스터 무변경 + 상품 연결 테이블만 적재(FK 안전). 프리미엄엽서를 레퍼런스 패턴화.
- **위젯 기반:** 정제된 9속성이 자동견적 위젯의 옵션 캐스케이드(줄수→오시·미싱, 개수→VDP, 별색→공정, 종이→자재)
  데이터 소스가 됨. 결함 잔존 시 위젯 옵션 누락·가격 오류로 직결되므로 R1·R3이 위젯 선결 조건.
