**프리미엄엽서는 잘못 연결된 가격구성요소를 데이터 재배선으로 교정하면 자기 가격공식으로 올바른 가격이 계산된다 — 3중 독립 입증.** 단가 정확성(우리 트랙)은 데이터만으로 닫히고, 최종 청구(출력매수 환산)는 개발자 엔진 경계로 정직히 분리된다.

## 2. 확정된 결함 (구성요소 잘못 연결)

- **F-1 print_opt_cd 축 이중의미**: 가격데이터 print_opt=도수(흑백/칼라), comp S1/S2=면(단/양). 상품 print_opt=면(단/양). 같은 코드 충돌 → 손님 "단면" 선택 시 흑백 단가 매칭. (값은 가격표 verbatim·날조 0)
- **F-2 S1+S2 이중배선**: 단·양면 두 comp가 PRF_DGP_A에 동시 addtn_yn=Y → 면 무관 합산.
- 교정(데이터만·칼라전용) = 단일 인쇄비 comp + print_opt=면(단면→칼라단면 D·양면→칼라양면 E·값 라이브 verbatim 이동·S2 제거). 엔진 코드 무변경.

## 3. ★검증이 생성을 교정한 2건 (개선 루프 Round-2)

- **D-1 (MAJOR)**: arbiter의 "6공식 전역 재키 안전·도수축 불요(ⓑ부결)"는 **흑백만 보고 별색 차원 누락**. PRF_DGP_A 형제 중 **020 화이트인쇄·021 핑크별색·022 금은별색**은 front=CLR_000005(칼라와 동일)이나 권위는 별색 컬럼(@25: 화이트 1250·핑크 1400 vs 칼라 800) 요구. 라이브 단가행 clr_cd 전부 NULL=별색 전달 경로 없음.
  → **적용 스코프 정정**: 프리미엄엽서 교정은 **평칼라 상품(16/17/18/26/41/42)으로 한정**(전역 재키 금지). 별색 3형제(020/021/022)는 **색 차원(칼라+별색5) 구별 경로 별도 설계**(arbiter ⓑ가 별색 한정 부활) = 분리 트랙.
- **D-2 (MINOR)**: `t_prc_component_prices`에 use_yn 컬럼 부재 → 가역 수단 = use_yn 토글 아님. **물리 백업 + apply_ymd 세대분리/undo**(기존 dbmap 관행)로 정정.

## 4. 잔여 경계 (데이터로 안 닫힘·개발자 엔진 — 정직 기록)

- **F-3 출력매수 환산**: pricing.py에 imposition/ceil/판걸이수 전무(검증 재확인). 호출측(앱/위젯)이 주문수량→출력매수=ceil(qty/판걸이수)로 환산해 qty=출력매수로 엔진 공급. (판걸이수는 가격표 시트에만·라이브 미적재→적재 위치 컨펌)
- **F-4 work→plate 임포지션**: 호출측이 work size→plate(국4절/3절)+판걸이수 산출 후 공급.
- → 단가 정확성(우리 트랙·입증됨) + 수량 환산(개발자) 둘 다 닫혀야 end-to-end 골든.
