# 가격테이블 적재 모델 — 4원 대조 확정 (2026-06-29)

> 목적: 인쇄상품 가격표 19시트를 "어떻게 적재할지" 숙고하기 위해, **현재 적재가 어떻게 되어있고 / 그게 가격공식·가격구성요소·가격테이블과 어떻게 연관되는지**를 4개 소스로 교차 확정.
> 소스: ① raw/webadmin 코드 ② 라이브 적재 DB(스냅샷 20260629_2011) ③ webadmin 라이브 화면(가격뷰어 gstack 캡처) ④ 인쇄상품 가격표 19시트(권위).

## 1. 적재 사슬 (4원 일치)

```
상품(t_prd_products)
  └─[바인딩 120] t_prd_product_price_formulas → 가격공식
가격공식 PRF_* (t_prc_price_formulas · 90/활성89)
  └─[배선 156] t_prc_formula_components → 구성요소 묶음
가격구성요소 COMP_* (t_prc_price_components · 173/활성137)
  └─ use_dims = 차원축 정의   ← ★단가표 그리드 컬럼 = use_dims (_comp_dims)
가격테이블/단가행 (t_prc_component_prices · 8,753행)
  └─ use_dims 로 키된 셀(판형·도수·면·수량 등 × 단가)
```

- **코드 근거**: `pricing.py evaluate_price → _evaluate_formula(frm_cd) → TPrcFormulaComponents.filter(frm_cd) → comp.use_dims → _component_rows(comp)=TPrcComponentPrices.filter(comp_cd)` 를 use_dims로 매칭 합산.
- **그리드 근거**: `price_views.py _comp_dims(comp)` = `comp.use_dims`(+ min_qty). 즉 **운영자가 보는 단가표 컬럼이 곧 use_dims**.

## 2. 적재 메커니즘 — 자동 로더 없음

- 가격표 xlsx → t_prc_* **자동 로더 부재**. 운영자가 가격뷰어 → 구성요소 단가표 **그리드(`price_grid_save`)로 수동 입력**(comp별 `/admin/price-viewer/comp/<comp_cd>/edit/`).
- 적재 신호 = **단가표 행수 배지**(`price_component_md`, "0행=미적재"). **거친 신호** — 행수만 봄.

## 3. 현재 적재 상태 (라이브 실측)

- 활성 구성요소 **137개 중 132개가 단가행 보유 · 5개만 0행**(대부분 TBD/실무진 확인필요). → 거친 신호로는 적재율 양호.
- **하지만 진짜 격차 = 적재된 132개 안의 "빠진 셀 / 빠진 차원축"** — 행수 배지로 안 보임.

### ★실증: 디지털인쇄 흑백 collapse (4원 전부 입증)
| 소스 | 증거 |
|---|---|
| ④ 권위 19시트 | 디지털인쇄비 = 도수(흑백/칼라/별색7) × 면(단/양) × 수량. 흑백 단가 존재(국4절 단면 qty1=3000). |
| ② 라이브 DB | COMP_PRINT_DIGITAL_S1 = **212행 (전부 칼라값)**. use_dims=`[proc_cd,plt_siz_cd,print_opt_cd,min_qty]` — **도수축 없음**. 흑백 단가 0행. |
| ③ 라이브 화면 | DIGITAL_S1 그리드 컬럼 = 판형·인쇄옵션(단/양)·수량 — **도수(흑백/칼라) 칸 없음**. 212행 비고만 "칼라(CMYK)". 배지 "212행" = DB 일치. |

**비고**: 행수 배지로는 "212행=정상"으로 보임 → 배지는 이 결함을 못 잡음. 메모리 `a2-price-conformance-remediation` 돈크리티컬과 일치(독립 재적발).

## 5. 함의 — 19시트 적재 진단은 2층 정합

1. **셀 정합** — 권위 격자의 모든 셀이 단가행에 있나(미적재/불일치). 배치 빌더 grid-diff.
2. **★차원축 정합** — 구성요소 use_dims가 시트의 가격축을 담는가(디지털 흑백처럼 축 자체가 빠지면 그리드에 칸이 없어 권위 데이터가 들어갈 곳이 없음). 이건 **셀 적재가 아니라 use_dims 재설계(§18)** = search-before-mint·인간 승인.

→ 적재 전 "전 19시트 셀+차원축 진단"이 선행(읽기전용·토큰0·COMMIT 없음). 배치 빌더 3 어댑터 패밀리로 수행. 결과 결함보드를 보고 (a) verbatim 셀 적재분 (b) 차원축 재설계분(§18/dbmap·인간 승인)으로 분기.
