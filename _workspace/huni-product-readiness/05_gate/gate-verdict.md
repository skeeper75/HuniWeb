## Q1 — 분모 누락 0 ✅ GO

- 척추(`product-spine.csv`) 실상품 = 라이브 등록 276 + 엑셀-only 미등록 7 = **283**. scorecard도 정확히 **283행**(276 prd_cd + 7 "(미등록)").
- 미등록 7 = 링바인더(보류중)·버즈케이스·블랙젤리·슬림하드폰케이스·에어팟케이스·임팩트젤하드·투명포스터 → scorecard L0 7건과 1:1 일치.
- 비상품 주석행(그레이배경 범례·Figma 안내 등 5~6행)은 분모에서 정당 제외.
- 라이브 `t_prd_products` count = **276**(재실측 일치).
- **누락·중복 0 — GO.**

## Q2 — 등급 근거 정합 🟡 GO(보정)

- L3/L4 83건은 "계산 가능" 게이팅(D5 PASS). 직접 재실측 결과 **계산은 실제로 됨**(Q3) → codex A1의 강한 함의("과대평가=견적 0 위험")는 **반증**.
- 다만 루브릭 D5 PASS의 엄밀 정의 = "PRICE≠0 **AND 골든 일치**". scorecard 근거 컬럼에 골든 대조 흔적 **0/83**(Claude·codex 합의) → **골든 정합은 미검증 상태**.
- **판정**: L3+ 83 등급을 L2로 일괄 내리지 **않음**(계산이 되는데 "계산 0/부분"이라 적는 건 사실 왜곡). 대신 **golden_status="골든 미대조"** 플래그를 데이터에 부착(투명성). 골든 전수대조는 후속(§26 pr_score·§15).
- **STALE 플래그 반증**: 아크릴 147~152·문구 다이어리류의 `pfm=LIVE_UNBOUND`/`DESIGNED_NOT_LOADED`는 옛 스냅샷 잔재 — 라이브 PRICE>0로 직접 반증(codex B3 "환각 PASS" 가설 기각).

## Q4 — 구성요소/오매핑 실재 ✅ GO(메커니즘 정정)

- 오매핑 리스트의 020·042·047(+034·037·040·048) "외래자재 오염" = **라이브 실재 확인**. 화이트인쇄엽서(020)·프리미엄쿠폰(042)·소량전단지(047)에 면끈·젤리볼펜·지비츠부속·핀버튼·미니배너거치대 등 **다른 상품 자재가 dflt_yn=Y로 등록**됨(용지 드롭다운 오염).
- **단, 메커니즘 정정**: 이 외래자재는 `COMP_PAPER`(용지비)의 단가행으로, use_dims=`[plt_siz_cd, mat_cd]` — 즉 **mat_cd로 구분되어 손님이 그 자재를 골라야만 가산**. "출력만 해도 ×수량으로 매번 silent 합산"하는 구조는 **아님**.
- 실제 위험 = ① 손님 용지 드롭다운에 엉뚱한 자재 노출 ② **잘못된 기본값**(여러 dflt_yn=Y) → 엔진이 틀린 자재행을 집어 오·과청구 가능. 돈영향은 있으나 "×1000 밴드총액 폭주"는 아님 → **분류를 "자재 드롭다운 오염+다중 오기본값(D7/D11)"로 정정**(라벨 "진짜 silent합산 돈크리티컬"은 메커니즘상 과장).
- **결론**: 오매핑 리스트는 날조 아님(GO). 단 위젯 얹기 전 정리 필수(Q6에서 위젯 제외 처리).
- FP 가드 적절성: 136·138·139(실사 자기부품 우드거치대/설치용끈/큐방)는 WARN 유지가 옳음(메모리 RC-2에서 addon 처리됨) — codex B2 재승격 보류.

## Q6 — 일정 선행 정합 🟡 GO(보정)

- 위젯 wave는 전부 L3(계산 가능) 토대 위 — 정상.
- **단 스케줄러의 핵심 보증 "돈크리티컬 silent합산은 전부 L2 이하로 자동 차단됨"은 020/042/047에 거짓**(codex C1 채택). 셋은 D11=FAIL인데 D5=PASS라 L3로 남아 W2(디지털 캐스케이드) wave에 산입됨 — **자재오염이 위젯에 얹힐 구조적 누수**.
- **게이트 보정**: 데이터에 **widget_eligible** 플래그 신설 — D11=FAIL(검증된 오염)인 020/042/047은 **N**(위젯 보류). W2 디지털 19 → **16**. 위젯 착수 가능 83 → **80**.
- 등급은 L3 유지(계산은 됨)·위젯만 보류(정리 선행).

## Q7 — codex 수렴 + 생성검증 독립성 ✅ GO

- 게이트는 codex/평가자 결론을 복사하지 않고 **라이브 DB(체인 SQL)·인증 시뮬레이터(13표본)·plate 재실측**으로 독립 재판정.
- codex 합의 6건 처리: A1=부분반증(계산됨)+부분채택(골든 미대조)·A2/C1=채택(위젯 누수→제외)·A4/A5/A6=유지·B3=반증(STALE 플래그)·B5=해소(위협 아님)·B2=보류(addon 처리됨).
- codex 가설을 라이브 확정 전 채택하지 않음(환각 경계 준수).
