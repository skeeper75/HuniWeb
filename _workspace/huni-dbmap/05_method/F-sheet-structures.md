## ② 시트별 상세 (전 컬럼 인벤토리 + 블록구조 + 특이점 + 추출 누락위험)

표기: 컬럼레터 `[row1 그룹명] / [row2 하위컬럼명]`. 헤더 원문은 영어/원문 그대로 유지.

### 1. 계산공식집초안 (비상품 — 가격공식)

- 병합 0. r1~2 공백, r3부터 데이터. B=공식유형+상품열거(`원자합산형: 프리미엄엽서 / 코팅엽서 / ...`), C=`참조시트명`. round-2 가격엔진 권위. 9속성 무관.
- **누락위험**: 본 시트는 9속성 추출 대상 아님. 단 round-2에서 공식유형(원자합산/고정가/면적)·참조시트 매핑의 권위이므로 가격 작업 시 별도 파싱 필요.

### 2. MAP (비상품 — 카테고리 트리)

- 병합 0. r1=대카테고리 12개 가로(`01 엽서`~`12 포장`), r2=중분류(`▶︎엽서` 등), r3↓=각 컬럼 아래 상품명 세로 리스트.
- **가로 12컬럼 × 세로 상품명** 구조 — 다른 시트의 세로블록과 정반대 축. 카테고리↔상품 매핑 참조표.
- **누락위험**: 고정기준(D=상품명) 적용 불가(D=`04 포스터`). 카테고리 매핑이 필요하면 컬럼=카테고리, 행=상품 으로 별도 파싱.

### 3. 디지털인쇄 (A형, maxcol AR)

전 컬럼:
```

F [ ]/[판수]  G [파일사양]/[블리드]  H [ ]/[작업사이즈]  I [ ]/[재단사이즈]
J [ ]/[출력용지규격]  K [ ]/[파일명약어]  L [ ]/[출력파일]  M [ ]/[폴더]
N [주문방법(필수)]/[업로드]  O [ ]/[편집기]  P [종이(필수)]  Q [인쇄(옵션)]
R~V [별색인쇄(옵션)]/[화이트][클리어][핑크][금색][은색]
W [코팅(옵션)]  X [커팅(옵션)]  Y [접지(옵션)]
Z [제작수량(필수)]/[건수(옵션)]  AA [ ]/[최소]  AB [ ]/[최대]  AC [ ]/[증가]
AD [후가공(옵션)]/[모서리]  AE [ ]/[오시]  AF [ ]/[미싱]  AG [ ]/[가변(텍스트)]  AH [ ]/[가변(이미지)]
AI [박(앞면)/박(뒷면)/형압]/[박/형압 가공]  AJ [ ]/[크기]  AK [ ]/[박칼라]
AL [추가상품(옵션)]/[추가상품]  AM [ ]/[추가가격]  AN [가격공식]
```
- **블록구조**: 프리미엄엽서 r3~r9(사이즈 7종 세로). 옵션 컬럼들이 행별 독립 ragged: 사이즈(E)는 7행, 인쇄(Q)는 단면/양면 2행, 후가공 모서리(AD) 직각/둥근, 별색(AE오시·AF미싱·AG·AH)은 1줄/2줄/3줄 식 세로 누적. **AL(추가상품)은 봉투류가 사이즈와 무관하게 세로로 더 길게 나열**(r3~r8, 사이즈보다 행수 많음).
- **특이점**:
  - 별색 5컬럼(R~V)은 row1 병합(`R1:V1`)에 묶임. row2가 실제 컬럼명.
  - Z의 row2 = `건수(옵션)` — 묶음수 신호(건당 묶음). AA/AB/AC가 제작수량 최소/최대/증가.
  - W(코팅)·X(커팅)·Y(접지) 각 1컬럼 = 각각 단일 공정축(택일그룹 후보).
  - AN(가격공식)은 footnote상 "무시" 지시(`AN열은 무시해주세요`)이나 round-2 가격엔진 권위로 보존가치.
  - C258 footnote 병합(`C258:AK258`): 노랑배경=신규(MES미등록), ★=제약, 그레이글씨=비노출, *오렌지=개발무시 코멘트, 종이는 MES통합관리(별도설정 무시) — **셀 배경색이 의미를 가짐**(openpyxl data_only로는 색 미취득, 확인 필요 시 fill 별도 파싱).
- **누락위험**:
  - 고정기준(E만 추출) 시 **F(판수)·H(작업사이즈)·I(재단사이즈)·J(출력용지규격) 행내 동반값 소실** → 사이즈 행이 작업/재단/판형을 가졌는지 판정 불가(실사 false-MISSING과 동일 메커니즘).
  - 별색 5컬럼을 단일 인쇄옵션으로 평면화하면 화이트/클리어/핑크/금색/은색 구분 소실.
  - AL(추가상품)의 ragged 길이가 E보다 길어 **블록경계를 E기준으로 잡으면 추가상품 일부 누락**.

### 4. 스티커 (A형, maxcol AK)

전 컬럼: A~M 디지털인쇄와 동일 backbone. N[주문방법]/[업로드] O[편집기] P[종이] Q[인쇄] R~V[별색5] W[코팅] X[커팅] **Y[조각수(옵션)]** Z~AB[제작수량 최소/최대/증가] AC[후가공]/[가변(텍스트)] AD[가변(이미지)] AE[추가상품] AF[추가가격].
- **블록구조**: 반칼자유형스티커 r3~r7(사이즈 A6/A5/A4 3행 + 종이 5행 + 출력파일 4행 ragged 불일치). 종이(P)가 사이즈보다 행 많음(유포/비코팅/미색/무광코팅/유광코팅 5종).
- **특이점**:
  - Y=`조각수(옵션)` — 묶음수 컬럼(`*최대20조각`/`*최대40조각` 텍스트값). 사이즈와 동축 아님.
  - G=`판수`가 아니라 `블리드`(F=판수, G=블리드). row2 확인 결과 F가 판수.
  - L(출력파일)에 `AI_CS5 (칼선)`·`*아이마크`·`*칼선분리` 같은 **파일사양 제약 텍스트**가 ragged로 추가.
- **누락위험**: Y(조각수) 텍스트가 `*최대20조각` 형태 — 숫자 묶음수가 아닌 제약문구. 고정기준으로 묶음수=정수 가정 시 파싱불가/누락. 디지털인쇄와 동일하게 H/I/J 행동반값 소실 위험.

### 5. 책자 (A형, maxcol AT, **내지/표지 2축**)

전 컬럼:
```
A~E backbone  F~L [내지파일사양](F=MES,G=블리드,H=작업,I=재단,J=약어,K=출력파일,L=폴더)

X [표지종이(필수)]  Y [표지인쇄(필수)]  Z~AA [표지옵션]/[표지코팅][투명커버]
```

- **블록구조**: 중철책자 r3~r5. **2축**: 내지(F~Q)와 표지(R~AD)가 각각 별도 파일사양·종이·인쇄. 페이지룰(O~Q)은 내지 전용. 표지옵션(Z표지코팅 ragged 코팅없음/무광/유광).
- **특이점**:
  - F·R의 row2 = `MES ITEM_CD`(내지/표지별 자체 MES코드) — backbone C의 MES와 별개.
  - AC(박 크기)에 `크기(가로x세로)\n★최소 30x30 / 최대170x170` 줄바꿈 제약텍스트.
  - AQ(COMMENT)에 `중철책자 표지 AI / 내지 AI` 제작메모.
- **누락위험**:
  - **내지/표지 2축을 단일 자재/인쇄로 평면화하면 한 축 전손실**(M내지종이 vs X표지종이, N내지인쇄 vs Y표지인쇄). 1차에서 표지축 누락 위험 큼.
  - 페이지룰(O/P/Q)은 내지에만 존재 → 표지 행과 혼동 금지.
  - 작업/재단사이즈가 F~L(내지)·R~W(표지) 두 곳에 분산.

### 9. 실사 (A형, maxcol AO, 비규격 + 인라인가격) — **false-MISSING 발원 시트**

  - **작업사이즈(I)가 헤더상 H가 아닌 I열**. row2 재확인: H=블리드, I=작업사이즈, J=재단사이즈. (※ 디지털인쇄는 H=작업/I=재단 — **시트마다 작업/재단 컬럼 위치가 다름**.)

- **누락위험 (사용자 우려 직접 대상)**:
  - **1차가 E(사이즈명)만 떠서 H(블리드)·I(작업)·J(재단)를 버림** → "A1 사이즈명 있으나 작업사이즈 공백"으로 오판, 포맥스보드 A1 false-MISSING 발생. 실제론 작업사이즈가 I열에 `594 x 841` 식으로 존재했고, H=0(블리드)을 작업사이즈 공백으로 착각. **추출 시 E·F·G·H·I·J·K를 한 행 단위로 묶어서 떠야 함**.
  - 사이즈명이 `A3 (297 x 420 mm)`처럼 규격+치수 복합, `사용자입력`은 비치수 → 단일 파싱규칙 불가.
  - H=0(유효 0mm) vs 공백(미정의) 구분 필수 — `0`을 결측으로 치환 금지.

### 공통 backbone
- **A~E 공통**: A=구분(대분류 그룹, 일부 시트만), B=ID, C=MES ITEM_CD(전부 또는 일부 NULL — JOIN KEY는 prd_nm), D=상품명, E=사이즈(필수). 전 13시트 동일(MAP 제외).
- **제작수량 3컬럼**: `제작수량(필수)` 그룹 = 최소/최대/증가 — 전 상품시트 보유(컬럼 위치만 시트별 상이: 디지털 AA~AC, 스티커 Z~AB, 책자 AK~AM, 실사 AA~AC, 아크릴 Y~AA, 문구 AD~AF, 굿즈 T~V, 악세사리 F~H, 포토북 AE~AG, 캘린더 V~X, 디캘 U~W).
- **파일사양 군**(블리드·작업사이즈·재단사이즈·출력용지규격·파일명약어·출력파일·폴더): 대부분 시트에 존재하나 **컬럼 레터·순서·심지어 작업/재단 위치가 시트마다 다름**.
- **footnote 병합셀**: 상품시트 대부분 시트 중간(데이터 끝~)에 `C258`·`C174` 등 안내문 병합(노랑=신규/그레이=비노출/★=제약/*오렌지=무시). 셀 **배경색이 의미** 보유.

## ④ "동일 고정기준 추출 시 누락될 컬럼/관계" 목록 (사용자 우려 직접 응답)

1차처럼 "속성별 단일 고정 컬럼만 평면화"하면 아래가 소실된다. 시트별 헤더 row2를 읽어 **행 단위(전 컬럼)로 떠야** 막힌다.

1. **작업사이즈·재단사이즈·블리드·출력용지규격·판수(행 동반값)** — 사이즈(E)만 추출하면 같은 행의 H/I/J(작업·재단·판형)를 전부 버림. → **실사 포맥스보드 A1 false-MISSING의 직접 원인**. 작업사이즈가 공백이 아니라 다른 컬럼(I)에 있었고, H=0(블리드)을 작업사이즈 공백으로 오인.

2. **H/블리드의 `0`(유효 0mm) vs 공백(미정의) 구분** — 실사 H=0은 정상, 투명포스터★은 H/I/J 전부 공백(미정의/준비중). `0`을 결측 치환하면 둘을 못 가림.

3. **작업/재단 컬럼 레터의 시트별 차이** — 고정 "H=작업사이즈" 가정 시 실사(I=작업)·아크릴(J=작업)에서 엉뚱한 컬럼(블리드) 추출.

4. **내지/표지 2축 한 축 전손실** — 책자·포토북·문구에서 단일 자재/인쇄 컬럼만 떠면 표지(또는 내지) 종이·인쇄 통째 누락.

5. **별색 5컬럼(R~V) 평면화** — 디지털·스티커의 화이트/클리어/핑크/금색/은색을 단일 인쇄옵션으로 합치면 5종 구분·각 단/양면 소실.

6. **사이즈별/옵션별 인라인 단가** — 실사 R/S/V, 아크릴 H/V, 굿즈 P/Q/S, 포토북 AK/AL. 사이즈만 떠면 사이즈에 박힌 단가 소실(round-2 가격 매핑 결손).

7. **구간할인적용테이블(문구 AJ, 굿즈 Y)** — 단일속성 추출에서 빠지면 상품↔할인테이블 연결 권위 소실.

8. **ragged 블록경계(추가상품/별색이 사이즈보다 긴 컬럼)** — 디지털 AL(봉투), 아크릴 W(볼체인)는 E보다 행 많음. E기준 블록경계로 자르면 초과 옵션 누락 또는 옆 상품으로 흡수.

9. **병합 D 앵커(상품악세사리)** — A형 ragged 규칙을 그대로 쓰면 병합 하위행 D=None을 continuation으로 정확히 잡으나, **병합/비병합 혼재**를 의식 못 하면 단일행 상품 누락 가능. 병합범위 자체가 블록경계 권위.

10. **사이즈명 내 묶음수 혼입((50장)/(20장))** — 상품악세사리 E, 디지털 AL 봉투명. 분리 안 하면 사이즈 노이즈 + 묶음수 누락 동시(SIZE_NAME_NOISE 재발).

11. **묶음수 비정수 텍스트** — 스티커 `*최대20조각`, 캘린더 `4(8P)/8(16P)`. 정수 가정 추출 시 파싱불가/누락.

12. **A컬럼 대분류 그룹(캘린더·문구·굿즈·아크릴·악세사리)** — 상품명(D)과 별개 카테고리축. 무시 시 그룹 소속 소실.

13. **footnote 셀 배경색 의미(노랑=신규/그레이=비노출/준비중)** — data_only 파싱은 값만 취득, 색 미취득. 신규/품절 판정 필요 시 cell.fill 별도 파싱 필요(현재 미수집 — **확인 필요**).

14. **포토북 단일 상품명의 커버타입×사이즈 매트릭스** — D 1개로만 추출 시 R(표지타입)·사이즈별 가격 매트릭스 전개 소실.

15. **MAP·계산공식집초안의 비표준 축** — MAP은 가로 카테고리×세로 상품(D=상품명 아님), 공식집은 공식텍스트. 상품시트 고정기준 적용 불가.
