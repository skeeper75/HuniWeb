## ② L1 추출 구현 내역 (G 기준 충실 적용)

### 2-1. 전 컬럼 행단위 보존 (속성별 단일컬럼 평면화 금지)
- A~AO(max_col 41) 전 컬럼을 1행=1레코드로 보존. 속성별 파이프 join 안 함.
- 산출 형태: tidy 1행1레코드(`silsa-l1.csv`) — 고정메타 6열(`sheet`·`row_seq`·`prd_nm`·`_anchor_ffilled`·`_work_size_col`·`_work_size_value`) + 41 데이터 컬럼(composite명) + `cell_meta_json`.

### 2-2. 그룹헤더 composite 귀속 (row1 가로병합 ffill)
실사 row1 가로병합 6그룹(원본 실측, `ws.merged_cells.ranges` 중 row1 다중컬럼):

| row1 가로병합 범위 | 그룹명 | composite 하위 컬럼 |
|------|------|------|
| F1:G1 | `비규격(최소/최대)` | `_가로`(F) `_세로`(G) |
| H1:N1 | `파일사양` | `_블리드`(H) `_작업사이즈`(I) `_재단사이즈`(J) `_출력용지규격`(K) `_파일명약어`(L) `_출력파일`(M) `_폴더`(N) |
| O1:P1 | `주문방법(필수)` | `_업로드`(O) `_편집기`(P) |
| W1:X1 | `가공(옵션)` | `_가공`(W) `_가격`(X) |
| Y1:Z1 | `추가(옵션)` | `_추가`(Y) `_추가옵션가격`(Z) |
| AA1:AC1 | `제작수량(필수)` | `_최소`(AA) `_최대`(AB) `_증가`(AC) |

세로병합(r1:r2 단일컬럼, composite 불필요): A `구분`·B `ID`·C `MES ITEM_CD`·D `상품명`·E `사이즈(필수)`·Q `소재(필수)`·R `price`·S `price\n(vat포함)`·T `화이트별색(옵션)`·U `코팅(옵션)`·V `price`·AD `COMMENT`.

### 2-3. 작업사이즈 컬럼 = row2 헤더 식별 (고정레터 가정 금지 — G HARD)
- row2 헤더 텍스트 `작업사이즈`로 컬럼을 동적 식별 → 실사에서 **I**(composite `파일사양_작업사이즈`). 고정 컬럼레터 가정 안 함. 전수 확장 시 시트별 row2 매핑으로 자동 식별(디지털 H, 아크릴 J 등 시트별 상이 대응).

### 2-4. 세로리스트 ffill = 화이트리스트 한정
- ffill 대상 = `구분`(A)·`ID`(B)·`MES ITEM_CD`(C)·`상품명`(D)만. E 사이즈 이하 옵션 컬럼은 ffill **금지**(빈셀 보존).
- ffill 적용 행은 `_anchor_ffilled=true` 메타 기록(round-trip 정확 역보정용).
- 검증: 포맥스보드 A1(row73) — ID `14576`·MES `004-0013`·상품명 `포맥스보드`는 ffill 전파, **작업사이즈는 빈값 보존**.

### 2-5. 빈셀 보존 + 유효0 구분
- 빈 셀 = `""` 보존(drop 금지). 타입 강제·trim 안 함.
- 유효0 구분: 블리드(H) `0`은 유효 0mm로 보존, 공백은 `""`. 실사 데이터행 다수가 H=`0`(예 아트프린트포스터 r3~5), 포맥스보드 r71~72는 H=`3`, A1(r73)은 H=`""`.

### 2-6. 실무진 메타 귀속
- **스레드댓글 9건**(U3·U13·U23·W38·W66·W71·W94·Y94·U98, 전부 `레이어창`, 작성자 `HP`) → 해당 셀 컬럼명에 귀속(`cell_meta`). 데이터행 댓글이므로 셀레벨.
- **배경색(fill.fgColor)** → 셀메타. 의미코드: 노랑(FFFFFF00)=신규는 실사에 **0건**(해당 시트 신규상품 없음). 미확정색(FFB7B7B7·FF92CDDC·FFCCCCCC·FF93CDDD 등)은 RGB+빈도만 보존, `fill_meaning=null`(단정 안 함).
- **글씨색(font.color)** → 별도 메타축 보존(예 price열 FFFF0000 빨강). 의미(비노출/무시)는 L2 해석.
- **★제약텍스트** → `has_constraint_star` 플래그(실사 1건). 줄바꿈(`\n`) 원형 보존.
- **footnote 범례셀**(초록 FFB6D7A8, C147) → 시트메타. ※ 실사 C147 전문은 `제목부분 (필수/빨강)는 모든 상품에 반드시 설정해야하는 부분 (옵션/오렌지)은 상품에 따라 있을수도 없을 수도 있습니다 / 모든 항목은 Figma를 참고해주세요` 로, G §5-2가 인용한 노랑/그레이 신규·품절 범례와 **다른 텍스트**다(아래 ⑤ 참조).

## ④ 포맥스보드 A1 재판정 시연 (false-MISSING 해소 토대)

### 4-1. 원본 실측 (실사 포맥스보드 블록, D=row71)

### 4-2. L1 추출본 보존 확인 (`silsa-l1.csv` row_seq 73)
```

파일사양_블리드=''        ← 공백 보존 (유효0 아님, 미정의)
파일사양_작업사이즈=''    ← ★1차가 버린 컬럼, 공백 그대로 보존
파일사양_재단사이즈=''
```

**다중 신호 동시 보존**: 포맥스 A1은 이제 (a)작업사이즈 공백 + (b)`_row_hidden=true` + (c)행 전체 회색 fill — **세 신호가 한 레코드에 함께 보존**된다. 사용자가 추궁한 "A1→작업사이즈공백→행숨김" 연쇄가 더 이상 땜질규칙이 아니라 정보축 전수로 한 번에 잡힌다. L2는 이 다중 신호로 "비활성/미생산" 판정.

### 4-3. L2 재판정 가능성 (판정 자체는 다음 단계 — 여기선 정보 보존까지 입증)
- L1은 "A1 사이즈명은 있으나 **작업사이즈(I) 공백**"이라는 사실을 같은 레코드에 함께 보존한다.
- 이 토대 위에서 L2는 "작업사이즈 컬럼 공백 = 생산 작업치수 미정의 → 정당 미적재(false MISSING 아님)"로 올바르게 판정할 수 있다. **G §②경계대로 공백/0/값 판정은 L2 영역**이며, L1은 판정에 필요한 I열 값을 행에 보존만 했다.
- 추가 신호: A1 행 전체가 FFB7B7B7 회색 fill(셀메타 보존). 회색 배경 의미는 footnote 무언급 → **확인 필요**(실사 회색의 의미가 "준비중/미생산"인지 실무 확인 대상). L2가 이 보조신호도 활용 가능.

### 4-4. 1차 평면화 대비 핵심 차이
| 항목 | 1차 평면화 (`04_audit/excel-size.csv` E만) | L1 충실추출 (`06_extract/silsa-l1.csv`) |
|------|------|------|
| 포맥스보드 표현 | `A3 (297x420) \| A2 (420x594) \| A1 (594x841)` 단일 파이프 문자열 1행 | 사이즈별 3행(71·72·73) 분리 |
| 작업사이즈(I) | **컬럼 통째 미추출** (drop) | 행별 보존 (71=`303 x 426`, 73=`''`) |
| A1 공백 신호 | **소실** (사이즈명만 남아 실사이즈로 오인 → false MISSING) | **보존** (작업사이즈 공백이 레코드에 명시) |
| 빈셀/유효0 | 구분 불가 | 블리드 0 vs 공백 구분 보존 |
| 실무 메타 | 없음 | 댓글·fill·★·footnote 셀 귀속 |
| 보존율 검증 | 없음 | non-empty 100%·round-trip 0 기계 보증 |

→ **근본 차이**: 1차는 "사이즈 속성=사이즈명만 중요"라는 **해석을 추출 시점에 박아** 같은 행의 작업사이즈를 버렸다. L1은 해석 없이 전 컬럼을 행단위 보존해, A1 작업사이즈 공백이라는 다운스트림 판단 근거를 살렸다.

| row_seq | 상품/사이즈 | `_row_hidden` | 신호 해석(L2 영역) |
|:---:|------|:---:|------|
| 18~21 | **투명포스터★**(A3/A2/A1/사용자입력) | **true** | 전행 숨김 = 비활성/품절 후보 |
| 68·69 | 폼보드 계열 | true | 비활성 후보 |
| **73** | **포맥스보드 A1** | **true** | 작업사이즈공백 + 숨김 다중신호 |
| 77 | 프레임리스 | true | 비활성 후보 |

- **8행 전부 데이터(사이즈명 등) 보존 + `_row_hidden=true` 플래그 병기**. 한 행도 드롭하지 않음(round-trip diff 0이 무손실 증명). r22(완전 빈 숨김행)는 데이터행 정의상 레코드 아님(보존할 값 없음 — 정보 손실 아님).
- 검증 `row_hidden_preservation`: 원본 숨김데이터행 8 == 추출 `_row_hidden=true` 8 **PASS**.

### 5-2. 열숨김 (col_hidden) — K 출력용지규격 보존
- 실사 숨김열 = **K `파일사양_출력용지규격`** 1개(전 13시트 숨김열은 전부 출력용지규격 — G §9-1 실측). 숨김열도 **컬럼으로 그대로 추출** + 셀메타 `col_hidden=true` + 시트메타 `hidden_column` 행 표기.
- 실사 K는 안내문 성격 값 소수만(캘린더 14·디캘 7행은 값보유). **데이터 컬럼 보존 + 숨김 플래그 둘 다** 유지.
- 검증 `col_hidden_preservation`: 원본 숨김열명 `출력용지규격` ⊆ 추출 플래그열명 + 컬럼 데이터 보존 **PASS**.

### 5-3. 수식여부 (is_formula) — S열 VAT파생 58건
- 실사 **S열** = `=SUM(R{r})*1.1` 수식 **58건**(VAT포함가, R열 판매가의 ×1.1 파생). `data_only=False` 워크북 병행 로드로 식별.
- 셀메타에 `is_formula=true` + `formula`(원본 수식 문자열) 보존. 계산값(예 7700)은 `data_only=True`로 `cols`에 별도 보존 → **값+수식 둘 다**.
- 의미: S는 독립 입력이 아닌 R가 파생값 — L2가 "S 직접 매핑 금지, R×1.1 공식 귀속"으로 활용 가능.
- 검증 `formula_preservation`: 58 == 58 **PASS**.

### 5-4. 하이퍼링크 (hyperlink) — 실사 0건
- 실사 시트에 하이퍼링크 없음(원본 0). 전 파일 하이퍼링크는 책자 1건(실사 무관). 검증 `hyperlink_preservation`: 0 == 0 **PASS**.

### 5-5. 전수 확장 검증 (캘린더 — 무수정 재실행)
- 동일 스크립트로 `--sheet 캘린더` 재실행: 숨김열 `출력용지규격`(I) 플래그+**14 non-empty 값 보존**, 숨김행 0, 수식 0, 20레코드, 검증 **OVERALL PASS**. 스크립트 무수정으로 전수 확장 동작 입증.
