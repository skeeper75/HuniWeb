## 1. 논리 블록 식별 (병합셀 앵커 기준)

병합셀 = `A1:C1`·`A15:D15`·`F15:I15`·`A27:D27`·`F27:I27`·`A32:D32`·`F32:I32`·`A12:B12` → 블록 경계.

| # | 블록명(병합 헤더) | 위치 | 옵션 축 | 수량구간 | 가격셀 | 라이브 comp |
|---|------------------|------|---------|---------|--------|------------|
| B1 | **모서리 (귀돌이, 합가)** | A1, 헤더 A2 | 직각 / 둥근 | A3~A11 (9구간: 1~5000) | 18 | CORNER_RIGHT·CORNER_ROUND |
| B2 | **오시 (합가)** | A15, 헤더 A16 | 1줄/2줄/3줄 | A17~A26 (10구간: 1~5000) | 30 | CREASE_1L·2L·3L |
| B3 | **미싱 (합가)** | F15, 헤더 F16 | 1줄/2줄/3줄 | F17~F26 (10구간) | 30 | PERF_1L·2L·3L |
| B4 | **가변(텍스트)** | A32, 헤더 A33 | 1개/2개/3개 | A34~A56 (23구간: 1~9500) | 69 | VARTEXT_1EA·2EA·3EA |
| B5 | **가변(이미지)** | F32, 헤더 F33 | 1개/2개/3개 | F34~F56 (23구간) | 69 | VARIMG_1EA·2EA·3EA |
| — | (제작수량 100000 행) | A57·F57 | — | — | 0 | 부유셀(가격 공란·노트) |

합계 = 18+30+30+69+69 = **216 가격셀** = 라이브 216행.

### 프롬프트 vs 실제 구조
- 프롬프트는 3블록(모서리·오시·가변텍스트)으로 명시했으나, **실측은 6블록** — 미싱(B3)·가변이미지(B5)가 추가 존재. 무손실 원칙(round-10 교훈: 침묵 삭제 금지)상 전부 분해.
- 프롬프트는 모서리 헤더를 `직각/둥근모서리 × 수량`이라 정확히 명시. 실측 일치.

## 2. 복합셀 식별 (분해 대상)

이 시트는 복합셀이 적다 — 각 셀은 단일 가격값이고, 옵션은 열 헤더(직각/둥근·1줄/2줄/3줄·1개/2개/3개)로 이미 분리되어 있다.

| 분해 포인트 | 내용 | 분해 방향 |
|-------------|------|-----------|
| 줄수/개수 헤더 | "1줄/2줄/3줄"·"1개/2개/3개"가 같은 후가공의 차등 가격 | **comp 분리**(CREASE_1L/2L/3L) — opt_cd 차원 아님(§ decomposition §2 근거) |
| 오시 vs 미싱 | A블록(오시)·F블록(미싱)이 같은 줄수 헤더·**동일 가격** | 공정이 다름(귀돌이≠오시≠미싱) → 별 comp군. 가격은 우연히 동일 |
| 증분룰 텍스트 | A12·A27·F27 (밴드 밖 수량 외삽 규칙) | **단가행으로 표현 불가** → note 보존 + 별 명시 (§ decomposition §5) |

## 4. 라이브 적재 상태 (실측 — 이 시트의 결정적 특이점)

> **이 시트는 round-16의 다른 시트(스티커=신규·아크릴=배선 단절·foil-small=전면 미적재)와 근본적으로 다르다: 이미 완전 적재 + 배선 완결 + 전건 정합.**

### 4-1. 적재된 그릇 (라이브 실측)

| t_prc 테이블 | 라이브 상태 |
|--------------|------------|
| `t_prc_price_components` | **14 comp 적재** (COMP_PP_CORNER_RIGHT/ROUND·CREASE_1~3L·PERF_1~3L·VARTEXT/VARIMG_1~3EA) |
| `t_prc_component_prices` | **216행 적재** (comp별 9·10·23구간), opt_cd·proc_cd·siz_cd 전부 NULL |
| `t_prc_formula_components` | **배선 완결** — 14 comp가 `PRF_DGP_A`(disp_seq 16~29)·`PRF_DGP_D`(7~20)에 합산 배선 |
| `t_prc_price_formulas` | 독립 후가공 공식 없음 — 후가공은 **디지털인쇄 합산형 공식의 구성요소**로 통합됨 |

### 4-2. 가격사슬 = 완결 (아크릴 단절과 정반대)
```
손님 선택(둥근모서리, 1줄오시 …) → PRF_DGP_A 합산형 공식
  → formula_components (후가공 comp 배선됨)
  → price_components (use_dims=["min_qty"])
  → component_prices (수량구간 → 가격)  ✅ 조회 가능
```
- 아크릴은 단가행 적재됐으나 **배선 0(엔진 조회 불가)**. 인쇄후가공은 **배선 실재** → 엔진이 디지털인쇄 견적에서 후가공 가격을 합산할 수 있음.

### 4-3. 전건 정합 대조 결과
- 가격표 216 가격셀 ↔ 라이브 216행 자동 대조: **mismatch 0 · EXCEL-ONLY 0 · LIVE-ONLY 0 = ALL MATCH**.
- 오시(CREASE) = 미싱(PERF) 동일 가격 — 가격표에서도 B블록=F블록 동일값 확인. 라이브가 정확히 별 comp로 분리 적재(공정 구분 보존).

## 5. round-16 산출의 성격 (이 시트 한정)

신규 구축이 아니므로 산출 목표가 다르다:
1. **기존 적재 그릇의 webadmin 복붙용 문서화** — import.xlsx는 라이브 실재값을 그대로 담아 "현재 적재 상태 = 가격표와 정합"을 webadmin에서 확인·재현 가능하게.
2. **단가/합가 충돌(P4) 명시** — 가격표 "합가" ↔ 라이브 .01 단가형 충돌을 컨펌으로 분리(§ decomposition §2). 이게 이 시트의 유일한 미해소 결정.
3. **증분룰 처리 명시** — DB가 담지 못하는 밴드 외 외삽 규칙의 거처 판정(§ decomposition §5).
