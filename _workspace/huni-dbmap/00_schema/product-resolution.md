# 상품 해소 — 7개 할인 그룹 → prd_cd

라운드 1d. 7개 수량구간 할인 테이블의 모든 상품을 실제 `t_prd_products.prd_cd`로 해소한다.
매핑 설계자를 위한 할인테이블 → 상품 연결의 권위 출처.

## 작업 1 — 조인 키 도출

마스터 CSV(`01_excel/product-discount-assignments.csv`)는 잠재적 키 세 개를 담고 있다:
`product_id`(숫자형, 예: 14671), `mes_item_cd`(예: 008-0020), `product_name`.

`t_prd_products` 실제 DB 검사:

| 후보 컬럼 | DB 실태 | 조인 키로 사용 가능? |
|------------------|-----------|---------------------|
| `prd_cd` (PK, varchar 50) | `PRD_000xxx`, 280행 | 가능 — 단 마스터에 prd_cd가 없음 |
| `"MES_ITEM_CD"` (varchar 30, 따옴표 대문자) | **280행 전부 NULL** (`count(MES_ITEM_CD)=0`) | 불가 — 컬럼은 존재하나 전부 비어 있음 |
| (마스터 `ID` 14671에 대응하는 숫자형 외부 id) | `t_prd_products`에 **그런 컬럼이 존재하지 않음** | 불가 |
| `prd_nm` (varchar 200) | 채워져 있음 | **가능 — 유일하게 실현 가능한 조인** |

**결론: 유일한 조인 키는 `prd_nm`(정확한 상품명 일치)이다.** 마스터의 숫자형 `product_id`도, `mes_item_cd`도 DB에 대응되는 것이 없다 — `MES_ITEM_CD`는 정의되어 있으나 100% NULL이고, 숫자형 외부 id 컬럼은 없다. 마스터 id는 이 DB에 한 번도 로드된 적 없는 MES 측 식별자다.

### 테스트 예시 (모두 확인됨)

| 마스터 이름 | 해소된 prd_cd | 일치 |
|-------------|-----------------|------|
| 틴거울 | PRD_000183 | OK |
| 말랑키링 | PRD_000221 | OK |
| 머그컵 | PRD_000193 | OK |
| 만년다이어리(소프트커버) (mes 008-0020) | PRD_000172 | 이름으로 OK (mes_item_cd 조인은 0 반환) |

### 이름 유일성 유의사항 (매핑 설계자용)

DB에서 `prd_nm` 값 7개가 비유일하다: 아크릴코롯토 (2), 떡메모지 (2), 와이드벽걸이캘린더 (2), 탁상형캘린더 (2), 벽걸이캘린더 (2), 엽서캘린더 (2), 미니탁상형캘린더 (2). **38개 마스터 행 중 중복 이름에 걸린 것은 없다** — 38개 모두 정확히 하나의 prd_cd로 해소되었다(모호 0, 미해소 0). 하지만 마스터가 향후 아크릴코롯토나 떡메모지로 확장되면 이름 조인은 모호해진다(아크릴코롯토는 PRD_000164/167; 떡메모지는 PRD_000097/182). 아크릴코롯토 모호성은 여기서 이름이 아니라 카테고리 서브트리로 처리된다.

[정정 2026-06-04] 라이브 DB 실측 결과 `CAT_000154 아크릴카라비너` 카테고리는 **비어있고(0건)**, PRD_000166은 나머지 아크릴 11개와 동일한 `CAT_000155 조합형`에 속한다. 즉 12개 아크릴은 카테고리로 구별 불가하며, acrylic_general vs acrylic_carabiner 분리는 카테고리가 아닌 **상품명** 기반이다. 두 그룹은 상호 배타적이다(이름에 카라비너를 포함하는 유일한 상품이 PRD_000166).

## 작업 3 — 원단 상품 집합

범위 = 에코백 서브트리(`CAT_000011`) + 단독 레더파우치 루트(`CAT_000305`). 모든 상품은 `PRD_TYPE.03`(기성상품). 서브트리는 **구별되는 상품 51개**(PRD_000230–PRD_000280, 연속)를 산출한다.

**부자재 플래그 (사용자 규칙에 따라 제외):** 카테고리 `CAT_000306` 에코백부자재는 정확히 **PRD_000280 레더라벨제작** — 유일한 진짜 부자재 — 만 보유한다. 제외.

→ **원단 상품 집합 = 50** (PRD_000230–PRD_000279).

### 매핑 설계자용 부자재 식별 주의사항
- 정규식 `라벨|스트랩`은 **CAT_000225 캔버스 스트랩 라벨파우치**(PRD_000241)와 **CAT_000226 광목 스트링 라벨파우치**(PRD_000242)도 잡는다. 이들은 부자재가 아니다 — 라벨/스트랩 기능을 가진 파우치일 뿐이다. **유지할 것.** 부자재로 분류된 상품만 제외한다.
- 카테고리명 정규식 `파우치|에코백|필통|캔버스|광목|린넨|타이벡|메쉬|패브릭|직물`은 포스터/배너/코스터 원단 카테고리(패브릭포스터 CAT_000072+, 메쉬배너 CAT_000089, 린넨코스터 CAT_000176 등)도 잡는다 — 하지만 **그중 범위 내 상품을 보유한 것은 없다**; 이들은 에코백 아래가 아니라 포스터/배너/굿즈 루트 아래에 있다. 권위 있는 범위는 단순 이름 정규식이 아니라 서브트리 재귀다.

| 리프 카테고리 | 상품 |

> 엑셀 분석가가 마스터로부터 권위 있는 원단/부자재 구분을 병렬로 산출한다; 이는 그것을 뒷받침하는 DB 카테고리→prd_cd 열거다. 마스터가 추가 부자재 상품(별도 판매 지퍼/스트랩/손잡이)을 지정하면 부자재 유형 카테고리 아래에 나타날 것이다 — 현재 에코백 서브트리에는 PRD_000280 외에 그런 것이 없다.

## 미해소 마스터 행

**없음.** 권위 있는 마스터 38행 모두 `prd_nm`을 통해 정확히 하나의 prd_cd로 해소되었다.

## 매핑 설계자를 위한 주의/리스크

1. `MES_ITEM_CD`는 280개 상품 전부에서 NULL이다 — MES id로 할인 연결을 로드하려고 시도하지 말 것. `prd_cd`만 사용하라.
2. 이름 기반 조인은 현재 38행에는 안전하지만, 중복된 7개 이름(특히 떡메모지, 아크릴코롯토)으로의 향후 확장에는 취약하다. 모든 하위 연결에서는 prd_cd를 선호하라.
3. acrylic/fabric은 category-wide → 나열된 집합의 상품마다 `t_prd_product_discount_tables` 행 하나(아크릴+원단 연결 행 61개). goods/squishy/stationery는 마스터에서 상품별로 명시적이다(연결 행 37개; 파우치 마스터 행 1개는 fabric으로 통합됨).
4. acrylic_general과 acrylic_carabiner은 상호 배타적이다(카라비너 = PRD_000166 단독).
