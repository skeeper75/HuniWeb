# golden-cases-photobook.md — 포토북 설계 대표 케이스 + 기대 골든값

> **핵심 설계가(hpe-engine-designer) 산출 — 포토북 종단 골든.** 설계 공식(PRF_PHOTOBOOK_SUM)으로 계산되는 대표 케이스와 기대 골든값.
> 검증가(hpe-validator)·codex가 라이브 `evaluate_price`를 실호출/재구현해 **이 골든값을 재현**한다(허용오차 0).
>
> **★순환참조 금지[HARD]**: 골든값은 **상품마스터260610 포토북 시트 inline verbatim**(`가격_기본(24P)`·`가격_추가(2P)당`) + row17 명문 산식에서 가져온다. 설계가 만든 값이 아니다.
> 출처 = `photobook-l1.csv` 셀 직독(verbatim·2026-06-22)·라이브 단가행(COMP_BIND_PUR·COMP_PAPER 재사용 비목).
> 계산 규칙 = engine-contract(P2~P4): 단가형(.01) `subtotal = unit_price × qty` + row17 산식 `base24 + ceil((N−base_min)/2)×per2p`(× 부수).

## 0. 골든 케이스 도출 원칙 — 포토북 (★캘린더와 정반대: inline 공식화 가능)

- **base24·per2p 골든** = 상품마스터 inline verbatim 직독. **재현 가능**(허용오차 0·시트 명시값=권위).
- **페이지 증분 골든** = base24 + ceil((N−base_min)/2) × per2p (× 부수). 증분횟수=앱 계산(off-grid ceiling 동류)·검증가가 산식 재현.
- **★캘린더 inline BLOCKED와 결정적 차이**: 캘린더는 inline이 비정수해(정찰가 스냅샷)라 BLOCKED. 포토북은 **base24/per2p가 시트 명시값 + per2p cost-driven**(imposition 단조)이라 **공식화 재현 가능**(추측 INSERT 아님·날조 0).

★ **양면표 원칙**(디지털 R-1·캘린더 선례): 골든 = ① **설계 기대값**(inline verbatim·옳은 값) vs ② **현 라이브 산출**(WIRE 결함=공식·바인딩 0행이라 0원/source=NONE). 진원 = WIRE 결함(공식·바인딩 전무)이지 inline 값 오류 아님.

## 1. ★base24 골든 (24P 기준·상품마스터 verbatim·허용오차 0)

> 가장 단단한 골든. 24P 1부 주문 = base24 그대로(증분 0·×1). prc_typ .01 ×qty(×1).

| per2p 항 | ceil((24−24)/2)=0 → 0원 | 증분 0 |

### GC-PB-2. 8x8 레더하드커버 24P 1부 (표지타입 분기 입증)

### GC-PB-3. A5 소프트커버 4P 1부 (소프트 base_min=4·Q-PB-PAGEBASE)

## 2. ★페이지 증분 골든 (base24 + per2p×증분횟수 · 산식 재현 · G-PB-PAGE 입증)

> row17 명문 산식. 페이지 증분이 가격에 선형 반영됨을 입증. 증분횟수=앱 계산(ceil).

### GC-PB-5. 8x8 하드커버 40P 1부 (benchmark 골든·★핵심 페이지 증분)
| 산식 | base24(15,000) + ceil((40−24)/2)×per2p(500) = 15,000 + 8×500 |
| **설계 기대값** | **19,000원** | base24 15,000 + 페이지증분 4,000(8×500) |
| ★페이지 누락 오답 | 15,000(증분 소실) — **금지**(G-PB-PAGE) |

### GC-PB-6. A4 레더하드커버 50P 1부 (benchmark 골든)
| 산식 | base24(26,000) + ceil((50−24)/2)×per2p(600) = 26,000 + 13×600 |
| **설계 기대값** | **33,800원** | base24 26,000 + 페이지증분 7,800(13×600) |
| ★페이지 누락 오답 | 26,000 — 금지 |

### GC-PB-7. 8x8 하드커버 150P 1부 (고페이지·G-PB-PAGE 3배 입증)
| 산식 | base24(15,000) + ceil((150−24)/2)×per2p(500) = 15,000 + 63×500 |
| **설계 기대값** | **46,500원** | base24 15,000 + 페이지증분 31,500(63×500) |
| ★페이지 누락 오답 | 15,000 (3.1배 과소청구) — **돈크리티컬**(G-PB-PAGE) |

### GC-PB-8. 10x10 하드커버 40P 1부 (per2p siz_cd 분기 입증·표지 무관)
| 산식 | base24(22,000) + ceil((40−24)/2)×per2p(1,000) = 22,000 + 8×1,000 |
| **설계 기대값** | **30,000원** | ★per2p=1,000(10x10)·8x8(500)과 siz 종속·표지 무관(C-PB4) 입증 |

### GC-PB-9. A5 하드커버 40P 1부 (per2p 최저·imposition cost-driven)
| 산식 | base24(12,000) + ceil((40−24)/2)×per2p(300) = 12,000 + 8×300 |
| **설계 기대값** | **14,400원** | per2p=300(A5 4-up imposition 최저·§4.3 입증) |

★ GC-PB-5~9 = **페이지 증분 산식 재현**·증분횟수=앱 ceil. **per2p siz 종속·표지 무관**(GC-PB-8 vs GC-PB-5: 같은 40P인데 10x10=1,000·8x8=500). 페이지 곱 누락 시 GC-PB-7이 46,500→15,000(3.1배 과소)로 붕괴=G-PB-PAGE 입증.

## 4. ★BLOCKED 골든 (정직 표기·추측 단가 금지)

> 시트 공란·미확정 부품은 추측 적재 금지·BLOCKED 정직(캘린더 inline·악세사리 카드봉투·책자 088 선례 동형).

### GC-PB-11. 10x10 소프트커버 (row8 시트 공란) — BLOCKED
| 항목 | 값 |
|------|-----|
| base24 | (시트 공란·미정) |
| **판정** | **❌ BLOCKED**(추측 INSERT 금지·Q-PB-SOFT8) |
| 사유 | 상품마스터 row8 `가격_기본/추가` 공란(누락)·8x8/A5/A4 소프트(12,000/10,000/13,000)에서 10x10 소프트 보간 추측 금지 |

### GC-PB-12. base24 부품 분해 정확 재현 (15,000 = 표지+내지+제본+면지) — 부분 BLOCKED
| 항목 | 값 |
|------|-----|
| 부품 역산 | PUR 5,000 + 내지24P 6,000 + 표지/면지 4,000 = 15,000(근사) |
| **판정** | **🟡 부분 BLOCKED**(base24 통째 적재가 정답·부품 정확 단가행 직매칭은 Q-PB-COAT/FACE 미해소) |
| 사유 | base24=완제 통합값(권위 verbatim)·부품 정확 역산은 표지 무광코팅 단가행 충전·면지 가격기여 컨펌 후. **base24 통째 적재로 골든 GC-PB-1~10 재현·부품 분해 안 함** |

★ GC-PB-11/12 = **BLOCKED 정직**(날조 0). 10x10 소프트는 시트 공란이라 단가행 INSERT 안 함·base24 부품 분해는 통째 적재로 우회(부품 정확 역산 불요).

## 5. 골든 종합표 (검증가 재현 대상)

| 골든 | 대상 | 설계 기대값(verbatim) | 재현 방법 | 상태 |
|------|------|-----------------------|-----------|------|
| GC-PB-1~4 | base24 ×1(증분0) | 15,000 / 23,000 / 10,000 / 16,000 | base24 단가행 직독 ×1 | ✅ 허용오차 0 |
| GC-PB-5~9 | 페이지 증분 산식 | 19,000 / 33,800 / 46,500 / 30,000 / 14,400 | base24 + ceil((N−base_min)/2)×per2p | ✅ 산식 재현(증분=앱 ceil) |
| GC-PB-10 | 부수 곱 | 190,000 | (base24+per2p×8)×부수 | ✅ base24·per2p 둘 다 ×부수 |
| GC-PB-11 | 10x10 소프트(공란) | (재현 불가) | — | **❌ BLOCKED**(추측 금지·Q-PB-SOFT8) |
| GC-PB-12 | base24 부품 정확 역산 | (통째 적재로 우회) | base24 통째 | **🟡 부분 BLOCKED**(부품 직매칭 Q-PB-COAT/FACE) |

★ **핵심 입증**:
- **GC-PB-5/7**(40P=19,000·150P=46,500 vs 페이지 누락 15,000)가 **G-PB-PAGE 페이지 증분 곱**을 단단히 입증(누락 시 3.1배 과소).
- **GC-PB-1 vs GC-PB-2**(하드 15,000 vs 레더하드 23,000)가 base24 **mat_cd 표지타입 판별**(G-PB-FLAT)을 입증.
- **GC-PB-5 vs GC-PB-8**(8x8 500 vs 10x10 1,000·같은 40P)가 per2p **siz_cd 종속·표지 무관**(C-PB4)을 입증.
- inline 공식화 재현(캘린더 BLOCKED와 정반대)·10x10 소프트만 BLOCKED 정직(날조 0).

검증가는 GC-PB-1~10을 재현하고, GC-PB-11/12 BLOCKED를 권위 충돌/누락으로 확인.
