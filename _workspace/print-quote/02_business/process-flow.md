## 1. 공정 라우트 17개 케이스 (p.1)

후니프린팅은 제품 사양·소재·후가공에 따라 **17가지 공정 라우트(Case 1~17)** 를 운영한다. 각 라우트는 노드 시퀀스이며, 단계별로 입고·출고 마커가 찍힌다.

### 1.1 라우트 표 (p.1)

| Case | 분류 | 시작 노드 → 종료 노드 | 자체 / 외주 |
|:--:|------|---|:--:|
| 1 | 낱장인쇄 | 디지털출력 → 코팅 → 재단 → **1차포장** | 자체 |
| 2 | 스티커 | 디지털출력 → 코팅 → 반칼커팅 / 완칼커팅 → 재단 → **1차포장** | 자체 |
| 3 | 인쇄후가공 | 디지털출력 → 코팅 → 재단 → 오시/접지/타공/귀돌이/수동도무송 → **1차포장** | 자체 |
| 4 | 책자(링제본) | 디지털출력 → 코팅 → 재단 → 링제본 → **1차포장** | 자체 |
| 5 | 책자 (PUR/떡/중철) | 디지털출력 → 코팅 → 재단 → 중철제본/PUR제본/떡제본 → 재단 → **1차포장** | 자체 (중철은 선택적 외주) |
| 5b | 커버/클립보드 (레더+싸바리) | 디지털출력 / 패브릭출력 → 싸바리 → 조립/후가공 → **1차포장** | 자체 |
| 6 | 실사/사인 | 실사출력 → 콜코팅 → 조립/후가공 → **1차포장 + 박스포장** | 자체 |
| 7 | 투명실사 | 라텍스(화이트)출력 → 조립/후가공 → **1차포장 + 박스포장** | 자체 |
| 8 | 커팅 | 레이저커팅 / 시트커팅 → **1차포장 + 박스포장** | 자체 |
| 9 | 봉제미싱 | 패브릭출력 → 봉제미싱 → **1차포장** | 자체 |
| 10 | UV단품 | UV평판출력 → **1차포장** | 자체 |
| 11 | 투명클립보드 | UV평판출력 → 레이저커팅 → 조립/후가공 → **1차포장** | 자체 |
| 12 | 아크릴가공 | UV평판출력 → 레이저커팅 → 아크릴가공 → **1차포장** | 자체 |
| 13 | 전사 | 전사인쇄 → **1차포장** | 자체 |
| 14 | 도장 | 도장제작 → **1차포장** | 자체 |
| 15 | 외주공정 | 동판제작 → 박작업 → **생산본부 입고** | 외주 |
| 16 | 재고상품 | 상품액세서리 → **1차포장** | 자체 (재고) |
| 17 | 외주제작 | 책자제작 / 실사현수막 / 합판인쇄 / 고주파 → **1차포장** | 외주 |

### 1.2 노드별 담당 부서 (p.1 우측 Summary)

| 노드(공정) | 담당팀 | 상태 마커 |
|----------|------|------|
| 디지털출력 | **디지털인쇄팀** | "출력완료" |
| 반칼/완칼 커팅 | **스티커가공파트** | "커팅완료" |

| 실사/UV/패브릭 등 특수 출력 | **특수인쇄가공팀** | (인입) |
| 봉제미싱 | **봉재가공파트** | "봉재시작" |
| 아크릴가공 | **아크릴가공팀** | "가공시작" |
| 1차포장 | **포장팀** | "제작완료" |
| 박스포장 | **포장팀** | "송장출력" |

### 1.3 외주공정 분기 (p.1)

| 구분 | 내용 |
|------|------|
| **외주공정 (필수)** | 동판제작, 박작업 — 박명함·박 후가공 시 |

## 2. 부서별 일별 현황 표 (p.2)

p.2는 `2026.02.10 공정별 현황표` — 운영용 일일 KPI 대시보드. 7개 부서 + 8개 추가 공정(외주 포함)을 4단계(대기/완료/주의/불량)로 트래킹.

### 2.1 일일 합계 KPI (p.2)

| 지표 | 값 (2026-02-10 표본) |
|------|--:|
| 당일발주 | **940건** |
| 당일생산 | **300건** |
| 당일출고대기 | **940건** |
| 당일출고완료 | **300건** |

### 2.2 7개 핵심 부서 + 8개 보조 공정

핵심 부서 (모두 자체):
1. 디지털인쇄팀
2. 스티커가공파트
3. 인쇄후가공팀
4. 제본
5. 특수인쇄가공팀
6. 봉재가공파트
7. 굿즈가공

보조 공정:
1. 패브릭출력
2. UV인쇄(단품)
3. UV인쇄(아크릴)
4. 전사상품
5. 책자(외주)
6. 현수막(외주)
7. 고주파(외주)
8. 합판인쇄

각 부서/공정마다 다음 4지표 트래킹 (p.2 표):
- **대기:** 940/3000 (당일 대기 / 누적 대기)
- **완료:** 300/865 (당일 완료 / 누적 완료)
- **주의:** 5/50 (당일 주의 / 누적 주의)
- **불량:** 2/100 (당일 불량 / 누적 불량)

표본은 데모용 동일 값. 실 운영 시 부서별 실측치 입력.

### 2.3 4단계 상태 정의

| 상태 | 의미 | 후속 조치 |
|------|------|------|
| 대기 | 입고됐으나 작업 미시작 | 작업 배정 필요 |
| 완료 | 부서 공정 종료, 다음 공정으로 이동 | 다음 부서로 자동 입고 |
| 주의 | 사양/품질 검토 필요 | 담당자 확인 후 진행/반려 |
| 불량 | 재작업 필요 | 재제작 또는 클레임 흐름 |

## 5. SLA (Service Level Agreement) — 명시·추정

### 5.1 명시된 SLA

p.1, p.2에 명시된 시간 SLA는 없음. **공정별 평균 리드타임은 "공정라우트 설정 후 추출 가능"** 으로 명기 (3.2). → To-Be에서는 **공정 라우트 정의 = SLA 측정 가능 인프라**가 1순위.

## 8. Pain Point 요약 (주문프로세스 PDF p.5와 교차)

공정관리 PDF는 As-Is 공정을 도식화하나, 주문프로세스 PDF p.5 `05. Pain point`에서 명시한 핵심 통점:

### 8.1 발주 단계 Pain

- 판매처/상품에 따라 **파일 포맷·파일명이 모두 다름** (4사이트 BIZ/FUJI/HUNI/컨티뉴 — 다른 OEM 채널)
- 자유형 스티커: 조판완료 파일과 칼선 파일을 분리/배포 필요
- 아크릴 상품: 화이트+칼선 별도 (5도/9도 구분)
- 디지털인쇄: 종이명·사양을 파일명에 추가 필요 (캘린더·떡메모지)
- 자사사이트(현 후니프린팅): AI/JPG 업로드 후 조판·파일명 100% 수정 필요
- 떡메모지: 판걸이 명시 필요

**1차 목표 (PDF p.5 명시):** "주문과 파일의 매칭으로 썸네일 자동생성 / 판매처에 상관없이 상품과 파일명의 일관화"

### 8.2 출력 단계 Pain

- 판매처별 상품·소재·출력사이즈가 모두 다른 표기
- 다도안으로 인한 판걸이(인포지션) 차이 → 구분 어려움
- 출력시간 분배 어려움
- **해결:** "MES의 상품과 사용자의 상품을 매칭하여 파일네임을 일관화"

### 8.3 출고 단계 Pain

- 합배송 리스트 알 수 없음
