## 0. 가설 ↔ 실측 차이 (반드시 먼저 기록)

| 항목 | 핸드오프 가설 | **실측 결과** | 판정 |
|------|--------------|-------------|------|
| 이 시트의 자리 | "세트/고정형 대표(마지막 미검증 구조)" | **두 블록 다 순수 수량구간 단가 MATRIX** | 🔴 가설 반증 |
| 떡메 가격 유형 | "합가형 세트(구성요소 합산)" | 라이브 `COMP_TTEOKME.prc_typ_cd=PRICE_TYPE.01 단가형` | 🔴 가설 반증 |
| 엽서북 가격 유형 | (수량구간 단가 MATRIX) | 라이브 엽서북 4 comp 전건 `PRICE_TYPE.01 단가형` | ✅ 일치 |
| 떡메 수량 범위 | "6~360" | **6~600**(28구간) | 정정 |

**결론(정직 평결):** 엽서북떡메는 **스티커(MATRIX)와 동형**이되 **인쇄축(단면/양면)·페이지축(20P/30P)·권당장수축(50/100장)이 추가된 다차원 단가 MATRIX**다. 스티커/아크릴에 없던 페이지축·묶음(권)축을 새로 검증했다는 점이 이 시트의 가치다. **"세트/고정형 신규 구조"는 이 시트로 발견되지 않았다** — 합가형/세트 합산 구성이 전혀 없다(전건 단가형 .01). 다음 후보(박/제본/봉투)는 별도 시트에서 검증해야 한다.

> 단, 페이지축·인쇄축·권당장수축이 "고정형 옵션"이라는 의미에서는 고정형 거동을 일부 보여준다(연속 차원이 아닌 이산 선택). 그러나 가격엔진 관점에서는 모두 단가형 MATRIX의 차원일 뿐, 별도 가격유형이 아니다.

## 2. B1 엽서북 다차원 매트릭스 상세 (핵심 구조)

```
행2 사이즈 │     100*150 (B:E)      │     150*100 (F:I)      │     135*135 (J:M)
행3 인쇄   │   단면(B:C)  양면(D:E)  │   단면(F:G)  양면(H:I)  │   단면(J:K)  양면(L:M)
행4 페이지 │ 20P  30P   20P  30P    │ 20P  30P   20P  30P    │ 20P  30P   20P  30P
수량 A열   │  B    C     D    E     │  F    G     H    I     │  J    K     L    M
 2         │11000 11500 11500 12500 │11000 11500 11500 12500 │12000 12500 12500 13500
 4         │ 9100  9900  9200 10000 │ ...
 ...       │  (수량↑ → 단가↓, 장당가 차등)
3000       │ 2200  2900  2300  3100 │ ...
```

- **행2(병합) = 사이즈 3**: `100*150`(B2:E2) · `150*100`(F2:I2) · `135*135`(J2:M2). 각 4컬럼.
- **행3(병합) = 인쇄 2**: 각 사이즈 아래 `단면`(2열) · `양면`(2열).
- **행4 = 페이지 2**: 각 인쇄 아래 `20P` · `30P`.
- **A열 = 수량(min_qty 축)**: 2,4,6,8,10,20,30…1000,1500,2000,2500,3000 (39구간 상향).
- **셀 = (수량, 사이즈, 인쇄, 페이지) 4차원 조합 단가** — 수량↑ 단가↓ 거동(장당 단가형).

## 3. B2 떡메모지 다차원 매트릭스 상세

```
행47 사이즈 │   90x90mm (B:C)   │   70x120mm (D:E)
행48 권당장수│ 50장1권  100장1권 │ 50장1권  100장1권
수량 A열    │   B       C       │   D       E
 6          │  3000    3200     │  3000    3200
 12         │  2300    2500     │  2300    2500
 ...        │  (수량↑ → 단가↓)
600         │   850    1050     │   850    1050
```

- **행47(병합) = 사이즈 2**: `90x90mm`(B47:C47) · `70x120mm`(D47:E47).
- **행48 = 권당장수 2**: 각 사이즈 아래 `50장 1권` · `100장 1권` (낱장 매수=권 단위 묶음).
- **A열 = 수량(권 수)**: 6,12,18…600 (28구간). ⚠️ "장수/수량" 라벨 — A열은 주문 권 수, 권당장수(50/100장)는 별 차원.
- **셀 = (수량, 사이즈, 권당장수) 3차원 단가** — 수량↑ 단가↓(단가형).

## 4. 복합셀 식별 (분해 대상)

| 복합 헤더 | 원본 | 섞인 요소 | 분해 방향 |
|-----------|------|----------|----------|
| 사이즈/인쇄/페이지 3중 헤더 | 행2~4 누적(`100*150`→`단면`→`20P`) | 사이즈 + 인쇄면 + 페이지 | siz_cd(사이즈) + **인쇄·페이지=comp_cd 분리**(라이브 4 comp) |
| 떡메 "장수/수량" | A48 `"장수 / 수량"` | 권당장수(차원) + 주문수량(축) | 권당장수=`bdl_qty` / 수량=`min_qty` (2축 분리·collapse 금지) |
| 떡메 "50장 1권" | B48 | 낱장 50장 = 묶음 1권 | `bdl_qty=50` (QTY_UNIT 권 단위) |

- **"/" 분해 원칙(HARD §3)**: 사이즈·인쇄·페이지·권당장수 각 축을 개별 차원값으로 펼침(묶지 않음). 라이브가 정확히 이대로 적재(엽서북=면·페이지 comp 분리, 떡메=bdl_qty 분리).
