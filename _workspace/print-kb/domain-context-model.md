# Domain Context Model — 맥락 모델 (단계 B · 가치사슬 골격)

> 작성 2026-06-05. CommonKADS 맥락 모델. **개별 사실 모으기 전에 골격부터.**
> 각 태스크 = 입력 · 출력 · 전제 · 수행기준 · 수행주체(역할·권한). 위키 엔트리의 뼈대.

## 0. 가치사슬 개관

```
[견적] → [주문] → [제작준비] → [제작] → [후가공] → [납품] → [정산]
  │        │         │          │        │         │        │
 CQ-PRICE CQ-FILE   CQ-FILE    CQ-PROC  CQ-FIN   CQ-SUB   CQ-SUB
 CQ-PROD  CQ-POL    CQ-PROC             CQ-PROC  CQ-POL   CQ-POL
```

- **고객 여정(앞단):** 견적 → 주문 → 파일입고 (셀프 견적 위젯 + 주문)
- **생산 여정(뒷단):** 제작준비(조판) → 제작(인쇄) → 후가공 → 납품 (MES 공정 라우트)
- **정산 여정(횡단):** 결제(앞) + 외주정산·매출정산(뒤)

아키타입 3종(통합/셋트/완제품)·인쇄방식 5종이 **제작~후가공 분기를 결정**하는 1차 축.

## T1. 견적 (Quote)

| 항목 | 내용 |
|---|---|
| **입력** | 상품 선택, 옵션 캐스케이드 값(사이즈·자재·인쇄도수·수량·후가공), 사용자입력 비규격 치수 |
| **출력** | 견적 금액(기본가+옵션가+후가공가), 예상 제작 Case, 장바구니 항목 |
| **전제** | 상품 카탈로그 로드, 옵션 의존성 그래프, 가격엔진(t_prc_*)·수량구간 할인표 |
| **수행기준** | 옵션 조합이 유효(캐스케이드 제약 통과) · 가격 = 서버 권위(클라 캐싱), PRICE=0 금지 |
| **수행주체** | 고객(셀프) / 위젯 프론트 + 가격 API. 권한: 익명 가능(주문 시 인증) |
| **연계 CQ** | CQ-PRICE-01~12, CQ-PROD-05~07 |

## T2. 주문·결제 (Order & Payment)

| 항목 | 내용 |
|---|---|
| **입력** | 장바구니 견적, 회원/배송 정보, 결제수단 |
| **출력** | 주문 레코드(7상태 머신 진입), 결제 케이스 확정, 주문번호 |
| **전제** | 회원 인증(또는 비회원 주문 정책), 결제 PG 연동, 재고/캐파 확인 |
| **수행기준** | 결제 완료 → "결제완료" 상태 전이 · 합배송 시 주문 1↔송장 N 허용 |
| **수행주체** | 고객 + 주문 시스템. 권한: 회원/비회원, 오프라인 주문은 관리자 별도 처리 |
| **상태 머신** | 7 정식 상태(order-flow.md §1.1) + 결제 케이스(§1.2) |
| **연계 CQ** | CQ-POL-01~06, CQ-SUB-08 |

## T3. 제작준비 — 파일입고·조판 (Pre-press)

| 항목 | 내용 |
|---|---|
| **입력** | 고객 업로드 파일(PDF/AI/JPG), 주문 사양, 판걸이/임포지션 규칙 |
| **출력** | 검수 통과 인쇄용 파일, 표준 파일명(RENAME), 조판 배치(판걸이수), 썸네일 |
| **전제** | 파일 검수 기준(해상도·CMYK·블리드·폰트), 4채널 파일명 일관화 매핑 |
| **수행기준** | G1 파일검수 통과 · 칼선/화이트 분리(스티커·아크릴) · 다도안 판걸이 산출 |
| **수행주체** | 디지털인쇄팀 + 자동검수(PitStop). 권한: 생산팀 파일다운로드 |
| **상태 머신** | 파일번호 부속 상태 머신(order-flow.md §1.3) |
| **연계 CQ** | CQ-FILE-01~09, CQ-PROC-08(G1) |

## T4. 제작 — 인쇄 (Production / Print)

| 항목 | 내용 |
|---|---|
| **입력** | 조판 완료 파일, 인쇄방식(옵셋/디지털/실사/실크/UV), 자재(종이/소재) |
| **출력** | 인쇄물(낱장/전지), 출력완료 바코드, 다음 공정 자동 입고 트리거 |
| **전제** | 인쇄방식=1상품1방식 확정, 자재 재고, 공정 라우트(17 Case) 셋업 |
| **수행기준** | 색상/출력 누락 검수(G2) · 앞공정("출력완료") 입력 후 다음공정 진입 |
| **수행주체** | 디지털인쇄팀 / 특수인쇄가공팀(실사·UV·패브릭). 상태마커 "출력완료" |
| **분기축** | 인쇄방식 → 가능 공정집합 결정 (process-recipe-tree.md L0) |
| **연계 CQ** | CQ-PROC-01~05, CQ-PROC-11 |

## T5. 후가공 (Finishing)

| 항목 | 내용 |
|---|---|
| **입력** | 인쇄물, 후가공 사양(코팅·박·형압·오시·미싱·제본·재단·커팅·봉제) |
| **출력** | 완성 반제품/완제품, 공정별 상태마커(제본시작·가공시작·봉재시작) |
| **전제** | 후가공 전후 의존(오시→접지, UV→레이저커팅), 외주 후가공(동판·박) 발주 |
| **수행기준** | 공정 택일그룹 제약 준수 · 제본별 필요 기초데이터(책등·링·면지) 충족 · G3 가공검수 |
| **수행주체** | 인쇄후가공팀·스티커가공파트·아크릴가공팀·봉재가공파트·굿즈가공 |
| **분기축** | 아키타입(통합/셋트/완제품) → 반제품 분해 여부(하드커버·포토북만 셋트) |
| **연계 CQ** | CQ-FIN-01~10, CQ-PROC-06~07 |

## T6. 납품 — 포장·출고 (Delivery)

| 항목 | 내용 |
|---|---|
| **입력** | 완제품, 배송 정보, 합배송 리스트, 송장 |
| **출력** | 포장물, 송장 출력, 출고완료, 배송 추적 |
| **전제** | G4 출고검수, 다중 송장(주문 1↔송장 N) 지원, 합배송 그룹핑 |
| **수행기준** | 사양/수량 일치 · 배송라벨·동봉물 확인 · "제작완료"→"송장출력" 마커 |
| **수행주체** | 포장팀. 상태마커 "제작완료"(1차포장)·"송장출력"(박스포장) |
| **연계 CQ** | CQ-SUB-08, CQ-POL-03(배송비) |

## T7. 정산 (Settlement) ⚠️ 최대 공백

| 항목 | 내용 |
|---|---|
| **입력** | 외주 매입(발주서·입고), 매출(주문 결제), 정산 주기 |
| **출력** | 외주사 정산 내역, 매출 집계, 마진/마크업 반영 |
| **전제** | 외주 발주서 생성 프로세스, 외주 단가 계약, 매입원가→판매가 마크업 규칙 |
| **수행기준** | **🔴 대부분 미수집** — 정산주기·불량책임·마진규칙 (D 인터뷰 필요) |
| **수행주체** | 관리자/정산 담당. 외주사(비즈하우스·후지필름·쿠마샵) |
| **연계 CQ** | CQ-SUB-02~07, CQ-SUB-09, CQ-PRICE-12 |
