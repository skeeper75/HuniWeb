## 0. 셋트 = 구성원(반제품) 조립 — 도메인 모델

부품조립형 셋트의 일반형:
```
SET(완제품) = Σ 구성원(member 반제품) + 조립공정(assembly/binding)
구성원 = { 역할(role), 자재, 인쇄(도수), 사이즈, 개수규칙(min/max/incr), (후가공) }
조립공정 = { 제본방식, 제본방향, 부속(면지/링/커버) }
가격 = Σ 구성원가 + 조립가 (구성원 합산형)  ―또는―  셋트 고정가(번들 단가)
```

인쇄 책자 도메인의 **역할(role)** 체계 [권위 booklet-l1 컬럼 직접 확인]:
- **표지(cover)** — 권위 컬럼 `표지종이/표지인쇄/표지옵션(코팅·투명커버)/박·형압`. 1부(앞뒤 한 장 또는 4면). 통상 두꺼운 용지(150g+).
- **내지(inner)** — 권위 컬럼 `내지종이/내지인쇄/내지페이지(min/max/incr)`. **페이지수 차원**(수량과 직교). 통상 얇은 용지(≤130g).
- **면지(end-paper)** — 권위 컬럼 `제본(옵션)_면지`. 내지 시작/끝의 컬러지(선택). 하드커버·고급 책자에서 등장.
- **간지(divider/interleaf)** — 권위 책자 시트 **컬럼 부재**(=후니 권위 침묵). 도메인상 챕터 구분 삽입지. → CONFIRM 후보(권위 미정의 구성원).
- **제본(binding 공정)** — 권위 컬럼 `제본(필수)` + `제본방향/링컬러/바인더링`. 셋트 조립의 핵심 공정.

### 1.1 제본방식별 셋트 조립 차이 (도메인)
- **중철(saddle-stitch)** — 표지+내지를 가운데 철심으로. 페이지 = **4의 배수**(접지 한 묶음 4면). 얇은 책자(권위 max 28p). 책등 없음.
- **무선(perfect)** — 표지로 내지 책등을 접착제로 감쌈. 책등 두께 = 내지 페이지×평량 함수(권위 무선책자 표지작업사이즈에 책등 반영). 박/형압 표지 가능.
- **PUR** — 무선의 강력접착제(PUR) 변형. 구성원 동형, 접착제만 상위. 더 두꺼운 책 가능.
- **트윈링/스프링** — 표지+내지 천공 후 **링(링컬러 구성원)** 삽입. 투명커버(앞)·뒷커버 추가 가능. 페이지 max 낮음(100p, 링 수용한계).
- **하드커버(양장)** — **하드보드(합지)** + 표지지(전용지/레더) 싸바리 + **면지(필수 구성원)** + 내지. 구성원 최다.

## 2. 엽서북 — 떡제본 카드 묶음 셋트 [권위 booklet-l1 실측]

```
엽서북 (ID=14535, MES=006-0007)  사이즈 100×150mm
구성원:
  - 표지: 스노우300g + 단면인쇄 + 무광코팅(단면)
  - 내지: 몽블랑240g + 단면인쇄
  - 제본: 떡제본(낱장 한쪽 끝 접착 — 한 장씩 떼어 쓰는 패드형)
개수규칙: 페이지(내지 장수) min20/max30/incr10 → 20·30장 택1 (권위 명시)
        수량(부수) min1/max10000/incr1
```
**도메인 해석**: 엽서북 = "한 장씩 떼어내는 엽서 패드". 표지(두꺼운 커버)+내지(엽서들)+떡제본 풀칠. 책자와 달리 **페이지=떼어낼 엽서 장수**(20/30장)·incr 10 단위. 표지가 1구성원, 내지 엽서가 N장. 가격 = 표지가 + 내지가(장수 선형) + 떡제본가.

## 3. 떡메모지 — 표지 없는 내지전용 떡제본 셋트 [권위 booklet-l1 실측]

```
떡메모지 (ID=14535, MES=008-0008)  사이즈 90×90mm
구성원:
  - 내지: 백모조120g + 단면인쇄   ← 유일 인쇄 구성원
  - 표지: 없음 (권위 표지 컬럼 전부 공란)
  - 제본: 떡제본(한쪽 끝 풀칠 — 떼어 쓰는 메모 패드)
개수규칙: 수량(부수) min3/max1000/incr3 → 3권 단위 (권위 명시)
        페이지(장수): 권위 공란 [모름] — 도메인상 떡메는 보통 100매/200매 등 매수 프리셋. 권위 미정의.
```
**도메인 해석**: 떡메모지 = "표지 없는 단순 메모 패드". 셋트 중 **최소 구성**(내지 1종 + 떡제본). 일부 떡메는 **바닥받침지(보드)**를 1구성원 추가하기도 하나 권위에 컬럼 없음 → 후니 권위는 내지+떡제본만으로 정의한 것으로 보임. 수량 incr=3은 3권 묶음 생산 단위(도메인 — 떡메는 보통 다권 묶음).

[모름] 떡메 내지 장수(매수) 개수규칙 = 권위 공란. 바닥받침지 구성원 유무도 권위 미정의 → CONFIRM 후보.

## 4. 개수규칙(min/max/incr) 도메인 관례 (권위 침묵 시 후보)

| 구성원/축 | 권위 명시(있으면 정답) | 도메인 관례(권위 침묵 시 후보) |
|---|---|---|
| 표지 부수 | (셋트 수량과 동일·1:1) | 표지 1 per 책 (책자=표지1·내지N) |
| 내지 페이지 | 책자별 min/max/incr (위 표) | 중철=4배수·무선/PUR=2배수·링=2배수 (접지/제본 단위) |
| 면지 | 하드커버=화이트면지(필수 보임) | 하드커버류만 1쌍(앞뒤) |
| 간지 | **권위 컬럼 없음** | 0~N장 선택 삽입(챕터수 종속) — 권위 미정의 |
| 셋트 수량(부수) | 책자별 min/max/incr (위 표) | 떡메=3배수, 엽서북=1단위 |

## 5. 셋트 가격구성 모델 (도메인 — §competitor-set-reference 와 연계)

두 모델이 도메인에 공존:
1. **구성원 합산형** = 표지가 + 내지가(페이지 선형) + 제본가 + 후가공가. → 후니 권위 골든(PRF_BIND_SUM)이 **제본비를 단독 항(권당가×권수)**으로 노출 = 이 모델.
2. **셋트 통합 산정형(흡수형)** = 인쇄/제본을 한 엔진이 통산, 제본비 별항 0. → 경쟁사(레드 PRBKYPR)가 BIND=0으로 인쇄에 흡수 = 이 모델.

★후니 권위는 제본비를 분리 노출하는 **구성원 합산형**에 가깝다(골든 G1~G8 제본비 단독). 설계가는 셋트 가격을 "구성원가 합산"으로 조립하되, 권위 골든 단가를 verbatim 사용해야 함. 상세는 `competitor-set-reference.md` §3.
