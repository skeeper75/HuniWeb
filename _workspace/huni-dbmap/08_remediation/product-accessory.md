# product-accessory(상품악세사리) 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 **누락·오매칭·모호**가 생겼다.
> 본 문서는 그 결함을 *올바로 처리(수정)*하기 위해 **라이브 Railway DB 직접 read-only SELECT를 권위**로
> product-accessory 상품군(15상품, 부자재·악세서리)을 종단 규명한다. 도메인 지식(07_domain KB)은 결함을 옳게 분류하는 렌즈다.
> 정제된 DB가 인쇄 자동견적 위젯의 기반이 된다.
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본·v2 stale 가능). 추정·임의판단 0 — 라이브·엑셀 L1·도메인 KB 근거만.
> 미지/추정은 가설+컨펌 플래그. INSERT/UPDATE/DDL 0. DB 미적재 유지. `db railway`·비표준포트·JOIN KEY=`prd_nm`.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. **라이브 SELECT 수행 약 7회**(전부 read-only).

## ⚠️ 머리말 — 본 검증의 가장 큰 사실 (권위 반전·정합 확인)

product-accessory 시트는 **goods-pouch와 정반대로, 라이브 적재 정합도가 높다.**

라이브 직접 확인 결과:
- **15상품 전부 라이브 등록**(`prd_nm` exact 매칭, unmatched 0, PRD_000001~015 연속 블록, 전부 `PRD_TYPE.03`).
- **엑셀 `사이즈(필수)`칸의 size성↔material성 분기가 엑셀 의도대로 정확히 적재됨.**
  봉투류(치수+장수, 예 `70x200mm(50장)`)는 `t_prd_product_sizes`에, 색상/규격 variant(볼체인 색상 8종, 와이어링
  색상 3종, 우드봉 길이 3종, 만년잉크 색상 7종)는 `t_prd_product_materials`에 분리 적재됨.
  - 투명케이스(009) size 3행 = 엑셀 `PP투명케이스 42x57x20mm(10개)` 등과 **정확 일치**.
  - 볼체인(006) material 8행 = `오렌지(3개1팩)`~`화이트(3개1팩)` variant **정확 적재**(USAGE.07).
- **size·material 둘 다 0인 완전 누락 상품 = 0** (15상품 전부 둘 중 하나 이상 적재).
- bundle_qty도 OPP봉투 2상품에 50장 묶음(`bdl_qty=50, QTY_UNIT.01`, reg_dt **2026-06-05 최신 보정**).

**교훈(HARD):** goods-pouch는 size 거의 전무(plate만 적재)였으나, product-accessory는 동일 시트군임에도
size/material 분기가 잘 됐다. 즉 **결함은 시트 전체가 아니라 상품군별로 편차가 크다** — 라이브 상품별 확인이 필수.
다만 **공정 컬럼 자체가 엑셀에 부재**(digital-print 분석서 확인)하고, addon·process가 전무한 것은 도메인상 정상인지
부분 누락인지 구분이 필요하다(§4).

## ① product-accessory 상품군 — 도메인 이해 (라이브 권위)

**라이브 SELECT:** 엑셀 L1 15상품명 → `t_prd_products` 매칭 = **15 matched / 0 unmatched**.
전부 `prd_typ_cd = PRD_TYPE.03`(부자재/악세서리), `prd_cd` PRD_000001~015 연속 블록.

product-accessory 시트 = 후니의 **포장 부자재 + 상품 악세서리**. 구분(엑셀 `구분`) 2개 군:

| 구분 | 상품(prd_nm → prd_cd) | 인쇄 도메인 의미 |
|------|----------------------|------------------|
| **봉투/케이스**(11) | OPP접착봉투(001)·OPP비접착봉투(002)·트래싱지 카드봉투(003)·카드봉투(004)·캘린더봉투(005)·볼체인(006)·와이어링(007)·천정고리(008)·투명케이스(009)·행택끈(010)·자석고정용고무판(011) | 포장재·걸이 부자재. 봉투=치수×장수 size, 걸이/끈=색상 variant material |
| **상품액세서리**(4) | 우드거치대(012)·우드봉(013)·우드행거(014)·만년스탬프 리필잉크(015) | 굿즈/스탬프 부속. 우드=길이 variant, 잉크=색상 variant |

- **이 시트는 "addon으로 다른 상품에 붙는 부속" + "자체 판매 부자재"의 이중 성격.** 예: 봉투/볼체인은 굿즈·엽서의
  addon(goods-pouch 말랑키링 addon=볼체인 PRD_000006, digital-print 엽서 addon=OPP접착봉투 PRD_000001)으로도 쓰임.
- **공정(process) 컬럼 부재(HARD):** digital-print 분석서가 확인한 대로 product-accessory 엑셀에는 인쇄옵션/별색/공정
  컬럼 자체가 없다(컬럼=`사이즈/수량(최소·최대·증가)/가격`뿐). 따라서 process 0행은 **결함 아닌 정상**(원천 부재).
- **use_yn=N (비활성, MISSING 아님):** 천정고리(008) 1상품. (material 1행은 적재돼 있어 데이터는 보존됨.)
- **신규(노랑 FFFFFF00):** OPP비접착봉투·천정고리·우드봉·만년스탬프 리필잉크 등 일부 셀 신규 표기(행 추가 진행).
- **행숨김 0 · 회색밴딩 0:** goods-pouch와 달리 그레이밴딩 없음(미적재 마킹 상품 없음).

## ② 엑셀 옵션 구조 (product-accessory L1, 무손실 추출 권위)

L1(`06_extract/product-accessory-l1.csv`, 67 데이터행·D 세로병합 블록 구조)이 표현하는 구조는 **단순**하다:

| 엑셀 컬럼 | 의미 | DB 정규화 대상 |
|-----------|------|----------------|
| `D 상품명`(세로병합) | 상품 블록 앵커(`_anchor_ffilled`로 ffill) | `t_prd_products.prd_nm` |
| `사이즈(필수)` | **치수×장수**(봉투 `70x200mm(50장)`) 또는 **색상/규격 variant**(볼체인 `오렌지(3개1팩)`) | **size**(봉투류) 또는 **material**(색상 variant) — 분기 |
| `수량(필수)_최소/최대/증가` | 수량 규칙 | (products.qty 필드) + bundle(묶음장수) |
| `가격` | 행별 단가 | (round-2 t_prc_*, 본 라운드 이연) |

**핵심 도메인 캐스케이드(반영):**
- **D 세로병합 블록:** 상품명이 첫 행에만 있고 아래로 병합 → L1이 `_anchor_ffilled=true`로 ffill 보존. 상품 경계 정확.
- **size↔material 분기(R-MAT-5 교차):** `사이즈(필수)`칸이 **치수문자열이면 size**, **색상/규격 variant면 material**.
  - size성: OPP봉투(치수+장수), 트래싱지/카드/캘린더봉투, 투명케이스(PP케이스 치수), 자석고무판(20x20).
  - material성(색상/규격 variant): 볼체인(색상8)·와이어링(색상3)·행택끈(규격3)·우드봉/우드행거(길이)·만년잉크(색상7)·우드거치대.
- **묶음장수(50장/20장/10개):** size 문자열의 괄호 수량 = bundle_qty 신호. OPP봉투에 bundle 50 적재됨.
- **addon vs 가격전용 구분:** 이 부자재들은 다른 상품의 addon으로 쓰이나, 본 시트에서는 자체 상품(가격행)으로 등록.

## ④ 결함 표 — 누락(MISSING)·오매칭(MISMATCH)·모호(CONFIRM) (도메인 렌즈, 라이브 재확인)

> product-accessory는 큰 BLOCKER 없음. 미세 검토 항목 위주.

### G-PA-1 [CONFIRM→정상] size↔material 분기 = 엑셀 의도대로 정합
- **라이브 재확인:** 투명케이스(009) size 3행 = 엑셀 PP케이스 치수와 정확 일치. 볼체인(006) material 8행 = 색상 variant 정확.
- **판정:** **결함 아님 · MATCH.** 봉투류=size, 색상/규격 variant=material 분기가 옳게 적재됨. (goods-pouch와 대조적 모범 사례.)

### G-PA-2 [CONFIRM·잠재 모호] size 문자열에 치수+장수+색상 혼재 (정규화 입도)
- **도메인 근거:** OPP비접착봉투 size = `60x90mm(20장)`·`60x90mm(50장)` — **같은 치수·다른 장수**가 별도 size 행.
  카드봉투 = `화이트 165x115mm(10장)`·`블랙 165x115mm(10장)` — **색상이 size 문자열에 포함**.
- **모호:** 장수(20/50장)는 size 차원인가 bundle_qty 차원인가? 색상(화이트/블랙 카드봉투)은 size에 묶을지 material로 뺄지?
  라이브는 카드봉투를 size 2행(색상 포함 문자열)으로 적재 — 볼체인은 색상을 material로 뺐는데 카드봉투는 size에 둠 → **분기 기준 비일관**.
- **판정:** **CONFIRM** — 장수=bundle 분리 여부, 색상=size/material 분기 일관 기준 확정 필요. (현 적재는 동작하나 위젯 옵션 입도에 영향.)

### G-PA-3 [CONFIRM] bundle_qty 부분 적재 (OPP봉투 2상품만)
- **라이브:** bundle_qty 적재 = OPP접착/비접착봉투 2상품(50장)뿐. 그러나 size 문자열에 `(50장)/(20장)/(10개)`를 가진
  봉투/케이스 상품은 더 많다(트래싱지·카드·캘린더·투명케이스 등).
- **MISSING(가설):** 묶음장수를 가진 다른 봉투류의 bundle_qty 미적재 가능. 단 장수가 size 문자열에 이미 포함돼 있어
  bundle 별도 적재가 필수인지 정책 의존.
- **판정:** **CONFIRM** — bundle을 size에서 분리해 일괄 적재할지, size 문자열 내 표기로 충분한지 정책 확정.

### G-PA-4 [정상] process / addon / discount / plate 전무 = 도메인상 정상
- 공정 컬럼 엑셀 부재 → process 0 정상. 자체 부자재 → addon 0·discount 0·plate 0 정상. price 0 → round-2 이연 정상.

### G-PA-5 [참고·MES_CD 중복] MES ITEM_CD 상품 간 공유
- **라이브/엑셀:** 우드거치대·우드봉 = `012-0012` 공유, 우드행거·만년잉크 = `012-0013` 공유 등 MES_CD가 여러 prd_nm에 중복.
- **판정:** JOIN KEY는 `prd_nm`이고 MES_ITEM_CD는 라이브 전부 NULL(메모리 권위) → 매핑에 무영향. **결함 아님**(참고 기록).

## ⑥ 실무 컨펌 질문 목록 (top)

1. **[R1·size 입도]** 카드봉투 size에 색상(화이트/블랙)이 포함됐는데, 볼체인은 색상을 material로 분리함.
   색상 variant를 size에 둘지 material로 뺄지 **일관 기준**을 정할 것인가? (위젯 옵션 표시 입도에 직결)
2. **[R2·bundle]** 봉투 size 문자열의 묶음장수(50장/20장/10개)를 별도 bundle_qty로 분리 적재할지, size 표기로 충분한지?
   현재 OPP봉투 2상품만 bundle 50 적재됨.
3. **[참고]** 이 부자재들(봉투·볼체인 등)이 다른 상품의 addon으로 참조되는데(엽서 addon=OPP봉투, 말랑키링 addon=볼체인),
   본 시트 자체 등록과 addon 참조의 관계 — 단가 이중 정의 우려는 없는가? (round-2 가격 설계 시 확인 대상)
4. **[use_yn=N]** 천정고리(008) 미출시 — 결함 없음(material 적재됨). 출시 시 추가 작업 불필요 확인.

## ⑦ 큰 그림 적용 (도메인 → 엑셀 → DB결함 → 처리 → 위젯)

- **도메인 KB(07_domain):** size↔material variant 분기·공정 컬럼 부재·부자재 addon 이중성이 "사이즈칸 문자열"을
  옳게 해석하는 렌즈 → 원적재가 **이 시트는 비교적 잘 처리했음**을 라이브로 확인(거짓 결함 양산 방지).
- **엑셀 L1(무손실):** D 세로병합 ffill·치수/색상 문자열 보존 덕에 size성/material성 분기를 정확 판정(추정 0).
