## 0. V1/V2 분류 원칙 (analyst inference — 사용자 추후 검토 전제)

huni-ia-master xlsx는 우선순위(K열)·규모(L열) 컬럼이 **전부 공란**이므로 우선순위를 명시하지 않는다(huni-ia-master §A). 따라서 본 사이트맵의 V1/V2는 다음 원칙으로 **잠정 분류**하며, 경영진/담당자 확정 전까지는 추론값이다.

- **V1 = 런칭 필수**: 상품 카탈로그·견적·옵션·디자인 에디터 진입·장바구니·주문·결제·기본 회원·핵심 어드민(상품/주문/가격관리)
- **V2 = 런칭 후**: 체험단·이벤트·통계 세분·세분 권한·게시판형 부가기능·마케팅 도구
- 모호 항목은 **`V1?`** (담당자 협의 대상)으로 마킹
- **별도트랙** = huni-ia-master IA 화면에 없으나 빌더/생산운영 메타에서 파생된 화면(공정관리 PDF·위젯빌더 메타). V1 운영 최소형 필요 여부는 `V1?`로 표기.

### 표기 규칙

- 페이지 ID 체계: `SM-{tier}-{NN}` (M=메인/정보, Q=견적/주문, AC=인증, MY=마이페이지, CS=고객센터, A=어드민)
- 모달/하위행은 부모 ID에 `.NN` suffix
- **출처** 컬럼: huni-ia-master의 No(예: `IA-30`) 또는 `↳`(하위행) / `BUILDER`(별도트랙) / `BENCH`(경쟁사 마이페이지 벤치마크 발 보강, D-DS-38)
- 접근권한: G=게스트, U=회원, A=관리자(단일)
