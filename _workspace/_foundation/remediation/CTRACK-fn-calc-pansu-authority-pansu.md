# C트랙 수정 명세 — fn_calc_pansu 권위 판걸이수 기반 계산 (2026-06-28)

> 위상: 개발팀/실무진 판단·인간 승인 C트랙(DB 함수 + 데이터). 사용자 결정 기록.
> ★이전 사이트 기준 가격 검토의 **키** — 이전 ASP도 실무진 엑셀 판수로 계산했을 가능성.

## 문제 (PR 골든서 적발·프리미엄엽서 PRD_000016)
가격엔진은 판형기준 구성요소(디지털인쇄비·용지비)를 **판수 = ⌈주문수량 ÷ 판걸이수⌉**(pricing.py:215 `plate_qty` ceil)로 계산.
판걸이수 = `fn_calc_pansu(판형, 완제품)` = **출력가능영역(판형 work − 여백) 기하 패킹** max(정상,회전).
→ 권위 엑셀(상품마스터 "판수" 컬럼=실무진 기록)보다 **과다 산출** → 판수 과소 → **인쇄비 저청구**. 전 디지털인쇄 systemic.

**근본:** fn_calc_pansu는 **출력가능영역은 맞게 뺌**(여백 5mm). 그러나 ① 아이템 간 **칼선 간격(gutter)** ② **물림변(그리퍼·보통 10mm+)** 미반영 → 꽉 채워 과다 패킹. 실제 인쇄는 1열/행 덜 앉음.

## 결정 (사용자·2026-06-28)
**실무진이 엑셀에 기록한 판걸이수(판수)를 권위로 하여 판수를 계산.** fn_calc_pansu를 변경해 그 권위 값을 쓰도록 수정. (기하 계산은 권위 부재 시 폴백.)

## 수정 명세 (개발팀)
1. **권위 판걸이수 저장소 신설**(현재 컬럼 없음): 후보
   - (a) 신규 테이블 `t_siz_pansu(plate_siz_cd, item_siz_cd, pansu)` — 판걸이수는 (판형×아이템) 종속이라 정규.
   - (b) `t_prd_product_sizes`에 `pansu` 컬럼 — 상품×사이즈 단위(판형 자동도출 맥락).
   - 권장=(a) 정규·재사용(동일 판형×아이템은 동일 판걸이수). 적재원=상품마스터 "판수" 컬럼.
2. **fn_calc_pansu 수정**: 권위 저장소 lookup 우선 → 있으면 그 값 반환·없으면 기존 기하 폴백.
   ```
   SELECT pansu INTO v FROM t_siz_pansu WHERE plate_siz_cd=p_plate AND item_siz_cd=p_item;
   IF v IS NOT NULL THEN RETURN v; END IF;
   -- else 기존 기하 계산(폴백)
   ```
3. **적재**: 상품마스터 디지털인쇄 시트 "판수" 컬럼 전 상품×사이즈 → t_siz_pansu(판형=출력용지규격·아이템=완제품사이즈). 프리미엄엽서=판형316x467×{73x98:15·98x98:12·100x150:8·135x135:6·95x210:6·110x170:4·148x210:4}.

## 영향·검증
- **영향:** 전 디지털인쇄 원자합산형(엽서군·상품권·슬로건…) 인쇄비. 판수↑ → 인쇄비↑(저청구 교정). 프리미엄엽서 100x150 qty100: 12판→13판(권위).
- **★검증 키:** 이전 후니프린팅 라이브 사이트(goods.asp)가 같은 판수로 계산하는지 교차검증 = 권위 판걸이수가 맞는지 최종 확인(실무진 엑셀=이전 사이트 산식일 가능성).
- 실무진 확인: 칼선간격·물림변 실측으로 엑셀 판수(15/12/8) 재확인.
