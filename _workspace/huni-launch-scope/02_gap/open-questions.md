## A. 돈/주문 크리티컬 — 1차 가격 브리지 (★최우선)

### OQ-G1. 동적 계산가 환원 전략 확정 (P-A vs P-B vs placeholder)
- **무엇:** final_price를 Shopby 주문 라인에 무손실로 싣는 1차 전략. 직접 주입은 스펙상 불가(확정).
- **선택지:** P-A 사전 옵션 동기화(정산 안전·조합 폭발) / P-B 직전 동적옵션(무손실·심사 리스크) / IA 비고 placeholder(수량1+옵션금액·정산 정합 검증).
- **권고:** GREEN 고정가형/소조합 = P-A 파일럿. 연속/매트릭스형은 OQ-G2 해소 전까지 보류.
- **종속:** OQ-G2·OQ-G3. **결정 주체:** integration-architect/scope-gate + PM.

### OQ-G2. P-B 동적옵션이 상품심사(judgement)를 유발하는가 (OQ-3 계승)
- **무엇:** 주문 직전 addPrice 옵션 동적 등록/변경이 "수정 후 승인대기"(판매 차단)를 유발하는지. 자동승인 예외 여부.
- **근거:** enterprise judgement.mdx:9,35 — 일부 정보 수정 시 재심사. addPrice 변경 예외 미확인.
- **영향:** 막히면 P-B 불가 → 무손실 길은 P-A(사전 동기화)만. **D 채택 결정적 관문.**
- **검증:** docs.shopby.co.kr 상품심사 정책 + 라이브 admin 읽기 탐색.

### OQ-G3. Shopby 정산이 라인 salePrice/addPrice를 권위로 쓰는가 (OQ-4 계승)
- **무엇:** 정산/세금계산서(NATIVE)가 라인 등록가 vs 결제 실청구액 중 무엇을 권위로.
- **영향:** placeholder 전략(후니 별도 권위가) 시 정산 왜곡 위험. 어느 경우든 "라인가=후니가" 동기화 필요 가능성.
- **검증:** settlement yml + 정산 매뉴얼 갭필.

## B. 회원/머니 모델 격차 (2차 결정·1차 영향)

### OQ-G4. 프린팅머니 = Shopby 적립금 재해석 vs 별도 원장
- **격차:** Shopby 적립금=구매적립형 / 후니=충전식 선불 잔액. 모델 상이.
- **선택지:** (a) 적립금 도메인에 충전 운용 / (b) 커스텀 잔액·원장 BFF.
- **종속:** PG 연동·legacy 머니 마이그레이션(잔액+트랜잭션 원장). **migration-designer 연계.**

### OQ-G5. 회원 사업자정보·다중주소를 Shopby 어디에 담는가
- **무엇:** 라이브 회원 = 개인+사업자(상호·사업자번호·대표자·업태/업종)+회사주소2(30+필드). Shopby 표준 프로필 외 확장분.
- **선택지:** customField vs 별도 BFF 저장. **검증:** Shopby member customField 수용 범위.

### OQ-G6. 회원등급 grade_cd ↔ Shopby memberGrade 매핑 (1차 가격 영향)
- **무엇:** evaluate_price grade_cd(후니 등급할인) ↔ Shopby memberGrade 도메인 매핑. 라이브 등급할인 0행(현재 비발화).
- **영향:** 1차 가격 calculate에 회원할인 적용순서(상품→회원할인→쿠폰→머니) 정합. **등급 정책 조기 확정.**
