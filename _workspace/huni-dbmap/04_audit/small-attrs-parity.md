## 속성 1: 페이지룰(page_rule)

### 4분류
| 분류 | 카운트 |
|------|:-----:|
| **MATCH** | **11** |
| MISSING (엑셀O DB X) | 0 |
| EXTRA (DB O 엑셀 X) | 0 |
| MISMATCH | **0** |

### 명세 (11/11 상품 min/max/incr 완전 일치)
DB 11행 ↔ 엑셀 11상품(값 보유), 모두 일치:

| prd_nm | DB (min/max/incr) | 엑셀 원천 | 판정 |
|--------|-------------------|-----------|------|
| 중철책자 | 4/28/4 | `4\|4\|28\|28\|4\|4` | MATCH |
| 무선책자·PUR책자·하드커버책자·레더 하드커버책자 | 24/300/2 | `24\|24\|300\|300\|2\|2` | MATCH |
| 트윈링책자·하드커버 링책자 | 8/100/2 | `8\|100\|2` / `8\|8\|100\|100\|2\|2` | MATCH |
| 엽서북 | 20/30/10 | `20\|30\|10` | MATCH |
| 포토북 [디자인명] | 24/150/2 | `24\|24\|24\|24\|150...\|2...` | MATCH |
| 먼슬리플래너 | 28/28/0 | `28\|28\|0` | MATCH |
| 떡메모지 | 3/3/3 | `3\|3\|3\|3\|3\|3` | MATCH |

- 엑셀 6열 중복(min min max max incr incr) → 3값 정규화 후 DB와 대조. 표기 형태만 다르고 값 동일.
- **자기 오탐 철회**: 1차 스크립트가 연속중복 dedup으로 `먼슬리플래너` `28\|28\|0`을 `(28,0)`으로 잘못 축약해 MISMATCH로 표시 → **철회**. 원천 `28\|28\|0`=min28/max28/incr0=DB `(28,28,0)`로 **MATCH**.

### 판정: **GO (페이지룰 완전 정합)**
모집단·값 전건 일치, EXTRA/MISSING 0. 후니 확인항목 없음.

## 속성 2: 공정택일그룹(process_select_group)

### ⚠️ 엑셀 원천 부재 — 정합 대조 불가

### sel_typ_cd 코드 정합
- 사용 코드 = `{SEL_TYPE.01}` 단일 → SEL_TYPE 유효 부분집합(`.01 단일`/`.02 다중`) 내 정합. **다중(.02) 미사용**.
- `max_sel_cnt`=1, `mand_yn`=Y 전건 일관(택일그룹=필수 단일선택 의미와 정합).

### 4분류: **대조 불가 (N/A)**
- MATCH/MISSING/EXTRA/MISMATCH **산정 불가** — 엑셀 권위 원천 부재로 비교 분모 없음.

### 판정: **확인 필요 (CONFIRM — 엑셀 원천 부재)**
- DB 13행 자체는 내부 정합(코드 유효·일관). 그러나 **엑셀에 대조할 원천이 없어 정합 입증 불가**.
- **후니 확인항목**:
  1. DB 택일그룹 도출 근거(공정 옵션-컬럼=택일그룹 추론)가 후니 의도와 일치하는지 확인.
  2. 책자 10상품의 trigger(제본방식)가 각 상품의 실제 제본옵션과 맞는지(예 포토북 trigger=`PUR` vs 포토북 시트 제본옵션) 검증.
  3. 캘린더 GRP-CAL-가공 3상품의 trigger(우드거치대/고리형트윈링제본)가 가공옵션 의미와 정합하는지.
  4. 향후 다중선택(SEL_TYPE.02) 택일그룹이 필요한 상품이 있는지(현재 전부 단일).

| MISMATCH | (미산정) | addon 값 세부 1:1 대조는 표기차 多 — 별도 작업(아래) |

### addon FK 정합: **PASS (위반 0)**
- DB 34행의 `addon_prd_cd` 전건이 `t_prd_products.prd_cd` 실존 상품으로 해소(**self-ref FK 위반 0**).
- 참조 addon 상품 9종 전부 실재: `OPP접착봉투`·`OPP비접착봉투`·`카드봉투`·`투명케이스`·`캘린더봉투`·`우드거치대`·`우드행거`·`우드봉`·`볼체인`. (이들은 §봉투/악세사리 상품군 = 묶음수 정정·모집단 분석의 PRD_TYPE.03 상품군과 일치)

### MATCH 19 parent 상품
엽서 7종(프리미엄/코팅/스탠다드/투명/화이트인쇄/핑크별색/금은별색엽서)·인쇄배경지 2종·인쇄헤더택·탁상형캘린더·엽서캘린더·캔버스 행잉포스터·린넨 우드봉 족자·아크릴키링·아크릴 포카키링·말랑키링·말랑증사홀더·말랑포카홀더. 양측 모두 추가상품 보유, addon_prd_cd FK 정합.

### MISSING 11 명세 (엑셀 추가상품 보유, DB addons 미적재)
- **실addon 7** (DB 적재 필요): `PET배너`·`메쉬배너`(배너거치대), `만년스탬프`(리필잉크 7색), `메쉬현수막`·`일반현수막`(끈/큐방/각목 추가), `족자포스터`(천정형고리), `아크릴쉐이커코롯토`(아크릴스탠드). → DB addons MISSING, 후속 적재 대상.
- **가격전용 4 (addon 아님)**: `방수포스터`·`접착방수포스터`·`폼보드`(`0.0 \| 0.0`), `지비츠★`(`500.0`). 추가상품명 없이 가격값만 → **실 addon 아님, addons 미적재 정당**(MISSING 아님으로 재분류 권고).

### 판정: **MAJOR (실addon 7 MISSING) + addon FK는 PASS**
- 적재된 34행은 FK·parent 전건 정합(거짓 정합 없음). EXTRA 0.
- 그러나 엑셀 실addon 보유 상품 중 7종(배너/현수막/만년스탬프/족자/아크릴쉐이커)이 DB 미적재 = 결손.
- **후니 확인항목**:
  1. 실addon 7종을 addons 적재할지(배너거치대·잉크·끈 등 addon_prd_cd가 될 상품이 t_prd_products에 등록돼야 함 — 미등록이면 선행 등록 필요).
  2. 가격전용 4종은 추가상품이 아닌 **옵션가격(가격엔진 round-2 영역)**으로 재분류 — addons 미적재가 옳음.
  3. addon 세부값(수량·사이즈 `OPP접착봉투 110x160 mm 50장`)의 1:1 대조는 DB note 자유텍스트 vs 엑셀 파이프리스트 표기차로 별도 정규화 작업 필요(현 단계는 parent·FK 정합까지).
