## 0. 보정 트리거 — 검증가 부분 GO + 사용자 directive 4축

REV3까지의 설계는 "면적→등급 환산 = 앱 계산 = 엔진 미지원 = C트랙(개발팀 코드 선결)"로 가변사이즈 7상품(접지카드·책자) 적재를 보류했다. 검증가가 라이브로 **"등급을 적재 시점에 단가에 미리 펼치면 박가공비가 1단 면적매트릭스가 되고 현 엔진 off-grid ceiling이 코드변경 0으로 처리한다"**를 실증(아크릴 79×85→(80,90) 비대칭 ceiling)했고, 그 과정에서 **박가공비 prc_typ을 designer가 잡은 `.02`가 off-band 수량에서 과청구**하는 새 결함(qty 850→56,100 vs 52,800)을 적발했다. REV4는 이 둘 + 사용자 directive 2건(박 크기 제한·comp 통합)을 흡수한다.

| 변경 | 종류 | 결과 |
|---|---|---|
| **C-1. 면적→등급 flatten → C트랙 제거** | directive #1 + 검증가 GO | 박가공비 = `(proc_cd, siz_width, siz_height, min_qty)→단가` 1단 면적매트릭스. **C트랙 소멸**·가변사이즈 7상품 지금 적재 가능·신규 테이블 불필요 |
| **C-2. 박가공비 prc_typ `.02`→`.03 FLAT`** | directive #2 + 검증가 적발(돈크리티컬) | off-band 수량 과청구 제거. 골든 8/8 불변 + off-band 2 신규 골든 |
| **C-3. 박 크기 제한 per-product 라이브 조사** | directive #3 | 박영역 ≤ 상품크기. 7상품 트림사이즈 라이브 실측 → 소형/대형 배정 **5/7 데이터 확정·2 CONFIRM** |
| **C-4. comp 통합 vs 과분리 재검토** | directive #4 | 통합/분리 기준 적용 → **5 comp 유지(통합 불가 입증)**·각 comp 통합/분리 근거 명문화 |

## 1. C-1 [directive #1·검증가 GO] — 면적→등급 flatten으로 C트랙 제거

### 1-1. REV3 모델(C트랙) → REV4 모델(데이터 흡수)

REV3 §3은 박가공비를 **2단**으로 봤다: ① 면적격자(B02/B04: 가로×세로→등급 A~E) = 앱 lookup, ② 등급단가표(B03/B05: 등급×수량→단가) = DB. 1단(면적→등급)이 엔진 미지원(NON_QTY_DIMS/TIER_DIMS에 grade 없음·`pricing.py:42-50`)이라 가변사이즈 상품을 C트랙으로 보류.

★ **REV4 = 적재 시점에 등급을 단가에 미리 펼친다(grade pre-expand)**. 면적격자 + 등급단가표를 **적재 전에 결정론으로 join**해, 박가공비 단가행을 `(proc_cd, siz_width, siz_height, min_qty) → 단가` 1단 면적매트릭스로 emit한다. 그러면:

- grade 차원 자체가 단가행에서 **소멸**(엔진이 grade를 환산할 필요 없음).
- 남는 건 width/height ceiling + min_qty band = **엔진이 이미 하는 연산**(아크릴 `COMP_ACRYL_CLEAR3T` 라이브 277행 동형·검증가 실증).
- 가변사이즈(접지카드·책자)도 **지금 적재 가능**·신규 `t_prc_foil_area_grades` 테이블 **불필요**(기존 `t_prc_component_prices`에 siz_width/siz_height/min_qty 컬럼 실재).

### 1-2. 검증가 실증 근거 (라이브·순수함수)

| 실증 | 결과 |
|---|---|
| 아크릴 79mm×85mm → matched (80,90) | 각 축 **독립 ceiling**(79→80 머묾·85→90 상승). 박 면적셀 선택과 동일 연산 |
| 순수함수 소형 일반박 flatten 135행으로 직접 호출 | 35×35 q1000 → (40,40,minq1000,57000=D) None err / 40×80 q1000 → (40,80,minq1000,64000=E) |
| 등급격자 monotonic 검사 | 대형 8×8·소형 3×5 **양 축 비감소·위반 0** → ceiling이 항상 권위 등급셀에 착지(잘못된 경계 넘음 0) |

### 1-3. ★결정론 flatten 규칙 (적재 명세)

박가공비 1단 면적매트릭스 행 생성 규칙(적재 빌더가 결정론으로 수행·날조 0):

```
입력: 면적격자(B02 소형/B03 대형 면적정의 블록·가로×세로→등급 A~E)
      등급단가표(B03 소형/B03·B05 대형 등급×수량→단가)
      박색상→{일반|특수} 그룹·시트별 매핑(§2-3 engine-design)

알고리즘 (per 시트 × {일반|특수} 그룹):
  FOR 각 면적셀 (w_tier, h_tier) in 면적격자:        # 격자 셀의 가로/세로 = 티어 상한값
      grade = 면적격자[w_tier][h_tier]              # A~E
      FOR 각 수량구간 min_q in 등급단가표.수량밴드:
          단가 = 등급단가표[grade][min_q]            # verbatim
          EMIT row {
            proc_cd     = (그 그룹의 박색상들·아래 색상 복제 규칙)
            siz_width   = w_tier   (NUMERIC·'이하' 상한 키)
            siz_height  = h_tier   (NUMERIC·'이하' 상한 키)
            min_qty     = min_q    (INTEGER·'이상' 하한·NOT NULL)
            unit_price  = 단가      (권위 verbatim)
            prc_typ_cd  = PRICE_TYPE.03 (FLAT·C-2)
            note        = grade   # 추적용: 이 면적셀의 권위 등급(검증가 재현·역추적)
          }
```

- **키 = 티어 상한값**(권위 '이하' 경계). 엔진 ceiling('주문값 이상 최소 임계')이 그 사이즈를 담는 가장 작은 셀 = 권위가 그 사이즈에 부여하는 등급셀과 정확히 동일(검증가 monotonic 입증).
- **note = grade 보존**: 단가행에 등급 라벨을 note로 남겨, 검증가·운영자가 "이 면적셀이 권위상 어느 등급인지" 역추적 가능(grade는 매칭에 안 쓰임·추적 전용).
- **off-grid**: 손님 가로/세로가 격자값 사이면 엔진이 각 축 독립 ceiling(코드변경 0). 최대 초과(>170대형·>80소형)=ERR_ABOVE_MAX(CONFIRM 영역·§6).

### 1-4. 행수 추정 (검증가 (d) 인용)

| comp | 면적셀 | 수량밴드 | 행수(색상1) |
|---|---|---|---|
| 소형 일반 박가공비 | 15 (3×5) | 18 | 270 |
| 소형 특수 박가공비 | 15 | 18 | 270 |
| 대형 일반 박가공비 | 64 (8×8) | 13 | 832 |
| 대형 특수 박가공비 | 64 | 13 | 832 |

★ **색상 복제 트레이드오프**: 일반/특수 그룹 내 색상은 단가 동일 → proc_cd 게이트 충전 시 그룹 색상 수만큼 행 복제(예 대형 일반 6색 → 832×6=4,992행). 대안 = 1색 키잉 후 CPQ가 색상→그룹 환원(행수 절감). **적재 명세 결정사항**(인간 승인 시 택1). 어느 쪽이든 라이브 7,293행과 동급·스키마 압박 없음.

### 1-5. 동판비도 flatten 정합 (불변·이미 면적매트릭스)

동판비 C-1(대형)은 REV3부터 이미 `(proc_cd, siz_width, siz_height)→단가` 면적매트릭스(72셀). flatten과 동형 — grade 개념 없음(동판비는 등급 무관·면적만). C-2(소형)는 단일 고정 5,000. 둘 다 변경 없음(단 §2 C-2/C-3 통합 재검토는 C-4 참조).

## 2. C-2 [directive #2·검증가 적발·돈크리티컬] — 박가공비 prc_typ `.02`→`.03 FLAT`

### 2-1. 검증가 발견 — `.02`가 off-band 수량 과청구

REV3 §4-가드는 박가공비를 `.02 합가형 + min_qty=구간하한`으로 잡고 "명함박 동형이라 작업 1건 고정 성립"이라 정당화했다. 검증가가 순수함수로 반증:

| qty | band min_qty | unit | **.02 소계** | **.03 소계** | 권위 intent(작업1건 고정) |
|---|---|---|---|---|---|
| 800 | 800 | 52800 | 52800 | 52800 | 52,800 |
| **850** | 800 | 52800 | **56,100** ❌ | **52800** ✅ | **52,800** |
| **899** | 800 | 52800 | **59,334** ❌ | **52800** ✅ | **52,800** |
| 1000 | 1000 | 64000 | 64000 | 64000 | 64,000 |

★ **`.02`는 qty가 band의 min_qty와 정확히 같을 때만 권위값**. 밴드 사이 수량(801~899 등)이면 `per_item = unit ÷ tier_min_qty` 후 `× qty` = 선형 스케일 → **과청구**(돈크리티컬). 명함박이 `.02`로 라이브 작동하는 건 명함이 사이즈 고정·수량이 항상 밴드 경계(1000 등)라 off-band 미발생 — **가변사이즈·자유수량 7상품엔 `.02`가 위험**.

### 2-2. `.03 FLAT`이 band-lookup을 올바로 하는가? — 라이브 코드 확인 (designer 독립 재확인)

검증가 주장의 핵심 = "`.03`은 band 선택은 그대로 하고 곱셈만 안 한다". designer가 `pricing.py`로 독립 재확인:

| 확인 항목 | 라이브 코드 | 결론 |
|---|---|---|
| band 선택(min_qty 티어) | `match_component`(`pricing.py:134`)이 prc_typ과 **무관하게** min_qty 티어 선택 수행(`:173-178` `eligible=[t for t in tiers if t<=qty]→max`) | **`.03`도 min_qty band-lookup 정상**(prc_typ은 곱셈 단계에만 영향) |
| `.03` 소계 계산 | `component_subtotal`(`:193-212`) — `if prc_typ == PRC_TYPE_FLAT: return up, up`(`:203-204`) = **매칭 구간 금액 그대로·×qty 0** | off-band 수량도 flat band total(권위 intent 정확) |
| `.03` 정의 docstring | `:198-199` "고정금액: unit_price(구간 금액) 그대로 — 수량 무관(곱셈 없음). 수량구간은 어느 금액을 쓸지(티어 매칭)만 결정." | **min_qty band 지원 명문화** |
| min_qty NULL 가드 | `.03`은 `tier_min_qty` 미사용(`:203` 곱셈 없음)이라 NULL이어도 ValueError 없음. 단 band 선택엔 min_qty 행 필요 | min_qty=구간하한 NOT NULL 유지(band 선택용)·`.02`의 `:207 base<=0 ValueError`는 `.03`서 무관 |

★ **결론: `.03`은 min_qty band-lookup을 올바로 수행**(match_component가 prc_typ 무관 티어 선택) + 곱셈만 안 함(component_subtotal `.03` 분기). 동판비(이미 `.03`)와 동일 메커니즘. **directive #2 = 정확·검증가 적발 = 유효.**

### 2-3. designer 정정

- **C-3~C-6 박가공비 prc_typ_cd = `.03 PRICE_TYPE_FLAT`** (REV3 `.02`는 off-band 부정확).
- min_qty=수량구간 하한 NOT NULL **유지**(band 선택에 필요·`.03`은 곱셈만 생략).
- 동판비 C-1/C-2도 `.03`(REV3부터 불변). ∴ **박 4 comp(가공비)+동판비 2 comp 전부 `.03 FLAT`** = 일관.

### 2-4. 골든 재검증 (off-band 신규 2건)

기존 골든 G-F1~F8 = 전부 qty가 밴드 경계(1000·10·200·500) → `.03`로도 **동일 총액**(band 선택 같고 곱셈 없음·`.02`==`.03` at band 경계). **8/8 불변.** off-band 사각지대 보강 위해 골든 2건 신규(§golden-cases REV4):

| 신규 골든 | 입력 | `.03` 기대(권위) | `.02`라면(틀림) |
|---|---|---|---|
| **G-F9 off-band** | 소형 일반박 40×80 q850 | **52,800**(E@800 band·flat)+5,000 동판=**57,800** | 56,100+5,000 과청구 |
| **G-F10 off-band** | 대형 일반박 90×90 q1500 | C등급@1000 band=**120,000**(flat)+18,000 동판=**138,000** | 120,000÷1000×1500=180,000 과청구 |

★ G-F9/F10은 `.02`였다면 FAIL·`.03`이면 PASS = 이 교정의 회귀 가드.

## 3. C-3 [directive #3·라이브 조사] — 박 크기 제한 per-product

### 3-1. directive — 박영역 ≤ 상품크기

각 박 상품은 박을 찍을 수 있는 영역이 **상품 크기에 제한**된다(박 면적 ≤ 상품 면적). 이 상한이 ① 소형/대형 시트 배정의 근거가 되고 ② 위젯 손님 박영역 입력의 상한이 된다.

### 3-2. 라이브 실측 (2026-06-30·읽기전용 SELECT)

7상품의 등록 사이즈(t_prd_product_sizes → t_siz_sizes·cut_width/height = 재단 트림사이즈) + product_processes 박색상 + 제약규칙:

| prd_cd | 상품 | 등록 트림(최대 cut W×H mm) | 박색상(product_processes) | 박 필수? | 제약규칙(박 min/max) |
|---|---|---|---|---|---|
| PRD_000027 | 2단접지카드 | **170×220**(170x110 행 cut_height=220 펼침) | 037~044 8종 | **선택(아님)** | 없음 |
| PRD_000029 | 3단접지카드 | **150×100** | 037~044 8종 | 선택 | 없음 |
| PRD_000031 | 프리미엄명함 | **90×55** | 037~044 8종 | 선택 | 없음 |
| PRD_000034 | 펄명함 | **90×50** | 037~044 8종 | 선택 | 없음 |
| PRD_000042 | 프리미엄쿠폰/상품권 | **148×75** | 037~044 8종 | 선택 | 없음 |
| PRD_000069 | 무선책자 | **210×297**(A4) | 037~044 8종 | 선택 | 없음 |
| PRD_000070 | PUR책자 | **210×297**(A4) | 037~044 8종 | 선택 | 없음 |

★ **전 7상품 박=선택(mand_proc_yn=f)·박색상 8종(037~044)·박 면적 제약규칙 0건**(JSONLogic 박 min/max 부재). → ① **proc_cd 게이트 필수 재확인**(REV3 R-FOIL-CDX1·박 선택형이라 미선택 시 0 보장 필요) ② 박색상 enum = 8종(금무광048/은무광049/펄박045/백박046/녹박047 미등록·§0 표 정합).

### 3-3. ★박영역 ≤ 트림 = 상한(실제 박 크기 아님) — 라이브 명함박이 입증

중요: **트림사이즈는 박영역의 이론적 상한일 뿐 실제 박 크기가 아니다.** 박은 보통 로고/텍스트 작은 영역. 라이브 명함박(PRD_000037) 실측이 이를 입증:

```
PRD_000037 트림 = 90×50 (90 > 소형 grid max 80)
COMP_NAMECARD_FOIL_S1_STD use_dims = ["min_qty"]  (siz_width/height 차원 없음·9행 단일등급 E collapse)
```

## 4. C-4 [directive #4] — comp 통합 vs 과분리 재검토 (통합/분리 기준 적용)

### 4-1. 사용자 질문 — 4~6 comp 과분리?

박 설계가 일반박/특수박 × 소형/대형 = 4 박가공비 comp + 동판비 2 = 6 comp. "차원이 같으면 하나로 통합 가능한데 과분리 아닌가?" 통합/분리 기준([[price-component-unify-vs-split-criterion-260630]]·[[dbmap-price-component-grouping]]) 적용:

> **기준**: 한 상품 안 손님선택 축(색상·수량·도수)→ **차원 통합**(같은 comp의 use_dims) / 종류·상품이 다름(제본종류·별색)→ **별도 comp**.

### 4-2. 축 1: 일반박 vs 특수박 통합 가능? — ❌ 분리 유지

| 검토 | 판정 |
|---|---|
| 차원 동일성 | 둘 다 `(proc_cd, siz_width, siz_height, min_qty)` 같은 use_dims. **표면상 통합 후보** |
| 단가 차등 | 같은 (등급×수량)이라도 단가 다름(대형 C등급 1000: 일반 120,000 vs 특수 150,000). **proc_cd가 단가행을 가른다**(색상→단가) |
| 통합 시 위험 | 일반/특수를 한 comp로 합치면 단가행이 `proc_cd→단가`로 자연 분기(손님 박색상 proc_cd = 1행) — **이론상 통합 가능·silent 이중합산 없음**(한 주문=1 박색상=1 proc_cd=1행) |
| ★분리 유지 이유 | 그러나 검증가 flatten으로 **각 색상별 단가행을 어차피 복제**(§1-4·proc_cd 게이트 충전)하면, 일반/특수가 같은 comp 안에서 색상 proc_cd로 자연 분기됨 = **통합해도 무방·분리해도 무방**(돈 영향 동일). **★[결정] 분리 유지** — 이유: ① 권위 가격표가 일반박/특수박을 **별도 블록(B03 vs B05·B04)**으로 정의(실무진 정리 존중) ② 면적격자가 일반/특수 동일하나 **단가표가 별개**라 적재·검수·교정 단위로 분리가 추적성 우월(검증가가 "일반 120,000 vs 특수 150,000" 별개 verbatim 대조) ③ 소형 특수박은 색상 3종(백/홀로/트윙클)만이라 일반(7종)과 색상집합 자체가 다름 → **종류 다름**(기준의 "종류·상품 다름→별도 comp")에 해당 |

★ **판정: 일반박/특수박 = 별도 comp 유지**(통합 가능하나 권위 블록 구조·색상집합 차이·추적성으로 분리가 정합). silent 이중합산 위험은 proc_cd 게이트가 원천 차단(한 주문 1 proc_cd 1행)이라 분리/통합 어느 쪽도 P3-8 안전.

### 4-3. 축 2: 소형 vs 대형 통합 가능? — ❌ 분리 필수(돈크리티컬)

| 검토 | 판정 |
|---|---|
| 면적격자 범위 겹침 | 소형 격자 가로/세로 10·20·40·60·80 ↔ 대형 30·50·70···170. **30·50···가 겹침** |
| ★통합 시 off-grid ceiling 충돌 | 소형 상품의 박영역 80×80 → 한 comp에 소형+대형 셀 공존 시, 엔진 ceiling이 **소형 80셀이 아닌 대형 80~90셀로 잘못 착지** 가능. 예: 소형 40×80(E등급 소형 64,000) vs 대형 50×80 셀(다른 단가). 겹치는 사이즈 영역에서 ceiling이 어느 시트 셀을 고를지 모호 → **틀린 시트 단가 청구**(돈크리티컬) |
| 상품-시트 고정이 충돌 막나? | 한 상품은 한 시트만 사용(명함=소형·접지=대형)하나, **한 comp에 두 시트 셀을 다 담으면** proc_cd가 같은 박색상이라 시트 구분 차원이 없어 ceiling이 양 시트 셀을 다 후보로 봄 → collision |
| 분리 시 안전 | 소형 comp는 소형 격자 셀만·대형 comp는 대형 격자 셀만. 상품이 자기 시트 comp만 바인딩 → ceiling이 그 시트 셀에서만 동작(충돌 0) |

★ **판정: 소형/대형 = 별도 comp 필수**(종류 다름·면적격자 범위 겹침이 통합 시 ceiling 오착지 → 틀린 단가·돈크리티컬). 기준의 "종류 다름→별도 comp" + off-grid ceiling 안전이 둘 다 분리를 요구.

### 4-4. 축 3: 동판비 소형(5000 고정) vs 대형(면적매트릭스) — ❌ 분리(구조 다름)

| 판정 | **별도 comp**(구조·use_dims 다름·기준의 "종류 다름"). 동판비 자체가 박가공비와도 별개(셋업 vs 가공·comp_typ_cd .05 vs .01) |
