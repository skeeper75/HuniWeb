# 아는 것 vs 모르는 것 — 종합 지식맵

> Phase 2 통합 (리더 종합) · 2026-06-18 · huni-price-engine-diag
> 사용자 directive: "결론을 내리기 전에 원리상 아는 것과 모르는 것을 분리." 미지를 결론으로 위장 금지.

## A. 아는 것 (확정 — 코드/스키마 실측 근거)

| # | 확정 사실 | 근거 |
|---|----------|------|
| K-1 | 가격은 단일 알고리즘 `evaluate_price` 하나로 산출. 시뮬레이터·위젯·주문 동일 호출 | pricing.py·price_views.py |
| K-2 | 공식=값 없는 레시피 헤더, 항상 구성요소 **단순 합산**(frm_typ·addtn_yn 미참조) | 코드 + sql/25 |
| K-3 | 모든 가격 차원 책임은 **가격구성요소+단가행**(7,293행). 공식은 묶음만 | 코드 + 라이브 |
| K-4 | 차원매칭 = 비수량 정확매칭(NON_QTY_DIMS·NULL 와일드카드) + 티어(siz_w/h 이하·min_qty 이상) | pricing.py |
| K-5 | 도수 차원 = **print_opt_cd**(1431행)이지 clr_cd 아님(clr_cd=dead 0행) | 라이브 실측 |
| K-6 | 할인 = base 후 수량구간→등급 **순차 후처리**(곱/차감). 가격 구성 아님 | pricing.py:496+ |
| K-7 | 현재 발화 경로 = FORMULA + 수량구간할인만. 템플릿단가·직접단가·등급할인 = 라이브 0행 | 라이브 실측 |
| K-8 | 설계 산출물 prcx01·pricing-erd = STALE(8차원·clr_cd 시절). 라이브가 권위 | 02_code_schema 추적 |

## B. 모르는 것 (미지·추정 — 결론 금지)

| # | 미지 항목 | 왜 모르나 | 누가 답하나 |
|---|----------|----------|------------|
| U-1 | clr_cd→print_opt_cd 도수 폐기가 **의도된 설계 변경인가**(prcx01 LOCKED 문서와 충돌) | 코드는 print_opt_cd, 설계는 clr_cd. 어느 게 최종 의도인지 문서 부재 | **사용자/도메인** |
| U-2 | addtn_yn='N' 행이 라이브에 있나·있으면 그 구성요소는 **과대합산**인가 | 엔진이 안 읽음. 차감 의도 행이 합산되면 가격 오류 | 코드실측+사용자 |
| U-3 | 단가행이 use_dims 선언대로 채워졌나(판별차원 없음·NULL 차원 오염) 전수 | 구성요소별 전수 정합은 본 진단 범위 밖 | **검증 트랙(huni-price-quote)** |
| U-4 | 합가형 구성요소 3종 단가행 min_qty가 NULL/0인가(0이면 ÷min_qty ValueError=계산불가) | 표본만 확인(아크릴 CLEAR3T=1 안전), 전수 미점검 | 검증 트랙 |
| U-5 | proc_grp: 토큰 비대칭 처리가 36 comp에 "판별차원 없음" 오라벨을 실제로 유발하나 | 코드 두 곳 비대칭 발견, 실효과 미측정 | 검증 트랙 |
| U-6 | 수량구간 할인이 출력매수(판수)인가 주문수량인가 기준축 | 직전 N-1b(임포지션 변환 부재)와 연결, 미해소 | **사용자/도메인** |

## C. 컨펌큐 (사용자/도메인 확인 선결 — 이게 안 풀리면 적재 오판)

- **C-1 (U-1)**: 도수는 print_opt_cd로 확정인가? clr_cd는 완전 폐기인가? → 가격구성요소 적재 시 도수 차원을 어디에 넣을지 결정.
- **C-2 (U-2)**: 차감형 비용 항목(addtn_yn='N')이 필요한가? 필요하면 엔진이 안 읽으므로 **개발자 백로그**(코드 수정)이고, 불필요하면 addtn_yn 컬럼 자체가 dead.
- **C-3 (U-6)**: 수량구간 단가행의 수량 기준 = 출력매수 vs 주문수량? (직전 합산형 엽서 N-1b 근본).
