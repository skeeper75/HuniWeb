## 4. 조사신호 (자동 교정 금지·설계가/게이트 보고)

### P-1 [중] 070 PUR A5 내지 단면/양면 — 설계 vs 이전사이트 불일치
- **설계 가정**: 070 A5 내지=단면(component-boundary.csv·booklet-l1 PUR A5 단면 명시).
- **이전사이트 실측**: PUR(pc37) `tmp_p09` 내지인쇄 = **양면만**(단면 옵션 부재).
- **해석**: 권위 엑셀(booklet-l1)이 A5=단면 명시 → **권위가 절대**. 이전사이트가 양면만 노출 = ⓐ 라이브 ASP가 단면 미구현(구 시스템) or ⓑ booklet-l1 PUR 행의 단면이 A4와 행 어긋남 가능. → 설계가 권위 재확인 필요(라이브에 맞춰 권위를 바꾸지 말 것). **돈영향**: 단면↔양면은 내지인쇄비 ~2배 차 → 검증 골든에서 면 매칭 정확성 확인 필수.
- 출처: design D 2.2 표·booklet-l1 PUR A5행·이전사이트 pc37 tmp_p09.

### P-2 [중] 071 트윈링 표지×2 — 이전사이트 폼 미실증
- **설계 가정**: 071 앞표지+뒤표지 별도 → 부모공식 표지인쇄·코팅 ×2 배수.
- **이전사이트 실측**: tmp_p20=맞춤표지\|표지없음 단일 select. **앞뒤 ×2 별 슬롯 없음**.
- **해석**: 도메인상 트윈링 앞뒤표지는 타당(앞=인쇄커버·뒤=백색/투명). 그러나 라이브 폼은 표지를 단일 입력으로 받음 → ×2는 **가격 산정 내부 처리**거나 뒤표지=투명커버(tmp_p06)로 대체 가능성. ×2를 부모공식에 강제하면 **단일표지 책자에 ~50% 과대청구 위험**. → 설계가: 071 표지×2는 "도메인 가설"로 두고, 게이트에서 골든(표지1 vs ×2) 양쪽 계산해 권위 가격표 대조로 확정. **자동 ×2 적용 보류**.
- 출처: design A10·§3.1 071 ×2·이전사이트 pc38 tmp_p20.

### P-3 [저] 이전사이트 폼에 내지 페이지 `<select>` 미노출
- **이전사이트 실측**: 4종 모두 `tmp_p01`(사이즈)·`tmp_p07`(용지)·`tmp_p09`(인쇄)는 있으나 **내지 페이지수 select 없음**(페이지/수량은 폼 하단 수량입력 필드로 추정·gstack select 스캔 범위 밖).
- **해석**: 권위 엑셀 booklet-l1에 페이지 min/max/incr 명시(절대 권위) → 이전사이트 페이지 입력 UI 미노출은 거울 한계(페이지는 텍스트 입력일 수 있음·select 아님). **설계의 페이지범위(068=4/28/4 등)는 권위 엑셀이 권위·라이브로 무효화 불가**. 조사신호 등급 저(권위로 마감).
- 출처: 이전사이트 select 스캔(페이지 select 부재)·booklet-l1 페이지컬럼.
