## C-1 — 볼체인 / 리필잉크

### ① 경쟁사 실제 처리

**WowPress (외부 경쟁사 · 로컬 캡처 권위)**
- WowPress는 **아크릴 키링·만년스탬프를 카탈로그에 취급하지 않음** → 볼체인·리필잉크 직접 대응 상품 **없음** `[근거부족: 직접 동일상품]`.
- 그러나 **결정적 간접 근거**: WowPress는 **모든 부착·연결·소모 부속을 `부자재` 카테고리의 독립 상품(별도 SKU)으로 판매**한다. 카탈로그 스캔 결과 `부자재` 카테고리 **39개 상품** 전수가 독립 prdname/slug/category를 갖는다:
  - 예: `40080 큐방(4cm)`, `40111 로프(3Mx4개)`, `40339 OPP봉투`, `40206 조각자석`, `40113 양모헤라` 등 — **본체에 끼우거나 동봉되는 소형 부속도 본체 옵션이 아니라 별도 상품**.
  - 근거: `docs/wowpress/catalog/products/{40080,40111,40339,40206,40113}.json` (각 `categoryPath=['부자재']`, 독립 `productId`).
- → WowPress 모델에서 "본체에 붙는/동봉되는 부속"의 기본 처리 = **별도 상품(=후니 TEMPLATE)**. 본체 옵션 그릇에 넣지 않는다.

**RedPrinting (프로젝트 소유자 본인 시스템 · 라이브)**
- 상품 코드 확인: 아크릴 키링 `AC/ACTHLKY`(렌티큘러), 만년스탬프 `GS/GSSMSTP`(자동/만년 스탬프). 근거: RedPrinting 랜딩 `/ko/landing/lifestyle/keyring`·`/ko/landing/stamps/self-inking-stamps` (WebFetch 2026-06-09).
- **옵션 구조 직접 캡처는 미확보** — 랜딩 페이지가 JS 위젯으로 옵션을 로드해 정적 fetch로는 볼체인/리필잉크의 옵션-vs-추가상품 노출 형태를 단정 불가 `[근거부족: RP 볼체인/리필 옵션 캡처]`.
- 보유 캡처(`s3_rp_GS*` 굿즈 8종, `s6_cal_*` 캘린더 5종)에는 볼체인·리필잉크 항목 **부재**(grep 무결과). → RP 측 직접 verdict는 보류, WowPress·후니 엑셀로 대체.

### ② 인쇄 도메인 관행
- 굿즈/아크릴 업계에서 **볼체인·O링·키링고리·낫셈 등 연결 부속**은 통상 두 형태로 처리:
  1. **무료 기본 동봉 부속**(키링 1개당 체인 1개 자동 동봉) — 이 경우 색상 선택만 옵션화 → **set(.07) 성격**.
  2. **유료 별도 부자재 상품**(체인만 추가 구매·수량 별도) — **별도 SKU/장바구니 라인** → **TEMPLATE 성격**.
- **리필잉크(만년스탬프 5cc)**는 거의 예외 없이 **소모품 별도 상품**으로 판매(스탬프 사용 후 보충용, 본체와 수명·수량이 분리됨). 본체 옵션으로 묶는 관행은 드물다.
- `(선택안함)` 항목의 의미: "이 부속을 함께 주문할지 말지"를 **택1 토글**로 노출하는 UI 관용 — 즉 **부속 자체는 별도 상품이되, 본체 주문 화면에서 함께 담을지 선택**하게 하는 형태. 부속이 본체 제작방식(OPTION)임을 뜻하지 않는다.

### ③ 권고 — **TEMPLATE** (볼체인·리필잉크 모두 별도 추가상품 SKU)
**근거(강→약):**
1. **[최강·후니 자체 사실]** 후니가 **리필잉크를 독립 상품 `PRD_000015`(부자재)로 이미 등록**했고, 만년스탬프 본체는 `PRD_000217`로 별개. 근거 `huni-goods-option-mapping.md §2.2`("리필잉크(PRD_000015)는 **별도 상품(부자재)**"). → 본체와 분리된 자기 prd_cd·재고를 가지므로 정의상 별도 주문라인 = TEMPLATE.
2. **[강·엑셀 권위]** 볼체인·리필잉크가 후니 L1에서 **`추가상품(옵션)_추가상품` 컬럼**에 들어있음(acrylic·goods-pouch 시트). guide 권위: "`추가상품(옵션)` 컬럼=TEMPLATE(값이 다른 상품)". 근거 `all-sheets-otc-extract.md §8·§9`(볼체인 9색·리필잉크 7색).
3. **[보강·경쟁사]** WowPress가 동류 부착/소모 부속을 전부 `부자재` 독립 SKU(39종)로 판매. 본체 옵션 그릇 미사용.
4. **[도메인]** 리필잉크=소모품 별도판매가 업계 표준.

**`(선택안함)` sentinel 처리(중요):** `볼체인(선택안함)`은 볼체인을 **OPTION으로 재분류해야 한다는 근거가 아니다.** 이는 "본체 주문 시 이 추가상품을 함께 담을지 선택"하는 UI 노출제어. 후니 모델에서는 **TEMPLATE ADDON을 본체 옵션 그룹의 enabled 조건(또는 선택형 add-on)으로 노출**하고, "선택안함"=add-on 미선택(template_selection 0건)으로 처리. 즉 **데이터 분류는 TEMPLATE, UX 노출은 본체 화면의 택1**.

### ④ 후니 CPQ 적용 (1줄)
볼체인(색 9종)·리필잉크(색 7종)는 각각 `t_prd_templates`(base 부자재 prd_cd: 리필잉크=`PRD_000015`, 볼체인=base 체인 prd_cd `[CONFIRM 라이브]`) + `t_prd_template_selections`(색=selection)로 적재하고, `(선택안함)`은 본체 화면에서 add-on 미선택(template 0건)으로 노출한다.

## C-4 — 캘린더 우드거치대 (wood stand/holder)

### ① 경쟁사 실제 처리
**WowPress (외부 경쟁사 · 로컬 캡처 권위) — 결정적**
- WowPress는 **거치대 일체형 캘린더와 거치대 단독판매를 별개 상품으로 분리**한다:
  - `40210 우드 스탠드 캘린더` — `categoryPath=['캘린더']`, 즉 **거치대가 캘린더 상품 정의에 내장된 완성 상품**. `additionalOptions=[]`(거치대를 옵션으로 노출하지 않음 — 상품 자체가 "스탠드형 캘린더"). 근거 `docs/wowpress/catalog/products/40210.json`.
  - `40211 우드스탠드` — `categoryPath=['부자재']`, **거치대만 단독 판매하는 독립 상품 SKU**. 근거 `docs/wowpress/catalog/products/40211.json` (`meta.name="우드스탠드"`, slug `우드스탠드-40211`).
  - 보강: `40621 우드 행잉 캘린더`(`캘린더`)도 거치/행잉 방식이 상품명에 내장. 거치대류 단독판매는 `부자재`에 `40078 LED라이트배너 거치대`·`40119 미니배너 거치대`·`40121 실내용 배너거치대`·`40577 등신대 종이거치대` 등 다수.
- → **WowPress 모델 = 거치 방식이 본체 제작과 묶이면 "스탠드형 ○○"라는 별도 상품(가공 일체), 거치대를 따로 사면 `부자재` 독립 SKU(추가상품).** 한 상품 안에서 "거치대 옵션"으로 토글하지 않는다.

**RedPrinting (소유자 본인 시스템 · 라이브)**
- 탁상캘린더 코드 `TP/TPCLSTD`(디자인 탁상 캘린더). 근거 RP 랜딩 `/ko/landing/calendars/design-calendar-templates` (WebFetch 2026-06-09).
- 보유 캘린더 캡처 5종(`s6_cal_{GSCLMGN,HLCLSTD,HLCLWAL,TPCLECO,TPCLWLB}.json`)에 **거치대/스탠드/우드 항목 부재**(grep 무결과). 그중 `TPCLECO`(에코 탁상캘린더)의 가공 그룹은 `재단·링제본(상철)·포장`뿐 — **거치대는 옵션에 없음**. 근거 `s6_cal_TPCLECO.json` PCS_INFO.
- → RP 탁상캘린더는 **삼각대/거치 구조가 상품 자체(탁상형)에 내장**되어 별도 옵션화하지 않음. 다만 우드거치대 단독 추가상품 캡처는 미확보 `[근거부족: RP 우드거치대 추가상품]`.

### ② 인쇄 도메인 관행
- 탁상캘린더의 **삼각대/스탠드 구조**는 통상 **상품 본체에 내장**(탁상형=세워지는 형태가 곧 상품). 별도 옵션화하지 않음.
- **우드거치대(원목 받침)**처럼 본체와 분리된 별도 부재는 두 형태로 갈림:
  1. **캘린더 제작 시 함께 가공·동봉**(우드받침 끼워 완성·함께 배송) → 본체 제작 일체 = **OPTION**(또는 동봉 set).
  2. **거치대만 별도 구매**(캘린더 없이도 팔리는 독립 원목 받침) → **별도 SKU** = **TEMPLATE**.
- 후니 시트가 `우드거치대`를 **캘린더가공 컬럼과 추가상품 컬럼 양쪽에 둔 것**은 위 두 형태가 **실제로 둘 다 존재**함을 뜻한다(WowPress 40210 가공일체 vs 40211 단독판매와 동형).

### ③ 권고 — **분기 (엑셀 컬럼이 의미를 가른다)**
- **캘린더가공 그룹 내 `우드거치대` = OPTION**(본체 제작 일체·함께 배송되는 거치 가공·BUNDLE). 근거: `캘린더가공_캘린더가공` 컬럼은 제본/거치 방식 택1 그룹(`all-sheets-otc-extract.md §5 GRP-CAL-가공`·"가공없음/우드거치대/1구타공+끈/고리형트윈링제본/2구타공+끈/제본없음").
- **추가상품 그룹 내 `우드거치대` = TEMPLATE**(별도 거치대 SKU·별도 주문라인). 근거: `추가상품_추가상품` 컬럼은 TEMPLATE 신호(`§6` 우드거치대=별도 거치 상품).
- **판별 근거(강→약):**
  1. **[강·경쟁사]** WowPress가 동일 분기를 상품 분리로 구현(40210 가공일체 vs 40211 부자재 단독). → 후니가 둘 다 둔 것은 의미 충돌이 아니라 **두 별개 케이스**.
  2. **[강·엑셀 권위]** 후니 자체가 가공 컬럼(=OPTION 신호)과 추가상품 컬럼(=TEMPLATE 신호)에 **각각** 배치 → guide의 컬럼 신호 그대로 따름.
  3. **[보강·도메인]** 가공 동봉 거치 vs 거치대 단독판매가 굿즈 업계에 공존.
- **확정 필요 잔여 `[CONFIRM 라이브]`:** 후니 실데이터에서 가공 컬럼의 `우드거치대`와 추가상품 컬럼의 `우드거치대`가 **물리적으로 같은 거치대(중복 노출)인지, 다른 거치 방식인지**. 같은 것이면 추가상품 쪽만 TEMPLATE로 두고 가공 쪽 중복 제거; 다른 것이면 둘 다 유지(가공=일체형, 추가상품=별매). 라이브 admin product-viewer 캘린더 상품의 가공·추가상품 탭 대조로 확정.

### ④ 후니 CPQ 적용 (1줄)
가공 그룹의 `우드거치대`는 `option_groups→options→option_items`(.04 process BUNDLE)로, 추가상품 그룹의 `우드거치대`는 `t_prd_templates`(base 우드거치대 prd_cd `[CONFIRM]`) ADDON으로 적재하고, 라이브 대조로 중복 여부만 해소한다.
