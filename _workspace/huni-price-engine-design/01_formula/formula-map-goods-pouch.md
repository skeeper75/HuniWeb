## 0. ★결론 먼저 — 굿즈/파우치는 가장 단순한 가격산식 + 가장 미완성인 라이브 (계산방식 단순·완성도 최저)

| 추정(과업 입력·dbmap 메모리) | 라이브·권위 실측 결론 |
|----------------|----------------------|
| "굿즈/파우치 = 가장 복잡한 상품군(혼재·면적/구간 혼합 가능성)" | **부분 거짓.** 가격 **계산방식 자체는 굿즈 전체 단일 유형**(`[고정가형: 굿즈/파우치]` calc-draft row 122~123)으로 **가장 단순**(면적매트릭스 없음·원자합산 없음·출력매수 곱셈사슬 없음). **복잡함은 "계산"이 아니라 "데이터 구조"**(86상품·303 variant행·19 상품군·혼합 인쇄방식 7종·변형단가 31상품·구간할인 4타입). |
| "면적/구간 혼재" | **면적매트릭스형 0건.** calc-draft가 굿즈 전체를 `고정가형` 한 유형으로 규정. 거울·코스터·머그·파우치·에코백·폰케이스·말랑 전부 고정가. (실사/아크릴의 면적과 결정적 차이.) |
| "size→option 재분류가 가격에 영향" | **참 + 결정적 함의.** 사이즈가 옵션으로 갔지만 **각 옵션 variant 행은 자기 고정가를 가짐**(S=5000/M=5500/L=6000). 그런데 라이브 `PRODUCT_PRICE`는 prd_cd당 unit_price 1개뿐 → **변형단가를 담을 그릇이 없음**(§6 핵심 결함). |

**계산공식집초안(절대 권위) 실측** (`24_master-extract-260610/calc-formula-draft-l1.csv` row 122~123):
```

판매가 = 고정가 + 수량별구간할인 타입 + 추가상품      (비고: "상품마스터 굿즈/파우치(구간할인 타입입력)")
```
- ★문구(row 119~120)와 **거의 동형이나 한 단어 차이**: 문구=`수량별구간할인`(단일 테이블), 굿즈=`수량별구간할인 **타입**`(구간할인 **타입입력**=A타입/B타입/말랑/파우치에코백 중 상품별 선택). 라이브 구간할인 테이블 4종 실재(DSC_GOODSA_QTY·DSC_GOODSB_QTY·DSC_SQUISHY_QTY·DSC_FABRIC_QTY).

**라이브 가격사슬 완성도 = 5종단 중 최저 (총 전무에 가까움):** PRD_000183~280 **98 활성상품 중 — 고정가(product_prices) 0행·공식 바인딩 0·CPQ option_items 0.** 단 **수량구간할인 바인딩 82행 + 자재 BOM 76상품/164링크는 실재**(round-1·round-22 적재분). 즉 **할인·자재 절반은 배선됐으나 가격의 본체(고정가)가 통째로 비어 있다.** `확신도: 높음(라이브 직접 카운트)`

## 2. 계산방식 분류 (굿즈/파우치 = 1 유형·4 변형 서브클래스)

★전 상품 단일 계산방식. 단 **"고정가"가 단일이냐 변형이냐**로 두 서브클래스(가격 그릇 설계가 달라짐):

|-----------|------|:--:|------|----------------|
| **GP-1 단일고정가** | prd_cd당 고정가 1개(변형 없음·variant 행이 가격동가) | **55** (41 single-row + 14 multi-row 동가) | 카드거울 2500·코르크코스터 3000·레더여권케이스 5000 | `t_prd_product_prices` unit_price 1행 = `PRODUCT_PRICE` 경로 그대로 OK |

## 3. 3층 지도 — (GP-1) 단일고정가 (완제품·55상품)

> calc-draft row 123: `판매가 = 고정가 + 수량별구간할인 타입 + 추가상품` · 라이브 엔진 경로 = `PRODUCT_PRICE`(pricing.py:312-317 `unit_price × qty`) → `_quantity_discount`(pricing.py:478-491).

### 3-A. 거울/코스터/단품 굿즈 (소품·라이프·기념품)
```
상품 [완제품]: 카드거울(PRD_000185·2500)·틴거울(183·3000)·컴팩트거울(184·3600)·
              코르크코스터(189·3000)·우드코스터(190·7000·원/사각 동가)·규조토코스터(192·5000)·
              레더여권케이스(196·5000)·만년스탬프(217·고정가)·핀버튼(200) 등
  ├─ 고정가 base   = 가격 C열 inline (상품별)              ★price source=상품마스터 / 엔진=PRODUCT_PRICE
  ├─ (구성·BOM)    본체 소재 = t_prd_product_materials(round-22 적재분·레더/캔버스/우드/규조토 등)

  └─ (추가상품)    만년스탬프=잉크 5cc 7색(추가상품 addon·2500원·§5)
```
계산방식: **고정가형 × 수량 − 구간할인(타입)** · 라이브 baseline: **product_prices 0행·바인딩 0·구간할인 바인딩 실재**(틴거울→DSC_GOODSB_QTY 등 82링크) `확신도: 높음(고정가값 명시·구간 바인딩 실측)·라이브 고정가 전무`

### 3-B. 파우치/에코백/필통 (레더·캔버스·타이벡·메쉬·린넨)
```
상품 [완제품]: 레더 플랫 파우치(230)~타이벡 보냉에코백(277)·메쉬에코백(279) 등 ~50상품
  ├─ 고정가 base   = 가격 C열 inline (대부분 단일고정가)
  ├─ (구성·BOM)    본체원단(레더/캔버스/타이벡/메쉬/린넨/광목) = t_prd_product_materials (round-22 41상품 COMMIT)
  ├─ 가공(옵션)    라벨없음(0)/라벨부착(+300) = 캔버스 파우치/필통/에코백 (§4 후가공가산)
  └─ 수량구간할인  = DSC_FABRIC_QTY(파우치/에코백 50링크 — 카테고리 단위 바인딩·[[dbmap-discount-authority]])
```
계산방식: 고정가형 × 수량 + 가공가산 − 구간할인 · baseline: 고정가 0·가공옵션 0·구간 바인딩 50 `확신도: 높음`
- ★구간할인 바인딩은 **카테고리(파우치/에코백) 단위** — 개별 상품마다 행이 아니라 family 일괄(round-1 권위·[[dbmap-discount-authority]]).

## 4. 3층 지도 — (GP-2) 변형고정가 (옵션 variant별 다른 고정가·31상품·★핵심 설계난제)

계산방식: **변형고정가 × 수량 − 구간** · ★엔진 그릇 부재 `확신도: 높음(변형단가 31상품 직접 카운트)`

### 4-B. 용량/온스 변형 (머그/보틀)

### 4-D. 폰케이스 기종 변형 (★Sheet-only·라이브 미등록)
```
상품 [완제품·variant·라이브 부재]: 슬림하드/블랙젤리/임팩트젤하드/에어팟케이스★/버즈케이스★ (ID 14715)

  ├─ 가공(옵션)    맥세이프추가(+6500·임팩트젤하드)
```

## 9. 핵심 메모

1. **계산방식 단일·단순** — 굿즈/파우치 전 86상품 = `[고정가형: 굿즈/파우치] = 고정가 + 구간할인 타입 + 추가상품`. 면적매트릭스·원자합산·세트조합·출력매수 곱셈 전부 없음. 엔진 경로 = `PRODUCT_PRICE`(unit_price×qty) + `_quantity_discount`.
2. **★변형고정가(GP-2·31상품)가 최대 설계난제** — S/M/L·온스·ml·면별 다른 고정가가 `가격` C열에 행별 실재하나, 라이브 PRODUCT_PRICE(unit_price 1개)·option_items(add_price 없음) 어디에도 담을 그릇 부재. designer 그릇 결정 선결(§6-1·Q-GP-1).
3. **가격 소스 = 상품마스터 inline 고정가** — "가격포함" 시트(C열). 인쇄상품 가격표 260527 매트릭스 미사용(굿즈=리지드 단품).
4. **size→option 재분류의 가격 함정** — 사이즈 대부분이 옵션(폰기종·M/L/XL·면·구수). 가격은 그 옵션 variant 행에 붙어있음. **기계적 단일고정가 평탄화 금지**(오청구·round-10 권위).
5. **할인·자재 절반 적재** — 구간할인 바인딩 82(A타입15·B타입11·말랑·파우치에코백 50 카테고리단위)·자재 BOM 76상품/164링크·할인테이블 4종 실재. 가격본체(고정가)만 0.
6. **색상 variant = 가격 비기여** — 선택_가격 전부 0(동가). 색상은 본체색=재질행 합성(생산 BOM)·UI 옵션표시만. 가공가산(라벨+300·맥세이프+6500)·추가상품(잉크+2500)만 가격 가산.
7. **폰케이스 기종 = Sheet-only·라이브 미등록** — 슬림하드/블랙젤리/임팩트/에어팟/버즈 상품 자체 라이브 부재(밴드톡·폰스트랩만). 상품 등록이 가격 적재보다 선행(round-24 Sheet-only 18).
8. **돈크리티컬 자체는 안전** — 고정가형은 ×qty 폭발·silent 이중합산 구조적 부재(comp 합산 없음). 유일 위험 = GP-2 평탄화 오청구(설계 선결) + (b)formula 채택 시 신규 component_prices min_qty=1 가드.
