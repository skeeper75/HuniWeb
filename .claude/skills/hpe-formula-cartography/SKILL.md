# hpe-formula-cartography — 가격공식·구성요소 지도 작성 방법론

가격계산 엔진을 설계하기 전, "후니가 가격을 어떻게 계산하려는가"의 전체 지도를 그린다. 설계의 자(尺)다.

## 왜 지도가 먼저인가

가격 장치 역할을 정확히 정의하지 않으면 잘못 설계된다(직전 검증에서 검증자조차 도수축 오해·SIZ 오선택). 설계가가 빈 캔버스에서 짜면 권위를 놓친다 — 먼저 상품마스터·가격표·역공학·라이브를 충실히 지도화해야 설계가 권위에 정합한다.

## 4대 원천 권위 순서 [HARD]

1. **상품마스터(260610)** — 가격계산공식 컬럼·상품별 자재/공정/사이즈/도수. 절대 권위.
2. **인쇄상품 가격표(260527)** — 차원·단가 매트릭스. 절대 권위. 둘이 어긋나면 결함(컨펌큐). v03/STALE 금지.
3. **라이브 t_prc_* + evaluate_price** — 현 적재 현황(설계 정답 아님·현 상태 기준선). 미설계/불완전 갭 포착용.
4. **역공학** — widget_monitor PRICE 계약·docs/reversing·rpmeta 메타모델(경쟁사 구성요소 축 후보·갭헌팅).

## 3층 지도 (상품군마다)

```
상품 (완제품 | 반제품=세트조합)
  └─ 가격구성요소 인벤토리 [자재 mat_cd · 공정 proc_cd · 사이즈 siz · 도수 print_opt_cd · 옵션 · 수량]
       각 요소: 차원(use_dims 후보) · 단가 소스(가격표 셀)
         └─ 계산방식 [원자합산 | 면적매트릭스(siz_width·siz_height) | 수량구간 | 고정가 | 세트조합]
```

## 완제품 vs 반제품(세트) [HARD]
- **완제품** — 단일 상품의 자기 공식.
- **반제품(세트상품)** — 여러 본체/구성품 조합. 구성품 각각의 완제품 공식 + 조합 레이어(합·번들). 세트 식별이 핵심(designer의 세트 모델 입력).

## 토큰 효율
- 엑셀은 1회 추출→CSV 캐시(`dbm-excel-parse`·기존 `24_master-extract-260610` 재사용). 반복 Read 금지.
- 라이브는 읽기전용 SELECT 최소(`dbm-schema-extract`). 기존 하네스 산출(§7/§13/§16) 우선 인용.
