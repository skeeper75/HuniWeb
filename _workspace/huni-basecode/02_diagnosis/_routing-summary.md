## 6. ★ del_yn / use_yn 권위 보정 (보완 진단 2026-06-18 — B5 blind-spot)

> 전체 보드가 use_yn만 추적한 blind-spot 적발 → **삭제/정리 라우팅의 권위 플래그 = del_yn**으로 정정. 상세: `diagnosis-material.md §del_yn`·`diagnosis-category.md §del_yn`.

| 사실(소스+라이브) | 라우팅 영향 |
|-------------------|-------------|
| **del_yn = 소프트삭제 권위**(조회/BOM/가격 선택지 차단 = `del_yn='N'` 필터)·**use_yn = 부차 토글**(조회 미차단) | 모든 "삭제/정리" 라우팅을 **use_yn='N' → del_yn='Y'(+del_dt)** 로 교체 |
| 자재 del_yn='Y' 143행 전건 use_yn='Y'(직교 미정리). .09 69행 100% del_yn='Y'+BOM link 113 활성 | del_yn='Y'는 **신규 UI 숨김까지만** — 진짜 미완 = **BOM link 제거(GPM-4)**. del_yn은 load_master 재적재 시 소멸(경로 Y 근본) |
| 카테고리도 del_yn 보유. 고아 14노드 중 진짜 소프트삭제 = CAT_000297 1노드뿐. 나머지 13(294 포함)=del_yn='N' 미삭제(round-22는 use_yn만 토글) | 카테고리 고아 정리 = **del_yn='Y'** 13노드(294 포함). 302/304는 BLOCKED 보존(del_yn='N' 노출 중) |
| 멱등 가드 | **`WHERE del_yn='N'`**(이미 'Y' skip) + **del_yn/use_yn 둘 다 추적**(단독 금지·blind-spot 재발 방지) |
