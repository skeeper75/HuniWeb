## 3. 셀 유형 분류 = 전부 "밴드 단가표(수량구간×옵션열)"

- 매트릭스이지만 한 축이 **수량구간(min_qty)**, 다른 축이 **제본방식(이산 카테고리)**. 면적매트릭스(size×size)와 다름.
- 한 셀 = 단일 스칼라 단가(복합셀 아님). **복합셀 분해 불필요** — 셀 자체는 1차원 값.
- 단, **제본방식 축을 어느 라이브 차원에 담느냐**가 분해의 핵심(아래 §4·decomposition §2).

## 4. 제본방식 축 → 라이브 차원 매핑 판정 (핵심)

핸드오프 가설: 제본 = `t_proc_processes` PROC_000017 자식 공정 → 제본방식 = **proc_cd 차원**?

### 4.1 PROC_000017 자식 실측 (라이브 권위)

```
PROC_000017 제본 (부모)
 ├ PROC_000018 중철제본      ├ PROC_000022 떡제본
 ├ PROC_000019 무선제본      ├ PROC_000023 하드커버무선제본
 ├ PROC_000020 PUR제본       ├ PROC_000024 하드커버트윈링제본
 ├ PROC_000021 트윈링제본    └ PROC_000025 레이플랫제본
```

- 시트 제본방식 11종 중 **8종은 PROC_000017 자식으로 실재**(중철·무선·PUR·트윈링·하드커버무선·하드커버트윈링).
- **3종은 PROC_000017 자식 부재**: 싸바리바인더, 캘린더제본 4종(벽걸이/탁상220/탁상130/탁상미니). → proc_cd 직매핑 불가(BLOCKED 후보였음).

### 4.2 라이브 실제 적재 방식 = comp_cd 분리 (proc_cd 미사용)

라이브 `t_prc_component_prices` 실측 결과 — 제본방식은 **proc_cd가 아니라 구성요소(comp_cd)로 분리**되어 적재됨:

|--------------|---------------|-----------|----------|
| 중철제본 | COMP_BIND_JUNGCHEOL | NULL | `["min_qty"]` |
| 무선제본 | COMP_BIND_MUSEON | NULL | `["min_qty"]` |
| PUR제본 | COMP_BIND_PUR | NULL | `["min_qty"]` |
| 트윈링제본 | COMP_BIND_TWINRING | NULL | `["min_qty"]` |

- **세 번째 패턴 확정**: 스티커=opt_cd/coat_side / 박=opt_cd(등급) / **제본=comp_cd 분리**. 제본방식이 PROC_000017 공정 트리에 있음에도 가격엔진은 component 분리로 표현. 의미는 후가공 공정이지만 가격 차원은 comp_cd.
- **장점**: 싸바리·캘린더처럼 PROC 트리에 없는 방식도 comp만 만들면 됨 → BLOCKED 없음. proc_cd NULL 강제 회피.

## 5. 단가/합가 판정 = 단가형(.01)

- B1 중철: 1매 3000 → 1000매 500. **수량↑ 단가↓**. 셀 값 = 권당(1권 제본) 장당 단가.
- 라이브 실측: 11 component 전건 `prc_typ_cd = PRICE_TYPE.01`(단가형), `use_dims=["min_qty"]`. → 엔진 = `해당 수량구간 장당가 × 주문수량(권수)`.
- **합가형 아님**: 구간총액 표기(예 "100권 20,000원") 없음. 헤더 `제본/수량` = 권당 단가표. 단가형 정당.

## 6. 가격경계 (decomposition에 명시)

- **B2 `표지비용 따로 계산`**: 하드커버 제본은 표지(하드보드+감싸기)가 별도 상품(PRD_000073 하드커버책자-표지 등). 이 시트는 **제본 후가공비만**. 표지비는 다른 시트/상품 가격.
- **B3 `삼각대 포함`**: 캘린더 제본비에 삼각대(받침) 부속 포함 → 별도 부속비 없음. 가격 완결 경계.
