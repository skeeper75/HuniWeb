# codex 연결검증 프롬프트 (디지털인쇄·gpt-5.5)

> codex exec -s read-only 로 투입. stdin: 레시피 + _live-facts.md + _authority-context.md(요약) + 본 지시.
> codex 산출 본문 = connection-audit-디지털인쇄.md 의 §codex 섹션.

## 역할
너는 후니프린팅 디지털인쇄 시트 31상품의 **구성요소 연결·가격공식↔가격구성요소 설계 정합** 독립 검증가다.
입력 = (A)레시피 작업지시서 (B)라이브 t_* 실측 근거(_live-facts.md, 절대 사실) (C)권위 가격표 차원.
라이브 접속 없음 — _live-facts.md 수치만 사실로 취급. 추가 추론은 [가설] 배지.

## 검증 2축
1. **구성요소 연결**: 각 상품의 자재(mat_cd)·공정(proc_cd)·옵션·사이즈(siz_cd)·도수(colrcnt/print_opt)가 올바른 t_*에 연결되고, 가격공식이 바인딩됐는가.
2. **가격공식↔가격구성요소 설계**: 바인딩된 PRF_*가 묶는 comp가 ① 그 상품군 가격표 차원경계 안인가(시트밖 silent 합산) ② use_dims↔단가행 충전 차원 일치 ③ 권위 가격표 차원(도수·면·사이즈·수량)대로 설계됐는가.

## 적발 분류
- **누락(견적 결과값 안 나옴)**: 미바인딩·단가행 0·필수 comp 미배선·고아 → no-match/0원.
- **오연결**: 축 의미충돌(print_opt 면/도수)·이중배선(같은 차원 두 comp 합산)·차원 미스매치(use_dims↔단가행 불일치)·잘못된 t_* 귀속.

## 출력 형식 (Markdown, 한국어)
1. **연결검증 요약표**: 31상품 × {바인딩·핵심결함·정상/오연결/누락 판정}.
2. **결함 상세**: 각 결함에 {결함ID·상품/공식·comp_cd·차원·원인유형·라우팅(우리트랙 데이터 / 개발자 엔진imposition)·확신도}.
3. **설계 적정성 판정**: PRF_DGP_A~F·FIXED·FOLD_SUM·ENV 각 공식의 차원경계·use_dims↔단가행 정합 평가.
4. 라이브 미확인 주장은 [가설] 배지. _live-facts.md로 확인된 건 [라이브-확정].

## 시드 (재유도 말고 검증 확장)
