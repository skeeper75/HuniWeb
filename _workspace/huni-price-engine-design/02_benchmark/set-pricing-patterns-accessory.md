# 반제품/세트(묶음) 가격 합성 패턴 — 상품악세사리

> `hpe-benchmark-analyst` 산출 3/3 — 악세사리류의 **묶음/세트/부속물 조합 가격 합성**을 경쟁사가 어떻게 다루는지 정리해 후니 세트상품 가격 설계 입력으로 제공.
> **흡수 vs 답습[HARD]**: 메커니즘만 흡수·naming/codes 후니 유입 금지·권위 엑셀 최종.
> 악세사리는 디지털인쇄(책자=다부품 합성)와 달리 **세트 합성보다 "묶음 단위 단가"와 "부속물 귀속"이 핵심** — 본체 인쇄물에 부속물(우드행거·고리)을 더하는 조합이 진짜 세트 패턴이다.

## 0. 악세사리 묶음/세트 가격 합성의 4가지 패턴

| 패턴 | 정의 | 경쟁사 관측 | 후니 그릇 |
|------|------|-----------|----------|
| **A-1 묶음수 단가(팩/세트/입)** | 한 SKU를 묶음 단위(50장/3개1팩/2개1세트/100개입)로 단가 적용 | 와우 papername 융합·ordqty tier·레드 팩 | `bdl_qty` + `t_prd_product_bundle_qtys`(`bdl_unit_typ_cd` FK QTY_UNIT) |
| **A-2 같은규격×묶음수 비례+볼륨할인** | 같은 규격에서 묶음수만 올려 비례 단가·대량 할인 | 와우 ordqty tier 단가하향·후니 트래싱지 20/40/100장 | `bdl_qty` 차원 단가행 + `t_dsc_*` |
| **A-3 부속물 귀속(본체+addon)** | 본체 상품(포스터/캘린더)에 부속물(우드행거·고리·끈) 결합 | 레드 부속물#8 `addons.tmpl_cd` 다중귀속·PD 거치대 | `t_prd_product_addons`(+ template) — 본체 가격 + addon 개당단가 Σ |
| **A-4 부속물 = 자재+공정 BUNDLE** | 한 부속물이 자재(우드/면끈)+공정(부착/봉제) 두 의미 | 레드 지퍼=FLX_ZIP(자재+공정)·끈=부착공정 `[red:GS §7·8]` | `t_prd_product_materials`(usage)+`t_prd_product_processes` BUNDLE |

★악세사리는 책자형 **다부품 합성(P-1)** 이 드물다 — 봉투/케이스/고리는 단일 완제품이라 부품을 합산하지 않는다(개당 고정단가·C-AC1). 진짜 "세트"는 **본체 인쇄물 + 부속물(A-3)** 또는 **묶음 팩(A-1)** 이다.

## A-1. 묶음수 단가 (팩/세트/입) ★

### 경쟁사 패턴
- **와우프레스** `[wow:acc]`: 거울버튼 `유포지(유광) 개별포장있음`(포장유무가 papername 융합)·케이스 "10개" 단위·뱃지 unit=개·ordqty discrete tier. 묶음/포장이 가격 단위.
- **레드프린팅**: 부속물 팩 단위(볼체인 3개1팩 동류).
- **후니** `[huni:acc]`: OPP봉투 `(50장)`·트래싱지 카드봉투 `(20장/40장/100장)`·볼체인 `(3개1팩)`·천정고리 `(2개1세트)`·자석고무판 `(20개입)`·행택끈 `(100개)`·만년잉크 `(5cc)`. **묶음 단위가 가격 단위**(개당이 아니라 1팩당 단가).

### 합성식 (관측)
```
악세사리 가격 = 묶음 단가(50장 1팩당)        # bdl_qty 단위 고정단가
            × 주문 묶음수(팩 수)            # 수량 = 팩 수·개수 아님
```

### 후니 매핑 (그릇 보유 — 단가 단위 명시 선결)
후니 `t_prd_product_bundle_qtys`(`bdl_qty`+`bdl_unit_typ_cd` FK QTY_UNIT: 장/개/팩/세트/입) — 11 악세사리 상품 보유(`[huni:live]`). `component_prices.bdl_qty` 차원으로 묶음수별 단가 표현. 단 ★현재 후니는 묶음수가 `siz_nm`에 융합(`60x90mm(50장)`)·`bdl_qty`와 미연결.
- 흡수 = ① 묶음 단가의 **단위를 명시**("50장 1팩당 1100"·`product_prices.note` or bdl_unit) → ×수량(팩수) 정당성 ② 묶음수가 가격 축이면(트래싱지) siz_nm 융합에서 `bdl_qty` 차원으로 정규화.

### trade-off / 가드
- ★**돈크리티컬: 단가 단위 평탄화 함정** — "50장 1팩당 1100"을 "1장당"으로 오해하면 ×수량 시 50배 오청구(굿즈 GP-2 평탄화·디지털 ×qty 과청구 동일 클래스). 묶음 단위 명시 필수.
- 단일 묶음(OPP봉투 50장 단일)은 siz_cd가 가격 축·bdl_qty=50 메타로 충분(과분할 금지). 묶음수가 변하는 경우(트래싱지)만 bdl_qty 차원 사용.

### naming 유입 가드 [HARD]
`papername`/`ordqty`/`papergroup` 후니 유입 금지. `bdl_qty`/`bdl_unit_typ_cd`(QTY_UNIT) 번역.

## A-2. 같은 규격 × 묶음수 비례 + 볼륨할인 ★

### 경쟁사 패턴
- **와우프레스 ordqty tier** `[wow:acc]`: 조각자석 1000~8000개 tier·수량 올라가면 개당단가 하향(볼륨디스카운트).
- **후니 트래싱지 카드봉투** `[huni:acc]`: `160x110mm` 같은 규격에서 `(20장)` 6000·`(40장)` 12000(정비례)·`(100장)` 28000(28000<30000=볼륨할인). `100x100mm`도 20장 6800·40장 13600·100장 32000 동형.

### 합성식 (관측)
```
가격 = 묶음수별 단가행 룩업(bdl_qty=20/40/100)   # 시트 verbatim·비례 + 대량 할인 베이크
```

### 후니 매핑 (이미 보유)
`component_prices`에 같은 siz_cd + 다른 `bdl_qty`(20/40/100장) 단가행 = **시트 단가 verbatim**(권위 정답). 정률할인 공식(`t_dsc_*`)은 시트에 할인율 명시될 때만.

### trade-off
- 단가행 verbatim이 정률 공식보다 안전(시트=정답·볼륨할인 이미 베이크). 흡수 불요 — 후니 bdl_qty 차원이 동형. 묶음수가 가격 축임을 인식하는 게 실질.

## A-3. 부속물 귀속 (본체 + addon) ★ — 진짜 세트 패턴

### 경쟁사 패턴
- **레드프린팅 부속물#8** `[red:meta]`·`[red:GS §0.4]`: 거치대/끈/링 = `t_prd_product_addons.tmpl_cd`→독립 SKU 다중귀속(PD 등신대 거치대 CDL_DFT·PRD_000016 5개 SKU 다중귀속 PASS). 본체(등신대/포스터) + 부속물(거치대) 결합.
- **후니** `[huni:acc]`: 우드거치대/우드봉/우드행거/천정고리/볼체인/와이어링/행택끈 = **다른 본체 상품에 붙는 부속물**(포스터+우드행거·캘린더+천정고리·태그+행택끈).

### 합성식 (관측)
```
세트 가격 = 본체 상품 가격(자기 공식)
          + Σ 부속물 개당단가(addon·고정가)   # 우드행거 16000·천정고리 6500
          (부속물 수량 × 부속물 단가)
```

### 후니 매핑 (그릇 보유 — 5행만 적재)
- 부속물 자체 = **독립 완제 상품**(시트 67행=단독 고정가·C-AC1).
- 본체에 결합 = `t_prd_product_addons`(라이브 5행·`[huni:live]`)로 귀속. 본체 가격 공식 + addon 개당단가 Σ.
- ★**구성(addons template)과 가격(고정단가 Σ)을 분리** — addons는 어떤 본체에 어떤 부속물이 붙을 수 있나(구성·생산 BOM), 가격은 본체 공식 + 부속물 고정단가(디지털 P-1b template/공식 분리 원칙 동형).

### trade-off / 가드
- 부속물이 단독 판매(시트 단가) + 본체 옵션 양면 → 두 경로 동시 지원(독립 고정가 + addon 귀속).
- 부속물 단가가 본체 가격에 silent 흡수되지 않도록(required addon vs optional addon 명확화·책자 codex DV-BK4 저청구 위험 가드 재적용).

### naming 유입 가드 [HARD]
`addons.tmpl_cd`(레드 CDL_DFT)·`PCS_INFO` 후니 유입 금지. `t_prd_product_addons` 번역.

## A-4. 부속물 = 자재 + 공정 BUNDLE ★ — 이중 의미

### 경쟁사 패턴
- **레드프린팅** `[red:GS §7·8]`: 지퍼=`FLX_ZIP`(부자재 + 박는 공정 BUNDLE)·끈=부착 공정·아일렛=금속링 자재 + 타공 공정. 한 부속물이 자재 + 공정 두 의미.
- **후니** `[huni:acc]`: 우드봉 `270mm + 면끈`·우드행거 `+ 면끈`(우드=자재 + 면끈=자재 + 부착=공정)·천정고리(금속링 자재 + 고정 공정).

### 합성식 (관측)
```
부속물 BUNDLE 가격 = 자재 단가(우드/면끈/금속)        # mat_cd + usage_cd
                  + 부착/봉제 공정 단가(있으면)        # proc_cd
```
단, 후니 시트는 이를 이미 **완제 고정단가**(우드봉 270mm+면끈 7000)로 통합 제시 → 분해 불요(개당 고정단가가 BUNDLE 비용 baked-in).

### 후니 매핑
- 가격은 완제 고정단가(C-AC1)로 충분(시트 7000 verbatim).
- 생산 BOM 관점에선 자재(우드+면끈)+공정(부착) 분해(dbmap 자재/공정축 진단)·가격엔진은 inline 고정가 verbatim만(굿즈 GP 경계 원칙 동형).

### trade-off / 가드
- ★가격엔진은 BUNDLE을 분해하지 말 것(완제 고정단가가 정답·verbatim). 자재/공정 분해는 생산 BOM 트랙(dbmap)·가격엔진 스코프 밖.

### naming 유입 가드 [HARD]
`FLX_ZIP`/`DIR_MTR`/`WRK_MTR` 후니 유입 금지.

## 99. 세트/묶음 패턴 종합 (designer 입력)

- **악세사리 = 다부품 합성(책자형) 드묾** — 봉투/케이스/고리는 단일 완제품·개당 고정단가(C-AC1). `t_prd_product_sets` 분해 불요.
- **진짜 세트 = 본체 + 부속물(A-3 addons)** + **묶음 팩(A-1 bdl_qty)** 두 패턴.
- **High**: A-1 묶음수 단가(단위 명시·평탄화 가드)·A-2 묶음수 비례+볼륨할인(단가행 verbatim)·A-3 부속물 귀속(addons 그릇 보유).
- **돈크리티컬**: ① 묶음 단가 단위(팩당 vs 개당) 명시 — ×수량 정당성 ② 부속물 required/optional 명확화(silent 흡수·저청구 가드).
- **신규 그릇 0건** — bundle_qtys·addons·product_prices·t_dsc 전부 보유. 실질 = 가격사슬 미설계 채움 + 묶음 단위 정규화 + 부속물 귀속 적재(dbmap 위임). 채택·E1~E7은 designer/validator 권한.
