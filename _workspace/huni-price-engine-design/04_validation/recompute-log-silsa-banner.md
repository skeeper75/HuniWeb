## 2. ★축 권위(W=가로·H=세로) 회귀 — 비대칭 매트릭스 실증

- 정상 {w:600, h:1800} → 린넨 (600,1800) 셀 = **32,400**.
- 축 스왑 {w:1800, h:600} → 린넨 매트릭스는 **비대칭**(width tier 600~3000·height tier 600/800/1000/1200/1800). (1800,600) 셀 단가 ≠ (600,1800) 셀 = **다른 값**.
- ∴ designer "W=가로·H=세로·축 뒤바뀌면 비대칭 셀 오룩업" 주장 = 라이브로 **확인**(축 순서 load-bearing·돈크리티컬).
- (재구현 1차 NO_MATCH는 임시 CSV가 부분 행만 export한 아티팩트·라이브 전체 grid로 정정 — 엔진 결함 아님.)

## 3. ★silent 합산 demo (GC-S10) — 배너 후가공 판별차원 미충전

라이브 단가행 verbatim으로 재현(일반현수막 900×900 + 타공):
- body(BANNER_NORMAL 900×900) = **8,000**·PUNCH_4=3,000·PUNCH_6=4,000·PUNCH_8=5,000.
- **미충전 배선(use_dims=[]·전 NON_QTY_DIMS NULL)**: `_row_matches` 와일드카드로 PUNCH_4/6/8 셋 다 통과 → `included=True` 합산 → `8,000+3,000+4,000+5,000 = 20,000` (손님이 6개만 골라도 셋 다 가산 = silent 과청구). **designer 기대 20,000 일치.**
- **판별차원 충전 후(타공6만 매칭)**: `8,000+4,000 = 12,000`. **designer 기대 12,000 일치.**
- 메커니즘 = `_match_entry`(pricing.py:414-419): 판별차원 없으면 note "판별차원 없음 — 항상 매칭"만 남기고 `included=True`로 합산(차단 아님). 디지털 V-DGP-1·아크릴 미러 동형. **돈크리티컬 결함 메커니즘 실증·designer 정확.**
