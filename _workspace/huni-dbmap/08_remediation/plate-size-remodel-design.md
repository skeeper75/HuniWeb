## 핵심 결론 한 줄

`t_prd_product_plate_sizes` 494행은 **압도적으로 작업사이즈(per-item 인쇄 입고 규격)** 를 담고 있고(493행), 전지(출력용지규격)는 **단 1행(프리미엄엽서 SIZ_000499)** 뿐이다. 그러나 작업사이즈는 **무손실 0** 관점에서 **327/494행이 완성품(`t_prd_product_sizes`)으로 재현 불가능한 독립 정보**(접지·펼침·변형으로 작업면이 완성품과 다름)임이 라이브 전수 대조로 입증됐다. 따라서 **"작업사이즈를 전지로 교체"는 작업사이즈를 그냥 지우는 게 아니라, 작업사이즈를 안전히 보존한 뒤 전지축을 새로 추가하는 양립(coexist) 설계**여야 하며, 327행에 대한 무손실 보존책 없이는 작업사이즈 DELETE 불가다(🔴 차단).

## 0. 무손실 전수 대조 결과 (가장 중요 — 데이터 손실 0 입증)

부속 CSV: `csv/worksize-lossless-proof.csv` (494행 전수)

라이브에서 **각 plate 작업사이즈 행의 work 치수가, 같은 상품의 `t_prd_product_sizes`(완성품) work 치수에 존재하는가**를 전수 비교했다.

| 무손실 판정 | 행 수 | 의미 | 작업사이즈 DELETE 가능? |
|------------|:-----:|------|:---:|
| `JEONJI_ROW` | **1** | 이 행 자체가 전지(316x467, 프리미엄엽서). 작업사이즈 아님 | 별도(전지축 이관) |
| `REPRODUCIBLE` | **166** | 작업사이즈 work 치수 = 완성품 work 치수 → `product_sizes`에서 런타임 재현 가능 | ✅ 보존 입증됨(완성품+런타임) |
| `NOT_REPRODUCIBLE` | **327** | 작업사이즈 ≠ 완성품(접지/펼침/변형으로 작업면이 큼) → 완성품으로 재현 **불가** | 🔴 별도 보존 없이 DELETE 시 손실 |

**근거 실측 (접지 계열이 결정적):**
- 코팅엽서(PRD_000017): 완성품 work = 작업사이즈 work가 **동일**(75x100·100x100·…·150x212) → 엽서류는 REPRODUCIBLE.
- 접지리플렛(PRD_000048): 작업사이즈 426x303·425x303·424x303(A3 펼친 인쇄면) ≠ 완성품 A4/A5 → 펼침으로 작업면이 큼 → **NOT_REPRODUCIBLE**.
- 2단접지카드(PRD_000027): 작업 274x139(2단 펼침) ≠ 완성 137x137 → **NOT_REPRODUCIBLE**.
- 봉투제작(PRD_000050): 작업 510x387(전개도) ≠ 완성품(봉투 접힘 후) → **NOT_REPRODUCIBLE**.

→ **단일 블리드 가산으로는 327행을 재현할 수 없다.** 접지단수·펼침·전개에 따라 작업면이 비선형으로 달라지므로, "완성품 + 일정 블리드 = 작업사이즈" 공식은 **166행에만 성립**한다.

### 무손실 보존 설계 (3-갈래)

1. **REPRODUCIBLE 166행** — 완성품(`product_sizes`) + 런타임 블리드로 재현 가능함이 입증. 단 작업사이즈 siz_cd(예 코팅엽서 SIZ_000112)와 완성품 siz_cd(SIZ_000001)는 **다른 siz 엔티티**이므로, plate 행을 지우면 "이 상품의 이 작업사이즈"라는 product↔작업사이즈 링크는 사라진다. 보존책: ① plate 유지(현행) 또는 ② 별도 작업사이즈 테이블 이관. **현 단계 권고: 유지**(전지축을 추가하되 작업사이즈 행은 보존).
2. **NOT_REPRODUCIBLE 327행** — 🔴 **DELETE 절대 불가**(보존 입증 실패분). 작업사이즈는 인쇄 입고 파일 규격으로 실재 필요 정보이며 완성품으로 재구성 불가. 보존 필수.
3. **JEONJI_ROW 1행** — 프리미엄엽서 전지 행. 작업사이즈 아님 → 전지축으로 이관 + 작업사이즈 7행 보강(아래 §5).

> **핵심 판정: 작업사이즈 493행 전부를 plate에서 제거하는 것은 327행 무손실 위반으로 🔴 차단.** 본 설계는 "작업사이즈 제거"가 아니라 **"전지축을 plate에 추가(또는 분리 모델)하되 작업사이즈는 보존"** 으로 방향 전환한다. 사용자 결정 1("전지 테이블이어야")을 **무손실과 양립**시키려면 작업사이즈를 별도 보존(전용 컬럼/플래그/테이블)해야 하며 — 이는 🟡 결정 필요(§6 D-1).

### 5.1 ①프리미엄엽서 오등록 교정 (🔴 명백 오류)
부속 CSV: `csv/immediate-fix-premium-postcard.csv`.

- **현황:** PRD_000016 plate = `SIZ_000499`(전지 316x467) 1행만. 작업사이즈 행 0개. 형제(코팅엽서)는 작업사이즈 7행.
- **교정:** 프리미엄엽서 작업사이즈 = **엽서 작업사이즈 7종 보강** = 코팅엽서(PRD_000017) plate의 `SIZ_000112(75x100)·SIZ_000116(97x212)·SIZ_000113(100x100)·SIZ_000114(102x152)·SIZ_000117(112x172)·SIZ_000115(137x137)·SIZ_000118(150x212)`.
  - 근거: 프리미엄엽서 `product_sizes` 완성품도 **7종**(73x98·95x210·98x98·100x150·110x170·135x135·148x210) — 코팅엽서 family와 동일. 엑셀 `excel-plate-size`는 프리미엄엽서 출력용지규격=316x467(전지)만 명시, 작업사이즈는 판걸이수 엽서행이 권위.
  - **진단(plate-size-live-diagnosis) 정정:** 진단은 "작업사이즈 5행 누락"이라 했으나, 코팅엽서 family는 **7행**(97x212·112x172 포함)이다. 프리미엄엽서 완성품도 7종이므로 **7행 보강**이 정확.
- **전지 행 처리:** 기존 `SIZ_000499` 전지 행은 §6 D-1 결정에 따라 (a) 전지축으로 유지 또는 (b) 제거. 작업/전지 혼재 상태 해소.

### 5.2 ②output_paper_typ_cd `.03` 노이즈 99행 정리
- MIXED 17상품의 81행 + 기타 18행 = 99행. **작업사이즈 행의 `.03`은 NULL화**(§4.3). 전지 분류는 전지 행으로 이관.
- 규칙: `UPDATE … SET output_paper_typ_cd=NULL WHERE output_paper_typ_cd='OUTPUT_PAPER_TYPE.03' AND siz_cd NOT IN (전지 siz_cd 집합)` — 설계만, 실행 아님.

## 6. 마이그레이션 설계 (설계만, 실행 아님)

### 6.1 적용 순서 (FK 위상 + 무손실 게이트)
1. **(선결) 무손실 결정 D-1 확정** — 작업사이즈 보존 방식(plate 유지 vs 전용 테이블). 미확정 시 후속 전부 차단.
2. **코드/siz 선등록** — 신규 전지 5종 siz(`SIZ_000501~`) + output_paper_typ_cd 계열(이미 .01/.02/.03 존재, 신규 없음). `dbm-ddl-proposer`/코드행 승인.
3. **작업사이즈 보존 확정** — 327 NOT_REPRODUCIBLE 행 보존 입증/이관 완료(손실 0 게이트 통과).
4. **작업사이즈 행 처리** — D-1=유지면 무변경. D-1=분리면 작업사이즈 행을 전용 보존처로 이관 후 plate에서 정리.
5. **전지 행 등록** — 상품별 전지 plate 행 추가(210행 매핑 권위). 프리미엄엽서 전지 행은 교정 반영.
6. **output_paper_typ_cd 백필** — 전지 행에 계열 부여 + 작업사이즈 .03 99행 NULL화.
