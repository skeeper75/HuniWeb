# 적재 로직 재구성 — loadlogic-notes (PRF_DGP_A 가격 영역)

> 적재 oracle 재구성: 가격표 셀이 t_prc_* 단가행으로 어떻게 변환됐나 + 발견된 적재 로직 결함.

## 1. 적재 경로 규명 (누가 이 가격을 넣었나)

| 후보 적재기 | 이 가격 영역 처리 여부 | 증거 |
|-------------|:--:|------|
| `raw/webadmin/tools/load_master.py` | **아니오** | `grep "t_prc_\|component_prices" load_master.py` = 0건. load_master = 상품마스터(t_prd/t_mat/t_proc) 전용 |
| huni-dbmap **round-2 가격엔진 파일럿** | **예** | 라이브 note `round-2 파일럿 자동생성` 명시(t_prc_price_components 전건) |
| huni-dbmap **디지털인쇄 가격엔진 트랙** | 예(보강) | COMP_PAPER note `신규: 디지털인쇄 용지비`·메모리 [[dbmap-digitalprint-atomic-formula-unbuilt]] 308행 COMMIT |
| siz 교정 트랙 | 예(후속) | 단가행 note `[siz-corrected: SIZ_PENDING_3JEOL→SIZ_000077]`·`SIZ_PENDING_GUK4→SIZ_000499` |

**중요**: 이 가격 영역은 **webadmin v03 마이그레이션 산물이 아니다**. 우리(huni-dbmap)가 가격표에서 직접 추출·적재했다. → 사용자 우려의 "webadmin이 분석 없이 적재" 프레임은 상품마스터(t_prd/t_mat/t_proc)에는 해당하지만, **이 가격 단가행에는 해당하지 않는다**. 교정 책임·트랙도 우리 쪽.

## 2. 셀→단가행 변환 규칙 재구성

### 2-1. 디지털인쇄비 / 코팅 / 별색 (출력축 단가형 — 올바름)

```
가격표 디지털인쇄비!A{r}=수량(국4절) → component_prices.min_qty
가격표 B{r}(흑백단면) → unit_price,  siz_cd=SIZ_000499(국4절)/SIZ_000077(3절), clr_cd=CLR_000002(흑백)
양면=C열→S2 comp, 칼라=D/E열→CLR_000005, 별색=F~K열→SPOT_* comp(clr_cd 미사용·comp로 분리)
prc_typ_cd=PRICE_TYPE.01 (단가형 — 출력매수×장당가)  ← 올바름
```
- 변환 충실: 값 verbatim, 5색 별색 개별 comp 분해(collapse 없음), 단/양면·흑백/칼라 분리. **결함 없음.**
- `A3="수량(국4절)"` 헤더를 라이브 note는 `출력매수≥N`으로 해석 — 토대 §3(이 수량표는 출력매수 기준)와 정합.

### 2-2. 인쇄후가공 오시·미싱·가변·둥근모서리 (주문축 합가형 — 결함)

```
가격표 인쇄후가공!A{r}=제작수량 → min_qty
가격표 B/C/D{r}(오시 1/2/3줄) → unit_price,  siz_cd=NULL(차원 무관)
prc_typ_cd=PRICE_TYPE.01  ← ❌ 결함: 가격표는 (합가)=구간고정총액형(.03)이어야 함
```
- 값 변환은 충실(verbatim·차원 NULL 정확). **단가유형 메타만 오적재.**

## 3. 발견된 적재 로직 결함

### 결함 ① prc_typ 일괄 .01 평면화 (주결함·돈 영향)

- **원인**: round-2 파일럿이 모든 component를 일괄 `PRICE_TYPE.01`로 생성. 라이브에 `.02 합가형`은 존재하나 그 엔진 의미가 명세식 `÷mq×qty`라 후가공 구간총액에 부적합 → 파일럿이 구간총액형을 표현할 적절한 코드(`.03` 또는 `.02` 재해석)를 두지 않고 .01로 흡수.
- **결과**: 출력축 단가형(디지털/코팅/별색)에는 정합하나, 후가공 합가형 13 comp가 .01로 잘못 표시 → 엔진 작동 시 ×수량 과대.
- **교정 진원**: 단가행이 아니라 **prc_typ 메타**. 정정 코드는 `.02` 재사용 vs 신규 `.03` 비교(root-cause §3-1·C-D1b). correction-manifest M-1.

### 결함 ② note에 내부 메모·코드값·교정마커 출력 (사용자 직접 지적)

- **원인**: 적재 스크립트가 디버그/추적용 메모(`round-2 파일럿 자동생성`·`comp_typ=.04`·`별색=공정,clr=NULL`·`[siz-corrected:...]`)를 note 필드에 그대로 기록.
- **결과**: 토대 §9-1 위반 — 실무진이 admin에서 못 알아봄. correction-manifest M-2.

### 결함 ③ off-grid 누진룰 미반영 (보정 — 옛 "꼬리행 누락" 재판정)

- **원인**: 가격표 `인쇄후가공`의 off-grid 누진 주석(`A12 "100장당 1000원씩 올립니다"`·`A27/F27 "1,000장당 20,000원씩 올립니다"`)을 적재 시 미보존. 격자 단가행만 옮기고 5000매 초과 누진 계수는 버림.
- **결과**: 모서리/오시/미싱 라이브 max min_qty=5000에서 끊김 → 5000매 초과 주문 시 후가공비가 105,000에 머물러 **과소청구**. correction-manifest **M-G2**(MISSING). 누진룰은 격자 평면화로 표현 불가능한 런타임 계산식(토대 §3) — 단가행 복제 아님.
- **부수 재판정**: 가변 `A57=100000`(B57~K57 None)은 단가행이 아니라 누진룰 경계 표지 → 라이브 미적재 정당(복제 INSERT 오진단 철회). correction-manifest **C-6**.

## 4. 적재 경로 불명 항목

없음 — 본 영역 전 구성요소의 적재 경로(round-2 파일럿 + 디지털 트랙 + siz 교정)가 note·메모리로 재구성됨.

## 5. 교훈

- **값 정확 ≠ 적재 정확.** 단가행 값이 가격표와 verbatim 일치해도, 그 값을 어떻게 계산할지(prc_typ)가 틀리면 가격이 수량배로 튄다. 정합 감사는 값뿐 아니라 **계산유형 메타·축**까지 봐야 한다.
- **누락 코드값이 평면화를 유발.** .03 단가유형이 없어 합가형이 .01로 흡수됐다 — 기초 코드값(base_code)을 먼저 제대로 등록해야 메타 적재가 정확해진다(적재 방법론 §선결 게이트와 동일).
- 이 영역 결함은 webadmin이 아니라 **우리 round-2 파일럿의 일괄 .01**이 진원 → 교정도 우리 트랙에서 닫는다.
