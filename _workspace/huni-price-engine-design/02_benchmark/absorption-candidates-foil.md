## 0. 종합 판정 요약

**후니 박 모델은 경쟁사 대비 이미 합리적·정밀하다**(동판비/박가공비 분리 + 면적등급×수량구간 + 일반/특수 2블록). 경쟁사로부터 박 **계산식 자체를 교체할 흡수는 없다.** 흡수 후보는 모두 **표현력/UX/엣지케이스 보강**(갭헌팅) 수준이며, 권위 데이터는 그대로 둔다.

## 1. F-1 — 동판비/박가공비 2-컴포넌트 분리 (이미 정합·흡수 불요)

**경쟁사 근거:** 레드프린팅 박/형압 스티커가 "동판 금형을 만들고"로 setup비 독립 명시. 박 공정의 물리(동판=1회 제작, 가공=수량비례 압인)를 정확히 반영.

**후니 현황:** 권위 엑셀에 **이미 동판비 블록(B01) + 박가공비 블록(B03~B05) 분리**. 박(대형)=동판비 면적매트릭스, 박(소형)=동판비 고정 5,000.

**후니 t_prc_* 매핑(설계 입력):**
```
박 = 동판비_component + 박가공비_component  (2개 price_component를 공식에 합산)
  동판비_component:
    대형 → use_dims=[siz_width, siz_height], 면적매트릭스 단가행
    소형 → 고정값(use_dims 없음, 단일 단가행 5,000)
  박가공비_component:
    use_dims=[박등급(A~E), 수량구간]   ← 등급은 면적→등급 lookup 선행
```
**trade-off:** 동판비는 **1회성**(수량 무관) — evaluate_price에서 ×qty 되지 않도록 prc_typ 주의(돈크리티컬: 명함 밴드총액 ×qty 과청구 교훈[bandtotal-x-qty-overcharge] 동일 함정). 동판비는 **고정 합산형(.02)**, 박가공비는 **수량구간 lookup**.

**판정: 흡수 불요(이미 보유) — 단 evaluate_price 배선 시 동판비 ×qty 금지 가드 필수.** naming 유입 가드: 레드 "동판 금형"·와우 라벨은 후니 comp_nm로 가져오지 않음(후니 권위 용어 사용).

## 2. F-2 — 일반박/특수박 2블록 차등 (이미 정합·흡수 불요)

**경쟁사 근거:** 레드(홀로그램=프리미엄 별도)·와우(홀로그램박 select). 후니=권위 엑셀에 이미 **일반박 블록(B03/B02) vs 특수박 블록(B04/B05)** 별도 단가표.

**후니 t_prc_* 매핑:** 박 종류 select → **일반박/특수박 분기**가 박가공비 단가표(component_prices) 선택을 바꿈. = rpmeta 캐스케이드 패턴(상위 선택→하위 단가표 전환)과 정합.
```
박종류(opt) → {일반박 | 특수박} 분기
  → 박가공비 단가표[등급][수량] 중 해당 블록 lookup
```
**trade-off:** 박 종류가 6~12종이나 가격은 **일반/특수 2tier로 collapse**(엑셀 권위가 그렇게 설계됨). 종류별 개별 단가가 아니라 **2그룹 단가** — 위젯에선 종류 라벨 12개 노출하되 가격은 2tier 매핑.

**판정: 흡수 불요(이미 보유). 흡수 후보로서의 가치 = "박 종류 select와 가격 tier를 분리하라"는 설계 원칙 확증.**

---

## 3. F-3 — 박 영역 면적비율 추가금 (표현력 갭 · Medium)

**경쟁사 근거(레드 명함, verbatim):** "박이 묻는 영역이 명함면의 **25%를 초과하면 추가금이 발생**". 박 사용 면적이 클수록 박 소재·압인 비용 증가를 반영.

**후니 현황:** 권위 엑셀의 박가공비는 **가로/세로 mm → 등급(A~E)** 으로 박 영역 절대크기를 양자화 — 즉 후니도 **박 영역 면적을 이미 등급으로 반영**(레드의 "25% 비율 추가금"을 절대 mm 등급으로 표현). 표현 방식이 다를 뿐 동일 동인(박 면적↑→비용↑).

**흡수 후보:** 레드의 "면 대비 비율(%)" 발상은 **상품면 크기 대비 박 면적**이라는 상대 축. 후니는 박 절대 mm만 봄. → 같은 박 50mm라도 큰 명함/작은 명함에서 비용 동일(후니). 레드는 상대비율로 큰 면에선 더 너그러움.

**후니 적용 가능성:** 후니 권위 엑셀이 **절대 mm 등급을 정답**으로 삼으므로(권위 절대), 비율 추가금을 **새로 도입하지 않는다**. 다만 설계 시 "박 등급이 상품 사이즈와 무관하게 박 절대크기로만 결정된다"를 명문화 — 이것이 권위 의도. **표현력 갭은 인지하되, 권위가 절대 mm로 결정 → 비율 모델 미흡수.**

**판정: 인지·기록만(권위 절대 mm 우선). 흡수 시 권위 덮어쓰기가 되므로 도입 금지. E3 검증 시 "면적비율 추가금 미도입 = 권위 정합" 확인.**

## 4. F-4 — 박 크기 UX = 단계/구간 양자화 셀렉터 (위젯 계약 · Low)

**경쟁사 근거:** 와우=박 크기 select 4단계(15mm이하/30mm이하/70mm이하/90mm미만). 레드=박 영역 4구간(20x20 이하/40x40 이하/40x40 초과/대형). 둘 다 **연속 크기를 이산 구간으로 양자화**해 손님이 고르게 함.

**후니 현황:** 후니 권위는 가로/세로 mm 격자(30~170mm 또는 10~80mm) → 등급. **연속 입력 후 격자 lookup**(off-grid=ceiling 큰 쪽) 또는 **드롭다운으로 격자값 선택** 둘 다 가능.

**흡수 후보(위젯 UX):** 경쟁사처럼 박 크기를 **드롭다운 구간 셀렉터**로 제시(자유 mm 입력보다 손님 친화·견적 안정). 후니 격자(30/50/70/90/110/130/150/170mm)를 그대로 드롭다운 노출.

**후니 적용:** 가격 엔진(t_prc_*)은 불변 — **위젯 계약(option_items)에서 박 크기를 격자값 드롭다운으로** 노출하면 off-grid 견적 불안정(ceiling 보간) 회피. 가격 모델 변경 아닌 **UX/위젯 흡수**.

**판정: 위젯 단계에서 흡수(가격 엔진 무변). disp 옵션을 격자값으로.** naming 유입 가드: 와우 "15mm이하"·레드 "20x20 이하" 라벨 문구는 후니 표시명 표준(siz 라벨)으로 재작성.

## 5. F-5 — 박 = 면(앞/뒷) 독립 선택 가능성 (정책 확인 · Low)

**경쟁사 근거:** 와우 명함이 **박(앞면) / 박(뒷면) 각각 select**(spdata 115282/116822). 즉 양면 박을 면별로 따로 고름.

**후니 현황:** pricing-rules.md PRC-005 관찰 = "박명함 양/단면 동일 단가(박 가공이 면 단위 아님)". 즉 **후니는 박을 면 무관**으로 보고 단가 동일.

**충돌 점검:** 와우는 면별 select(앞/뒷 각각 박 가능 → 비용 2배 가능), 후니는 면 무관 동일단가. → **모델 차이.** 후니 권위(상품마스터·가격표)가 면 무관이면 그대로. 단 "양면 박 = 앞박+뒷박 두 항목 합산"이 실무라면 후니도 박 항목을 면별 2회 합산할 수 있어야 함(현 권위는 동일단가).

**판정: 정책 확인 항목(흡수 아님).** 후니 권위 = 면 무관 동일단가 유지. 단 설계 시 "양면 박이 앞+뒷 두 번 청구되는가, 1회인가"를 권위 엑셀/실무 확인(AskUser 후보). 와우의 면별 select는 **흡수하지 않되, 양면 박 청구 정책을 명문화하라는 신호.**

## 6. F-6 — 박 전용 상품 vs 후가공 옵션 2경로 (IA · Low)

**경쟁사 근거:** 와우 = 박을 ① 일반 명함 후가공 옵션 ② **엠보싱금박명함/은박명함/홀로그램박명함 = 독립 상품** 두 경로. 레드 = 박/형압 명함(BCFOXXX) = **박 전용 상품군**.

**후니 현황:** 후니 박 = 후가공 시트(대형/소형)로 **여러 상품에 공유되는 후가공 component**. 박 전용 상품은 별도 IA 결정.

**흡수 후보:** 박을 **공유 후가공 component(현 후니 모델)** 로 두되, 마케팅상 "박명함" 같은 **프리셋 상품 진열**은 IA/카탈로그 층에서. 가격 엔진은 동일 박 component 재사용(search-before-mint — 박 공식·component 중복 mint 금지).

**판정: 가격 엔진은 공유 component 유지(흡수 불요). IA 차원에서 박 프리셋 상품 진열은 §5 print-quote/카탈로그 결정.**

## 7. 흡수 가드 체크리스트 (E3 검증 입력)

- [ ] **권위 덮어쓰기 0:** 동판비·등급·수량구간·일반/특수 단가 = 전부 후니 권위 엑셀 verbatim. 경쟁사 값 주입 0.
- [ ] **naming 유입 0:** 레드/와우 라벨(동판 금형/15mm이하/박(앞면)/엠보싱박명함)·코드(BCFOXXX/spdata_*/11529)는 후니 comp_nm·코드로 유입 안 함.
- [ ] **계산식 미교체:** 후니 "동판비+박가공비(면적등급×수량구간)" 모델 유지. 경쟁사 면적비율 추가금(F-3) 미도입.
- [ ] **돈크리티컬 가드:** 동판비 = 1회성(×qty 금지). 박가공비 = 수량구간 lookup(이미 수량 반영). 이중 ×qty 방지.
- [ ] **일반/특수 2tier collapse:** 박 종류 다수 → 가격 2tier 매핑(개별단가 mint 금지).
- [ ] **양면 박 정책:** 면 무관 동일단가(권위) — 앞+뒷 이중청구 여부 확인 필요(F-5).

## 8. 후니 박 t_prc_* 설계 권고 (engine-designer 입력)

```
[공식] PRF_FOIL_<상품>  (또는 후가공 공유공식)
  = 동판비_comp + 박가공비_comp        ← 합산(addtn)
  
동판비_comp (price_component):
  prc_typ = 고정합산(.02, ×qty 금지)
  대형: use_dims=[siz_width, siz_height] → 면적매트릭스 단가행(11k~64k)
  소형: 단일 단가행 5,000

박가공비_comp (price_component):
  prc_typ = 수량구간 lookup
  use_dims=[박등급, 수량구간]
  박등급 = 면적분류(siz_width,siz_height → A~E)  ← 선행 lookup(앱계산 또는 차원행)
  단가표 분기 = 박종류(opt) → {일반박표 | 특수박표}
```
- **search-before-mint:** 박 공식·동판비/박가공비 component·등급 차원이 라이브 t_prc_*에 이미 있으면 재사용(중복 mint 금지). 미등록은 BLOCKED → dbmap 위임(인간 승인).
- **단가값 verbatim:** 권위 CSV 값 그대로(날조 0).
- **DB 미적재:** 설계 명세까지. 실 COMMIT/DDL은 인간 승인 후 dbmap.
