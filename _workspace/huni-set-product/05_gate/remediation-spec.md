# 교정 명세 — 엽서북 셋트 게이트 BLOCKED/결함 (인간 승인 큐)

생성: hsp-set-gate · 라이브 읽기전용 재실측 기반 · 실 COMMIT/DDL은 **인간 승인 후 라우팅 트랙 위임**(게이트는 명세만).

## RM-1 ★ 30P 가격결함 — 진단 정정(부재 아님·미바인딩+오청구) — 돈크리티컬 · High

| 항목 | 내용 |
|---|---|
| **결함(게이트 정정)** | 설계/codex는 "30P comp·단가행 라이브 부재 → 견적불가(BLOCKED)"라 진단. **라이브 실측은 다름**: COMP_PCB_S1_30P·S2_30P **존재**(use_dims=[siz_cd,min_qty]) · 30P 단가행 각 117행 **존재**(note "…/30P"). |
| **실제 동작** | (a) 30P comp가 PRF_PCB_FIXED `formula_components`에 **미바인딩**(고아·0행 실측). (b) 전 comp use_dims에 **page 차원 부재**(0건) → 사용자가 30P 선택해도 evaluate_price가 바인딩된 **20P comp를 매칭**해 견적. (c) 30P가 20P보다 비쌈(SIZ_000003 단면 수량4~20: +800/장). → **30P 선택 시 20P 단가로 과소청구(under-charge)**. |
| **권위 정답** | booklet-l1 row61: 엽서북 내지페이지 20/30 택1 — 각 페이지별 정답 단가가 권위(가격표 260527). 30P 단가행은 이미 라이브에 verbatim 존재. |
| **교정 방법(선택지)** | ① 페이지(20/30)를 가격 차원으로 도입: 20P/30P comp를 페이지 선택에 따라 분기 매칭(use_dims에 page 차원 추가 or 공식 분리). ② 또는 PRF_PCB_FIXED에 30P comp 배선 + evaluate_price가 페이지 선택으로 20P/30P comp 택1하도록 로직. **둘 다 신규 단가 데이터 생성 불요**(데이터는 존재). |
| **대상 t_*** | t_prc_formula_components(30P 배선)·t_prc_price_components(use_dims 페이지 차원)·가격엔진 코드(pricing.py 페이지 분기) |
| **돈 영향** | 30P 주문 전건 과소청구. 즉시성 High. |
| **라우팅** | **§18 가격엔진 설계 / dbmap 적재 트랙**(공식 배선·차원 도입). webadmin 코드 직접수정 금지. **인간 승인 필요**. |
| **적재본 영향** | **없음** — 엽서북 셋트 보정 적재본(95/96 행·94 유형)에 30P 변경 미포함. 본체 GO와 분리. |

> ★ 이 정정은 게이트 직접 재실측의 핵심 산물(생성≠검증). 생성측 "부재" 진단을 그대로 신뢰했다면 교정 경로(신규 단가 데이터 생성)를 오설계할 뻔함. 실제는 "배선+차원" 문제.
