## 2. 멤버별 4축 판정 (1:1 대조 — 일괄판정 금지)

### 삼각대(그레이): MAT_000032 vs MAT_000252

| 축 | 검사 | 판정 |
|---|---|---|
| ① 권위추출+표시↔실제 정합 | mat_nm↔mat_typ_cd·부모·BOM↔권위(상품마스터) | **252=정합**(권위 실재·BOM 1·USAGE.07 완성품 거치). **032=오적재 신호**(권위 부재·BOM 0·cp 0·.02 링제본부자재인데 삼각대=거치부속 의미축 불일치) |
| ② 표시 중복 | 화면 표시명 '삼각대(그레이)' 동일·코드 다름 | **YES** — 032/252 둘 다 mat_nm='삼각대(그레이)' |
| ③ 내부값 중복 | 표시명+유형+부모+치수 동일 | **부분** — 표시명·치수(NULL) 동일하나 **mat_typ_cd 상이(.02 vs .07)·부모 유무 상이** → 완전 동일 아님 |
| ④ 의미구분 보존 (가드) | mat_typ_cd/부모로 정당구분 가능? | **032=stale 잔재 강함**(.02 링제본은 삼각대와 무관·권위·BOM 전무). 의미축 분리(.02 카탈로그 vs .07 생산)는 약함 — 032의 .02는 정당한 별도 의미가 아니라 잘못 붙은 유형. |

→ 표면상 **A안**(032 stale 잔재 논리삭제·252 정본)으로 수렴하는 듯 보임. **그러나 ③④에서 252 자체가 정본 자재로 확정되지 않음** — 다음 §3 참조.

### 삼각대(블랙): MAT_000033 vs MAT_000254 (평행구조·★Phase1 미적시)

표시명은 상이('캘린더부자재 삼각대블랙' vs '삼각대(블랙)')라 ②표시중복은 NO이나, **구조가 그레이와 완전 평행**: 033=032의 블랙 짝(부모 MAT_000031 공유·권위부재·BOM0), 254=252의 블랙 짝(root·권위실재·BOM1[PRD_000109]). C-CAL-01이 252/254를 **함께** 자재→공정 오적재로 명시하므로 동일 BLOCKED 판정.

## 3. ★축이동 경계 분석 (판정 C의 핵심 근거)

상위 dbmap round-13/22 **C-CAL-01** 진단(인용):

- `32_axis-staged-load/02_axis-error-diagnosis.md:85` — **"삼각대=자재→공정(calendar C-CAL-01)"** = MIS-LOADED.
- `23_remediation-apply/calendar/cpq-plan.md:17` — 삼각대 색 → "자재 오적재(MAT_000252/254) → **option_items(공정 param)** — 삼각대 거치 공정 mint 후".

- `cpq-plan.md:81` — "삼각대 거치 공정 마스터 **부재(0행 실측)** → mint 선행(ddl-proposer·CL-A)".
- `cpq-plan.md:83` — "자재 오적재 정리: MAT_000252/254(삼각대) 자재 연결행은 공정/param 이전 후 **del_yn='Y' 논리삭제**".
- `cpq-plan.md:121` — "삼각대색 공정 param | 삼각대 거치 공정 0 | **BLOCKED** | 삼각대 거치 공정 mint 선행(CL-A)".

**경계 판정:**
- 본 하네스(표시중복) 범위 = "화면 같은 게 두 번 보이나(032/252)" + "표시↔실제 어긋났나".
- 축이동 트랙(round-22/basecode §12) 범위 = "삼각대가 자재축에 있어야 하나 공정축으로 가야 하나".
- ★표시중복 표면정리(032 논리삭제·252를 '정본 자재'로 채택)는 **252를 자재 정본으로 못박아** 상위 "252/254 둘 다 공정 param 이전 후 논리삭제" 결정을 선점·충돌.
- 따라서 **032도, 252도 지금 손대지 않음** = 판정 C(BLOCKED). 사이즈·공정·카테고리 교훈("1차 판정을 권위로 재검증·표시중복 정리가 상위 결정 덮어쓰면 안 됨") 정합.

**A안을 채택하지 않는 이유:** A안(032 삭제·252 정본)은 표시중복은 닫지만 252의 자재 지위를 확정 — 이는 C-CAL-01이 "252 자체가 공정으로 가야 한다(자재 아님)"고 본 것과 정면 충돌. 표시중복 한 그룹을 닫으려고 더 큰 축이동 결정을 잠그는 것은 false-resolution.

**B안(의미축 분리 keep)도 아닌 이유:** 032의 .02 링제본부자재는 삼각대(거치부속)와 무관한 유형 — "카탈로그 vs 생산" 같은 정당한 의미축 분리가 아니라 단순 오적재. 그러므로 정당구분 keep으로 영구화할 근거도 없음. → C(보류)가 유일하게 정직한 판정.

## 4. false-positive 가드 (D2)

- 032/252는 표시명 동일하나 **mat_typ_cd(.02/.07)·부모 유무 상이** → 기계적 "byte-identical 진짜중복"(사이즈 SIZ_104/105 사례)과 다름. 진짜중복으로 **승격하지 않음**(가드 통과).
- 그레이·블랙을 **각각** 평가(일괄 흡수 금지) — 블랙은 표시명 상이라 ②표시중복 NO지만 평행구조·C-CAL-01 공동대상이라 함께 BLOCKED.
- 잘못된 false-positive(032/252를 진짜중복으로 단정해 252 정본 merge) **0건**.
