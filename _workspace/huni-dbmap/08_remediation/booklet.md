# booklet 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 **누락·오매칭·모호**가 생겼다.
> 본 문서는 그 결함을 *올바로 처리(수정)*하기 위해, **라이브 Railway DB 직접 read-only SELECT를 권위**로
> booklet(책자) 상품군을 종단 규명한다. 도메인 지식(07_domain KB)은 결함을 옳게 분류하는 렌즈다.
> booklet 핵심 = **내지/표지/면지 2~3축 반제품 구조**(R-MAT-4·R-SET-1)와 **제본 택일그룹**(GRP-BOOK-제본).
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본 `ref-*.csv`·v2는 stale 가능).
> 추정·임의판단 0 — 라이브·엑셀 L1·도메인 KB·PDF 실재 근거만. 미지/추정은 가설+컨펌 플래그.
> INSERT/UPDATE/DDL 0. DB 미적재 유지(본 문서는 처리 *방향* 설계). `db railway`·비표준포트·JOIN KEY=`prd_nm`.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. **라이브 SELECT 수행 15회**(전부 read-only).

## ⚠️ 머리말 — 본 검증이 v2/B-rules 대비 뒤집은 가장 큰 사실 (권위 반전)

라이브 직접 확인 결과, 기존 산출물(B-normalization-rules·v2)의 다음 가설이 **라이브에서 반전**됐다:

1. **B-rules §⑤-2 "중철책자 PRD_000068 sets 0행 → 셋트 미적재 의심(MISSING)"** →
   **라이브 반전(false MISSING)**. 中철·무선·PUR·트윈링(068~071)은 **sets 0행이 정상**이다.
   이들은 **통합상품**으로, 내지/표지가 독립 반제품(sub_prd)으로 분해되지 않고 parent 한 상품에 직접 적재되는
   구조다(엑셀 표지/내지가 단일 행 캐스케이드). BLOCK-2 기수확과 정합. → **셋트 결함 아님**.
2. **R-MAT-4 "parent 자재 비어있음이 정상(자재는 sub_prd에 붙음)"** → **라이브 반전**.
   라이브는 정반대 모델로 적재됐다: **하드커버류(072/077/082/088/094/097/100)는 자재·공정·사이즈를 전부
   parent에 적재**하고 `usage_cd`(USAGE.01 내지/.02 표지/.03 면지)로 슬롯을 구분한다.
   **모든 sub_prd(PRD_TYPE.02 반제품 073~107)는 9속성 전부 0행**(빈 껍데기). sets만 parent→sub_prd를 가리킨다.
   → "자재를 sub_prd에서 찾으면 전량 MISSING 오판" 위험. **자재 권위 위치 = parent + usage_cd**.
3. **B-rules가 예측한 "page_rule 대량 MISSING"** → **라이브 반전(false)**. booklet 10 parent 전부
   `t_prd_product_page_rules` 적재됨(068=4/28/4, 069·070=24/300/2 등). page_rule 미적재 결함 **없음**.
4. **제본 택일그룹(GRP-BOOK-제본)** → **라이브 정상 적재 확인**. booklet 10 parent(068~072·077·082·094·097·100)
   전부 excl_group 1행 + `trigger=<제본명>`. R-PSG-1 추론이 라이브와 정합. → 택일 결함 아님.

**교훈(HARD 재확인):** booklet의 진짜 결함은 "셋트/페이지룰/택일 미적재"가 **아니라**,
**내지/표지 종이 자재(`*별도설정`→IMPORT)의 미적재**다. 추출본 기반 가설은 결함 위치를 오인했다.

## ④ 결함 표 — 누락(MISSING)·오매칭(MISMATCH)·모호(CONFIRM) (도메인 렌즈, 라이브 재확인)

> 분류 기준: **MISSING**=정규화 규칙대로 엑셀을 변환하면 적재돼야 하는데 라이브에 없는 행.
> **MISMATCH**=잘못 연결/모호 매칭. **CONFIRM**=도메인 의미 미확정(실무 컨펌). 모든 판정 라이브 SELECT 재확인.

### G-BK-1 [MISSING · BLOCKER급] 내지종이 자재(`*별도설정`→IMPORT) 미적재 (R-MAT-2/4)

- **도메인 근거:** 엑셀 8상품 내지종이칸 `*별도설정` = IMPORT 종이 별도 정의 포인터. 내지는 책자의 본문 자재 —
  미적재 시 위젯 종이선택·가격 산정 불가.
- **라이브 재확인(SELECT #5,10):** 068·069·070·071·072·077·082·088 **내지종이(USAGE.01) 0행**.
- **IMPORT 해소(`import-resolution-resolved.csv`·`seoljeong-import-map`):**
  중철내지 13종·무선내지 7종·트윈링내지 16종은 IMPORT 컬럼 ● 매핑 확정. **PUR·하드커버 3종은 IMPORT 컬럼 부재**(자재 권위 매트릭스 밖 — 별도 소스 컨펌).
- **MISSING 합계:** 내지종이 적재 대상 8상품 × N종이(IMPORT ● 수). 매핑확정 3상품(068/069/071)=즉시 적재 가능, 미확정 5상품(070·072·077·082·088)은 자재 소스 컨펌 후.
- **처리방향(§5 R1):** IMPORT ● 종이를 `t_mat.mat_nm` 매치 → parent `t_prd_product_materials(usage_cd=USAGE.01)` 다행 적재.

### G-BK-5 [MISMATCH] 떡메모지(097) page_rule 3/3/3 + 백모조120 USAGE 중복
