# 가격엔진 fit-gap (round-2, 15 단가시트 전수) + 평면화 파일럿 5시트

가격표 15 단가시트가 `t_prc_*` 6테이블 엔진(component_prices 6차원)에 들어가는지 시트(블록)별로 확정한다.
F2(`05_method/F2-price-sheet-structures.md`)의 "차원 귀속 **후보**"를 본 문서가 "**확정**"으로 승격한다.

[HARD] 권위: dbm-price-formula SKILL 규칙 1~10 · `00_schema/price-engine-ddl.md`(6차원·자연키8·C-1~9·G-1~3).
식별자/컬럼/코드/CSV헤더는 영어, 해석은 한국어. DB 미적재·스키마 무변경.

## 0. fit-gap verdict 분포 (15시트)

| verdict | 수 | 시트 |
|---------|----|------|
| ADEQUATE | 6 | 코팅·디지털인쇄·접지옵션·봉투제작·제본·인쇄후가공 |
| ADEQUATE-WITH-PROPOSALS | 7 | 아크릴·포스터사인·커팅타공·스티커·합판도무송·명함포토카드·엽서북떡메 |
| GAP | 2 | 후가공_박(소형)·후가공_박(대형) — 면적→분류(A~E)→가격 2단 룩업 |

- **ADEQUATE** = 기존 6차원으로 그대로 표현(siz_cd/clr_cd/mat_cd/coat_side_cnt/bdl_qty/min_qty + comp_cd 분리).
- **ADEQUATE-WITH-PROPOSALS** = 표현은 되나 **문서화된 모델링 결정** 필요(주로 ① comp_cd에 옵션/차원 흡수, ② placeholder siz_cd 후니 등록, ③ 면적 좌표 siz_cd화). 침묵 드롭 없음.
- **GAP** = 6차원으로 직접 표현 불가(2단 룩업의 중간키 A~E가 차원 슬롯 부재). 모델링 워크어라운드 또는 에스컬레이션.

## 1. 핵심 모델링 결정 (전 시트 공통 — 근거)

| ID | 결정 | 근거(왜) |

| M-2 | **별색 5종 = clr_cd NULL + 별색인쇄비 comp 분리** | 규칙①·G-1. clr_cd는 0~4도/CMYK 5종뿐(별색 자리 없음). 별색 clr_cd 매핑=FK위반. 별색단가는 `COMP_PRINT_SPOT_*` comp(comp_typ=.01 인쇄비)로 분리, clr_cd=공란→NULL |

## 2. 평면화 파일럿 5시트 (이번 단계 — 매핑+CSV 산출)

### 2-1. 코팅 (`coating`) — ADEQUATE

- **공식 유형**: 원자합산형 공식(엽서/전단 등)의 **구성요소**(코팅비). 계산공식집초안 행8: `(4) 코팅비 = [수량행단가]×[코팅타입열]×출력매수`.
- **comp_cd**: 무광/유광 별 단가 프로파일 상이 → `COMP_COAT_MATTE`/`COMP_COAT_GLOSSY`(comp_typ=**.02 코팅비**).
- **차원 확정**:
  | 엑셀 축 | → 차원 | 근거 |
  |--------|--------|------|
  | 무광/유광 | comp_cd 분리 | 코팅종별 단가 프로파일 상이(M-1). 자재 아닌 코팅비 |
  | 단면/양면 | `coat_side_cnt`(1/2) | 규칙②·DDL §4. 코팅면수 전용 차원 존재 |
  | 수량(출력매수) | `min_qty` | M-6 |
  | 국4절/3절 | `siz_cd`(placeholder) | M-4 |
- **산출**: 184행. 역대조 PASS(B4=2000, C22=260 양면, 상한개방 q1000000=100).
- **FK**: comp_typ.02 OK · coat_side_cnt FK없음(정수) · siz_cd placeholder 2종(후니 대기).

### 2-2. 디지털인쇄비 (`digital-print-price`) — ADEQUATE

- **공식 유형**: 원자합산형의 인쇄비 구성요소. 계산공식집초안 행6~7: `(2)인쇄비(4도)=[수량행단가]×출력매수` + `(3)별색인쇄비`.
- **comp_cd**: 도수(흑백/칼라)=`COMP_PRINT_DIGITAL_S1/S2` + 별색5종=`COMP_PRINT_SPOT_{WHITE,CLEAR,PINK,GOLD,SILVER}_S1/S2`(전부 comp_typ=**.01 인쇄비**).
- **차원 확정**:
  | 엑셀 축 | → 차원 | 근거 |
  |--------|--------|------|
  | 흑백(1도) | `clr_cd=CLR_000002` | 도수=clr 축(흑백=1도) |
  | 칼라(CMYK) | `clr_cd=CLR_000005` | CMYK 4도 |
  | 별색 화이트/클리어/핑크/금/은 | **clr_cd=NULL** + comp 분리 | **M-2/규칙①/G-1: 별색=공정(clr 아님). clr_cd 매핑=FK위반** |
  | 단면/양면 | comp_cd `_S1/_S2` 분리 | M-1·규칙②. 인쇄 단/양면 전용 차원 부재(G-2) → comp 흡수 |
  | 수량(출력매수) | `min_qty` | M-6 |
  | 국4절/3절 | `siz_cd`(placeholder) | M-4 |
- **특이**: 3절 블록(B02)은 흑백/칼라만(별색 없음) — F2 노트 "3절 별색인쇄 상품 없음" 확증.
- **산출**: 954행(국4절 14leaf×53q + 3절 4leaf×53q). 역대조 PASS(칼라양면6000≠8000, 별색 clr=NULL 530행).
- **FK**: comp_typ.01 OK · clr CLR_000002/000005 OK · 별색 clr=NULL · siz placeholder.

### 2-3. 아크릴 (`acrylic-price`) — ADEQUATE-WITH-PROPOSALS

- **공식 유형**: 면적매트릭스형(G-3: 합산형+siz_cd 차원으로 흡수, 신규 FRM_TYPE 불요). 계산공식집초안 행106~108: `[면적매트릭스형: 아크릴]`, `(1)인쇄가공비=소재별 면적[세로mm행][가로mm열]`, 합성=`아크릴+인쇄+레이저커팅`.
- **comp_cd**: 아크릴종별 `COMP_ACRYL_CLEAR3T`/`CLEAR15T`/`MIRROR3T`(comp_typ=.01 인쇄가공비).
- **차원 확정**:
  | 엑셀 축 | → 차원 | 근거 |
  |--------|--------|------|
  | 가로(row2 leaf)×세로(A열) mm | `siz_cd`(면적, placeholder) | 면적=siz(work/cut 치수). **단 196 면적좌표 siz_cd 부재** → PROPOSAL(아래) |
  | 아크릴종(투명3T/1.5T/미러) | comp_cd 분리 | 단가 프로파일 상이 |
  | 도수(양면9도/단면7도/전면5도) | 미반영(블록 title 메모) | "통용 단가"라 도수가 단가에 미영향(시트 명시) — 차원 불요 |
- **수식81(미러=투명×2)**: `=B3*2` 수식이나 **data_only value(5000=투명2500×2)를 그대로 적재**(규칙: L1 value 변조 금지). 파생관계는 note 기록. 미러를 투명참조 동적계산으로 만들지 않음(스키마는 정적 단가).
- **🔴 M-1 정정 (dodge 해소 — 직전 검증 적발)**: 직전 검증이 "196 좌표 중 47종 라이브 실재인데 전수 SIZ_PENDING 발명(dodge)"을 적발. **라이브 권위 정확매칭 재실행으로 정정**:
  | 매칭 | 종 | 처리 | 근거 |
  |------|----|------|------|
  | **DIRECT(WxH 정방향)** | 47 | **라이브 실코드 교체**(SIZ_000336=20x20 등) | siz_nm이 `NNxNN` 정확일치. 35종은 용도없음(범용좌표 안전), 12종은 다른용도(반칼스티커/명함 등)나 면적=치수의미라 재사용 정당 |
  | **REVERSED(HxW 역방향만)** | 21 | **SIZ_PENDING 유지 + 에스컬레이션** | 아크릴`20x60`이 라이브엔 `60x20`만 존재 → 가로/세로 의미축 불일치. **직접입력형이라 섣불리 교체 금지**(검증 "의미축 상이 위험" 정합) |

- **PROPOSAL P-1 (잔여)**: REVERSED 21종 의미축 + NONE 128종 등록방식(좌표 일괄등록 vs 면적함수 모델)은 **여전히 후니 결정**. "직접입력형(사용자 가로/세로)" 특성상 196 고정좌표보다 면적함수가 적합할 수 있음 → 에스컬레이션 유지. 매칭표: `09_load/poster-sign/_acryl_siz_match.csv`.
- **round-1 연계**: B04/B07(수량별 구간할인)은 가격 아닌 **할인율** → round-1 t_dsc 영역(규칙⑨ 재매핑 금지). 평면화 스킵.
- **고정가 분리**: B05 아크릴코롯토·B06 카라비너는 "외주비용 고정가"(계산공식집초안 행113/116 `[고정가형]`) → `t_prd_product_prices` 경로(이번 평면화는 면적블록 B01~B03만).
- **산출**: 358행(투명3T 196 + 투명1.5T 81 + 미러3T 81). 역대조 PASS(20×20=2500, 미러=5000). siz_cd M-1 정정 후 실코드 47종 교체.

## 4. GAP (2시트) — 6차원 직접표현 불가 (박 모델링 제안 §4-3 상술)

### 4-1. 후가공_박(소형) (`foil-small`) — GAP
- **구조 정량(L1 실측)**: 4논리블록. ① 동판비(B01) 가로×세로(80x40)→단가(2D 매트릭스, ADEQUATE). ② 면적→분류(B02/B04): 가로(10~80mm 5축)×세로(10~80mm) → **분류문자 A~E** (매핑 좌표 13종). ③ 분류×수량→가격(B03/B05): 분류 A~E × 수량 18구간(200~) → 가격 90셀.
- **GAP 본질**: 중간키 **분류문자(A~E)가 component_prices 6차원(siz/clr/mat/coat/bdl/min) 슬롯에 없다**. 면적→A~E 룩업과 A~E→가격 룩업이 직렬. "분류군" 축 부재.
- 박종 그룹: 일반박 7종(금유광/은유광/먹유광/청박/적박/동박/펄박, **동일가격 공유**) + 특수박 3종(백박/홀로그램박/트윙클박).

### 4-2. 후가공_박(대형) (`foil-large`) — GAP
- 소형박과 **동형 2단 룩업**. 동판비(B01)는 2D 가격매트릭스(가로×세로→원, 아크릴형 ADEQUATE). 가공비(B03~B05): 면적→분류(A~E)→수량별가격 = 동일 GAP. 박종: 일반박 6종 + 특수박 6종(먹유광/백박/홀로그램/트윙클/적박/녹박).

### 4-3. 🔴 박 GAP 모델링 제안 (억지 평면화 금지 — 사용자 HARD 준수)

**원칙**: 박은 진짜 2단 룩업(면적→A~E→가격)이라 6차원 직접 평면화는 의미 왜곡. **억지 평면화하지 않고** 모델링 워크어라운드 3안 + 에스컬레이션을 제시한다. 침묵 드롭·발명 금지.

**🔴 에스컬레이션(사용자 결정 필요)**: 박 가공비의 면적→분류(A~E/A~I) 중간 룩업표를 **어디에 둘 것인가**가 핵심 미결. 스키마 무변경 원칙상 신규 테이블 불가. 3안 중 택일(또는 동판비만 우선 적재) 필요. 후니 운영상 "박은 분류등급 입력 UI인가 면적입력 UI인가"에 따라 A(등급) vs B(면적)가 갈림 — **비전문가 이해 질문**: "박 주문 시 고객이 ① 사이즈(가로×세로 mm)를 입력하면 시스템이 알아서 가격 매기게 할지, ② A~E 등급을 직접 고르게 할지" 확인 필요. 결정 전 박 가공비 **적재 보류**(BLOCKED-LEGIT, 침묵드롭 아님).
