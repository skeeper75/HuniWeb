## 4. BC-3 해소 — 캘린더 제본 이중표현 판정

라이브 실측으로 COMP_BIND_CAL_WALL vs COMP_BIND_TWINRING 값/구간 비교:

| min_qty | 벽걸이캘린더(CAL_WALL) | 트윈링(TWINRING) | 동일? |
|--:|--:|--:|:--:|
| 1 | 5000 | 4000 | ❌ |
| 4 | 4000 | 3000 | ❌ |
| 10 | 3000 | 2000 | ❌ |
| 30 | (없음) | 1500 | 구간 다름 |
| 50 | 2500 | 1500 | ❌ |
| 70 | (없음) | 1300 | 구간 다름 |
| 100 | 2000 | 1300 | ❌ |
| 1000 | 2000 | 1000 | ❌ |

- **판정: 이중표현 아님.** 캘린더 제본비(CAL_*)와 트윈링(TWINRING)은 **별개 가격 부품** — 값 전건 상이·구간 수 상이(캘린더 6 vs 트윈링 8).
- **권위**: 캘린더 상품 제본비 = **COMP_BIND_CAL_*** (가격표에 "캘린더 제본비" 블록 별도 존재). 캘린더 BOM이 공정상 "트윈링제본"(PROC_000021)으로 적재된 것은 **생산 공정 분류**일 뿐, **가격 부품은 캘린더 전용 comp**가 옳음.
- 함의: 캘린더 상품(벽걸이111·와이드112 등) 공식 배선 시 트윈링 comp가 아니라 **CAL_* comp를 배선해야** 함(공정≠가격부품·BC-3 확정).

## 5. 배선·바인딩 활성화 경로 (7종)

| 검사 | 7종 상태 |
|---|:--:|
| 가격 comp 존재 | ✅ 7/7 |
| 단가행 적재 | ✅ 7/7(각 6행) |
| 값 가격표 정합 | ✅ 42/42 |
| prc_typ 정확 | ✅ 7/7 |
| use_dims 정확 | ✅ 7/7 |
| **공식 배선** | 🔴 0/7(전건 미배선) |
| **상품 바인딩(공식有)** | 🔴 0/7(하드커버책자·캘린더 등 ❌미구성) |

- **결론: comp·단가행·메타 모두 정상 = 데이터 그릇 건강.** 잔여 결함은 **순수 배선+바인딩**(공식설계). 추가 데이터 결함(값 손상·차원 오적재·comp 신설) **없음**.
- 단 "배선+바인딩만 하면 활성화"는 **공식 설계 결정 선행**이 조건(BC-5):
  - 책자4와 동일한 B-FORMULA 미스매치 — 하드커버3은 상품별 제본 1고정(하드커버책자→하드커버무선)이라 [[PRF_BIND_SUM]] A안(제본방식별 공식 분리·1:1 바인딩) 그대로 적용 가능.
  - 캘린더4도 동형(벽걸이→CAL_WALL). 단 캘린더 가격은 제본비만이 아닌 완제품 합산(인쇄+용지+제본) → 캘린더 완제품 공식의 제본 항으로 편입.
  - 싸바리는 상품 미확인(BC-1)이라 바인딩 대상 불명 → 배선만으로 활성화 불가.
