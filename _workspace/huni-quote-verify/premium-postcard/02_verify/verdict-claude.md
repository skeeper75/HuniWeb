## 1. 결함의 정체 — print_opt_cd 축 이중의미 충돌 (F-1) + S2 이중배선 (F-2)

라이브 실측(t_prc_component_prices @ SIZ_000499 국4절):

| comp | 면(의미) | POPT_000001 | POPT_000002 |
|------|------|------|------|
| COMP_PRINT_DIGITAL_S1 (plt_siz키) | **단면** | 흑백단면 B(450/400/350/130) | 칼라단면 D(900/850/800/310) |
| COMP_PRINT_DIGITAL_S2 (siz키) | **양면** | 흑백양면 C(900/800/700/260) | 칼라양면 E(1800/1700/1600/620) |

- 가격표 매트릭스는 **도수(흑백/칼라/별색6) × 면(단/양) × 출력매수** 3D. 라이브는 이를 `print_opt(도수) × comp S1/S2(면)`로 적재 — 값은 전부 verbatim 정합(날조 0·결손 0).
- **그러나 상품 PRD_000016의 print_opt_cd 정의 = 면(POPT_000001=단면 front CMYK, POPT_000002=양면 CMYK·front_colrcnt=CLR_000005 고정=칼라전용).**
- → **같은 코드 POPT_000001이 상품에선 "단면", 가격표 데이터에선 "흑백"** = 축 의미 충돌. 손님이 "단면" 선택 → 흑백 단가 매칭(F-1) + S1(단면)·S2(양면) 둘 다 발화(F-2 이중).

## 2. 교정(데이터만) + 개선 루프 Round-1 증명

칼라전용이므로 도수 고정 → 올바른 모델 = **단일 인쇄비 comp + print_opt=면**:
- POPT_000001(단면) ← 칼라단면 D열(= 현 S1/POPT_000002 값 verbatim 이동)
- POPT_000002(양면) ← 칼라양면 E열(= 현 S2/POPT_000002 값 verbatim 이동)
- 흑백 행 제외(칼라전용), COMP_PRINT_DIGITAL_S2 배선 제거(F-2 해소)

## 3. 잔여 경계 (데이터로 안 닫힘 — 개발자/호출측)

- **F-3 출력매수 환산**: 엔진(pricing.py)에 판걸이수/출력매수/imposition/ceil **전무**(grep 0). `component_subtotal`=단가×qty / 합가÷min_qty×qty(둘 다 qty 선형). 출력매수=ceil(qty/판걸이)=계단함수 → 현 엔진 2모드로 표현 불가. **호출측(앱/위젯)이 주문수량→출력매수 환산해 엔진에 qty=출력매수로 공급**해야 함. (qty=200 그대로 주면 62,000 오산 — 데이터 결함 아님)
- **F-4 work→plate 환산**: 가격 단가행은 plate(국4절 SIZ_000499)에만 키. 손님 work size 선택 시 호출측이 임포지션으로 plate+판걸이수 산출 후 공급.
- **F-5 유광코팅 0행 / F-6 투명엽서 바인딩**: 별도(엽서 인쇄비 핵심 아님). 유광 노출/투명 출시 컨펌 후.
