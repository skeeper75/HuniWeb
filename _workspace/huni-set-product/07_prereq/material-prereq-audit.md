# 셋트 가격 적재 전 기초데이터(자재) 정합 감사 (material-prereq-audit)

생성: §(huni-set-product) 07_prereq · 라이브 Railway DB 읽기전용 SELECT 실측(2026-06-24) · DB 미적재(진단·교정 명세까지) · 권위=상품마스터(260610) booklet-l1·계산공식집 · 사용자 자재 원칙[HARD] 권위

> **결론 한 줄**: 셋트 가격 적재 자체는 **안전**(오염 자재 MAT_000001~004는 가격 단가행 0건 참조). 그러나 ① **출력소재 계층이 데이터상 붕괴**(상위자재 13개 전부 논리삭제·활성 자식 64건 고아), ② **면지 4종이 자재 오염**(권위에서 면지=옵션값인데 자재로 등록), ③ **셋트 표지/면지 배선 12행이 삭제된 자재(FK_DEAD)를 참조**. 상품마스터 적재 원칙 **부분 위반**.

## 0. 감사 범위·자(尺)

- 대상 테이블: `t_mat_materials`(343행/활성 194), `t_prd_product_materials`(활성 756), `t_prc_component_prices`(mat_cd 충전 3,342행).
- 7셋트 = PRD_000072(하드커버책자)·077(레더)·082(하드커버링)·088(레더링바인더)·094(엽서북)·097(떡메모지)·100(포토북).
- 권위: 상품마스터 booklet-l1 컬럼(`내지종이(필수)`·`표지종이(필수)`·`제본(옵션)_면지(옵션)`) + 계산공식집(면지인쇄비 비목).
- 사용자 원칙[HARD]: ① 상위자재→자재 계층(`upr_mat_cd`) ② 자재=실재고 스톡만 ③ 면지·내지·표지=용도(`usage_cd`)이지 자재 아님 ④ 적재 원칙 준수 확인.

## 1. 항목① 출력소재 계층 무결성 — **위반(심각)**

### 1.1 라이브 실측

```sql
-- 활성 자식의 부모 상태 분포
SELECT count(*) FILTER (WHERE p.del_yn='N') AS child_parent_active,
       count(*) FILTER (WHERE p.del_yn='Y') AS child_parent_deleted,
       count(*) FILTER (WHERE p.mat_cd IS NULL) AS child_parent_dangling
FROM t_mat_materials c LEFT JOIN t_mat_materials p ON c.upr_mat_cd=p.mat_cd
WHERE c.del_yn='N';
-- 결과: 13 | 64 | 0
```

| 지표 | 값 | 판정 |
|---|---|---|
| 활성 자재(del_yn=N) | 194 | — |
| 그중 상위자재(upr_mat_cd) 보유 | 77 | — |
| **활성 자식인데 상위자재가 논리삭제됨** | **64** | 🔴 계층 붕괴 |
| dangling FK(상위자재 자체가 없음) | 0 | OK |
| child 보유 root 18개 중 OK | 5 | — |
| child 보유 root 18개 중 **DEAD_ROOT_LIVE_CHILDREN** | **13** | 🔴 |

### 1.2 핵심 결함

- 종이 상위자재(root)인 **아트(MAT_000075)·스노우(085)·몽블랑(103)·백모조(071)·앙상블(094)·랑데뷰(100)·띤또(122)·투명(143)·반투명(146)** 등 13개가 **전부 `del_yn='Y'`(논리삭제)** 상태인데, 그 평량 자식(아트지 100g~300g 등) **64개는 `del_yn='N'`(활성)** 으로 살아 가격/상품에 사용 중.
  - 예: `MAT_000075 아트`(del=Y) ← `MAT_000076 아트지 100g`(del=N) … `MAT_000082 아트지 300g`(del=N). 자식 8개 전부 활성.
- 원칙1(상위자재→자재) 그릇(`upr_mat_cd`)은 **존재하나, 부모 노드가 죽어 계층 탐색이 끊김**. UI/관리에서 "아트지" 묶음으로 조회 불가.
- 추가로 **상위자재 없는 평량 종이 31건**(MAT_000169 레자크줄무늬백색 110g 등): 평량 변형인데 `upr_mat_cd IS NULL` = **root 미연결 고아**.

### 1.3 상품마스터 대조

- booklet-l1 `내지종이(필수)`·`표지종이(필수)`는 평량 단위 실제 종이(몽블랑240·스노우300·백모조120) — 라이브 자식 자재(MAT_000109·092·073)와 1:1 정합.
- 권위는 "상위자재" 추상명을 명시하지 않음(평량 종이 직접 지정). 따라서 root 계층은 라이브가 자체 도입한 그룹핑 — **그 그룹 노드를 삭제한 것이 결함**(자식을 살린 채 부모만 삭제).

→ **판정: 원칙1 위반.** 계층 그릇은 옳으나 데이터 무결성(부모-자식 del_yn 동기) 붕괴. `material-hierarchy-board.csv` 참조.

## 2. 항목② 자재 vs 용도 오염 전수 스윕 — **오염 4건 확정 / 면지류**

### 2.1 용도 그릇은 정상 작동

```sql
SELECT count(*) AS total, count(*) FILTER (WHERE usage_cd IS NULL) AS null_usage
FROM t_prd_product_materials WHERE del_yn='N';
-- 결과: 756 | 0   → usage_cd NOT NULL 제약·전 행 충전. 용도(USAGE) 그릇 정상.
```

- `t_cod_base_codes` USAGE 그룹 = 01내지·02표지·03면지·04간지·05투명커버·06표지타입·07공통. **면지·내지·표지가 base code(용도)로 이미 존재** → 원칙3("면지=용도")이 그릇으로 입증됨.

### 2.2 오염 확정 = 면지 4종

| mat_cd | mat_nm | mat_typ_cd | 치수/평량 | 계층 | 가격참조 | usage 사용 | 판정 |
|---|---|---|---|---|---|---|---|
| MAT_000001 | 화이트면지 | MAT_TYPE.02 | NULL | NULL | 0 | 면지(4상품) | 🔴 용도오염 |
| MAT_000002 | 블랙면지 | MAT_TYPE.02 | NULL | NULL | 0 | 면지(4상품) | 🔴 용도오염 |
| MAT_000003 | 그레이면지 | MAT_TYPE.02 | NULL | NULL | 0 | 면지(4상품) | 🔴 용도오염 |
| MAT_000004 | 인쇄면지 | MAT_TYPE.01 | NULL | NULL | 0 | 면지(2상품) | 🔴 용도오염 |

- **권위 근거**: booklet-l1 `제본(옵션)_면지(옵션)` 컬럼 = 행별 **면지 색상 택1 옵션값**(화이트/블랙/그레이/인쇄). 자재가 아니라 **CPQ 옵션값**. 면지의 비용은 계산공식집 "면지인쇄비" 비목으로 별도 처리(색상은 차원 아님).
- **이중 표현**: 동일 의미(면지)가 ① `mat_nm`("화이트면지") ② `usage_cd`(=면지) ③ 별개 반제품 구성원으로 **3중** 등록됨 = 원칙2·3 위반(용도/의미를 자재로 기록).

### 2.3 오염 아님(false-positive 가드) — 정당한 실 스톡

치수/평량 NULL인 무차원 자재 69건 중 면지 4종 외 **65건은 정당한 스톡**:
- 부자재/악세사리(링·D링·볼체인 3개1팩·우드봉/우드행거·만년스탬프 리필잉크 5cc·천정고리)·실사소재(메쉬·현수막천·PET·PVC·인화지)·사입자재(화이트머그 11온스·미니CD키링)·아크릴 색상/두께·레더/린넨/캔버스/타이벡(MAT_TYPE.05 특수소재) 등.
- 이들은 평량이 무의미하나 **개수/규격 단위로 재고관리 가능한 실재고**. MAT_000242(미색스티커)는 **468개 단가행** 참조 = 가격 깊이 배선된 실 스톡.
- 표지 자재 **MAT_000186(레더 화이트)·MAT_000172(하드커버전용지+무광코팅)·MAT_000250(아트250+무광코팅)** = 권위 `표지종이(필수)`의 실제 출력소재명 → **스톡(정당)**.

→ **판정: 자재 오염 = 면지 4건(MAT_000001~004).** 표지 자재는 정당. `material-pollution-board.csv` 참조(전 194 활성행 분류).

## 3. 항목③ 셋트 구성원 자재 배선 — **혼합(OK 14 / 오염 12 / FK_DEAD 5)**

### 3.1 배선 구조 실측(중요)

- **구성원 반제품(PRD_000073~107)에는 자재가 배선되지 않음** — `t_prd_product_materials`에서 sub_prd_cd로 조회 시 0행.
- 대신 자재는 **셋트 부모(PRD_000072 등 7개)에 직접** 배선됨(usage_cd로 역할 구분). 즉 "셋트 부모 1개 = 표지자재 + 면지자재(들) + 부속(링)" 평면 배선.
- `usage_cd`는 **전 행 충전**(NULL 0건) — 용도 분리 그릇 정상 사용.

### 3.2 7셋트 부모 배선 판정(34행)

| 배선상태 | 행수 | 내용 |
|---|---|---|
| **OK** | 17 | 내지/표지 종이(몽블랑240·스노우300·백모조120·몽블랑130·레더화이트·아트250) + 링/D링 부속 |
| **POLLUTED_REF** | 12 | 면지 4종(MAT_000001~004)을 mat_cd로 직접 참조(072·077·082·088에 분산) |
| **FK_DEAD** | 5 | 삭제된 자재 참조: 전용지(MAT_000246·072/082)·하드커버(005)·소프트커버(007)·레더하드커버(006)·그레이면지(251·포토북) |

- 핵심: 면지 색상이 **셋트 부모에 mat_cd로 박혀 있음**(usage_cd=면지 동반). 권위상 면지는 옵션이므로 이는 오염 참조.
- FK_DEAD 5건: `t_prd_product_materials`가 `del_yn='Y'`인 자재를 가리킴(전용지·표지 추상명·포토북 그레이면지). 표지 종이 그릇(MAT_000172 등)은 살아 있으나 셋트는 죽은 MAT_000246(전용지)을 가리킴.

→ **판정: 셋트 배선 부분 위반.** usage_cd 충전은 OK, 그러나 면지=옵션을 자재로 박고(12) + 삭제 자재 참조(5). `set-member-wiring-board.csv` 참조.

## 4. 항목④ 가격 영향 — **셋트 적재 안전(오염 자재 가격참조 0)**

```sql
-- 오염 면지자재를 mat_cd로 참조하는 단가행
SELECT count(*) FROM t_prc_component_prices WHERE mat_cd IN ('MAT_000001','MAT_000002','MAT_000003','MAT_000004'); -- 0
-- opt_cd로도?
SELECT count(*) FROM t_prc_component_prices WHERE opt_cd IN ('MAT_000001','MAT_000002','MAT_000003','MAT_000004'); -- 0
```

| 점검 | 결과 | 의미 |
|---|---|---|
| 면지 오염자재 4종 → component_prices 참조 | **0행** | 셋트 가격 적재 시 오염 자재가 단가에 안 걸림 → **교정 안전** |
| 셋트 표지/면지 자재(전용지·레더화이트·아트250) → 단가행 | 0행 | 셋트 가격은 면지/표지 **인쇄비 비목**(계산공식집)으로 계산·표지자재 자체는 가격키 아님 |
| 셋트 내지 종이(몽블랑240·스노우300·백모조120·몽블랑130) → 단가행 | 존재(정상) | 가격 배선 정상 |
| **component_prices가 삭제(del_yn=Y) 자재 참조** | **21행** | MAT_000159 모조120g(20)·MAT_000119 리브스250g(1) — 면지 오염과 무관·전 카탈로그 잔존 결함 |

→ **판정: 셋트 가격 적재의 직접 가격 영향 없음.** 다만 전 카탈로그 차원에서 product_materials가 삭제 자재 참조 **185행**(전체 756 중)·component_prices 삭제 자재 참조 21행 = 별도 정리 대상(이번 셋트 적재 차단 사유 아님).

## 5. 상품마스터 적재 원칙 준수 판정

| 원칙 | 그릇 존재 | 데이터 준수 | 판정 |
|---|---|---|---|
| 1. 출력소재 상위자재→자재 계층(upr_mat_cd) | ✅ 존재 | 🔴 부모 13개 삭제·자식 64 고아·root 미연결 31 | **위반(데이터)** |
| 2. 자재=실재고 스톡만 | ✅ | 🔴 면지 4종이 무차원 의미값으로 등록 | **위반(4건)** |
| 3. 면지/내지/표지=용도(usage_cd) | ✅ USAGE 7코드·NOT NULL·충전100% | ⚠️ 용도는 맞으나 면지 색상을 자재로도 이중등록 | **부분 위반** |
| 4. 셋트 배선 정합 | ✅ usage_cd 분리 | 🔴 면지=자재 박힘 12·FK_DEAD 5 | **부분 위반** |

**종합: 그릇(스키마)은 원칙대로 설계됨**(upr_mat_cd 계층 + usage_cd 용도 분리 존재). **데이터 적재가 원칙을 부분 위반** — 면지 옵션을 자재로 등록, 상위자재 노드 삭제, 삭제 자재 참조.

## 6. 교정 우선순위(명세까지 · 실 교정 인간 승인 후 dbmap/basecode 위임)

| 우선 | 결함 | 교정 방향 | 라우팅 | 위험 |
|---|---|---|---|---|
| **P1** | 면지 4종 자재 오염(MAT_000001~004) | 자재 논리삭제 → 셋트 배선 면지는 usage_cd=면지 유지하되 **mat_cd는 실 종이로 교체 or NULL 허용**·면지 색상은 CPQ option_items로 표현 | basecode(자재 마스터)+dbmap(셋트 배선) | 中(가격참조 0=안전·단 셋트 옵션 재설계 동반) |
| **P2** | 셋트 FK_DEAD 5행(전용지·하드/소프트/레더하드커버·포토북 그레이) | 삭제 자재 참조를 **살아있는 표지 종이(MAT_000172 등)로 재배선** or 자재 부활(del_yn=N) | dbmap | 中(셋트 가격공식 표지 비목 의존) |
| **P3** | 계층 붕괴(상위자재 13삭제·자식 64고아·root 미연결 31) | 상위자재 **부활(del_yn=N)** or 자식 upr_mat_cd 재지정·31 평량종이 root 신규/연결 | basecode(자재 마스터 수술) | 高(공유 마스터·전 카탈로그 영향·search-before-mint) |
| **P4** | 전 카탈로그 삭제자재 참조(pm 185·cp 21) | 셋트 범위 밖 일괄 정리 | dbmap 별도 트랙 | 中(이번 적재 비차단) |

**셋트 가격 적재 가부**: 오염 자재가 가격에 0건 걸리므로 **적재는 진행 가능**. 단 P1·P2를 동반 교정하지 않으면 셋트 면지/표지 옵션 UI·배선이 권위와 어긋난 채 적재됨 → **P1·P2를 셋트 적재와 함께 처리 권장**.
