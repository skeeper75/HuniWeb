## 1. 신규 리서치 — 미싱제본 (07_domain 제본 8자식 부재 → 표준 보강)

**갭:** 만년다이어리(소프트커버)·만년다이어리(레더소프트커버)·먼슬리플래너는 **C28 제본사양 빈값**이나, COMMENT에 `*출력+미싱` 명시(미싱제본). 07_domain 제본 8종(중철/무선/PUR/트윈링/떡/하드커버무선/하드커버트윈링/레이플랫)에 **미싱제본 부재**.

**표준 보강(WebSearch — 후니 PDF에 미싱제본 Case 미확인):**
- **미싱제본 = 사철(stitch) 방식.** 접은 종이 가운데 접은선에 미싱기로 **실(색실 선택 가능)** 박음. 소프트커버 노트/다이어리 대표 제본.
- **4배수 페이지** 작업, 80g 기준 40장 최대(중철과 유사한 박음 한계).
- 소프트커버 책=미싱(사철)/떡제본/선제본으로 묶임. → 만년다이어리 소프트커버=미싱제본이 표준 정합.

**후니 권위 우선:** 표준상 미싱제본은 별 제본방식이나, **후니 제본 8자식에 부재**. 두 해석:
- (A) **미싱제본 = PROC_000017 신규 자식**(중철과 별개·실 param). 후니가 운영하나 코드 미등록(준비중 시트 신규 가능).
- (B) 미싱제본 = 후가공(미싱=재봉선 가공)으로 처리, 제본은 별도.
→ **CONFIRM-ST2-5로 인간 컨펌.** 표준을 후니에 묵시 흡수 금지(추정 0).

## 2. 발견한 충돌 — 없음 (오모델 미발견)

엑셀 실측 × 07_domain × booklet 3자 정합. **문구는 의미축이 깨끗**(booklet 정합):
- 내지/표지/면지가 별 컬럼 블록으로 명시 → usage 분리가 엑셀 구조에 내장(평면화 위험 낮음).
- page_rule(C16~18)과 묶음수(떡메모 C28)가 별 축 → 떡제본 잡음 위험 회피(떡메모 묶음수=50/100장1권이 증거).
- 제본(C28)이 명시 enum → 공정 변환 1:1(미싱만 코드 부재).

> 단, **스티커 G-SK-2(형상 size 흡수)·굿즈파우치 size→option 같은 오모델은 문구에 없음.** 문구 시트는 반제품·페이지를 처음부터 별 축으로 분리 설계(booklet 동형).

**검증된 도메인 정합(07_domain 예측 ↔ 엑셀 실측 일치):**
- 트윈링 좌철(노트)/상철(수첩) 방향 차등 ✅ (db-structure §161 방향 enum)
- 떡메모지 묶음수(50/100장1권) 활성·page=장수 ✅
- 하드커버=면지 보유, 일반노트=면지 없음 ✅
- 표지사양 `아트250+무광코팅` 복합표기(자재+공정) ✅ (entity-semantic §23)
- 만년다이어리 4종 130x190 동일 size(커버타입만 분기) ✅

## 3. 🔴/🟡 컨펌 질문 (인간 결정 대기)

### CONFIRM-ST2-1 [🟡] 레더(C25 표지소재) — 자재유형?
- **상황:** 레더하드커버/레더소프트커버 표지사양=`레더`. 종이 아닌 가죽 소재. 표지폴더=특수인쇄.
- **가설:** MAT_TYPE.06 가죽(db-structure §191). 색상(화이트)=variant.
- **질문:** 레더를 MAT_TYPE.06 가죽으로 등록할까요? 색상=variant? (booklet BK-4·포토북 PB-3·실사 일괄)

### CONFIRM-ST2-2 [🟡·핵심] 만년다이어리 커버타입 4종 — 별상품 vs variant?
- **상황:** 소프트/하드/레더하드/레더소프트커버 = 130x190 동일 size, MES 008-0020~0023 **별 코드**, 가격 차등(9000/12000/15000/15000).
- **두 모델:** (A) 별 prd_cd 4상품(booklet 하드커버=별상품 패턴·별 MES 시사) (B) 1상품 만년다이어리 + 표지타입 variant(USAGE.06, 포토북 패턴).
- **가설(유력):** (A) 별상품 — **별 MES 코드 4개**가 별상품 강력 시사(포토북은 1 MES). 다만 size 동일·커버만 차이=variant 여지.
- **질문:** 만년다이어리를 커버타입별 4상품으로 둘까요, 1상품+표지타입 variant로 둘까요? (실무진 Q6 디자인=1상품+에디터템플릿 확정과 다른 축 — 커버타입은 물리 반제품)

### CONFIRM-ST2-3 [🟡] 제본옵션(C27) 분기 — 링컬러 vs 면지?
- **상황:** C27이 두 의미 혼재 — 트윈링류 `실버링`(링)·하드커버류 `하드커버(면지?)`(면지).
- **가설:** 실버링=링 부속(MAT_TYPE.07) 또는 트윈링 param(링컬러)·면지=USAGE.03 자재(D-29).
- **질문:** 실버링을 링 부속 자재(mat_cd)로 둘까요, 트윈링 공정 param(링컬러)으로 둘까요? 면지는 USAGE.03 자재 확정? (booklet BK-3 동류)

### CONFIRM-ST2-4 [🟡] 떡메모지 3축 — page/묶음수/가격표 결합?
- **상황:** 떡메모지=page(C16~18=3)·묶음수(C28=50/100장1권)·가격(`*가격표참고`)이 동시 존재.
- **가설:** page=장수 단위(3장 증가)·묶음수=권 단위(50/100장1권 QTY_UNIT.03)·가격=묶음수×size 매트릭스. 3축 독립.
- **질문:** 떡메모지 page(C16~18 = 3)와 묶음수(50/100장1권)의 관계? page=낱장 매수, 묶음수=권당 장수가 맞나요?

### CONFIRM-ST2-5 [🟡·신규] 미싱제본 — PROC_000017 신규 자식 vs 후가공?
- **상황:** 만년다이어리 소프트/레더소프트·먼슬리=C28 제본 빈값, COMMENT `*출력+미싱`. 후니 제본 8자식에 미싱 부재(§1 리서치).
- **가설:** (A) PROC_000017 신규 자식(미싱제본, 실 param)·(B) 미싱 후가공(재봉선) + 별 제본·(C) 무선/중철 변형.
- **질문:** 미싱제본을 후니 제본 방식으로 신규 등록할까요(준비중 시트라 신규 가능)? 표준상 사철(실박음) 방식입니다.

### CONFIRM-ST2-6 [🔴] C10/C22 파일명약어 · C37 COMMENT — 견적 DB 귀속?
- **상황:** 내지/표지 파일명약어(먼슬리/떡메모)·COMMENT(오렌지=개발무시) = 생산 메타.
- **가설:** 견적 밖(생산/MES). 디지털 DP-2·스티커 ST-5·책자 BK-6·실사 SL-6과 동일. 단 COMMENT `*합지보드`/`*PVC커버`=BOM 힌트로 활용(t_* 귀속은 아님).
- **질문:** (a) 견적 밖 (b) note 보존 (c) 신규 컬럼? (전 시트 일괄)

### CONFIRM-ST2-7 [🟡] 떡메모지 가격(`*가격표참고`) — round-2 적재 위치?
- **상황:** 떡메모지만 inline 고정가 아닌 `*가격표참고`(묶음수×size 매트릭스). 다른 문구는 inline(9000·4500…).
- **가설:** 떡메모지=별도 가격표(묶음수 50/100 × size 90x90/70x120 매트릭스)=round-2 component_prices(siz 차원). 다른 상품=고정가.
- **질문:** 떡메모지 가격표(묶음수×size)를 어디에 적재? inline 고정가와 다른 처리 필요.

### CONFIRM-ST2-8 [🟡] 합지보드·PVC커버·하드보드 — 부속 자재 등록?
- **상황:** COMMENT 힌트 `*합지보드`(스프링/메모 뒷판)·`*PVC커버`(소프트커버)·하드보드(하드커버 싸바리).
- **가설:** MAT_TYPE.07 부속(합지보드·하드보드)·PVC=USAGE.05 투명커버. 표지(USAGE.02) 결합 또는 별 부속.
- **질문:** 합지보드/하드보드/PVC커버를 부속 자재로 등록할까요? 표지 결합인가 별 부속행인가?
