# Huni-Product-Readiness (§29) — 상품별 가격계산 준비도 평가 + 위젯/제약 개발일정

## 목적
이전사이트(huniprinting.com) 상품리스트를 **분모**로, 라이브 DB 상품+가격이 **실제 가격계산 가능 수준**까지 연결됐는지 상품별 세부 평가(진척도)하고, **위젯+제약조건 개발 부분만** 준비도 등급별 묶음 일정으로 분리. 현재 구현 수준 확인 + 프로젝트 일정 체크.

## 디렉토리
- `00_spine/` — 이전사이트×엑셀(260610)×라이브 3자 교차 척추·종이류 판정·미등록 구멍 (hpr-catalog-spine)
- `01_rubric/` — D1~D11 루브릭·L0~L4 등급·위젯 클래스·재사용 증거 맵 (hpr-rubric-curator)
- `02_readiness/` — 상품별 점수표 + 4 리스트(누락/오매핑/판형재처리/가격0) + other-checks (hpr-readiness-evaluator)
- `03_schedule/` — 위젯/제약 개발일정(등급별 wave·선행) (hpr-widget-scheduler)
- `04_codex/` — codex 독립 2차 reconcile (hpr-codex-verifier)
- `05_gate/` — Q1~Q7 게이트 + **최종 점수표 문서**(xlsx 요청 시 시트 빌드) (hpr-scorecard-gate)

## 평가 루브릭 (베스트프랙티스)
- **차원 D1~D11**: 구성요소 요건·공식 바인딩·단가행·차원충전·계산가능성(evaluate_price)·기초마스터·옵션·추가상품템플릿·제약·판형·매핑정합.
- **등급 L0~L4**: 미적재→구성요소만→공식바인딩(불완전)→계산가능→위젯준비.
- **위젯 클래스**: 고정가by-siz·면적입력·셋트조립·옵션캐스케이드·addon템플릿.

## 핵심 결정 (2026-06-30 초기 구성)
1. 새 하네스(평가·일정 overlay)·기존 스코어링/RTM/§21/§26/§13 재사용(중복 채점 금지).
2. 진척도 분모=이전사이트 상품리스트(이전사이트→엑셀→라이브 3자 교차).
3. 위젯/제약 일정=준비도 등급별 묶음 단위.
4. ★[HARD] 판형=종이류 출력소재에만 유효 — 종이류인데 판형 미/오매핑=재처리 대상.
5. 생성≠검증·codex 주장=가설·라이브 읽기전용·평가/계획까지(실 교정·적재 인간 승인 후 §7/§18/§6).
