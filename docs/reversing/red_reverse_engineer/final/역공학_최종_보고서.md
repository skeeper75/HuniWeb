## 1. 프로젝트 개요

레드프린팅(redprinting.co.kr)의 제품 상세 주문 페이지에서 사용되는 minified JavaScript 소스코드를 역공학하여, 사람이 읽을 수 있는 코드로 변환하였다. 모든 단일 문자 변수를 의미 있는 이름으로 리네이밍하고, 비즈니스 로직에 한글 주석을 추가하였다.

### 1.1 역공학 대상

| 파일 | 원본 크기 | Beautified | 역공학 결과 | 등급 |
|------|----------|------------|------------|------|
| widget.js | 450KB (24줄) | 22,747줄 | 앱 코드 5,405줄 역공학 | **A+** (API/SDK), **A** (Components) |
| RedEditorSDK.min.js | 250KB (1줄) | 12,245줄 | 12,630줄 (주석 포함) | **B** |
| widget.css | 28KB | 1,624줄 | (CSS는 beautify만) | - |

### 1.2 방법론

학술 기반 역공학 방법론을 적용하였다:
- **Semantic Anchor 기반 네이밍** (Allamanis et al., 2015) — API 엔드포인트, 필드명, DOM 셀렉터를 앵커로 활용
- **Data Flow 추적** (Reaching Definitions Analysis) — 변수의 정의-사용 체인을 추적하여 목적 추론
- **Framework Pattern Recognition** — Vue 3, Pinia, TanStack Query 패턴 자동 인식
- **Cross-reference Verification** — 기존 HTML 분석 보고서와 교차 검증

## 3. 주요 발견사항

### 3.1 가격 계산 로직

- **API**: `fetchPriceCalculation(orderOptionData)` → POST `/ko/product_price/get_ajax_price_vTmpl`
- **요청 구조**: `{dataJson: {ORD_INFO: [...], PCS_INFO: [...], price_gbn, mb_cust_cod}}`
- **응답**: 공정별 가격 배열 `[{PCS_CD, PRICE, PRICE_VAT, PRICE_MALL}]`
- **가격 구분**: `price_gbn` 값으로 가격 체계 선택 (예: `book2025_price`)
- **고객 등급**: `mb_cust_cod`로 회원/비회원 가격 차등

### 3.2 옵션 의존성 트리

```
규격 (sizes) ──→ 작업사이즈 자동 계산 (CUT→WRK +10mm 도련)
    │
    ├──→ 내지 용지 (paper/weight) ──→ 내지 자재코드 (INN_MTRL_CD)
    ├──→ 표지 용지 (paper/weight) ──→ 표지 자재코드 (CVR_MTRL_CD)
    ├──→ 수량 (QTY) ──→ 인쇄수량 (PRN_CNT)
    │
    └──→ 후가공 (PCS_INFO[])
         ├── 재단 (CUT_DFT)
         ├── 제본 (PER_DFT) ──→ 좌철/상철
         ├── 코팅 (COT_DFT) ──→ 무광/유광/무코팅
         └── 제본방향 (BIND_DIRECTION)
```

## 6. 제한사항 및 후속 작업

### 6.1 제한사항

1. **Editor SDK 벤더 코드 미처리**: Sentry v5.22.0 (2,468줄)과 Babel polyfill (6,640줄)은 의도적으로 역공학하지 않았다. 공개 오픈소스 라이브러리이므로 원본 소스를 직접 참조하는 것이 효율적이다.
2. **Vue 렌더 헬퍼 유지**: `g()`, `V()`, `M()`, `S()` 등 Vue 3 내부 렌더 함수는 리네이밍하면 코드가 동작하지 않으므로 주석으로만 매핑을 제공하였다.
3. **동적 코드 미분석**: `eval()`, `new Function()` 등 동적 코드 생성은 발견되지 않았으나, 런타임 Pinia 스토어 상태는 정적 분석만으로 완전히 파악할 수 없다.

### 6.2 권장 후속 작업

1. **런타임 검증**: 브라우저 DevTools에서 역공학 결과를 Pinia 스토어 실시간 상태와 대조 확인
2. **API 샘플링**: 다양한 제품 코드(PRBKYPR, PRBKORD 등)로 가격 API를 호출하여 응답 구조 완전 매핑
3. **후니프린팅 적용**: 역공학 결과를 기반으로 후니프린팅 가격 엔진 및 admin 화면 설계 진행 (→ `red-reverse-plan` 스킬 연계)
