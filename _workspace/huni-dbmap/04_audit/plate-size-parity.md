# 판형사이즈(plate_size) 속성 연결 정합 — round-3 Phase 4 (9속성 마지막)

권위: **엑셀=원천**, DB 적재값이 엑셀과 정합하는지 검증. DB 재조회 없음(`00_schema/` 추출본). EXTRA 삭제금지(플래그+출처). JOIN KEY=`prd_nm`(prd_cd→`ref-products.csv`, siz_cd→`ref-sizes.csv`). 분모=245 공통.

근거: `00_schema/ref-product-plate-sizes.csv`(509행, siz_cd·output_paper_typ_cd·output_file_typ)·`ref-sizes.csv` / `04_audit/excel-plate-size.csv`(54상품 값보유) + 가격표 `판걸이수`·`출력소재(IMPORT)` 브리지. 불일치: `plate-size-mismatches.csv`.

## ⚠️ 핵심 발견 — 의미 축 불일치 (전지 vs 작업사이즈)
브리지(판걸이수 시트) 대조 결과, **DB와 엑셀이 "판형사이즈"라는 같은 이름으로 서로 다른 것을 담고 있음**:
- **엑셀 `출력용지규격`** = **전지**(press sheet): `316x467`·`330x470`·`330x660`·`315x467` 등. 디지털인쇄 판걸이용 전지.
- **DB `t_prd_product_plate_sizes.siz_cd`** = **작업/출력 사이즈**(per-item work size, bleed 포함): `102x152`(=cut 100x150 + 블리드)·`88x54`(명함)·`112x172` 등. `output_file_typ`(PDF/JPG/AI)와 짝 → 출력파일 규격.

근거(실측):
- DB plate siz_nm **80건이 판걸이수 시트 `작업사이즈`(D열)와 일치**(102x152·105x148·114x84 등). cut 사이즈 + 블리드 형태.
- 예: 프리미엄엽서 DB plate `[100x100,102x152,112x172,137x137,150x212]` = 작업사이즈, 엑셀 plate `316x467` = 전지. **overlap 0**.
- 예: 코팅명함 DB `[88x54,92x52]`(작업) vs 엑셀 `316x467`(전지). 명함 DB plate = 명함 작업사이즈(cut 86x52+블리드).
- **엑셀 전지값(316x467·330x470·330x660)은 t_siz 마스터에 아예 없음** → DB는 전지를 siz로 저장하지 않음.

→ 단순 4분류 대조 시 51상품이 거짓 MISMATCH로 보이나, 이는 **행값 충돌이 아니라 속성 정의 차이**. 단정 금지, 의미불일치로 정직 분류.

## 전지/판걸이수 브리지 해소 규모
- **판걸이수 시트**: A(사이즈옵션명)·B(재단)·D(작업사이즈)·F(판걸이수)·G(전지). 전지 G는 디지털인쇄 전부 `316x467` 단일.
- DB plate siz_nm **80/298이 작업사이즈(D)와 정규화 일치** → DB plate = 작업사이즈 축임을 브리지로 확증.
- 엑셀 전지(G축)는 DB에 미저장. 전지↔작업사이즈는 판걸이수 시트로 연결 가능하나 **DB는 전지축을 별도 보유 안 함** → 전지 정보 적재 시 판형/판걸이 별도 모델 필요(확인 필요).

## output_paper_typ_cd 정합
- DB: `OUTPUT_PAPER_TYPE.03`(기타)만 107행 사용. `.01 국전계열`·`.02 46계열` **미사용**.
- 엑셀 전지: `316x467`(국전계열=국4절), `330x470`(46전지계열) 등 계열 구분 있음. → DB가 전지 계열을 `.03 기타`로만 적재 = **OUTPUT_PAPER_TYPE 미세분류**(국전/46 미구분). 코드 자체는 유효 FK이나 분류 활용 안 됨. **확인 필요**(전지 계열 구분 적재 의도).
- output_file_typ는 PDF/JPG/AI 등 출력파일형 — DB plate가 출력사이즈축임을 재확인.

## 명세

### 의미불일치 51 (엑셀 전지 vs DB 작업사이즈)
공통 54 중 51상품에서 엑셀 전지(316x467 등)와 DB 작업사이즈가 축 상이로 비교 불가. 예: 프리미엄엽서·코팅명함·라벨/택·캘린더·접지카드 등. **행값 충돌 아님 — 속성 정의 차이**.

### MATCH 3 (A규격 우연일치)
- 낱장 자유형 스티커·낱장 자유형 투명스티커(`210x297·297x420·420x594`)·대형 자유형 스티커(`400x600`): 엑셀이 전지 대신 A규격을 판형으로 기재 → DB 작업사이즈(대형 스티커는 A규격=작업)와 우연 일치.

### EXTRA 150 (DB-only, 삭제금지)
- DB plate 보유하나 엑셀 전지컬럼 공란인 150상품(PRD_TYPE.03 굿즈 93 + .04 디자인 57). 굿즈/파우치는 엑셀에 출력용지규격 미기재이나 DB가 작업/출력사이즈를 plate로 적재. **출처=DB, 삭제 단정 금지**.

## 판정: **CONDITIONAL — 의미축 정의 확인 선행 필요**
- 단순 행대조로는 MISSING 0·EXTRA 150이나, **본질은 엑셀(전지)과 DB(작업사이즈)의 속성 정의 불일치**. 행 충돌(MISMATCH)로 단정 불가.
- DB plate 자체는 작업사이즈축으로 내부 정합(509행, output_file_typ 짝). 엑셀 전지를 같은 테이블에 매핑할지/별도 판형·전지 모델을 둘지 **정의 확정 선행 필요** → BLOCK 아닌 CONDITIONAL.

## 후니 확인항목
1. **`t_prd_product_plate_sizes` 의미 확정(최우선)**: 이 테이블이 (a)출력 작업사이즈인지 (b)전지(판걸이용)인지. 현 DB=작업사이즈, 엑셀 출력용지규격=전지 → 둘 다 필요하면 전지축 별도 컬럼/테이블 필요.
2. **전지 마스터 부재**: 316x467·330x470·330x660이 t_siz에 없음 → 전지를 siz로 적재할지, 판걸이수 시트를 전지 마스터로 별도 적재할지 확인.
3. **output_paper_typ_cd 미세분류**: 전지 계열(국전 .01/46 .02)을 `.03 기타` 대신 구분 적재할지 확인.
4. EXTRA 150 굿즈/파우치 작업사이즈가 plate로 적재된 의도 확인(삭제금지).
5. 사이즈 표기차 브리지(판걸이수 A↔B↔D)는 사이즈 속성과 동일 적용 — 작업사이즈 대조 시 정규화 일관.
