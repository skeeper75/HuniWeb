## 2. ★공통 선결 블로커 (한 셋트가 아니라 전체에 걸림 — piecemeal 함정의 핵심)

### B-1. 가격엔진 코드 결함 2건 (webadmin·전 책자 공통·후니 개발팀 트랙)
1. **DBLPANSU (이중 ÷pansu)**: 뷰 derive_inner_sheets가 ÷pansu한 총판수를 _evaluate_formula가 plate_qty로 또 ÷pansu → 내지 ~0.4배 과소. 정석=price_views.py:1707 `derive_inner_sheets→copies*pages` 1줄. 데이터로는 over-mint 우회뿐(열위). → **072/077/082 내지 공통**.
2. **S1/S2 내지인쇄 이중합산**: 양면 주문 시 S1·S2 단가행이 NON_QTY_DIM 동일하게 둘 다 매칭 → 배타선택 부재 → S1+S2 합산. print_opt_cd는 색상축(흑백/칼라)이지 면축 아님. → **094도 가진 결함(R-3)·전 책자 공통**. 해법=면별 PRF 분리 or only_comps or side 차원.
- ★둘 다 webadmin 코드 직접수정 금지 → §6/dbmap 코드 트랙·인간 승인·배포. **이 2건이 풀려야 책자류 전체가 정확 청구.**

### B-2. 기초데이터 정정 (축1·dbmap/basedata 트랙)

### B-3. 가격테이블 전수 적재정합 (축2·미완 확인)

## 5. set-products webadmin 사용방법 (축3·코드 숙지)

- URL `/admin/set-products/` → 좌측 셋트 목록 / 우측 iframe 변경폼 + "셋트 구성" 인라인.
- 인라인 = `ProductSetsInline`(admin.py:1041~1054·autocomplete sub_prd_cd·PRD_TYPE.02만 후보)·뷰 `views.py:622~663`·템플릿 `templates/catalog/set_products.html`.
- 구성원 추가 = 하위상품(반제품 택)·min/max/incr·역할(semi_role_cd는 products에·드롭다운 라벨 `[역할]`). disp_seq 순서.
- 가격 = `evaluate_set_price`(pricing.py:718) 구성원별 자기공식 + 셋트 본체 공식(제본) 합산. 시뮬 `price_simulate_set`(price_views.py:1649).
- ★적재 후 확인 = price_simulate_set 호출로 evaluate_set_price 골든 재현(코드결함 해소 후라야 정확).
