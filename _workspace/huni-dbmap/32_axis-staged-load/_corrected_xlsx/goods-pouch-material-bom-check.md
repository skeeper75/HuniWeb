## 0. 핵심 결정 사실 (라이브 실측 · 재현 SELECT)

### 사실 ① — 파우치 자재축(`.09`) 74행에 진짜 소재명이 0행

```sql
SELECT mat_cd, mat_nm FROM t_mat_materials WHERE mat_typ_cd='MAT_TYPE.09' ORDER BY mat_cd;
-- 74행 전부: 원형 90mm/사각 110mm/11온스/350ml/단면/양면/2구/화이트 M/청보라/꽃/마카롱 ...
-- 레더·우드·규조토·벨벳·린넨·코르크·캔버스·타이벡·메쉬·세라믹·유리·PVC = 단 1행도 없음
```

`.09 파우치` 자재유형 74행은 **전부 형상·치수·색·인쇄면·구수**다. 본체 소재명은 하나도 없다. → 어떤 상품이 `.09` 행에 연결돼 있어도 그것은 **본체 소재 BOM이 아니다.**

### 사실 ② — 진짜 본체 소재 자재행은 라이브에 **존재하지만 굿즈파우치 상품에 미연결(고아)**

```sql
-- 원단(.05)·가죽(.06) = 진짜 본체 소재 축
SELECT m.mat_cd,m.mat_nm,COALESCE(p.prd_nm,'(연결없음)')
FROM t_mat_materials m
LEFT JOIN t_prd_product_materials pm ON pm.mat_cd=m.mat_cd
LEFT JOIN t_prd_products p ON p.prd_cd=pm.prd_cd
WHERE m.mat_typ_cd IN ('MAT_TYPE.05','MAT_TYPE.06');
```

| mat_cd | mat_nm | typ | 라이브 연결 상품 | 판정 |
|--------|--------|-----|------------------|------|
| MAT_000061~067 | 파우치 / 파우치 블랙·화이트·반투명·투명·멜란지·청보라 | .05 원단 | **(전부 연결없음)** | 본래 파우치 본체소재(색 합성)로 의도된 행 — **고아** |
| MAT_000183 | 메쉬 | .05 원단 | 메쉬배너·메쉬프린트·메쉬현수막 (실사) | 굿즈 메쉬파우치/메쉬에코백 **미연결** |
| MAT_000184 | 린넨 | .05 원단 | 린넨패브릭포스터·린넨 우드봉 족자 (실사) | 굿즈 린넨파우치/린넨에코백 **미연결** |

### 사실 ③ — `MAT_TYPE` 코드 라벨(권위 base_codes)

```sql
SELECT cod_cd,cod_nm FROM t_cod_base_codes WHERE cod_cd LIKE 'MAT_TYPE.%';
```
`.05=원단 · .06=가죽 · .07=부속 · .08=실사소재 · .09=파우치 · .10=악세사리 · .12=굿즈`

→ `.05 원단`·`.06 가죽`이 굿즈파우치 본체 소재가 들어갈 정답 축. `.09 파우치`는 후니가 "파우치 옵션값 모음" 버킷으로 쓴 오염 축(형상/색/사이즈)이다.

## 1. 상품별 본체 자재 BOM 확인 판정표 (distinct 103상품)

판정 기준:
- ✅ **확인됨** — 라이브 연결 자재 = 진짜 본체 소재(권위 정합).
- 🟡 **오염행에만 연결** — `.09` 형상/색/사이즈 또는 `.08/.01/.12` 표면값에만 연결, 본체 소재 불명.
- ❌ **미연결** — `product_materials` 0행.

### 1-A. 연결 자재 분류 집계 (라이브 매칭 98상품 · 재현 SELECT)

  FROM t_prd_products p
  LEFT JOIN t_prd_product_materials pm ON pm.prd_cd=p.prd_cd

## 2. 근본 원인 (적재 로직 + v03 진원)

`load_master.py`는 무변환 전파기다(03 판정). 따라서 라이브 상태 = **v03 엑셀이 인코딩한 그대로**다.

1. **상품마스터 굿즈파우치 시트에 "본체 소재" 컬럼이 없다.** 시트 컬럼은 `상품명·상품(옵션)·파일사양·주문방법·선택(옵션)·가공·제작수량·추가상품`뿐 — 소재 명시 칼럼 부재. 소재는 **상품명에만 박혀 있다**(레더코스터·캔버스파우치). load_master가 변환할 소재 셀이 원천에 없으므로 `product_materials`에 본체 소재를 넣을 입력이 없었다.
2. **`상품(옵션)` 셀값("M"·"원형 90mm"·"단면"·"청보라")이 `.09 파우치` 자재행으로 무변환 적재됐다.** v03이 옵션값을 자재 슬롯에 인코딩 → load_master가 그대로 INSERT → 라이브 `.09` 74행 = 옵션값 버킷. 이것이 🟡(오염행 연결)의 진원.
3. **진짜 소재 자재행(.05 원단·.06 가죽)은 실사·책자 상품용으로만 연결됐고 굿즈파우치엔 재배선 안 됨.** `MAT_000061~067`(파우치 색 합성행)은 의도는 있었으나 고아로 방치 = 본체 소재 슬롯 부재의 결과.

## 3. B-3 분류와의 관계 — "80/82 BOM 의존 경고" 재확인

round-22 B-3(`material-nonmaterial-cpq-spec.md`)는 `.08/.09/.10`의 **비소재 오염행을 어느 축으로 옮길지**(색→option·형상→siz·구수→bundle·인쇄면→print_side)까지였다. 본 진단이 추가하는 결정적 사실:

- **오염 행을 전부 제거(use_yn='N')하면 파우치 본체 소재는 0이 남는다.** `.09` 74행이 전부 옵션값이고 진짜 소재행이 없으므로, B-3가 색/형상/사이즈를 정확 축으로 옮긴 **뒤에는 본체 자재가 통째로 비게 된다.** = B-3 §1-B "파우치 본체 소재는 상품명이 곧 상품이므로 자재 분기 불필요"라는 가정은 **DB BOM 관점에서는 GAP**이다. 위젯/MES BOM이 본체 소재를 필요로 하면 별도 신규 적재가 필요.
- **84상품 144 link 재배선 경고가 그대로 유효** — 본 진단으로 그 144 link가 **전부 비소재(형상/색/사이즈)**이고 본체 소재 link는 0임이 확정됐다. 따라서 재배선 = "비소재 link 제거 + **본체 소재 link 신규 생성**"의 2단계여야 한다(B-3는 1단계만 다룸).
