## 1. MAP 07 캘린더 권위 구조 ↔ 라이브 1:1 대조 (★비동형 진단)

사용자 제시 MAP 07 캘린더 시트 (원본 이미지 = 절대 권위):

```
07 캘린더
▶︎탁상형캘린더   → 탁상형캘린더 · 미니탁상형캘린더 · 엽서캘린더   (상품)
▶︎벽걸이캘린더   → 벽걸이캘린더 · 와이드벽걸이캘린더              (상품)

▶︎디자인캘린더   → 탁상형캘린더 · 엽서캘린더 · 벽걸이캘린더        (상품)

사용자 명시: "캘린더 → ▶︎[하위분류] → 텍스트는 **상품**임."
→ MAP 모델 = **L1(캘린더) > L2(▶︎하위분류) > 상품(junction 귀속)**. ▶︎ 아래 텍스트는 전부 상품이지 더 깊은
카테고리 노드가 아니다. **모든 하위분류가 동일 깊이(L2)·동일 모델**이어야 한다.

  → MAP 모델("L2 > 상품")과 **동형**.
- ▶︎디자인캘린더(118)만 **L2 노드 아래 또 L3 카테고리 노드(318 탁상형캘린더·319 벽걸이캘린더·320 엽서캘린더)**.

  만든 것**. MAP 모델과 **비동형 = 오모델**.

```

## 3. 올바른 목표 구조 (MAP 권위 동형)

MAP 모델("캘린더 > ▶︎하위분류(L2) > 상품")에 맞춰 디자인캘린더를 다른 하위분류와 **동형**으로 정정:

```
CAT_000007 캘린더 (L1)
├─ CAT_000112 ▶︎탁상형캘린더 (L2)  ──junction── PRD_108(main=Y후보)·109·110     [현행 정상]
├─ CAT_000115 ▶︎벽걸이캘린더 (L2)  ──junction── PRD_111(main=Y후보)·112          [현행 정상]
├─ CAT_000317 ▶︎부자재/캘린더액세서리 (L2)                                        [별 검토]
└─ CAT_000118 ▶︎디자인캘린더 (L2)  ──junction── PRD_108·PRD_110·PRD_111 (main='N') [★정정 목표]
      └─ (318/319/320 L3 노드 제거 — 상품을 노드로 오모델한 것)
```

목표 = **CAT_000118 아래에 L3 노드를 두지 않고, 상품 PRD_108/110/111을 CAT_000118에 직접 junction 귀속**
(▶︎탁상형/▶︎벽걸이와 동일 패턴). 이 상품들은 이미 112/115에 main 분류돼 있으므로 디자인캘린더 귀속은
**다중분류(main_cat_yn='N')**.

junction PK = **(prd_cd, cat_cd) 복합**(라이브 information_schema 실측). 따라서:
- PRD_000108 → CAT_000112 (기존) + PRD_000108 → CAT_000118 (신규) = **서로 다른 PK 2행·합법 다중분류**.
- 가격 비종속(pd=N·junction에 가격 컬럼 없음).

## 5. round-24 D-1 dedupe 재평가 — ★MAP 권위와 충돌·round-24 해석이 틀림

round-24 `06_product-category-mapping.md` 33행 원문:
> "**D-1 캘린더 동일셀 중복 3행 dedupe**(PRD_000108/110/111이 각각 MAP 2셀 G3+G17·G5+G18·G8+G19에
> **중복 등재** → junction **1행으로 통합**)."

**진단: round-24 D-1은 카테고리 무시(category-blind) dedupe 오류다.**

- round-24는 G3(탁상형캘린더 하위)와 G17(디자인캘린더 하위)를 "동일셀 중복"으로 보고 1행 통합.
- 그러나 **G3와 G17은 중복이 아니다**: G3 → 목표 cat=CAT_000112(탁상형캘린더), G17 → 목표 cat=CAT_000118
  (디자인캘린더). **다른 카테고리**다.
- junction PK = **(prd_cd, cat_cd) 복합**(실측 확인). PRD_108→112 와 PRD_108→118 은 **서로 다른 PK·합법 2행**.
  PK 충돌이 아니다.
- round-24 D-1("(prd_cd,cat_cd) PK 충돌 0·prd_cd당 main='Y' ≤1") 규칙 자체는 옳으나, **적용을 오판**했다:
  같은 prd_cd가 두 *다른* cat_cd에 등재된 것을 "중복"으로 잘못 보고 통합 → **MAP 권위의 다중분류 의도를 소실**.

→ **round-24 해석이 틀렸다(MAP 권위 우선).** D-1이 dedupe한 PRD_108/110/111의 디자인캘린더 귀속은
**버려선 안 되는 정당 다중분류**였다. v1이 이 dedupe를 "권위의 부결"로 채택한 것도 함께 정정 — round-24의
오류를 권위로 오인한 것이다. (round-24 본 트랙에 D-1 정정 escalate 권고.)

## 6. 사용자 승인용 적재 명세 (가역·무손실 · DB 직접 쓰기 0·설계까지)

### (가) 노드 처리 — 318/319/320 논리삭제 (이 하네스 범위)

```sql
-- 가역·무손실: 노드 논리삭제(물리삭제 0). 상품을 노드로 오모델한 L3 3건 제거. 118은 유지.
UPDATE t_cat_categories
SET del_yn='Y', use_yn='N', upd_dt=now()
WHERE cat_cd IN ('CAT_000318','CAT_000319','CAT_000320')
  AND del_yn='N';   -- 멱등 가드(이미 Y면 no-op)
-- 영향: 노드 3행. junction 무관(318/319/320 귀속 0=고아 발생 0). 가격 무관(pd=N).
-- undo: del_yn='N', use_yn='Y' 토글.
```
멱등·가역. 318/319/320에 귀속 상품 0이라 **junction 고아 발생 0**. CAT_000118 보존.

### (나) 상품 귀속 — PRD_108/110/111 → CAT_000118 다중분류 (round-24 트랙 위임)

```sql
-- 다중분류 append(main='N'). PK=(prd_cd,cat_cd) 복합이라 기존 112/115 귀속과 충돌 0.
INSERT INTO t_prd_product_categories (prd_cd, cat_cd, main_cat_yn, reg_dt)
VALUES
  ('PRD_000108','CAT_000118','N', now()),
  ('PRD_000110','CAT_000118','N', now()),
  ('PRD_000111','CAT_000118','N', now())
ON CONFLICT (prd_cd, cat_cd) DO NOTHING;   -- 멱등
-- 영향: junction +3행. PK 충돌 0(다른 cat_cd). main='Y' 단일성 위반 0(전부 N).
-- undo: DELETE WHERE (prd_cd,cat_cd) IN (...) AND main_cat_yn='N'.
```
★(나)는 **round-24 dbm-category 영역**(상품 귀속). 본 하네스는 명세만 제공·실행은 round-24 트랙 + 인간 승인.
순서: (가)와 (나)는 독립(318/319/320 ≠ 118). 동시/개별 모두 안전. **둘 다 가역·멱등·가격무관.**
