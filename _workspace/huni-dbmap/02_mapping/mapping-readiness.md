# 가격 매핑 준비도 평가 (round-2)

"DB 구조 + 상품마스터 계산공식집을 기반으로 매핑할 정보가 충분히 수집되었는가"에 대한 종합 평가. 권위=라이브 DB DDL + `계산공식집초안`(상품마스터). 식별자/컬럼/코드는 영어.

## 0. 한줄 결론

**정보 수집 ≈ 90% 완료, 매핑 설계 착수 가능.** 단 실제 적재 전 **모델링 결정 7건**이 선행돼야 한다(미결정 시 적재거부/데이터손실). 권위 자료(DB 구조·공정 도메인·27 공식블록·19 가격시트)는 모두 확보됐고, 남은 것은 "수집"이 아니라 "결정과 설계".

## 2. 계산공식집 27블록 × 매핑 준비도

유형 분포: 원자합산형 10 · 고정가형 15 · 면적매트릭스형 2.

| 유형 | 블록수 | 대표상품 / 참조시트 | DB 매핑 경로 | 준비도 |
|------|--------|---------------------|--------------|--------|
| **원자합산형** | 10 | 엽서·전단·리플렛·책자·캘린더 / 디지털인쇄비·코팅·출력소재·인쇄후가공·제본·접지옵션·커팅타공·박 | FRM_TYPE.01 + formula_components + component_prices(.01~.05) | 🟢 정보충분 (결정 후 매핑가능) |
| **고정가형** | 15 | 명함·스티커·봉투·문구·굿즈·상품액세서리 / 명함포토카드·스티커·봉투제작·(가격포함)시트 | t_prd_product_prices(합가) 또는 단일 comp | 🟡 variant 결정 필요 |
| **면적매트릭스형** | 2 | 실사·현수막 / 아크릴 | component_prices(siz_cd) + 공식외부 곱셈 | 🔴 추가설계 필요 |

블록별 참조시트가 모두 식별돼 **"상품군 → 공식 → 참조 가격시트(구성요소)" 연결고리는 완성**. 즉 매핑의 척추는 섰다.

## 3. 확정된 매핑 규칙 (수집·검증 완료)

1. **별색 = 공정** (clr 아님). 선택=`t_prd_product_processes`(PROC_000007~012), 단가=별도 comp_cd(인쇄비.01, 화이트=클리어/핑크=금=은 2프로파일). clr_cd 매핑 금지(FK위반).
2. **박 = 공정** (`PROC_000033`+17자식), 박단가=박형압비(.05) comp_cd.
3. **단가 → `t_prc_component_prices`** (구성요소 다차원 단가, 분해 저장).
4. **합가 → `t_prd_product_prices`** (상품단가=합가 테이블). 한 시트에 단가와 합가가 공존하면 **합가는 분해·재합산하지 말고 합가 그대로 상품단가로 반영**(이중원천 불일치 방지). 적용 시트: 커팅타공("인쇄비+소재+커팅"), 포스터사인("코팅포함가"), 인쇄후가공("(합가)"), 제본("삼각대 포함"), 스티커(소재+코팅 결합). [단, 합가가 사이즈/수량별로 변하면 §4-③ variant 결정에 종속]
5. **적재 컬럼명 = 라이브 DDL** (frm_typ_cd/comp_typ_cd/unit_price). 최신 스펙 260603 정합.
6. **수량행 = 출력매수**(주문수량 아님). 주문수량→출력매수 변환(÷판걸이수)은 공식단계, 단가행 baked 금지. `판걸이수` 시트=변환계수(비가격).
7. **비가격 시트 분리**: 판걸이수(변환계수), 굿즈파우치구간할인(t_dsc round-1), 박백업(중복 제외).

## 4. 모델링 결정 7건 — 전건 확정 (2026-06-04)

| # | 결정 | 확정 내용 |
|---|------|-----------|
| ① 별색 단가 | **각 시트 단가 그대로** 사용. 별색=공정(PROC_000007~012)으로 상품에 부여, 단가는 `디지털인쇄비` 시트 F~O 값을 별색인쇄비 comp_cd로 적재. 시트가 화이트=클리어/핑크=금=은으로 동일하면 그대로 반영. clr_cd 사용 금지 |
| ② 인쇄 단/양면 | **각 시트의 단면·양면 단가를 각각 그대로 저장**. 양면=단면×2 계산 금지(양면≠2배 입증). 저장축: 코팅 단/양면=coat_side_cnt, 인쇄 단/양면=별도 comp_cd(단면인쇄비/양면인쇄비) — 파일럿서 최종화 |
| ③ variant 가격 | **`t_prc_component_prices` 차원으로** 저장. 사이즈 variant(손거울 S/M/L)→siz_cd, 색상/재질 variant(머그 화이트/반투명/투명)→mat_cd. 단일가 상품만 `t_prd_product_prices`. 스키마 변경 0, `t_prd_product_sizes`(444링크) 활용. variant 상품은 FRM_TYPE.02 단순형 경유 |
| ④ 면적형 곱셈/할인 | **곱셈·수량할인은 공식 외부(적용단)** 처리, 매트릭스 셀은 룩업 적재. addtn_yn=합산 플래그뿐이라 필연 |
| ⑤ siz_cd FK(면적) | **기존 `t_siz_sizes`(497, mm 사이즈 대량 보유) 활용 + 면적매트릭스 부족분만 추가**. "수천 신규" 아님 |
| ⑥ 이중 가격원천 | **합가 우선** 규칙 적용(§3-④). 단가와 합가 공존 시 합가를 권위로 |
| ⑦ apply_ymd | **변경된 가격을 적용할 일자**(가격 효력일). NOT NULL `'yyyy-MM-dd'`. go-live 확정 시 실제 일자 일괄 주입, round-1과 형식 정합 |
