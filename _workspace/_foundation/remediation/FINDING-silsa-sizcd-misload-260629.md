## 2. 엔진 매칭 메커니즘 (왜 0인가 — pricing.py 근거)

- `_row_matches`(pricing.py:94): 단가행의 **NON_QTY_DIMS 전부**(`siz_cd, plt_siz_cd, print_opt_cd, mat_cd, proc_cd, opt_cd, siz_width, siz_height`)를 매칭 조건으로 본다. 행 값이 **NULL 이면 와일드카드**(line 99), 값이 있으면 선택값과 일치해야 함.
- silsa 본체 단가행: `siz_cd` 외 NON_QTY 차원이 **전부 NULL**(라이브 `IS NULL` 검증 완료) → `siz_cd` 가 **유일한 매칭 키**.
- 옵션 siz_cd(정본 174…) ≠ 단가행 siz_cd(중복본 315…) → no_match → subtotal=0 → final_price=0.
- **캔버스133 본체** comp `use_dims=[siz_width,siz_height,min_qty]` 이지만 단가행은 siz_cd 만 값 보유(w/h NULL=catch-all). `use_dims` 는 TIER 선택용일 뿐, NON_QTY 매칭은 행 값 있는 차원(=siz_cd)으로 수행 → 동일하게 siz_cd 재키잉으로 해소(검증쿼리 OK).
