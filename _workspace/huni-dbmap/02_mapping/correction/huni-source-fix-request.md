# 후니 원천 데이터 정정 요청 — SIZE_NAME_NOISE

SIZE_NAME_NOISE 81건 정정 중 후니 확인/등록이 필요한 항목을 정리한다. 명칭 파싱(치수·묶음수 분해)은 81건 전부 우리측 완료(파싱불가 0).

> **[v2 권위 정정 — validator 라이브 검증 반영]** 라이브 `t_siz_sizes`의 work/cut 치수(`numeric(8,2)`)는 **이미 77/77 채워져 있다**(후니 D-C 등록분 반영). 따라서 **치수 적재(UPDATE)는 불요**. 1단계가 본 `ref-sizes.csv`는 stale 추출본이었다. **실제 우리측 정정 산출 = 묶음수 18행**(`load/t_prd_product_bundle_qtys.csv`, validator GO). 아래는 묶음수 적재 권한 + 후니 의미 확정/등록 영역. 근거: `03_validation/product-viewer/correction-validation-report.md`.

## D. 검증 주의 — 도무송 묶음수 사이즈 종속 (PRD_000066)

합판도무송스티커는 사이즈마다 구성 EA가 다르다(정사각10x10=8EA, 25x25=3EA, 45x45=1EA...). 묶음수를 (prd_cd, bdl_qty) PK로 분리하면 "어느 사이즈가 몇 EA"인지 연결이 끊긴다.

- 현 CSV는 PRD_000066에 대해 bdl_qty {8, 6, 3, 2, 1} 5행만 등록(distinct 값).
- **사이즈별 고정 구성수량**은 bundle_qtys 단독으로 표현 불가 → 옵션 캐스케이드 또는 `t_prd_products.constraint_json` 영역일 수 있음.

→ **확정 요청**: 도무송의 "사이즈→EA" 종속 관계를 어떻게 모델링할지. (본 정정은 묶음수 distinct 값만 분리하고 사이즈 연결은 보류.)
