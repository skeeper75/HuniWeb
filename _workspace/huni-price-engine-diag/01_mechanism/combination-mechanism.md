## 0. 공통 골격 — 한 번의 견적이 흐르는 길

`evaluate_price(target, selections, qty, grade_cd, mode, as_of)` 한 줄이 전부를 한다 `[확정·코드근거 pricing.py:247]`.

1. **타깃 해석** — prd_cd로 가격소스 선택: 템플릿단가>직접단가>**공식** `[확정·코드근거 pricing.py:285-328]`. (라이브는 직접단가·템플릿단가 0행 → 항상 공식 경로 `[확정·데이터]`.)
2. **공식 → 구성요소 순회** — `_evaluate_formula`가 배선(`formula_components`)을 `disp_seq` 순으로 돌며 각 구성요소를 매칭 `[확정·코드근거 pricing.py:444-475]`.
3. **구성요소별 단가행 매칭(`match_component`)** — 선택값↔단가행 차원 자동매칭. 호출자가 구성요소 목록을 넘기지 않음. 차원 NULL=와일드카드 `[확정·코드근거 pricing.py:15-16, 118-174]`.
   - 비수량 차원: 정확매칭(NON_QTY_DIMS) `[pricing.py:81-86]`
   - 티어 차원: siz_w/h='이하' 상한 중 최소, min_qty='이하' 중 최대 `[pricing.py:144-162]`
   - 동일 선택값에 차원조합 2개↑ 매칭 = `ERR_AMBIGUOUS`(흡수 안 함) `[pricing.py:136-138]`
4. **소계 환산(`component_subtotal`)** — 단가형: 단가×수량. 합가형: (총액÷구간min_qty)×수량 `[확정·코드근거 pricing.py:177-192]`.
5. **합산** — included 구성요소 subtotal 전부 더해 base_amount `[확정·코드근거 pricing.py:346-352]`.
6. **할인** — 수량구간 → 등급 순차 적용 `[확정·코드근거 pricing.py:359-368]`.
7. **최종가** — running을 원 단위 ROUND_HALF_UP `[확정·코드근거 pricing.py:369, 63-65]`.

**누가 어느 차원을 책임지나(요약)** `[확정·코드근거 + 데이터]`:
| 차원 | 책임 장치 | 매칭 방식 |
|------|----------|----------|
| 자재(두께 등) | 구성요소 단가행 `mat_cd` | 정확매칭(NON_QTY_DIMS) |
| 사이즈(규격) | 단가행 `siz_cd` | 정확매칭 |
| 사이즈(가로/세로 면적) | 단가행 `siz_width`/`siz_height` | 티어('이하' 상한) |
| 도수/인쇄옵션 | 단가행 `print_opt_cd` | 정확매칭 (⚠ clr_cd 아님 — D-2) |
| 공정(택1) | 단가행 `proc_cd` + `proc_grp:` 스코프 | 정확매칭 + 다중 공정 개별평가 |
| 공정 상세 파라미터 | 단가행 `dim_vals` jsonb | 키별 정확매칭 |
| 코팅 | 단가행 `coat_side_cnt` | 정확매칭 |
| 묶음 | 단가행 `bdl_qty` | 정확매칭 |
| 출력판형 | 단가행 `plt_siz_cd` | 정확매칭 |
| 수량(가격) | 단가행 `min_qty` | 티어('이상' 하한, 이하 중 최대) |
| 수량(할인) | 할인 디테일 `min_qty~max_qty` | 구간 포함 |
| 등급 | 등급할인율(주카테고리) | grade_cd 조회 |

**라이브 실측**: 프리미엄엽서(PRD_000016) 등 → `PRF_DGP_A` "디지털인쇄 원자합산형A" `[확정·데이터 2026-06-18]`.
이 공식의 구성요소 18개를 `disp_seq` 순으로 배선(`formula_components`). 각 구성요소의 `use_dims`(라이브 실측) `[확정·데이터]`:

| seq | comp_cd | comp_nm | prc_typ | use_dims (라이브) |
|-----|---------|---------|---------|-------------------|
| 1 | COMP_PRINT_DIGITAL_S1 | 디지털인쇄비 | 단가형 | proc_cd, plt_siz_cd, print_opt_cd, min_qty, proc_grp:PROC_000001 |
| 2 | COMP_PRINT_DIGITAL_S2 | 디지털인쇄비 | 단가형 | proc_cd, siz_cd, print_opt_cd, min_qty, proc_grp:PROC_000001 |
| 13 | COMP_COAT_GLOSSY | 유광코팅비 | 단가형 | siz_cd, coat_side_cnt, min_qty |
| 15 | COMP_PAPER | 용지비 | 단가형 | siz_cd, mat_cd |
| 16 | COMP_PP_CREASE_1L | 오시비 | 단가형 | proc_cd, min_qty, proc_grp:PROC_000029 |
| 22 | COMP_PP_VARTEXT_1EA | 가변텍스트 | 단가형 | proc_cd, min_qty, proc_grp:PROC_000085 |

**조합 메커니즘 (합산형)** `[확정·코드근거 + 데이터]`:
- 사용자가 사이즈·출력판형·인쇄옵션(도수)·용지(자재)·코팅면수·후가공(공정)·수량을 선택.
- 엔진이 **각 구성요소를 독립 매칭**: 인쇄비는 plt_siz_cd+print_opt_cd+min_qty로, 용지비는 siz_cd+mat_cd로, 코팅비는 siz_cd+coat_side_cnt로 각각 단가행을 찾음.
- 후가공(오시·미싱·가변텍스트…)은 `proc_grp:` 스코프가 붙은 구성요소 — 사용자가 그 공정을 선택했을 때만 매칭(미선택=no_match=합산 제외). 다중 공정은 `proc_sels`로 공정마다 개별 평가 `[확정·코드근거 pricing.py:462-471]`.
- **합산형 핵심**: included 모든 구성요소 subtotal을 **단순 합** → base. 각 항목 = 장당가×수량(단가형) `[확정·코드근거 pricing.py:346, 192]`.
- 이후 수량구간 할인(엽서 상품에 할인테이블 연결 시) 적용.

**어느 장치가 어느 차원**: ① 공식이 18개 구성요소를 묶음 / ② 각 구성요소 use_dims가 자기 차원 책임 / 단가행이 차원조합별 단가 보유 / 수량은 min_qty 티어 / ③ 할인 후처리.

## 2. 파일럿 B — 실사/현수막(면적 매트릭스) : `siz_width × siz_height` 티어

**라이브 실측**: 면적 단가행을 가진 구성요소 — `COMP_POSTER_BANNER_NORMAL`(79행), `COMP_POSTER_CANVAS_FABRIC`(52행), `COMP_POSTER_LEATHER_ARTPRINT`(52행) 등 `[확정·데이터 2026-06-18]`.
use_dims = `["siz_width","siz_height", ...]` (가로/세로 면적 차원) `[확정·데이터]`.

**조합 메커니즘 (면적 매트릭스)** `[확정·코드근거 + 데이터]`:
- 사용자가 가로(mm)·세로(mm)·수량을 입력. 자재(현수막천/캔버스/레더 등)는 **구성요소 선택**으로 갈림(상품군별 본체 comp).
- 엔진이 단가행에서 **siz_width·siz_height를 티어('이하' 상한)로 매칭** `[확정·코드근거 pricing.py:46, 150-154]`:
  - 주문 가로값 ≤ 임계 중 **가장 작은(최소) 임계**(그 값을 담는 가장 작은 구간)를 선택.
  - 주문값이 가장 큰 구간도 초과 → `ERR_ABOVE_MAX`(상한 초과, 다음 구간 미정의) `[pricing.py:152-154]`.
  - NULL 임계 = ∞(catch-all 상한) `[pricing.py:99-102]`.
- 즉 면적가는 **[가로 구간][세로 구간] 2D 매트릭스 셀의 단가**. 좌표 siz_cd 채번이 아니라 w/h 구간 차원으로 표현 `[확정·코드근거 + 메모 dbmap-area-matrix-wh-dimension]`.

**어느 장치가 어느 차원**: ② 단가행 siz_width/siz_height 티어가 면적 책임 / 자재는 본체 구성요소 선택으로 갈림 / 수량은 min_qty.

## 3. 파일럿 C — 아크릴(면적 + 두께) : `siz_w/h` 티어 + `mat_cd` 정확매칭, 합가형

**라이브 실측**: `COMP_ACRYL_CLEAR3T`("투명아크릴 인쇄가공비") `[확정·데이터 2026-06-18]`:
- prc_typ_cd = **PRICE_TYPE.02(합가형)** `[확정·데이터]`
- use_dims = `["siz_width","siz_height","mat_cd"]` `[확정·데이터]`
- 단가행 예(같은 w/h, 두 가격):
  | siz_width | siz_height | min_qty | unit_price |
  |-----------|-----------|---------|-----------|
  | 20 | 20 | 1 | 2000.00 |
  | 20 | 20 | 1 | 2500.00 |
  | 30 | 30 | 1 | 3100.00 |
  | 30 | 30 | 1 | 2480.00 |

**조합 메커니즘 (면적 + 두께, 합가형)** `[확정·코드근거 + 데이터]`:
- 같은 가로/세로(20×20)에 두 단가(2000/2500)가 공존 — 이는 **두께(mat_cd)가 비수량 정확매칭 차원**이기 때문 `[확정·코드근거 pricing.py:38 (mat_cd ∈ NON_QTY_DIMS)]`. 3T vs 1.5T가 mat_cd로 갈려 동시매칭이 아니라 정상 분기(`ERR_AMBIGUOUS` 회피).
  - ⚠ **이 보드의 존재 이유**: 만약 mat_cd가 NULL이면 같은 w/h에 2행 → 동시매칭 데이터 오류로 흡수 거부 `[확정·코드근거 pricing.py:136-138]`. 두께를 정확매칭 차원으로 둔 것이 분기의 열쇠. (라이브 mat_cd 충전 여부는 code-schema-auditor 확인 대상.)
- **합가형 환산**: unit_price=구간 총액 → `총액 ÷ min_qty(=1) = 장당가` → ×수량 `[확정·코드근거 pricing.py:185-190]`. min_qty가 0/NULL이면 `ValueError`(장당가 환산 불가) `[확정·코드근거 pricing.py:187-188]` — 그래서 CLEAR3T 단가행 min_qty=1 충전이 중요(메모 dbmap-acrylic 교훈과 정합).

**어느 장치가 어느 차원**: ② 단가행 siz_w/h(면적 티어) + mat_cd(두께 정확매칭) / 합가형 환산 / 수량은 min_qty(=1 base).

## 3.5 ★SOT 2 — 결합형 vs 독립형 구성요소 (디지털인쇄비 결합 · 별색 독립) `[확정·사용자SOT + 데이터]`

가격구성요소는 가격표에서 차원을 추출해 만들되, **같은 가격테이블 내 차원을 공유하면 결합, 독립이면 분리**한다. 파일럿 A 엽서가 그 실증:

**결합형 — 디지털인쇄비:**
- 국4절 1도/4도 + 3절은 **판형·도수·수량 차원을 공유** → 같은 룩업 행집합으로 표현 가능 → 한 구성요소(`COMP_PRINT_DIGITAL_S1`/`S2`)로 결합.
- 엔진은 plt_siz_cd(①판형)·print_opt_cd(③도수)·min_qty(수량)로 한 번에 매칭 — 국4절·3절·도수가 **차원 값으로 분기**되지 별도 comp가 아니다.

**독립형 — 별색인쇄비:**
- 별색은 **국4절에만 존재(3절엔 없음)** → 인쇄비와 차원 비공유 → 합치면 3절 행에 별색이 잘못 매칭될 위험 → **별도 구성요소** `COMP_PRINT_SPOT_*`로 분리.
- 그 별색 구성요소 **안에서** 화이트/핑크/은/금/클리어를 색 차원(proc_cd)으로 결합(메모 별색 dedup: 5색 통합 정본 `COMP_PRINT_SPOT_WHITE_S1`).

**판별 기준 = "차원 공유로 한 룩업 행집합 표현 가능한가"** (가능=결합 / 불가=독립).
→ ⚠ **오독 주의(메모 grouping-model 교훈):** 결합 시 종류축(proc_cd·색)을 빠뜨리고 키 검사하면 정상통합을 "중복·오염"으로 오판(별색 530행 오판 사례). 확장키(종류축 포함)로 재검사해야 중복 0 확인.

→ ★ **이 결합/독립 판별은 SOT 4(제약 부재)와 짝.** 별색을 잘못 결합형으로 넣으면(3절에 별색 합산), 엔진은 막을 제약이 없어(SOT 4·F-3) 오적재가 통과한다.

## 3.6 ★SOT 6 — 수량×단가(추가상품·부자재) 처리 원리 `[확정·사용자SOT + 라이브실측]`

엽서(PRD_000016)는 추가상품(OPP봉투·카드봉투 SKU)을 끼워 판다 — 수량×단가 요소의 실증. 두 갈래로 처리된다:

**(a) 추가상품(addon) = 별도 SKU 재귀평가 `[확정·라이브실측 2026-06-18]`:**
- 라이브: `t_prd_product_addons`(엽서→TMPL-000005 OPP봉투 50장 등·**가격 컬럼 없음**) → `t_prd_templates`(base_prd_cd=PRD_000001·dflt_qty=50).
- 시뮬레이터가 addon마다 **개별 `evaluate_price(SKU)` 호출**(고정수량 dflt_qty) → addon_total → grand_total = base + addon_total `[device-roles §⑤·price_views.py:1300-1327]`.
- → **수량×단가 of addon = "SKU별 고정수량 재귀 견적"**. 메인 구성요소 사슬(10차원)에 **안 들어옴.** SKU 단가 그릇 = `t_prd_template_prices`(현재 0행 → SKU의 base_prd_cd 공식사슬로 평가).

**(b) 부자재/악세사리(옵션 환원형) = 10차원으로 흡수 `[추정·도메인원리 + 라이브실측]`:**
- 라이브: option_items에 **add_price 컬럼 부재**(qty=BOM소요량만) → 옵션은 가격 미보유, 자재(mat_cd)/공정(proc_cd) BUNDLE으로 환원(메모 option=자재+공정 BUNDLE).
- 부자재(아일렛·끈·각목) 가격 = 환원된 ⑤자재·⑥공정 구성요소 단가행이 책임. 수량 = 단가행 min_qty 티어 또는 ④bdl_qty.
- → **10차원으로 표현 가능, 별도 가격장치 불요.** 단 "수량" 축(주문수량/BOM소요량/출력매수) 미확정(U-6).

**★판정 종합:** 가격구성요소에 "수량×단가" 전용 차원 신설 **불요.** (a)는 이미 addons+templates 별도 장치, (b)는 기존 10차원(자재/공정 + 수량축) 흡수.

## 4. 세 파일럿이 보여주는 일반 원리 (정의 — 결론 아님)

| 합쳐지는 방식 | 메커니즘 | 파일럿 |
|---------------|----------|--------|
| **합산형** | included 구성요소 subtotal 단순 합 | A 엽서 (18 구성요소) |
| **면적 매트릭스** | 한 구성요소 단가행을 siz_w/h 2D 티어로 셀 단가 룩업 | B 실사/현수막 |
| **면적 + 추가 정확매칭 차원** | 면적 티어 × mat_cd(두께) 정확매칭으로 셀 분기 | C 아크릴 |
| **수량 처리** | (가격) min_qty 티어 '이상' → (할인) 수량구간 후처리 | 전 파일럿 |
| **할인** | base 산출 후 수량→등급 순차 곱/차감 | 전 파일럿 |
