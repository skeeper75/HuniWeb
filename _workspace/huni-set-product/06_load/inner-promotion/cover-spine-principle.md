## 1. 책등(spine) 공식 — 도메인 권위 + 가격표 baseline

### 1.1 책등 두께 공식 (인쇄 도메인 표준)

```
책등(무선/소프트) = (페이지수 ÷ 2) × 종이 1장 두께(mm)          ← 장수 = 페이지/2
하드커버(양장)    = 책등(무선) + 보드/감싸기 보정 (≈ +5mm)        ← 양장 가산
```
- 종이 1장 두께(caliper) 예시: 모조/백모 80g≈0.10mm · 100g≈0.13mm · **120g≈0.15mm** · 아트 150g≈0.13mm · 두꺼운 120g≈0.17mm.
  - 예: 백모120 100p → 책등 = (100/2)×0.15 = **7.5mm** (+양장 5 ≈ 12.5mm).
- 출처: photomon 동인지 책등 가이드(`(페이지÷2)×장당두께`·양장 +5mm), sodaprint 책등높이계산법, aprint 세네카 작업팁(세네카 가로 +4mm·양옆 +11mm·사방 +20mm).

### 1.2 책등은 표지 펼침의 **가로(width)**를 키운다 (A5 세로제본)

A5 책은 긴변(210mm)이 제본변(spine 높이=210). 따라서:
```
표지 펼침 세로(height) = 책 세로(210) + turn-in 상하  ≈ 268mm  (책등 무관·고정)
표지 펼침 가로(width)  = 앞표지 + 책등 + 뒤표지 + turn-in 좌우  ≈ 390mm @ baseline
                       → 책등이 커지면 width만 증가
```
- 가격표 r64 권위 baseline: **하드커버무선책자 A5 재단(=펼침) = 390 × 268** (블리드 0). 이 390은 특정 페이지(≈100p region) baseline의 펼침 가로.
- aprint 세네카 공식과 정합: 펼침 가로 = 책폭×2 + 세네카(책등) + 사방 +20mm 여분 → 책등이 width를 직접 가산.

### 1.3 상품마스터·가격표에 책등 명시값/공식은 **없다** (실측)

| 원천 | 표지 사이즈 표기 | 책등 산식 | 판정 |
|---|---|---|---|
| 상품마스터 booklet-l1 (072/077) | 표지작업사이즈 = **`*책등에따라다름`** (병합·고정값 없음) | — | 🔴 명시 부재(가변 표시만) |
| 상품마스터 calc-formula-draft-l1 | — | `grep 책등\|펼침\|두께\|spine` = **0행** | 🔴 책등 산식 부재 |
| 가격표 260527 판걸이수 r64/r65 | 재단(펼침) A5=390×268·A4=532×355 (고정 1쌍) | 판걸이(F열) 공란 | 🟡 펼침 baseline만(페이지별 분기 없음) |

- ★**핵심**: 후니 권위는 책등을 **수식으로 박지 않는다**. 표지작업사이즈를 `*책등에따라다름`으로 표시하고 실제 펼침 치수는 **앱 런타임 임포지션**(주문 페이지수→책등→펼침)에서 계산하는 아키텍처. DB는 절가 룩업·판형만 저장(중간계산=앱·DB=룩업 철학·MEMORY [dbmap-compute-in-app-db-stores-lookup]).

### 1.4 후니가 책등을 discrete bucket으로 모델한 실증 (레더 링바인더·088)

라이브·마스터 실측 — 책등 가변을 **이산 펼침 siz로** 다룬 선례가 후니 안에 이미 존재:

- ★즉 후니는 책등을 "수식"이 아니라 **두께 버킷 = 별 사이즈코드/펼침코드**로 표현한다. 가변폭이 큰 바인더류(책등 31~56mm)에서만 버킷을 쓰고, 책등 가변이 작은 일반 HC책자(A5)는 펼침 1쌍(390×268)으로 충분하다는 설계 일관성.

## 2. 표지 펼침이 페이지수로 가변할 때 1-up 깨지나 — fn_calc_pansu 정량 (라이브 시뮬)

### 2.1 국4절 출력 실제영역 (라이브 verbatim)

```
SIZ_000499 국4절: work 316×467, margin 5/5/5/5 → 실제영역 306 × 457 (cut 306×457·impos_yn=Y)
fn_calc_pansu = GREATEST( ⌊pw/iw⌋×⌊ph/ih⌋ , ⌊pw/ih⌋×⌊ph/iw⌋ )   (normal, rotate)
```
- 표지 펼침 390(W)×268(H)는 국4절 306×457에 **normal 불가**(390>306) → **rotate만**: W(390)↦457변, H(268)↦306변. ⌊306/268⌋×⌊457/390⌋ = 1×1 = **1-up**.
- 1-up 유지 조건: 펼침 **width ≤ 457** (rotate 긴변) **AND** 펼침 **height ≤ 306** (rotate 짧은변).

### 2.2 책등 성장 → 펼침 width → pansu (백모120·아트150 등 caliper별·정량)

펼침 width = 390(baseline) − baseline책등보정 + 해당페이지책등보정. height=268 고정(책등 무관).

| 종이(caliper) | 24p width | 100p width | 200p width | 300p width | 1-up(국4절·전범위)? |
|---|---|---|---|---|---|
| 백모120 (0.15) | 384.3 | 390.0 | 397.5 | 405.0 | 🟢 YES (≤457·≤306) |
| 백모80 (0.10) | 386.2 | 390.0 | 395.0 | 400.0 | 🟢 YES |
| 아트150 (0.13) | 385.1 | 390.0 | 396.5 | 403.0 | 🟢 YES |
| 두꺼운120 (0.17) | 383.5 | 390.0 | 398.5 | 407.0 | 🟢 YES |

- ★**전 종이·전 페이지(24~300p)에서 펼침 가로 383~407mm** → 국4절 rotate 한계 **457mm**까지 **여유 ~50mm**. **1-up이 깨지는 페이지 없음**(width가 457을 넘으려면 책등이 ~+50mm 더 = 양장 백모120 기준 **~700p**가 필요·082는 100p상한·072는 300p상한이라 도달 불가).
- height 268은 책등과 무관히 고정·306 한계 안(여유 38mm) → 세로축 위반도 없음.

## 3. 정적 vs 동적 단일판정

### 3.1 A5 — 정적 단일 siz로 충분 (page-bucket 불요)

- pansu가 **전 페이지 범위에서 상수 1**이므로, 표지 출력매수 = copies(불변)·절가/코팅 = 판형(국4절) 고정. **페이지수가 표지 3비목 가격을 바꾸지 않는다.**
- 따라서 표지 펼침 siz는 **단일 정적 코드 1개**면 정확(동적·버킷 불요). 그 코드는 fn_calc_pansu(국4절, 코드)=1을 만족하는 임의 펼침 치수면 되고(390×268 권위치 권장), **completed A5(148×210)를 주면 4-up=과소**라는 함정만 피하면 된다.
- ★현 셋트 근사 **SIZ_000326(390×290)도 pansu=1** (390×290 on 306×457 rotate=⌊306/290⌋×⌊457/390⌋=1×1=1). 즉 **390×290 근사는 돈영향 0**(pansu 동일)이나, 권위 치수는 **390×268**(가격표 r64)이라 정정이 깔끔.

### 3.2 A4 — 정적/동적 무관 BLOCKED

- 페이지 버킷을 만들어도 532×355가 3절에 안 들어가므로 무의미. **출력판형 재정의(3절 margin/회전 또는 다른 판형)·3절 절가 적재·펼침 코드 신설 3건 선결**이 먼저(dbmap·실무진). 본 원칙 범위 밖(BLOCKED 유지).

### 3.3 결론

> 🟢 **단일 정적 표지 펼침 siz가 방어 가능 — 단 A5 한정, 그리고 펼침 치수(390×268)로 등록할 때만.** page-bucket·동적 siz는 A5 책자(072/077/082 全)에서 **불요**(책등 최대치가 1-up을 깨지 못함). A4는 별건 BLOCKED.
> ★전제: 후니가 책등 가변을 **앱 임포지션**으로 흡수(DB는 펼침 1-up 절가 룩업). 책등 두께가 커도 "펼침 1장=국4절 1판"이라는 출력 단위는 불변이므로 가격 키가 페이지에 의존하지 않는다.

- ★**표지 절가는 spread-size에 민감하지 않다 = plate-based.** 라이브 COMP_PAPER MAT_000078은 `(SIZ_000499 국4절, 46.65)` 단 1행 — 펼침이 390이든 405든 절가 동일. 책등이 커져도 **종이를 더 쓰는 비용은 표지 가격에 반영되지 않는다**(후니 가격모델: 펼침 1장=국4절 1판 절가 고정). 이는 후니의 명시적 단순화(절가 plate-based)이며, 책등 무시 = 가격 결함 0원(A5).
- 🔴 **진짜 돈 결함은 "책등 가변"이 아니라 "표지 펼침 siz 미등록"**(CFM-HC-COVER-PANSU): evaluate_price가 완제 A5(SIZ_000170)를 fn_calc_pansu에 넣으면 4-up → 출력매수 13판(50권 기준) = **3.8배 과소청구**. 이건 펼침 siz를 selections.siz_cd로 주입하면 해소(pansu=1)되고, **책등 정량과 무관**하다.

> ★**판정**: 현 정적 근사(SIZ_000326 390×290)의 책등 미모델링은 **A5 표지 가격에 0원 영향**(pansu=1 불변·절가 plate-based). "100p와 300p 표지 가격이 같을 수 없다"는 직관은 **후니 가격모델에서는 성립하지 않는다** — 책등 종이 추가분을 절가에 반영하지 않는 plate-based 단순화이기 때문(경쟁사도 펼침 1-up이면 표지 출력비 동일·책등은 제본/물리만). 만약 실무진이 책등 종이비를 청구하려면 그건 **새 가격 축**(현 권위 가격표에 없음)이며 별도 CONFIRM 사항.
