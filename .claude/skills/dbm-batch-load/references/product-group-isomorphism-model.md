# 인쇄상품 옵션·가격구성요소·상품군 동형 모델 (토대)

> 후니프린팅 DB매핑 하네스의 **단일 진실 토대**. 출력소재(IMPORT)를 출력 결정축으로 놓고,
> 옵션 → 가격구성요소 → 가격표 매트릭스 → 가격산출식 → 상품군 동형성까지 한 줄로 꿴다.
> 전 dbm-* 에이전트(domain-researcher·price-*·readiness-auditor·load-builder·option-mapper)가 공유.
> 권위 = 두 엑셀(`docs/huni/후니프린팅_상품마스터_*.xlsx` · `후니프린팅_인쇄상품_가격표_*.xlsx`) 실측. 추측 0.

## 목차
1. 두 엑셀의 연결 구조
2. 출력소재(IMPORT) — 출력 결정축
3. 판걸이수 — 출력매수 계산 룩업
4. 가격표 2분류 (가공/공정 vs 상품자체 포함가)
5. 상품군 동형성 = 계산공식집초안 권위
6. 매핑 효율 원리 (상품군 동형 파이프라인)
7. 실무진 주석 = 권위
8. 매핑 범위 (미출시 제외)
9. 적재 산출 기준 (실무진 가시성·정규화)

## 1. 두 엑셀의 연결 구조

- **상품마스터(13시트)** = 상품 정체·옵션구성·출력소재 지정·판걸이. 11 상품군 시트(디지털인쇄·스티커·책자·포토북·캘린더·디자인캘린더·실사·아크릴·문구·굿즈파우치·상품악세사리) + `계산공식집초안`(가격산출식) + `MAP`(카테고리).
- **인쇄상품 가격표(19시트)** = 가격구성요소 단가. 토대 2종(판걸이수·출력소재) + 가공/공정 8종 + 상품자체 8종 + 백업 1.
- **두 엑셀은 연결된다.** 상품마스터의 옵션이 가격표의 어느 단가행을 참조하는지가 `계산공식집초안`에 명시. **특히 상품마스터 `실사` 시트 = 가격표 `포스터사인` 시트** (실사 가격 = 포스터사인 [가로×세로] 면적매트릭스).

## 2. 출력소재(IMPORT) — 출력 결정축

출력소재는 인쇄상품의 **출력을 결정한다**(종이만이 아니다). 대분류 실측: 디지털인쇄용지·색지·PET·특수지·스티커용지·합판인쇄·하드커버전용·**실사출력(인화지)·패브릭(린넨)·시트커팅·아크릴**.

**가격 유무의 의미 — 핵심:**
- **종이류**(디지털인쇄용지·색지·PET·특수지)는 전지(국전 939×636)를 연(連) 단위로 사서 재단한다 → `연당가(H)` → `가격(국4절 I)`·`가격(3절 J)`로 환산. **출력소재 시트에 단가 있음.**
- **비종이류**(실사 인화지·패브릭 린넨·시트커팅·아크릴·합판스티커)는 **출력소재 시트에 단가가 없다.** 이유: 재고관리 없이 운영돼, 이 소재들의 가격은 출력소재 시트가 아니라 **각 상품자체 가격표(실사=포스터사인·아크릴·스티커 시트)에 "합가(포함가) 매트릭스"로** 들어있다. → §4와 직결.

비종이가 가격 없다고 "미적재 GAP"으로 단정하지 말 것 — **설계상 가격이 다른 시트(상품자체)에 있는 것**이다.

## 3. 판걸이수 — 출력매수 계산 룩업

판수(판걸이)는 임포지션으로 자동 계산되지만, `판걸이수` 시트가 따로 존재하는 이유는 그 계산의 **입력/검증 룩업 테이블**이기 때문이다.

- 구성: `사이즈옵션명 → 재단사이즈(B) + 블리드(C) → 작업사이즈(D) → 상품(E) → 판걸이(F)`. + 인쇄가능영역(J)·원형아이마크 완칼(K)·격자아이마크 반칼(L).
- **출력매수 = 주문수량 / 판걸이수** (계산공식집초안 B5 명시).
- **인쇄가능영역 기준 = `사방 5mm 제외`** (국4절 316×467 / 3절 330×660). 완칼(원형아이마크: 좌우 6mm·상 13mm·하 16mm 제외)·반칼(격자아이마크)은 **여백 기준이 가공방식마다 다르다.**
- 따라서 판수/출력가능영역은 DB에 저장하는 값이 아니라 **앱이 판걸이수 룩업 + 작업사이즈로 런타임 계산**한다(중간계산=앱, DB=룩업 원리).

## 4. 가격표 2분류 (가공/공정 vs 상품자체 포함가)

가격표 16개 상품/가공 시트는 두 성격으로 갈린다 — **반드시 인지해야 매핑이 흔들리지 않는다.**

| 분류 | 시트 (8+8) | 매트릭스 축 | 성격 |
|------|-----------|------------|------|
| **가공/공정 (추가가)** | 디지털인쇄비·코팅·접지옵션·인쇄후가공·커팅타공·후가공_박(소형)·후가공_박(대형)·제본 | [수량]×[판수/면적/옵션] | 옵션 선택분만큼 **가산** |
| **상품자체 (포함가)** | 스티커·합판도무송스티커·봉투제작·명함포토카드·엽서북떡메·아크릴·포스터사인·굿즈파우치(구간할인) | 다축([옵션·소재·수량] 또는 [가로]×[세로]) | 소재+출력+가공 **묶음 포함가** |

- **포함가 증거**: 포스터사인 첫행 `코팅포함가` 명시 / 계산공식집초안 `명함/포토카드 시트 가격은 용지가 포함되어있는 가격임`.
- 상품자체 시트는 **추가가격(옵션)을 제외한 포함가**로 구성된다. 즉 상품자체 가격 + (선택한 가공/공정 추가가)가 최종가.
- 면적매트릭스형: 아크릴·포스터사인·후가공_박(대형) = `[가로]×[세로]` 2축 매트릭스(off-grid는 한 단계 큰 크기 가격).

## 5. 상품군 동형성 = 계산공식집초안 권위

`계산공식집초안` 시트가 **상품군 동형 분류의 권위 원천**이다. 공식을 상품 하나가 아니라 **여러 상품을 묶어** 정의한다 — 같은 상품군은 옵션 구성과 가격공식이 **동형**(인쇄상품을 이루는 요소가 동일하게 쓰이기 때문). RedPrinting 역공학(`docs/reversing/`)도 같은 현상을 보인다.

**공식 유형(실측):**
- **원자합산형** — `판매가 = Σ 구성요소`. 예) 엽서류(프리미엄/코팅/스탠다드/투명엽서) = `인쇄비 + 코팅비 + 용지비 + 후가공비 + 추가상품`. 모양엽서/라벨택 = `인쇄비 + 용지비 + 커팅비`. 접지류 = `인쇄비 + 코팅비 + 용지비 + 접지비 + 후가공비`.
- **고정가형** — `[수량행][소재×면열] + 후가공비`. 예) 명함류(스탠다드/프리미엄/코팅/펄/투명명함). 용지 포함가.
- **면적매트릭스형** — `[가로][세로]` 면적 룩업 + ceiling. 예) 실사(=포스터사인)·아크릴·현수막.

**가격구성요소 사슬:** 각 공식의 구성요소가 **어느 가격표 시트를 참조하는지** 계산공식집초안 C열에 명시(판걸이수·디지털인쇄비·코팅·출력소재·인쇄후가공·후가공_박 등). 이 사슬이 가격산출의 배선도다.

## 6. 매핑 효율 원리 (상품군 동형 파이프라인)

**목적은 검증 시스템이 아니다.** 실제 고객이 주문 가능한 형태 — 상품마스터(구성요소) + 옵션 + 템플릿 + 제약조건 + 가격이 어우러져 **UI 선택과 생산 데이터 전달까지 처리되는 형태** — 로 DB에 적재하는 것이 목적이다. 단일 대표 1건이 목적이 아니라, **전 상품군 각각의 대표가 주문가능 형태로 종단되는 것**이 목표다. 모든 상품을 하나씩 대조하는 것은 비효율이므로 동형성을 이용한다:

1. **분류(S1)**: 계산공식집초안 × 옵션구성으로 상품군(동형 클래스) 분류. 후니 전체 **50개 미만**. 미출시·옵션 미완비 상품은 먼저 제외(§8).
2. **대표 완전 종단(S2)**: 상품군당 대표 1개를 골라, 아래 5레이어를 어우러지게 **실제 주문가능 형태로 적재**한다(단순 가격검증이 아니다). [[batch-patterns]]의 동형 클래스 = 이 상품군.
3. **자동 전파(S3)**: 동형 나머지 상품은 대표의 superset 매핑에서 부분집합으로 **스크립트 자동 적재 + 집계 전수 검증**(예외만 사람). 하나씩 수동 대조 금지.

이 동형성이 깨지는 지점(옵션 누락·공식 불일치·포함가/추가가 혼동·제약 차이)만 사람이 본다.

### 6-1. 대표 선정 기준 — 시트 실측 기반 (일반화 금지)

대표를 추상 원리로 미리 정의하지 말 것 — 그러면 일반화의 오류에 빠진다. **각 상품군 시트를 실측해, 그 시트에서 옵션·구성요소·제약이 가장 완전하게(superset) 반영된 주문가능 상품을 대표로 선정한다.** 대표는 상품군마다 다른 상품이다 — 프리미엄엽서가 superset인 것은 디지털인쇄 시트에서 옵션이 잘 반영돼 있어서일 뿐, 실사 시트에서는 일반현수막이 그 위치(옵션이 비슷하거나 더 많이 반영)다. 한 상품(프리미엄엽서)을 전 상품군의 일반 공식으로 박지 말 것.

superset을 대표로 잡는 이유: 부분집합 상품은 "해당 항 빼기"로 안전 전파되지만, 최소 상품을 대표로 뽑으면 그 상품에 없는 옵션·구성요소를 영영 못 잡아 누락이 전파된다. **superset 여부는 추상 판정이 아니라 그 시트의 실제 옵션 반영도로 가린다.** 5레이어(§6-2)는 "대표를 뽑는 잣대"가 아니라 "그 대표의 종단이 완성됐다고 볼 잣대"다.

### 6-2. 대표 완전 종단 = 5레이어 어우러짐

| 레이어 | 의미 | 목표 t_* | 산출 가치 |
|--------|------|----------|----------|
| ① 상품마스터 | 구성요소(자재·공정 BOM)·정체 | t_mat_*·t_proc_*·t_prd_products | 생산 데이터 |
| ② 옵션(CPQ) | option_groups/options/option_items (polymorphic ref_dim_cd) | t_prd_product_option_* | **UI 선택** |
| ③ 템플릿 | SKU·추가상품 묶음 | t_prd_templates·template_selections | 주문 구성 |
| ④ 제약조건 | 조합 규칙(잘못된 선택 차단) | t_prd_product_constraints | 주문 무결성 |
| ⑤ 가격 | 공식+구성요소 배선 → **전체 옵션 조합** 산출 | t_prc_* 4단 | 가격 처리 |

이 5개가 대표 1건에서 어우러져 적재되면 "이 상품군은 주문가능"이다. round-19 견적가능 3요소(①UI·②차원환원·③가격)에 템플릿·제약을 명시적으로 더한 완전체.

### 6-3. 가격 검증 = 전체 옵션 × 전체 공식 × 다양한 조합

가격을 인쇄비로 좁히지 않는다. 대표의 전체 계산공식을 적용해 **사이즈 × 용지 × 코팅 유무 × 후가공 유무 × 수량 구간**을 교차한 다양한 가격 케이스를 산출하고, 가격표 known값과 대조한다. 한 조합만 맞는 것은 "가격 처리 가능"의 증거가 아니다.

## 7. 실무진 주석 = 권위

실무진이 엑셀에 적은 주석(특히 **인쇄상품 가격표**의 주석)은 운영 현실의 권위다. 각 시트의 가격테이블은 **첫행/첫열부터 컨텐츠가 있는 마지막행/열까지 전체를 확인**한 뒤 DB 매핑 계획을 세운다. 주석을 놓치면 매트릭스 축 해석·포함가/추가가 구분·소재 적용범위를 오판한다.

## 8. 매핑 범위 (미출시 제외)

매핑 대상 = **기본적으로 주문이 가능한 형태의 상품**(옵션 구성이 제대로 된 상품). **미출시 상품은 고려하지 않는다.** 분류(S1) 단계에서 미출시·옵션 미완비 상품을 먼저 걸러낸 뒤 동형 클래스를 구성한다.

## 9. 적재 산출 기준 (실무진 가시성·정규화)

분석·리서치로 끝내지 않는다. 실제 주문 상품을 만들 라이브 t_* DB에 적재하고, 정확히 매핑됐는지 + **실무진이 `raw/webadmin` admin에서 알아보는지**까지 확인한다. 적재 시 다음을 지킨다.

### 9-1. 코드·명칭·비고 3축 분리

한 엔티티 행은 세 표현을 가진다 — 혼동하면 실무진이 admin에서 못 알아본다:
- **시스템 코드**(`*_cd`, 예 `SIZ_000xxx`·`PRD_000xxx`): 순번 surrogate. 시스템 정렬·FK용. 사람이 외울 필요 없음.
- **관리 인식 명칭**(`*_nm`·tags): 실무진이 관리 화면에서 이 행이 무엇인지 인식하는 이름.
- **고객 UI 명칭**(disp_nm 류): 고객이 위젯/사이트에서 보는 표시명.
- **비고(note)**: 전문가용 식별자·내부 행번호 나열이 아니라 **일반 실무진이 알아볼 수 있는 쉬운 한국어 라벨**.

### 9-2. 권위 = 후니 용어

인쇄 도메인지식·경쟁사 리서치로 인쇄용어·상품 구성을 확인하되, 명칭·용어의 최종 권위는 **후니프린팅이 실제 쓰는 용어**다. 경쟁사 용어로 후니 명칭을 덮어쓰지 않는다(참조·갭발견용).

### 9-3. 사이즈 정규화 (중복 저장 방지)

사이즈는 세 성격으로 갈린다 — 구분해 **동일 사이즈가 중복 행으로 저장되지 않게** 한다:
- **공용 사이즈**: 여러 상품군이 공유(A4 등) → 한 행을 공유 참조.
- **상품군 전용 사이즈**: 특정 상품군만 사용.
- **판형(출력규격) 사이즈**: 출력용지규격·전지 등 판형정보. `t_siz_sizes.impos_yn='Y'`로 작업/재단 사이즈와 다른 축으로 구분(스키마 실재).

적재 전 **search-before-mint**(같은 치수·이름이 이미 있는지 조회)로 중복 mint를 막고, 반드시 **데이터 정규화**(무손실·무중복·함수종속 정합)를 통한 적재를 한다.

### 9-4. webadmin 가시성 게이트

대표 종단 적재 후, `raw/webadmin` admin(product-viewer·change form)에서 그 상품의 옵션·가격·제약이 실무진이 알아보는 형태로 보이는지 확인한다. 코드만 뜨고 명칭이 안 보이거나 비고가 전문가 식별자면 §9-1 위반이다.
