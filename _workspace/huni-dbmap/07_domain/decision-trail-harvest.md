# Decision 트레일 수확 — DB note의 설계 결정 근거

작성: 2026-06-05 · 전략 H Phase 0(§4-0 0순위). 설명 한국어, 식별자/코드/SQL 영어.

> **목적**: DB note(`ref-*.csv`)에 화석화된 `Decision N`·`D-XX`·`b.X`·`R4-3`·`Wave N`가 가리키는 설계 결정의 {출처·결정 내용·도메인 의미}를 PDF·DB스키마·B규칙에서 수확. 못 찾으면 "원문 미발견(실무 컨펌 대상)".
>
> **수확 방법**: 전체 `ref-*.csv` note 컬럼을 grep으로 전수 추출 → Decision ID별 분류 → PDF/B-rules/표준에서 근거 교차.
>
> **확정도**: ✅근거확보(DB note+PDF/B근거) · 🟡부분(DB note만, 의미 추론) · 🔴원문미발견(별도 결정문서 부재).

## 0. 수확 결과 요약

- **수확한 Decision ID: 14종** (Decision 12·14 / D-14·17·23·24·25·26·29·30·32 / R4-3 / b.9·b.12 / Wave5 / D8 4-1)
- **별도 "Decision 문서"(D-XX 결정 원문 모음): 미발견** 🔴 — `docs/huni/DB스키마/`는 테이블 스키마 xlsx 29종만, Decision 결정록 PDF/문서 부재. DB note 자체가 결정 화석의 유일 소재.
- **PDF 교차 가능**: Decision 14(1상품=1방식)·Decision 12(별색 다중)·b.9(실사 전용)는 PDF로 교차 확증. 나머지(D-17/23/24/25/26/29/30/32)는 DB note만 — 의미 추론 가능하나 원문 미발견.

> ⚠️ **pushback 결론**: 전략 H §4-0 "Decision 문서 존재 시 수확"의 전제 — **별도 결정문서는 존재하지 않음**. Decision 트레일은 *DB note에 압축 인코딩*된 형태로만 존재. → 의미 재구성은 가능하나 *완전한 결정 근거*는 실무진 승계 필요(§4-3 인터뷰로). 단정 금지: D-1~D-13, D-15·16·18~22·27·28·31·33+ 등 **note에 안 나타난 Decision은 추적 불가**(번호 연속성으로 보아 26+ 결정이 있었으나 note에 일부만 화석화).

## 2. Decision 트레일에서 도출되는 도메인 공리 (L0/L3 입력)

수확된 Decision이 공통으로 가리키는 후니 설계 공리:

1. **인쇄방식=최상위 축, 1상품=1방식** (Decision 14 v3) — PDF 독립 확증. L0 taxonomy의 앵커.
2. **별색=공정·다중** (Decision 12) — 도수 아님. 결함2 차단 공리.
3. **방식별 전용 공정 레시피** (b.9 족자=실사전용 / b.12 에폭시=파우치전용 / 실사소재=실사전용) — L0 "방식별 가능공정 부분집합" 증거.
4. **자재 vs 공정 재분류 이력** (D-17 투명커버 공정→자재 / D-23 아크릴·파우치 자재 root) — "투명커버/아크릴은 자재축"이 후니 명시 결정. 결함(자재 MAJOR) 해석 토대.
5. **복합 공정 합성** (D8 4-1 오시+접지) — 전후 의존 공정을 단일 leaf로. L2 레시피 트리의 의존 모델 증거.
6. **버전 진화** (v1.0→v2→v3, Wave 1~5, NO-GO) — 설계가 반복 개정됨. note의 버전 표기 = 어느 결정이 최신인지 신호. **최신 버전이 권위**(D-17 v1.0 "폐기" 명시처럼).

## 3. 미발견·미추적 (실무 컨펌/승계 대상)

| 구분 | 내용 | 처리 |
|------|------|------|
| 별도 결정문서 | D-XX 결정 원문을 모은 문서/PDF | 🔴 **존재하지 않음**(DB스키마 디렉토리=테이블 xlsx만). DB note가 유일 소재 |
| 미화석 Decision | D-1~13·15·16·18~22·27·28·31·33+ 등 note 미등장 | 🔴 추적 불가. 번호 연속성으로 보아 다수 결정이 note 외부에 존재 → 실무진 승계 |
| enum 원천 | UV변형 5종·박색상 16종·접지 16종 등 enum 값의 결정 근거 | 🟡 DB에만, PDF 미수록. 실무 컨펌 |
| NO-GO 사유 | D-32 볼체인 NO-GO 이유 | 🔴 미수록 |
| 버전 충돌 해소 | "제작완료"(PDF p1) vs "출고준비중"(PDF p10) 상태명 | 🟡 문서간 불일치. 실무 컨펌 |

> **결론**: Decision 트레일은 *수확 가능한 14종*을 확보했고 핵심 공리(1상품=1방식·별색=공정·방식별 레시피·자재재분류·복합공정)를 도출. 그러나 **별도 결정문서 부재**로 완전 추적은 불가 — 전략 H §4-3 인터뷰에서 **note에 화석화된 14종을 "이 결정이 현행인가" 확인 + 미화석 결정 승계**가 다음 단계. 단정은 PDF 교차된 Decision 14·12·b.9·D-30에만.
