## 0. 한 줄 결론 (커버리지 verdict)

**실사 29상품은 CPQ 옵션 레이어로 PARTIAL 커버 가능하다.** 7개 옵션축 중 **5축은 스키마 차원에서 완전 커버**(사이즈·소재·별색·코팅·수량), **가공·추가 2축은 대부분 COVERED + 일부 SCHEMA-GAP**(보드종류·복합옵션 결합의미). 차단의 압도적 다수는 **DATA gap**(차원행 미적재 — 선적재로 해소) 또는 **이미 식별된 SCHEMA gap**(GAP-PARAM/COMPOSITE/SHAPE — ddl-proposer 소관)이며, 본 시트가 새로 발견한 **신규 SCHEMA gap은 1건**(보드종류 = 가공 vs 자재 vs 형태 모호, GAP-BOARD)이다.

- **COVERED 축: 5/7** (사이즈·소재·별색·코팅·수량) — 스키마 표현력 충분.
- **PARTIAL 축: 2/7** (가공·추가) — 차원행·param·복합결합 보강 필요.
- **신규 SCHEMA gap: 1건** (GAP-BOARD, 폼보드/포맥스보드 보드종류축).
- **상품 적재 가능성(상품별 재룩업 실측):** 28상품 라이브 등록(1상품 미등록 = 투명포스터★) · **21 FULLY INSERTABLE / 7 PARTIALLY BLOCKED**(+보드 2상품 GAP-BOARD 종속) · 사이즈축 28/28 COVERED · **PARTIAL 7상품 차단은 전부 DATA gap**(선적재로 해소, SCHEMA 차단 0).
- **[검증 정정·교훈]** 복합옵션 차원행 COVERED는 **상품별 재룩업 필수** — 138 파일럿의 "부착081 실재"를 124로 이월한 over-claim(validator MAJOR) 정정. 부착081은 138에만 적재 → 124·139의 끈/면끈/리본끈은 081 선적재 필요(DATA gap). **파일럿 1상품의 COVERED를 widening 시 이월 금지.**

## 4. 복합옵션 전수 — 가공·추가의 복합 패턴 (polymorphic 2+ item 확인)

> **복합옵션 종합 (상품별 재룩업 정정 반영):** 복합 패턴 **8종**(가공 5 + 추가 3). polymorphic 2+ item으로 **전부 표현 가능**(스키마) — 이종 차원(공정+셋트, 봉제+부착, 동일공정 param 2회) 결합이 typed FK로는 불가하나 polymorphic ref_dim_cd로 자연 표현. 단 **차원행 적재는 상품별로 다름**(아래 재룩업). **DATA BLOCKED seq2 = 5건**: 말아박기+면끈(124·부착081 미적재)·오버로크+리본끈(124·부착081 미적재+enum)·각목+끈 ×2(138·셋트 미적재)·천정고리(135·addon링크 미적재). **COVERED 유지 = 3건**: 오버로크+봉미싱(134·080 2회)·우드봉+면끈(134·addon)·우드행거+면끈(133·addon). **상품별 080/081 적재 현황**: 124=080만(081❌) · 133=080만 · 134=080만 · 135=082만 · 138=079·080·081(파일럿 상품만 081 실재). **부착 081은 138에만 적재** — 124의 면끈/리본끈은 081 선적재 필요(DATA gap). **리본끈은 enum(라벨/맥세이프/끈/테입) 미존재** = enum 확장 + 차원행 선적재 이중 필요. **복합 결합 의미**(AND동반)는 `GAP-COMPOSITE`(기식별·SCHEMA) — item_combine_typ 컬럼 부재라 "한 옵션 전 item 동반" 암묵 규약 의존.
>
> **[교훈·over-claim 정정]** 복합옵션 차원행 COVERED는 **상품별 재룩업 필수** — 138 파일럿이 입증한 "PROC_081 실재"는 **138 한정**인데, widening 시 124로 이월해 `말아박기+면끈=COVERED`로 over-claim했다(validator MAJOR 적발). 차원행 적재는 상품마다 다르므로 파일럿 1상품의 COVERED를 widening에 이월 금지 — L2 load-bearing 검사의 정의는 "그 prd_cd에 그 차원행이 실재하는가"다.

## 6. GAP 종합 — DATA gap vs SCHEMA gap (→ cpq-option-gaps.md)

silsa 전체 widening이 노출한 GAP을 DATA(선적재로 해소) vs SCHEMA(ddl-proposer)로 분류. **기존 8 GAP에 silsa 전수가 추가/강화한 분만 기술.**

### 6.1 신규 SCHEMA gap — GAP-BOARD (폼보드/포맥스보드 보드종류)

- **내용:** 폼보드 가공축 `화이트보드`/`블랙보드`, 포맥스보드 `화이트포맥스(3mm)`/`화이트포맥스(5mm)`가 **가공(proc) vs 자재(보드 substrate) vs 형태(두께 spec)** 어디인지 모호. 라이브 마스터에 "보드" 공정·"보드종류" 자재 **둘 다 부재**(공정명 키워드 0건, 자재명 0건). 해당 5상품(129·130·131·132·144) **materials 0행**.
- **3 후보 (침묵 선택 안 함):**
  - **후보 A — 자재 `.03`:** 보드 substrate = 물리 소재(폼보드 화이트 vs 블랙 = 본체색→재질 합성과 동류). 두께(3mm/5mm) = 자재 spec(depth). → 마스터 지도 §3.1 "본체색=재질 합성" 규칙 적용 시 가장 정합. **권고 후보**.
  - **후보 B — 가공 `.04`:** L1이 `가공(옵션)` 컬럼에 둔 사실(W열). 보드 부착/마감 공정으로 해석.
  - **후보 C — 형태(siz 융합):** 두께가 형상 사양이면 규격 융합.
- **증거:** L1 `가공(옵션)_가공`에 위치(W열) but 마스터 공정·자재 둘 다 미등록 = "L1 배치 ≠ 엔티티 확정". 폼보드/포맥스 materials 0행 = 자재로 의도됐을 가능성(보드가 본체 substrate).
- **판정: DESIGN DECISION NEEDING CONFIRMATION (신규).** 권고 = **후보 A(자재)** — 보드는 본체 substrate이고 화이트/블랙 = 본체색(재질 합성), 두께 = depth. 단 L1이 가공축에 둔 의도 확인 필요. ddl-proposer 라우팅(자재행 신설 vs 공정행 신설).
