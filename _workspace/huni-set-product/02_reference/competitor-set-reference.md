## 0. 한 줄 요약

레드프린팅은 책자를 **표지/내지 역할별 자재·도수 슬롯 분리 + 페이지차원 + 제본 가공군**으로 구성하고, 가격은 **셋트 통합 산정형(제본비를 인쇄비에 흡수·BIND 별항=0)**으로 만든다. 후니 권위는 **제본비를 단독 항으로 노출하는 구성원 합산형**. 이 가격축 차이가 셋트 설계의 핵심 의사결정(CONFIRM).

## 1. 레드프린팅 책자 셋트 구성 (PRBKYPR 윤전 무선책자 — 풀 실측)

출처: `huni-rpmeta/PR/reverse.md §2`(huni-widget s3 캡처 product_PRBKYPR.json 풀 infoCall + price 8조합). **후니 용어로 번역**:

### 1.1 역할별 구성원(반제품) 슬롯 — ★핵심 흡수
레드는 책자에서 **표지와 내지를 별도 스키마 슬롯**으로 분리(경쟁사 `pdt_mtrl_info` vs `inner_pdt_mtrl_info`):

| 구성원(후니 번역) | 레드 원슬롯(유입 금지) | 자재 | 도수 | 평량 제약 |
|---|---|---|---|---|
| 표지 자재/도수 | pdt_mtrl_info / pdt_dosu_info | 아트지300g 등 | 단면4색/양면8색 독립 | 표지 최소평량 150g |
| 내지 자재/도수 | inner_pdt_mtrl_info / inner_pdt_dosu_info | 윤전전용 백색모조80g 등 | 양면(내지 색도) 독립 | 내지 최대평량 130g |

→ **후니 흡수 후보**: 셋트 구성원을 `role=표지|내지`로 구분, 각각 자재·도수를 독립 입력받는 그릇. 가격 단가도 표지/내지 분리(레드 단가축 12종: 표지자재비·표지인쇄비·내지자재비·내지인쇄비·제본비·코팅비…). **★단, 후니 권위(booklet-l1)도 표지종이/내지종이·표지인쇄/내지인쇄 컬럼을 이미 분리** → 이 흡수는 권위와 정합(충돌 아님).

### 1.2 페이지차원 — 수량과 직교
레드: 내지 페이지 min10/max300/step1 + 부수 min30/step10. 가격 = (부수 볼륨디스카운트 곡선) × (페이지 선형가산). 실측 페이지 10→20 시 인쇄비 +1,120/page, 코팅비 불변.
→ **후니 흡수 후보**: 페이지수 = 수량과 별개 차원. 가격은 페이지에 선형가산. **★후니 권위(booklet-l1)도 내지페이지 min/max/incr를 명시**(중철 4/28/4, 무선 24/300/2 등) → 정합. 단 후니 incr는 제본방식별(중철=4배수)로 더 엄격.

### 1.3 제본 가공군 구성원
레드 제본 관련 구성원(`pdt_pcs_info` 그룹·후니 번역):
- **제본방식** — 레드는 **상품(코드)으로 분기**(무선/스프링/트윈링/스테플러=중철/실제본 각 별 상품). 후니 권위는 **한 시트에 제본방식별 별 상품**(중철책자/무선책자/PUR/트윈링/하드커버…) → **동일 패턴**(제본방식=상품 분리).
- **제본방향** — 좌철/상철 (책 펼침 방향). 후니 권위 컬럼 `제본방향(옵션)` 존재 → 정합.
- **면지(end-paper)** — 10색 컬러지 + 양면인쇄 삽입. **자재(컬러지)+공정(삽입) BUNDLE**. 후니 권위 컬럼 `면지(옵션)` 존재(하드커버=화이트면지) → 정합.
- **날개커버/소프트커버** — 책날개·커버형태 공정. 후니 권위 컬럼엔 명시적 날개 없음 → [모름] 후니 미정의.

### 1.4 자재→후가공 disable 제약
레드: 저평량지(모조80g) 선택 시 코팅/접지/미싱 비활성(24건 실측). 후니 권위도 ★제약(예 무선책자 박 크기 "★최소30x30") 존재 → 제약 룰엔진 동형.

## 2. 레드프린팅 엽서북·떡메(메모패드) 셋트 (그룹 횡단 추정)

출처: `huni-rpmeta/PR/reverse.md §4-B 카드/엽서군` + ST(문구류) — **라이브 풀 실측 아님(unobserved 다수)**.
- 레드는 **카드/엽서**를 PR 카테고리에 두고, 엽서북류(엽서 묶음)는 PR 책자군 또는 카드군에 분기. 떡메모지류(메모패드)는 ST(문구/오피스) 또는 떡제본 카드군.
- 레드 카드/엽서군 = 소형 규격 + 고급 후가공(박·형압·스코딕스·레이저커팅). **떡제본 패드(엽서북/떡메)**는 reverse.md에 명시 옵션 미추출([live:catalog] 상품명만).
- → **흡수 가능분 제한**: 엽서북/떡메의 경쟁사 구성원·개수규칙은 **레드 라이브 풀 실측 부재(unobserved)**. 후니 권위(booklet-l1)가 엽서북(표지+내지+떡제본)·떡메(내지+떡제본)를 명시하므로 **권위가 더 구체적** → 경쟁사 보강 불요(권위 우선).

[모름] 레드 엽서북·떡메 옵션 트리·가격축 = 기존 자산에 없음. 필요 시 라이브 읽기전용 보강(현 단계 불요).

## 3. 와우프레스(buysangsang/wowpress) 책자 셋트 — 가격축 대조

출처: `huni-dbmap/27_competitor-benchmark/PRF_BIND_SUM/benchmark.md`(2026-06-15). 와우프레스 직접 책자 옵션 트리 자산은 print-quote 02_business에 IA/가격규칙 수준(셋트 구성원 단위 풀 캡처 아님).
- 본 벤치마크의 경쟁사 대조축은 **레드 PRBKYPR** 중심. 와우프레스 책자 셋트의 구성원별 풀 캡처는 현 자산에 부재 → [모름].

### 3.1 ★가격축 핵심 발견 (셋트 가격 모델 결정에 직결)
| 항목 | 레드프린팅(PRBKYPR) | 후니 권위 골든(PRF_BIND_SUM) |
|---|---|---|
| 가격 단위 | **완제품 총액**(표지+내지+코팅+제본 통산) | **제본비 단독 항**(권당가×권수) |
| 제본비 노출 | **BIND 별항 = 0**(인쇄비에 흡수·통합 산정형) | 제본비 명시(G1 중철100권=70,000·G2 무선100권=50,000·G4 PUR100권=200,000·G5 트윈링100권=130,000) |
| 모델 분류 | **셋트 통합 산정형** | **구성원 합산형** |

→ **흡수 결론**: 후니는 제본비를 분리 노출하는 **구성원 합산형**이 권위(골든이 제본방식별 제본비를 단독 단가로 정의). 레드의 통합산정형(BIND=0)은 후니에 **이식 금지**(naming/모델 유입 금지). 단 레드의 "표지/내지 역할 분리·페이지 선형가산" 구조는 후니 권위와도 정합하므로 흡수 가능.

### 3.2 벤치마크 동형 매핑 = 0건 (정직 기록)
benchmark.md 결론: 레드 캡처는 좌철(BPLFT) 1종·완제품 총액·BIND=0 → 후니 무선/중철/PUR/트윈링 제본비 골든과 동형 0건. **셋트 가격 합리성 교차검증 불가**(레드 신규 캡처 + 후니 B-WIRE 해소 전까지 보류). → 셋트 가격은 **후니 권위 골든을 verbatim** 사용, 경쟁사 수치 대입 금지.
