## ① 사이트엔 있는데 엑셀 미등재 / 사이트-only

★[한계·중요] **이전사이트 표시명 ≠ 엑셀 `prd_nm`** 이라 사이트↔엑셀/라이브의 행단위 1:1 조인이 불가하다.
- 실증: 사이트 `인화지`(pcode=1) = 엑셀 `아트프린트포스터` / 사이트 `무선제본·PUR제본·트윈링제본`(pcode 36~39) = 엑셀 책자류 / 사이트 `명함(디지털)`(pcode 26) = 엑셀 디지털명함류.
- 사이트는 **고객 표시명(소재·제본방식 중심)**, 엑셀/라이브는 **내부 상품명(prd_nm)** 으로 명명 → 의미(semantic) 매핑이지 이름 매핑이 아님.
- 따라서 "사이트-only 진짜 구멍"은 이름만으로 자동 판정 불가 → **확인 필요(의미 대조)**.

채집 실측(`site-product-sample.tsv`):
- `/product/list.asp?pcode=N` **견적형 주문 universe = 85개**(pcode 1~104 전수, 105+ 무효) 채집.
- 이 중 정규화 매칭으로 척추 prd_nm과 연결된 것 **22개**, 이름 상이로 미연결 **63개**(= 사이트-only가 아니라 표시명 차이 = 확인 필요).
- `/goods/view.asp?pcode=`(굿즈 카탈로그형, sitemap상 pcode 98~271)은 **별도 템플릿이라 미전수** → 척추에서는 엑셀 `굿즈파우치 103` + `상품악세사리 15`로 대표.

→ **결론:** 사이트 universe(견적형 85 + 굿즈 카탈로그형 다수)는 척추(엑셀∪라이브 sellable)와 **상품군 수준에서 정합**하나, 표시명 차이로 사이트-only 진짜 구멍의 자동 적발은 보류. 후속 평가에서 의미 대조가 필요한 사이트 라벨 63건은 `site-product-sample.tsv` 참조.

## 한계 (정직 표기·pending 금지)

1. **사이트↔엑셀 행 조인 불가** — 이전사이트 표시명이 고객 라벨(소재/제본 중심)이라 `prd_nm`과 1:1 아님. 사이트-only 진짜 구멍은 의미 대조 필요(자동 미판정 63건).
2. **사이트 굿즈 카탈로그형 미전수** — `/goods/view.asp`(별도 템플릿·~170 pcode)는 전수 enum 미수행(브라우저 단일 인스턴스 안정성 한계). 척추에서는 엑셀 굿즈파우치/상품악세사리로 대표.
3. **분모 채택** — 사용자 directive상 universe=이전사이트지만, 표시명 차이로 라이브/엑셀이 더 정밀한 sellable 카탈로그라 **분모=엑셀 distinct prd_nm 249 ∪ 라이브 276 = 285행**을 분석 척추로 채택(사이트는 상품군 수준 정합·85개 견적형 표본으로 검증).
4. **종이류=N/A 2행** = 비상품(주석/가이드) → 진척도 분모에서 제외 권고(실 상품 분모 = 283).
