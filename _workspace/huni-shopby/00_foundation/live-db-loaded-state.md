## 4. 가격사슬 완전성 — "카트 전달 가능 상태" 정의

Shopby 카트로 보낼 수 있으려면 **가격사슬이 완전**해야 한다(아래 4조건 모두):

| 조건 | 의미 | 라이브 점검 키 | 결함 시 증상 |
|------|------|---------------|-------------|
| ① 가격소스 존재 | 공식 바인딩 또는 직접단가 1개 이상 | `t_prd_product_price_formulas` / `t_prd_product_prices` | 소스부재 → 0원(NONE) |
| ② 배선 정합 | 공식의 구성요소가 disp_seq로 배선·과대/누락 없음 | `t_prc_formula_components` | 미배선 → 매칭0 0원 / 과배선 → 과청구 |
| ③ 단가행 충전 | use_dims 차원조합별 단가행 존재·판별차원 NULL 아님 | `t_prc_component_prices` | NULL 와일드카드 → 과청구(R-4) |
| ④ 옵션선택 가능 | (선택형 상품) CPQ 옵션그룹·제약 적재 | `t_prd_product_option_groups/items/constraints` | 미적재 → 옵션 UI 부재 |

**카트 전달 가능 상태 분류:**

- **GREEN (가능):** ①+②+③ 충족 → `evaluate_price(strict).final_price>0`. ④는 옵션형 상품만 추가
  요구. **현재 ≈ 104 상품이 ①을 충족**(가격소스). 그중 CPQ까지 적재된 51 상품이 옵션 UI도 가능.
  → Shopby 브리지 1순위 = §2 ✅ 카테고리(스티커·포스터·엽서/카드·책자·접지카드·포토카드 등).
- **YELLOW (부분):** 일부 상품만 가격소스 보유(인쇄홍보물·라이프·에코백). 상품 단위 선별 후 가능.
- **RED (막힘):** 가격소스 0(하드커버책자 0/19·포토북 0/8·아크릴 0/2·캘린더류·조합형·포토카드 외
  액세서리). §21 NO-GO 드라이버(공식 전무·미바인딩·orphan)와 일치. **Shopby 브리지 대상에서 제외**
  하거나, 인간 승인 후 dbmap 적재 트랙 선행 필요.

## 6. CPQ 옵션 레이어 구조 (§7 — 카트 옵션 모델 입력)

- L1(상단 9탭)=차원 데이터 적재(사이즈·도수·판형·자재·공정·묶음수·추가상품·페이지룰).
- L2(하단 3탭)=차원을 참조·결합하는 옵션 레이어: **옵션그룹/옵션/옵션아이템**(polymorphic `ref_dim_cd`
  로 차원행 묶음·`sel_typ` 단일/다중) · **제약규칙**(차원값 간 호환/금지/필수동반·JSONLogic var 7키:
  `siz_cd·plt_siz_cd·mat_cd__usage_cd·proc_cd·bdl_qty·opt_id·sub_prd_cd`) · **구성템플릿(SKU)**.
- **★STALE 정정:** §7 groundtruth(2026-06-08) "L2 전 상품 0행"은 **무효** — §1 재실측 51 prd 적재.
- ★ `ref_param_json` 부재 확정(옵션아이템 스키마에 없음) — 타공 구수·각목 규격 보존처 미존재(GAP 유지).

> **Shopby 함의:** Shopby 주문옵션 모델로 보낼 "옵션 선택지" = L2 옵션그룹/아이템. 옵션 선택 → `ref_dim`
> 풀이 → selections 차원값 → `evaluate_price`로 가격. **옵션그룹 적재된 51 상품이 Shopby 옵션 매핑
> 1순위 후보**(브리지 무손실 SB2 게이트 입력).
