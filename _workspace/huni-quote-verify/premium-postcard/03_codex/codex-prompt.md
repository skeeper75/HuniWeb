# 독립 가격계산 검증 의뢰 (당신 = 독립 2nd opinion 검증가)

당신은 후니프린팅 인쇄 가격엔진의 **독립 교차검증가**입니다. 아래 work-spec(상품 정의·가격엔진 계약·라이브 적재 상태·권위 가격표·골든 케이스)만 보고 **독립적으로** 판정하세요. 다른 누군가의 판정은 제공되지 않습니다 — 당신 스스로 추론하세요. **모르면 모른다고 명시**하세요. 근거 없는 단정 금지.

---

## 0. 대상 상품
- 상품: **프리미엄엽서** (코드 PRD_000016)
- 상품군: 디지털인쇄
- 가격공식: **PRF_DGP_A** (디지털인쇄 합산형 = 구성요소 subtotal 합산)
- 권위[HARD]: 상품마스터(260610) + 인쇄상품 가격표(260527, 디지털인쇄비 B01 시트 "국4절")

## 1. 상품 정의 (상품마스터 260610 발췌)
- `front_colrcnt_cd = CLR_000005` (= CMYK 4도, 고정 단일값)
- 인쇄옵션 차원 `print_opt_cd` 두 값:
  - `POPT_000001` = 단면 (front, CMYK)
  - `POPT_000002` = 양면 (CMYK)
- 즉 상품 정의상 print_opt_cd 축은 **면(단면/양면)** 을 의미하고, 도수는 CLR_000005(칼라) 고정.

## 2. 가격엔진 계약 (라이브 webadmin catalog/pricing.py 의 evaluate_price, 정확한 동작)
- 공식 = formula_components 의 구성요소(component)들의 subtotal 합산.
- 각 구성요소 subtotal 계산 두 모드:
  - 단가형(prc_typ=unit): `단가 × qty`
  - 합가형(prc_typ=sum): `단가 ÷ min_qty × qty`
- 구성요소 단가는 `t_prc_component_prices` 에서 차원키(use_dims)로 룩업. 차원키 예: siz_cd, plt_siz_cd, print_opt_cd 등.
- **엔진에는 판걸이수·출력매수·imposition·ceil 개념이 전혀 없음**(grep 결과 0). qty 는 그대로 선형 곱.

## 3. 라이브 적재 상태 — 디지털 인쇄비 구성요소 (t_prc_component_prices @ 국4절 SIZ_000499 실측)
PRF_DGP_A 에는 인쇄비 구성요소가 **두 개**(COMP_PRINT_DIGITAL_S1, COMP_PRINT_DIGITAL_S2) 가 둘 다 addtn_yn=Y 로 배선되어 있고, 각각 print_opt_cd 두 값에 단가가 적재됨:

| 구성요소 | 차원키(use_dims) | POPT_000001 단가(출력매수1당) | POPT_000002 단가(출력매수1당) |
|------|------|------|------|
| COMP_PRINT_DIGITAL_S1 | plt_siz 키 | 450/400/350/130 (수량구간별) | 900/850/800/310 |
| COMP_PRINT_DIGITAL_S2 | siz 키 | 900/800/700/260 | 1800/1700/1600/620 |

## 4. 권위 가격표 (인쇄상품 가격표 260527, 디지털인쇄비 B01 "국4절")
가격표 원본은 **3D 매트릭스**: [도수] × [면] × [출력매수]. 도수 = 흑백 / 칼라 / 별색6. 면 = 단면 / 양면.
국4절 시트의 출력매수1당 단가(열별):
- B열 = 흑백·단면: 450/400/350/130
- C열 = 흑백·양면: 900/800/700/260
- D열 = 칼라·단면: 900/850/800/310
- E열 = 칼라·양면: 1800/1700/1600/620

(즉 위 §3 라이브의 S1=흑백단면·칼라단면 행, S2=흑백양면·칼라양면 행을 print_opt_cd 두 값 슬롯에 적재한 것.)

## 5. 골든 케이스 (검증 기준값)
- 단면, 출력매수 25 → 인쇄비 골든 = **20,000**
- 양면, 출력매수 25 → 인쇄비 골든 = **40,000**

## 당신이 독립적으로 판정할 질문 (근거 명시)
1. 프리미엄엽서(PRF_DGP_A)가 자기 가격공식으로 **가격계산이 제대로 되는가?** 3축 중 어디가 깨지는가:
   - (a) SOT 일치: 상품마스터 ↔ 가격표 데이터가 일치하는가
   - (b) 공식↔구성요소 매핑: PRF_DGP_A 에 배선된 구성요소가 올바른가
   - (c) 차원 매칭: 구성요소 차원(print_opt_cd 등)이 가격표 매트릭스 차원과 올바로 대응하는가
2. 라이브 인쇄비 S1/S2 가 가격표 매트릭스(도수×면×출력매수)를 올바로 차원 매핑했는가? `print_opt_cd` 축이 **상품 정의에선 면, 가격데이터에선 도수**를 가리키는데 — 이 충돌이 실재하는가? 손님이 "단면"(POPT_000001) 선택 시 어떤 단가가 매칭되는가?
3. 이 상품은 칼라전용(front_colrcnt=CLR_000005 고정)이다. 그렇다면 올바른 인쇄비 단가는 가격표 **칼라단면(D열)/칼라양면(E열)** 이어야 하는가? 국4절 출력매수25 기준 칼라단면 단가는 골든(20,000)과 어떻게 대응하는가?(D열 출력매수1당 = 800 → ×25 = 20,000 인지 확인)
4. S1 과 S2 가 둘 다 addtn_yn=Y 로 동시 배선되면 가격계산에 이중 합산이 발생하는가?
5. (있다면) 위에서 다루지 않은 추가 결함·리스크를 독립적으로 제기하라. 없으면 "없음"이라고 하라.

## 출력 형식
- 판정: GO / NO-GO / NO-GO(교정가능) 중 하나 + 한 줄 요약
- 3축 각각 PASS/FAIL + 근거
- 각 질문 1~5 답
- 확신도(High/Med/Low)와 "모르는 것" 명시
