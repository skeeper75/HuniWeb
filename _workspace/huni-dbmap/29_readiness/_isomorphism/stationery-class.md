## 0. 한눈에 (결론)

- **문구 = 책자(booklet)와 구조 동형이나 가격은 고정가형(inline).** 한 문구 = `내지 블록 + 표지 블록 + 제본` 반제품 구조(전부 책자와 동형)이나, 두 핵심 차이: ① **가격 inline(C29 고정가형·포토북/캘린더 패턴)** — 책자는 제본비 합산형. ② **박/형압 블록 없음**(문구류 박 운영 안 함).
- **핵심 = 문구는 가격공식이 두 갈래의 다중 동형 클래스:**
  - **(A) 고정가형(PRF 미존재) — 만년다이어리·먼슬리·스프링·메모패드·중철노트**: C29 inline 단가(9000/4500/2500…), 라이브 고정가 공식 **전무**(명함 PRF_NAMECARD_FIXED 같은 후니 고정가형 공식이 문구엔 없음).
  - **(B) 떡메모지 매트릭스(PRF_TTEOKME_FIXED 존재) — 떡메모지(097)**: `사이즈×권당장수×장수구간` 매트릭스, 단가행 112개 라이브 적재 완료·가격표 verbatim. **단 바인딩 0행**(상품에 미연결).
- **9차 사이클 종단 대상 = (A) 고정가형 클래스 + (B) 떡메모지를 함께.** 대표 = **만년다이어리(소프트커버) 172 + 스프링노트 177 보완**(트윈링 제본 전형) + **떡메모지 097(B 클래스 단독 대표)**.
- **결론 선행**: 문구는 책자(WIRE 단일·데이터 멀쩡)와 **달리 다중 전무** — CPQ 옵션 레이어 0/0/0(책자는 069=8그룹) + 가격 바인딩 0 + 고정가 공식 자체 부재. **아크릴(이중전무)에 더 가까운 결함 프로파일**, 단 떡메모지 단가행은 멀쩡(verbatim). §5에서 확증.

## 1. 동형 클래스 분류 — 계산공식집초안 + 가격표 권위

### 1-1. 계산공식집초안 (문구 가격 구조)

문구 = **가격포함 시트 → 고정가형**(토대 §5·§4 상품자체 포함가). `계산공식집초안`이 문구를 명함/포토카드와 같은 고정가 묶음으로 본다:
```
(A) 문구 고정가형 = [상품][고정 단가]   (용지·인쇄·코팅·제본 포함가, C29 inline)
(B) 떡메모지     = [사이즈][권당장수 50/100][장수구간] 단가 매트릭스   (참조시트: 엽서북떡메)
```
(A)는 옵션이 거의 없는 **단일 고정가**(만년다이어리 9000원 등), (B)는 명함류처럼 다축 매트릭스.

### 1-2. 가격표 `엽서북떡메` 시트 실측 (떡메모지 블록 = B 클래스)

```
사이즈      90x90mm           70x120mm
장수/수량  50장1권 100장1권   50장1권 100장1권
6          3000   3200        3000   3200
12         2300   2500        2300   2500
18         2200   2400        2200   2400
...        ...    ...         ...    ...
600         850   1050         850   1050   (28 장수구간)
```
→ 떡메모지 = `[사이즈 2종]×[권당장수 2종]×[장수구간 28]` = 112 단가행. 라이브 COMP_TTEOKME 112행과 **셀 수 일치**(§5 골든 대조).

### 2-1. "문구 CPQ 전무" = 대표 선정의 제약

### 2-2. 대표 보정 — 일반화 오류 회피 (토대 §6-1 준수)

- **(A) 주축 대표 = 스프링노트(177)** — 문구 시트에서 **내지 블록(무지내지+백모조) + 표지 블록(아트250+무광코팅) + 트윈링 제본(좌철·방향 param) + 부속(실버링·합지보드)**가 가장 풍부하게 반영(superset). 5레이어 종단의 본체.
- **(A) B 셋트 보완 대표 = 만년다이어리(하드커버 173)** — 177이 못 가진 **하드커버 제본(면지·하드보드 sub_prd·sets)** 차원을 종단에서 함께 측정(177 단독 GO가 하드커버 셋트 결함을 못 보는 거짓 GO 방지).
- **(B) 떡메모지 대표 = 떡메모지(097)** — 유일하게 가격 그릇(PRF_TTEOKME_FIXED·단가행 112)이 라이브에 있는 상품. (A)와 가격 산정방식이 달라 별도 종단(매트릭스 골든 대조).
- **제본방식 자체**(미싱/트윈링/떡/중철/하드커버)는 PROC_000017 자식 적재 여부를 종단에서 확인 — 미싱제본 부재가 (A)의 핵심 갭(round-13 ST-06)이며 §5·cycle-report Q3에서 판정.

→ 5레이어 종단(S0~S5)은 **(A) 177(주축) + 173(B셋트 보완) + (B) 097(매트릭스)**를 함께 측정한다.

## 4. 책자와의 동근/이질 명시 (사용자 directive 핵심)

| 축 | 책자(7차) | 문구(9차) | 동근/이질 |
|----|----------|----------|:--:|
| 구조 | 내지+표지+제본 반제품 | 내지+표지+제본 반제품 | **동근** |
| 가격 클래스 | 합산형(제본비 항) PRF_BIND_SUM | 고정가형(inline) + 떡메모 매트릭스 | **이질**(합산 vs 고정/매트릭스) |
| CPQ 옵션 | 069=8그룹·차원환원 완비 | **전 상품 0/0/0** | **이질**(책자 완비 vs 문구 전무) |
| 가격 그릇 | PRF_BIND_SUM 존재·바인딩 4 | (A) 공식 부재 / (B) 공식·단가 있으나 바인딩 0 | **이질**(책자 바인딩 vs 문구 미바인딩/부재) |
| 단가행 값 | 제본 verbatim 32/32 | 떡메모 verbatim 112/112 | **동근**(값 결함 0) |
| C32 제본방향 | 그릇 부재(param 슬롯 없음) | **동일 그릇 부재**(PROC_000017 prcs_dtl_opt만) | **동근** |
| 미싱제본 | 책자엔 무관 | PROC_000017 자식 부재(ST-06) | **문구 고유** |

→ **결론: 문구는 책자와 구조는 동근이나, 가격·CPQ 결함은 동근이 아니다.** 책자 WIRE(데이터 멀쩡·배선만)는 문구에 **그대로 재발하지 않는다** — 문구는 그 앞 단계(CPQ 전무·공식 부재·바인딩 0)에서 막힌다. **즉 책자 BOOKLET-BIND-WIRE와 동근이 아닌, 아크릴 이중전무에 가까운 더 이른 결함**. 단 단가행 값 결함 0·C32 그릇 부재는 동근.
