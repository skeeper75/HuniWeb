## C. frm_typ_cd 백필(공식유형 구조화) — D-5 · DDL 트랙

라이브에 frm_typ_cd 부재(formula-inventory §0). 소스 DDL은 선언했으나 미적용 → round-14 "선언≠적용" 해소.

- **제안**: `ALTER TABLE t_prc_price_formulas ADD COLUMN frm_typ_cd varchar(50)` + FK→FRM_TYPE(seed `sql/05_seed.sql:286` 이미 .01 합산형/.02 단순형 존재) + 16공식 값 백필(합산형 8=FRM_TYPE.01·단순형 8=FRM_TYPE.02).
- **효과**: 가격뷰어가 공식 유형을 배지/필터로 노출 가능, 같은 유형 묶어보기. note 접두어 의존 해소.
- **주의**: NOT NULL 강제 시 기존 행 백필 후 제약 추가(2단계). **DDL 변경이라 dbm-ddl-proposer + 인간 승인 트랙**. Q-2(단순형 분류) 컨펌 선행. round-16 빌더들이 frm_typ_cd 미사용으로 작업한 것과 충돌 없음(추가 컬럼).
