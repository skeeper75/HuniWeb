# ★중대 발견 — addon 가산형(opt_cd 키) 모델이 라이브 엔진에서 미작동 (2026-06-28)

> 라이브 시뮬레이터 검증(사용자 directive "실제 사이트에서 견적 나오는지 확인")에서 적발.

## 증상 (라이브 price-simulator 실측)

- **146 아크릴키링**(30x30+은색고리+볼체인블랙 기대 5200) → **0원**. 본체·고리·볼체인 구성요소 전부 "제외".
  - 본체(투명아크릴 인쇄가공비): "제외 · 해당 없음 — 단가 없는 조건 — 자재=MAT_000208"(볼체인 자재가 본체 자재 자리에).
  - 고리·볼체인 가산 comp: "제외 · 해당 없음".
  - 액세서리 전부 해제(본체만)해도 **0원**(본체 자재 MAT_043 미주입).

## 근본 원인 (코드 정독 확정)

`raw/webadmin/webadmin/catalog/price_views.py`:
1. **`_opt_maps`(1378~1391)**: 옵션그룹의 option_items를 가격차원으로 환원하는데 매핑이
   `OPT_REF_DIM.01→siz_cd · .02→plt_siz_cd · .03→mat_cd · .04→proc · .05→bdl_qty` 뿐 — **opt_cd 생성 없음**.
   → 가산 comp(use_dims=[opt_cd])는 옵션그룹 선택으로 opt_cd를 못 받음 → 영원히 "해당 없음".
2. **`OPT_REF_DIM.03 → dims["mat_cd"]=k1`(1386)**: 액세서리 옵션(고리 MAT_051·볼체인 MAT_208·머리끈 MAT_057)을
   고르면 **전역 mat_cd가 액세서리 자재로 덮임** → 본체 면적격자(mat_cd=MAT_043 필요) 매칭 실패.
3. **covered 숨김(1411~1416)**: 옵션그룹이 커버하는 차원(mat_cd)은 raw 드롭다운에서 **숨김**.
   → 액세서리 그룹이 있으면 본체 자재(MAT_043) 선택 UI 자체가 사라짐 → 본체 영원히 미가격(선택해도 못 함).
