## 0. 비전문가 한눈 요약 (1단락)

후니 디지털인쇄(엽서류)는 큰 종이(전지) 1장에 작은 제품을 여러 개 앉혀 한 번에 찍습니다.
"한 장에 몇 개 앉느냐"가 **판걸이수**이고, 이 숫자로 **필요한 전지 매수**가 정해져 가격이 매겨집니다.
그런데 지금 가격엔진(`fn_calc_pansu`)은 종이 면적만 보고 **꽉 채워서 계산**해, 실제 인쇄에 필요한
칼선 간격·기계가 종이를 무는 여백(물림변)을 빼먹습니다. 그 결과 **한 장에 더 많이 앉는 것으로 착각**해
전지를 적게 잡고 → 인쇄비를 **덜 청구(저청구)**합니다. 반대로 일부 사이즈는 더 많이 잡아 **과청구**도
납니다. 실무진이 엑셀에 손으로 기록한 판걸이수가 정답이므로, 엔진이 그 값을 먼저 쓰도록 고치면 됩니다.
**이건 돈이 어긋나는 결함이고, 전 디지털인쇄 상품에 걸쳐 일어납니다(systemic).**

## 1. 증상·근본원인·영향

### 증상
가격엔진이 판형기준 구성요소(디지털인쇄비·용지비)를 **판수 = ⌈주문수량 ÷ 판걸이수⌉**
(`pricing.py:215 plate_qty`, ceil)로 계산한다. 판걸이수 = `fn_calc_pansu(판형, 완제품사이즈)`.
이 함수가 실무진 엑셀(상품마스터 "판수" 컬럼)과 **불일치**하여, 같은 주문이 권위와 다른 전지 매수로
계산된다 → 인쇄비·용지비가 어긋난다.

### 근본원인 (코드 실증)
`raw/webadmin/sql/32_fn_calc_pansu.sql` (라이브 함수 본체):

```sql
n_normal := floor(pw / iw)::int * floor(ph / ih)::int;  -- 정상 배치
n_rotate := floor(pw / ih)::int * floor(ph / iw)::int;  -- 90° 회전 배치
RETURN GREATEST(n_normal, n_rotate);                    -- 더 많이 앉는 쪽
```

- `pw,ph` = 판형 **인쇄 실제영역** = 판형 작업사이즈 − 4면 여백.
- `iw,ih` = 아이템 **작업사이즈(여분포함)**.
- **순수 기하 패킹**(면적÷면적)이라 ① 아이템 간 **칼선 간격(gutter)** ② **물림변(그리퍼)**
  ③ **자재별 임포지션 차이**(투명·특수지)를 **전혀 반영하지 못함** → 꽉 채워 **과다 패킹**.
- 실제 인쇄는 1열/1행 덜 앉으므로, 엔진 판걸이수 > 권위 판걸이수가 systemic하게 발생한다.

라이브 실측(판형 SIZ_000499 316x467·여백 4면 5mm → 실제영역 306×457):
- 100x150 아이템 work 102×152 → `floor(306/102)×floor(457/152)=3×3=9`. **권위=8**(간격/물림 1행 손실).
- 투명엽서는 자재 임포지션이 더 커서 권위가 더 낮음(아래 §2 투명 50% 과다).

### 영향 (저청구·돈크리티컬·systemic)
- **방향①(주 결함·저청구):** 엔진 판걸이수 > 권위 → 전지 과소 → 인쇄비/용지비 **저청구**.
  전 디지털인쇄 원자합산형(엽서군·상품권·슬로건…)에 systemic.
- **방향②(역방향·과청구):** 일부 사이즈는 엔진 판걸이수 < 권위(예 스탠다드 148x210) → 전지 과다 →
  **과청구**. 즉 결함은 **양방향**이며 "무조건 저청구"가 아니다 — 교정은 양쪽 다 바로잡는다.
- 돈크리티컬: 단일 사이즈에서 **±7~50%** 가격 오차 실측(아래 매트릭스).

## 3. 권장 수정 (개발팀)

### 코드 위치
- **함수 본체(고칠 대상):** `raw/webadmin/sql/32_fn_calc_pansu.sql` — `fn_calc_pansu(plate, item)` plpgsql.
  현재 기하 패킹 `floor(pw/iw)*floor(ph/ih)` (L56~59).
- **엔진 호출부:** `raw/webadmin/webadmin/catalog/pricing.py:287 _calc_pansu()`(`SELECT fn_calc_pansu(...)`),
  `:215 plate_qty()`(판수 ceil), `:670 needs_plate` 분기.
- **셋트/시뮬레이터 호출부:** `price_views.py:1495 _fn_calc_pansu()`, `:1888`(내지 파생).
- **판걸이수 저장소:** **현재 없음** — 라이브에 `pansu` 이름의 컬럼/테이블 0건(실측 확인).

### 수정 명세
1. **권위 판걸이수 저장소 신설**(현재 컬럼 없음): 후보
   - **(a·권장)** 신규 테이블 `t_siz_pansu(plt_siz_cd varchar, item_siz_cd varchar, pansu int, PK(plt_siz_cd,item_siz_cd))`
     — 판걸이수는 (판형 × 아이템) 종속이라 정규·재사용(동일 판형×아이템은 동일값). 적재원=상품마스터 "판수" 컬럼.
   - (b) `t_prd_product_sizes`에 `pansu` 컬럼 — 상품×사이즈 단위(판형 자동도출 맥락이라 판형 미표현).
2. **`fn_calc_pansu` 수정**: 권위 저장소 lookup 우선 → 있으면 그 값 반환·없을 때만 기존 기하 폴백.
   ```sql
   -- (신규) 권위 판걸이수 우선
   SELECT pansu INTO v FROM t_siz_pansu
    WHERE plt_siz_cd = p_plate_siz_cd AND item_siz_cd = p_item_siz_cd;
   IF v IS NOT NULL THEN RETURN v; END IF;
   -- (폴백) 기존 기하 계산 — 권위 미등록 사이즈 대비
   ... floor(pw/iw)*floor(ph/ih) ... RETURN GREATEST(n_normal, n_rotate);
   ```
   - 호출부(`pricing.py`·`price_views.py`)는 **변경 불필요** — 함수 시그니처/반환 동일.
   - 폴백 유지로 권위 미등록 사이즈는 기존 동작 보존(회귀 위험 격리).
3. **적재**: 상품마스터 디지털인쇄 시트 "판수" 컬럼 전 상품×사이즈 → `t_siz_pansu`
   (plt=출력용지규격·item=완제품사이즈). 예 프리미엄엽서 판형 316x467(SIZ_000499) ×
   {73x98:15·98x98:12·100x150:8·135x135:6·95x210:6·110x170:4·148x210:4}. 투명엽서는 별도값
   {98x98:8·100x150:6·135x135:4} — **상품(자재)별로 판걸이수가 다르므로 키에 자재/상품 맥락 필요**
   (판형×아이템만으로 부족 시 (b) 또는 (prd_cd, item) 키 고려 — 투명이 동일 사이즈에서 다른 판수인 점이 근거).

> ★설계 주의: 투명엽서(SIZ 동일하나 판수 6 vs 일반 8)가 입증하듯 판걸이수는 **자재/상품 임포지션에도 종속**.
> (a) 순수 (판형×아이템) 키로는 투명을 표현 못 한다 → 키에 상품/자재 축 추가 또는 (b) 상품×사이즈 단위 권장.

## 4. 검증 방법 (회귀 케이스)

수정·적재 후 시뮬레이터 재실측으로 판걸이수 = 권위 일치 확인:
- **회귀①(저청구):** 투명엽서 PRD_000019 100x150 qty120 → 판걸이수 **6**(현재 9)·매수 20 →
  기본자재 약 **42,000**(현재 29,400). 별색 화이트+양면 케이스는 ~71,000~75,500 근접(원 진단 공급가).
- **회귀②(systemic):** 프리미엄/코팅/스탠다드 73x98 → 판걸이수 **15**(현재 18), 100x150 → **8**(현재 9).
- **회귀③(과청구 역방향):** 스탠다드 148x210 → 판걸이수 **6**(현재 4)·매수 20 → 약 **16,413**(현재 24,619 과청구 해소).
- **무회귀:** 권위·엔진 일치 사이즈(98x98·135x135·95x210·110x170 등)는 폴백/등록값 모두 동일 유지.
