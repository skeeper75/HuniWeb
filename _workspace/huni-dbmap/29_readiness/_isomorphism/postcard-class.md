## 0. 한눈에 (결론)

- **엽서 동형 클래스(PRF_DGP_A 원자합산형) = 7상품**: 프리미엄엽서·코팅엽서·스탠다드엽서·투명엽서·화이트인쇄엽서·핑크별색엽서·금은별색엽서. (계산공식집초안 row 3 권위가 묶음)
- **대표 = 프리미엄엽서(PRD_000016)** — 디지털인쇄 시트 실측에서 옵션·구성요소·추가상품 반영도가 **superset**(size 7·후가공 4종 풀·추가상품 6·종이=옵션화). 단 코팅 차원만 없음 → 코팅 차원은 **코팅엽서(017)가 보유**(부분 보완).
- **상품권(041/042)은 엽서 동형 클래스에 포함되지 않는 별도 클래스**다. 같은 PRF_DGP_A 공식을 공유하나 (a) 구분이 다르고(상품권), (b) 핵심 옵션 축이 미싱/가변/박(상품권)이며 엽서는 코팅/모서리 축이 핵심이다. **공식은 동형, 옵션 구성은 비동형** → §4에서 판정.

## 1. 동형 클래스 분류 — 계산공식집초안 권위

`계산공식집초안` 시트 실측(row 3~12):

```
[원자합산형: 프리미엄엽서 / 코팅엽서 / 스텐다드엽서 / 투명엽서 / 화이트인쇄엽서 / 핑크별색엽서 / 금은별색엽서 / 종이슬로건 / 스텐다드 상품권 / 프리미엄상품권]   (참조시트: 판걸이수)
판매가 = 인쇄비 + 코팅비 + 용지비 + 후가공비 + 추가상품
 (1) 출력매수 = 주문수량 / 판걸이수            [판걸이수]
 (2) 인쇄비(4도) = [수량행단가] x 출력매수        [디지털인쇄비]  *국4절 또는 3절
 (3) 별색인쇄비 = [인쇄비단가] + [출력매수 수량행]x[칼라열]   [디지털인쇄]
 (4) 코팅비 = [수량행단가]x[코팅타입열] x 출력매수   [코팅]
 (5) 용지비 = [크기별 기준단가] X [출력매수]+5장(손지율)  [출력소재]
 (6) 후가공비 = [제작수량행]                  [인쇄후가공]
 (7) 후가공박(대형) = [면적별동판비] + [면적별 A,B,C,D,E군][군별/칼라별 수량행 합가]  [후가공_박(대형)]
 (8) 추가상품 = 목록별 가격적용
```

이 공식 1개가 **여러 상품을 묶어** 정의된다 = 동형성의 직접 증거(토대 §5). 가격구성요소 사슬(참조시트 C열)이 가격산출 배선도다.

계산공식집초안 row 3 목록 중 **디지털인쇄 시트에서 구분이 엽서로 묶이고 PRF_DGP_A 공식을 쓰는 상품**:

|---|--------|:--:|------|

| 2 | 코팅엽서 | 엽서 | 코팅 차원 보유 · PRD_000017 |
| 3 | 스탠다드엽서 | 엽서 | 프리미엄 부분집합 · PRD_000018 |
| 4 | 투명엽서 | 엽서 | 투명PET·화이트언더 |

### 1-2. 미출시·미완비 제외 (§8)

본 7상품은 전부 디지털인쇄 시트에 옵션 구성(size·인쇄·종이·후가공·추가상품)이 완비된 **주문가능 형태** → 미출시 제외 대상 없음. (디지털인쇄 시트 전 상품 스캔 결과 엽서 구분에 미완비 행 없음.)

## 2. 대표 선정 — 시트 실측 옵션 반영도 (superset 판정)

디지털인쇄 시트 per-상품 옵션차원 distinct 값 카운트(실측):

| 상품 | size | 종이 | 인쇄(도수) | 코팅 | 모서리 | 오시 | 미싱 | 가변T | 가변I | 추가상품 | 별색(W/C/P/G/S) |
|------|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|
| **프리미엄엽서** | **7** | 옵션화(별도설정) | 2 | 0 | 2 | **3** | **3** | **3** | **3** | **6** | 0 |
| 코팅엽서 | 7 | 2 | 2 | **4** | 2 | 0 | 0 | 0 | 0 | 6 | 0 |
| 스탠다드엽서 | 5 | 옵션화 | 2 | 0 | 2 | 3 | 3 | 3 | 3 | 6 | 0 |
| 투명엽서 | 4 | 2(PET) | 1 | 0 | 2 | 0 | 0 | 0 | 0 | 6 | W=3 |
| 화이트인쇄엽서 | 4 | 4 | 0 | 0 | 0 | 0 | 0 | 0 | 0 | 6 | W=3·C=3 |
| 핑크별색엽서 | 4 | 3 | 2 | 0 | 0 | 0 | 0 | 0 | 0 | 5 | P=3 |
| 금은별색엽서 | 4 | 3 | 2 | 0 | 0 | 0 | 0 | 0 | 0 | 5 | G=3·S=3 |

### 2-1. "프리미엄엽서 = superset" 가설 검증 결과 = **조건부 확정**

- 프리미엄엽서가 **size·후가공(오시/미싱/가변텍/가변이미지) 풀·추가상품·종이 옵션화**에서 최대 = 옵션 반영도 최상위. 토대 §6-1의 "옵션이 가장 완전하게 반영된 주문가능 상품" 정의에 부합 → **대표 확정**.
- **단 superset이 strict하지 않음 — 1개 차원 누락:** 프리미엄엽서는 **코팅(coat=0)**이 없다. 코팅 차원은 코팅엽서만 보유(coat=4). 따라서 "프리미엄엽서를 전 엽서의 일반공식으로 박으면" 코팅 차원을 영영 못 잡는다(토대 §6-1 일반화 오류 경고).
- **별색(W/C/P/G/S) 차원도 프리미엄엽서엔 없음** — 별색은 투명/화이트/핑크/금은 엽서가 보유. 단 별색은 계산공식집초안 (3)별색인쇄비로 PRF_DGP_A 내 분기항이고 라이브 PRF_DGP_A formula_components에 별색 comp가 이미 풀로 존재(round-19 §1 C18~22 확인).

### 2-2. 대표 보정 — 일반화 오류 회피 (토대 §6-1 준수)

추상 원리로 "프리미엄엽서=전부"라 박지 않는다. **대표 종단의 범위를 다음으로 명시**:
- **주축 대표 = 프리미엄엽서(016)** — 종이·후가공·추가상품·size·도수 차원의 superset.
- **코팅 차원 보완 대표 = 코팅엽서(017)** — 016이 못 가진 코팅 옵션 그룹·COMP_COAT_* 사슬을 종단 검증으로 함께 본다.
- **별색 차원** = PRF_DGP_A formula_components에 별색 comp 풀 적재 확인(라이브)으로 종단 범위에 포함하되, 별색 엽서(투명/화이트/핑크/금은) 상품 자체는 Sp 자동전파 시 "프리미엄 + 별색항" 부분집합으로 닫는다.

→ 5레이어 종단(S0~S5)은 **016(주축) + 017(코팅 보완)**을 함께 측정한다. 단일 016만으로는 코팅 사슬을 못 보므로(=거짓 GO 위험), 코팅 차원을 017로 명시 보완.

## 3. 동형 나머지 = 대표의 부분집합 매핑 (Sp 자동전파 전제)

대표(프리미엄엽서 016 + 코팅 보완 017) 기준 각 동형 상품이 빠지거나 더하는 항:

| 상품 | 대표 대비 차이 (부분집합/추가항) | Sp 전파 방식 |
|------|------------------------------|------------|
| 코팅엽서(017) | 016 − 후가공(오시/미싱/가변) + **코팅항** | 코팅 그룹·COMP_COAT_* 추가, 후가공 그룹 제거 |
| 스탠다드엽서(018) | 016 − 추가상품 일부·size 일부(7→5) | 부분집합(항 빼기) |
| 투명엽서 | 016 base − 후가공 + 투명PET 종이 + **화이트언더(별색 W)** | 종이축 PET 한정 + 별색 W 항 추가 |
| 화이트인쇄엽서 | 016 base − 인쇄(도수0) + 흑지 + **별색 W·C** | 인쇄=화이트/클리어 별색 전용 |
| 핑크별색엽서 | 016 base − 후가공 + **별색 P항** | 별색 P comp 배선 |
| 금은별색엽서 | 016 base − 후가공 + **별색 G·S항** | 별색 G/S comp 배선 |

> **Sp 핵심:** 코팅(017)·별색(투명/화이트/핑크/금은)은 단순 "항 빼기"가 아니라 **"016에 없는 차원 더하기"**다. 그래서 §2-2처럼 대표를 016 단독이 아니라 016+017로 잡고, 별색 comp가 PRF_DGP_A에 풀 적재됐는지를 종단에서 확인해야 Sp가 안전(누락 전파 방지). 단순 부분집합(항 빼기)인 것은 018(스탠다드)뿐.

## 4. 상품권(041/042) 클래스 귀속 판정

- **공식**: 041/042도 계산공식집초안 row 3 목록(스텐다드/프리미엄 상품권)에 포함 = **PRF_DGP_A 공유**(RTM도 둘 다 PRF_DGP_A로 봄).
- **구분**: 디지털인쇄 시트 구분=상품권(엽서 아님).
- **옵션 구성(시트 실측)**: 스탠다드상품권 = crease/perf/vartext/varimg 각 3 + paper 4; 프리미엄상품권 = + foil 2 + paper 8. **핵심 옵션 축이 미싱(perf)·가변·박(042)** — 엽서의 코팅/모서리 축과 다르다.
- **판정**: **공식은 동형(PRF_DGP_A) / 옵션 구성은 비동형 → 별도 동형 클래스(상품권 클래스)**. 엽서 동형 클래스에 포함하지 않는다.
  - 단 D-1b(후가공 prc_typ 오적재)는 PRF_DGP_A 공식 레벨 결함이라 **엽서·상품권 양 클래스 공통**(한 번 정정으로 동시 해소) — 클래스는 분리하되 가격 결함은 공유(토대 §5 가격사슬 공유).
  - 상품권은 별 사이클에서 종단(이미 round-19 `voucher/quote-gate.md` 조건부 GO).
