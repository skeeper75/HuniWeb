## G6 (재검증) — 자기충족 단언 제거  → **PASS**

"전역 0"·"완제품 .01"·"전 시트 0" 등 라이브 불일치 자기충족 단언이 전 binding에서 제거/정정됐는지 2-pass(grep + 라이브 재측정).

**2-pass 결과:**
- **pass-1 (grep):** "전역 0"·"라이브 0"·"완제품 .01"·"option_items 0"·"전 시트 0" 토큰을 전 파일 grep → 잔존 출현은 **전부 부정문맥**("전역 0 아님"·"라이브 0은 부정확"·"전역 0이 아니라 충족률"·"`.01` 아님"). **긍정 단언으로 남은 자기충족 거짓 = 0건.**
- **pass-2 (라이브 재측정):** 정정 단언의 수치(25·3,481·0·63·`.01`=3건=프리미엄3종)를 라이브로 재측정 → **전부 일치.**

**근거:** index.md §0 L11~13이 NO-GO 두 결함을 명시 반영. matrix §1.2 L96~101이 "전역 0 아님" 정정. 굿즈 L17~18이 "완제품 PRD_TYPE.01 단언은 도메인/라이브 뭉갠 오류→정정" 자기-철회. 실사 L11이 "종전 PRD_TYPE.01 표기는 라이브 미실측 오기" 명시 철회. **긍정형 거짓 단언 잔존 0.** **G6 PASS.**
