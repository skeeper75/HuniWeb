### ★핵심 판정 — 계산방식 정체 (절대 권위로 결판)

**캘린더 = 원자합산형(atomic-sum) · 디지털인쇄 PRF_DGP 패밀리 직계 동형.**
1차 가설(variant 고정가형 = 굿즈 GP-2 동형)은 **반증됨.** 근거:
- 계산공식집초안 r94 `[원자합산형: 캘린더]` 명시 라벨 — B열 정본.
- r95 `판매가 = 인쇄비 + 용지비 + (제본비 or 캘린더가공비)` — Σ(비목) 구조.
- `디자인캘린더(가격포함)`의 inline `가격`은 **대표 1 variant의 합성 결과 스냅샷**(인쇄비+용지비+제본가공비)이지 가격 소스가 아님. 같은 상품이 `캘린더` 시트에서 다수 사이즈×장수×가공으로 전개되는데 `디자인캘린더`는 product당 1행만 가격 보유 → 매트릭스 단가표 아님 = inline 고정가형일 수 없음.

`확신도: 높음` (calc-draft 명시 라벨 + 라이브 COMP_BIND_CAL_* 가 제본비를 수량행 매트릭스로 적재한 사실이 합산형 입증).

### ★가격 소스 결판 (과업 (a)/(b)/(c) 질문)

**(c) 둘 혼합 — 但 `디지털인쇄비` 시트 참조형이 본질, `디자인캘린더` inline은 검증용 골든.**
- **인쇄비** = `디지털인쇄비` 시트(국4절/3절 수량행 × 인쇄면×색 열) 참조 (b).
- **제본비** = `제본` 시트(수량행 × 제본종류열, 트윈링제본 컬럼) 참조 — 但 라이브는 캘린더 전용 COMP_BIND_CAL_* 로 별도 적재(아래 §2).
- **`디자인캘린더` inline 가격** = 위 합산의 대표 결과값 → **골든 케이스 검증 자료**로만 사용(가격 소스 아님).

## 2. 3층 지도 — 원자합산형 (완제품) → **PRF_CAL_* (신규 설계 필요·라이브 미배선)**

> calc-draft r95~98: `판매가 = 인쇄비 + 용지비 + (제본비 or 캘린더가공비)`
> `(1) 인쇄비 = [총출력장수/판걸이수] × 인쇄비`  `(2) 제본비 = [수량행][제본종류열]`  `(3) 캘린더가공비 = [가공비] × [제작수량]`

```
상품 [완제품]: 탁상형캘린더·미니탁상형캘린더·엽서캘린더·벽걸이캘린더·와이드벽걸이캘린더
  ├─ (1) 인쇄비   = [수량행단가] × 총출력장수        comp=COMP_PRINT_DIGITAL_S1  [원자합산]  (재사용)
  │        ★총출력장수 = 주문수량 × 페이지수(장수) / 판걸이수(판수)
  │        ★페이지수(장수)가 엽서(단장)와 캘린더를 가르는 load-bearing 차원 — 4(8P)/8(16P)/12(24P)…
  │        use_dims=[proc_cd, plt_siz_cd, print_opt_cd, min_qty, proc_grp:PROC_000001]
  │        plt_siz_cd: 국4절(SIZ_000499) 4상품 / 3절(SIZ_000077) 와이드 1상품
  │        print_opt_cd: 단면/양면 × 색(흑백·CMYK) → 가격표 디지털인쇄비 열차원
  ├─ (2) 용지비   = [크기별 기준단가] × 총출력장수    comp=COMP_PAPER             (재사용)
  │        use_dims=[siz_cd, mat_cd]   mat_cd: 몽블랑 190g/240g·스노우지 250g 등
  ├─ (3a) 제본비  = [제작수량행][제본종류]  comp=COMP_BIND_CAL_{DESK220·DESK130·DESKMINI·WALL}  [원자합산·수량구간형]
  │        ★라이브 실재(단가행 적재) — 但 use_dims=[proc_cd, min_qty, proc_grp:PROC_000017]
  │        ★단가 = 수량구간(min_qty) 룩업이지 calc-draft 문자 "× 제작수량" 아님(아래 §4 GAP)
  │        탁상220: 1→5000·4→4000·10→3000·50→2500·100→2300·1000→2000  (PROC_000100)
  │        벽걸이: 1→5000·4→4000·10→3000·50→2500·100→2000·1000→2000   (PROC_000099)
  └─ (3b) 캘린더가공비(add-on 택1) = [가공비] × 제작수량   comp=신규 COMP_CALOPT_*  [×제작수량 평면가산]
           우드거치대 4000 / 1구타공+끈 1000 / 2구타공+끈 1500 / 고리형트윈링제본 2000 / 삼각대·가공없음 0
           (★상품마스터 `캘린더가공_추가가격` 칸 verbatim)
```
계산방식: **원자합산형** · 라이브 baseline: **미배선(WIRE 결함)** — COMP_BIND_CAL_* 단가행은 적재됐으나 ① 어떤 공식에도 wire 안 됨(formula_components 0행) ② 캘린더 상품에 공식 바인딩 0건(t_prd_product_price_formulas 0행) ③ 인쇄비·용지비 comp 미배선.
`확신도: 높음(calc-draft+가격표+라이브 제본단가행 일치)`

### ★디지털인쇄 PRF_DGP_A 와의 동형성 (흡수 후보 = 직계 재사용)

캘린더 인쇄비·용지비는 **PRF_DGP_A(엽서류)와 완전 동일 chassis**:
- COMP_PRINT_DIGITAL_S1 (212 단가행·plt_siz_cd×print_opt_cd×min_qty) — 그대로 재사용.
- COMP_PAPER (siz_cd×mat_cd) — 그대로 재사용.
- 차이는 단 하나: **페이지수(장수) 곱셈** — 엽서=1장, 캘린더=4~16장. 출력매수 산식에 페이지수가 곱해짐.
- 제본/가공이 추가비목으로 합산 = PRF_DGP_E(접지카드, 인쇄+용지+접지+후가공) 구조와 유사.

→ 설계 권고: PRF_CAL_DESK / PRF_CAL_WALL 등 신규 공식이되, 인쇄비·용지비 comp는 PRF_DGP 재사용(mint 금지·search-before-mint). 제본비는 COMP_BIND_CAL_* 재사용. add-on 캘린더가공만 신규 mint 후보.

## 4. 캘린더가공 추가가격 — prc_typ 판정

> calc-draft r98: `(3) 캘린더가공비 = [가공비] × [제작수량]`

| 가공 종류 | 추가가격(상품마스터) | 상품 | prc_typ 판정 |
|-----------|---------------------|------|--------------|
| 우드거치대 | 4000 (엽서캘린더는 `추가상품`으로도 표기 4000) | 엽서캘린더 | **× 제작수량** (개당 평면 가산) |
| 1구타공+끈 | 1000 | 엽서캘린더 | × 제작수량 |
| 2구타공+끈 | 1500 | 벽걸이캘린더 | × 제작수량 |
| 고리형트윈링제본 | 2000 | 벽걸이·와이드 | ★제본비로 분기(아래) |
| 삼각대(그레이/블랙) | 0 | 탁상형·미니 | 0 (무료·옵션) |
| 가공없음(재단만)/제본없음 | 0 | 다수 | 0 |

### ★중요: 제본비는 calc-draft 문자와 라이브 적재가 어긋남(컨펌큐)

- calc-draft r95: `제본비 = [수량행][제본종류열]` (수량구간 룩업) **vs** r98 `캘린더가공비 = [가공비] × 제작수량` (평면 가산).
- **라이브 COMP_BIND_CAL_* = 수량구간형**(min_qty 1/4/10/50/100/1000 tier × unit_price, proc_grp:PROC_000017). 즉 r97 제본비 패턴으로 적재됨(개당 단가가 수량 늘수록 하락).
- → **설계 권고: 제본비(트윈링/캘린더제본) = 수량구간형 COMP_BIND_CAL_*(라이브 실재 재사용)** / **단순 add-on(우드거치대·타공+끈) = 고정단가 × 제작수량**.
- 돈크리티컬: 제본비를 `× 제작수량` 평면 가산으로 잘못 모델하면 수량 할인이 사라짐(벽걸이 1000개 주문 시 제본비 2000원/개 = 2,000,000원 적정 vs 평면 가산 시 동일하나 tier 미반영). 반대로 add-on을 수량구간으로 오모델하면 단가행 결손.

`확신도: 중(calc-draft r95↔r98 문자 충돌 — designer 컨펌큐)`

## 5. 제작수량 구간할인 (t_dsc_*) — 바인딩 없음

라이브 실측: `t_prd_product_discount_tables` 에 캘린더 PRD 바인딩 **0건**.
- 굿즈 DSC 4타입 같은 캘린더 전용 수량구간할인 **부재.**
- 단, 제작수량 효과는 인쇄비(COMP_PRINT_DIGITAL min_qty tier)·제본비(COMP_BIND_CAL min_qty tier) 단가행 자체의 수량구간 하락으로 흡수됨 → **별도 t_dsc_* 불요**(디지털인쇄 PRF_DGP 패밀리 동형, 굿즈와 결정적 차이).
`확신도: 높음`

## 6. 두 시트 관계 — `캘린더` vs `디자인캘린더(가격포함)`

| 항목 | `캘린더` 시트 | `디자인캘린더(가격포함)` 시트 |
|------|--------------|------------------------------|
| 상품 | 동일 5상품(PRD_000108~112) | 동일 5상품 |
| 행 수 | variant 다수(사이즈×장수×가공 전개) | product당 대표 1행(+추가상품 행) |
| `가격` 칸 | **없음** | **있음**(10400/6500/4000/9900/24000) |
| 주문방법 | 업로드(Y) | 편집기(Y) |

## 7. 완제품 vs 반제품(세트) 구분

| 구분 | 캘린더 상품 | 근거 |

`확신도: 높음`

## 8. 핵심 메모

1. **계산방식 = 원자합산형**(calc-draft 절대 권위). 1차 가설 variant 고정가형 반증. 디지털인쇄 PRF_DGP_A/E 직계 동형.
2. **페이지수(장수) = 캘린더의 식별 차원** — 인쇄비 출력매수에 곱해짐(엽서=1장 vs 캘린더=4~16장). 4(8P)=4장 양면=8페이지. 설계 시 출력매수 산식에 페이지수 곱 누락 금지(돈크리티컬·인쇄비 4~16배 차이).
3. **제본비 = 수량구간형(라이브 COMP_BIND_CAL_* 실재)**, add-on 가공비(우드거치대·타공+끈) = ×제작수량 평면가산. calc-draft r95↔r98 문자 충돌은 라이브 적재(수량구간)가 결판.
4. **공정택일그룹**: 캘린더가공 택일 = 가격축, 但 하위 색선택(링칼라·삼각대컬러) = 가격무관 옵션. `t_prd_product_process_excl_groups` 라이브 부재(메모리 stale 정정).
5. **현 라이브 = WIRE 결함**(round-21 패턴): COMP_BIND_CAL_* 단가행 적재됐으나 공식·바인딩·인쇄용지 comp 배선 전무 → designer가 인쇄비·용지비·제본비를 조립한 PRF_CAL_* 신설 + 캘린더 5상품 바인딩.
6. **재사용 우선**(mint 최소화): COMP_PRINT_DIGITAL_S1·COMP_PAPER·COMP_BIND_CAL_* 재사용. 신규 mint = PRF_CAL_* 공식 + 캘린더가공 add-on comp(우드거치대·타공+끈)뿐.
