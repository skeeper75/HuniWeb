# stationery(문구) 시트 결함 처리 — 도메인 렌즈 라이브 규명 (round-3 remediation)

> **작업 목적(내재화):** 누군가 엑셀을 인쇄 도메인 이해 부족으로 DB에 적재해 **누락·오매칭·모호**가 생겼다.
> 본 문서는 그 결함을 *올바로 처리(수정)*하기 위해, **라이브 Railway DB 직접 read-only SELECT를 권위**로
> stationery(문구) 상품군을 종단 규명한다. 도메인 지식(07_domain KB)은 결함을 옳게 분류하는 렌즈다.
> 정제된 DB가 인쇄 자동견적 위젯의 기반이 된다.
>
> **권위·안전 원칙(HARD):** 라이브 SELECT = 진실(추출본 `ref-*.csv`·v2 2026-06-04은 stale 가능).
> 추정·임의판단 0 — 라이브·엑셀 L1·도메인 KB·PDF 실재 근거만. 미지/추정은 가설+컨펌 플래그.
> INSERT/UPDATE/DDL 0. DB 미적재 유지(본 문서는 처리 *방향* 설계). `db railway`·비표준포트·JOIN KEY=`prd_nm`.
>
> **작성** 2026-06-05 · 식별자/컬럼/코드/SQL 영어, 설명 한국어. **라이브 SELECT 본 시트 분담분 9회**(전부 read-only, 3시트 공통).

## ⚠️ 머리말 — 본 검증이 v2 대비 뒤집은 가장 큰 사실 (권위 반전)

**문구 시트의 핵심 결함은 v2 9속성 대시보드의 process MAJOR로 *집계만* 되었지, 상품별 치명도가 드러나지 않았다.**

라이브 직접 확인 결과(가장 큰 반전):

- **노트류 제본 공정이 라이브에서 전량 미적재.** 스프링노트·스프링수첩(트윈링제본), 메모패드(떡제본),
  중철노트(중철제본), 만년다이어리 4종(하드커버/소프트커버 제본)은 엑셀 `제본사양`에 명시돼 있으나
  라이브 `t_prd_product_processes`에는 **수축포장(PROC_000076) 2건만** 존재(다이어리 하드커버·스프링노트).
  → **제본은 책자·문구 도메인의 핵심 필수 공정**인데 0건 적재 = 위젯 옵션 캐스케이드에서 제본 선택 자체가 사라짐. **BLOCKER급.**
- **떡메모지(PRD_000097)는 묶음수(bundle_qty)·페이지룰·제본·excl_group이 *이미* 정상 적재**돼 있다(50장/100장 1권, 3/3 page, 떡제본 GRP-BOOK-제본).
  → 떡메모지는 round-3 작업 중 **레퍼런스로 손질된 단일 예시**로 보인다(reg/upd_dt 2026-06-05). 다른 노트류는 미손질 잔존.
  → digital-print의 프리미엄엽서와 동일 패턴: **"예시 1건만 처리, 나머지 잔존".** 무비판 복제 금지, 떡메모지 적재 정확성 검증 선행 후 패턴화.

**교훈(HARD 재확인):** v2 속성단위 집계(process MISSING 48)는 "어느 상품의 무슨 필수 공정이 빠졌는지"를 가리지 않는다.
라이브 상품별 규명으로만 "문구 전 노트류 제본 0건"이라는 치명 결함이 드러난다.

## ① stationery 상품군 — 도메인 이해 (라이브 권위)

**라이브 SELECT #1:** 문구 시트 11 prd_nm 전수 매칭.

메모(메모패드·떡메모지)·메모패드(내지커스텀 준비중)** = 11상품. **노트·다이어리류는 내지/표지 2축 + 제본**을 가진 booklet 유사 구조다.

## ④ 결함 표 — 누락(MISSING)·오매칭(MISMATCH)·모호(CONFIRM) (도메인 렌즈, 라이브 재확인)

> 분류: **MISSING**=정규화 규칙대로면 적재돼야 하나 라이브에 없음. **MISMATCH**=잘못 연결. **CONFIRM**=의미 미확정(실무 컨펌).
> 모든 판정 라이브 SELECT 재확인. v2 수치 stale → 본 표 권위.

### G-ST-1 [MISSING · BLOCKER] 노트·다이어리 제본 공정 전량 미적재 (R-PROC-6)

- **도메인 근거(R-PROC-6):** 엑셀 `제본사양` enum = 제본 공정(PROC_000017 family). 트윈링제본=PROC_000021,
  떡제본=PROC_000094 계열(또는 PROC_000017 자식), 중철제본=PROC_000018, 하드커버=PROC_000022 등.
  제본은 책자·문구의 **필수·단일(택일)** 공정(PROC_000017 note "필수,단일").
- **라이브 재확인(SELECT #2,4):** 문구 노트·다이어리 8상품 중 제본 공정 적재 = **떡메모지(097)뿐.** 나머지 0.

  떡메모지(097)를 레퍼런스 패턴으로 복제하되 적재 정확성 검증 선행. **단일 제본=택일그룹 단독이므로 excl_group 동반 여부 컨펌(§4 G-ST-3).**

### G-ST-2 [MISSING] 페이지룰 노트류 미적재 (R-PAGE)

- **도메인 근거:** 엑셀 `페이지사양_최소/최대/증가`는 내지 페이지 규칙 = `t_prd_product_page_rules`.
- **라이브 재확인:** page_rule 적재 = 먼슬리플래너(176, 28/28/0)·떡메모지(097, 3/3/3)뿐. v2 page_rule "11/11 완전정합"은
  **적재된 11행 내에선 정합**이나, **엑셀에 페이지값이 있는데 미적재된 상품을 분모에 안 넣은 착시**.
- **라이브 MISSING:** 스프링노트·수첩·메모패드·중철노트 등 내지 페이지가 있어야 하나 엑셀상 `페이지사양` 공란 → **엑셀 원천 부재**.
  → 노트류는 내지 페이지 가변 상품인데 엑셀에 min/max 미기재. **CONFIRM**(페이지 규칙 출처가 엑셀 외 어디인가).

### G-ST-3 [CONFIRM] 제본 택일그룹(excl_group) — 문구 단일제본 적용 여부

- **라이브:** 문구 11상품 중 excl_group = 떡메모지(097, GRP-BOOK-제본)뿐. 다른 노트·다이어리 0행.
- **도메인 해석:** 문구 각 상품은 제본이 **1종 고정**(스프링노트=트윈링만, 중철노트=중철만)이라
  책자처럼 "여러 제본 택일"이 아니다 → **excl_group 부재가 정상일 수 있음**(제본은 상품에 1개 고정 공정).
  단 떡메모지만 GRP-BOOK-제본에 든 것은 booklet 떡제본(094)과 묶인 결과로 보임(컨펌).
- **판정:** excl_group **CONFIRM** — 문구 제본이 단일 고정이면 G-ST-1은 excl_group 없이 process 단독 적재.

### G-ST-4 [CONFIRM→일부 MATCH] 표지 코팅 공정 / 자재 표기

- **도메인 근거:** 엑셀 `표지사양 = 아트250 + 무광코팅` = 자재(아트250) + **코팅 공정(무광=PROC_000015)** 복합.
- **라이브:** material에 `아트250 + 무광코팅`이 **자재 1행으로 통째 적재**(코팅을 분리 안 함). 코팅 공정(PROC_000015) 행은 별도 미적재.
- **판정:** 자재행 자체는 MATCH(표기 보존)이나, **코팅이 공정으로 분리 적재 안 됨 = 부분 MISSING(코팅 공정).** R-PROC-5 적용 시 무광코팅→PROC_000015 적재 대상. **CONFIRM**(자재명 안에 코팅 포함시킬지 vs 공정 분리).

### G-ST-5 [CONFIRM] bundle_qty(묶음수/권) — 떡메모지 외 적용 여부

- **라이브:** bundle_qty 적재 = 떡메모지(097, 50장1권·100장1권 = QTY_UNIT.03 권)뿐.
- **도메인:** 엑셀 떡메모지 `제본옵션 = 50장1권/100장1권`이 묶음수 원천. 다른 문구는 `제본옵션`에 묶음 표기 없음(실버링은 링색).
- **판정:** 떡메모지 bundle_qty = **MATCH(정상)**. 다른 문구 bundle 부재 = 정상(묶음 단위 없음). **단 메모패드도 떡제본인데 묶음수 무관한지 컨펌.**

### G-ST-6 [글로벌 갭, 문구 한정 아님] qty_unit_typ_cd 전 NULL

- **라이브:** `t_prd_products.qty_unit_typ_cd` = 문구 11 전부 NULL(272 전상품 NULL).
- **판정:** 문구 고유 결함 아님 → 글로벌 적재 갭(QTY_UNIT 코드 부여 정책 컨펌). digital-print G-DP-9와 동일.

## ⑦ 큰 그림 적용 (도메인 → 엑셀 → DB결함 → 처리 → 위젯)

- **도메인 KB:** 제본=필수·단일(PROC_000017 note), 내지/표지 2축(USAGE 코드)이 "엑셀 제본사양·종이/표지사양"을 옳게 해석하는 렌즈 → 원적재의 제본 누락을 결함으로 규명.
- **엑셀 L1(무손실):** 내지/표지 컬럼 분리 보존 덕에 자재 usage(내지/표지)·제본종류를 상품별 정확 추출.
- **DB 라이브 권위:** v2 속성집계(process MAJOR)가 가리지 못한 "문구 전 노트류 제본 0건"이 라이브 상품별 규명으로 드러남.
  material 표기차는 false EXTRA(정규화 MATCH) — 라이브로 확인.
- **처리방향:** 마스터 무변경 + 제본·코팅 공정 연결만 적재(FK 안전). 떡메모지를 레퍼런스 패턴화.
- **위젯 기반:** 제본(트윈링/떡/중철/하드커버)이 위젯 옵션 캐스케이드의 핵심 선택축. 제본 0건 잔존 시 위젯에서 제본 옵션이 통째로 사라짐 → R1이 위젯 선결 조건.
