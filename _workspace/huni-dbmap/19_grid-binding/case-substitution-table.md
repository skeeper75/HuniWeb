## 0. 치환이란 무엇인가 (이 문서의 작동)

round-13 BATCH는 **결함을 패턴축**(카테고리 고아·MAT_TYPE 오염·v03 진원)으로 묶었다. 그러나 "이 속성을 어느 그릇에 담나"의 **결정 원칙**은 케이스별로 남았다. round-15 §2가 `구성요소 11그릇 × 생산형태 3`의 배정 원칙을 세웠으므로, 각 BATCH를 그 셀에 대응시키면:

- **그릇 원칙으로 도출되는 BATCH** → "생산형태별 원칙 적용" 1줄로 치환(개별 컨펌 불요·원칙이 답).
- **그릇 축과 직교한 BATCH** → 원칙으로 안 닫힘(분류 무결성·식별코드·교정 방향·정체). **진짜 예외/별도 축**으로 격리.
- **그릇 원칙 vs 다른 바인딩(주로 ③가격)이 충돌하는 BATCH** → 원칙은 명확하나 **경계 쟁점**으로 표기(결정 필요분 압축).

→ 결과: 14 BATCH가 **{원칙 도출 N · 직교 축 N · 경계 쟁점 N}** 로 재정렬되어, 실무진 결정이 "그릇 원칙을 채택할 것인가" + "진짜 예외 몇 건"으로 수렴한다.

## 3. 생산형태별 원칙 재구성 (치환 결과 — 그릇이 어떻게 답하나)

🟢 원칙 도출 9건이 §2 셀로 어떻게 닫히는지 생산형태축으로 재정렬(개별 컨펌 → 생산형태 원칙):

### 완제품 (PRD_TYPE.01 — 낱장 인쇄물·굿즈)
- **색**(B2) = 본체색은 재질행 합성(2행)·도수는 `print_options`·자유색은 옵션. 자재 행 폭증 금지.
- **사이즈**(B6 경계) = 이산 옵션 or 면적매트릭스(siz). 폰기종/등급의 siz↔opt 경계는 OM-2 의존.
- **출력판형**(B10) = `plate_sizes`(절수=판형). 인쇄방식 폴더는 생산메타(견적밖).
- **가격**(B7) = 면적매트릭스형/고정가형/원자합산형(§2 #10).
- **후가공**(B3) = 코팅·커팅·박 = `processes`(스티커 가격모델만 경계).
- **묶음**(B8) = 떡제본=`bundle_qtys`(권), page_rule 부적용.

### 반제품 (PRD_TYPE.02 — 책자·포토북·노트)
- **소재**(B4·B9) = parent+`usage_cd`(표지.02/내지.01/면지.03). 레더=mat_typ .06 가죽. usage 분리는 반제품 한정.
- **제본**(B13) = `processes` PROC_000017 + GRP-BOOK-제본 택일그룹. 미싱제본 등 0행이면 그릇 내 신설.
- **page_rule**(B8) = 반제품 전용 축(완제품 부적용).
- **가격**(B7) = 고정가형(완제품가 포함).

### 기성상품 (PRD_TYPE.03 — 매입 부자재)
- **추가상품성**(B5) = 1 base SKU + host의 `t_prd_templates` 참조 + 사이즈 자동매칭. 별 PRD 폭증·이중등록 금지(OM-1).
- **색**(B2) = variant·자재행(볼체인 8색). 인쇄/공정 BOM 거의 없음.
- **가격**(B7) = 고정가(`t_prd_product_prices`).

> **핵심:** 9개 🟢 BATCH가 "생산형태별 원칙 적용" 단일 진술로 수렴 → round-13의 개별 ~70 컨펌 중 색·소재·후가공·가격·제본·추가상품·usage·묶음 관련분이 **생산형태 3 × 그릇 원칙**으로 치환됨(케이스별 재논의 불요).
