# dbm-column-domain — 컬럼 도메인 의미 + 상품 BOM 도출 방법론

상품마스터를 "셀 더미"에서 **두 개의 모호함 없는 도메인 산출물**로 바꾼다: ① 컬럼 도메인 사전(각 시트 각 컬럼의 의미) ② 상품 BOM 명세(각 상품의 자재+공정). 매핑은 그 아래 도메인 의미가 확정돼야 비로소 옳다 — 이 스킬은 그 의미를 확정한다. round-9 교훈("기계적 매핑 금지·스키마 설계의도 선행")의 엑셀 측 실현이다.

## 권위 순서 [HARD]

1. **후니 공식 PDF** — `docs/huni/후니프린팅_공정관리_시행초안_20260210.pdf`(17 Case 공정플로우·14 공정팀), `docs/huni/후니프린팅_주문프로세스_20251001.pdf`(인쇄타입·조판·파일명). 회사 운영 사실.
2. **라이브 DB / 07_domain KB** — `_workspace/huni-dbmap/07_domain/`(`entity-semantic-model.md` L3 의미축, `process-recipe-tree.md` L2 인쇄방식·제본·생산방식, `db-domain-structure-live.md`, `pdf-domain-knowledge.md`, `benchmark-competitors.md`). **의미축은 이미 여기 확정됨 — 재유도 금지, 인용 후 구체화.**
3. **국내외 인쇄 표준 (보조)** — WebSearch/WebFetch로 PDF/DB가 못 채운 갭만, 또는 물리/원리 보강. **표준이 후니 운영 사실과 충돌하면 후니가 권위**이고 충돌을 명시한다.

**추정 0.** 모든 컬럼 의미·BOM 요소는 실재 출처가 있다. 미지는 "가설 + 출처 + 컨펌질문"으로 적되 빈칸으로 두지 않는다.

## 절차

### 1. 시트 파싱 (dbm-excel-parse 재사용)
- `data_only=True`로 로드(이 워크북은 `read_only=True`에서 dims=None 반환 가능).
- **다단 헤더 복원:** 디지털인쇄처럼 R1=구분 그룹(사이즈/파일사양/종이/인쇄/별색인쇄/코팅/커팅/접지/제작수량) + R2=세부(판수/블리드/작업사이즈/…/화이트/클리어/핑크/금/은/최소/최대/증가)면 composite 헤더(`그룹 > 세부`)로 재구성한 뒤 컬럼을 정의한다.
- 병합셀·세로 캐스케이드(상품 1행 + 사이즈 여러 행) 구조를 식별한다.

### 2. 컬럼 도메인 사전 작성
각 컬럼에 대해:
- **의미축 식별** — 07_domain 9속성(size=재단치수·material=자재+usage슬롯·print_option=인쇄면도수·process=공정·process_excl_group=공정택일·plate=작업판형·bundle=묶음수·page=페이지룰·addon=추가상품) 중 어디인지 인용.
- **도메인 의미(정의)** — 그 컬럼이 인코딩하는 실제 인쇄 속성을 한 문장으로. (예: `판수` = 출력용지 1장에 작업물이 몇 판 들어가는가 = 임포지션 판걸이수, 가격·생산 단위.)
- **단위/포맷·값 도메인** — mm/장/판/Y·N/enum, 샘플값.
- **목표 DB 축** — 어느 t_* 속성으로 가야 하는가.
- **확정도** — ✅PDF/DB · 🟡PDF부분+표준 · 🔴미수록(가설).
- **함정** — 의미축 평면화·variant·복합표기 주의.

### 3. 의미축 평면화 함정 분리 (07_domain 횡단 교훈)
한 컬럼이 2개 의미축을 담으면 단일 축 평면화로 한 축이 소실된다. 적발 시 두 목표 축을 모두 명시:
- `아트250+무광코팅` = material(아트250) + process(무광코팅) → 단일 자재 평면화 금지.
- `배면양면/풀빼다/투명테두리` = UV 변형(process PROC_000002 변형) → print_side 오적재 금지.
- `화이트인쇄(단면)` = 별색 공정(PROC_000008) → 인쇄옵션 텍스트로 오해 금지.
- size 컬럼 속 형상 enum(원형 25~90mm) → size축에 형상 drop 금지.
- `*별도설정` = 자재 IMPORT 포인터(빈값 아님) → 제외 시 종이옵션 소멸.

### 4. 상품 BOM 명세 작성
각 상품명(또는 BOM 공유 클러스터)에 대해:
- **상품명 리서치** — 이름 자체의 도메인 의미(프리미엄엽서·투명클립보드·트윈링노트)를 07_domain 인쇄방식/제본 레시피로 해석.
- **인쇄방식** — 디지털/실사/UV/(옵셋/실크) 중 무엇인가(1상품=1인쇄방식, process-recipe-tree §1).
- **자재(BOM)** — 종이/소재/부속 + usage 슬롯(.01내지/.02표지/.03면지/.07공통), 두께·색상 variant 포함.
- **공정(순서 체인)** — 출력→(코팅)→재단→후가공→포장. 인쇄방식이 가능 공정 부분집합을 게이팅(process-recipe-tree §2). 선행→후행 의존(오시→접지, UV출력→레이저커팅→아크릴가공).
- **variant 분해** — 색상→material, 사이즈→size, 두께→material(별 mat_cd), 표지타입→반제품 sub_prd, 복합(색×사이즈)=2차원 분리.
- **근거·확정도** — 셀/PDF/표준 인용 + ✅🟡🔴.

> **자재+공정 둘 다.** 옵션=자재+공정 BUNDLE(사용자 HARD). BOM은 종이만이 아니라 공정도 포함한다. 순수공정(열재단·타공bare-hole)은 자재 없음, 무인쇄 재고는 공정 없음 — 있는 그대로.

### 5. 확정도 게이트 (애매모호 0)
- 모든 컬럼·BOM 요소가 ✅/🟡/🔴 중 하나로 표시됐는가(빈칸 0)?
- 🔴은 가설+출처+컨펌질문이 붙었는가?
- 07_domain이 이미 답한 컬럼은 "07_domain §X 인용"으로 닫혔는가(재리서치 0)?
- 의미축 2개를 담은 컬럼이 모두 분리됐는가?
