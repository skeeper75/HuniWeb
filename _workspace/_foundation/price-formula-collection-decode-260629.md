## 0. 공식집 ↔ 라이브 엔진 1:1 대응 (구조 모델)

라이브 `evaluate_price`(pricing.py)의 핵심 = **"공식은 항상 구성요소 합산, 구성요소는 선택값↔차원 자동매칭"**. 공식집의 `판매가 = 인쇄비 + 코팅비 + 용지비 + …` 가 정확히 이 모델이다.

| 공식집 표기 | 라이브 대응 | 비고 |
|---|---|---|
| `판매가 = A + B + C …` | 가격공식(PRF_*) = Σ 가격구성요소(COMP_*) | t_prc_formula_components 배선 |
| `[수량행]` | min_qty 구간(이상 하한) | TIER_DIMS min_qty |
| `[소재열]`/`[옵션열]`/`[제본종류열]`/`[도수열]` | mat_cd·opt_cd·proc_cd·print_opt_cd 등 비수량 차원(정확매칭) | use_dims 컬럼 |
| `출력매수 = 주문수량 / 판걸이수` | `comp_qty = ⌈주문수량 ÷ fn_calc_pansu(판형, 사이즈)⌉` | 판형기준 구성요소(use_dims에 plt_siz_cd) |
| `[세로mm행][가로mm열]` | siz_height·siz_width 구간(이하 상한)·면적매트릭스 | TIER_UPPER |
| `참조시트명`(예: 디지털인쇄비·코팅·출력소재…) | 그 구성요소의 component_prices 단가 출처 = 가격표(260527) 해당 시트 | 단가 verbatim |
| `단가형/합가형/고정금액` | prc_typ_cd .01/.02/.03 | .01=단가×수량 / .02=구간총액÷min_qty×수량 / .03=고정 |

라이브 상품구조(실측): 상품 = 기초축(사이즈·도수·판형·자재·공정) → 옵션그룹(손님 선택) → 제약규칙 → 추가상품. 셋트 = 부모 셋트공식 + 구성원(반제품, 내지=페이지파생 qty) 2레이어(evaluate_set_price).

## 4. 일반상품 vs 셋트상품

- **일반(단순)**: 단일 PRF_* = 구성요소 합산. 엽서·명함·스티커·실사·아크릴·문구·굿즈·보드/배너.
- **셋트(조립형)**: 공식집 63~92행 = 책자류·엽서북·떡메. 판매가에 `내지인쇄비/표지인쇄비/면지인쇄비/제본비`가 분리 등장 = 라이브 `evaluate_set_price`(부모 셋트공식 + 구성원 반제품, 내지=페이지파생 qty)와 정합. 책자 = 표지·내지(·면지) 구성원 + 제본 셋트공식.

## 5. ★확인필요·불일치 신호 (자동교정 금지·권위=공식집)

1. **하드커버링책자 면지 가격(81~89행)** — 공식집은 `면지인쇄비 + 면지코팅비 (각 ×2)`를 **판매가에 포함**. 그러나 이전사이트/라이브 관찰은 "면지=무료(블랙/화이트/그레이 동일·0원)". **공식집(정본)은 면지에 인쇄·코팅비 부과 / 거울(이전사이트)은 무료** → 어느 쪽이 현행인지 실무진·이전사이트 재확인.
2. **썬캡(48행)** — 공식 구조는 있으나 신규·미출시(가격 미정). 가격 확정 전 견적불가 정상.
3. **봉투제작(45행)** — `[수량행][소재열]` 표가 라이브에서 단가형(.01)↔합가형(.02) 오타이핑 위험(§26 무결성 BLOCKER 기보고).
4. **실사/현수막(100행)** — 면적격자 권위인데 라이브 포스터사인 셀 97% 미채번(§26 최대 갭).
5. **판걸이수** — `출력매수=주문수량/판걸이수`의 판걸이수가 라이브 fn_calc_pansu 기하계산이 권위 엑셀 판수보다 과다(저청구 systemic·C트랙).
