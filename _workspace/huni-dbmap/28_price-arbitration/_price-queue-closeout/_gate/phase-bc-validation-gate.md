## 게이트별 판정

### V1 — D-1b 그룹① comp 카운트 (13 확정) → **PASS**
- 라이브 실측: 후보 15 comp 전부 실재·전부 `prc_typ_cd='PRICE_TYPE.01'`·`comp_typ='.04'`.
- **CORNER_ROUND 결판**: 라이브 단가행 = 2000(1)·2000(100)·4000(300)·6000(500)·11000·21000·31000·41000·51000(5000) = **진성 누진 총액형**(9행). 가격표 `인쇄후가공!A1='모서리 (귀돌이, 합가)'` C열과 byte 일치 → **13에 포함 정당**.
- **"13→12" vs "13" 결판**: Phase A의 "13→12"는 CUT_PERF_1H6 빼기를 가리킨 표현이나 CUT_PERF는 큐 §1-4 IN절에 애초 부재 → **정정 대상은 13 그대로**(CREASE3+PERF3+VARTEXT3+VARIMG3+CORNER_ROUND1). Phase B §3-1이 정확히 13으로 채택. **→ Phase A를 13으로 통일 정정(반영 완료)**.
- CUT_PERF_1H6(23행 0.00)·CORNER_RIGHT(9행 0.00) → 가격표 "타공(단가)" 0원·"직각모서리" 0원과 정합. 제외 정당.

### V2 — base_code PRICE_TYPE.03 충돌 0 → **PASS**
- 라이브 `t_cod_base_codes` PRICE_TYPE 그룹 = .01(단가형)·.02(합가형)만. **.03 미점유** → 신규 INSERT 충돌 0·기존 .02 무회귀. ⓒ-2 채택 근거 실증.

### V3 — 033 mat_cd 귀속 + MATGROUP 동일가 → **PASS**
- 라이브 `t_mat_materials`: 074=백색모조지220·081=아트지250·082=아트지300·091=스노우지250·092=스노우지300 → Phase C 귀속 주장과 **정확 일치**.
