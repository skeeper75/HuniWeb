## G1. 🔴 일반 박 가공비 comp 라이브 부재 (7상품 박 가격 0)

- **현황**: 라이브에 일반 박 가공비 comp 없음. 명함박 전용(COMP_NAMECARD_FOIL_*) 4 + 동판셋업 2만 존재.
  - 확인: `t_prc_price_components.csv`에서 박/FOIL comp grep → NAMECARD 외 0건.
- **영향 상품 7** (박 process 등록됐으나 공식에 박 가공비 미포함 → 박 걸어도 0원):
  - 2단접지카드(27)·3단접지카드(29)·프리미엄명함(31)·펄명함(34)·프리미엄 쿠폰/상품권(42)·무선책자(69)·PUR책자(70)
- **designer 작업**: 대형/소형 × 일반/특수 박가공비 comp + 대형 동판비 comp mint, 7상품에 addon/공식확장으로 바인딩. 면적→등급 2단 lookup 살릴 것(명함박처럼 E등급 collapse 불가, 면적 가변).
- 우선도: **High**(돈크리티컬·박 무료 청구 = 매출 누락).

## G2. 🟡 박색상축 표현 미결 (일반박 vs 특수박)

- **현황**: 일반박/특수박은 같은 (등급×수량)이라도 단가 다름(예 대형 A등급 1000매 일반 75,000 vs 특수 95,000). 반드시 구분 필요.
- **미해결 질문**: 박색상을 무엇으로? → proc_cd(박 자식공정 034~049·라이브 존재) / comp 분리(명함박 선례 STD/HOLO) / print_opt_cd(도수용·오용우려) / opt_cd(addon 미작동 이력).
- **단서**:
  - 라이브 박 자식공정 18종 이미 정리됨(PROC_000033 아래) → proc_cd 안이 search-before-mint 충족.
  - 명함박은 comp 분리(S1_STD/S1_HOLO) 채택 → 동형 전파 일관성은 comp 분리 유리.
  - 일반/특수 그룹핑 매핑 필요: 대형 일반=금유광/금무광/은유광/은무광/동박/청박 · 대형 특수=먹유광/백박/홀로그램/트윙클/적박/녹박 (출처 B03/B04 제목). 소형은 펄박이 일반, 백박/홀로그램/트윙클이 특수 — ★대형과 색상 그룹이 다름(먹유광이 소형=일반, 대형=특수).
- **designer 작업**: 박색상축 방식 확정 + 일반/특수 그룹 매핑(대형/소형 별도). 우선도: **High**.

## G3. 🟡 면적→등급 2단 lookup 스키마 미정

- **현황**: 박가공비는 (가로×세로 → 등급 A~E) → (등급 × 수량 → 단가) 2단계. 라이브 component_prices 스키마에 등급 라벨 컬럼 없음.
- **미해결 질문**: 면적→등급 변환을 어디에 저장?
  - (가) dim_vals(JSON)에 등급 저장 + 면적 구간 → 등급 별도 룩업 테이블
  - (나) 면적(siz_width×siz_height) 단가행으로 직접 펼침(등급 중간단계 제거, 면적 격자 셀마다 단가 = 등급×수량 단가를 면적격자로 펼침 → 행 폭증)
  - (다) 앱 계산(면적→등급은 evaluate_price 코드에서 환산·MEMORY compute-in-app-db-stores-lookup 철학)
- **단서**: dbmap 철학 = "중간계산=앱·DB=룩업"(판수·박 등급 동일). 면적→등급은 앱 계산, DB는 등급별 단가만 → (다)가 기존 아키텍처 정합. 단 evaluate_price 코드 지원 여부 = 코드트랙 CONFIRM.
- **designer 작업**: 면적→등급 변환 위치 결정 + 등급격자 verbatim 적재(대형 9×8·소형 5×5 등급라벨). 우선도: **High**.

## G4. 🟠 소형/대형 경계 자동분기 불가 (CONFIRM)

- **현황**: 권위 시트에 "면적 ≤ X = 소형" 임계값 명시 없음. 소형 격자 최대 80×80(6,400㎟) ↔ 대형 최소 30×30(900㎟) → **면적 범위 겹침** → 단순 면적 임계 분기 불가.
- **미해결 질문**: 소형 vs 대형 시트 선택 기준은? 상품/박 크기 카테고리(명함·소형굿즈=소형 / 책자·접지카드=대형)로 사람이 고르나?
- **CONFIRM 큐**: 실무진에게 "소형/대형 박 구분 기준" 질의. 또는 상품별로 소형/대형 시트를 고정 지정(라이브 명함박=소형 차용).
- 우선도: **Medium**(상품별 시트 고정 지정으로 우회 가능).

## G5. 🟠 7상품 바인딩 방식 (addon vs 공식확장)

- **현황**: 박은 mand=N(선택). addon 가산 comp(opt_cd 키)는 라이브 미작동 이력(MEMORY addon-optcd-model-broken-live — opt_cd 미생성·본체 mat_cd 덮임).
- **미해결 질문**: 박가공비를 (가) t_prd_product_addons 템플릿(작동 입증된 모델) (나) 공식에 박 comp 추가(명함박처럼·단 명함박은 박이 mand) 중 무엇?
- **단서**: 박이 선택(optional)이므로 addon 템플릿이 자연스러움. 단 addon은 고정가/면적격자만 작동 입증(MEMORY: 155볼펜·159코스터). 면적×등급×수량×박색상 4축 addon은 미검증 → 코드트랙 CONFIRM.
- **designer 작업**: 박 선택의 표현(addon vs 공식 분기 comp) 결정 + evaluate_price/evaluate_set_price 지원 확인. 우선도: **High**.

## G6. 🟡 명함박 라이브 자체 결함 (참조모델 불완전)

- **현황**:
  - PRF_NAMECARD_FOIL에 STD comp만 배선(S1_STD+SETUP_S1_STD). **HOLO·양면(S2) 미배선** — note "HOLO/양면=CPQ 선결"(미완).
  - 라이브 FOIL_S1_STD 1000매=**63,000** vs 권위 소형 일반박 E등급 1000(M18)=**64,000** → **1,000원 차이**(CONFIRM: 오적재 또는 다른 등급 차용?).
- **designer 작업**: 7상품 그릇 설계 시 명함박 차용 규칙(E등급 verbatim)을 그대로 답습하지 말 것 — 1,000원 갭과 STD-only 미완은 검증 트랙(§13/§21)이 별도 판정. 박색상·양면 완전 배선을 7상품엔 처음부터 반영.
- 우선도: **Medium**(참조모델 신뢰도 주의 신호).
